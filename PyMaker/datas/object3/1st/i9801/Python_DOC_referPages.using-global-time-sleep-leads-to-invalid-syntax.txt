When ran, this returns invalid syntax pointing to "global time.sleep()":
Only identifier(s) can follow the global statement:global_stmt ::=  "global" identifier ("," identifier)*where identifiers are plain names such as:when you break that requirement and python isn't able to find an identifier after global, a SyntaxError is going to get raised.What you've entered after the global keyword isn't an identifier; it's a function call which is syntactically recognized by the () after the identifier.
