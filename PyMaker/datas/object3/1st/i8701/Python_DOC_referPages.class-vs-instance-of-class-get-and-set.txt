Q: why hh.a = 2 is not what I expect but b.a = 2 is Okay?here's the code:and...when i set hh.a = 2 it wont call the print('setting') but b.a = 2 is Okay. Can you explain this ?
Descriptor setters are only activated when setting the attribute on an instance of the class that owns the descriptor. They don't apply to the class itself or its subclasses. This is documented in the Python data model.3.3.2.1. Implementing Descriptors...object.__set__(self, instance, value)Called to set the attribute on an instance instance of the owner class to a new value, value.
