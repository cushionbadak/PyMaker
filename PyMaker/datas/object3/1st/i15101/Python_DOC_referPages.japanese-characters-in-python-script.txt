Im writing a python script that attach a word in Japanese to a common type url. My IDE (spyder) shows the Japanese characters perfectly, and so is the browser and my service, yet the language seems to have problems with it. For example, my service gives me the correct answer when Im inputting:http://example.com/translate?lang=ja&word=こんにちは Yet in my IDE:And that url is not recognised by my service.How can It be fixed?
Browser url-encode the query string for you. You need to do it manually in Python:In Python 2.x, using urllib.quote or urllib.quote_plus:In Python 3.x, using urllib.parse.quote or urllib.parse.quote_plus:Alternative using urllib.urlencode (Python 2.x)using urllib.parse.urlencode (Python 3.x)
