<div class="post-text" itemprop="text">
<blockquote>
<h2>Either replace all backslashes <code>\</code> with frontslashes <code>/</code> or place a <code>r</code> before your filepath string to avoid this error. <em>It is not a matter of your folder name being too long.</em></h2>
</blockquote>
<hr/>
<p>As Bohun Mielecki mentioned, the <code>\</code> character which is typically used to denote file structure in Windows has a different function when written within a string.</p>
<blockquote>
<p><strong>From Python3 Documentation:</strong> The backslash <code>\</code> character is used to escape characters that otherwise have a special meaning, such as newline, backslash itself, or the quote character.</p>
</blockquote>
<p>How this particularly affects your statement is that in the line</p>
<pre><code>student_data = pd.read_csv("C:\Users\xxxx\Desktop\student-intervention- 
system\student-data.csv")
</code></pre>
<p><code>\Users</code> matches the escape sequence <code>\Uxxxxxxxx</code> whereby <code>xxxxxxxx</code> refers to a <code>Character with 32-bit hex value xxxxxxxx</code>. Because of this, Python tries to find a 32-bit hex value. However as the <code>-sers</code> from <code>Users</code> doesn't match the <code>xxxxxxxx</code> format, you get the error:</p>
<blockquote>
<p>SyntaxError: (unicode error) 'unicodeescape' codec can't decode bytes in 
  position 2-3: truncated \UXXXXXXXX escape</p>
</blockquote>
<hr/>
<p>The reason why your code works now is that you have placed a <code>r</code> in front of <code>'C:\Users\xxxx\Desktop\project\student-data.csv'</code>. This tells python not to process the backslash character <code>/</code> as it usually does and read the whole string as-is. </p>
<p>I hope this helps you better understand your problem. If you need any more clarification, do let me know.</p>
<p>Source: <a href="https://docs.python.org/3/reference/lexical_analysis.html#string-and-bytes-literals" rel="nofollow noreferrer">Python 3 Documentation</a></p>
