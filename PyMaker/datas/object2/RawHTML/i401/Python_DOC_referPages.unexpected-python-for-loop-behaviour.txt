<div class="post-text" itemprop="text">
<p>The <code>_</code> variable is set in the Python interpreter, always holding the last non-<code>None</code> result of any expression statement that has been run.</p>
<p>From the <a href="https://docs.python.org/3/reference/lexical_analysis.html#reserved-classes-of-identifiers" rel="nofollow noreferrer"><em>Reserved Classifiers and Identifiers</em> reference</a>:</p>
<blockquote>
<p>The special identifier <code>_</code> is used in the interactive interpreter to store the result of the last evaluation; it is stored in the <code>builtins</code> module.</p>
</blockquote>
<p>and from <a href="https://docs.python.org/3/library/sys.html#sys.displayhook" rel="nofollow noreferrer"><code>sys.displayhook()</code></a>:</p>
<blockquote>
<p>If value is not <code>None</code>, this function prints <code>repr(value)</code> to <code>sys.stdout</code>, and saves value in <code>builtins._</code>. [...] <code>sys.displayhook</code> is called on the result of evaluating an <a href="https://docs.python.org/3/glossary.html#term-expression" rel="nofollow noreferrer">expression</a> entered in an interactive Python session.</p>
</blockquote>
<p>Here, that result was <code>9</code>, from an expression you must have run before the code you shared.</p>
<p>The <code>NameError</code> indicates you restarted the Python interpreter and did not yet run an expression statement yet that produced a non-<code>None</code> value:</p>
<pre><code>&gt;&gt;&gt; _
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
NameError: name '_' is not defined
&gt;&gt;&gt; 3 * 3
9
&gt;&gt;&gt; _
9
</code></pre>
