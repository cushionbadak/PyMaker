<p>As said by @Robin Zigmond in a comment, you are trying to <strong>call</strong> a list, as you would call a function, using the <code>(...)</code> syntax. The following:</p>
<pre><code>&gt;&gt;&gt; def f(x): return 2*x
... 
&gt;&gt;&gt; f(2)
4
</code></pre>
<p>Is different from:</p>
<pre><code>&gt;&gt;&gt; L=[1,2,3]
&gt;&gt;&gt; L(2)
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'list' object is not callable
</code></pre>
<p>The latter doesn't work because <code>[1,2,3]</code> is not callable. <a href="https://docs.python.org/3/reference/datamodel.html#the-standard-type-hierarchy" rel="nofollow noreferrer">The Python documentation</a> enumerates callable types:</p>
<ul>
<li>User-defined functions</li>
<li>Instance methods</li>
<li>Generator functions</li>
<li>Coroutine functions</li>
<li>Asynchronous generator functions</li>
<li>Built-in functions</li>
<li>Built-in methods</li>
<li>Classes</li>
<li>Class Instances: <em>Instances of arbitrary classes can be made callable by defining a <code>__call__()</code> method in their class.</em></li>
</ul>
<p>A list (ie a <code>list</code> instance) is none of them, since the <code>list</code> class does not have a <code>__call__() method</code>:</p>
<pre><code>&gt;&gt;&gt; dir(list)
['__add__', '__class__', '__contains__', '__delattr__', '__delitem__', '__dir__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__getitem__', '__gt__', '__hash__', '__iadd__', '__imul__', '__init__', '__init_subclass__', '__iter__', '__le__', '__len__', '__lt__', '__mul__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__reversed__', '__rmul__', '__setattr__', '__setitem__', '__sizeof__', '__str__', '__subclasshook__', 'append', 'clear', 'copy', 'count', 'extend', 'index', 'insert', 'pop', 'remove', 'reverse', 'sort']
</code></pre>
<p>In your example, the first lines states that <code>plottest</code> is an iterable. The error shows it is a list. And you try to call it with <code>index = plottest(doc)</code>. My guess is that you want to get the index of <code>doc</code> in <code>plottest</code>. To achieve this in Python, you can write:</p>
<pre><code>for index, doc in enumerate(plottest):
    ...
</code></pre>
