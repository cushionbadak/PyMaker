<p><strong>Caveat</strong> be aware of ordering rules for <code>values</code> method for dictionaries. </p>
<p>Ordering rules <a href="https://docs.python.org/2/library/stdtypes.html#dict.items" rel="nofollow noreferrer">2.x documentation</a> and <a href="https://docs.python.org/3/library/stdtypes.html#dictionary-view-objects" rel="nofollow noreferrer">3.x documentation</a>.</p>
<p>EDIT2:</p>
<p>To prevent any weirdness with dictionary ordering and the provided solution, wrap your data into an <a href="https://docs.python.org/2/library/collections.html#collections.OrderedDict" rel="nofollow noreferrer">OrderedDict</a>:</p>
<pre><code>from collections import OrderedDict

x = {'ID':[OrderedDict({
         'key-1': 'First Name',
         'key-2': 'John'
         }),OrderedDict({
         'key-1': 'Last Name',
         'key-2': 'Doe'
         }),OrderedDict({
         'key-1': 'Age',
         'key-2': '25'
        })]}
</code></pre>
<p><code>dict()</code> is a nice option for something like this:</p>
<pre><code>d = dict(each.values() for each in x['ID'])
</code></pre>
<p>output:</p>
<pre><code>{'Age': '25', 'First Name': 'John', 'Last Name': 'Doe'}
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Try a dict comprehension and build a new dictionary with <code>key-1 as the  key and</code>key-2` as the value.</p>
<pre><code>x = {'ID':[{
    'key-1': 'First Name',
    'key-2': 'John'
},{
    'key-1': 'Last Name',
    'key-2': 'Doe'
},{
    'key-1': 'Age',
    'key-2': '25'
}]}


print({el['key-1']: el['key-2'] for el in x['ID']})
</code></pre>
<p>Result</p>
<blockquote>
<p>{'Age': '25', 'First Name': 'John', 'Last Name': 'Doe'}</p>
</blockquote>
