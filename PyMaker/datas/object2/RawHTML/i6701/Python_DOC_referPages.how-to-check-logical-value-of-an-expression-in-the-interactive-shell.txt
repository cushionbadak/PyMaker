<div class="post-text" itemprop="text">
<blockquote>
<p>Is there a shorter way for checking the logical value of an
  expression?</p>
</blockquote>
<p>Yes, it's the <a href="https://docs.python.org/3/library/functions.html#bool" rel="noreferrer"><code>bool()</code></a> function:</p>
<pre><code>&gt;&gt;&gt; string = None
&gt;&gt;&gt; bool(string)
False
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>All expressions <strong>except those that produce <code>None</code></strong> are echoed.</p>
<p>If you execute an expression that doesn't result in data being echoed, then you probably have a <code>None</code> result. You can test for <code>None</code> explicitly:</p>
<pre><code>&gt;&gt;&gt; string = None
&gt;&gt;&gt; string is None
True
</code></pre>
<p>Using <code>is None</code> produces a non-<code>None</code> result.</p>
<p>If you just wanted to test for the <em>boolean</em> value of an expression, use the <a href="https://docs.python.org/3/library/functions.html#bool" rel="nofollow noreferrer"><code>bool()</code> function</a>; it'll return a result following the standard <a href="https://docs.python.org/3/library/stdtypes.html#truth-value-testing" rel="nofollow noreferrer">truth value testing rules</a>:</p>
<pre><code>&gt;&gt;&gt; string = None
&gt;&gt;&gt; bool(string)
False
</code></pre>
<p>Again, this is guaranteed to not be <code>None</code> and echoed. You can't distinguish between <code>None</code> and other false-y values this way, however. An empty string <code>''</code> will also result in <code>False</code>, for example.</p>
<p>Another alternative is to explicitly print all expression results, using the <a href="https://docs.python.org/3/library/functions.html#repr" rel="nofollow noreferrer"><code>repr()</code> function</a>:</p>
<pre><code>&gt;&gt;&gt; print(repr(string))
None
</code></pre>
<p>This then produces the exact same output as echoing does, with the single exception that <code>None</code> is printed too.</p>
</div>
<div class="post-text" itemprop="text">
<p>Yes, create a boolean directly from the expression, which will always be <code>True</code> or <code>False</code>:</p>
<pre><code>&gt;&gt;&gt; bool(string)
False
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>string is None</code> or <code>string is not None</code> will return the boolean you need.</p>
<pre><code>&gt;&gt;&gt; string = None
&gt;&gt;&gt; string is None
&gt;&gt;&gt; True
</code></pre>
</div>
