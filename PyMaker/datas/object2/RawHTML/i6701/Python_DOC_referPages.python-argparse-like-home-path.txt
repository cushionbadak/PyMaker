<p>I have a question, where I can't find an answer for. This is my codeblock:</p>
<pre><code>import argparse

parser = argparse.ArgumentParser()
parser.add_argument('-t', '--test', help='just for testing')
parser.add_argument('--home-path', help='specify the homepath')
args = parser.parser_args()
if args.test:
        print('Test')
</code></pre>
<p>How can I access the '--home-path' argument like the test argument?</p>
</div>
<div class="post-text" itemprop="text">
<p>Just replace the <code>-</code> with <code>_</code> in the attribute name:</p>
<pre><code>args = parser.parse_args()
if args.home_path:
    # Do stuff
</code></pre>
<p>This behavior is documented under <a href="https://docs.python.org/3/library/argparse.html#dest" rel="nofollow noreferrer"><code>dest</code></a> in <a href="https://docs.python.org/3/library/argparse.html#the-add-argument-method" rel="nofollow noreferrer"><code>ArgumentParser.add_argument()</code></a>:</p>
<blockquote>
<p>For optional argument actions, the value of <code>dest</code> is normally inferred
  from the option strings. <a href="https://docs.python.org/3/library/argparse.html#argparse.ArgumentParser" rel="nofollow noreferrer"><code>ArgumentParser</code></a> generates the value of <code>dest</code> by
  taking the first long option string and stripping away the initial <code>--</code>
  string. If no long option strings were supplied, <code>dest</code> will be derived
  from the first short option string by stripping the initial <code>-</code>
  character. Any internal <code>-</code> characters will be converted to <code>_</code> characters
  to make sure the string is a valid attribute name.</p>
</blockquote>
