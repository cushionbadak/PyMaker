<p>I want the value of id 1400 how to get it ?    </p>
<pre><code> x = {'type': 'Asset', 'id': 1400}
</code></pre>
<p>OUTPUT SHOULD BE </p>
<pre><code>1400
</code></pre>

<div class="post-text" itemprop="text">
<p><code>x</code> in your example is a <a href="https://docs.python.org/3/tutorial/datastructures.html#dictionaries" rel="nofollow noreferrer">dictionary</a>. In python, you access dictionary items via their key, which in this case is <code>"id"</code>. </p>
<pre><code>output = x["id"]
</code></pre>
<p>If you attempt to use a key which does not exist in the dictionary, you will get a <code>KeyError</code>. If you are unsure if the key you will be using is in the dictionary, you can either use a <code>try-except</code>:</p>
<pre><code>try:
    output = x["id"]
except KeyError:
    #Handle error
</code></pre>
<p>Or, you can use <code>.get()</code> to provide a default value. If the key you use does not exist in the dictionary, the default value is returned and no exception is raised.</p>
<pre><code>output = x.get("id", default_value)
</code></pre>
<p>If you omit <code>default_value</code>, then <code>None</code> is returned if the key does not exist.</p>
</div>
<div class="post-text" itemprop="text">
<p>The simplest way is using square bracket notation. But be careful if the element is not found it will throw and exception.</p>
<pre><code>x['id']
</code></pre>
<p>A more flexible way would be with the <code>.get()</code> method.</p>
<pre><code>x.get('id')
</code></pre>
<p>The difference is that with <code>.get()</code> you can pass a default value.</p>
<pre><code>x.get('id', 0)
</code></pre>
<p>And you can do some more complex navigation. For example:</p>
<pre><code>x.get('y', {}).get('z', None)
</code></pre>
<p>This would try to get <code>y</code>. If it can't find it it will return an empty dict from which it will try to get <code>z</code>. If it can't find it, it will return <code>None</code> without throwing an exception.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can also do, x.get("id"). In this case, you wont get KeyError Exception in case id isn't in the dict.</p>
