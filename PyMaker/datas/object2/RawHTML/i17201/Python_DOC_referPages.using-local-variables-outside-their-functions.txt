<p>However, I am not sure how to return these variables as they are local(?). Therefore, I can not use these variables outside the function. I am just wondering if someone could help edit this function in a way where I could use the drawnCard and drawnCardValue variables outside the function?</p>

<div class="post-text" itemprop="text">
<p><a href="http://docs.python.org/3/reference/simple_stmts.html#the-return-statement" rel="nofollow">Use <code>return</code></a>:</p>
<pre><code>def drawCard():
    drawnCard = random.choice(cardDeck)
    adPos = cardDeck.index(drawnCard)
    drawnCardValue = cardValues[adPos]
    return drawnCard, drawnCardValue

drawnCard, drawnCardValue = drawnCard()
</code></pre>
<p>Note, you could also write <code>drawCard</code> this way:</p>
<pre><code>def drawCard():
    adPos = random.randrange(len(cardDeck))
    drawnCard = cardDeck[adPos]
    drawnCardValue = cardValues[adPos]
    return drawnCard, drawnCardValue
</code></pre>
<p>These two functions behave differently if <code>cardDeck</code> contains duplicates, however. <code>cardDeck.index</code> would always return the <em>first</em> index, so <code>drawnCardValue</code> would always correspond to the first item which is a duplicate. It would never return the second value (which in theory could be different.)</p>
<p>If you use <code>adPos = random.randrange(len(cardDeck))</code> then every item in <code>cardValue</code> has an equal chance of being selected -- assuming <code>len(cardValue) == len(cardDeck)</code>.</p>
