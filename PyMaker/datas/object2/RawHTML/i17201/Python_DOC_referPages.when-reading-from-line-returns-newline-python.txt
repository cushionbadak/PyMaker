<p>I am trying to write a python script to generate some code by reading from a file. However it does not work as expected. </p>
<p>Code: </p>
<pre><code>with open('continent.txt') as fp:
    for line in fp:
        print "text ", line ," here"
</code></pre>
<p>Results</p>
<p>$ python ContinentCode.py</p>
<pre><code>text  North America
here
text  South America
here
text  Central America
</code></pre>
<p>Desired</p>
<pre><code>text  North America here
text  South America here
text  Central America here
</code></pre>

<div class="post-text" itemprop="text">
<p>Call the <a href="http://docs.python.org/3/library/stdtypes.html?highlight=str.strip#str.strip" rel="nofollow"><code>.strip()</code></a> method on <code>line</code>.</p>
<pre><code>with open('continent.txt') as fp:
    for line in fp:
        print "text ", line.strip('\r\n') ," here"
</code></pre>
<p>Note the argument to strip specifies what characters to remove. By default <code>.strip()</code> removes spaces and tabs as well... not sure if this is what you want.</p>
</div>

<div class="post-text" itemprop="text">
<p>Your question seems equivalent to one asked <a href="https://stackoverflow.com/questions/275018/how-can-i-remove-chomp-a-newline-in-python">here</a>.</p>
<p>Summarizing the above thread, you can use <code>print "text ", line.rstrip("\n"), " here"</code>. line.rstrip("\n") will return remove any "\n" contained in the string <code>line</code>.</p>
<p>Or you could use <code>for line in fp.splitlines()</code>. .splitlines() returns a list of the lines WITHOUT newline characters. Personally, I'm a fan of splitlines. If you want, you can do something like
<code>lines = fp.splitlines()
for line in lines:
    [code goes here]</code></p>
