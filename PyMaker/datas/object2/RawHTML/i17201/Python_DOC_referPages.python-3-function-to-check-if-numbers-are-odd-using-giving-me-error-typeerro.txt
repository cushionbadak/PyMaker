<div class="post-text" itemprop="text">
<p><a href="http://docs.python.org/3/library/functions.html#input" rel="nofollow"><code>input</code></a> returns a string in Python 3.x.  Meaning, you need to explicitly convert the input to an integer:</p>
<pre><code>num = int(input("num &gt; "))
</code></pre>
<p>The second example does not throw an error because multiplying strings by integers is perfectly legal:</p>
<pre><code>&gt;&gt;&gt; 'a' * 2
'aa'
&gt;&gt;&gt;
</code></pre>
<hr/>
<p>Also, you need to do <code>num % 2</code> to check if <code>num</code> is odd:</p>
<pre><code>&gt;&gt;&gt; 3 % 2
1
&gt;&gt;&gt; 4 % 2
0
&gt;&gt;&gt;
</code></pre>
<p><code>num % 1</code> will always return <code>0</code>, regardless of the value of <code>num</code>:</p>
<pre><code>&gt;&gt;&gt; 3 % 1
0
&gt;&gt;&gt; 4 % 1
0
&gt;&gt;&gt;
</code></pre>
<hr/>
<p>Finally, <code>x = num % 2</code> should be placed <em>inside</em> the function:</p>
<pre><code>num = int(input("num &gt; "))
def OddCheck(num):
    ############
    x = num % 2
    ############
    if x == 1:
        print('num',num,'is odd')
    else:
        print(num,'is not odd')

print(OddCheck(num))
</code></pre>
<p>Otherwise, you will not be checking the argument <code>num</code> passed into the function.</p>
</div>
<span class="comment-copy"><code>input</code> returns a string. Using the <code>%</code> operator on it tries to do string formatting. Your OddCheck function also doesn't actually check if its input is odd, it checks the value of a global variable. This is probably going to get you a poor grade...</span>
<span class="comment-copy">Incidentally, any number <code>% 1</code> is always 0.  You mean to be doing <code>% 2</code>.  It also looks like you meant for the <code>x = num % 1</code> line to be inside the function, not outside it.</span>
