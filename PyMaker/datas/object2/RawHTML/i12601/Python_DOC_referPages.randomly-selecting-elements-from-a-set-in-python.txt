<p>If I have a set like so:</p>
<pre><code>s = set(range(1,100))
</code></pre>
<p>How might I randomly generate a subset of <code>s</code> containing <code>k</code> distinct elements? </p>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/2/library/random.html#random.sample" rel="nofollow"><code>random.sample</code></a>:</p>
<pre><code>import random
random.sample(s, k)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/3/library/random.html?highlight=random#random.sample" rel="nofollow"><code>random</code></a> module:</p>
<pre><code>&gt;&gt;&gt; random.sample(s,  10) 
[14, 43, 42, 18, 80, 63, 15, 59, 49, 57]
</code></pre>
<p>There is no need for you to <code>s=set(range(100))</code> as <code>range</code> will provide a list of numbers in ascending order from 0 to 99.So, all of them are unique.</p>
<p>Just feed that <code>range(1,100)</code> to the <code>random.sample</code> method:</p>
<pre><code>&gt;&gt;&gt; random.sample(range(1,100), k)
</code></pre>
<p>Quoting from <a href="https://docs.python.org/3/library/random.html?highlight=random#random.sample" rel="nofollow">Python Docs</a>:</p>
<blockquote>
<p><code>random.sample(population, k)</code> Return a k length list of <strong>unique elements</strong>
  chosen from the population sequence or set. Used for random sampling
  without replacement.</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/3/library/random.html#random.sample" rel="nofollow"><code>sample</code></a> from the <code>random</code> module.</p>
<pre><code>import random
items = set(range(1, 100))
selection = set(random.sample(items, 10))
</code></pre>
</div>
<span class="comment-copy">Note, if you can manage it, avoid passing <code>set</code> to <code>random.sample</code>.  If I'm reading the source properly, it executes <code>random.sample(tuple(s), k)</code> -- at least for sets with length greater than 21.</span>
<span class="comment-copy">Good point. There is no need to call <code>set()</code> on the <code>range</code> object.</span>
