<div class="post-text" itemprop="text">
<p>Yes, that means that the default value for encoding is 'utf-8'. </p>
<p>Also you could check this on: <a href="https://docs.python.org/3/library/stdtypes.html#str.encode" rel="nofollow">https://docs.python.org/3/library/stdtypes.html#str.encode</a></p>

<div class="post-text" itemprop="text">
<p>It is an additional initializer for <code>str</code> objects that <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow">is used when an object of type <code>bytes</code> (or <code>byte-like</code>) is supplied</a> (indicated by the prefix <code>b'text'</code>.</p>
<p>It's default value is <code>utf-8</code> because the default encoding for Python 3.5 is <code>utf-8</code> (and it rocks).</p>
<p>It simply means that if a <code>bytes</code> object is passed in, the encoding <code>utf-8</code> is going to be used <strong>by default</strong> and the <code>errors=strict</code> means that encoding errors will raise a UnicodeError. It essentially issues a call to <a href="https://docs.python.org/3/library/stdtypes.html#bytes.decode" rel="nofollow"><code>bytes.decode()</code></a>.</p>
<p>You can of course change these if you wish and use <a href="https://docs.python.org/3.5/library/codecs.html#standard-encodings" rel="nofollow">another encoding</a> (like <code>latin-1</code>) <a href="https://docs.python.org/3/library/codecs.html#codecs.register_error" rel="nofollow">or another level of errors</a>. </p>
