<p><code>uniform(a,b)</code> returns values in the range [a,b). How to turn this interval into (a,b)?</p>
</div>
<div class="post-text" itemprop="text">
<p>This seems simple enough:</p>
<pre><code>def my_uniform(a, b):
    while True:
        result = uniform(a, b)
        if a &lt; result &lt; b:
            return result
</code></pre>

<span class="comment-copy">Note that <code>uniform(a, b)</code> can generate the upper value <code>b</code> too, in many cases.</span>
<span class="comment-copy">It's either <code>[a,b)</code> or <code>(a,b)</code>. <a href="https://docs.python.org/3/library/random.html?highlight=random.uniform#random.uniform" rel="nofollow noreferrer">"The end-point value b may or may not be included in the range depending on floating-point rounding in the equation a + (b-a) * random()."</a></span>
<span class="comment-copy">@Phillip: The sentence you quote is correct: <code>b</code> may be included. For an extreme example, try <code>a = 1.0</code>, <code>b = 1.0 + 2**-52</code>. You'll see that <code>uniform(a, b)</code> produces <code>a</code> half the time and <code>b</code> half the time. For a less extreme example, <code>uniform(1.0, 1.25)</code> can theoretically generate <code>1.25</code>.</span>
