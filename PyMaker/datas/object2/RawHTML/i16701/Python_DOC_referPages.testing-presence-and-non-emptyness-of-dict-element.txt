<p>Given a dictionary dict, is there a more elegant way to do this test?</p>
<pre><code>if 'key' in dict and dict['key']:
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/stdtypes.html#dict.get" rel="nofollow"><code>dict.get</code></a>:</p>
<pre><code>if dict.get('key'):
</code></pre>
<p>If the key is not found, the method will return <code>None</code> (which evaluates to <code>False</code>).  Otherwise, it will return the value associated with the key.</p>
<p>See a demonstration below:</p>
<pre><code>&gt;&gt;&gt; dct = {'a':0, 'b':1}
&gt;&gt;&gt;
&gt;&gt;&gt; dct.get('a')
0
&gt;&gt;&gt; dct.get('b')
1
&gt;&gt;&gt; dct.get('c')  # Returns None with non-existent keys
&gt;&gt;&gt;
&gt;&gt;&gt; bool(dct.get('a'))  # 0 evaluates to False
False
&gt;&gt;&gt; bool(dct.get('b'))  # 1 evaluates to True
True
&gt;&gt;&gt; bool(dct.get('c'))  # The None returned by dict.get evaluates to False
False
&gt;&gt;&gt;
</code></pre>
<p>Note that you can also specify a default return value:</p>
<pre><code>&gt;&gt;&gt; dct.get('c', 'not found')
'not found'
&gt;&gt;&gt;
</code></pre>
</div>

<span class="comment-copy">@Laizer - If the key is not found, <code>dict.get</code> will return <code>None</code>, which the if-statement will then evaluate as <code>False</code>.  If the key is found, <code>dict.get</code> will return the value associated with it, which the if-statement will then evaluate.  If the value is falsey (<code>False</code>, <code>None</code>, <code>0</code>, empty, etc.), the if-statement will evaluate it as <code>False</code>.  Otherwise, it will evaluate it as <code>True</code>.  So, the code I posted is just a shorter way of doing what you have already.</span>
