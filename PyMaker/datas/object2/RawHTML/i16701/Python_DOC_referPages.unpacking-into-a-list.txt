<p>Is there any difference in Python between unpacking into a tuple:</p>
<pre><code>x, y, z = v
</code></pre>
<p>and unpacking into a list?</p>
<pre><code>[x, y, z] = v
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Absolutely nothing, even down to the bytecode (using <a href="https://docs.python.org/2/library/dis.html"><code>dis</code></a>):</p>
<pre><code>&gt;&gt;&gt; def list_assign(args):
    [x, y, z] = args
    return x, y, z

&gt;&gt;&gt; def tuple_assign(args):
    x, y, z = args
    return x, y, z

&gt;&gt;&gt; import dis
&gt;&gt;&gt; dis.dis(list_assign)
  2           0 LOAD_FAST                0 (args) 
              3 UNPACK_SEQUENCE          3 
              6 STORE_FAST               1 (x) 
              9 STORE_FAST               2 (y) 
             12 STORE_FAST               3 (z) 

  3          15 LOAD_FAST                1 (x) 
             18 LOAD_FAST                2 (y) 
             21 LOAD_FAST                3 (z) 
             24 BUILD_TUPLE              3 
             27 RETURN_VALUE         
&gt;&gt;&gt; dis.dis(tuple_assign)
  2           0 LOAD_FAST                0 (args) 
              3 UNPACK_SEQUENCE          3 
              6 STORE_FAST               1 (x) 
              9 STORE_FAST               2 (y) 
             12 STORE_FAST               3 (z) 

  3          15 LOAD_FAST                1 (x) 
             18 LOAD_FAST                2 (y)

</code></pre>

<div class="post-text" itemprop="text">
<p>No. In fact the <code>x, y, z = v</code> is a shorthand for:</p>
<pre><code>(x, y, z) = v
</code></pre>
<p>... which is unpacking in a tuple. The same behavior happens:</p>
<pre><code>&gt;&gt;&gt; v = (1, 3, 4)
&gt;&gt;&gt; [x, y, z] = v
&gt;&gt;&gt; x, y, z
(1, 3, 4)
&gt;&gt;&gt; x, y, z = v
&gt;&gt;&gt; x, y, z
(1, 3, 4)
</code></pre>
</div>
