<p>I'm working on gathering up a list of image sequences from multiple directories.  I'm using a TCL expression within Nuke (compositing package) to generate a sting that lists the contents of each directory.  The returned string looks like so:</p>
<pre><code>Draft {Test_02_v002_Global_alp_%04d.png 1-12} {Test_02_v002_Global_col_%04d.png 1-12} Thumbs.db
</code></pre>
<p>The Draft and Thumbs.db are unwanted parts of the string (non-image files within the directory).  What I want to get is this:</p>
<pre><code>Test_02_v002_Global_alp_%04d.png 1-12 

Test_02_v002_Global_col_%04d.png 1-12
</code></pre>
<p>I am able to do this if there is only one image sequence in the directory using the following code:</p>
<pre><code>start = "{"
end = "}"
re.search('%s(.*)%s' % (start, end), string).group(1) 
</code></pre>
<p>But I can't figure out how to get this to work when more than one image sequence exists.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/re.html#re.findall" rel="nofollow"><code>re.findall</code></a>:</p>
<pre><code>&gt;&gt;&gt; from re import findall
&gt;&gt;&gt; mystr = "Draft {Test_02_v002_Global_alp_%04d.png 1-12} {Test_02_v002_Global_col_%04d.png 1-12} Thumbs.db"
&gt;&gt;&gt; findall("{(.*?)}", mystr)
['Test_02_v002_Global_alp_%04d.png 1-12', 'Test_02_v002_Global_col_%04d.png 1-12']
&gt;&gt;&gt;
</code></pre>
