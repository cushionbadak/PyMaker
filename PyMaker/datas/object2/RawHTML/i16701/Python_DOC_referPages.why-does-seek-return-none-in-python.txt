<div class="post-text" itemprop="text">
<p>You are reading the <em>wrong documentation</em>. You need to look at <a href="https://docs.python.org/2/library/stdtypes.html#file.seek" rel="noreferrer"><code>file.seek()</code></a> when using Python 2:</p>
<blockquote>
<p>There is no return value.</p>
</blockquote>
<p>Using <a href="https://docs.python.org/2/library/io.html#io.open" rel="noreferrer"><code>io.open()</code></a> is fine, and if you do, you'll get a <em>different</em> object, whose <code>seek()</code> method <em>does</em> return the current position:</p>
<pre><code>Python 2.7.6 (default, Apr 28 2014, 17:17:35) 
[GCC 4.2.1 Compatible Apple LLVM 5.1 (clang-503.0.40)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
&gt;&gt;&gt; import io
&gt;&gt;&gt; f = io.open('data.json')
&gt;&gt;&gt; f.seek(0, 2)
39L
&gt;&gt;&gt; type(f)
&lt;type '_io.TextIOWrapper'&gt;
&gt;&gt;&gt; f = open('data.json')
&gt;&gt;&gt; f.seek(0, 2)
&gt;&gt;&gt; type(f)
&lt;type 'file'&gt;
</code></pre>
<p>The <a href="https://docs.python.org/2/library/io.html" rel="noreferrer"><code>io</code> module</a> is the new I/O architecture for Python 3, available in Python 2 as well. The Python 3 <a href="https://docs.python.org/3/library/functions.html#open" rel="noreferrer">built-in <code>open()</code> function</a> is an alias for <code>io.open()</code>, but not yet so in Python 2.</p>
</div>
<div class="post-text" itemprop="text">
<p>Following up on Martjin's answer, use <code>type()</code> to inspect the variable's type:</p>
<pre><code>Python 2.7.5 (default, Mar  9 2014, 22:15:05) 
[GCC 4.2.1 Compatible Apple LLVM 5.0 (clang-500.0.68)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
&gt;&gt;&gt; f = open("foo", "r")
&gt;&gt;&gt; type(f)
&lt;type 'file'&gt;
</code></pre>
<p>By inspecting the object type, you will see that the variable <code>f</code> is not a member of <code>io</code>, but of <code>file</code>, and so the documentation to look up would be different.</p>
