<p>Whats wrong with <a href="https://docs.python.org/3.6/tutorial/introduction.html#lists" rel="nofollow noreferrer">list slicing</a>?</p>
<pre><code>def stringRange(chars, x):
    return chars[:x]
</code></pre>
<p>Whats going on here is we are treating the string as a list of characters, and we are telling python to give us the first <code>x</code> elements of the list (first <code>x</code> characters of string). Keep in mind, list slicing follows the syntax <code>list[start:end]</code> (in our case, <code>start</code> is implied to be 0). This means that, since lists start at 0, list slicing really returns a list of all elements with index from 0 (inclusive) to <code>x</code> (non inclusive).</p>
<p>For a better, more detailed explanation, see <a href="https://stackoverflow.com/a/509295/7662085">this</a> great answer by <a href="https://stackoverflow.com/users/893/greg-hewgill">Greg Hewgill</a> on how list slices work.</p>
