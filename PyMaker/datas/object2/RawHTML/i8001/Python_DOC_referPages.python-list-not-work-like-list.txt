<div class="post-text" itemprop="text">
<p>your data is a <a href="https://docs.python.org/3/library/json.html" rel="nofollow noreferrer">json</a> encoded string. you can decode it to a python list like this:</p>
<pre><code>import json

strg = '''[["admin", "123", "2017-May-21 01:43:37"], ["nimal", "456", "2017-Apr-10 21:24:48"], ["kamal", "8745", "2017-Apr-15 21:25:46"]]'''

data = json.loads(strg)

print(data[0])
# ['admin', '123', '2017-May-21 01:43:37']
</code></pre>
<p>or you can read directly from the file:</p>
<pre><code>with open('user.txt', 'r') as file:
    data = json.load(file)
print(data[0])
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Here you assign data to an empty list:</p>
<pre><code>data =[]
</code></pre>
<p>But then you assign data to the text contents of a file:</p>
<pre><code>data = openFile.read())
</code></pre>
<p>... and it's no longer a list.  It seems like you're expecting <code>data</code> to somehow magically remember that it was previously a list and interpret its new content according to that type, but it doesn't do that. </p>
<p>The contents of <code>user.txt</code> <em>look like</em> a 2D list, but it's just text.  But because the text looks <em>exactly</em> like a 2D list, you can use the <code>json</code> module to do what you want:</p>
<pre><code>import json
openFile = open('user.txt', 'r')
data = json.load(openFile)
print (data[0])
</code></pre>
