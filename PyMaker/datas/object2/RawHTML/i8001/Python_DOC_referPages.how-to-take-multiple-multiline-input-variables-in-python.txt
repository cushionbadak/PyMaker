<p><code>a, b = map(int, input().split("\n"))</code></p>

<div class="post-text" itemprop="text">
<p>As others have said it; I don't think you can do it with <code>input()</code>.</p>
<p>But you can do it like this: </p>
<pre><code>import sys
numbers = [int(x) for x in sys.stdin.read().split()]
</code></pre>
<p>Remeber that you can finish your entry by pressing <code>Ctrl+D</code>, then you have a list of numbers, you can print them like this (just to check if it works):</p>
<pre><code>for num in numbers:
    print(num)
</code></pre>
<p>Edit:
for example, you can use an entry like this (one number in each line):</p>
<pre><code>1
543
9583
0
3
</code></pre>
<p>And the result will be: <code>numbers = [1, 543, 9583, 0, 3]</code></p>
<p>Or you can use an entry like this:</p>
<pre><code>1
53          3
3 4 3 
      54
2
</code></pre>
<p>And the result will be: <code>numbers = [1, 53, 3, 4, 3, 54, 2]</code></p>

<p>You really cannot, <code>input</code> and <code>raw_input</code> stop reading and return when a new line is entered; there's no way to get around that from what I know. From <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow noreferrer"><code>input</code>s documentation</a>:</p>
<blockquote>
<p>The function then reads a line from input, converts it to a string (stripping a trailing newline), and returns that.</p>
</blockquote>
<p>A viable solution might be calling <code>input</code> in a loop and joining on <code>'\n'</code> afterwards.</p>
</div>

<pre><code>[ int(x.strip()) for x in sys.stdin.read().split() ]

<pre><code>1 43 43

[EOF]
</code></pre>
