<div class="post-text" itemprop="text">
<p>When I type <code>python3 --version</code> in terminal it shows Python 3.6.0.</p>
<p>When I type in <code>pip --version</code> it shows </p>
<pre><code>pip 9.0.1 from /Library/Python/2.7/site-packages/pip-9.0.1-py2.7.egg (python 2.7)
</code></pre>
<p>How do I get it from <code>Python/2.7</code> to <code>Python/3.6</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>From Python 3.4 <code>pip</code> is baked into standard Python. You can use it like,</p>
<pre><code>python3 -m pip install SomePackage
</code></pre>
<p>as mentioned in the <a href="https://docs.python.org/3/installing/#basic-usage" rel="noreferrer">Python Docs</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>The guaranteed cross-platform way to run a module 'mod' with a particular python version 'pythonx' is</p>
<pre><code>pythonx -m mod
</code></pre>
<p>On Windows, with the py launcher, I might run any of </p>
<pre><code>py -2.7 -m pip
py -3.5 -m pip
py -3.6 -m pip
</code></pre>
<p>There is then no question which python and pip combination I will get.  Other systems with one 2.x and 3.x version installed can use</p>
<pre><code>python2 -m pip
python3 -m pip
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Mac OS X 10.8+ come with Python 2.7 installed by default. This is why when you try to see what your pip version is, it comes back with 9.0.1 from Python 2.7. You shouldn't ever uninstall or move the system-installed Python because it's used internally by the Mac OS and other third-party developers, but a quick workaround is to just use <code>pip3</code> in place of <code>pip</code>.</p>
<p>A helpful resource on using Python 3 with Mac OS X can be found <a href="https://docs.python.org/3/using/mac.html" rel="nofollow noreferrer">in the Python docs</a>.</p>
</div>
