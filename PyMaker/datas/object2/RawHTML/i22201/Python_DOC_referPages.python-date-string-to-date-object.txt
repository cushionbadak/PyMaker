<div class="post-text" itemprop="text">
<p>How do I convert a string to a date object in python? </p>
<p>The string would be: <code>"24052010"</code> (corresponding to the format: <code>"%d%m%Y"</code>)</p>
<p>I don't want a datetime.datetime object, but rather a datetime.date.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="http://docs.python.org/library/datetime.html#strftime-and-strptime-behavior" rel="noreferrer"><code>strptime</code></a> in the <a href="http://docs.python.org/library/datetime.html" rel="noreferrer"><code>datetime</code></a> package of Python:</p>
<pre><code>&gt;&gt;&gt; datetime.datetime.strptime('24052010', "%d%m%Y").date()
datetime.date(2010, 5, 24)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<pre><code>import datetime
datetime.datetime.strptime('24052010', '%d%m%Y').date()
</code></pre>

<p><strong>Directly related question:</strong></p>
<p>What if you have</p>
<pre><code>datetime.datetime.strptime("2015-02-24T13:00:00-08:00", "%Y-%B-%dT%H:%M:%S-%H:%M").date()
</code></pre>
<p>and you get:</p>
<pre><code>Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
  File "/usr/local/lib/python2.7/_strptime.py", line 308, in _strptime

<p>and you tried:</p>
<pre><code>&lt;-24T13:00:00-08:00", "%Y-%B-%dT%HH:%MM:%SS-%HH:%MM").date()

<p>but you still get the traceback above. </p>
<p><strong>Answer:</strong></p>
<pre><code>&gt;&gt;&gt; from dateutil.parser import parse
&gt;&gt;&gt; from datetime import datetime
&gt;&gt;&gt; parse("2015-02-24T13:00:00-08:00")
datetime.datetime(2015, 2, 24, 13, 0, tzinfo=tzoffset(None, -28800))
</code></pre>

<div class="post-text" itemprop="text">
<p>If you are lazy and don't want to fight with string literals, you can just go with the <code>parser</code> module. </p>
<pre><code>from dateutil import parser
dt = parser.parse("Jun 1 2005  1:33PM")
print(dt.year, dt.month, dt.day,dt.hour, dt.minute, dt.second)
&gt;2005 6 1 13 33 0
</code></pre>
<p>Just <strong>a side note</strong>, as we are trying to match <code>any</code> string representation, it is 10x slower than <code>strptime</code></p>
</div>
<div class="post-text" itemprop="text">
<p>There is another library called <a href="https://github.com/crsmithdev/arrow" rel="nofollow"><code>arrow</code></a> really great to make manipulation on python date.</p>
<pre><code>import arrow
import datetime

a = arrow.get('24052010', 'DMYYYY').date()
print(isinstance(a, datetime.date)) # True
</code></pre>

<div class="post-text" itemprop="text">
<p>you have a date string like this, "24052010" and you want date object for this,</p>
<pre><code>from datetime import datetime
cus_date = datetime.strptime("24052010", "%d%m%Y").date()
</code></pre>
<p>this cus_date will give you date object.</p>
<p>you can retrieve date string from your date object using this,</p>
<pre><code>cus_date.strftime("%d%m%Y")
</code></pre>
