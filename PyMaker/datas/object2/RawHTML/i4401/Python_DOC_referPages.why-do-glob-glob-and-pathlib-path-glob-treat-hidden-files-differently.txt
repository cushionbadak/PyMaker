<div class="post-text" itemprop="text">
<p>Consider this folder containing two files:</p>
<pre><code>test/
    foo
    .bar
</code></pre>
<p>Calling <a href="https://docs.python.org/3/library/glob.html#glob.glob" rel="nofollow noreferrer"><code>glob.glob('*')</code></a> on this folder won't list the hidden <code>.bar</code> file:</p>
<pre><code>&gt;&gt;&gt; glob.glob('test/*')
['test/foo']
</code></pre>
<p>But <a href="https://docs.python.org/3/library/pathlib.html#pathlib.Path.glob" rel="nofollow noreferrer"><code>pathlib.Path.glob('*')</code></a> will:</p>
<pre><code>&gt;&gt;&gt; list(Path('test').glob('*'))
[PosixPath('test/.bar'), PosixPath('test/foo')]
</code></pre>
<p>I'd like to know if this is intended or possibly a bug/oversight.</p>
<hr/>
<p>The <a href="https://docs.python.org/3/library/glob.html" rel="nofollow noreferrer"><code>glob</code> module documentation</a> states that files starting with a dot are special cased:</p>
<blockquote>
<p>glob treats filenames beginning with a dot (.) as special cases</p>
</blockquote>
<p>Meaning that the result given by <code>glob.glob('*')</code> is intended. But what about pathlib's <code>glob</code>? I couldn't find any relevant information in the docs. Is this the intended behavior? Shouldn't both functions produce the same results?</p>
</div>
<div class="post-text" itemprop="text">
<p>As per <a href="https://bugs.python.org/issue26096" rel="nofollow noreferrer">issue #26096</a> on the official bug tracker, this difference has been deemed <code>not a bug</code> and is therefore completely intended.</p>
<p>Credits to <a href="https://stackoverflow.com/users/2301450/vaultah">@vaultah</a> for the find.</p>
