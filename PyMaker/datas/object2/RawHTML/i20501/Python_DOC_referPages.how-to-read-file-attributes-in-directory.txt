<p>How to get file modification time fo all files in directory ?</p>

<div class="post-text" itemprop="text">
<p>When looking for file attributes for all files in a directory, and you are using Python 3.5 or newer, use the <a href="https://docs.python.org/3/library/os.html#os.scandir" rel="nofollow noreferrer"><code>os.scandir()</code> function</a> to get a directory listing <em>with file attributes combined</em>. This can potentially be more efficient than using <code>os.listdir()</code> and then retrieve the file attributes separately:</p>
<pre><code>import os

with os.scandir() as dir_entries:
    for entry in dir_entries:
        info = entry.stat()
        print(info.st_mtime)
</code></pre>
<p>The <a href="https://docs.python.org/3/library/os.html#os.DirEntry.stat" rel="nofollow noreferrer"><code>DirEntry.stat()</code> function</a>, when used on Windows, doesn't have to make any additional system calls, the file modification time is already available. The data is cached, so additional <code>entry.stat()</code> calls won't make additional system calls.</p>
<p>You can also use the <a href="https://docs.python.org/3/library/pathlib.html" rel="nofollow noreferrer"><code>pathlib</code> module</a> Object Oriented instances to achieve the same:</p>
<pre><code>from pathlib import Path

for path in Path('.').iterdir():
    info = path.stat()
    print(info.st_mtime)
</code></pre>
<p>On earlier Python versions, you can use the <a href="http://docs.python.org/library/os.html#os.stat" rel="nofollow noreferrer"><code>os.stat</code></a> call for obtaining file properties like the modification time.</p>
<pre><code>import os

for filename in os.listdir():
    info = os.stat(filename)
    print(info.st_mtime)
</code></pre>
<p><code>st_mtime</code> is a float value on python 2.5 and up, representing seconds since the epoch; use the <a href="http://docs.python.org/library/time.html" rel="nofollow noreferrer"><code>time</code></a> or <a href="http://docs.python.org/library/datetime.html" rel="nofollow noreferrer"><code>datetime</code></a> modules to interpret these for display purposes or similar.</p>
<p>Do note that the value's precision depends on the OS you are using:</p>
<blockquote>
<p>The exact meaning and resolution of the st_atime, st_mtime, and st_ctime attributes depend on the operating system and the file system. For example, on Windows systems using the FAT or FAT32 file systems, st_mtime has 2-second resolution, and st_atime has only 1-day resolution. See your operating system documentation for details.</p>
</blockquote>
<p>If all you are doing is get the modification time, then the <a href="http://docs.python.org/library/os.path.html#os.path.getmtime" rel="nofollow noreferrer"><code>os.path.getmtime</code></a> method is a handy shortcut; it uses the <code>os.stat</code> method under the hood.</p>
<p>Note however, that the <code>os.stat</code> call is relatively expensive (file system access), so if you do this on a lot of files, and you need more than one datapoint per file, you are better off using <code>os.stat</code> and reuse the information returned rather than using the <code>os.path</code> convenience methods where <code>os.stat</code> will be called multiple times per file.</p>
</div>
<div class="post-text" itemprop="text">
<p>If you only want the modified time, then <code>os.path.getmtime(filename)</code> will get it for you.  If you are using <code>listdir</code> with an argument, you'll need to also use <code>os.path.join</code>:</p>
<pre><code>import os, os.path

for filename in os.listdir(SOME_DIR):
    print os.path.getmtime(os.path.join(SOME_DIR, filename))
</code></pre>
