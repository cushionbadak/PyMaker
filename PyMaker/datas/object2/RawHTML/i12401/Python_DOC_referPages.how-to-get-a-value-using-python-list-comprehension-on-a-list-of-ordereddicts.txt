<p>I have a list called <code>catalogue</code> given to me like this:</p>
<pre><code>[OrderedDict([(u'catalogue_id', 240)]), OrderedDict([(u'catalogue_id', 240)])]
</code></pre>
<p>I need to create a new list of <code>ID</code>'s from above. This is what I have tried:</p>
<pre><code>x = [x.catalogue_id for x in catalogue]
</code></pre>
<p>But I get the error: </p>
<pre><code>'OrderedDict' object has no attribute 'catalogue_id'. 
</code></pre>
<p>I assume this is because it is a list of dicts, how can this be done?</p>
</div>
<div class="post-text" itemprop="text">
<p><em><a href="https://docs.python.org/2.7/library/collections.html#collections.OrderedDict" rel="nofollow">Ordered Dictionaries</a></em> are subclasses of dictionaries, as such they behave just like ordinary dictionaries when accessing elements. Use key access <code>x['catalogue_id']</code> for every element in <code>catalogue</code> to access the values:</p>
<pre><code>from collections import OrderedDict

catalogue = [OrderedDict([(u'catalogue_id', 240)]), OrderedDict([(u'catalogue_id', 240)])]

x = [x['catalogue_id'] for x in catalogue]

print(x) # [240, 240]
</code></pre>
<p><strong>Note:</strong> You might be confusing it with <em><a href="https://docs.python.org/3/library/collections.html#collections.namedtuple" rel="nofollow"><code>namedtuples</code></a></em> which support named dot <code>.</code> access to their elements.</p>
</div>
<div class="post-text" itemprop="text">
<p>You access the content of a dictionary with <code>__getitem__</code>, not with <code>__getattribute__</code>, i.e. you need to use the bracket notation instead of the dot notation.</p>
<pre><code>&gt;&gt;&gt; a=[OrderedDict([(u'catalogue_id', 240)]), OrderedDict([(u'catalogue_id', 240)])]
&gt;&gt;&gt; [d['catalogue_id'] for d in a]
[240, 240]
</code></pre>
</div>
