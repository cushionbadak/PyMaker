<p>Well <code>Account_num</code> is of type <code>int</code> and the <code>[]</code> slicing notation you're trying to use is supported for sequences (i.e objects that contain other objects). </p>
<p>In order to get the last <em>three</em> digits from a number you can use the <strong><a href="https://docs.python.org/3/reference/expressions.html#binary-arithmetic-operations" rel="nofollow"><code>%</code></a></strong> operator that yields the remainder from the division, with <code>1000</code>:</p>
<pre><code>Disk = Account_num % 1000
</code></pre>
<p>So for a given <code>Accound_num = 9230939</code> </p>
<pre><code>Disk = Account_num % 1000
Print(Disk) # prints 939
</code></pre>

<p>You can't use [-3:] on an int, but you can on a string so just get the input and then cast to a string to grab the last three characters and then cast to an int.</p>
<pre><code>Account_num = input("Enter account number: ")
Disk = int(str(Account_num)[-3:])
if Account_num &lt;= 99999999:
    print("Your disk storage location is:",Disk
          )
else:
    print("Invalid account number entred")
</code></pre>
