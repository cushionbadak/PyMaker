<div class="post-text" itemprop="text">
<p>I am using <code>argparse</code> to parse command line arguments.</p>
<p>To aid debugging, I would like to <code>print</code> a line with the arguments which with the Python script was called. Is there a simple way to do this within <code>argparse</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/argparse.html#argparse.ArgumentParser.parse_args" rel="noreferrer"><code>ArgumentParser.parse_args</code></a> by default takes the arguments simply from <a href="https://docs.python.org/3/library/sys.html#sys.argv" rel="noreferrer"><code>sys.argv</code></a>. So if you donâ€™t change that behavior (by passing in something else to <code>parse_args</code>), you can simply print <code>sys.argv</code> to get all arguments passed to the Python script:</p>
<pre><code>import sys
print(sys.argv)
</code></pre>
<p>Alternatively, you could also just print the namespace that <code>parse_args</code> returns; that way you get all values in the way the argument parser interpreted them:</p>
<pre><code>args = parser.parse_args()
print(args)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If running <strong><em>argparse within another python script</em></strong>, like when testing it inside <a href="https://stackoverflow.com/a/18161115/658497">unittest</a>, then printing <strong><em>sys.argv</em></strong> will only print the arguments of the main script, e.g.:</p>
<blockquote>
<p>['C:\eclipse\plugins\org.python.pydev_5.9.2.201708151115\pysrc\runfiles.py', 'C:\eclipse_workspace\test_file_search.py', '--port', '58454', '--verbosity', '0']</p>
</blockquote>
<p>In this case you should use <a href="https://docs.python.org/2/library/functions.html#vars" rel="noreferrer">vars</a> to iterate over argparse args:</p>
<pre><code>parser = argparse.ArgumentParser(...
...
args = parser.parse_args()
for arg in vars(args):
    print arg, getattr(args, arg)
</code></pre>

</div>
