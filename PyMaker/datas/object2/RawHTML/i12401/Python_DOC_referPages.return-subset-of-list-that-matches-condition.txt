<div class="post-text" itemprop="text">
<p>Let's say I have a list of <code>int</code>s:</p>
<pre><code>listOfNumbers = range(100)
</code></pre>
<p>And I want to return a list of the elements that meet a certain condition, say:</p>
<pre><code>def meetsCondition(element):
    return bool(element != 0 and element % 7 == 0)
</code></pre>
<p>What's a Pythonic way to return a sub-<code>list</code> of element in a <code>list</code> for which  <code>meetsCondition(element)</code> is <code>True</code>?</p>
<p>A naive approach:</p>
<pre><code>def subList(inputList):
    outputList = []

    for element in inputList:
        if meetsCondition(element):
            outputList.append(element)

    return outputList

divisibleBySeven = subList(listOfNumbers)
</code></pre>
<p>Is there a simple way to do this, perhaps with with a list comprehension or <code>set()</code> functions, and without the temporary outputList?</p>
</div>
<div class="post-text" itemprop="text">
<p>Use list comprehension,</p>
<pre><code>divisibleBySeven = [num for num in inputList if num != 0 and num % 7 == 0]
</code></pre>
<p>or you can use the <code>meetsCondition</code> also,</p>
<pre><code>divisibleBySeven = [num for num in inputList if meetsCondition(num)]
</code></pre>
<p>you can actually write the same condition with Python's <a href="https://docs.python.org/3/library/stdtypes.html?highlight=truth#truth-value-testing">truthy</a> semantics, like this</p>
<pre><code>divisibleBySeven = [num for num in inputList if num and num % 7]
</code></pre>
<hr/>
<p>alternatively, you can use <code>filter</code> function with your <code>meetsCondition</code>, like this</p>
<pre><code>divisibleBySeven = filter(meetsCondition, inputList)
</code></pre>
</div>
<span class="comment-copy">Note that <code>divisibleBySeven</code> is actually the same <code>list</code> object as <code>outputList</code>, it's not a copy of it.</span>
<span class="comment-copy">In Python 3, <a href="https://docs.python.org/3.3/library/functions.html#filter" rel="nofollow noreferrer"><code>filter</code></a> returns an iterator (IOW a generator), so if you actually need a list then you need to do something like <code>list(filter(meetsCondition, inputList))</code>.</span>
