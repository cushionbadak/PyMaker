<div class="post-text" itemprop="text">
<p>For <code>import PythonMDK</code> to expose any of your functions in <code>code1.py</code>, the file <code>PythonMDK/__init__.py</code> must import them like <code>from code1 import myfunction</code>. Then you could do <code>import PythonMDK; PythonMDK.myfunction()</code>.</p>
<p>See <a href="https://docs.python.org/3/tutorial/modules.html#packages" rel="nofollow noreferrer">https://docs.python.org/3/tutorial/modules.html#packages</a>, <a href="https://docs.python.org/3/reference/import.html#packages" rel="nofollow noreferrer">https://docs.python.org/3/reference/import.html#packages</a>, and <a href="https://docs.python.org/3/reference/import.html#regular-packages" rel="nofollow noreferrer">https://docs.python.org/3/reference/import.html#regular-packages</a></p>

<span class="comment-copy">Just install via pip: <code>pip install .</code> (supposing you're in the folder with the <code>setup.py</code> file). Then you can <code>import PythonMDK</code>. By the way the same should be possible after doing <code>python setup.py install</code>.</span>

<span class="comment-copy">Also why are "dist" and "build" folders created?</span>
<span class="comment-copy">You can either import the desired modules in <code>__init__</code> or you can directly import them: <code>import PythonMDK.code1</code>.</span>
<span class="comment-copy">Distutils/setuptools use <code>build</code> directory for their temporary files; <code>dist</code> is for thr resulting source and binary packages. You can clean them with the command <code>python setup.py clean</code>.</span>
