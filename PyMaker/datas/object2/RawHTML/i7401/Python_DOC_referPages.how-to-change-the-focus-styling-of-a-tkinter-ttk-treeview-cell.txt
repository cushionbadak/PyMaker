Specifically the text column when I do <code>tree.insert('', tk.END, text='Some Text', values=5)</code> is taking focus, meaning there is a dashed line around <code>Some Text</code>. I was able to find this <a href="https://stackoverflow.com/questions/23354303/removing-ttk-notebook-tab-dashed-line/23399786#23399786">resource</a>, but I am not sure of the layout string I need to change.</p>

<div class="post-text" itemprop="text">
<p>According to the <a href="http://wiki.tcl.tk/26993" rel="nofollow noreferrer">Tcl/Tk</a> wiki, the following 5 styles can be used to customize a <code>ttk.Treeview()</code> widget:</p>
<pre><code>"Treeview"
"Treeview.Heading"
"Treeview.Row"
"Treeview.Cell"
"Treeview.Item"
</code></pre>
<p>Using <a href="https://docs.python.org/3/library/tkinter.ttk.html#tkinter.ttk.Style.layout" rel="nofollow noreferrer"><code>.layout()</code></a>, you can retrieve the layout specifications of each style:</p>
<pre><code>style = ttk.Style()
style.layout("Treeview.Item") 
</code></pre>
<p>It turns out that the <code>"Treeview.Item"</code> style has a <code>"Treeitem.focus"</code> layout mark. If you <a href="https://stackoverflow.com/a/23399786/4084269">comment it out</a> when overwriting the layout, the focus drawing behavior (and the dashed line) will disappear:</p>
<pre><code>style = ttk.Style()

style.layout("Treeview.Item",
[('Treeitem.padding', {'sticky': 'nswe', 'children': 
    [('Treeitem.indicator', {'side': 'left', 'sticky': ''}),
    ('Treeitem.image', {'side': 'left', 'sticky': ''}),
    #('Treeitem.focus', {'side': 'left', 'sticky': '', 'children': [
         ('Treeitem.text', {'side': 'left', 'sticky': ''}),
    #]})
    ],
})]
)
</code></pre>
