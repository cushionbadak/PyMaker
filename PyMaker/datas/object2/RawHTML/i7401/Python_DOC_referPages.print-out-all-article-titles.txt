<div class="post-text" itemprop="text">
<p>First, let's see what printing one <code>story_heading</code> alone gives us:</p>
<pre><code>&gt;&gt;&gt; story_heading
&lt;h2 class="story-heading"&gt;&lt;a href="https://www.nytimes.com/real-estate/mortgage-calculator"&gt;Mortgage Calculator&lt;/a&gt;&lt;/h2&gt;
</code></pre>
<p>To extract <strong>only</strong> the <code>a</code> tag, we access it using <code>story_heading.a</code>:</p>
<pre><code>&gt;&gt;&gt; story_heading.a
&lt;a href="https://www.nytimes.com/real-estate/mortgage-calculator"&gt;Mortgage Calculator&lt;/a&gt;
</code></pre>
<p>To get only the <strong>text</strong> inside the tag itself, and not it's attributes, we use <code>.text</code>:</p>
<pre><code>&gt;&gt;&gt; story_heading.a.text
'Mortgage Calculator'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Here,
.a gives you the first anchor tag
.text gives you the text within the tag</p>

<span class="comment-copy">Thank you very much. One more question: what does the dot '.' mean?</span>
<span class="comment-copy">In Python, and other Object Oriented Programming (OOP) languages, a dot is used to call a method from a certain class. For example, the built-in class <code>str</code>(string) has a method called <code>upper</code>, that converts all characters to uppercase, to call it, we do: <code>a = "this is a string"</code> and then <code>a.upper()</code>, or simply <code>"This is a string".upper()</code>. It seems you are really new to Python, the <a href="https://docs.python.org/3/library/string.html" rel="nofollow noreferrer">documentation</a> has very good texts about it's syntax and methods of built-in class if you wish to go a little further with the topic.</span>
<span class="comment-copy">Just a correction, writing without <code>()</code> in the end is not actually calling the method, instead it is just referencing the object associated with the method.</span>
