<div class="post-text" itemprop="text">
<p>This seems like a bug in <code>pelican-import</code>. <code>feedparser</code>'s date/time parser <a href="https://github.com/kurtmckee/feedparser/blob/2b11c8028321ed43cbaf313f83b0c94820143d66/feedparser/datetimes/rfc822.py#L118" rel="nofollow noreferrer">returns</a> the result of <a href="https://docs.python.org/3/library/datetime.html#datetime.datetime.utctimetuple" rel="nofollow noreferrer">datetime.utctimetuple()</a> which is <code>time.struct_time</code> instance while <code>pelican-import</code> <a href="https://github.com/getpelican/pelican/blob/6f0743b340cfaa98ced38fc41727c8ea9f1a52f8/pelican/tools/pelican_import.py#L566" rel="nofollow noreferrer">expects</a> an object with <code>.strftime()</code> method.</p>
<p>Report the bug to <a href="https://github.com/getpelican/pelican/issues" rel="nofollow noreferrer">the Pelican tracker</a>.</p>
