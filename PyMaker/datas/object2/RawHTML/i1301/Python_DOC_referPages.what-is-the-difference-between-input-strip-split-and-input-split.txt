<p>Split() function uses whitespace string as separator and removes the empty strings, so I think there is no use using strip() or rstrip() function to remove extra whitespace in head or tail. And here is my example:</p>
<pre><code>a = ' \n   1 2 3 4     \n\n 5 \n\n \t'
b = a.rstrip().split()
c = a.split()
print('b =',b)
print('c =',c)
</code></pre>
<p>The result turns out to be:</p>
<pre><code>b = ['1', '2', '3', '4', '5']
c = ['1', '2', '3', '4', '5']
</code></pre>
<p>It seems that there is no difference in betweeen. However, the former one( <code>intput().strip().split()</code>) seems more widely used. So what is the difference in these two expressions? </p>

<div class="post-text" itemprop="text">
<p>There's no difference. <code>split()</code> ignores whitespace on the ends of the input by default. People call <code>strip()</code> first either because they think it's clearer or because they don't know this behavior of <code>split()</code>.</p>
<p><a href="https://docs.python.org/3/library/stdtypes.html#str.split" rel="nofollow noreferrer">Docs</a>:</p>
<blockquote>
<p>If sep is not specified or is None, a different splitting algorithm is applied: runs of consecutive whitespace are regarded as a single separator, and the result will contain no empty strings at the start or end if the string has leading or trailing whitespace. Consequently, splitting an empty string or a string consisting of just whitespace with a None separator returns [].</p>
</blockquote>
</div>
<span class="comment-copy">As a side note, this is not clearly documented for Python 2, hence the confusion: <a href="https://docs.python.org/2/library/string.html#string.split" rel="nofollow noreferrer">docs.python.org/2/library/string.html#string.split</a></span>
<span class="comment-copy">@Selcuk: Yup, I'd probably call <code>strip()</code> too if I'd only seen those docs.</span>
<span class="comment-copy">There's no difference in <i>the output</i> if you use the default parameters for <code>.strip()</code> and <code>.split()</code>, but that will not always be the case.  Compare these examples:      <code>a = ' \n 1,2,3,4,5 \n\n \t'</code> <code>b = a.strip().split(',')</code> <code>c = b.split(',')</code> <code>print(b)</code> <code># Out: ['1', '2', '3', '4', '5']</code> <code>print(c)</code> <code># Out: [' \n 1', '2', '3', '4', '5 \n\n \t']</code>  Also, while just using <code>.split()</code> is a useful shortcut, <code>.strip().split()</code> more clearly explains what is happening.</span>
