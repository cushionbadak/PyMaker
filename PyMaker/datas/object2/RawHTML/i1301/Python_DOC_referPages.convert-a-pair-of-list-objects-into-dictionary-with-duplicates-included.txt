<div class="post-text" itemprop="text">
<p>Instead you can create the dictionary with values as list:</p>
<pre><code>from collections import defaultdict
d = defaultdict(list)

for k,v in zip(list3, list4):
    d[k].append(v)

defaultdict(list, {1: ['a'], 2: ['b'], 3: ['c', 'd'], 4: ['e', 'f']})
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can't have duplicate keys in a dictionary. However, you can have multiple values(a list) mapped to each key.</p>
<p>An easy way to do this is with <a href="https://docs.python.org/3/library/stdtypes.html#dict.setdefault" rel="nofollow noreferrer"><code>dict.setdefault()</code></a>:</p>
<pre><code>list3 = [1,2,3,3,4,4]
list4 = ['a','b','c','d','e','f']

d = {}
for x, y in zip(list3, list4):
    d.setdefault(x, []).append(y)

print(d)
# {1: ['a'], 2: ['b'], 3: ['c', 'd'], 4: ['e', 'f']}
</code></pre>
<p>The other option is to use a <a href="https://docs.python.org/3/library/collections.html#collections.defaultdict" rel="nofollow noreferrer"><code>collections.defaultdict()</code></a>, as shown in @YOLO's <a href="https://stackoverflow.com/questions/53688658/convert-a-pair-of-list-objects-into-dictionary-with-duplicates-included/53688679#53688679">answer</a>.</p>
