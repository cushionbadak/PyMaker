<div class="post-text" itemprop="text">
<p>You assigned the variable name <code>str</code> to something elsewhere in your code, shadowing the <a href="https://docs.python.org/3/library/functions.html#func-str" rel="nofollow"><code>str</code></a> built-in. A minimal reproduction would be: </p>
<pre><code>&gt;&gt;&gt; str = 'something'
&gt;&gt;&gt; try_string = str(5)
TypeError: 'str' object is not callable
</code></pre>
<p>Python tries to use your assigned variable for the function call, resulting in your error. Find whatever you named <code>str</code> in your code, and rename it. </p>
<p>In general, <strong>always make sure</strong> you don't shadow <a href="https://docs.python.org/3/library/functions.html#func-str" rel="nofollow">built-ins</a> with your variable names, to prevent this from happening. </p>
