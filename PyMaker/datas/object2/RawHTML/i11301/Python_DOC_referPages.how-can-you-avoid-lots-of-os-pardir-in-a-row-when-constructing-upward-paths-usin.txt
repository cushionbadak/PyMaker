<div class="post-text" itemprop="text">
<p>I'm trying to avoid this kind of thing:</p>
<pre><code>sibling_location = os.path.join(leaf_dir, os.pardir, os.pardir, os.pardir, sibling_name)
</code></pre>
<p>I thought it would be cool to say</p>
<pre><code>sibling_location = os.path.join(leaf_dir, *[os.pardir]*3, sibling_name)
</code></pre>
<p>but unfortunately that <code>*</code> argument expansion trick doesn't allow further arguments after the list it expands.</p>
</div>
<div class="post-text" itemprop="text">
<p>How about using <a href="https://pypi.python.org/pypi/pathlib2/" rel="nofollow"><code>pathlib2</code></a> (backport of Python 3 standard library module <a href="http://docs.python.org/3/library/pathlib.html" rel="nofollow"><code>pathlib</code></a>)?</p>
<pre><code>&gt;&gt;&gt; leaf_dir = '/path/to/some/deep/deeper/leaf_dir'
&gt;&gt;&gt; sibling_name = 's'
&gt;&gt;&gt; 
&gt;&gt;&gt; # Using os.path.join
&gt;&gt;&gt; import os
&gt;&gt;&gt; os.path.join(leaf_dir, *([os.pardir]*3 + [sibling_name]))
'/path/to/some/deep/deeper/leaf_dir/../../../s'
&gt;&gt;&gt; os.path.normpath(os.path.join(leaf_dir, *([os.pardir]*3 + [sibling_name])))
'/path/to/some/s'
&gt;&gt;&gt;
&gt;&gt;&gt; # Using pathlib / pathlib2
&gt;&gt;&gt; import pathlib2
&gt;&gt;&gt; str(pathlib2.Path(leaf_dir).parents[2] / sibling_name)
'/path/to/some/s'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Heh - found a way.</p>
<pre><code>sibling_location = os.path.join(leaf_dir, *([os.pardir]*3 + [sibling_name]))
</code></pre>
<p>It doesn't read wildly well, but it is concise.   Happy to accept other better answers :)</p>
