<p>Disclaimer: I never used poplib myself but I did read the official docs here: <a href="https://docs.python.org/3/library/poplib.html#poplib.POP3.retr" rel="nofollow">https://docs.python.org/3/library/poplib.html#poplib.POP3.retr</a></p>
<p>When you call .retr(i+1) here:</p>
<pre><code>for i in range(numberOfMails):
    for email in myEmailConnection.retr(i+1)[1]:
        print email
</code></pre>
<p>retr(i+1) returns 3 values: <code>(response, ['line', ...], octets)</code>.
Here you take only the second value (by doing <code>.retr(i+1)[1]</code>), which contains a list of the message lines.</p>
<p>Just join all these lines to get a single string message by doing:</p>
<pre><code>for i in range(numberOfMails):
    for emailLines in myEmailConnection.retr(i+1)[1]:
        print "\n".join(emailLines)
</code></pre>
<p>And if you want to parse the actual MIME content, you can use the <code>email</code> lib as suggested by @larsks</p>
