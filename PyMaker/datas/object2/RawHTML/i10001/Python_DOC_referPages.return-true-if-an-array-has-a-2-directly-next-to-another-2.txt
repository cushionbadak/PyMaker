<p>Given an array of ints, return True if the array contains a 2 next to a 2 somewhere.</p>
<p>has22([1, 2, 2]) → True</p>
<p>has22([1, 2, 1, 2]) → False</p>
<p>has22([2, 1, 2]) → False</p>
<p>My attempt at this was:</p>
<pre><code>def has22(nums):
    if 2 in nums:
        if (nums[nums.index(2) + 1] == 2):
            return True
        elif (nums[nums.index(2) - 1] == 2):
            return True
        else:
            return False
    else:
        return False
</code></pre>
<p>Which results in <code>Error: list index out of range</code>. If anyone can explain why this is happening that'd be appreciated.</p>
</div>
<div class="post-text" itemprop="text">
<p>First, <a href="https://docs.python.org/3/library/itertools.html#itertools-recipes" rel="nofollow"><code>pairwise()</code></a>. And then:</p>
<pre><code>any(x == y == 2 for (x, y) in pairwise(seq))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>I like the <a href="https://docs.python.org/3/library/itertools.html#itertools-recipes" rel="nofollow">pairwise</a> idea, and here is other without importing anything </p>
<pre><code>any( seq[i-1] == seq[i] == 2 for i in range(1,len(seq)) )
</code></pre>
<hr/>
<p>The problem with your code is that not only will fail when 2 is the first or last element, it also will fail when the pair of 2s is not the with the first 2 like for example [1,2,3,2,2] because <code>.index</code> only give you the first position of the first 2. </p>
<p>The solution is simple, ask for a index that you know will be there, as show in the example above starting in 1 ask for all indexes and the previous one so in the first time you ask for index 0 and 1, then 1 and 2, and so on</p>
</div>
<div class="post-text" itemprop="text">
<p>If the input is a small <a href="https://docs.python.org/3/library/stdtypes.html#sequence-types-list-tuple-range" rel="nofollow">sequence</a>:</p>
<pre><code>any(a == b == 2 for a, b in zip(seq, seq[1:]))
</code></pre>
