<p>There is a <a href="http://datastax.github.io/python-driver/dates_and_times.html#write-path" rel="nofollow noreferrer">doc page</a> that discusses the details of working with time with the Python driver.</p>
<p>You are correct that it can be either a <code>datetime</code>, or integer milliseconds from epoch.</p>
<p>Integer input would be marginally more efficient, but you will probably not observe any performance difference, since [query time] &gt;&gt; [parameter encoding].</p>
</div>
