<div class="post-text" itemprop="text">
<p>Using Python's <code>unittest</code> module, the assertion</p>
<pre><code>self.assertTrue(a &gt; b - 0.5 &amp;&amp; a &lt; b + 0.5, "The two values did not agree")
</code></pre>
<p>outputs the following on failure:</p>
<pre><code>AssertionError: False is not true : The two values did not agree
</code></pre>
<p>I don't want the <code>False is not true</code> to be printed. Ideally, <code>AssertionError</code> shouldn't be printed either. Only <code>The two values did not agree</code> should be printed.</p>
<p>Can I do this?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can supress the <code>False is true</code> part, however, one thing to keep in mind here, you are <em>raising</em> an exception, and what you are seeing is the standard output of an assertion being raised in Python. You want to see this. Furthermore, this is being called directly in the assertion methods, as you can see from the <code>assertTrue</code> below:</p>
<pre><code>def assertTrue(self, expr, msg=None):
    """Check that the expression is true."""
    if not expr:
        msg = self._formatMessage(msg, "%s is not true" % safe_repr(expr))
        raise self.failureException(msg)
</code></pre>
<p>To supress the 'False is true' part, change the <a href="https://docs.python.org/3/library/unittest.html#unittest.TestCase.longMessage" rel="nofollow noreferrer">longMessage</a> class attribute to False: </p>
<pre><code>class TestCompare(unittest.TestCase):

    longMessage = False

    def test_thing(self):
        self.assertTrue(5 == 6, "The two values did not agree")
</code></pre>
<p>Output: </p>
<pre><code>Failure
Traceback (most recent call last):
  File "/Users/XXX/dev/rough/test_this.py", line 21, in test_things
    self.assertTrue(5 == 6, "The two values did not agree")
AssertionError: The two values did not agree
</code></pre>
