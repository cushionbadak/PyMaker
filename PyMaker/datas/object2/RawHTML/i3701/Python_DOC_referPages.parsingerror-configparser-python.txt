<p>I am using <a href="https://docs.python.org/3/library/configparser.html#supported-ini-file-structure" rel="nofollow noreferrer">configparser</a> to read ini file. However, I am getting <a href="https://docs.python.org/3/library/configparser.html#configparser.ParsingError" rel="nofollow noreferrer">ParsingError</a>, which can be seen below:</p>
<pre><code>[line  2]: '  % ---------------- File created by User ----------------------\n'
[line  9]: '  % ---------------------------------------------------------------------------\n'
[line 11]: '  % ---------------------------------------------------------------------------\n'
</code></pre>
<p>I cannot use the option "allow_no_value=True" as line 9 and line 11 are same and in Document. </p>
<p>The example document is as follows:</p>
<pre><code>'  % ---------------- File created by User ----------------------\n'

[OrderID]
    orderName       = Ord1           
    orderNumber     = Ord2#02        
</code></pre>
<p>Is there any way to parse the document? The code which Im currently using is as follows:</p>
<pre><code>import configparser

config = configparser.ConfigParser()
config.read(path,encoding='cp1250')
</code></pre>
<p>Thank you</p>

<div class="post-text" itemprop="text">
<p>From the documentation for <code>configparser</code>: "<em>Config parsers allow for heavy customization.</em>"</p>
<p>In particular, you can specify that <code>'</code> introduces a comment line.</p>
<pre><code>import configparser

config = configparser.ConfigParser(comment_prefixes=("#", ";", "'"))
config.read(path, encoding='cp1250')
</code></pre>
<p>See: <a href="https://docs.python.org/3/library/configparser.html#customizing-parser-behaviour" rel="nofollow noreferrer">https://docs.python.org/3/library/configparser.html#customizing-parser-behaviour</a></p>
