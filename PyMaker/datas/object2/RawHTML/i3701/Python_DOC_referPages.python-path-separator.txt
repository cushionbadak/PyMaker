<p>Is there a proper platform-dependent (or independent?) path separator character somewhere in the Python standard library?</p>
<p>I am not asking about the <em>directory</em> separator <code>/</code> and <code>\</code>, but rather about the <code>PATH</code> separator: <code>:</code> and <code>;</code>.</p>
<p>Java has <a href="https://docs.oracle.com/javase/7/docs/api/java/io/File.html#pathSeparatorChar" rel="nofollow noreferrer"><code>java.io.File.pathSeparatorChar</code></a> and <a href="https://docs.oracle.com/javase/7/docs/api/java/io/File.html#pathSeparator" rel="nofollow noreferrer"><code>java.io.File.pathSeparator</code></a> to fulfill this need. Does Python have something similar? A fairly exhaustive search in <a href="https://docs.python.org/3/library/os.path.html" rel="nofollow noreferrer"><code>os.path</code></a> and <a href="https://docs.python.org/3/library/pathlib.html" rel="nofollow noreferrer"><code>pathlib</code></a> docs yielded nothing, so I am beginning to lose hope.</p>
<p>I am not particularly hung up on the character itself. A function that behaves like <a href="https://docs.python.org/3/library/os.path.html" rel="nofollow noreferrer"><code>os.path.join</code></a> but for entire paths instead of path elements would be perfectly acceptable, preferable even.</p>

<p>Maybe this is what you're looking for:</p>
<pre><code>import os

os.pathsep
</code></pre>
<p><code>os.pathsep</code> is <code>:</code> or <code>;</code> while <code>os.path.sep</code> is <code>\</code> or <code>/</code>.</p>
