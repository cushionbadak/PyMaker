<div class="post-text" itemprop="text">
<p>build_args' instantiation is simply <em>evaluating a logical boolean OR statement</em> inside of the list a list structure. After the OR statement has been evaluated there will simply be a single boolean value stored in build_args. (props to ukemi, beat me to the punch)</p>
<p>Since it was also included in the code snippet I'd add that info for '.extend()' following can be found <a href="https://docs.python.org/3/tutorial/datastructures.html" rel="nofollow noreferrer">here</a>. Essentially .extend() then just appends all items of an iterable structure on the end of the list, so build_args' content would then be <code>[&lt;boolean&gt;, '-j', &lt;job_string&gt;]</code></p>

<div class="post-text" itemprop="text">
<pre><code>build_args = [NINJA or MAKE]
</code></pre>
<p>If <code>NINJA</code> is equivalent to <code>True</code>, then <code>build_args = [NINJA]</code>.</p>
<p>If <code>NINJA</code> is equivalent to <code>False</code>, then <code>build_args = [MAKE]</code>.</p>
<hr/>
<p><strong>Note:</strong> In python there are <a href="http://docs.python.org/reference/expressions.html#boolean-operations" rel="nofollow noreferrer">other values</a> equivalent to <code>True</code> and <code>False</code>, other than the booleans themselves:</p>
<blockquote>
<p>In the context of Boolean operations, and also when expressions are used by control flow statements, the following values are interpreted as false: </p>
<ul>
<li><code>False</code></li>
<li><code>None</code></li>
<li>numeric zero of all types</li>
<li>empty strings and containers (including strings, tuples, lists, dictionaries, sets and frozensets)</li>
</ul>
<p>All other values are interpreted as true.</p>
</blockquote>
<hr/>
<pre><code>x     | y     | x or y
------|-------|---------
True  | True  | x
True  | False | x
False | True  | y
False | False | y
</code></pre>

<span class="comment-copy">What don't you understand about it?</span>
<span class="comment-copy"><code>or</code> when used in this way means "use the second value if the first one evaluates to false".  So if the value of <code>NINJA</code> is <code>None</code> or <code>""</code> or <code>0</code>, etc., the value of <code>MAKE</code> will be used instead.  The brackets are irrelevant; they are just using the result of that statement to initialize a new <code>list</code>.</span>
<span class="comment-copy">In this idiom, it's almost certain that <code>NINJA</code> and <code>MAKE</code> are <i>not</i> Boolean.  See the comment by <code>0x5453</code></span>
