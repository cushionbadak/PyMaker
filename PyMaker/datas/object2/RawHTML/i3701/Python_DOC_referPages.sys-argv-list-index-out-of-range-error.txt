<p><code>sys.argv</code> is a list in Python, which contains the command-line arguments passed to the script. <code>sys.argv[0]</code> contains the name of the script, <code>sys.argv[1]</code> contains the first argument and so on.</p>
<p>To prevent this error, you need to give command line arguments when starting the script. For example, you can start this script without any errors by </p>
<pre><code>python rssfeedfour.py TRIBUNE http://chicagotribune.feedsportal.com/c/34253/f/622872/index.rss
</code></pre>
<p>You can also modify the script so that it works using the default arguments if you don't provide any command line arguments.</p>
<pre><code>try:
    feed_name = sys.argv[1]
except IndexError:
    feed_name = 'TRIBUNE'

try: 
    url = sys.argv[2]
except IndexError:
    url = 'http://chicagotribune.feedsportal.com/c/34253/f/622872/index.rss'
</code></pre>
<p>You can learn more about handling errors <a href="https://docs.python.org/3/tutorial/errors.html" rel="nofollow noreferrer">here</a>.</p>
<p>Although it is much more convenient to use <a href="https://docs.python.org/3/library/argparse.html" rel="nofollow noreferrer">argparse</a> library. </p>
