<div class="post-text" itemprop="text">
<p>I have a time String like this：</p>
<pre><code>07/01/2015-14:31:58.520
</code></pre>
<p>I use this command line to convert it:</p>
<pre><code>import time
timeStr = "07/01/2015-14:31:58.520"
time.strptime(timeStr,'%d/%m/%y-%H:%M:%S.%f')
</code></pre>
<p>But this returns:</p>
<blockquote>
<p>ValueError: time data '07/01/2015-14:31:58.520' does not match format
  '%d/%m/%y-%H:%M:S.%f'</p>
</blockquote>
<p>My python version is 2.7.7</p>
</div>
<div class="post-text" itemprop="text">
<p><code>%y</code> denotes a 2 digit year, but your string has a 4 digit year.  Use <code>%Y</code> (capital Y) to denote a 4 digit year.  See <a href="https://docs.python.org/3/library/time.html#time.strftime" rel="nofollow">the docs</a> for more information.</p>
<pre><code>time.strptime(timeStr, '%d/%m/%Y-%H:%M:%S.%f')
</code></pre>
<p>Note that <a href="https://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow"><code>datetime.strptime</code></a> may be more useful, as it will return a full <code>datetime</code> object rather than a tuple.  <a href="https://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow">The format syntax is essentially the same</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>It should have been capital <strong>Y</strong> for year (%Y in place of %y)</p>
<pre><code>time.strptime(timeStr,'%d/%m/%Y-%H:%M:%S.%f')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You need to use %Y instead of %y</p>
<pre><code>time.strptime(timeStr,'%d/%m/%Y-%H:%M:%S.%f')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>To get a <code>datetime</code> object, use <code>python-dateutil</code></p>
<p>To install</p>
<pre><code>pip install python-dateutil
</code></pre>
<p>Then</p>
<pre><code>t = "07/01/2015-14:31:58.520"
from dateutil import  parser
&gt;&gt;&gt;parser.parse(t)
datetime.datetime(2015, 7, 1, 14, 31, 58, 520000)

tim = parser.parse(t)
&gt;&gt;&gt;str(tim.date())
'2015-07-01'
</code></pre>
<p>All operations to <code>datetime</code> objects is possible.</p>
</div>
<div class="post-text" itemprop="text">
<p>the time.strptime syntax <code>%d/%m/%y-%H:%M:%S.%f</code> is incorrect, it should be</p>
<blockquote>
<p>"%d/%m/%Y-%H:%M:%S.%f"</p>
</blockquote>
<p>where the only difference is that <code>%y</code> has become <code>%Y</code>. The reason is because from <a href="https://docs.python.org/2/library/time.html#time.strftime" rel="nofollow"> the docs </a> <code>%y</code> is without century number ( [00,99] ), whereas <code>%Y</code> is with century number, which is the syntax you use with "2015"</p>
<p>Tested and functinal in python 2.7.5 and 3.4.1</p>
