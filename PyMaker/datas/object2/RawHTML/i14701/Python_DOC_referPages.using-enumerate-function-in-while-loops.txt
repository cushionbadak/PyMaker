<div class="post-text" itemprop="text">
<p>I have several mathematical algorithms that use iteration to search for the right answer. Here is one example:</p>
<pre><code>def Bolzano(fonction, a, b, tol=0.000001):
    while abs(b - a) &gt; tol:
        m = (a + b) / 2
        if sign(fonction(m)) == sign(fonction(a)):
            a = m
        else:
            b = m
    return a, b
</code></pre>
<p>I want to count how many times the algorithm goes through the loop to get <em>a</em> and <em>b</em>. However this is not a <code>for</code> function and it isn't a list, so I can't clearly indicate what objects do I want to count if I use <code>enumerate</code>. Is there a way to count those loops?</p>
<p>Note: I am not trying to change the code itself. I am really searching for a way to count iterations in a <code>while</code> loop, which I can then use for other cases.</p>
</div>
<div class="post-text" itemprop="text">
<p>The simplest answer if you need a counter outside of a <code>for</code> loop is to count manually using a simple variable and addition inside your <code>while</code> loop:</p>
<pre><code>count = 0
while condition:
    ...
    count += 1
</code></pre>
<p>There is an alternative case - if each step of your iteration has a meanigful value to <code>yield</code>, you may want your loop to be a <a href="https://stackoverflow.com/a/1756156/722121">generator</a>, and then use <code>for</code> loop and <code>enumerate()</code>. This makes most sense where you care about which step you are on, not just the count at the end. E.g:</p>
<pre><code>def produce_steps():
    while condition:
        ...
        yield step_value

for count, step_value in enumerate(produce_steps()):
    ...
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>For a counter I use <code>count</code> from <code>itertools</code>:</p>
<pre><code>from itertools import count
c = count(1)
&gt;&gt;&gt;next(c)
1
&gt;&gt;&gt;next(c)
2
</code></pre>
<p>and so on...</p>
<p><strong>Syntax</strong></p>
<pre><code>count(start=0, step=1)
</code></pre>
<p><strong>Docs</strong></p>
<blockquote>
<p>Make an iterator that returns evenly spaced values starting with
  number start.</p>
</blockquote>
<p>Ref. <a href="https://docs.python.org/3/library/itertools.html#itertools.count" rel="nofollow noreferrer">itertools.count</a></p>
</div>
<span class="comment-copy">Using <code>count()</code> without a <code>for</code> loop doesn't really buy you anything, it just obscures your code a little. You just get a counter where you can't inspect the current value.</span>
