<div class="post-text" itemprop="text">
<p>In python strings have a method <code>isnumeric</code> and <code>isdigit</code>.</p>
<p>I was under the (wrong) impression that <code>isnumeric</code> and <code>isdecimal</code> would return <code>True</code> for strings such as <code>123.45</code> and false otherwise. I'm running the following version of python:</p>
<pre><code>Python 3.4.2 (default, Jan  7 2015, 11:54:58) 
[GCC 4.2.1 Compatible Apple LLVM 6.0 (clang-600.0.56)] on darwin
</code></pre>
<p>And I found that <code>isnumeric</code> and <code>isdecimal</code> return true if all characters in the string are integers, but false if a <code>'.'</code> (dot) is present. What causes this behaviour? Shouldn't <code>'123.45'.isnumeric()</code> return <code>True</code>?</p>
<pre><code>&gt;&gt;&gt; mystr_a = '123.45'
&gt;&gt;&gt; mystr_b = '123'
&gt;&gt;&gt; 
&gt;&gt;&gt; mystr_a.isnumeric()
False
&gt;&gt;&gt; mystr_a.isdecimal()
False
&gt;&gt;&gt; mystr_b.isnumeric()
True
&gt;&gt;&gt; mystr_b.isdecimal()
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As defined in the <a href="https://docs.python.org/3/library/stdtypes.html#str.isnumeric" rel="nofollow">Python documentation</a>, <code>isnumeric</code> returns <code>True</code> if all characters within the string are numeric, otherwise <code>False</code>. A dot is not considered to be numeric.</p>
