<p>Python 3.4 .
Trying to find what is the default timeout in urllib.request.urlopen() .</p>
<p>Its signature is:
urllib.request.urlopen(url, data=None, [timeout, ]*, cafile=None, capath=None, cadefault=False, context=None)</p>
<p>The <a href="https://docs.python.org/3/library/urllib.request.html">doc</a> says that its "global default timeout", and looking at the code its:
socket._GLOBAL_DEFAULT_TIMEOUT</p>
<p>Still what is the actual value in secs?</p>
</div>
<div class="post-text" itemprop="text">
<p>I suspect this is implementation-dependent. That said, for CPython:</p>
<p>From <a href="https://github.com/python-git/python/blob/master/Lib/socket.py#L485" rel="noreferrer"><code>socket.create_connection</code></a>,</p>
<blockquote>
<p>If no <em>timeout</em> is supplied, the global default timeout setting returned by :func:<code>getdefaulttimeout</code> is used.</p>
</blockquote>
<p>From <a href="https://github.com/python-git/python/blob/715a6e5035bb21ac49382772076ec4c630d6e960/Modules/socketmodule.c#L4154" rel="noreferrer"><code>socketmodule.c</code></a>,</p>
<pre><code>static PyObject *
socket_getdefaulttimeout(PyObject *self)
{
    if (defaulttimeout &lt; 0.0) {
        Py_INCREF(Py_None);
        return Py_None;
    }
    else
        return PyFloat_FromDouble(defaulttimeout);
}
</code></pre>
<p><a href="https://github.com/python-git/python/blob/715a6e5035bb21ac49382772076ec4c630d6e960/Modules/socketmodule.c#L719" rel="noreferrer">Earlier</a> in the same file,</p>
<pre><code>static double defaulttimeout = -1.0; /* Default timeout for new sockets */
</code></pre>
<p>So it looks like <code>Py_None</code>, aka <code>None</code>, is the default timeout. In other words, <code>urlopen</code> never times out. At least not from the Python end. I guess a timeout can still occur if the networking functions supplied by the OS have timeouts themselves.</p>
<hr/>
<p>Edit: oops, I guess I didn't need to go source diving for the answer at all, since it's right there in the <a href="https://docs.python.org/3/library/socket.html?highlight=socket#socket.getdefaulttimeout" rel="noreferrer">docs</a>.</p>
<blockquote>
<p>A value of <code>None</code> indicates that new socket objects have no timeout. When the socket module is first imported, the default is <code>None</code>.</p>
</blockquote>
