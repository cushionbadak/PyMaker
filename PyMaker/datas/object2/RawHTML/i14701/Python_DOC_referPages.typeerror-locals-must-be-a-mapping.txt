<div class="post-text" itemprop="text">
<p><code>eval()</code> takes three arguments, the latter two of which are optional. The first is the expression (as a <code>str</code>ing) to evaluate; the latter two, if present, must be mappings that will be used for the global and local namespaces in which to evaluate the expression. See <a href="https://docs.python.org/3/library/functions.html#eval" rel="nofollow">the documentation on <code>eval()</code></a> for more information.</p>
<p>As <a href="https://stackoverflow.com/questions/29567183/typeerror-locals-must-be-a-mapping/29567268#comment47283919_29567183">Kevin</a>, I do not condone the use of <code>eval()</code>. You're already using <code>int()</code> to parse strings into integers; that's a good startâ€”consider also simply adding or subtracting those integers to get the real answer.</p>
