<div class="post-text" itemprop="text">
<p>That's why people use <a href="https://docs.python.org/3.6/tutorial/venv.html" rel="nofollow noreferrer">virtual environments</a>. You should not mix your system Python installation with your project-dependant stuff.</p>
<p>Starting with Python 3.3, you can create new virtual environment <code>venv</code> in current directory like this:</p>
<pre><code>python3 -m venv venv
</code></pre>
<p>Then do</p>
<pre><code>cd venv
source venv/bin/activate
pip install -U pip &lt;project-dependencies&gt;
</code></pre>
<p>and you should be good to go.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can upgrade/reinstall <code>pip</code>, which should fix the link, with </p>
<pre class="lang-py prettyprint-override"><code>python3 -m pip install -U --ignore-installed pip
</code></pre>
<p>This will (probably) make your global <code>pip</code> command run it on that Python 3 interpreter.</p>
