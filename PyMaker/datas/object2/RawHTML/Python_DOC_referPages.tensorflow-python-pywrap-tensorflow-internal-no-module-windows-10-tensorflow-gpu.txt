<div class="post-text" itemprop="text">
<p>If your are sure you have <strong>Visual C++ 2015 Redist</strong> installed on your Windows machine then probably it is the version conflict between <strong>CUDA</strong> version and the <strong>Tensorflow</strong> version. Because newer versions have some conflicts and more over tensorflow requires the exact version of CUDA and the CUDnn Library for it to work properly.</p>
<p>The best way to get it configured correctly is using <strong>Anaconda</strong> for installing and configuring tensorflow-gpu as conda will automatically download all the necessary versions of CUDA and CUDnn for the corresponding tensorflow</p>
<p>The problem is that the new Anaconda uses Python 3.7.x by default which is not supported by tensorflow or tensorflow-gpu</p>
<p>So install Anaconda You can download it <a href="https://www.anaconda.com/distribution/" rel="nofollow noreferrer">here</a> </p>
<p>Now create a virtual environment using conda with python 3.6.x configured by typing the following in Anaconda Prompt</p>
<pre><code>conda create -n yourenvname python=3.6 anaconda
</code></pre>
<p>Now change switch to your environment by typing in</p>
<pre><code>conda activate yourenvname
</code></pre>
<p>Now Check the python version by using which should show you something like this</p>
<pre><code>&gt; python --version
Python 3.6.7 :: Anaconda custom (64-bit)
</code></pre>
<p>Now finally install tensorflow-gpu</p>
<pre><code>conda install tensorflow-gpu
</code></pre>
<p>Now open python and try importing tensorflow</p>
<pre><code>&gt;&gt;&gt; import tensorflow as tf
&gt;&gt;&gt; sess = tf.Session(config=tf.ConfigProto(log_device_placement=True))
</code></pre>

<p>Congrats ! You are all set.</p>
