<div class="post-text" itemprop="text">
<p>Because the last value <a href="https://docs.python.org/2/library/functions.html#range" rel="nofollow noreferrer"><code>range()</code></a> prints is the biggest value for <code>start + step * i</code> that is less than <code>end</code>. If <code>end</code> is less than <code>start</code> and the <code>step</code> is positive, there is no number which satisfies that condition:</p>
<p><code>10 + i * 1</code> will never be smaller than 1 (where <code>i</code> is a positive integer)</p>
<p>Instead, do this before you range:</p>
<pre><code>if a &gt; b:
    a, b = b, a
</code></pre>
<p>which will ensure a is always the smaller number moving forward, by swapping their values if that is not the case.</p>
<p>If you want to preserve the ascending/descending order:</p>
<pre><code>if a &gt; b:
    a, b = b, a
    rev = True
</code></pre>
<p>Then, after building the list</p>
<pre><code>if rev:
    numbers.reverse()
</code></pre>
</div>
<span class="comment-copy">That's what it's supposed, documented, and expected to do.</span>
<span class="comment-copy">See <a href="https://docs.python.org/3/library/stdtypes.html#range" rel="nofollow noreferrer">docs.python.org/3/library/stdtypes.html#range</a>. The last example corresponds to your situation.</span>
<span class="comment-copy">The range function can have up to 2 arguments: start, stop and step. If you don't specify step it will be set to 1 by default. If you want to get from a higher number to a lower one your step has to be negative or the result will be an empty list.</span>
