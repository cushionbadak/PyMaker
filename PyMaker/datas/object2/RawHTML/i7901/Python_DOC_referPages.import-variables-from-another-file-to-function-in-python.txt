<p>You should be very careful with global variables. It is considered a better practice to avoid using global variables, updating them in different locations in the code. It should not be a problem if those are only constants, though. Usually I do it this way:</p>
<p><strong>variables.py</strong>:</p>
<pre><code>MY_PATH1 ="c:/something"
MY_PATH2 ="c:/somethng2"
MY_PATH3 ="c:/something3"
</code></pre>
<p><strong>main.py</strong></p>
<pre><code>from variables import *

def run():
    print(MY_PATH1)
    ...
</code></pre>
<p>If you are working with path strings, you may also want to take a look in the <a href="https://docs.python.org/3/library/os.path.html" rel="nofollow noreferrer">documentation</a> for the <code>os.path</code> module. It helps making path descriptors compatible with the different platforms, among other useful things.</p>
