<p>So what does <code>next</code> keyword mean in python 3? I am using 3.4.0 and sublime 3 text editor and when I write <code>next</code> in it it turns blue? I tried to search about it but I haven't seem to find anything more specific.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>next</code> is not a keyword but a <a href="https://docs.python.org/3/library/functions.html#next" rel="nofollow">built-in function</a>.</p>
<p>The Python documentation also includes a <a href="https://docs.python.org/3/reference/lexical_analysis.html#keywords" rel="nofollow">list of keywords</a>; you'll see it is not listed there.</p>
<p>In Python 2, iterators defined an <a href="https://docs.python.org/2/library/stdtypes.html#iterator.next" rel="nofollow"><code>iterator.next()</code> <em>method</em></a>, but that has been renamed to <a href="https://docs.python.org/3/library/stdtypes.html#iterator.__next__" rel="nofollow"><code>iterator.__next__()</code></a> in Python 3. It is probably that method name the syntax highlighter tries to color-code; the <a href="https://sublime.wbond.net/packages/Python%203" rel="nofollow">Python 3 package for Sublime Text</a> adds <code>__next__</code> as highlighted too.</p>
</div>
