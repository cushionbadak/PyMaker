<p>For example, when adding lists together:</p>
<pre class="lang-py prettyprint-override"><code>list = [1,2,3,4,5]

list_sum = 0
for x in list:
   list_sum += x
</code></pre>

<div class="post-text" itemprop="text">
<p><code>list_sum += x</code> means add the contents of <code>list_sum</code> variable with the contents of variable <code>x</code> and again store the result to <code>list_sum</code> variable.</p>
<p>Code explanation:</p>
<pre><code>list_sum = 0     # At first, 0 is assigned to the `list_sum` variable .
for x in list:   # iterating over the contents which are present inside the variable `list`
list_sum += x    # list_sum = 0+1 . After the first iteration, value 1 is stored to list_sum variable. Likewise it sums up the values present in  the list and then assign it back to list_sum variable. Atlast `list_sum` contains the sum of all the values present inside the given list.
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>It's <a href="http://pymbook.readthedocs.org/en/latest/operatorsexpressions.html" rel="nofollow">shorthand</a> for <code>list_sum = list_sum + x</code></p>
<p><code>for x in list:</code> will loop once through every element in <code>list</code>, assigning the value to a temporary variable <code>x</code></p>

<span class="comment-copy">It's not exactly the same as <code>list_sum = list_sum + x</code>. That implies that a new object is created and then assigned to the name <code>list_sum</code>. This is true for immutable objects, but for mutable objects <code>+=</code> is an in-place operator and modifies the existing object. The main difference being that other references will also se the change. eg. <code>x = []; y = x; y += [1]; assert x == y</code>, whereas <code>x = 0; y = x; y += 1; assert x != y</code>.</span>
