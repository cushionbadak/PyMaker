<p>I parse the following request <code>param1=value1&amp;param2=&amp;param3=value3</code> using <code>urllib.parse.parse_qs</code> (python 3 wsgi application) but this function returns a dict with only <code>param1</code> and <code>param3</code> keys. What function can I use to get also the empty <code>param2</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>It's right there in the documentation for <a href="https://docs.python.org/3/library/urllib.parse.html#urllib.parse.parse_qs" rel="noreferrer"><code>parse_qs()</code></a>:</p>
<blockquote>
<p>The optional argument <em>keep_blank_values</em> is a flag indicating whether blank values in percent-encoded queries should be treated as blank strings. A true value indicates that blanks should be retained as blank strings. The default false value indicates that blank values are to be ignored and treated as if they were not included.</p>
</blockquote>
<p>In other words, instead of </p>
<pre><code>&gt;&gt;&gt; parse_qs(querystring)
{'param1': ['value1'], 'param3': ['value3']}
</code></pre>
<p>... you need:</p>
<pre><code>&gt;&gt;&gt; parse_qs(querystring, keep_blank_values=True)
{'param2': [''], 'param1': ['value1'], 'param3': ['value3']}
</code></pre>
</div>
