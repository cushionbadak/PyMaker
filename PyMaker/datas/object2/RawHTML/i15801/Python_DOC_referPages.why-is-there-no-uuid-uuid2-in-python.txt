<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3.4/library/uuid.html" rel="noreferrer">There's a <code>uuid.uuid1</code>, <code>uuid.uuid3</code>, <code>uuid.uuid4</code>, and <code>uuid.uuid5</code> but no <code>uuid.uuid2</code>.</a></p>
<p>What happened to <code>uuid2</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>From <a href="https://docs.python.org/3/whatsnew/2.5.html">What's new in Python 2.5</a> in which <code>uuid</code> module was added for the first time:</p>
<blockquote>
<p>(Version 2 UUIDs are not specified in <a href="http://tools.ietf.org/html/rfc4122.html">RFC 4122</a> and are not
  supported by this module.)</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>The UUID2 is a special UUID generated in combination with <a href="http://en.wikipedia.org/wiki/Universally_unique_identifier#Version_2_.28DCE_Security.29">DCE Security</a> server. It's rarely used nowadays. I guess the effort to implement that was in no relation to the request for such a functionality.</p>
</div>
