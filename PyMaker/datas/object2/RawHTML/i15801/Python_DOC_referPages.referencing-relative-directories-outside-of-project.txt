<p>In one of my apps, I want to load some data from another directory on my machine. My Django project is located at <code>C:/Projects/MyProject</code>, my app is located at <code>C:/Projects/MyProject/myapp</code>, and my data directory is located at <code>C:/Data/MyAppData</code>. For various reasons, I do not want to store this data directly in the app's static directory. How can I do this?</p>
<p>Here is what I have tried. In <code>C:/Projects/MyProject/settings.py</code>, I have the following:</p>
<pre><code>import os
DATA_ROOT = `C:/Data`
DATA_DIR = os.path.join(DATA_ROOT, 'MyAppData')
</code></pre>
<p>But how do I now reference <code>DATA_DIR</code> in my views file?</p>
<p>Also, suppose that I want to keep everything relative, and avoid hard-coding <code>C:/Data</code>. Is this possible? Something like the following:</p>
<pre><code>import os
BASE_DIR = os.path.dirname(os.path.dirname(__file__))
DATA_ROOT = BASE_DIR + '../../../Data'
DATA_DIR = os.path.join(DATA_ROOT, 'MyAppData')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>BASE_DIR + '../../../Data'</code> does not contains appropriate separate in between. Use <code>os.path.join</code> there, too.</p>
<p>BTW, <code>os.path.join</code> accepts multiple arguments. So you can write as follow:</p>
<pre><code>import os
BASE_DIR = os.path.dirname(os.path.dirname(__file__))
DATA_ROOT = os.path.join(BASE_DIR, '../../../Data', 'MyAppData')

# To get absolute path
DATA_ROOT = os.path.abspath(os.path.join(BASE_DIR, '../../../Data', 'MyAppData'))
</code></pre>
<p>To access the <code>DATA_ROOT</code> in views, import <code>settings</code> in the view:</p>
<pre><code>from django.conf import settings

# Do something with `settings.DATA_ROOT`
</code></pre>
<p><strong>UPDATE</strong></p>
<p>If you use Python 3.4+, you can  use <a href="https://docs.python.org/3/library/pathlib.html" rel="nofollow"><code>pathlib</code></a>:</p>
<pre><code>DATA_ROOT = pathlib.Path(__file__).resolve().parents[3] / 'Data' / 'MyAppData'
</code></pre>
