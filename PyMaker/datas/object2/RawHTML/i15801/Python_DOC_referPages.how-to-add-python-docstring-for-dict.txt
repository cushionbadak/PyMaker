<div class="post-text" itemprop="text">
<p>I generally use the <a href="http://sphinxcontrib-napoleon.readthedocs.org/en/latest/example_google.html" rel="noreferrer">Google docstring style</a>, so a dictionary parameter would look like:</p>
<pre><code>def func(a_dict):
    """Some function to do something to a dictionary.

    Args:
      a_dict (dict of str: int): Some mapping, I guess?

    """
    ...
</code></pre>
<p>A function that takes <code>**kwargs</code> (note: this is <strong>not</strong> quite the same as having a dictionary parameter), would look like:</p>
<pre><code>def func(**kwargs):
    """Some function to do stuff to arbitrary keyword arguments.

    Args:
      **kwargs: Arbitrary keyword arguments.

    """
    ...
</code></pre>
<p>If there are specific parameters that should be present (e.g. your <code>key1</code>), they should be separate, not rolled into <code>**kwargs</code>.</p>
<hr/>
<p>In Python 3.x, you can also use <a href="http://legacy.python.org/dev/peps/pep-3107/" rel="noreferrer">function annotations</a>:</p>
<pre><code>def func(a_dict: dict):
    """Some function to do something to a dictionary."""
    ...
</code></pre>
<hr/>
<p>From Python 3.5, you can be even more explicit using <a href="https://docs.python.org/3/library/typing.html" rel="noreferrer"><code>typing</code></a>:</p>
<pre><code>from typing import Mapping

def func(a_dict: Mapping[str, int]):
    """Some function to do something to a dictionary."""
    ...
</code></pre>
