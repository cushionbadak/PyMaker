<div class="post-text" itemprop="text">
<p>As Bryan Oakley pointed out, you do not call <code>unshow_me</code>. However, <code>unshow_me</code> is not part of the lambda at all:</p>
<pre><code>&gt;&gt;&gt; def f():pass
... 
&gt;&gt;&gt; def g():pass
... 
&gt;&gt;&gt; lambda x: f(), g()
(&lt;function &lt;lambda&gt; at 0x7f7737a267b8&gt;, None)
&gt;&gt;&gt; lambda x: (f(), g())
&lt;function &lt;lambda&gt; at 0x7f7737a267b8&gt;
</code></pre>
<p>The reason is that the <code>lambda</code> keyword has the <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer">lowest precedence</a> in Python. Adding brackets to the <code>lambda</code> body, which includes the second function in your body, and calling <code>unshow_me</code> with the argument <code>e</code> makes your <code>lambda</code>:</p>
<pre><code>lambda e, text=text:(handle_click(text), unshow_me(e))
</code></pre>
<p>Your complete program is then:</p>
<pre><code>import Tkinter
root = Tkinter.Tk()
def unshow_me(event):
    event.widget.grid_forget()

def handle_click(text):
    print text
    a=text

for r in range(3):
    for c in range(6):
        text = 'R=%s,C=%s'%(r,c)
        label = Tkinter.Label(root, text=text, borderwidth=1 )
        label.grid(row=r,column=c)
        label.bind("&lt;Button-1&gt;", lambda e, text=text:(handle_click(text), unshow_me(e)))

root.mainloop()
</code></pre>
<p>Which produces the desired output (with several clicks):</p>
<p><img alt="enter image description here" src="https://i.stack.imgur.com/RlDmQ.png"/></p>
<p>However, it is worth noting that once hidden, the clicks will <strong>not</strong> be recorded, which is what the OP seems to want. If you want to find where the user clicks, try (modified from <a href="http://effbot.org/tkinterbook/tkinter-events-and-bindings.htm" rel="nofollow noreferrer">effbot</a>):</p>
<pre><code>import Tkinter
root = Tkinter.Tk()
def callback(event):
    print "clicked at", event.x, event.y

root.bind("&lt;Button-1&gt;", callback)
root.mainloop()
</code></pre>
