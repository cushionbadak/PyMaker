<p>I'm hitting a problem I don't seem to be able to solve in my algorith.</p>
<p>Let's say I have:</p>
<pre><code>func1(x)
</code></pre>
<p>which returns a 2 element tuple, and I also have </p>
<pre><code>func2(y,z)
</code></pre>
<p>which I want to use that tuple on. But everytime I try to call it on function2 as</p>
<pre><code>func2(func1)
</code></pre>
<p>I get the error "missing 1 required positional argument:" because the function is receiving the tuple as:</p>
<pre><code>func2((tuple1st_element, tuple2nd_element),) 
</code></pre>
<p>and not</p>
<pre><code>func2(tuple1st_element, tuple2nd_element) 
</code></pre>
<p>How can I get it to do the latter?</p>
</div>
<div class="post-text" itemprop="text">
<p>Use the <a href="https://docs.python.org/3/tutorial/controlflow.html#unpacking-argument-lists" rel="nofollow"><code>*</code> argument unpacking syntax</a>:</p>
<pre><code>func2(*func1())
</code></pre>
<p>Below is a demonstration:</p>
<pre><code>&gt;&gt;&gt; def func1():
...     return 1, 2
...
&gt;&gt;&gt; def func2(a, b):
...     return a + b
...
&gt;&gt;&gt; func2(*func1())
3
&gt;&gt;&gt;
</code></pre>
<p>Or, in simpler terms, doing this:</p>
<pre><code>func(*(1, 2))
</code></pre>
<p>is equivalent to this:</p>
<pre><code>func(1, 2)
</code></pre>
</div>
