<div class="post-text" itemprop="text">
<p>You need to convert the number to <code>str</code> (not <code>int</code>) and iterate it:</p>
<pre><code>def sum_digits(n):
   s = 0
   for i in str(n):
      s += int(i)
   return s
</code></pre>
<p>usage:</p>
<pre><code>&gt;&gt;&gt; sum_digits(123)
6
</code></pre>
<p>Using <a href="https://docs.python.org/3/library/functions.html#sum" rel="nofollow"><code>sum</code></a> and <a href="https://docs.python.org/3/tutorial/classes.html#generator-expressions" rel="nofollow">generator expression</a>:</p>
<pre><code>def sum_digits(n):
    return sum(int(i) for i in str(n))
    # OR  return sum(map(int, str(n)))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The problem here is that your are passing an <code>int</code> to your function, which you cannot iterate over. </p>
<p>I suggest the solution</p>
<pre><code>&gt;&gt;&gt; n = 123
&gt;&gt;&gt; sum(int(x) for x in str(n))
6
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If you want <code>i</code> to range over the <em>digits</em> of <code>n</code>, you'd need something like:</p>
<pre><code>for i in [int(x) for x in str(n)]:
</code></pre>
<p>Or, you could just pass <code>"123"</code> instead of <code>123</code> to <code>sum_digits</code>, which the title of your post suggests is what you actually mean to do.  Of course, then you'd need to convert each character to its numeric value.</p>
</div>
<div class="post-text" itemprop="text">
<p>You cannot iterate over an int but you can still do it without casting to <code>str</code>:</p>
<pre><code>def sum_digits(n):
    n, i = divmod(n, 10)
    while n:
        n, r = divmod(n, 10)
        i += r
    return i
</code></pre>
<p>Demo:</p>
<pre><code>In [2]:  sum_digits(123456789)
Out[2]: 45

In [3]:  sum_digits(123)
Out[3]: 6

In [4]:  sum_digits(100)
Out[4]: 1
</code></pre>
<p>It does not make much sense to convert from int to str then back to int again.</p>
