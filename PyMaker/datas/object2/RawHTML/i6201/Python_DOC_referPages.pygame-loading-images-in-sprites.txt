<div class="post-text" itemprop="text">
<p>First <a href="http://www.pygame.org/docs/ref/image.html#pygame.image.load" rel="nofollow noreferrer">load</a> the image in the global scope or in a separate module and import it. Don't load it in the <code>__init__</code> method, otherwise it has to be read from the hard disk every time you create an instance and that's slow. </p>
<p>Now you can assign the global <code>IMAGE</code> in the class (<code>self.image = IMAGE</code>) and all instances will reference this image. </p>
<pre><code>import pygame as pg


pg.init()
# The screen/display has to be initialized before you can load an image.
screen = pg.display.set_mode((640, 480))

IMAGE = pg.image.load('an_image.png').convert_alpha()


class Player(pg.sprite.Sprite):

    def __init__(self, pos):
        super().__init__()
        self.image = IMAGE
        self.rect = self.image.get_rect(center=pos)
</code></pre>
<p>If you want to use different images for the same class, you can pass them during the instantiation: </p>
<pre><code>class Player(pg.sprite.Sprite):

    def __init__(self, pos, image):
        super().__init__()
        self.image = image
        self.rect = self.image.get_rect(center=pos)


player1 = Player((100, 300), IMAGE1)
player2 = Player((300, 300), IMAGE2)
</code></pre>
<hr/>
<p>Use the <a href="http://www.pygame.org/docs/ref/surface.html#pygame.Surface.convert" rel="nofollow noreferrer"><code>convert</code></a> or <code>convert_alpha</code> (for images with transparency) methods to improve the blit performance.</p>
<hr/>
<p>If the image is in a subdirectory (for example "images"), construct the path with <a href="https://docs.python.org/3/library/os.path.html#os.path.join" rel="nofollow noreferrer"><code>os.path.join</code></a>:</p>
<pre><code>import os.path
import pygame as pg

IMAGE = pg.image.load(os.path.join('images', 'an_image.png')).convert_alpha()
</code></pre>
