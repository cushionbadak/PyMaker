<div class="post-text" itemprop="text">
<p>For the os library what is the difference between</p>
<p>os.listdir('.') vs os.listdir()</p>
<p>They both seem to produce the same results (a list of everything in the active directory) however:</p>
<p><a href="https://www.tutorialspoint.com/python/os_listdir.htm" rel="nofollow noreferrer">https://www.tutorialspoint.com/python/os_listdir.htm</a></p>
<p>says that os.listdir specifically excludes '.' and '..' even if they are present in the directory.  What does that mean?</p>
</div>
<div class="post-text" itemprop="text">
<p>There is no functional difference, see the <a href="https://docs.python.org/3/library/os.html" rel="nofollow noreferrer">docs</a>.  The definition of <code>os.listdir()</code> looks like this</p>
<pre><code>os.listdir(path='.')
</code></pre>
<p>So the default value for path when you call <code>os.listdir()</code> is <code>'.'</code></p>
</div>
<div class="post-text" itemprop="text">
<p>From <code>help os.listdir</code>:</p>
<pre><code>listdir(path=None)
    Return a list containing the names of the files in the directory.

    path can be specified as either str or bytes.  If path is bytes,
      the filenames returned will also be bytes; in all other circumstances
      the filenames returned will be str.
    If path is None, uses the path='.'.
</code></pre>
<p>That is, <code>os.listdir()</code> is the same as <code>os.listdir('.')</code>.</p>
<blockquote>
<p>[...] says that <code>os.listdir</code> specifically excludes '.' and '..' even if they are present in the directory. What does that mean?</p>
</blockquote>
<p>That concerns the returned values.
In UNIX filesystems, every directory has <code>.</code> and <code>..</code> entries,
where <code>.</code> refers to the current directory,
and <code>..</code> to the parent directory.
The documentation says that these entries will not be included in the list returned by <code>os.listdir</code>.</p>
</div>
