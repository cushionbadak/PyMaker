<p>Problem:
I'm trying to store a python dictionary which has tuples as keys, I didn't tried to change structure of dictionary because a value in this dictionary can be another dictionary. I tried using Json but since it only supports string type keys, it failed.</p>
<p>Structure of the dictionary:</p>
<pre><code>dict[(s1,s2)] = f
</code></pre>
<p>where s1,s2 -&gt; strings, f-&gt;float</p>
<p>How do I store this dictionary as a file.</p>
</div>
<div class="post-text" itemprop="text">
<p>If you just need to save the object for later, then you could use pickle. Try this:</p>
<pre><code>import pickle

a = {('a','b') : 1.2}
pickle.dump(a, open('test.pickle','w'))
b = pickle.load(open('test.pickle','r'))

print a
# {('a', 'b'): 1.2}
print b
# {('a', 'b'): 1.2}
</code></pre>
</div>
<span class="comment-copy">Side note: don't name variables after classes, use <code>d</code> or <code>dict_</code> instead of <code>dict</code>.</span>
<span class="comment-copy">Possible duplicate of <a href="https://stackoverflow.com/questions/12734517/json-dumping-a-dict-gives-typeerror-keys-must-be-a-string">Json dumping a dict gives TypeError: keys must be a string</a></span>
<span class="comment-copy">You could just write the string repersentation to a file, and then use <a href="https://docs.python.org/3/library/ast.html#ast.literal_eval" rel="nofollow noreferrer"><code>ast.literal_eval</code></a> to convert the contents of that file back to a python object.</span>
