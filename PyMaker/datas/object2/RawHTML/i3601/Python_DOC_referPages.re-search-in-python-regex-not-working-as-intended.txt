<div class="post-text" itemprop="text">
<p>Your pattern is:</p>
<pre><code>regex_rep = r" repeat "
</code></pre>
<p>This will match the word <code>repeat</code> with a space on each end.</p>
<p>But your lines look like this:</p>
<pre><code>repeat 25&gt; flash_writ .d0000 .d00 1 1 1 0001  0 1 X 1; // wait program time
</code></pre>
<p>There's no space before <code>repeat</code>, so it doesn't match your pattern.</p>
<p>It's hard to suggest how to fix this, because I'm not sure why you put those spaces in the pattern in the first place.</p>
<hr/>
<p>If they're there for no reason, just get rid of them:</p>
<pre><code>regex_rep = r"repeat"
</code></pre>
<p>However, in that case, you're not using any features of <code>re</code> at all, so your test would be better written as:</p>
<pre><code>if "repeat" in eachline:
</code></pre>
<hr/>
<p>If they're there to make the pattern more readable, and you want <code>re</code> to ignore the spaces, you can use the <a href="https://docs.python.org/3/library/re.html#re.VERBOSE" rel="nofollow noreferrer"><code>VERBOSE</code></a> flag to tell it to ignore spaces in your pattern:</p>
<pre><code>if re.search(regex_rep, eachline, flags=re.IGNORECASE|re.VERBOSE) is not None:
</code></pre>
<p>You can see this working <a href="https://regex101.com/r/Q8yUrA/1" rel="nofollow noreferrer">at regex101</a>.</p>
<hr/>
<p>If you want to make sure you're matching <code>repeat</code> as a whole word, rather than as part of a larger word like <code>repeatable</code>, you can use the <code>\b</code> special character, which:</p>
<blockquote>
<p>Matches the empty string, but only at the beginning or end of a word…</p>
</blockquote>
<pre><code>regex_rep = r"\brepeat\b"
</code></pre>
<p>You can see this in action <a href="https://regex101.com/r/OVWNi7/1" rel="nofollow noreferrer">at regex101</a>.</p>

<span class="comment-copy">You're searching for <code>r" repeat "</code>, with spaces on both ends. Your text doesn't have <code>repeat</code> with a space before it anywhere. So there are no matches.</span>
<span class="comment-copy">By the way, you don't need a regex here at all. You could just do <code>if "repeat" in eachline:</code>. Don't drag in <code>re</code> unless you actually need regular expression features.</span>
