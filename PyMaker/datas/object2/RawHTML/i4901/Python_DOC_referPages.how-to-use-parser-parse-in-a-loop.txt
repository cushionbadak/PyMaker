<div class="post-text" itemprop="text">
<p><code>append</code> adds to the end of the array <a href="https://docs.python.org/3/library/array.html?highlight=append#array.array.append" rel="nofollow noreferrer">[0]</a>. Whereas <code>=</code> reassigns the name to the new contents <a href="https://docs.python.org/3/tutorial/introduction.html#numbers" rel="nofollow noreferrer">[1]</a>.</p>
</div>
<span class="comment-copy">In your first code sample you're changing the object that <code>y</code> refers to. Each time <code>y</code> is assigned a reference to a new object. You never save previous values. In your second code sample, your saving each value you parse by <code>append</code>ing them to the list <code>y</code> refers to.</span>
<span class="comment-copy">because you didn't apply <code>.strftime("%H:%M:%S")</code> the second time</span>
