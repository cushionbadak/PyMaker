<div class="post-text" itemprop="text">
<p>Your instructor is right about a tuple being immutable but your "tuple" is actually a list (with square brackets).</p>
<pre><code>&gt;&gt;&gt; tuple1 = [3, 4, 5, 6]
&gt;&gt;&gt; type(tuple1)
&lt;class 'list'&gt;
&gt;&gt;&gt; tuple1 = (3, 4, 5, 6)
&gt;&gt;&gt; type(tuple1)
&lt;class 'tuple'&gt;
&gt;&gt;&gt; tuple1[1] = 44
Traceback (most recent call last):
  File "&lt;input&gt;", line 1, in &lt;module&gt;
TypeError: 'tuple' object does not support item assignment
</code></pre>
<p>The other problem is that you are trying to get an item from the built-in <a href="https://docs.python.org/3/library/functions.html#func-list" rel="nofollow noreferrer"><code>list</code></a> not an actual list.</p>
<pre><code>&gt;&gt;&gt; my_list = [1, 2, 3, 4]
&gt;&gt;&gt; my_list[3] = 66
&gt;&gt;&gt; my_list
[1, 2, 3, 66]
&gt;&gt;&gt; type(list)
&lt;class 'type'&gt;
&gt;&gt;&gt; list[3]
Traceback (most recent call last):
  File "&lt;input&gt;", line 1, in &lt;module&gt;
TypeError: 'type' object is not subscriptable
</code></pre>

<span class="comment-copy">In the example above, <code>my_list</code> has a type of <code>list</code>. It is a <a href="https://docs.python.org/3/library/stdtypes.html#list" rel="nofollow noreferrer">built-in data structure in Python</a> and because of that you shouldn't ever use <code>list</code> as a variable name. You can create a list in a bunch of ways, <code>list('123')</code>, <code>list(range(5))</code>, <code>[x**2 for x in range(5)]</code>, <code>[1, 2, 3]</code>, etc. For more information, you can try looking at the official <a href="https://docs.python.org/3/tutorial/introduction.html#lists" rel="nofollow noreferrer">Python tutorial</a>.</span>
