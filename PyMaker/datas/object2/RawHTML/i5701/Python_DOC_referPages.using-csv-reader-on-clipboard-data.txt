<p>I use csv.reader successfully on a .csv file but if I copy the contents of the same file to the clipboard using Notepad++ and feed that to the csv.reader, I get a different result. Each character on the string gets written to a separate row.</p>
<p>e.g.</p>
<pre><code>import csv
import win32clipboard

win32clipboard.OpenClipboard()
data = win32clipboard.GetClipboardData()
win32clipboard.CloseClipboard()

board = csv.reader(data, dialect='excel')
for row in board:
    print(','.join(row))
</code></pre>
<p>Any ideas? Thank you already.</p>
</div>
<div class="post-text" itemprop="text">
<p>Your problem is that you hand <code>csv.reader</code> a string, but <a href="https://docs.python.org/3/library/csv.html#csv.reader" rel="nofollow noreferrer">it expects some iterable that yields a line for every call of <code>next</code></a>.
A string is iterable, but yields one character each.</p>
<p>The solution is to hand <code>csv.reader</code> a list of lines:</p>
<pre><code>board = csv.reader(data.splitlines(), dialect="excel")
</code></pre>
</div>
