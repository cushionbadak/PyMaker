<div class="post-text" itemprop="text">
<p>The <a href="https://bitbucket.org/dhellmann/virtualenvwrapper/overview" rel="noreferrer">latest version of virtualenvwrapper</a> is tested under Python3.2. Chances are good it will work with Python3.3 too.</p>
</div>
<div class="post-text" itemprop="text">
<p>If you already have python3 installed as well virtualenvwrapper the only thing you would need to do to use python3 with the virtual environment is creating an environment using:</p>
<pre><code>which python3 #Output: /usr/bin/python3
mkvirtualenv --python=/usr/bin/python3 nameOfEnvironment
</code></pre>
<p>Or, (at least on OSX using brew): </p>
<pre><code>mkvirtualenv --python=`which python3` nameOfEnvironment
</code></pre>
<p>Start using the environment and you'll see that as soon as you type python you'll start using python3</p>
</div>
<div class="post-text" itemprop="text">
<p>You can make virtualenvwrapper use a custom Python binary instead of the one virtualenvwrapper is run with. To do that you need to use VIRTUALENV_PYTHON variable which is utilized by virtualenv:</p>
<pre><code>$ export VIRTUALENV_PYTHON=/usr/bin/python3
$ mkvirtualenv -a myproject myenv
Running virtualenv with interpreter /usr/bin/python3
New python executable in myenv/bin/python3
Also creating executable in myenv/bin/python
(myenv)$ python
Python 3.2.3 (default, Oct 19 2012, 19:53:16) 
[GCC 4.7.2] on linux2
Type "help", "copyright", "credits" or "license" for more information.
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>virtualenvwrapper now lets you specify the python executable without the path.</p>
<p>So (on OSX at least)<code>mkvirtualenv --python=python3 nameOfEnvironment</code> will suffice.</p>
</div>
<div class="post-text" itemprop="text">
<p>On Ubuntu; using <code>mkvirtualenv -p python3 env_name</code> loads the virtualenv with python3.</p>
<p>Inside the env, use <code>python --version</code> to verify.   </p>
</div>
<div class="post-text" itemprop="text">
<p>You can add this to your .bash_profile or similar:</p>
<pre><code>alias mkvirtualenv3='mkvirtualenv --python=`which python3`'
</code></pre>
<p>Then use <code>mkvirtualenv3</code> instead of <code>mkvirtualenv</code> when you want to create a python 3 environment.</p>
</div>
<div class="post-text" itemprop="text">
<p>I find that running </p>
<pre><code>export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python3
</code></pre>
<p>and </p>
<pre><code>export VIRTUALENVWRAPPER_VIRTUALENV=/usr/bin/virtualenv-3.4
</code></pre>
<p>in the command line on Ubuntu forces mkvirtualenv to use python3 and virtualenv-3.4. One still has to do </p>
<pre><code>mkvirtualenv --python=/usr/bin/python3 nameOfEnvironment
</code></pre>
<p>to create the environment. This is assuming that you have python3 in /usr/bin/python3 and virtualenv-3.4  in /usr/local/bin/virtualenv-3.4.</p>
</div>
