<p>I was trying to convert a date string to a valid date python format, but I got a erroneous date:</p>
<pre><code>&gt;from datetime import datetime
&gt;mydate = '300413' # is 2013-04-30
&gt;print datetime.strptime(mydate,'%d%m%Y')
&gt;0413-02-01 00:00:00
</code></pre>
<p>How can I parse to a valid date, this date string?</p>
</div>
<div class="post-text" itemprop="text">
<p>If you read <a href="http://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow"><code>strptime()</code> and <code>strftime()</code> Behavior</a> in the docs, <code>%Y</code> means:</p>
<blockquote>
<p>Year with century as a decimal number [0001,9999]</p>
</blockquote>
<p>But you're giving it a 2-digit year, with <em>no</em> century.</p>
<p>That's supposed to raise a <code>ValueError</code>. For example, if you try it with 2.5.6, 2.7.2, 3.2.1, or 3.3.0, you get this:</p>
<blockquote>
<p>ValueError: time data '300413' does not match format '%d%m%Y'</p>
</blockquote>
<p>But, thanks to a bug in older versions of Python, you might get garbage instead.</p>
<p>If you want a 2-digit year, use <code>%y</code>, when means:</p>
<blockquote>
<p>Year without century as a decimal number [00,99].</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>As listed in the <a href="http://docs.python.org/2/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow">documentation</a>, <code>%Y</code> is the code for a four-digit year. You should be using the code for a two-digit year, <code>%y</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p>try:</p>
<pre><code>print datetime.strptime('300413','%d%m%y')
2013-04-30 00:00:00
</code></pre>
<p>only change <code>'%Y'</code> with <code>'%y'</code></p>
