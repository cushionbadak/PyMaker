<div class="post-text" itemprop="text">
<p>It appears to rely on <code>import foo</code> doing a <a href="http://www.python.org/dev/peps/pep-0328/" rel="nofollow">relative import</a>. It was the default behavior in 2.x (from 2.5 onwards there's a <code>__future__</code> directive overriding that). Python 3 <a href="http://docs.python.org/3/whatsnew/3.0.html#removed-syntax" rel="nofollow">removed this</a>. So the maintainers probably didn't even try to be 3.x-compatible, and the installation just worked by chance. Appropriately, there is no claim of Python 3 compatibility: No <code>Programming Language :: Python :: 3</code> trove classifier, no mention in the documentation, no use of 2to3 in <code>setup.py</code>, etc. Short of patching <code>rtree</code> to fix this, and probably numerous other issues, you won't get it to work.</p>
