<p>I'm learning Python with version 3.2 at the moment.</p>
<p>Given two list variables, how do you distinguish if the variables reference the <strong>same list</strong> vs two separate lists with the <strong>same values</strong>.</p>
<p>For example:</p>
<pre><code>&gt;&gt;&gt; foo = [1,2,3,4]
&gt;&gt;&gt; bar = foo
&gt;&gt;&gt; foo.append(5)
&gt;&gt;&gt; foo
[1, 2, 3, 4, 5]
&gt;&gt;&gt; bar
[1, 2, 3, 4, 5]
&gt;&gt;&gt; foo == bar
True
</code></pre>
<p>In the above, "foo" and "bar" are clearly referencing the same list. (as evidenced by appending "5" to foo and seeing that change reflected in bar as well).</p>
<p>Now, let's define a third list, called "other" with the same values:</p>
<pre><code>&gt;&gt;&gt; other = [1,2,3,4,5]
&gt;&gt;&gt; other == foo
True
</code></pre>
<p>They definitely look like the same list given the comparison operator here also returns True.  But if we modify "other", we can see that it is a different list where changes in either variable don't impact the other.</p>
<pre><code>&gt;&gt;&gt; other.append(6)
&gt;&gt;&gt; other == foo
False
&gt;&gt;&gt; other
[1, 2, 3, 4, 5, 6]
&gt;&gt;&gt; foo
[1, 2, 3, 4, 5]
</code></pre>
<p>I think it would be useful to know when two variables are aliases for each other vs. being identical in structure.  But I suspect I might be misunderstanding something else fundamental to the language.</p>

<div class="post-text" itemprop="text">
<p>You can use the <a href="http://docs.python.org/3/reference/expressions.html#not-in" rel="nofollow"><code>is</code> operator</a> to determine object identity:</p>
<pre><code>&gt;&gt;&gt; foo is bar
True
&gt;&gt;&gt; foo is other
False
</code></pre>
<p>To quote the documentation:</p>
<blockquote>
<p>The operators <code>is</code> and <code>is not</code> test for object identity: <code>x is y</code> is true if and only if <code>x</code> and <code>y</code> are the same object. <code>x is not y</code> yields the inverse truth value.</p>
</blockquote>
<p>Another way to detect if two variables refer to the same object (such as a list) is to check te return value of the <a href="http://docs.python.org/3/library/functions.html#id" rel="nofollow"><code>id()</code> function</a>:</p>
<pre><code>&gt;&gt;&gt; id(foo)
4432316608
&gt;&gt;&gt; id(bar)
4432316608
&gt;&gt;&gt; id(other)
4432420232
</code></pre>
