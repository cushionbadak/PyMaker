<div class="post-text" itemprop="text">
<p>I want to write a program in Python but I cant find any help on where to start. </p>
<p>All I want to do first is make Python open a simple window when I run the program. I want it to be green and about 500 pixels square. How would I do this?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can do this using <a href="https://docs.python.org/2/library/turtle.html" rel="nofollow"><code>turtle</code></a>, a builtin Python module made for drawing.</p>
<pre><code>import turtle
screen = turtle.Screen()
screen.setup(500, 500)
screen.bgcolor('green')
</code></pre>
<p>First you initialize the screen using <a href="https://docs.python.org/2/library/turtle.html#turtle.Screen" rel="nofollow"><code>turtle.Screen()</code></a>. Then you use <a href="https://docs.python.org/2/library/turtle.html#turtle.setup" rel="nofollow"><code>screen.setup()</code></a> and pass in the screen size you want. Lastly, you use <a href="https://docs.python.org/2/library/turtle.html#turtle.bgcolor" rel="nofollow"><code>screen.bgcolor()</code></a> to change the background color.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>tkinter</code></p>
<pre><code>from tkinter import Tk, Frame, BOTH

class Example(Frame):

    def __init__(self, parent):
        Frame.__init__(self, parent, background="green")   

        self.parent = parent

        self.parent.title("Window")
        self.pack(fill=BOTH, expand=1)

root = Tk()
root.geometry("500x500")
app = Example(root)
root.mainloop() 
</code></pre>
</div>
<span class="comment-copy">You want the window to be green or the content?</span>
<span class="comment-copy">You need to pick a windowing library. Python comes with <code>Tkinter</code>, and <code>turtle</code> (which is based on it); there are others like GObject and PySide (see <a href="https://docs.python.org/3/library/othergui.html#other-gui-packages" rel="nofollow noreferrer">Other Graphical User Interface Packages</a> in the docs). Or you can use a lower-level library like PyGame (which doesn't handle widgets, just plain windows that you have to draw on manually).</span>
