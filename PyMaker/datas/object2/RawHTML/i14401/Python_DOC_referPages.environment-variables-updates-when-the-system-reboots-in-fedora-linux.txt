<div class="post-text" itemprop="text">
<p>The recommended solution is to use <a href="https://pypi.python.org/pypi/virtualenv" rel="nofollow">virtualenv</a> and even better with <a href="https://pypi.python.org/pypi/virtualenvwrapper-win" rel="nofollow">virtualenvwrapper</a></p>
<p><strong>virtualenv/virtualenvwrapper Quick Start:</strong></p>
<pre><code>pip install virtualenvwrapper
source $(which virtualenvwrapper.sh)

mkvirtualenv foo

workon foo
</code></pre>
<p><strong>Note:</strong> For Python 3.x users; Python 3.x now comes with <a href="https://docs.python.org/3/library/venv.html" rel="nofollow">venv</a></p>
<p>However putting an environment variable in your <code>$HOME/.bash_profile</code> typically only gets executed/evaluated once per login session (<em>you don't really need to rebot as such</em>).</p>
<p>Typically in Bash this means:</p>
<pre><code>bash -l  # enter a new login session
</code></pre>
<p>However if you want the environment variable everywhere without having to start a new "login session" you should put it in <code>$HOME/.bashrc</code>.</p>
<p><strong>Example:</strong></p>
<pre><code>echo -e "export PYTHONPATH=\$HOME/lib/python\n" &gt;&gt; $HOME/.bashrc
</code></pre>
