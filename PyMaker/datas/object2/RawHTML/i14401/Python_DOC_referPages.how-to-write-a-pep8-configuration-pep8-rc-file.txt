<div class="post-text" itemprop="text">
<p>The preferred way is to use a <code>setup.cfg</code> in the top-level of the project (.cfg has the same syntax as a <a href="https://docs.python.org/3/library/configparser.html#supported-ini-file-structure" rel="noreferrer">.ini file</a>), which should contain a <code>[pep8]</code> section. For example:</p>
<pre><code>[pep8]
ignore = E226,E302,E41
max-line-length = 160
</code></pre>
<p><em>Note: the error codes are defined in the <a href="http://pep8.readthedocs.org/en/latest/intro.html#error-codes" rel="noreferrer">pep8 docs</a>.</em></p>
<hr/>
<ul>
<li>autopep8 find this same <code>[pep8]</code> section as pep8.</li>
<li>flake8 needs a <a href="http://flake8.readthedocs.org/en/latest/config.html" rel="noreferrer"><code>[flake8]</code> section</a> in setup.cfg.</li>
<li>yapf looks for a <a href="https://github.com/google/yapf#formatting-style" rel="noreferrer"><code>[yapf]</code> section</a> in setup.cfg.</li>
</ul>
</div>
<div class="post-text" itemprop="text">
<p>Sadly, the answer from Andy Hayden does not work for <code>pytest</code> / <code>pytest-pep8</code> / <code>flake8</code>.</p>
<h2>pytest-pep8</h2>
<p>For that, you have to use either</p>
<pre><code># content of setup.cfg
[pytest]
pep8maxlinelength = 99
</code></pre>
<p>or</p>
<pre><code>[pytest]
max-line-length=99
</code></pre>
<p>Strangely, <strong>the following does not work</strong></p>
<pre><code>[tool:pytest]
max-line-length=99
</code></pre>
<h2>pytest-flake8</h2>
<p>Add</p>
<pre><code> [flake8]
 max-line-length=99
</code></pre>
</div>
