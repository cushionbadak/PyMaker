<p>Yes it is called blender. Using python you can tell blender to <a href="https://www.blender.org/api/blender_python_api_current/bpy.ops.render.html#bpy.ops.render.render" rel="nofollow noreferrer">render a specific frame</a>, you can also tell it to <a href="https://www.blender.org/api/blender_python_api_current/bpy.ops.render.html#bpy.ops.render.opengl" rel="nofollow noreferrer">render a viewport preview</a> using opengl. A python script can be generated on the fly and given as a <a href="https://www.blender.org/manual/advanced/command_line/arguments.html" rel="nofollow noreferrer">cli argument</a> for blender to run without using a GUI, <a href="https://blender.stackexchange.com/a/8405/935">this answer</a> provides an example of passing arguments from the cli to be used within the script inside blender, the <a href="https://docs.python.org/3/library/subprocess.html" rel="nofollow noreferrer">subprocess module</a> can be used as a cli equivalent. There is also an option to <a href="https://wiki.blender.org/index.php/User:Ideasman42/BlenderAsPyModule" rel="nofollow noreferrer">build blender as a python module</a> that can be imported in any python interpreter.</p>
<p>To manually render a live 3D view of your model within your app you could use one of several modules that provides opengl support in python to draw the surface, such as <a href="https://pypi.python.org/pypi/PyOpenGL" rel="nofollow noreferrer">PyOpenGL</a> or <a href="https://pypi.python.org/pypi/pyglet" rel="nofollow noreferrer">pyglet</a></p>
