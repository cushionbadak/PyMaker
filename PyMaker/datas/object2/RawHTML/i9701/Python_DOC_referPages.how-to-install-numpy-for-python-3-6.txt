<p>As long as binary packages (so-called 'wheels') for 3.6 have not been released to PyPi yet, you can resort to unofficial (but working) ones available at <a href="http://www.lfd.uci.edu/~gohlke/pythonlibs/" rel="noreferrer">http://www.lfd.uci.edu/~gohlke/pythonlibs/</a>. Download the file and install it like this:</p>
<pre><code>pip install C:\path\to\numpy‑1.11.3+mkl‑cp36‑cp36m‑win_amd64.whl
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Pip itself is not part of Python.  However, since version 3.4 Python includes the <a href="https://docs.python.org/3/library/ensurepip.html" rel="nofollow noreferrer"><code>ensurepip</code> module</a> that allows bootstrapping a pip installation.  After calling</p>
<pre><code>python3.6 -m ensurepip
</code></pre>
<p>you should be able to install numpy via pip.</p>
</div>
<div class="post-text" itemprop="text">
<p>Pip is always be installed in subdirectory <code>\Scripts</code> of the Python 3.6 install directory.</p>
<p>Open command prompt in windows, change directory to <code>\Scripts</code> and input <code>pip list</code>, you will see what has been installed, then input <code>pip install numpy</code>, and the numpy will be installed. </p>
</div>

<p>Here is the list of steps needed to be performed for installing numpy on windows. This worked for me.</p>
<ol>
<li>Set path variable for python- C:\Users\XXXXXXX\AppData\Local\Programs\Python\Python36-32</li>
<li>Download get-pip.py locally</li>
<li>Open cmd and navigate to get-pip.py file</li>
<li>run python get-pip.py for installation of PIP</li>
<li>set pip path in env variable path -  C:\Users\XXXXXXX\AppData\Local\Programs\Python\Python36-32\Scripts;</li>
<li>pip install numpy</li>
</ol>
