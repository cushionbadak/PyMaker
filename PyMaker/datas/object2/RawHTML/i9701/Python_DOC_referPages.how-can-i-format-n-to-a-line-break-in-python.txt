<div class="post-text" itemprop="text">
<p>As the documentation for <a href="https://docs.python.org/3/library/subprocess.html#subprocess.check_output" rel="nofollow noreferrer"><code>subprocess.check_output</code></a> explains:</p>
<blockquote>
<p>By default, this function will return the data as encoded bytes. The actual encoding of the output data may depend on the command being invoked, so the decoding to text will often need to be handled at the application level.</p>
</blockquote>
<p>So what you get back is a <a href="https://docs.python.org/3/library/stdtypes.html#bytes" rel="nofollow noreferrer">bytes</a> object that contains the output of your command as encoded bytes. You can see this when you print the object; bytes literals have a <code>b</code> in front of the quotes both when printed and when the <code>repr</code> is printed:</p>
<pre><code>&gt;&gt;&gt; x = b'foo bar'
&gt;&gt;&gt; print(x)
b'foo bar'
&gt;&gt;&gt; x        # basically the same as print(repr(x))
b'foo bar'
</code></pre>
<p>In order to get a proper string out of this, you need to <em>decode</em> the bytes object using <a href="https://docs.python.org/3/library/stdtypes.html#bytes.decode" rel="nofollow noreferrer"><code>bytes.decode()</code></a>. Note that in order to decode bytes into a string, you need to know what encoding the data is encoded as. Very commonly, this will be utf-8 in which case you do not need to pass any argument to it:</p>
<pre><code>&gt;&gt;&gt; x.decode()
'foo bar'
&gt;&gt;&gt; print(x.decode())
foo bar
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You need not to worry about it. <code>\n</code> in string means new line in the string. If you will print the content of <code>check_output()</code>, you will see the <code>\n</code> replaced with new line in the console. For example:</p>
<pre><code>&gt;&gt;&gt; my_text = "1234\n\n1234"
&gt;&gt;&gt; my_text
'1234\n\n1234'
#    ^ value of `my_text` holds `\n`

# on printing the `my_text`, `\n` is replaced with new line
&gt;&gt;&gt; print(my_text)
1234

1234
</code></pre>
<hr/>
<p>In your case, you should do it like:</p>
<pre><code>my_output = subprocess.check_output(...)
print(my_output)
</code></pre>
