<p>For new versions of Python, you can create a new dictionary with <a href="https://docs.python.org/3/library/functions.html#func-dict" rel="nofollow noreferrer"><code>dict()</code></a> + <a href="https://docs.python.org/3/library/functions.html#zip" rel="nofollow noreferrer"><code>zip()</code></a>:</p>
<pre><code>dict(zip(dic, mylist))
# {'Bag': 0.2577, 'Door': 0.2311, 'Duck': 0.6393, 'Car': 0.159, 'Girl': 0.6135}
</code></pre>
<p>Or update the dictionary in place:</p>
<pre><code>for k, v in zip(dic, mylist):
    dic[k] = v

print(dic)
# {'Bag': 0.2577, 'Door': 0.2311, 'Duck': 0.6393, 'Car': 0.159, 'Girl': 0.6135}
</code></pre>
<p>But this assumes ordered dictionaries for <strong>Python3.6+</strong>. If you are using lower versions which have <em>unordered</em> dictionaries, you can use <a href="https://docs.python.org/3/library/collections.html#collections.OrderedDict" rel="nofollow noreferrer"><code>collections.OrderedDict</code></a> to remember the order in which keys are inserted. </p>
<p><strong>OrderedDict Example:</strong></p>
<pre><code>from collections import OrderedDict

mylist = [0.2577, 0.2311, 0.6393, 0.159, 0.6135]

items = ['Bag', 'Door', 'Duck', 'Car', 'Girl']

d = OrderedDict(zip(items, mylist))

print(d)
# OrderedDict([('Bag', 0.2577), ('Door', 0.2311), ('Duck', 0.6393), ('Car', 0.159), ('Girl', 0.6135)])
</code></pre>
<p>You can also combine the above examples to only use lists, which are <em>ordered</em>:</p>
<pre><code>mylist = [0.2577, 0.2311, 0.6393, 0.159, 0.6135]

items = ['Bag', 'Door', 'Duck', 'Car', 'Girl']

d = dict(zip(items, mylist))

print(d)
# {'Bag': 0.2577, 'Door': 0.2311, 'Duck': 0.6393, 'Car': 0.159, 'Girl': 0.6135}
</code></pre>
<p>and is the most straightforward solution to your problem, since it creates a dictionary directly. </p>
