Now what I'm trying to achieve is rather than asking for the colour in a different line and the status of the led in another line, is to combine them two. So for example I'd write on the console:</p>

<p>Use <a href="https://docs.python.org/3/library/stdtypes.html#str.split" rel="nofollow noreferrer">string.split()</a>:</p>
<pre><code>while True: 
    what = input("Color [r,g,y] and state [on,off] (ex.: 'r on'): ").strip().lower().split()
    if len(what)==2:
        leds_col,leds_stat = what
        # sort the color input to reduce possible values
        leds_col = ''.join(sorted(leds_col))
        if leds_col not in "r g y gr ry gy gry" or leds_stat not in "on off":
            continue
    else:
        continue
    led(leds_col, leds_stat)
</code></pre>
<p>If invalid input is given, this will <code>continue</code> to ask until inputs are valid.
See <a href="https://stackoverflow.com/questions/23294658/asking-the-user-for-input-until-they-give-a-valid-response">Asking the user for input until they give a valid response</a> for more ideas on input validation.</p>

<p>Unrelated - but you can optimize your <code>led</code>-function:</p>
<pre><code>def led(color,state):
    d = {"on":GPIO.HIGH, "off":GPIO.LOW,
         "g":27, "y":22, "r":17}

    for c in color:
        GPIO.output(d[c],d[state])
    print("LED",color,state)
</code></pre>
<p>by using a lookup dictionary: see <a href="https://docs.python.org/3/library/stdtypes.html#mapping-types-dict" rel="nofollow noreferrer">dict()</a></p>

<p>This approach might help. By taking two inputs with a space in between. You can split the input based on a space or any other character. It will then place the color and state into two different variables.</p>
<pre><code>&gt;&gt;&gt; leds_col,leds_stat = input('Color(r,g,y) and State(on,off):').split(' ')

&gt;&gt;&gt; print(leds_stat)

<p>Alternatively, you can also use comma or any other character as a separator between the color and state.</p>
