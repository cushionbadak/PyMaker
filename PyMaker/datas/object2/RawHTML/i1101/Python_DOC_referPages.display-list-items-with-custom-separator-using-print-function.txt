<p>Can someone explain to me why when I'm trying to pass unpacked data into print function using asterisk, the optional argument "end" is applied only for the last list's element, and for the rest is default (space)</p>
<pre><code>l = ['a', 'b', 'c']
print(*l, end='-')
</code></pre>
<p>I expected <code>a-b-c-</code> instead of <code>a b c-</code></p>
</div>
<div class="post-text" itemprop="text">
<p>According to the <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow noreferrer"><code>print()</code> docs</a>, </p>
<blockquote>
<p>Print objects to the text stream file, separated by <code>sep</code> and followed
  by <code>end</code>. </p>
<p>All non-keyword arguments are converted to strings like <code>str()</code> does and
  written to the stream, separated by <code>sep</code> and followed by <code>end</code>.</p>
</blockquote>
<p>The <code>sep</code> argument is used to control the separator between arguments to <code>print</code>. <code>end</code> just controls the line terminator.</p>
<pre><code>l = ['a', 'b', 'c']
print(*l, sep='-', end='-')
# a-b-c-
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>In addition to previous answer, you may find the following useful as well:</p>
<pre><code>&gt;&gt;&gt; l = ['a', 'b', 'c']
&gt;&gt;&gt; print('-'.join(l) + '-')
a-b-c-
</code></pre>
</div>
<span class="comment-copy">As the name implies the <code>end</code> character is placed at the end of what you print, the very end.</span>
<span class="comment-copy">That's pretty much what <code>print</code> does internally for convenience.</span>
