<p>From Python's documentation (<a href="https://docs.python.org/3/library/sqlite3.html#sqlite3.Cursor.execute" rel="nofollow noreferrer">https://docs.python.org/3/library/sqlite3.html#sqlite3.Cursor.execute</a>), I should be able to execute SQL statements using parameterized placeholders. Yet, the code below doesn't work.</p>
<pre><code>import sqlite3

conn = sqlite3.connect("temp.db")
c = conn.cursor()

c.execute("create table ? (foo text, bar text)", ("table_name",))

conn.commit()
conn.close()
</code></pre>
<p>I'm getting an error:</p>
<pre><code>Traceback (most recent call last):
  File "main.py", line 6, in &lt;module&gt;
    c.execute("create table ? (foo text, bar text)", ("table_name",))
sqlite3.OperationalError: near "?": syntax error
</code></pre>
<p>But if I switch out of using parameterized placeholders, it works. </p>

<div class="post-text" itemprop="text">
<p>In a word - no. You can only parameterize values, not object names (in this case, the table's name). If you want to do something like this, you'd have to resort to string manipulation, e.g.:</p>
<pre><code>c.execute("create table %s (foo text, bar text)" % ("table_name"))
</code></pre>
