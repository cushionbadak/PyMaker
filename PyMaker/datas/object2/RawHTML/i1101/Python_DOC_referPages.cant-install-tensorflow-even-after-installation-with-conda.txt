<p>I think you just use another version of Python so you can use this command to get the path of tensorflow, for some platform the default <code>pip</code> is for Python2, so maybe you will get:</p>
<pre><code>C:\Users\Nick&gt;pip show six
Name: six
Version: 1.11.0
Summary: Python 2 and 3 compatibility utilities
Home-page: http://pypi.python.org/pypi/six/
Author: Benjamin Peterson
Author-email: benjamin@python.org
License: MIT
Location: c:\python27\lib\site-packages
Requires:
</code></pre>
<p>Also from the doc <a href="https://docs.python.org/3/installing/index.html?highlight=pip#work-with-multiple-versions-of-python-installed-in-parallel" rel="nofollow noreferrer">Work with multiple versions of Python installed in parallel</a> .</p>
<p>You can try this:</p>
<p>On Linux, Mac OS X, and other POSIX systems, use the versioned Python commands in combination with the -m switch to run the appropriate copy of pip:</p>
<pre><code>python2   -m pip install SomePackage  # default Python 2
python2.7 -m pip install SomePackage  # specifically Python 2.7
python3   -m pip install SomePackage  # default Python 3
python3.4 -m pip install SomePackage  # specifically Python 3.4
</code></pre>
<p>Appropriately versioned pip commands may also be available.</p>
<p>On Windows, use the py Python launcher in combination with the -m switch:</p>
<pre><code>py -2   -m pip install SomePackage  # default Python 2
py -2.7 -m pip install SomePackage  # specifically Python 2.7
py -3   -m pip install SomePackage  # default Python 3
py -3.4 -m pip install SomePackage  # specifically Python 3.4
</code></pre>
<p>If you have a broken <code>pip</code> on Anaconda, you can follow these steps:</p>
<ol>
<li>Find pip folder under "../Anaconda3/Lib/site-packages/", delete it manually.</li>
<li>In Anacoda prompt do : <code>conda uninstall pip</code></li>
<li>Install pip using your familiar way. E.g: <code>conda install -c anaconda pip</code></li>
</ol>
<p>Hope this helps.</p>
