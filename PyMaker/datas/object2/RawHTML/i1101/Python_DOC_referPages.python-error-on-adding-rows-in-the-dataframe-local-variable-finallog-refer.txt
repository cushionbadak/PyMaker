<p>I suppose, you declare <code>finallog</code> somewhere above your code, but in other scope, but assign to it in a "current" local scope (may be somewhere below, because i don't see any assignment in your code):</p>
<pre><code>finallog = pd.concat([finallog, df_2],axis=0)
...
&gt; UnboundLocalError: local variable 'finallog' referenced before assignment
</code></pre>
<p>As the <a href="https://docs.python.org/3/faq/programming.html#why-am-i-getting-an-unboundlocalerror-when-the-variable-has-a-value" rel="nofollow noreferrer">docs</a> advices, try to use <code>global</code>. Simplified example:</p>
<pre><code>x = 10

def p():
    print(x)
    x += 1

p()
Traceback:
UnboundLocalError                         Traceback (most recent call last)
&lt;ipython-input-3-d40139f363ae&gt; in &lt;module&gt;
----&gt; 1 p()

&lt;ipython-input-2-271014bcda23&gt; in p()
      1 def p():
----&gt; 2     print(x)
      3     x += 1
      4
</code></pre>
<p>With <code>global</code>:</p>
<pre><code>x = 10

def p():
    global x
    print(x)
    x += 10

p()
Out: 
10
</code></pre>
