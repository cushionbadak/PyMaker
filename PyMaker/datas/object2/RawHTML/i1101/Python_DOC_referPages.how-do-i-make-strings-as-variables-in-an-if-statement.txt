<p>I'm making a text based game in python and want to check how many trees are in the field that i'm in.</p>
<p>I'm running the latest Python (3.7.1) (Not using IDLE it is just named Python3.7.1.exe and it looks like CMD)
I'm trying to use variables like this but it is saying that I can't compare strings to integers (but how do I turn strings into "variable text"?).</p>
<pre><code>trees_A1 = random.randint(…)
trees_A2 = random.randint(…)
…
Field = A3
if "trees_"+Field &gt; 0
</code></pre>
<p>I expected it to compare trees_A3 but just sees the string trees_A3.</p>

<div class="post-text" itemprop="text">
<p>Use a <a href="https://docs.python.org/3/tutorial/datastructures.html#dictionaries" rel="nofollow noreferrer">dictionary</a>!</p>
<pre><code>trees = dict()
trees["A1"] = random.randint(…)
trees["A2"] = random.randint(…)
…
Field = "A3"
if trees[Field] &gt; 0:
    …
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>globals()</code>:</p>
<pre><code>if globals()["trees_"+Field] &gt; 0:
</code></pre>
<p>Or use a dictionary.</p>
</div>
<div class="post-text" itemprop="text">
<p>You are looking for <code>eval()</code> operator </p>
<pre><code>a = 5
eval("a")
</code></pre>
<p>this will result </p>
<blockquote>
<p>5</p>
</blockquote>
<p>so you can evaluate strings with <code>eval()</code>. However you should not do that. Since eval is like an evil operator. They will cause security issues in your program. You should use dictionaries for your purpose.</p>
<pre><code>trees = {}
trees["A1"] = random.randint(…)
</code></pre>
<p>then you should call <code>trees["A1"]</code> this way you will be write more pythonic and more proper code.</p>
