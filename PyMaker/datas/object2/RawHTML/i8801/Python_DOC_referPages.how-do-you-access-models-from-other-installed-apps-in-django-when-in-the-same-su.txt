<div class="post-text" itemprop="text">
<p><code>from .data.models</code> is an import <em>relative</em> to the current directory where that file exists, e.g., it would be looking for a <code>data</code> module directory inside your <code>dashboard</code> directory. If you want to refer to the <code>data</code> module relative to your <code>dashboard</code> module you should be able to use </p>
<pre><code>from ..data.models import Foo
</code></pre>
<p>Using absolute imports depends on your <a href="https://docs.python.org/3/using/cmdline.html#envvar-PYTHONPATH" rel="nofollow noreferrer"><code>PYTHONPATH</code></a> environment variable. You can play around with it by adjusting <code>sys.path</code> in a Python REPL.</p>
</div>

<p>From the top of my head I can advise a few things:</p>
<ol>
<li>Check your <code>__init__.py</code> files if they are not empty you'll have
access to only whats imported in them.</li>
<li>Check if you are not running into circular imports - if in <code>dashboard.models</code> you import from <code>data.models</code> when it imports from <code>dashboard.models</code>.</li>
<li><code>from apps.data import models</code> or <code>from apps.data.models import ModelName</code> seems the right way to go - knowing this look for solutions.</li>
</ol>
