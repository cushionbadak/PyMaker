<div class="post-text" itemprop="text">
<p>This <strong>is</strong> a timezone issue.  <code>time.strptime</code>, which relies on posix <code>strptime</code> returns a local time in all cases.  </p>
<p>Please read the documentation of <a href="http://man7.org/linux/man-pages/man3/strptime.3.html" rel="nofollow noreferrer"><code>strptime</code></a> carefully, you will note that while <code>%Z</code> is supported as a format character "fields are parsed, but <em>no field in <code>tm</code> is changed</em>"</p>
<p>Generally speaking, a posix implementation of <code>strptime</code> that does support <code>%Z</code> will only support the values of <code>UTC</code>, <code>GMT</code> or the systems <code>tzname</code> (available in python from <code>time.tzname</code>).  But <em>it will not alter the return value based upon them</em>.</p>
