<p>How can non-iterable kwargs be included in the <code>map</code> function of <code>concurrent.futures</code>? For example, in the code below, how can <code>c</code> be called as False for all function calls of <code>single</code>? That is, <code>single(iterable_a[i], iterable_b[i], c=False)</code> where <code>i</code> is an int between 0 and 3.</p>
<pre><code>import concurrent.futures

def single(a, b, c=True):
    return (a, b, c)

if __name__ == '__main__':
    iterable_a = range(4)
    iterable_b = range(4,8)
    with concurrent.futures.ProcessPoolExecutor() as executor:
        results = executor.map(single, iterable_a, iterable_b)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can use a <a href="https://docs.python.org/3/library/functools.html#functools.partial" rel="nofollow noreferrer"><code>functools.partial</code></a> for that:</p>
<pre><code>from functools import partial

single2 = partial(single,c=False)
</code></pre>
<p>will generate a function <code>single2</code> where <code>c</code> is set to <code>False</code>. So you can use:</p>
<pre><code><b>from functools import partial</b>

results = executor.map(<b>partial(single,c=False)</b>, iterable_a, iterable_b)</code></pre>
<p>So your <code>map</code> calls a function generated from <code>partial(single,c=False)</code> and you <code>map</code> with that function.</p>
