<p>I'm trying to get the coordinates from a scanned QR-code using the pyzbar module. If I use the following code below to do this:</p>
<pre><code>    test = decode(img)
    print(test)
</code></pre>
<p>I get the following result:</p>
<p>[Decoded(data=b'Part4', type='QRCODE', rect=Rect(left=172, top=332, width=75, height=76))]</p>
<p>The problem however, is that I do not know how to just extract the rect part of this string. I'm using spyder, and if I go to the variable explorer to try and see how the variable is built-up, I get the following error:</p>
<p>AttributeError("Can't get attribute 'Rect' on ",)</p>
<p>Anyone know what could be it?</p>

<div class="post-text" itemprop="text">
<p>As you can see in <a href="https://github.com/NaturalHistoryMuseum/pyzbar/blob/8e3e87da17ddd794842b2fcbbe0dd86c692d7511/pyzbar/pyzbar.py#L26" rel="nofollow noreferrer">the source code</a>, <code>Decoded</code> is a <a href="https://docs.python.org/3/library/collections.html#collections.namedtuple" rel="nofollow noreferrer">namedtuple</a>. So your <code>test</code> is a list of namedtuples. You can get the rect of the first result like this:</p>
<pre><code>rect = test[0].rect
</code></pre>
</div>
