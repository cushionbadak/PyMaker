<div class="post-text" itemprop="text">
<p>I am importing data from a csv file; and trying to determine a user type. In specific, I am using the python module collections.</p>
<p>My data set has blank fields thus when I execute the following script <code>type_of_users=user_data.most_common()</code> I get the following result: </p>
<p>The type of users are :</p>
<pre><code>[('Subscriber', 269149), ('Customer', 30159), ('', 692)]
</code></pre>
<p>Is there a way to update the <code>('',692)</code> with <code>('Unknown User Type',692)</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>You could use a list comprehension on the results of <code>.most_common()</code> after reading the file:</p>
<pre><code>&gt;&gt;&gt; type_of_users = [('Subscriber', 269149), ('Customer', 30159), ('', 692)]

&gt;&gt;&gt; type_of_users = [(i, j) if i else ('Unknown User Type', j)
...                  for i, j in type_of_users]

&gt;&gt;&gt; type_of_users
[('Subscriber', 269149), ('Customer', 30159), ('Unknown User Type', 692)]
</code></pre>
<p>This takes advantage of the <a href="https://docs.python.org/3/library/stdtypes.html#truth-value-testing" rel="nofollow noreferrer">property</a> that empty sequences ('') are considered False while non-empty strings are considered True.</p>
<p>Note that there are a number of other objects that will also evaluate in this way, so if you need to be more explicit, you should use <code>if i != ''</code>.</p>
</div>
<span class="comment-copy">In case someone is wondering how to apply this to non-empty strings, this case could be re-written as <code>type_of_users = [(i, j) if i == '' else ('Unknown User Type', j) for i, j in type_of_users]</code>. You could swap the condition to whatever you need.</span>
