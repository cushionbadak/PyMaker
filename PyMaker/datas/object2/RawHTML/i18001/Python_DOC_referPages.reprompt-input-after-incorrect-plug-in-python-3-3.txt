<div class="post-text" itemprop="text">
<p>How can one reprompt this this input if the input is anything other than 1 or 2?</p>
<pre><code>status = eval(input("Are they single(1) or married(2)? \n"))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Use a loop:</p>
<pre><code>while True:
    status = input("Are they single(1) or married(2)? \n")
    if status in ('1', '2'):
        break
</code></pre>
<p>BTW, don't use <code>eval</code>. It can evaluate arbitrary expression. If you want convert the input string to int, use <a href="http://docs.python.org/3/library/functions.html#int" rel="nofollow"><code>int</code></a> instead.</p>
</div>
<div class="post-text" itemprop="text">
<p>@falsetru, has answered with my preferred way already. Here is a couple of alternatives you may see elsewhere</p>
<pre><code>status = input("Are they single(1) or married(2)? \n")
while status not in ('1', '2'):
    status = input("Are they single(1) or married(2)? \n")
</code></pre>
<p>This is not desirable due to the duplication of the prompt</p>
<pre><code>status = -1 # or 0 or None or some other invalid value
while status not in ('1', '2'):
    status = input("Are they single(1) or married(2)? \n")
</code></pre>
<p>This is just ugly in my opinion, but some people dislike using <code>break</code></p>
<p>One reason I prefer @falsetru's version is that it's easy to convert for a <code>for</code> loop when you want to count/limit the number of retries.</p>
</div>
<span class="comment-copy">In this case, converting to <code>int</code> means you'll possibly raise an exception. Better to just leave as <code>str</code></span>
