<p>Here's my code:</p>
<pre><code>try:
    entry_date = entry.updated_date
except AttributeError:
    try:
        entry_date = entry.published_date
    except AttributeError:
        entry_date = manual_parse_from_string(entry)
</code></pre>
<p>In this scenario I'm parsing a feed and trying to get the date. RSS comes with <code>updated_date</code>, <code>published_date</code> or date in string format, which need further processing into the DateTime object. I have a feeling it can be done in an other way, such as in this pseudo-code:</p>
<pre><code>entry_date = entry.updated_date
or
entry.published_date or manual_parse_from_string(entry)
</code></pre>
<p>None of those seems right to me. What would be the best way to do it?</p>

<div class="post-text" itemprop="text">
<p>In this particular case, you can use <a href="https://docs.python.org/3/library/functions.html#getattr" rel="nofollow noreferrer">getattr</a> with a <code>None</code> default value:</p>
<pre><code>entry_date = (getattr(entry, 'updated_date', None)
              or getattr(entry, 'published_date', None)
              or manual_parse_from_string(entry))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Homogenise your different getters into uniform callables and iterate over them:</p>
<pre><code>from operator import attrgetter

getters = attrgetter('updated_date'), attrgetter('published_date'), manual_parse_from_string

for getter in getters:
    try:
        entry_date = getter(entry)
    except AttibuteError:
        pass
</code></pre>
