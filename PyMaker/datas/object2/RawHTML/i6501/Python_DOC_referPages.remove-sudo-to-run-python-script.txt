<div class="post-text" itemprop="text">
<p><code>ImportError: No module named 'matplotlib'</code> happens when your Python doesn't find the module. <code>sudo</code> changes the <strong>enviornment</strong> variables; That's why.</p>
<p>To fix this, locate where <code>matplotlib</code> is installed in your computer, and verify the folder is part of your <code>sys.path</code>.</p>
<pre><code>import sys
sys.path
['C:\\Python27\\tests', ..., ...]
</code></pre>
<p>Then you've two options: insert that path from your script, i.e adding a line such <code>import sys; sys.path.append(&lt;folder&gt;)</code> or configure <code>PYTHONPATH</code> env variable under your user appending the folder to the path.</p>
<p><code>PYTHONPATH</code> env variable gets loaded to <code>sys.path</code> on startup.</p>
</div>
<div class="post-text" itemprop="text">
<p>Best solution to me is a general workflow for all projects: use virtualenviroment]<a href="https://docs.python.org/3/tutorial/venv.html" rel="nofollow noreferrer">1</a>.</p>
<pre><code>sudo pip3 install virtualenv
virtualenv myenv
source mynenv/bin/activate
</code></pre>
<p>Then you should install your libraries again with pip and they will be installed in your virtualenviroment, isolated from everything else. </p>
