<p>Are you running python2 or python3? Additionally, what are using for your time metric?</p>
<p>Your bug may be the result of <code>input()</code> having different functionality given the python version you are running</p>
<p><code>input()</code> in python2 takes in a string and runs <code>eval()</code> on it which turns it into its most-likely type. For example, if I inputted 2 at the command line, python2 would take in 2 as a string, <code>"2"</code>, and then run <code>eval()</code> on it to turn it into an integer, <code>2</code>. <a href="https://docs.python.org/2/library/functions.html#input" rel="nofollow noreferrer">See the documentation for <code>input()</code> here</a>.</p>
<p><code>input()</code> in python3 on the other hand just takes in input from the command line and processes it directly as a string. So if you were to input 2 for <code>num1</code>, <code>num1</code> would live as <code>"2"</code> until otherwise specified. <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow noreferrer">See documentation for python3 <code>input()</code> here</a>.</p>
<p>Thus, if you are running python2, this program should work fine. If you are running python3, you will run into error when you try to subtract strings. </p>
