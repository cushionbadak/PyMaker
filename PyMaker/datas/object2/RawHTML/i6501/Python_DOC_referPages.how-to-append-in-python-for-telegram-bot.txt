<p>I believe the problem lies in this part of the code:</p>
<pre><code>    amt= []
    for query_data in amt:
        amt.append(query_data)
        z = list[amt]
</code></pre>
<p>I'm not entirely sure about the python syntax<sup>1</sup>, but this seems to create a new variable <code>query_data</code> for the scope of the for loop.<br/>
This means that you do your loop for every element of <code>amt</code>. But <code>amt</code> is empty, so you don't do the loop at all.<br/>
I'm not certain what your loop is intended to do, but wouldn't just using a variable that exists over the time of all the inputs (for the easiest example: a global variable) suit your needs better? If you had a global <code>amt</code>, you could replace that code block with</p>
<pre><code>amt.append(query_data)
z = list[amt]
</code></pre>
<p>Be aware though, that a global variable will cause trouble as soon as you have more than one user simultaneously.</p>
<hr/>
<p>[1] Note that python just replaces your <code>amt</code> variable. Here's an example:</p>
<pre><code>&gt;&gt;&gt; alpha = []
&gt;&gt;&gt; beta = [1,2,3]
&gt;&gt;&gt; for alpha in beta:
...     print(1)
...
1
1
1
&gt;&gt;&gt; alpha
3
</code></pre>
<p>So even though my <code>alpha</code> was a list, it became an integer of value <code>3</code></p>
