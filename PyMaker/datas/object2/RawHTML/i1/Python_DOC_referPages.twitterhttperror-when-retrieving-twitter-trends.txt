<div class="post-text" itemprop="text">
<p>The problem seems to be due to the <code>LOCAL_WOE_ID</code> of 2344925 not being valid for the Twitter API. </p>
<p>Twitter provides another API <code>twitter.trends.available()</code> which gives a list of all the available WOEIDs supported. API docs are at: <a href="https://developer.twitter.com/en/docs/trends/locations-with-trending-topics/api-reference/get-trends-available.html" rel="nofollow noreferrer">https://developer.twitter.com/en/docs/trends/locations-with-trending-topics/api-reference/get-trends-available.html</a></p>
<p><strong>2344925 does NOT appear in this list</strong> (though it is found in some other WOEID lookups e.g. <a href="https://www.flickr.com/places/info/2344925" rel="nofollow noreferrer">https://www.flickr.com/places/info/2344925</a>), the discrepancy presumably due to Yahoo! not fully supporting their "Where On Earth" data currently (<a href="https://en.wikipedia.org/wiki/WOEID" rel="nofollow noreferrer">https://en.wikipedia.org/wiki/WOEID</a>).</p>
