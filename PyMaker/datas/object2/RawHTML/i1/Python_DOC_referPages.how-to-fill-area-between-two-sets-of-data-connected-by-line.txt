<div class="post-text" itemprop="text">
<p>You have to use the <code>ydata</code> as arguments for your <a href="https://matplotlib.org/api/_as_gen/matplotlib.axes.Axes.fill_between.html" rel="nofollow noreferrer"><code>fill_between</code></a>, not the curves.</p>
<p><strong>Either use <code>ydata</code> directly, or get them from your <code>curve1/2</code> objects like <code>ydata=curve1.get_ydata()</code></strong>.</p>
<p>Here is an example adapted from the <a href="https://matplotlib.org/gallery/pyplots/whats_new_98_4_fill_between.html#sphx-glr-gallery-pyplots-whats-new-98-4-fill-between-py" rel="nofollow noreferrer">docs</a>:</p>
<pre><code>import matplotlib.pyplot as plt
import numpy as np

x = np.arange(-5, 5, 0.01)
y1 = -5*x*x + x + 10
y2 = 5*x*x + x

c1, = plt.plot(x, y1, color='black')
c2, = plt.plot(x, y2, color='black')

# If you want/have to get the data form the plots
# x = c1.get_xdata()
# y1 = c1.get_ydata()
# y2 = c2.get_ydata()

plt.fill_between(x, y1, y2, where=y2 &gt;y1, facecolor='yellow', alpha=0.5)
plt.fill_between(x, y1, y2, where=y2 &lt;=y1, facecolor='red', alpha=0.5)
plt.title('Fill Between')

plt.show()
</code></pre>

<span class="comment-copy">no, there is not problem when using <code>plt</code> instead of <code>ax</code> to plot. If you use <code>plt</code> it simply plots on the current axes.</span>
<span class="comment-copy">If you don't want the lines then simply do the <code>fill_between</code> using your <code>xdata</code> and <code>ydata</code>. The line is not needed for this, so you do not need to do <code>plot</code> at all if you don't need the lines.</span>
