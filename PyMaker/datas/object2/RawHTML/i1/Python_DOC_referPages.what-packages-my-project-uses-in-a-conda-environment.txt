<div class="post-text" itemprop="text">
<p>I have a project and I am looking to create an isolated python distribution that ONLY contains the packages/modules required by my project.</p>
<p>I have been developing in an anaconda environment. </p>
<p>If I do <code>conda list --export &gt; requirements.txt</code>, this gives all the packages my environment uses. </p>
<p>I don't want my project to install every one of these packages on another machine as it only needs a handful I'm sure.</p>
<p>Is there a way to only list the packages my project requires (in conda).</p>
</div>
<div class="post-text" itemprop="text">
<p>You can try <a href="https://github.com/bndr/pipreqs" rel="nofollow noreferrer">pipreqs</a>, it generates your requirements from the imports of your project.</p>
</div>
<div class="post-text" itemprop="text">
<p>The traditional approach is to test your project in a virtual environment:</p>
<ol>
<li><a href="https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html#creating-an-environment-with-commands" rel="nofollow noreferrer">create</a> an virtual environment</li>
<li>run your project and install the required dependencies, e.g. <code>&gt; conda install &lt;dependency&gt;</code></li>
<li>save dependencies to <code>requirements.txt</code> via the OP command</li>
</ol>
<p>This will create an isolated <code>requirements.txt</code>.</p>
<p>See also <a href="https://pipenv.readthedocs.io/en/latest/" rel="nofollow noreferrer"><code>pipenv</code></a> and <a href="https://poetry.eustace.io/" rel="nofollow noreferrer"><code>poetry</code></a>.  These tools create virtual environments of your project automatically and track your dependencies reliably.</p>
</div>
