<span class="comment-copy">@EliasRub Because <code>requests</code>' chain of dependencies ends up doing <code>import urllib.parse</code> when executing <code>import requests</code> in your script. And since the name of your script is <code>urllib.py</code>, the interprter looks in <i>your</i> script for a module named <code>parse</code>, instead of the Python standard lib package <code>urllib</code>. <i>Do not name your scripts after standard lib or 3rd party packages!</i></span>
