<div class="post-text" itemprop="text">
<p>Python list are zeroed indexed. That means the first index of a list starts from <code>0</code>. So the indexes for <code>hastag_list</code> are <code>0, 1, 2</code>. However, <code>randint(1,3)</code> is inclusive, so <code>tag</code> will sometimes randomly be assigned <code>3</code>.</p>
<p>It should start to become clear what the problem is. Since the maximum index of <code>hashtag_list</code> is <code>2</code>, and <code>tag</code> is sometimes <code>3</code>, then sometimes you will be indexing <code>hashtag_list</code> out of range. </p>
<p>The simple solution would be to use <code>randint(0, 2)</code> rather than <code>randint(1, 3)</code>. However, a cleaner solution would be to use <a href="https://docs.python.org/3/library/random.html#random.choice" rel="nofollow noreferrer"><code>random.choice</code></a> to choose a random option from <code>hashtag_list</code>:</p>
<pre><code>from random import choice

# ...

hashtag_list = ['urban', 'hipster', 'retro']
for hashtag in hashtag_list:
    tag = choice()    
    driver.get('https://www.instagram.com/explore/tags/' + hashtag_list[tag] + '/')
    sleep(5)
</code></pre>
</div>

<p><code>randint(a, b)</code> produces a random integer in the range <code>[a, b]</code> inclusive, those sometimes when it doesn't work it's producing <code>3</code>, and <code>3</code> is beyond the list. Change it to <code>randint(0, 2)</code>, because in Python (and many other languages), a list with <code>n</code> elements' indices start from <code>0</code> and end at <code>n - 1</code>. Thus:</p>
<pre><code># elements:  ['urban', 'hipster', 'retro']
# indices:       0         1         2
</code></pre>
</div>
