<div class="post-text" itemprop="text">
<p>The expression you actually should be using is <code>if value == 1</code>. </p>
<p><code>if 1 in value</code> expects that <code>value</code> is an iterable (e.g., an array or a dict) and returns True if 1 is contained in <code>value</code>, and False otherwise.</p>
<p>If <code>value</code> is not an iterable, you'll get an exception:</p>
<pre><code>&gt;&gt;&gt; arr = [1, 2, 3]
&gt;&gt;&gt; 1 in arr
True
&gt;&gt;&gt; 4 in arr
False
&gt;&gt;&gt; not_arr = 1
&gt;&gt;&gt; 1 in not_arr
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: argument of type 'int' is not iterable
&gt;&gt;&gt; not_arr == 1
True
&gt;&gt;&gt; not_arr == 2
False
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Using <code>and</code> along with comparing the <code>'1'</code> with the <code>str</code> type of <code>value</code>:</p>
<pre><code>def test(**kwargs):
    for key, value in kwargs.items():
        if 'sdk' in key and '1' in str(value): # True in case of test(sdk=19)
                print('HelloWorld')

test(sdk=1)
</code></pre>
<p><strong>OR</strong></p>
<p>if you do not wish to cast the <code>1</code> into a <code>str</code> <code>'1'</code>,  <code>==</code> is what you could use:</p>
<pre><code>if 'sdk' in key and 1 == value: # False in case of test(sdk=19)
</code></pre>
<p><strong>OUTPUT</strong>:</p>
<pre><code>HelloWorld
</code></pre>
