<p>I have a variable X, it contains a list (Python list), of 10 Numpy 1-D arrays (basically vectors).
If I ask for X[100], it throws an error saying: IndexError: list index out of range</p>
<p>Which makes total sense, but, when I ask for X[:100], it doesn't throw an error and it returns the entire list! 
Why is that?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>X[:100]</code> means slice <code>X</code> from 0 to 100 or the end (whichever comes first)
But <code>X[100]</code> means the 100th element of <code>X</code>, and if it doesn't exist it throws an <code>index out of range</code> error</p>
</div>
<span class="comment-copy">first one is indexing, another is slicing... slicing does not raise error.</span>
