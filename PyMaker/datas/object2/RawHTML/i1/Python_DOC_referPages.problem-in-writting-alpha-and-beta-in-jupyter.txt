<div class="post-text" itemprop="text">
<p>it's because <code>'\a'</code> is a string literal for ascii character 7 <a href="https://www.asciitable.com/" rel="nofollow noreferrer">ascii table</a> just like <code>'\t'</code> is for tab and <code>'\n'</code> is for newline.</p>
<p>we can check this using the built-in function <code>ord()</code> which returns the corresponding ASCII table index for the supplied character - <a href="https://docs.python.org/3/library/functions.html#ord" rel="nofollow noreferrer">doc</a></p>
<pre><code>&gt;&gt;&gt; ord('\a')
7
&gt;&gt;&gt; ord('\n')
10
&gt;&gt;&gt; ord('\t')
9
</code></pre>
<p>on the other hand, <code>'\o'</code> doesn't correspond to any character.</p>
<pre><code>&gt;&gt;&gt; ord('\o')
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: ord() expected a character, but string of length 2 found
</code></pre>
<p>to get around this, just prepend <code>r</code>(as in <code>raw string</code>) before the string like this </p>
<p><code>my_string_variable = r'alpha\omega'</code></p>
</div>
