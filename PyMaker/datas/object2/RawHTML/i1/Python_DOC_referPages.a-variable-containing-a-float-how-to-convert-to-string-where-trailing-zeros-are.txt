<div class="post-text" itemprop="text">
<p>I have a variable <code>K = 25.00</code> and I need to search a value in a dataframe column such as </p>
<pre><code>data[f"Item {K}"][99] 
</code></pre>
<p>but <code>K</code> reverts to <code>25.0</code> when turned into a string. 
Is there a nice way to make <code>K</code> stay <code>25.00</code> throughout?
I read <a href="https://stackoverflow.com/questions/28399533/converting-float-to-string-without-truncating-leading-or-trailing-zeros">this</a> but that method looks so messy, there must be a simpler way?</p>
</div>
<div class="post-text" itemprop="text">
<p>Since you're using the modern Python 3 formatting:</p>
<pre><code>data[f"Item {K:.2f}"][99]
</code></pre>
<p>Note that you shouldn't normally call variables 'K' but rather 'k', since the use of uppercase indicates a class type, not a variable.</p>
<p>As @Tushar already suggested, the above is equivalent to:</p>
<pre><code>data["Item {:.2f}".format(Variable)][99]
</code></pre>
<p>But I do like the new Python 3 formatting.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can format the variable to two decimal places while converting it to string</p>
<pre><code>"{:.2f}".format(Variable)
</code></pre>
