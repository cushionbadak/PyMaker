<p>I have a django url:</p>
<pre><code>path('question/&lt;slug:question_slug&gt;/add_vote/', views.AddVoteQuestionView.as_view())
</code></pre>
<p>It work fine with english slug but when slug is persian something like this:</p>
<p><code>/question/سوال-تست/add_vote/</code> </p>
<p>django url throw <code>404 Not Found</code>, is there any solution to catch this perisan slug url?</p>

<p>I'm using django 2.1.5.</p>
<p>It work fine with this url:</p>
<pre><code>re_path(r'question/(?P&lt;question_slug&gt;[\w-]+)/add_vote/$', views.AddVoteQuestionView.as_view())
</code></pre>

<p>This is an addition to <a href="https://stackoverflow.com/users/2011147/selcuk">Selcuk</a> answer <a href="https://stackoverflow.com/a/55175589/8283848">given here</a></p>
<hr/>
<p>to pass such language/unicode characters you have to </p>
<ol>
<li><a href="https://docs.djangoproject.com/en/2.1/topics/http/urls/#registering-custom-path-converters" rel="nofollow noreferrer">Write some custom path converter</a></li>
<li>Use <a href="https://docs.djangoproject.com/en/2.1/ref/urls/#re-path" rel="nofollow noreferrer"><strong><code>re_path()</code></strong> function</a>
<hr/></li>
</ol>
<h3>1 . Custom path converter</h3>
<p>If we look into the source code of Django, the <a href="https://github.com/django/django/blob/master/django/urls/converters.py#L35" rel="nofollow noreferrer"><strong><code>slug</code></strong></a> path converter uses this regex,<br/> <strong><code>[-a-zA-Z0-9_]+</code></strong> which is inefficent here (see Selcuk's answer).<br/><br/>
So, Write your own custom slug converter , as below</p>
<pre><code><b>from django.urls.converters import SlugConverter</b>


class CustomSlugConverter(<b>SlugConverter</b>):
    <b>regex = '[-\w]+' # new regex pattern</b></code></pre>
<p>Then register it,</p>
<pre><code><b>from django.urls import path, register_converter

register_converter(CustomSlugConverter, 'custom_slug')</b>

urlpatterns = [
    path('question/<b>&amp;ltcustom_slug:question_slug&gt;</b>/add_vote/', views.AddVoteQuestionView.as_view()),
    ...
]</code></pre>
<h3>2. using <a href="https://docs.djangoproject.com/en/2.1/ref/urls/#re-path" rel="nofollow noreferrer"><strong><code>re_path()</code></strong></a></h3>
<p>You've already tried and succeeded with this method. Anyway, I'm c&amp;p it here :)</p>
<pre><code>from django.urls import re_path

urlpatterns = [
    re_path(r'question/<b>(?P&amp;ltquestion_slug&gt;[\w-]+)/</b>add_vote/$', views.AddVoteQuestionView.as_view()),
    ...
]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>According to <a href="https://docs.djangoproject.com/en/2.1/topics/http/urls/#path-converters" rel="nofollow noreferrer">Django 2.1 documentation</a> you can only use ASCII letters or numbers for <code>slug</code> patterns:</p>
<blockquote>
<p><code>slug</code> - Matches any slug string consisting of ASCII letters or numbers, plus the hyphen and underscore characters. For example, <code>building-your-1st-django-site</code>.</p>
</blockquote>
<p>whereas regex <code>\w</code> pattern also matches Unicode word characters:</p>
<p><a href="https://docs.python.org/3/library/re.html#index-32" rel="nofollow noreferrer">https://docs.python.org/3/library/re.html#index-32</a></p>
<blockquote>
<p>For Unicode (str) patterns:
  Matches Unicode word characters; this includes most characters that can be part of a word in any language, as well as numbers and the underscore. If the ASCII flag is used, only <code>[a-zA-Z0-9_]</code> is matched.</p>
</blockquote>
