<p>I have started to experiment with Cython and ran into the following problem. Consider the following class representing a vertex in the 3D space:</p>
<pre><code>#Vertex.pyx
cdef class Vertex(object):
    cdef double x, y, z
    def __init__(self, double x, double y, double z):        
        self.x = x
        self.y = y
        self.z = z
</code></pre>
<p>Now I try to create an object from the Python console:</p>
<pre><code>import Vertex as vt
v1 = vt.Vertex(0.0, 1.0, 0.0) 
</code></pre>
<p>which works fine. However, when I'm trying to access the class attributes, I'm getting an <code>AttributeError</code>:</p>
<pre><code>print v1.x
---------------------------------------------------------------------------
AttributeError                            Traceback (most recent call last)
&lt;ipython-input-6-83d928d774b7&gt; in &lt;module&gt;()
----&gt; 1 print v1.x

AttributeError: 'Vertex.Vertex' object has no attribute 'x'
</code></pre>
<p>Any ideas why this could happen?</p>
</div>
<div class="post-text" itemprop="text">
<p>By default <code>cdef</code> attributes are only accessible from within Cython. If <a href="https://cython.readthedocs.io/en/latest/src/userguide/extension_types.html#static-attributes" rel="nofollow noreferrer">you make it a public attribute</a> with <code>cdef public</code> then Cython will generate suitable properties to be able to access it from Python.</p>
