<p>How to convert a date Object to millisecond in python 2.6.6 </p>
<pre><code> date_dob = getattr(model, value)
                 a = date_dob.timestamp() * 1000
                 print(a)
</code></pre>
<p>Here date_dob is something like  "1993-05-29 13:42:10.298Z"</p>
<p>I am getting below error </p>
<pre><code>AttributeError: 'datetime.datetime' object has no attribute 'timestamp'
</code></pre>
<p>I am expecting result of 738639730000(epoch)</p>
</div>
<div class="post-text" itemprop="text">
<p><code>timestamp()</code> was introduced in Python 3.</p>
<p>Use the below code for python 2.</p>
<pre><code>from datetime import datetime

date_dob = datetime.now()

print date_dob

#print dir(date_dob) ## To list all the attributes

date_sec_str = date_dob.strftime("%s.%f")
date_sec_float = float(date_sec_str)

a = int(date_sec_float*1000)

print a
</code></pre>
<p>Result:</p>
<blockquote>
<p>2019-03-15 15:32:05.869709<br/>
  1552663925869</p>
</blockquote>
</div>
<span class="comment-copy">Milliseconds are a unit of an <i>interval</i> of time; a date represents an <i>instant</i> of time. You can't convert an instant into an interval unless you have another instant that you are <i>relative</i> to, called the "epoch". Given your numbers, it sounds like you're working with an epoch of approximately 1969-12-31 00:00:00 UTC? That's an unusual epoch: the Unix epoch of 1970-01-01 00:00:00 UTC is much more common.</span>
