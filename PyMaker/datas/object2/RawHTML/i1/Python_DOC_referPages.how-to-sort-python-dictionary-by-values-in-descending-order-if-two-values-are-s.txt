<div class="post-text" itemprop="text">
<p>Your python version may not keep the order in which you entered your key-value pairs. In which case you can use <a href="https://docs.python.org/3/library/collections.html?highlight=ordereddict#collections.OrderedDict" rel="nofollow noreferrer"><code>collections.OrderedDict</code></a></p>
<pre><code>from operator import itemgetter
from collections import OrderedDict

dictionary={'b': 5, 'a': 2, 'k': 5}

s = OrderedDict(sorted(dictionary.items(), key=itemgetter(1), reverse=True))
print(s)
# OrderedDict([('b', 5), ('k', 5), ('a', 2)])
</code></pre>
<p>if your python version is &lt;= 3.5 dicts will lose their order; in python 3.6 the order-keeping is considered an implementation detail and should not be relied on. <a href="https://mail.python.org/pipermail/python-dev/2017-December/151283.html" rel="nofollow noreferrer">starting from python 3.7 this is considered</a> to be part of the language specification.</p>
<p>so starting from python 3.7 your code will not need <code>OrderedDict</code> and you can work with the regular <code>dict</code>.</p>
