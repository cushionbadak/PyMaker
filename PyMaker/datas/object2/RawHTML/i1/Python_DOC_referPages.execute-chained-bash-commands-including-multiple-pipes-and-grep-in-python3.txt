<div class="post-text" itemprop="text">
<p>This works without having to use the subprocess library or any other os cmd related library, only Python.</p>
<pre><code>my_file = open("./file.txt")
line = True
while line:
    line = my_file.readline()
    line_array = line.split()
    try:
        if line_array[0] == '"name":':
            print(line_array[1].replace('"', '').replace(',', ''))
    except IndexError:
        pass
my_file.close()
</code></pre>

<div class="post-text" itemprop="text">
<p>If you not trying to parse a json file or any other structured file for which using a parser would be the best approach, just change your command into:</p>
<pre><code>grep -oP '(?&lt;="name":[[:blank:]]").*(?=",)' file.txt
</code></pre>
<p>You do not need any pipe at all.</p>
<p>This will give you the <strong>output</strong>:</p>
<pre><code>Tom
Jack
</code></pre>
<p><strong>Explanations:</strong></p>
<ul>
<li><code>-P</code> activate perl regex for lookahead/lookbehind</li>
<li><code>-o</code> just output the matching string not the whole line</li>
<li>Regex used: <code>(?&lt;="name":[[:blank:]]").*(?=",)</code>
<ul>
<li><code>(?&lt;="name":[[:blank:]]")</code> Positive lookbehind: to force the constraint <code>"name":</code> followed by a blank char and then another double quote <code>"</code> the name followed by a double quote <code>"</code> extracted via <code>(?=",)</code> positive lookahead</li>
</ul></li>
</ul>
<p><strong>demo</strong>: <a href="https://regex101.com/r/JvLCkO/1" rel="nofollow noreferrer">https://regex101.com/r/JvLCkO/1</a></p>
