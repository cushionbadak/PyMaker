<p>I am now calling a C++ library in Python with the help of <a href="https://cffi.readthedocs.io/en/latest/" rel="nofollow noreferrer">CFFI</a>. This C++ library uses <a href="http://rpg.ifi.uzh.ch/docs/glog.html" rel="nofollow noreferrer">Google Logging Library</a> for logging message. I know that when I set up a C++ application that calls the library, I can easily control where the log file is located:</p>
<pre><code> #include &lt;glog/logging.h&gt;

   int main(int argc, char* argv[]) {
     // Initialize Google's logging library.
     google::InitGoogleLogging(argv[0]);

     // ...
     LOG(INFO) &lt;&lt; "Found " &lt;&lt; num_cookies &lt;&lt; " cookies";
   }
</code></pre>
<p>However, now I am calling it from the Python codes. Then my question is how can I set up the log file so that the log information in the C++ library can be written to the defined log file. By the way, I have the source code of the C++ library, and I can compile it myself. </p>
</div>
<div class="post-text" itemprop="text">
<p>According to the <a href="https://rawgit.com/google/glog/v0.3.5/doc/glog.html" rel="nofollow noreferrer">docs</a>, setting environment variable <code>GLOG_log_dir</code> should work:</p>
<pre><code>GLOG_log_dir='/my/log/dir/' python my_code.py
</code></pre>
<p>According to the <a href="https://github.com/google/glog/blob/v0.3.5/src/logging.cc#L154" rel="nofollow noreferrer">code</a>, setting environment variable <code>GOOGLE_LOG_DIR</code> may also work.</p>
</div>
