<div class="post-text" itemprop="text">
<p>No, it is not broken.  <a href="http://docs.python.org/3.3/library/urllib.request.html">The <code>urllib.request</code> documentation</a> is pretty clear on how this works:</p>
<pre><code>import urllib.request
req = urllib.request.urlopen('http://www.google.com')
data = req.read()
</code></pre>
<p><strong>Edit:</strong> If you need to write the file directly to disk rather than process the data, use <code>urlretrieve</code>.</p>
<pre><code>urllib.request.urlretrieve('http://example.com/big.zip', 'file/on/disk.zip')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>To download an url into a file, you could use <a href="http://docs.python.org/3/library/urllib.request#urllib.request.urlretrieve" rel="nofollow"><code>urlretrieve()</code> function</a>:</p>
<pre><code>from urllib.request import urlretrieve
url = "http://www.petercollingridge.co.uk/sites/files/peter/particle_tutorial_7.txt"
urlretrieve(url, "result.txt")
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The urllib2 module has been split across several modules in Python 3.0 named urllib.request and urllib.error. The 2to3 tool will automatically adapt imports when converting your sources to 3</p>
<pre><code>from urllib.request import urlopen

data = urlopen(r"http://www.petercollingridge.co.uk/sites/files/peter/particle_tutorial_7.txt")

print(data)
</code></pre>
