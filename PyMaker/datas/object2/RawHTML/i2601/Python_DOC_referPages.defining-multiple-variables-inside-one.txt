<div class="post-text" itemprop="text">
<p>In your case hiddenLayerinput is a dictionary,
you can use it as :</p>
<pre><code>hiddenLayerinput = {
"units" : 64,
"activation": "relu"
"input_dim": 5
}
</code></pre>
<p>Then you can acces it as : <code>hiddenLayerinput["units"]</code> , and you will get value of "units" : 64 </p>
</div>
<div class="post-text" itemprop="text">
<p>Python's dictionaries use <code>{}</code> and hence </p>
<pre><code>hiddenLayerinput = {
    units=64, 
    activation='relu', 
    input_dim=5
}
</code></pre>
<p>is identified as a dictionary, <code>=</code> is <code>SyntaxError</code> you get. You need to change all the <code>=</code>s to <code>:</code>s</p>
<pre><code>hiddenLayerinput = {
    "units":64, 
    "activation":'relu', 
    "input_dim":5
}
</code></pre>
<p><strong>Edit</strong> - <code>units</code>, <code>activation</code>, <code>input_dim</code> are keys. If you have <code>units</code>, <code>activation</code>, <code>input_dim</code> as variables, you can remove the quotes and the values from those variables will be used as keys.
Read more about <a href="https://docs.python.org/3/tutorial/datastructures.html#dictionaries" rel="nofollow noreferrer">dictionaries</a></p>
</div>

<span class="comment-copy">I think what you are looking for is a dictionary. You fill it with key-value-pairs like this <code>hiddenLayerinput = {'unit':64, 'activiation':'relu', 'input':5}</code>. Afterwards, you can access it like this: <code>hiddenLayerinput['unit']</code> to get <code>64</code>.</span>
<span class="comment-copy">You probably want to use string literals as keys here - it is technically legal to use variables, but I strongly doubt that was the intent in this case ;-)</span>
