<div class="post-text" itemprop="text">
<p>In Python3, the <code>print</code> statement of Python2 was changed to function: <code>print(...)</code>. So Python2 interprets this Python3 statements as calling the <code>print</code> statement with a tuple.</p>
<pre><code>$ python2 -c 'print(1, 2, 3)'
(1, 2, 3)
$ python3 -c 'print(1, 2, 3)'
1 2 3
</code></pre>
<hr/>
<p>Relevant documentation: <a href="https://docs.python.org/3/whatsnew/3.0.html#print-is-a-function" rel="nofollow noreferrer">https://docs.python.org/3/whatsnew/3.0.html#print-is-a-function</a>.</p>
</div>

<p>The difference is in the version of Python. 
If you're using Python 2.XX you don't need the brackets when you are calling the print function; however in Python 3.XX you must use print() with the brackets.</p>
<p>If you want to solve your problem, you should change your Python version or just remove every brackets from the print function like this:</p>
<pre><code>print("I will now count my chickens:")

print "Hens", 25 + 30 / 6 
print "Roosters", 100 - 25 * 3 % 4

print("Now I will count the eggs:")

print 3 + 2 + 1 - 5 + 4 % 2 - 1/ 4 + 6

print("Is it true that 3 + 2 &lt; 5 - 7?")

print 3 + 2 &lt; 5 - 7

print "What is 3 + 2?", 3 + 2
print "What is 5 - 7?", 5 - 7

print("Oh, that's why it's False.")

print("How about some more.")

print "Is it greater?", 5 &gt; -2
print "Is it greater or equal?", 5 &gt;= -2
print "Is it less or equal?", 5 &lt;= -2
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As others have identified, your <code>print</code> calls in Python 2 call a statement, whereas in Python 3, they call a function. The addition of brackets is necessary in Python 3 whereas they indicate a tuple in Python 2.</p>
<p>In Python 2.6+, you can bring in the new behaviour of the print function from Python 3 by importing the print function from the <code>__future__</code> package:</p>
<pre><code>from __future__ import print_function
</code></pre>
<p>This provides the Python 3 behaviour and will allow your print statements to be executed with similar behaviour under both Python 2 and 3 interpreters. (Other Python 2-specific constructs may, of course, require separate treatment.)</p>
