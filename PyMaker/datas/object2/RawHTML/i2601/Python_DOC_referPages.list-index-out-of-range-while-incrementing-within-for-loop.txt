<div class="post-text" itemprop="text">
<p>This is what happens on the first iteration of your loop:</p>
<pre><code>arr = []
i = 0
j = 0
arr[i][j] = ...
</code></pre>
<p>Try it out in the <a href="https://docs.python.org/3/tutorial/interpreter.html#interactive-mode" rel="nofollow noreferrer">REPL</a>.</p>
<pre><code>&gt;&gt;&gt; arr = []
&gt;&gt;&gt; i = 0
&gt;&gt;&gt; j = 0
&gt;&gt;&gt; arr[i][j] = 1
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
IndexError: list index out of range
</code></pre>
<p>So what happened?</p>
<ul>
<li><p><code>arr[0][0] = 1</code> means "set the first item of list <code>arr[0]</code> to be 1."</p></li>
<li><p><code>arr[0]</code> means "the first item of list <code>arr</code>."</p></li>
<li><p><code>arr</code> is an empty list; it does not have a first item. So the expression <code>arr[0]</code> for that program state is erroneous.</p></li>
</ul>
<p>How do we fix it? We need to make sure we create a list for each row.</p>
<pre><code># Create a blank matrix
for j in range(order):
    arr[j] = []

# Now fill in the values
for i in range(order):
    for j in range(order):
        arr[j][i] = ...
</code></pre>

<p>You have to modify your code a bit for it to work properly.</p>
<pre><code>import fileinput

order = int(input())
arr_input = [int(num) for num in input().split(' ')]
arr = []
index = 0

for j in range(order):
    arr.append([])

for i in range(order):
    for j in range(order):
        print(index, i , j)
        arr[j].append(arr_input[index])
        index += 1

print(arr)
</code></pre>
<p>I can explain why you are getting IndexError: list index out of range</p>
<pre><code>arr[j][i] = arr_input[index] 
</code></pre>
<p>through this line of code you are trying to access jth row and ith column of the array arr but arr is just an empty array.
so first you have to do this </p>
<pre><code>for j in range(order):
        arr.append([])
</code></pre>
<p>p.s: while assigning values in an empty array always use append() method to avoid "list index out of range error".</p>
