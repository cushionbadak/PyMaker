<div class="post-text" itemprop="text">
<p><code>map</code> yields the result of the transform function on a single element of the provided input iterable each time you pull a value from it. So on the first pull, it's yielding the result of <code>list("66")</code>; since <code>str</code> are iterables composed of <code>len</code> <code>1</code> <code>str</code>s (the individual characters of the string), <code>list("66")</code> produces a <code>list</code> of <code>'6'</code> and <code>'6'</code>. The next time, <code>list("67")</code> produces <code>['6', '7']</code>, and so on.</p>
</div>
<div class="post-text" itemprop="text">
<p>Map receives as params a function and an iterable. So, it will iterate over the iterable and run the function for each element. That means it will run in order: <code>list("66"); list("67"); list("68")</code>
list() is a pretty versatile function in Python, it will convert your string into an array of characters that create the string. That means "66" will get ["6", "6"]. Finally, your initial list will become a generator for the above mentioned lists, thanks to ShadowRanger for pointing this out.</p>
<p>Additionally, you can check the <a href="https://docs.python.org/3/library/functions.html#map" rel="nofollow noreferrer">official documentation</a> for the map function</p>
