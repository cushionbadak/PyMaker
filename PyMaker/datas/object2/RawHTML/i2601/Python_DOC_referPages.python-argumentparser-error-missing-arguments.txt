<div class="post-text" itemprop="text">
<p>You must launch the application providing the arguments indicated in the error message. In your code, both arguments are <code>required</code>. Hence, you must supply them to run the application.</p>
<p>Here is what it may be expecting</p>
<pre><code>python ipykernel_launcher.py -q "&lt;query&gt;", -o "&lt;dir&gt;"
</code></pre>
<p>Note, the <code>&lt;query&gt;</code> and <code>&lt;dir&gt;</code> above are fillers. You must provide a query inline with applications definition of a query. The <code>-o</code> flag indicates a requirement to pass a directory. Hence, it could be as simple as <code>./output</code> or something between these lines.</p>
</div>
<div class="post-text" itemprop="text">
<p>You set the <code>required</code> option to true:</p>
<pre><code>ap.add_argument("-q", "--query", **required=True,** help="search query to search Bing Image API for")
ap.add_argument("-o", "--output", **required=True**, help="path to output directory of images")
</code></pre>
<p>(** added for visibility)
writing</p>
<pre><code>ap.add_argument("-q", "--query", **required=False,** help="search query to search Bing Image API for")
ap.add_argument("-o", "--output", **required=False**, help="path to output directory of images")
</code></pre>
<p>instead, should fix the problem. Unless of course you want them to be required, then you have to call the script with the arguments.</p>
