<p>I have been searching for a way that could read gz file in python, and I did something like</p>
<pre><code>with gzip.open(filepath) as infile:
    ...
</code></pre>
<p>However, it seems like the read-in data is byte-like and I cannot do something like <code>for l in infile</code>. Is there a way to solve this?</p>
</div>
<div class="post-text" itemprop="text">
<p>Pass <code>mode='rt'</code> (text mode) to the <a href="https://docs.python.org/3/library/gzip.html#gzip.open" rel="nofollow noreferrer"><code>gzip.open</code></a> call.</p>
<p>From the documentation (linked above):</p>
<blockquote>
<p>The mode argument can be any of <code>'r', 'rb', 'a', 'ab', 'w', 'wb', 'x'</code> or <code>'xb'</code> for binary mode, or <code>'rt', 'at', 'wt',</code> or <code>'xt'</code> for text mode. The default is <code>'rb'</code>.</p>
</blockquote>
</div>
