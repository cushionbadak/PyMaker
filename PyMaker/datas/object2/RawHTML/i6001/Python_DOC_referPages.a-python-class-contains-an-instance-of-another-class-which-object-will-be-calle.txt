<h1>Old school debugging</h1>
<pre><code>print(0)
class Foo:
    print(1)
    def __init__(self):
        print(2)
        self.age = 'Age'

    def __str__(self):
        print(3)
        return str(self.age)

class Bar:
    print(4)
    obj_foo = Foo()
    print(obj_foo)
    def __init__(self, name):
        print(5)
        self.name = name

    def __str__(self):
        print(6)
        return str(self.name)

print(7)
obj_bar = Bar("Sam")
print(obj_bar)
print(8)
</code></pre>
<p>My FORTRAN teacher would be proud!</p>
<p>It outputs:</p>
<pre><code>0
1
4
2
3
Age
7
5
6
Sam
8
</code></pre>
<p>So here's the order:</p>
<ul>
<li>Start of the script</li>
<li>class Foo is defined</li>
<li>class Bar is defined</li>
<li>one Foo is instantiated</li>
<li>the Foo is printed</li>
<li>Foo's Age is printed</li>
<li>Script continues</li>
<li>one Bar is instantiated</li>
<li>the Bar is printed</li>
<li>Bar's Name is printed</li>
<li>Script finishes</li>
</ul>
<h1>Spyder + pdb</h1>
<p>For more complex scripts, it might be a good idea to use specialized tools. For example <a href="https://pythonhosted.org/spyder/" rel="nofollow noreferrer">spyder</a> with <a href="https://docs.python.org/3/library/pdb.html" rel="nofollow noreferrer">pdb</a>.</p>
<p>If you installed Python with <a href="https://anaconda.org/anaconda/python" rel="nofollow noreferrer">Anaconda</a>, you don't need to install anything else.</p>
<p>Run your script with Debug &gt; Debug (Ctrl + F5), press Ctrl + F10 multiple times to see the execution, press Ctrl + F11 if you want to step into a function or method.</p>
