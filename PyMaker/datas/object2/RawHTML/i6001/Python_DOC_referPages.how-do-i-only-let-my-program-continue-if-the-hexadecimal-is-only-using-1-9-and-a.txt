<p>Exactly. The expression <code>a or b</code> evaluates to <code>a</code> if <code>a</code> can be converted to <code>True</code>, and to <code>b</code> otherwise. Thus your <code>if</code> check reduces to <code>if hexa == "1":</code></p>
<p>You should have written the longer <code>hexa == "1" or hexa == "3" or ...</code>. Or use the <code>in</code> keyword like <code>hexa in ["1", "3", "4", "5", ...]</code>. Or, since the righthand side is all single character strings, you can use <code>hexa in "134567..."</code> because a string is also iterable by character, and a character is a string itself.</p>

<p>If you want to parse any hexadecimal number recognized by <code>int()</code>, you can simply catch the <code>ValueError</code> exception:</p>
