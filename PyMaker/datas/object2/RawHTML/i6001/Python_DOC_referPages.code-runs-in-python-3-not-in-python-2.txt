<div class="post-text" itemprop="text">
<p>That's because in the atom editor, you use <code>python3</code> and your IDLE uses <code>python2</code>. In Python 2, the function for reading user's input was called <code>raw_input()</code>; <a href="https://docs.python.org/3/whatsnew/3.0.html#builtins" rel="nofollow noreferrer">it was renamed to <code>input()</code> in Python 3</a> (section starting with <code>PEP 3111: raw_input() was renamed to input()</code>).</p>
<p>You can either ensure you are using <code>python3</code> by default or make the code <code>python2</code>-compatible: add a code block</p>
<pre><code>import sys
compatible_input = raw_input if sys.version_info &lt; (3, 0) else input
</code></pre>
<p>and replace all usages of <code>... = input(...)</code> in your code with <code>... = compatible_input(...)</code>.</p>

<span class="comment-copy"><a href="https://docs.python.org/2/library/functions.html#input" rel="nofollow noreferrer">docs.python.org/2/library/functions.html#input</a>  In python2 string returned from <code>input()</code> is immediately evaluated which causes code to behave like <code>risposta=y</code> where <code>y</code> is not defined yet in first case, and <code>risposta=Y</code>  where <code>Y</code> is an array in second.</span>
