<h2>send_keys(*value)</h2>
<p>As per the documentation of <a href="https://seleniumhq.github.io/selenium/docs/api/py/webdriver_remote/selenium.webdriver.remote.webelement.html#selenium.webdriver.remote.webelement.WebElement.send_keys" rel="nofollow noreferrer"><strong><code>send_keys()</code></strong></a> simulates typing into the element.</p>
<ul>
<li><strong><code>send_keys(*value)</code></strong></li>
<li><p>Args :</p>
<pre><code>value - A string for typing. For setting file inputs, this could be a local file path.
</code></pre></li>
<li><p>Use this to set file inputs:</p>
<pre><code>driver.find_element_by_id("attachFile_nPaintUploadAll").send_keys("path/to/profilepic.gif")
</code></pre></li>
</ul>
<p>But as per your code trial as you are passing an <em>url</em> as a <em>string</em> hence you see the error as:</p>
<pre><code>selenium.common.exceptions.InvalidArgumentException: Message: File not found: http://bdfjade.com/data/out/89/5941587-natural-image-download.jpg
</code></pre>
<h2>Solution</h2>
<p>If your <em>usecase</em> is to use <em>Selenium</em> for file uploading you have to have the download the file in your local system and pass the <em>absolute path</em> of the file as an argument within <code>send_keys()</code> method.</p>
<hr/>
<h2>Alternative (urllib.request.urlretrieve)</h2>
<p>As an alternative, you can also use <a href="https://docs.python.org/3/library/urllib.request.html?highlight=request#module-urllib.request" rel="nofollow noreferrer"><strong><code>urlretrieve</code></strong></a> method from <em>Python 3.x</em> as follows:</p>
<h2><sub>urllib.request.urlretrieve(url, filename=None, reporthook=None, data=None)</sub></h2>
<ul>
<li><p>Copy a network object denoted by a URL to a local file. If the URL points to a local file, the object will not be copied unless filename is supplied. Return a tuple (filename, headers) where filename is the local file name under which the object can be found, and headers is whatever the info() method of the object returned by <code>urlopen()</code> returned (for a remote object). Exceptions are the same as for <code>urlopen()</code>.</p></li>
<li><p>Code Block:</p>
<pre><code>import urllib.request

urllib.urlretrieve("http://andrew.com/selfie.jpg", "andrew_selfie.jpg")
driver.find_element_by_id("attachFile_nPaintUploadAll").send_keys("andrew_selfie.jpg")
</code></pre></li>
</ul>
</div>
