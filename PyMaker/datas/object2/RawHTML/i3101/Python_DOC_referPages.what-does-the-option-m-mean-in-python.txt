<p>What does the <code>-m</code> mean while creating a virtual environment?</p>
<p>For example, <code>py -3 -m venv env</code>.</p>
<p>What does the <code>-m</code> mean? What's the difference of <code>venv</code> and <code>virtualenv</code>?</p>
<p>Thanks.</p>

<div class="post-text" itemprop="text">
<blockquote>
<p>What does the -m mean?</p>
</blockquote>
<p>From python manual page:</p>
<blockquote>
<p>-m module-name<br/>
  Searches sys.path for the named module and runs the corresponding .py file as a script. </p>
</blockquote>
<p><code>py -m venv env</code> will tell Python (the Windows version uses <code>py</code>) to use execute the <a href="https://docs.python.org/3/library/venv.html" rel="nofollow noreferrer"><code>venv</code></a> module and create a virtual environment that will be stored in the <code>env</code> directory. </p>
<p>From <a href="https://virtualenv.pypa.io/en/stable/userguide/#usage" rel="nofollow noreferrer">virtualenv documentation</a>:</p>
<blockquote>
<p>Virtualenv has one basic command:</p>
<p><code>$ virtualenv ENV</code></p>
<p>Where <code>ENV</code> is a directory to place the new virtual environment.</p>
</blockquote>
<hr/>
<blockquote>
<p>What's the difference of venv and virtualenv?</p>
</blockquote>
<p><strong>venv</strong> and <strong>virtualenv</strong> are just shorter versions of <strong>virtual environment</strong>, meaning that <strong>venv</strong> is the same as <strong>virtualenv</strong> -- there is no difference.</p>
