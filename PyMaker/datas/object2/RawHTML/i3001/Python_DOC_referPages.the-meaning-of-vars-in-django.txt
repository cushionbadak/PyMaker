<p>In many functions in views.py the is a parameter called vars() that is passed to the render.
I want to know what is the use of this parameter and is it better than passing local variable individually to render?</p>
<pre><code>return render(request, 'report.html', vars())
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.djangoproject.com/en/2.0/topics/http/shortcuts/#render" rel="nofollow noreferrer"><strong><code>render():</code></strong></a>  accepts a 3rd argument to be <code>dictionary</code> </p>
<blockquote>
<p><strong>vars():</strong> 
  Return the <code>__dict__</code> attribute for a module, class, instance, or any other object with a <code>__dict__</code> attribute.</p>
</blockquote>
<p>So we can make use of <code>vars()</code> as an argument in <code>render()</code>.</p>
</div>
<span class="comment-copy">See Python documentation on <a href="https://docs.python.org/3/library/functions.html#vars" rel="nofollow noreferrer"><code>vars</code></a>.</span>
<span class="comment-copy">Also see <a href="https://stackoverflow.com/a/1901720/2011147">this answer</a> and please don't use <code>vars()</code> or <code>locals()</code> shortcuts as they may lead to sensitive data leaks and unmaintainable code.</span>
