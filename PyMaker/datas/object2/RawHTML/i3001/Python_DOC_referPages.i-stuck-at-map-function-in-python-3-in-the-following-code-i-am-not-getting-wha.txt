<p>Let's say I type <code>5</code> and then enter at the first prompt:</p>
<pre><code>n = int(input())
</code></pre>
<p>Would take the input "5" and make it into the integer <code>5</code>. So we are going from a <code>string</code> to an <code>int</code></p>
<p>Then we will get another input prompt because we have input() again in the next line:
This time I'll type <code>123 324 541 123 134</code> and then enter.</p>
<p>the <code>.split()</code> will split this into "123", "324", "541", "123", "134" which is a list (well a <code>map</code>) of strings. Then we'll map <code>int</code> onto them to give ourselves a <code>map</code> of <code>int</code>s rather than strings. <code>int</code> converts the strings to integers.</p>
<p>When checking out code it is often helpful to try things in a REPL (read execute print, looper). In your command promt just type <code>python</code> or <code>python3</code> if you have it installed or use <a href="http://repl.it" rel="nofollow noreferrer">replt.it</a>. Type <code>a = "123" + "321"</code> then try `a = int("123") + int("321") </p>
<p>Wrap this with list(map(int, input().split())) to get a <code>list</code> rather than a <code>map</code>.</p>

<div class="post-text" itemprop="text">
<ul>
<li><a href="https://docs.python.org/3/library/functions.html#map" rel="nofollow noreferrer">map(function, iterable)</a> takes a function which to apply to all elements of the list that is passed as second argument.</li>
<li><a href="https://docs.python.org/2/library/functions.html#int" rel="nofollow noreferrer">int(x)</a> is a function that returns an integer object constructed
from a number or string x.</li>
</ul>
<p>Therefore, it will just make a iterable where it applied the int() function to each string from <code>.split()</code>, meaning it casts every entry to int. </p>
<p>Example:</p>
<pre><code>arr = map(int, "12 34 56".split())
print(list(arr)) # prints: [12, 34, 56]
</code></pre>
