<div class="post-text" itemprop="text">
<p>I'm currently attempting to write a program that takes in a user input of units and magnitude and converts them to a user defines the unit. So basically just a unit converter (still doing beginner projects with Python).</p>
<p>But for the module <code>pint</code>, it uses a dot separator as the way to get the units, like so:</p>
<pre><code>from pint import *

ureg = UnitRegistry()
dist = 8 * ureg.meter
print(dist)
&gt;&gt;&gt; 8 meters
</code></pre>
<p>Now 8 is tied to meters. But say I want to let the user define the unit. So have a variable defined by an input attached to <code>ureg</code> like so:</p>
<pre><code>from pint import *

ureg = UnitRegistry()
inp = input()
&gt;&gt;&gt;inp = meter 
unit = ureg.inp # this is a variable defined by user
step = 8 * unit
print(step)

&gt;&gt;&gt;AttributeError: 'UnitRegistry' object has no attribute '_inp_'
</code></pre>
<p>The problem is python thinks I'm trying to access an attribute called <code>inp</code> from <code>ureg</code>, which doesn't exist. if <code>inp=meters</code>, why won't it work as <code>ureg.meters</code> worked? How can you attach variables to call attributes from a module?</p>
</div>
<div class="post-text" itemprop="text">
<p>You could use <a href="https://docs.python.org/3/library/functions.html#getattr" rel="nofollow noreferrer"><code>getattr(object, name)</code></a>:</p>
<pre><code>ureg = UnitRegistry()

user_value = input()
unit = getattr(ureg, user_value) # this is a variable defined by user
step = 8 * unit
print(step)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Or:</p>
<pre><code>ureg = UnitRegistry()
user_value = input()
unit = vars(ureg)[user_value] # this is a variable defined by user
step = 8 * unit
print(step)
</code></pre>
<p>Or:</p>
<pre><code>eval('ureg.%s()'%user_value)
</code></pre>
