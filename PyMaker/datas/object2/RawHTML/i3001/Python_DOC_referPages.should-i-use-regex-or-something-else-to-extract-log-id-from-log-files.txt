<p>I have list with the following data: </p>
<pre><code>["asdf mkol ghth", "dfcf 5566 7766", "7uy7 jhjh ffvf"]
</code></pre>
<p>I want to use regular expressions in python to get a list of tuples like this </p>
<pre><code>[("asdf", "mkol ghth"),("dfcf", "5566 7766"),("7uy7", "jhjh ffvf")]
</code></pre>
<p>I tried using re.split, but I am getting an error saying too many values to unpack. following is my code:</p>
<pre><code>logTuples = [()]
    for log in logList:
        (logid, logcontent) = re.split(r"(\s)", log)
        logTuples.append((logid, logcontent))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Regex is overkill here:</p>
<pre><code>l = ["asdf mkol ghth", "dfcf 5566 7766", "7uy7 jhjh ffvf"]

lst = [tuple(i.split(maxsplit=1)) for i in l]

print(lst)
</code></pre>
<p>Prints:</p>
<pre><code>[('asdf', 'mkol ghth'), ('dfcf', '5566 7766'), ('7uy7', 'jhjh ffvf')]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>From the documentation:</p>
<p><a href="https://docs.python.org/3/library/re.html" rel="nofollow noreferrer">https://docs.python.org/3/library/re.html</a></p>
<blockquote>
<p>\s </p>
<p>For Unicode (str) patterns: Matches Unicode whitespace characters
  (which includes [ \t\n\r\f\v], and also many other characters, for
  example the non-breaking spaces mandated by typography rules in many
  languages). If the ASCII flag is used, only [ \t\n\r\f\v] is matched.</p>
</blockquote>
<p>there are 2 whitespaces, and thus 3 items. </p>
<p>If all of your log entries have 3 items separated by spaces, and you always organize them as (1, 2 + ' ' + 3), you don't need to use a regex to format them as such:</p>
<pre><code>logtuples = []
for log in loglist:
    splitlog = log.split(" ") #3 total elements
    logtuples.append (splitlog[0], splitlog[1] + " " + splitlog[2])
</code></pre>
