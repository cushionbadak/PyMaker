<p>I'm trying to write a desktop application that takes the current time and adds increments of 90 minutes to tell the user when they should wake up if they go to bed now (sleep cycles occur reliably in 90 min cycles in most people). </p>
<p>To do this, I need the program to take the current datetime and add 6 repetitions of 90 minutes. I've tried several approaches thus far, here's what I feel is the closest I've come (using Python 3.7): </p>
<pre><code>import datetime

now = datetime.datetime.now()

for i (0,7):

    gotosleep = now + datetime.datetime(0, 0, 0[, 0[, (90*i)[, 0[, 0]]]])
    print(gotosleep)
</code></pre>
<p>I've also tried formatting <code>datetime.datetime(0, 0, 0[, 0[, (90*i)[, 0[, 0]]]])</code> as <code>datetime.datetime(0, 0, 0[ 0[ (90*i)[ 0[ 0]]]])</code> because I kept getting syntax errors, to no avail.</p>

<div class="post-text" itemprop="text">
<p>You could use timedelta:</p>
<pre><code>import datetime

now = datetime.datetime.now()

for i in range(7):

    gotosleep = now + datetime.timedelta(0,i*60*90)
    print(gotosleep)
</code></pre>
<p>Output:</p>
<pre><code>2018-08-02 09:26:11.631513
2018-08-02 10:56:11.631513
2018-08-02 12:26:11.631513
2018-08-02 13:56:11.631513
2018-08-02 15:26:11.631513
2018-08-02 16:56:11.631513
2018-08-02 18:26:11.631513
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Using <a href="https://docs.python.org/3/library/datetime.html#timedelta-objects" rel="nofollow noreferrer"><code>datetime.timedelta</code></a> we can sum the desired interval to the current time.</p>
<pre><code>from datetime import timedelta, datetime

sleep_cycles = 7    
result = datetime.now() + timedelta(minutes=90 * sleep_cycles)
</code></pre>
<p><strong>Further notes</strong></p>
<p>I see that in your question you wrote <code>datetime.datetime(0, 0, 0[, 0[, (90*i)[, 0[, 0]]]])</code>. This suggests a difficulty reading the documentation: when a syntax such as <code>[, argument]</code> appears in the documentation, it means that the given argument is optional, <em>not</em> that the argument has to be wrapped in squared brackets.</p>
<p>For the same reason, you do not need to add all these arguments. For example when you'd like to use the second optional argument your would proceed as follows:</p>
<pre><code>datetime.datetime(
    required_1,
    required_2,
    required_3,
    optional_1,
    optional_2
)
</code></pre>
<p>I hope you can now read the documentation more clearly.</p>
