<p>This condition doesn't do what you think it does:</p>
<pre><code>&gt;&gt;&gt; n = 18
&gt;&gt;&gt; n &gt;= 2 &amp; n &lt;= 5
True
</code></pre>
<p>It is actually doing this:</p>
<pre><code>&gt;&gt;&gt; n &gt;= (2 &amp; n) &lt;= 5
True
</code></pre>
<p>Proof:</p>
<pre><code>&gt;&gt;&gt; import ast
&gt;&gt;&gt; ast.dump(ast.parse('n &gt;= 2 &amp; n &lt;= 5'), annotate_fields=False)
"Module([Expr(Compare(Name('n', Load()), [GtE(), LtE()], [BinOp(Num(2), BitAnd(), Name('n', Load())), Num(5)]))])"
&gt;&gt;&gt; ast.dump(ast.parse('n &gt;= (2 &amp; n) &lt;= 5'), annotate_fields=False)
"Module([Expr(Compare(Name('n', Load()), [GtE(), LtE()], [BinOp(Num(2), BitAnd(), Name('n', Load())), Num(5)]))])"
</code></pre>
<p>docs reference on operator precedence is <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer">here</a>.</p>
<p>Instead, use this:</p>
<pre><code>2 &lt;= n &lt;= 5
</code></pre>
