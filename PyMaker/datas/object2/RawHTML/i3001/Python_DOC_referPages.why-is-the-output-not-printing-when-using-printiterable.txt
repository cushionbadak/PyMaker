<div class="post-text" itemprop="text">
<p>From the Python documentation:</p>
<p><a href="https://docs.python.org/3/library/functions.html?highlight=reversed#reversed" rel="nofollow noreferrer"><strong>reversed(seq)</strong></a></p>
<blockquote>
<p>Return a reverse iterator. seq must be an object which has a
  <code>__reversed__()</code> method or supports the sequence protocol (the
  <code>__len__()</code> method and the <code>__getitem__()</code> method with integer
  arguments starting at 0).</p>
</blockquote>
<p><a href="https://docs.python.org/3/library/stdtypes.html#list" rel="nofollow noreferrer"><strong>list([iterable])</strong></a></p>
<blockquote>
<p>The constructor builds a list whose items are the same and in the same
  order as iterable’s items. iterable may be either a sequence, a
  container that supports iteration, or an iterator object. If iterable
  is already a list, a copy is made and returned, similar to
  iterable[:]. For example, <code>list('abc')</code> returns <code>['a', 'b', 'c']</code> and
  <code>list( (1, 2, 3) )</code> returns <code>[1, 2, 3]</code>. If no argument is given, the
  constructor creates a new empty list, <code>[]</code>.</p>
</blockquote>
<p>So:</p>
<p><code>reversed(string)</code> will return an iterator that travels backwards through the string. <code>list(iterator)</code> turns that iterator into a list. <code>list(string)</code> will split a string up into its constituent characters.</p>
<p>Thus</p>
<p><code>list('india')</code> returns <code>['i', 'n', 'd', 'i', 'a']</code></p>
<p><code>reversed('india')</code> returns <code>&lt;reversed object at 0x1090b48d0&gt;</code>, which is an iterator</p>
<p>you can convert the iterator into a list using the <code>list</code> operator:</p>
<p><code>list(reversed('india'))</code> returning  <code>['a', 'i', 'd', 'n', 'i']</code></p>
<p>or you could iterate through it:</p>
<pre><code>for n in reversed('india'):
   print(n)
</code></pre>
<p>which prints out</p>
<pre><code>a
i
d
n
i
</code></pre>

<div class="post-text" itemprop="text">
<p>If you want the reversed string, try the following:</p>
<pre><code>string = ''.join(reversed(s))
</code></pre>
<p>This is because reversed outputs an iterator, which doesn't hold values, but rather computes them on the fly. The ''.join part consumes the generator and makes a string.</p>
<p>See <a href="https://wiki.python.org/moin/Generators" rel="nofollow noreferrer">here</a> for more information on generators/iterators.</p>
