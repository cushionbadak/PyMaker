<pre><code>print('C:/Users/Ratul/Downloads/Machine_Learning_Data/reddit_data/reddit_data/{}/RC_{}'.format(
       timeframe.split('-')[0],timeframe))

<p>instead of <code>open</code>. Check if all exists - because for some of your values <em>it does not exist</em>. Hence the error. </p>
<p>If most of your files exist, it is far easier to handle the error itself:</p>
<pre><code>myname = 'C:/Users/Ratul/Downloads/Machine_Learning_Data/reddit_data/reddit_data/{}/RC_{}'.format(timeframe.split('-')[0],timeframe)

<p>The <code>open()</code> command does not care about you using <code>\</code> or <code>/</code> - if using <code>\</code> you should escape it or use raw strings (aka: <code>r'C:\some\dir\file.txt'</code>) - your syntax is ok as is - open() will use the appropriate directory delimiters under windows even if you give it <code>'c:/somedir/file.txt'</code></p>
<p>Readup: <a href="https://docs.python.org/3/tutorial/errors.html#handling-exceptions" rel="nofollow noreferrer">About error handling</a></p>
