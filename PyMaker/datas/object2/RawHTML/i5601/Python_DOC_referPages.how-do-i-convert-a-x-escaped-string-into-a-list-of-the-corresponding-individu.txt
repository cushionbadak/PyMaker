<p>I have strings of ASCII characters represented like this:</p>
<pre><code>'\\xaa\\xaa\\x02\\xba\\x04A\\xaa\\xaa\\x02\\xbc\\x05&gt;\\xaa\\xaa\\x04\\x80\\x02\\xf8\\x00\\x85\\xaa\\xaa\\x04\\x80\\x02\\xf8'
</code></pre>
<p>I would like to convert these strings to lists where each element of the list is an ASCII character, and the order and values of the characters are conserved from the original string:</p>
<pre><code>['\xaa', '\xaa', '\x02', '\xba', '\x04', 'A', '\xaa', '\xaa', '\x02', '\xbc', '\x05', '&gt;', '\xaa', '\xaa', '\x04', '\x80', '\x02', '\xf8', '\x00', '\x85', '\xaa', '\xaa', '\x04', '\x80', '\x02', '\xf8']
</code></pre>
<p>This wouldn't be too difficult if every character was in the form <code>\xa0</code> (representing a hex byte), but sometimes ASCII escape characters such as <code>\n</code> newline and <code>\\</code> backslash are included. This throws off the converter method I wrote.</p>
<p>Is there a simple built-in function that can do what I desire? Or is there some package I can download? Or does someone have a simple code solution?</p>

<div class="post-text" itemprop="text">
<p>What you appear to have there is a string containing the Python representation of another string. One way to decode that is to use <a href="https://docs.python.org/3/library/codecs.html#codecs.decode" rel="nofollow noreferrer"><code>codecs.decode()</code></a> with the <code>unicode_escape</code> <a href="https://docs.python.org/3/library/codecs.html#text-encodings" rel="nofollow noreferrer">encoding</a>:</p>
<pre><code>&gt;&gt;&gt; s = '\\xaa\\xaa\\x02\\xba\\x04A\\xaa\\xaa\\x02\\xbc\\x05 ... \\x02\\xf8'
&gt;&gt;&gt; import codecs
&gt;&gt;&gt; t = codecs.decode(s, 'unicode_escape')
&gt;&gt;&gt; t
'ªª\x02º\x04Aªª\x02¼\x05 ... \x02ø'
</code></pre>
<p>Note that although printable non-ASCII characters are represented here as-is, rather than as <code>\xNN</code> escapes, they are literally identical to those escapes:</p>
<pre><code>&gt;&gt;&gt; 'ª' is '\xaa'
True
</code></pre>
<p>Once you have your decoded string <code>t</code>, it's trivial to convert it into a list:</p>
<pre><code>&gt;&gt;&gt; list(t)
['ª', 'ª', '\x02', 'º', '\x04', 'A', 'ª', 'ª', '\x02', '¼', '\x05', ..., '\x02', 'ø']
</code></pre>
<p><strong>However:</strong> I can't help but wonder why you need to decode this string in the first place. There aren't many circumstances where it makes sense to be passing the result of <code>repr(some_string)</code> around, so it's possible you have an <a href="https://meta.stackexchange.com/questions/66377/what-is-the-xy-problem">XY Problem</a>. You should consider whether it would make more sense not to generate these repr-strings in the first place.</p>
