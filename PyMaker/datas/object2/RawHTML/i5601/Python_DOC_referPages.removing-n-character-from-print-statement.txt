<div class="post-text" itemprop="text">
<p>You can set the <code>end</code> parameter of <code>print</code> to be an empty string (or some other character):</p>
<pre><code>print(details, file=userquiz, end='')
</code></pre>
<p>From the <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow noreferrer">docs</a>, you can see that it defaults to a newline:</p>
<blockquote>
<p><code>print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)</code></p>
<p>Print <code>objects</code> to the text stream <code>file</code>, separated by <code>sep</code> and followed by <code>end</code>. <code>sep</code>, <code>end</code>, <code>file</code> and <code>flush</code>, if present, must be given
  as keyword arguments.</p>
</blockquote>
<p>This is what is being printed to the file currently.</p>
</div>
<div class="post-text" itemprop="text">
<p>If you are using <code>python 2.x</code>, you can do the following:</p>
<pre><code>print &gt;&gt; userquiz, details, # &lt;- notice the comma at the end
</code></pre>
<p>If using <code>pytnon 3.x</code>, you can do this:</p>
<pre><code>print(details,file=userquiz, end = " ")
</code></pre>
<p>Check <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow noreferrer">print</a> documentation. </p>
