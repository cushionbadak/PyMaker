<div class="post-text" itemprop="text">
<p>I am working on a project with following tree structure:</p>
<pre><code>Project
  app
    main.py
    svg_to_png.py
    __init__.py
    tests
       __init__.py
       unittests
          basic_test.py
          __init__.py
</code></pre>
<p>I am importing app functions to basic_tests by using </p>
<pre><code>from app.main import *
</code></pre>
<p>when I execute the <code>basic_test.py</code>, it gives the following error:</p>
<pre><code>ImportError: No module named 'app'
</code></pre>
<p>Also, while in the top directory of the project, when I type <code>python3 -m app.main</code>, the program executes for a while, then gives the same import error at the following line</p>
<p><code>from app.svg_to_png import do_svg2png</code></p>
</div>
<div class="post-text" itemprop="text">
<p>You can't import from higher up the directory structure that way. One way to do it would be</p>
<pre><code>import sys
sys.path.append("../..")
from main import *
</code></pre>
<p>For the second import, you'd want to do</p>
<pre><code>from .svg_to_png import do_svg2png
</code></pre>
<p>See <a href="https://docs.python.org/3/reference/import.html#submodules" rel="nofollow noreferrer">this</a> section of the Python documentation specifically</p>
</div>
