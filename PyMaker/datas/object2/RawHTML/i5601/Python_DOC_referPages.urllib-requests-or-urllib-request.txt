<p><code>urllib</code> is a module one it own, same thing for <code>requests</code>. 
you do : </p>
<pre><code>import requests
from urllib.request import urlopen
</code></pre>
<p>It's <code>urllib.request</code> not <code>urllib.requests</code> and it explains the error the python interpreter is giving you. 
Here's the <a href="https://docs.python.org/3/library/urllib.request.html" rel="nofollow noreferrer">docs</a> for <code>urllib</code></p>
<p>As for the first shell error you're getting, python is already giving you a hint as how to solve it.
So, instead of : </p>
<pre><code>bsObj = BeautifulSoup(html.read())
</code></pre>
<p>You should do this : </p>
<pre><code>bsObj = BeautifulSoup(html.read(),"html.parser")
</code></pre>

<p>Just use <code>requests</code> module, this is not a built-in module but a third party library.</p>
<pre><code>import requests
from bs4 import BeautifulSoup

html = requests.get("http://pythonscraping.com/pages/page1.html").text
bsObj = BeautifulSoup(html, "html.parser")
print (bsObj.h1)
</code></pre>
<p>By the way that warning you have has nothing to do with <code>request(s)</code> library, but it is with the <code>BeautifulSoup</code> parser.</p>
