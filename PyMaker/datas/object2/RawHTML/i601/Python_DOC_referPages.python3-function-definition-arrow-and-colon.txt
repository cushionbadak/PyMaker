<div class="post-text" itemprop="text">
<p>This is an instance of python3 type hinting. The use of <code>-&gt; 'None'</code> indicates that the function does not have a return statement. </p>
<p><code>List[str]</code> is more interesting: The <code>List</code> part indicates it will return a list type, and its argument <code>[str]</code> indicates it is a parameterized type. In practice, python lists may contain any type of object, but in strongly-typed language a list is a homogeneous collection.</p>
<p>Using this hint both indicates to a caller of the function that <code>s</code> must contain only strings, thus avoiding any exceptions for whatever operation will be performed, and it also indicates to an intelligent IDE (e.g. PyCharm, VSCode) that the objects contained in the list have string instance methods for autocompletion indicators.</p>
<p>The python interpreter does not do anything with this information in terms type checking, however the <a href="http://mypy-lang.org/" rel="nofollow noreferrer">mypy</a> interpreter will typecheck your code.</p>
<p>For more information see <a href="https://www.python.org/dev/peps/pep-0484/" rel="nofollow noreferrer">PEP 484</a> and the <a href="https://docs.python.org/3/library/typing.html" rel="nofollow noreferrer">typing</a> module, which has also been <a href="https://pypi.org/project/typing/" rel="nofollow noreferrer">backported</a> to pre-3.5 python3 and 2.7.</p>
