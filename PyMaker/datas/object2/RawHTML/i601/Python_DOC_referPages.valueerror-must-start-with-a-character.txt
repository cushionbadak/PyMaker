<p>It seems that the paths you are supplying (for example <code>"D:\python learning\IMG_20130614_000526.jpg"</code>) are intended as default values for the arguments <code>-i</code>, <code>-p-</code>, <code>-m</code>. If that is what you are trying to do, specify them as defaults. Your code is specifying them as <em>argument names</em> (like <code>--confidence</code>) which is why <code>argparse</code> is telling you they must begin with a hyphen.</p>
<p>For example:</p>
<pre><code>ap.add_argument("-i", "--input_image", required=True, help="path to input image", default=r"D:\python learning\IMG_20130614_000526.jpg")
</code></pre>

<span class="comment-copy">The positional arguments to <code>add_argument</code> are argument names given on the command line, like you have correctly for <code>"--confidence"</code>. You seem to be trying to add argument values, which is a misunderstanding of how argparse works. See <a href="https://docs.python.org/3/library/argparse.html" rel="nofollow noreferrer">argparse</a> for how to use this library.</span>
