<p>The argument to <code>set</code> has to be an iterable whose elements are the desired set items. <code>set( (0,0) )</code> has an iterable as an argument, but its elements are 0 and 0, not a tuple <code>(0,0)</code>. You need <code>set([(0,0)])</code>. The same goes for <code>deque</code>; you need an iterable <em>containing</em> the tuple <code>((0, 0), [ (0, 0) ] )</code>, so</p>
<pre><code>collections.deque([((0, 0), [ (0, 0) ] )])
</code></pre>
<p>As for the error you are seeing, <code>function</code> doesn't have an <code>__mro__</code>, because it doesn't need one. It's base class is always <code>object</code>, and you cannot inherit from <code>function</code> to create a class whose MRO would need to access <code>function</code>'s hypothetical MRO. </p>
<p>As to <em>why</em> your data class raises that error, it's because the <code>dataclass</code> decorator has to decorate a <em>class</em>, not a function.</p>
<pre><code>from dataclasses import dataclass

@dataclass
class DequeItems:
    current:tuple
    path:list

a = DequeItems(
        current=(0, 0),
        path=[(0, 0)]
    )
print(a)
</code></pre>
