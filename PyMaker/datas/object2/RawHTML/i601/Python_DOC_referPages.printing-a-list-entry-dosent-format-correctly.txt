<div class="post-text" itemprop="text">
<p>You're using <a href="https://docs.python.org/3/library/random.html#random.sample" rel="nofollow noreferrer"><code>random.sample(iterable, k)</code></a> which always returns a list of length <code>k</code> or in your case 1. So the object you're printing is a list of a string.</p>
<p>You could change the line</p>
<pre><code>print (random.sample(listofrecipes, 1))
</code></pre>
<p>to something like</p>
<pre><code>print(random.sample(listofrecipes,1)[0])  # note the [0]
</code></pre>
<p>That will return the <em>first</em> element in the list (which happens to be the only one).</p>
<p>A cleaner solution might be instead to use <a href="https://docs.python.org/3/library/random.html#random.choice" rel="nofollow noreferrer"><code>random.choice(iterable)</code></a>. It always returns a single choice, not a list. It is effectively equivalent as the line above, but it's a little clearer.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>random.sample</code> <a href="https://docs.python.org/3.7/library/random.html#random.sample" rel="nofollow noreferrer">returns a list</a>, which is exactly what you're getting. This output is just how lists are formatted. If you want to extract the string from this list, do it:</p>
<pre><code>random.sample(listofrecipes, 1)[0]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The output of <code>random.sample</code> is a list of a certain length, 1 in this instance. When you print this you are seeing the output that is generated when displaying a list, including newline <code>\n</code> characters.</p>
<pre><code>print(random.sample(listofrecipes, 1)[0])
</code></pre>
<p>would print out the string, the first element of the list, which would reinstate the original display across several lines.</p>
</div>
<span class="comment-copy">Where and how is listofrecipes defined? The definition of <code>random.sample</code> is to return a list of a certain length; that is what you have, a list consisting of a single item.</span>
<span class="comment-copy">If you <code>print (random.sample(listofrecipes, 1)[0])</code> you will probably see the original formatting. The output of the list shows the list structure and newline \n characters.</span>
<span class="comment-copy">An alternative solution that comes to mind (if you ever wanted to get 2 or more random recipes at once) would be to join them all together with a newline delimiter. Something like <code>'\n'.join(random.sample(listofrecipes, 2)</code> would print two random ones, one on top of the other.</span>
