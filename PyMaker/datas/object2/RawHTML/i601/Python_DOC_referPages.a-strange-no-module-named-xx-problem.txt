<p>For a better explanation, I've created the following folder structure:</p>
<pre><code>test/     
-- __init__.py
-- greeter.py # Greets in German
-- subfolder/  
    -- __init__.py
    -- greeter.py # Greets in Italian
    -- test.py
    -- deepfolder/
        -- __init__.py
        -- greeter.py # Greets in France
</code></pre>
<p>As you may notice, we have 3 files with the same name, each one greets in a different language using a function with the same name. The only function in a <code>greeter.py</code> file is:  </p>
<pre><code>def says():
    print("Hello World!")
</code></pre>
<p><strong>IMPORT FROM THE SAME FOLDER</strong><br/>
If from <code>test.py</code> file we import <code>greeter</code> and run the <code>says</code> function, we'll have:</p>
<pre><code>import greeter as greeter

greeter.says()
</code></pre>
<p><strong>Output:</strong> </p>
<pre><code>Buongiorno Mondo! # Italian output
</code></pre>
<p><strong>IMPORT FROM A SUBFOLDER</strong><br/>
But what if we want to import from a subfolder?<br/>
To import from a subfolder (i.e., <code>deepfolder/</code>), we simply add an empty <code>__init__.py</code> file into the folder, then we can specify the path in the import:</p>
<pre><code>import deepfolder.greeter as greeter

greeter.says()
</code></pre>
<p><strong>Output:</strong> </p>
<pre><code>Bonjour le Monde! # France output
</code></pre>
<p><strong>IMPORT FROM A PARENT FOLDER</strong><br/>
At last, you may want to import from a parent folder.<br/>
You should try to have your <code>main</code> running file at the top of the folder tree, but things happens and you find yourself looking to import a module from a parent folder.<br/>
For doing this, you need to add the parent folder to the sys.path:</p>
<pre><code>import sys
sys.path.append("/path/to/dir")
from test import greeter as greeter

greeter.says()
</code></pre>
<p><strong>Output:</strong> </p>
<pre><code>Guten Morgen Welt! # German output
</code></pre>
<p>Importing scripts and modules isn't really the most pythonic way to solve things, you may want to have a look on <a href="https://docs.python.org/3/tutorial/modules.html" rel="nofollow noreferrer">Python's documentation about packages</a>.</p>
<p><strong>TL;DR</strong><br/>
In your <code>project/example.py</code> use</p>
<pre><code>import swmm5_extend_function.swig.SWMM5ReadInpFile as swmm
</code></pre>
<p>instead of</p>
<pre><code>import swig.SWMM5ReadInpFile as swmm
</code></pre>
