<div class="post-text" itemprop="text">
<p>I'm searching a method in python to show me the platform details. So I find <code>platform.system()</code>, but I got shock when I saw <a href="https://docs.python.org/3/library/platform.html#platform.system" rel="nofollow noreferrer">the help doc</a>:</p>
<blockquote>
<p>system()</p>
<p>Returns the system/OS name, e.g. 'Linux', 'Windows' or 'Java'.</p>
</blockquote>
<p>Any idea on why 'Java' is listed there but not 'Darwin'?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>Darwin</code> is indeed one of the possible outputs of the <code>platform.system()</code> method. I did a quick Google search and came up with <a href="https://pymotw.com/2/platform/" rel="nofollow noreferrer">this</a>.</p>
<pre><code>import platform

print 'uname:', platform.uname()
print 'system   :', platform.system()
print 'version  :', platform.version()
</code></pre>
<p>Java is probably there for Jython implementations, as someone pointed out above.</p>
<p><strong>EDIT</strong>: Further Googling led me to this: </p>
<p><a href="https://i.stack.imgur.com/5tj1a.png" rel="nofollow noreferrer"><img alt="Jython proof" src="https://i.stack.imgur.com/5tj1a.png"/></a></p>
<p><code>platform.system()</code> uses <code>sys.platform</code>, so I think this is where the <code>Java</code> comes from.
This excerpt is from the book <strong>Jython Essentials: Rapid Scripting in Java</strong> and therefore can be regarded as conclusive. <a href="https://books.google.co.in/books?id=Dm1gsjr1gN8C&amp;pg=PA180&amp;lpg=PA180&amp;dq=sys.platform%20java&amp;source=bl&amp;ots=czfqqWn2K7&amp;sig=KZC4WH7rPGrQZraOtWVf9VcHb5c&amp;hl=en&amp;sa=X&amp;ved=0ahUKEwih0dmi783RAhUGTI8KHby5CY0Q6AEITTAI#v=onepage&amp;q=sys.platform%20java&amp;f=false" rel="nofollow noreferrer">Link included for reference</a>.</p>
