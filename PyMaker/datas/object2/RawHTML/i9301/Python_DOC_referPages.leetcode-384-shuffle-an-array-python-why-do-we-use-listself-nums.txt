<pre><code>import random

class Solution(object):
    def __init__(self, nums):
        """
        :type nums: List[int]
        :type size: int
        """
        self.nums = nums

    def reset(self):
        """
        Resets the array to its original configuration and return it.
        :rtype: List[int]
        """
        return self.nums

    def shuffle(self):
        """
        Returns a random shuffling of the array.
        :rtype: List[int]
        """
        ret = list(self.nums)
        random.shuffle(ret)
        return ret
</code></pre>

<div class="post-text" itemprop="text">
<p>Calling <a href="https://docs.python.org/3.1/library/functions.html?highlight=list#list" rel="nofollow noreferrer"><code>list()</code></a> here will <em>create a copy</em> of <code>self.nums</code> (see <a href="https://stackoverflow.com/q/2612802/78845">How to clone or copy a list?</a> ):</p>
<pre><code>&gt;&gt;&gt; l = [1, 2, 3]
&gt;&gt;&gt; x = l
&gt;&gt;&gt; l.append(4)
&gt;&gt;&gt; x, l
([1, 2, 3, 4], [1, 2, 3, 4])
&gt;&gt;&gt; y = list(l)
&gt;&gt;&gt; l.append(5)
&gt;&gt;&gt; y, l
([1, 2, 3, 4], [1, 2, 3, 4, 5])
</code></pre>
<p>This means that <code>shuffle()</code> leaves the underlying list of numbers untouched while returning a shuffled list (since <a href="https://docs.python.org/3/library/random.html#random.shuffle" rel="nofollow noreferrer"><code>random.shuffle()</code></a> shuffles the input <em>in place</em>).</p>
<p>Keeping this list untouched allows the <code>reset()</code> method to work.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>ret = list(self.nums)</code> creates a new list since shuffle does a in place shuffle. So <code>ret</code> holds a copy of the original list which gets shuffled and returned and <code>self.nums</code> holds the contents of the original list.</p>
