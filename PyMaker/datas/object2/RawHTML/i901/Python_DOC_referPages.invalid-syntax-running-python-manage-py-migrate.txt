<div class="post-text" itemprop="text">
<p>In the traceback we see that the caret points to a <a href="https://www.python.org/dev/peps/pep-0498/" rel="nofollow noreferrer"><em>string interpolation</em> [PEP-0498]</a>. This feature is supported since <a class="post-tag" href="/questions/tagged/python-3.6" rel="tag" title="show questions tagged 'python-3.6'">python-3.6</a>, but you run your code with:</p>
<pre><b><s>python3.5</s></b> manage.py runserver</pre>
<p>so the interpreter does not understand this string interpolation.</p>
<p>You basically have two options:</p>
<ol>
<li><p>try to run this with <a class="post-tag" href="/questions/tagged/python-3.6" rel="tag" title="show questions tagged 'python-3.6'">python-3.6</a> (or higher), although there is of course always a risk that something is broken then:</p>
<pre><b>python3.6</b> manage.py runserver</pre></li>
<li><p>perform proper formatting, replace:</p>
<pre><code>prefix=<s>f"{org.id}/"</s></code></pre>
<p>with:</p>
<pre><code>prefix=<b>"{}/".format(org.id)</b></code></pre></li>
</ol>
<p>or some other way to format this.</p>
</div>
<div class="post-text" itemprop="text">
<p>It looks like you're using format strings with python 3.5, however support for formatted string literals was not added until python 3.6. More information can be found in the <a href="https://docs.python.org/3/reference/lexical_analysis.html#f-strings" rel="nofollow noreferrer">python documentation</a>. You will either need to update your python version or format the string another way. e.g. <code>"{}/".format(org.id)</code></p>
