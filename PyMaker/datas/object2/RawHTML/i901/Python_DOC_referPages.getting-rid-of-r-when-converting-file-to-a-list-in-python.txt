<p>The key thing to notice is that python only reads one big line with all the <code>\r</code> characters embedded within. Based on that, I'm guessing you're using Python 2.x, which didn't enable <a href="https://docs.python.org/2/glossary.html#term-universal-newlines" rel="nofollow noreferrer">universal newlines</a> mode by default. Changing your mode to <code>rU</code> should do what you're expecting:</p>
<pre><code>with open(filename, 'rU') as f:
    for i in f:
        i = i.rstrip().split("\t")
        print(i)
</code></pre>
<p>For more information, see the <a href="https://docs.python.org/2/library/functions.html#open" rel="nofollow noreferrer"><code>open()</code></a> documentation.</p>

<p>as suggested, the <code>csv</code> module is good for dealing with this sort of data.  I'd do something like:</p>
<pre><code>import csv

with open(filename) as fd:
  inp = csv.reader(fd, delimiter='\t')

  header = next(inp)
  print(header)

  for row in inp:
    print(row)
</code></pre>
<p>Python has support for magic <a href="https://docs.python.org/3/glossary.html#term-universal-newlines" rel="nofollow noreferrer">universal newlines</a> which means it does something sensible with "old-style" Mac line-endings by default.  your can then use the <code>csv</code> module with a custom delimiter to parse the tab delimited file</p>
