<p>Not looking for a work around. Looking to understand why Python sorts this way.</p>
<pre><code>&gt;&gt;&gt; a = ['aaa','Bbb']
&gt;&gt;&gt; a.sort()
&gt;&gt;&gt; print(a)
['Bbb', 'aaa']

&gt;&gt;&gt; a = ['aaa','bbb']
&gt;&gt;&gt; a.sort()
&gt;&gt;&gt; print(a)
['aaa', 'bbb']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>This is because upper case chars have an ASCII value lower than that of lower case. And hence if we sort them in increasing order, the upper case will come before the lower case </p>
<ul>
<li>ASCII of <code>A</code> is 65</li>
<li>ASCII of <code>a</code> is 97</li>
</ul>
<p>65&lt;97</p>
<p>And hence <code>A &lt; a</code> if you sort in increasing order </p>
</div>
<div class="post-text" itemprop="text">
<p><code>str</code> is sorted based on the raw byte values (Python 2) or Unicode ordinal values (Python 3); in ASCII and Unicode, all capital letters have lower values than all lowercase letters, so they sort before them:</p>
<pre><code>&gt;&gt;&gt; ord('A'), ord('Z')
(65, 90)
&gt;&gt;&gt; ord('a'), ord('z')
(97, 112)
</code></pre>
<p>Some locales (e.g. <code>en_US</code>) will change this sort ordering; if <a href="https://docs.python.org/3/library/locale.html#locale.strxfrm" rel="nofollow noreferrer">you pass <code>locale.strxfrm</code> as the <code>key</code> function</a>, you'll get case-insensitive sorts on those locales, e.g.</p>
<pre><code>&gt;&gt;&gt; import locale
&gt;&gt;&gt; locale.setlocale(locale.LC_COLLATE, 'en_US.utf-8')
&gt;&gt;&gt; a.sort(key=locale.strxfrm)
&gt;&gt;&gt; a
['aaa', 'Bbb']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Python treats uppercase letters as lower than lowercase letters. If you want to sort ignoring the case sensitivity. You can do something like this:</p>
<pre><code>a = ['aaa','Bbb']
a.sort(key=str.lower)
print(a)

Outputs:
['aaa', 'Bbb']
</code></pre>
<p>Which ignores the case sensitivity. The key parameter "str.lower" is what allows you to do this. The following documentation should help. <a href="https://docs.python.org/3/howto/sorting.html" rel="nofollow noreferrer">https://docs.python.org/3/howto/sorting.html</a></p>
</div>
