<p>I want to set up my <code>tkinter</code> program so that it will call a specific function (in this case <code>self.executecommand()</code> when I press the enter key, to eliminate the need for the <code>self.open_button</code>. Is there a way to make my code so that when I press the enter key, it will call the function?</p>
<p>This code (part of a larger peice of code) basically executes commands in windows. It is made to run on machines that don't allow users to run cmd.exe. I've looked around on stack overflow, stack exchange and some other websites but I can't find anything that clearly fits into my code.</p>
<pre><code>class App2(tk.Frame):
    def __init__(self, master=None):
        super().__init__(master)
        self.master = master
        self.grid()
        self.create_buttons()

    def create_buttons(self):
        self.open_button = tk.Button(self)
        self.open_button['text'] = 'Run'
        self.open_button.grid(row=1, column=2)
        self.open_button['command'] = self.executecommand

        self.text_input = tk.Entry(self)
        self.text_input.grid(row=1, column=1)

    def executecommand(self):
        self.command = self.text_input.get()
        from subprocess import call
        call(self.command)
root = tk.Tk()
app = App2(master=root)
app.mainloop()

<div class="post-text" itemprop="text">
<p>You can create a binding to the <code>&lt;Key-Return&gt;</code> event:</p>
<pre><code>    self.text_input.bind("&lt;Key-Return&gt;",
                         lambda event: self.executecommand())
</code></pre>
<p>See <a href="https://www.tcl.tk/man/tcl8.4/TkCmd/bind.htm" rel="nofollow noreferrer"><code>bind</code></a> and the <a href="https://www.tcl.tk/man/tcl8.4/Keywords/B.htm#binding" rel="nofollow noreferrer"><code>keysyms</code></a>.  Note that unlike the <code>command</code> attribute, bindings are called with an event argument, hence the use of the lambda.</p>
<p><a href="https://docs.python.org/3/library/subprocess.html#subprocess.call" rel="nofollow noreferrer"><code>subprocess.call</code></a> expects a sequence as the command argument, so you probably to use something like this:</p>
<pre><code>    call([self.command], shell=True)
</code></pre>
<p>(In general, <code>shell=True</code> is insecure, but it is probably okay in this context.)</p>
