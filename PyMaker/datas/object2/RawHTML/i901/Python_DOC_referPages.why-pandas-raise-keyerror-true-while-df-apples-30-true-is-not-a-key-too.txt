<p>Based on official pandas documentations and help on:</p>
<pre><code>pandas.DataFrame.loc
</code></pre>
<p>the loc allows to access a group of rows and columns by label(s) or a Boolean array.
(<a href="https://pandas.pydata.org/pandas-docs/stable/generated/pandas.DataFrame.loc.html" rel="nofollow noreferrer">https://pandas.pydata.org/pandas-docs/stable/generated/pandas.DataFrame.loc.html</a>)</p>
<p>but it look a Boolean array does not work really!</p>
<pre><code>import pandas as pd

data=[[30, 21],[0, 121],[4, 121]]
df= pd.DataFrame(data, columns=['Apples', 'Bananas'])

print(df.loc[df.Apples==30 &amp; True])      # It's OK
print(df.loc[True])                      # **doesn't work**
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The answer is in the <a href="https://pandas.pydata.org/pandas-docs/stable/generated/pandas.DataFrame.loc.html#pandas-dataframe-loc" rel="nofollow noreferrer"><code>loc</code> docs</a> <em>you</em> linked to:</p>
<blockquote>
<p>Allowed inputs are:</p>
<ul>
<li>[...]    </li>
<li>A boolean array of the <strong>same length as the axis being sliced</strong>, e.g. <code>[True, False, True]</code>.</li>
</ul>
</blockquote>
<p>If the mask is not of the same length, it assumes you are searching for a label in the column, and failing to find it, raises a <code>KeyError</code>.</p>
<hr/>
<p>Note that <code>df.Apples == 30 &amp; True</code> has the same length as <code>df</code>.</p>
<pre><code>df.Apples == 30 &amp; True
</code></pre>
<p>Evaluates to </p>
<pre><code>df.Apples == 0
</code></pre>
<p>Which is </p>
<pre><code>0    False
1     True
2    False
Name: Apples, dtype: bool
</code></pre>
<p>Because <code>==</code> has lower precedence than <code>&amp;</code>. See <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer">Operator Precedence</a>.</p>
</div>
