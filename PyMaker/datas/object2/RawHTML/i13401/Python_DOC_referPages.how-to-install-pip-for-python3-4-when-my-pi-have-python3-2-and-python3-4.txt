<p>How to install pip for python3.4 when my pi have python3.2 and python3.4</p>
<p>when I used <code>sudo install python3-pip</code></p>
<p>it's only for python3.2</p>
<p>but I want install pip for python3.4</p>
</div>
<div class="post-text" itemprop="text">
<p>Python 3.4 has <code>pip</code> included, see <a href="https://docs.python.org/3/whatsnew/3.4.html#whatsnew-pep-453" rel="nofollow"><em>What's New in Python 3.4</em></a>.</p>
<p>Just execute:</p>
<pre><code>python3.4 -m ensurepip
</code></pre>
<p>to install it if it is missing for you. See the <a href="https://docs.python.org/3/library/ensurepip.html" rel="nofollow"><code>ensurepip</code> module documentation</a> for further details.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can go to your python 3.4 directory scripts and run it's pip in: <br/></p>
<p><code>../python3.4/scripts</code></p>
</div>
<div class="post-text" itemprop="text">
<p>You should compile python 3.4 and use venv for python3 environment:</p>
<ol>
<li><p>Check if you have installed required dependencies:</p>
<pre><code>sudo apt-get install build-essential

sudo apt-get install libc6-dev libreadline-dev libz-dev libncursesw5-dev libssl-dev libgdbm-dev libsqlite3-dev libbz2-dev liblzma-dev tk-dev
</code></pre></li>
<li><p>Download and compile Python 3.4.3. You should not <code>sudo make install</code> it because we don't need it systemwide: </p>
<pre><code>wget https://www.python.org/ftp/python/3.4.3/Python-3.4.3.tgz
tar -xvf Python-3.4.3.tgz
cd Python-3.4.3 
./configure &amp;&amp; make
</code></pre>
<p>This may take a while on pi.</p></li>
<li><p>While you still in current folder, create python environment:</p>
<pre><code>mkdir -p ~/.virtualenvs
./python -m venv ~/.virtualenvs/py34
</code></pre></li>
<li><p>launch your virtual environment:</p>
<pre><code>source ~/.virtualenvs/py34/bin/activate
</code></pre></li>
</ol>
<p>Now you have Python 3.4 and pip inside of it. Try:</p>
<pre><code>    pip install bpython
</code></pre>
<p>To exit virtual environment use:</p>
<pre><code>    deactivate
</code></pre>
</div>
