<div class="post-text" itemprop="text">
<p><code>sys.stdin.read()</code> will read until an EOF (end of file) is encountered. That's why "pressing enter" doesn't seem to do anything. You can send an EOF on Windows by typing <kbd>Ctrl</kbd>+<kbd>Z</kbd>, or on *nix systems with <kbd>Ctrl</kbd>+<kbd>D</kbd>.</p>
<p>(Note that you probably still need to hit <kbd>Enter</kbd> before hitting <kbd>Ctrl</kbd>+<kbd>Z</kbd>. I don't think the terminal treats the EOF correctly if it's not at the start of a line.)</p>
<p>If you just want to read input until a newline, use <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow noreferrer"><code>input()</code></a> instead of <code>sys.stdin.read()</code>.</p>

<div class="post-text" itemprop="text">
<p>This happens because <code>sys.stdin.read</code> attempts to read <em>all the data</em> that the standard input can provide, including new lines, spaces, tabs, <em>whatever</em>. It will stop reading only if the interpreter's interrupted or it hits an EndOfFile (Ctrl+D on UNIX-like systems and Ctrl+Z on Windows).</p>
<p>The standard function that asks for input is simply <code>input()</code></p>
