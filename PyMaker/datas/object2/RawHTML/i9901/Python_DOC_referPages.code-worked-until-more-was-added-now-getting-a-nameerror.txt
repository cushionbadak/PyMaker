<div class="post-text" itemprop="text">
<h2>TL;DR</h2>
<p>Use <code>raw_input()</code> instead of <code>input()</code> in Python 2.x, because <code>input()</code> is "shorthand" for <code>eval(raw_input))</code>.</p>
<hr/>
<p>In Python 2.7, you need to use <code>raw_input()</code> instead of <code>input()</code>. <code>raw_input()</code> lets you read in user input as a string, while <code>input()</code> is "shorthand" for <code>eval(raw_input())</code> which attempts to evaluate you user input as literal Python code</p>
<p>This is documented in the python 2.x documentation:</p>
<blockquote>
<p>[<code>input()</code> is] Equivalent to <code>eval(raw_input(prompt))</code>.</p>
<p>This function does not catch user errors. If the input is not syntactically valid, a <code>SyntaxError</code> will be raised. Other exceptions may be raised if there is an error during evaluation.</p>
<p>If the readline module was loaded, then input() will use it to provide elaborate line editing and history features.</p>
<p>Consider using the <code>raw_input()</code> function for general input from users.</p>
</blockquote>
<p>This however was later changed in Python 3.x. In Python 3.x <code>raw_input()</code> became <code>input()</code> and the old input(<code>eval(raw_input())</code>) was dropped. This is documented in <a href="https://docs.python.org/3/whatsnew/3.0.html" rel="nofollow noreferrer">What's new in Python 3</a>:</p>
<blockquote>
<p>PEP 3111: <code>raw_input()</code> <strong>was renamed to</strong> <code>input()</code>. That is, the new <code>input()</code> function reads a line from <code>sys.stdin</code> and returns it with the trailing newline stripped. It raises <code>EOFError</code> if the input is terminated prematurely. To get the old behavior of <code>input()</code>, use <code>eval(input())</code>.</p>
</blockquote>
<p><sub>(Emphasis mine)</sub></p>
<hr/>
<p>So use <code>raw_input()</code> instead of <code>input()</code> in Python 2.x, because <code>input()</code> is "shorthand" for <code>eval(raw_input))</code>. eg. Change:</p>
<p><code>name = input ("Can you tell me your name!")</code></p>
<p>to</p>
<p><code>name = raw_input ("Can you tell me your name!")</code></p>
