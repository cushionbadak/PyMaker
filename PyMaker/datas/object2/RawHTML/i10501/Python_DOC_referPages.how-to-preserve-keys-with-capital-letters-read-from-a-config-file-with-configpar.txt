<p>When reading out a config file with python's <code>configparser</code> package, all key names are lowercase strings. Does anybody know how to read the strings preserving capital and uppercase words?</p>
<p>For example:</p>
<pre><code>$cat config.cfg 
[DEFAULT]
Key_1 = SomeWord
KEY_2 = Another Word

$ python3
&gt;&gt;&gt; from configparser import ConfigParser
&gt;&gt;&gt; cf = ConfigParser()
&gt;&gt;&gt; cf.read('./config.cfg')
['./config.cfg']
&gt;&gt;&gt; print(cf.defaults())
OrderedDict([('key_1', 'SomeWord'), ('key_2', 'Another Word')])
</code></pre>
<p>Thanks for any help!</p>
</div>
<div class="post-text" itemprop="text">
<p>Yes, all keys are automatically transformed to lowercase during read/write operations. This is mentioned in the last sentence of <a href="https://docs.python.org/3/library/configparser.html#quick-start" rel="nofollow">the Quick Start section of the <code>configparser</code> docs</a>.</p>
<p>To not have this effect you can set your parsers' <a href="https://docs.python.org/3/library/configparser.html#configparser.optionxform" rel="nofollow"><code>optionxform</code></a> (a callable) to simply return the <code>option</code> rather than transform it to lowercase:</p>
<pre><code>&gt;&gt;&gt; from configparser import ConfigParser
&gt;&gt;&gt; c = ConfigParser()
&gt;&gt;&gt; c.optionxform = lambda option: option
&gt;&gt;&gt; c.read('./config.cfg')
['./config.cfg']
</code></pre>
<p>Now keys are preserved as they were defined:    </p>
<pre><code>&gt;&gt;&gt; c.defaults()
OrderedDict([('Key_1', 'SomeWord'), ('KEY_2', 'Another Word')])
</code></pre>
<p>Of course you can customize this to your liking, if, for example, you wanted all keys to uppercase you could set that in <code>optionxform</code>:</p>
<pre><code>&gt;&gt;&gt; cf = ConfigParser()
&gt;&gt;&gt; cf.optionxform = lambda option: option.upper()
&gt;&gt;&gt; cf.read('./config.cfg')
['./config.cfg']
&gt;&gt;&gt; cf.defaults()
OrderedDict([('KEY_1', 'SomeWord'), ('KEY_2', 'Another Word')])
</code></pre>
