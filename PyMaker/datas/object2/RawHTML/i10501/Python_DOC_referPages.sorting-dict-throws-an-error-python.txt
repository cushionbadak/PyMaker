<div class="post-text" itemprop="text">
<p>A dict doesn't have a <code>sort</code> attribute. You can sort it by keys using <code>sorted</code>:</p>
<pre><code>for key in sorted(d.iterkeys()):
    print("%s: %s" % (key, d[key]))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Here's the thing, your original code works in python 2.x:</p>
<pre><code>d = {'a': 10, 'b': 1, 'c': 22}
print(d.items())

t = d.items()
t.sort()
print(t)
</code></pre>
<p>Because d.items() returns a <code>&lt;type 'list'&gt;</code> class, but not with python 3.x, with python 3.x it returns <code>&lt;class 'dict_items'&gt;</code>, which doesn't have the sort method, so a possible workaround would be doing like this:</p>
<pre><code>d = {'a': 10, 'b': 1, 'c': 22}

t = list(d.items())
print(t)
t.sort()
print(t)
</code></pre>
<p>As you can see, casting d.items() to list will allow you to use <a href="https://docs.python.org/3/library/stdtypes.html#list.sort" rel="nofollow">list.sort</a></p>
</div>
<span class="comment-copy">I'd also note that <code>sorted()</code> can be used with dict_item objects, so all the asker really needed to do with his code was change <code>t.sort()</code> to <code>t = sorted(t)</code>.</span>
