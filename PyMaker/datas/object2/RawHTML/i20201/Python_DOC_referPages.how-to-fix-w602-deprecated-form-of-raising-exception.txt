<p>If I use pylint (via sublimerlinter) I get following warning message: </p>
<p><code>W602 deprecated form of raising exception</code></p>
<p>This I how I use exceptions in my code:</p>
<pre><code>if CONDITION == True:
    raise ValueError, HELPING_EXPLANATION
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Raise your exception like this:</p>
<pre><code>if CONDITION == True:
    raise ValueError(HELPING_EXPLANATION)
</code></pre>
<p>From <a href="http://www.python.org/dev/peps/pep-0008/#id39" rel="nofollow noreferrer">PEP 8 -- Style Guide for Python Code - Programming Recommendations</a>:</p>
<blockquote>
<p>When raising an exception, use raise <code>ValueError('message')</code> instead of the older form <code>raise ValueError, 'message'</code>.</p>
<p>The paren-using form is preferred because when the exception arguments are long or include string formatting, you don't need to use line continuation characters thanks to the containing parentheses. The older form will be removed in Python 3.</p>
</blockquote>
</div>
<span class="comment-copy">I added this rather trivial Q&amp;A because googling for <code>W602 deprecated form of raising exception</code> does not result in any helpful links.</span>
<span class="comment-copy">Similarly, if you want to add a traceback in Python 2.x, use <code>raise ValueError(HELPING_EXPLANATION), None, tb</code> instead of <code>raise ValueError, HELPING_EXPLANATION, tb</code> to avoid that warning.</span>
<span class="comment-copy">For what its worth, there is no python3-forward-compatible way to add a traceback in Python 2.x.</span>
