<div class="post-text" itemprop="text">
<p>I am writing a script to launch remote desktop sessions using rdesktop. The relevant portion of the code looks like this:</p>
<pre><code>subprocess.call(["rdesktop", "-a 16", "-u user", "-g 1280x1024",, server])
</code></pre>
<p>When this happens, the terminal is locked up until I exit the rdesktop session. Would it be possible to launch multiple desktop sessions with this script?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>subprocess.Popen</code> (<a href="https://docs.python.org/2/library/subprocess.html#subprocess.Popen" rel="nofollow">py2 docs</a>, <a href="https://docs.python.org/3/library/subprocess.html#subprocess.Popen" rel="nofollow">py3 docs</a>) is the correct answer here. 
<code>subprocess.call</code> waits for the command to complete, while <code>subprocess.Popen</code> calls it in the background, and immediately executes the next line.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can fork the python process or use threads, or run the process in the background.</p>
</div>
<span class="comment-copy">Use <code>subprocess.Popen</code> instead of <code>subprocess.call</code></span>
