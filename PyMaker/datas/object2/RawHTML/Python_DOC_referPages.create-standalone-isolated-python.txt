<div class="post-text" itemprop="text">
<p>If you don't want to do like what Amir is suggesting above, then 2 other options are available:</p>
<ol>
<li>Copy those modules and place them in the same folder where your script is installed</li>
<li>Create a requirements.txt file with the name &amp; version of those modules and then run "pip install -r requirements.txt" to install these modules in your site-packages folder</li>
</ol>

<div class="post-text" itemprop="text">
<p>To manage your python packages you can use great <strong>virtualenv</strong> tool, it looks really simple and works well on <em>linux/macOS/Windows</em>. Any package which will be installed in <strong>activated virtualenv</strong> will be <strong>available only in this virtualenv</strong>, so you can have for example 3 different versions of "Django" package on your machine and work with them using different virtual environments:</p>
<p>Install virtualenv:</p>
<pre><code>$ pip3 install virtualenv 
</code></pre>
<p>Create your virtualenv:</p>
<pre><code>$ virtualenv -p python3 my_virtualenv_name
</code></pre>
<p>Activate your virtualenv:</p>
<pre><code>$ . my_virtualenv_name/bin/activate
</code></pre>
<p>Check what packages have been installed:</p>
<pre><code>$ pip freeze
</code></pre>
<p>Install any package for example "Django":</p>
<pre><code>$ pip install Django
</code></pre>
<p>Confirm installation:</p>
<pre><code>$ pip freeze | grep Django
</code></pre>
<p>Uninstall any package from your virtual environment:</p>
<pre><code>$ pip uninstall Django -y
</code></pre>
<p>Uninstall all packages from your virtual environment:</p>
<pre><code>$ pip freeze | xargs pip uninstall -y
</code></pre>
<p>Deactivate virtualenv</p>
<pre><code>$ deactivate
</code></pre>
<p>More info in the official documentation: <a href="https://virtualenv.pypa.io/en/latest/" rel="nofollow noreferrer">https://virtualenv.pypa.io/en/latest/</a></p>
</div>
