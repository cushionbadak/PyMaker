<pre><code>from . import x

<p>I am not Python novice so I understand what <code>from</code> and  <code>import</code> keywords do. But what is <code>from .</code> ? Could someone explain with an example?</p>

<div class="post-text" itemprop="text">
<p><code>from . import x</code></p>
<p>Will import locally to that script.</p>
<p>Python includes a <a href="https://docs.python.org/2/library/json.html" rel="nofollow noreferrer"><code>json</code></a> library so </p>
<pre><code>from . import json
</code></pre>
<p>Is used to avoid importing the <a href="https://docs.python.org/2/library/json.html" rel="nofollow noreferrer">standard library <code>json</code></a> and instead import a local python module</p>
</div>
<div class="post-text" itemprop="text">
<p>It is an example of Intra-package references in python. (Check out section 6.4.2 on <a href="https://docs.python.org/3/tutorial/modules.html" rel="nofollow noreferrer">https://docs.python.org/3/tutorial/modules.html</a>)</p>
<p>When writing packages in python, <code>from . import X</code> is used to import sub-packages from the parent class, i.e. importing a sibling class. The <code>.</code> is only used to refer to the parent package in the relative path. </p>
<p>However, you can always import parent package without using the <code>.</code> in import statement, by using the name of the parent package name/path. Using <code>.</code> is just a handy shortcut.</p>
</div>
