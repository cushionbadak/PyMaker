<div class="post-text" itemprop="text">
<p>I have a dataset with the dates given in numbers from 1995 to 2016. 
1/1/1995 is equal to 1, 2/1/1995 is 2 and 31/12/2016 is 8036, so every day is one number.</p>
<p>How do I turn my time array [1,2,3,...,8034,8035,8036] into dates of dd/mm/yyyy?</p>
</div>
<div class="post-text" itemprop="text">
<p>Use the <code>datetime.fromordinal(x)</code> method. This creates a date from the number x, signifying the number of days since 0001-01-01.</p>
<p>If you want to start with <code>x==1</code> being 1995-01-01, you can do</p>
<pre><code>d = datetime.date(1,1,1) # actual date does not matter
print d.fromordinal( datetime.date(1994,12,31).toordinal() + x)
</code></pre>
<p>where <code>x</code> is the number of days since the last day of 1994.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use the epoch conversion which is also supported by the <a href="https://docs.python.org/2/library/time.html" rel="nofollow noreferrer">python time library</a>. The unix epoch time starts on 01/01/1970 and is counted in seconds. If you add 788932800 seconds to reach 01/01/1995, you can then add 86400 (seconds per day) and can use standard techniques to calculated the "seconds from epoch" back to a correct date. </p>
<p>Example code:</p>
<pre><code>#!/usr/bin/python
import time
entryid = 5
epochdate = 788932800 + entryid * 86400
print time.strftime("%d/%m/%y", time.gmtime(epochdate))
</code></pre>
