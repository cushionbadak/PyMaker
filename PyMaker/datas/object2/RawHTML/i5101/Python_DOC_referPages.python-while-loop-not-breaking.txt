<div class="post-text" itemprop="text">
<p>You never change <code>n</code> or <code>x</code> here:</p>
<pre><code>while n &gt; x:
    password = password + random.choice(l2)
    x + 1
</code></pre>
<p>So if the condition was <code>True</code> initially it will always stay <code>True</code> and loop infinitely. Need to do <code>x = x + 1</code></p>
<p>Incidentally this is the exact sort of bug that Pylint would catch for you.</p>
</div>
<div class="post-text" itemprop="text">
<p>Please consider the following:</p>
<p><strong>1) Obvious condition</strong></p>
<pre><code>    x = 0
    if x == 0:
        password = ''
</code></pre>
<p>You define x = 0, and then checks if x equals 0. It is invariably True.
Hence, you can change it this way:</p>
<pre><code>    x = 0
    password = ''
</code></pre>
<p><strong>2) While loop never ends</strong></p>
<p>Before you had: </p>
<pre><code>while n &gt; x:
    [some code]
    x + 1        # here was your mistake
</code></pre>
<p>Consider these two ways you can add <code>1</code> to the variable <code>x</code>:</p>
<pre><code>x = x + 1
</code></pre>
<p>or</p>
<pre><code>x += 1
</code></pre>
<p>Both mean the same thing.</p>
<p>For further enlightment:
<a href="https://docs.python.org/3/reference/simple_stmts.html#augmented-assignment-statements" rel="nofollow noreferrer">https://docs.python.org/3/reference/simple_stmts.html#augmented-assignment-statements</a></p>
</div>
