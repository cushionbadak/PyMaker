<div class="post-text" itemprop="text">
<p><code>b</code> is used to set break points. So whatever follows <code>b</code> is expected to be a function or line number.</p>
<p>If you type <code>ipdb&gt; help</code> you will see the full list of commands (reserved words).</p>
<p>You could use, say, <code>x</code> or <code>y</code> as a variable:</p>
<pre><code>ipdb&gt; y = variable
</code></pre>
<p>or</p>
<pre><code>ipdb&gt; exec 'b = variable'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>I think you need to use '!' (<a href="https://docs.python.org/3/library/pdb.html" rel="nofollow noreferrer">pdb documentation</a>):
! statement
Execute the (one-line) statement in the context of the current stack frame. The exclamation point can be omitted <em>unless the first word of the statement resembles a debugger command</em>. To set a global variable, you can prefix the assignment command with a global statement on the same line, e.g.:</p>
<pre><code>global list_options; list_options = ['-l']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>you can use</p>
<pre><code>locals()["b"] = variable
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>In python3 you can use a single exclamation mark to override ipdb commands</p>
<p><code>!b</code></p>
