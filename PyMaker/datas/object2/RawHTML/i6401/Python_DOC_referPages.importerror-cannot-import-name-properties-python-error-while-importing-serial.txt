<p>Looks like you want to use <code>pyserial</code>, but pyserial has no <code>properties.py</code>, no <code>meta.py</code> and no <code>model.py</code>, see: <a href="https://github.com/pyserial/pyserial/tree/master/serial" rel="nofollow noreferrer">https://github.com/pyserial/pyserial/tree/master/serial</a> so the stacktrace you're showing does not come from pyserial.</p>
<p>So there's a conflict between another package and pyserial.</p>
<p>The day to day solution to isolate work environments in Python is to use <a href="https://docs.python.org/3/library/venv.html" rel="nofollow noreferrer">virtual environment</a>, like:</p>

$ mkdir -p ~/.venvs/
$ python3 -m venv ~/.venvs/my_project_using_pyserial/
