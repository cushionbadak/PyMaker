<p>I just started working with hexadecimal values in python and am a bit surprised with what I just encountered. I expected the following code to first print a hexadecimal string, and then a decimal value.</p>
<p>Input:</p>
<pre><code>n = 0x8F033CAE74F88BA10D2BEA35FFB0EDD3
print('Hex value for n is:', n)
print('Dec value for n is:', int(str(n), 16))
</code></pre>
<p>Output:</p>
<blockquote>
<p>Hex value for n is: 190096411054295805012706659640261275091 </p>
<p>Dec value for n is: 8921116140846515089057635273465667902228615313</p>
</blockquote>
<p>How is it possible that 2 different different numbers are shown? I expected the first number to be a hexadecimal string and the second it's decimal equivalent, what is this second value in this case?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>0x</code> is a way to input an integer with an hexadecimal notation.</p>
<pre><code>&gt;&gt;&gt; n = 0x8F033CAE74F88BA10D2BEA35FFB0EDD3
</code></pre>
<p>This hexadecimal notation is forgotten directly after instantiation, though:</p>
<pre><code>&gt;&gt;&gt; n
190096411054295805012706659640261275091
&gt;&gt;&gt; str(n)
'190096411054295805012706659640261275091'
</code></pre>
<p>So when you call <code>int(str(n), 16)</code>, Python interprets <code>'190096411054295805012706659640261275091'</code> as an hexadecimal number:</p>
<pre><code>&gt;&gt;&gt; int(str(n), 16)
8921116140846515089057635273465667902228615313
</code></pre>
<p>You need to input the original hex string:</p>
<pre><code>&gt;&gt;&gt; int("8F033CAE74F88BA10D2BEA35FFB0EDD3", 16)
190096411054295805012706659640261275091
</code></pre>
<p>or use <a href="https://docs.python.org/3/library/functions.html#hex" rel="nofollow noreferrer"><code>hex</code></a>:</p>
<pre><code>&gt;&gt;&gt; int(hex(n), 16)
190096411054295805012706659640261275091
</code></pre>
</div>
