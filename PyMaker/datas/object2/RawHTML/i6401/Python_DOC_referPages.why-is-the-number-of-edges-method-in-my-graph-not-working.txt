<p>I have the following command:</p>
<pre><code>print g.number_of_edges
</code></pre>
<p>and get the following output</p>
<pre><code>&lt;bound method Graph.number_of_edges of &lt;networkx.classes.graph.Graph object at 0x00000000096512E8&gt;&gt;
</code></pre>
<p>what does this output mean?</p>
</div>
<div class="post-text" itemprop="text">
<p>Since <code>g.number_of_edges</code> is a class method and there is no print method defined for the class, Python will simply print what is knows about the object (the class name and memory location where the object is stored).</p>
<p>Instead of <code>print g.number_of_edges</code> use <code>print g.number_of_edges()</code>. The difference is that <code>print g.number_of_edges()</code> calls the method <code>number_of_edges</code> and then prints its return value.</p>
</div>
<span class="comment-copy">if <code>number_of_edges</code> is a method then you should do <code>print g.number_of_edges()</code> to actually call the method. I usually encounter this output when I try to print the object itself rather then calling the object and printing its output.</span>
