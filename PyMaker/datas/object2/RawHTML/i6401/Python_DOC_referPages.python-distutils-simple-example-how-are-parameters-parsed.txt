<p>Referring to <a href="https://docs.python.org/3/distutils/introduction.html#a-simple-example" rel="nofollow noreferrer"><strong>A Simple Example</strong> from python.org</a>:</p>
<p>This is <code>setup.py</code>:  </p>
<pre><code>from distutils.core import setup
setup(name='foo',
      version='1.0',
      py_modules=['foo'],
      )
</code></pre>
<p>And then it says to run this command:</p>
<p><code>python setup.py sdist</code></p>
<p>How/what/where is the parameter <code>sdist</code> being parsed?  </p>
</div>
<div class="post-text" itemprop="text">
<p>The <code>setup</code> function parses the command line arguments: it reads <code>sys.argv</code>. See <a href="https://github.com/python/cpython/blob/master/Lib/distutils/core.py" rel="nofollow noreferrer">source code on GitHub</a>. </p>
<p>You're certainly confusing between function arguments (the ones defined in the <code>setup.py</code> script) and the command line arguments: (which are <code>setup.py sdist</code>). </p>
<p>A piece of information is given in the <code>run_setup</code> function comment:</p>
<blockquote>
<p>'sys.argv[0]' will be replaced with 'script' for the duration of the call.  'script_args' is a list of strings; if supplied,</p>
<p>'sys.argv[1:]' will be replaced by 'script_args' for the duration of the call.</p>
</blockquote>
</div>
