<blockquote>
<pre><code>class ResultIterable(collections.Iterable):
</code></pre>

<p><a href="https://docs.python.org/3/library/collections.abc.html#collections-abstract-base-classes" rel="nofollow noreferrer">specifically you can assume</a> that it implements <a href="https://docs.python.org/3/reference/datamodel.html#object.__iter__" rel="nofollow noreferrer"><code>__iter__</code></a> dunder method - it means it can be iterated or converted to another collection and can be used whenever iterable objects is expected.</p>
<blockquote>
<p>I would have expected a list </p>
</blockquote>
<p><code>list</code> requires specific implementation of the collection. <code>Iterable</code> allows other options, including larger than memory collections, and specific implementation  can be change whenever needed.</p>
