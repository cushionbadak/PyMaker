<p>I have a text file that is reading in another application's script. 
This how it looks like in notepad:</p>
<pre><code>Одинцовская РЭС: М.О., Одинцовский район
Климовская РЭС: М.О., г.о. Подольск
Кульшовская РЭС: М.О., г.о. Подольск
</code></pre>
<p>This file is needed for two things:
1. Creating a dict of values, separated by ':'. I use this dict in another part of script
2. Allows user to select desirable value</p>
<p><a href="https://i.stack.imgur.com/tnxXA.png" rel="nofollow noreferrer"><img alt="enter image description here" src="https://i.stack.imgur.com/tnxXA.png"/></a></p>
<p>Here is what user see when he launhes script.
When a certain value is selected I have to use it in dictionary. But the problem is that selection is in <code>unicode</code> format (because of features of script building in ArcGIS) while the dictionary's keys are <code>str</code>.
So I need a value in dictionary which looks like
<code>'\xce\xe4\xe8\xed\xf6\xee\xe2\xf1\xea\xe0\xff \xd0\xdd\xd1'</code> to be converted in unicode. But when I make <code>.encode('utf-8')</code> it throws an error </p>
<p><code>UnicodeDecodeError: 'ascii' codec can't decode byte 0xce in position 0: ordinal not in range(128)</code></p>
</div>
<div class="post-text" itemprop="text">
<p>This should work</p>
<pre><code>&gt;&gt;&gt; c = b'\xce\xe4\xe8\xed\xf6\xee\xe2\xf1\xea\xe0\xff \xd0\xdd\xd1'
&gt;&gt;&gt; c
b'\xce\xe4\xe8\xed\xf6\xee\xe2\xf1\xea\xe0\xff \xd0\xdd\xd1'
&gt;&gt;&gt; c.decode('unicode_escape')
'Îäèíöîâñêàÿ ÐÝÑ'
</code></pre>
<p>The b'' prefix denotes sequence of 8-bit bytes.</p>
<p>Take a look at SO <a href="https://stackoverflow.com/questions/3198765/how-to-write-russian-characters-in-file">read russian characters</a></p>
