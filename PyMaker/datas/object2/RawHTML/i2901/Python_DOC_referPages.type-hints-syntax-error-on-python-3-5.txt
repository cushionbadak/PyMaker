<p>I've got a code base that I recently sprinkled liberally with type hints. It is python 3.5+ exclusively and the python 3.5 changelog claims type hinting is supported. Unfortunately all the hints raise syntax errors, likewise with simple hints in the interpreter:</p>
<pre><code>$ python3
Python 3.5.2 (default, Nov 23 2017, 16:37:01) 
&gt;&gt;&gt; a: int = 5
  File "&lt;stdin&gt;", line 1
    a: int = 5
     ^
SyntaxError: invalid syntax
</code></pre>
<p>Any ideas?</p>

<div class="post-text" itemprop="text">
<p>That's an example of a variable annotation, which wasn't introduced until Python 3.6. A variable annotation, defined by <a href="https://www.python.org/dev/peps/pep-0526/" rel="nofollow noreferrer">PEP-526</a>, is distinct from a function annotation. Type hints are just one possible use for either type of annotation.</p>
</div>
