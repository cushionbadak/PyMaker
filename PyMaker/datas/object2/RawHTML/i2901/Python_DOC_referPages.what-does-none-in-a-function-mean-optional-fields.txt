<p>The <code>=...</code> part gives a parameter a <em>default value</em>. Here that's <code>None</code>, and that does mean the parameter is optional when calling the function. If you don't specify it, the default is used. See <a href="https://docs.python.org/3/tutorial/controlflow.html#default-argument-values" rel="nofollow noreferrer"><em>Default Argument Values</em></a> in the Python tutorial, as well as the <a href="https://docs.python.org/3/reference/compound_stmts.html#function-definitions" rel="nofollow noreferrer">reference documentation for function definitions</a>:</p>
<blockquote>
<p>When one or more parameters have the form <em>parameter = expression</em>, the function is said to have “default parameter values.” For a parameter with a default value, the corresponding argument may be omitted from a call, in which case the parameter’s default value is substituted.</p>
</blockquote>
<p>When calling such a function, you could pass in the same value as the default, to this specific function that makes no difference; so when there is no <code>starting_date</code>, you can pass in <code>None</code>:</p>
<pre><code>start_date = starting_date or None  # when determining the starting date, default to None
requested_report=report_api.request_report(
    report_type=report_name,
    start_date=starting_date
)
</code></pre>
<p>Another option is to set any optional keyword arguments in a dictionary, then pass those in with the <code>**</code> call syntax to <a href="https://docs.python.org/3/tutorial/controlflow.html#unpacking-argument-lists" rel="nofollow noreferrer">unpack the dictionary as keyword arguments</a>:</p>
<pre><code>kwargs = {}
if ...:  # test that decides if start_date is needed
    kwargs['start_date'] = starting_date  # expression to set the start date

requested_report=report_api.request_report(
    report_type=report_name, **kwargs)
</code></pre>
<p>It is fine for the <code>kwargs</code> dictionary to be empty.</p>
