<p>I'm converting a source code written in Python 2 to Python 3 and I stumbled into this:</p>
<pre><code>from Queue import Queue, Empty
</code></pre>
<p>I changed it to:</p>
<pre><code>from multiprocessing import Queue, Empty
</code></pre>
<p>But this gives me an exception:</p>
<pre><code>ImportError: cannot import name 'Empty'
</code></pre>
<p>How do I fix this?</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/multiprocessing.html#exchanging-objects-between-processes" rel="nofollow noreferrer"><code>multiprocessing.Queue</code></a> is used for processes, don't let the capitalization confuse you. <a href="https://docs.python.org/2.7/library/queue.html" rel="nofollow noreferrer"><code>Queue</code></a>, which was renamed to <a href="https://docs.python.org/3.6/library/queue.html" rel="nofollow noreferrer"><code>queue</code></a> in Python 3, is for threads.</p>
<p>Both <a href="https://docs.python.org/3.6/library/queue.html#queue.Empty" rel="nofollow noreferrer"><code>Empty</code></a> and <a href="https://docs.python.org/3.6/library/queue.html#queue.Queue" rel="nofollow noreferrer"><code>Queue</code></a> are located in the <code>queue</code> module, so grab them from there.</p>
</div>
<span class="comment-copy"><code>Queue</code> is <code>queue</code> in python 3.</span>
<span class="comment-copy"><code>from queue import Queue, Empty</code></span>
