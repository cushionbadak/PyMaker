<div class="post-text" itemprop="text">
<p>I'd suggest turning <code>\r</code> into an <em>optional</em> subpattern and optimize the Group 2 subpattern as follows:</p>
<pre><code>nameRegex = re.compile(r"(899\*0)\r?\n[ ]+([a-z, A-Z-]+(?:[ ][a-z,A-Z-]+)?)")
</code></pre>
<p>See the <a href="https://regex101.com/r/crrZOs/1" rel="nofollow noreferrer">regex demo</a>. <strong>Note</strong> that the commas inside <code>[...]</code> are meaningful, if you used them as range delimiters, consider removing the commas. I left the spaces inside <code>[ ]</code> only because it might be handy if you plan to use <code>re.VERBOSE</code> flag that will make all whitespace in the pattern (outside character classes) meaningless (used for formatting purposes only).</p>
<p>Note that <code>re.MULTILINE</code> flag is redundant because it only redefines the behavior of <code>^</code> and <code>$</code> anchors, and does not have any affect on the input string parsing.</p>
<p><strong>Pattern description</strong>:</p>
<ul>
<li><code>(899\*0)</code> - Group 1 capturing <code>899*0</code></li>
<li><code>\r?\n</code> - an optional CR followed with LF</li>
<li><code>[ ]+</code> - 1+ regular spaces</li>
<li><code>([a-z, A-Z-]+(?:[ ][a-z,A-Z-]+)?)</code> - Group 2 capturing a sequence of:

<ul>
<li><code>[a-z, A-Z-]+</code> - 1 or more ASCII letters, <code>,</code>, space, <code>-</code></li>
<li><code>(?:[ ][a-z,A-Z-]+)?</code> - one or more occurrence of:

<ul>
<li><code>[ ]</code>  - a space</li>
<li><code>[a-z,A-Z-]+</code> - 1 or more ASCII letters, <code>,</code> or <code>-</code>.</li>
</ul></li>
</ul></li>
</ul>
