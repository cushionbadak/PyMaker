<div class="post-text" itemprop="text">
<p>You need to escape the backslash so it won't be interpreted as a special character:</p>
<pre><code>file_handle.write("\\t")
</code></pre>
<p><strong>EDIT:</strong> It looks like your best option will be to use python's excellent <code>string.replace()</code> function:</p>
<pre><code>&gt;&gt;&gt; v = "aaa \t bbb"
&gt;&gt;&gt; print v
aaa      bbb
&gt;&gt;&gt; x = v.replace("\t", "\\t")  # &lt;-- replace all '\t' in string with '\\t'
&gt;&gt;&gt; print x
aaa \t bbb
</code></pre>
<p>Note: <code>v</code> will be unchanged, since strings are immutable in python</p>
<p><strong>EDIT 2:</strong></p>
<p>Looks like converting to a raw string will take care of escapeing all escaped characters for you:</p>
<pre><code>&gt;&gt;&gt; a = 'aaa\nbbb\n\tccc'
&gt;&gt;&gt; print a
aaa
bbb
        ccc
&gt;&gt;&gt; b = a.encode('string-escape')
&gt;&gt;&gt; print b
aaa\nbbb\n\tccc
</code></pre>

<p>Lets assume v is 'foo'.
You need the escape your single backslash.</p>
<pre><code>v = '\\t %s' % v
print(v)
</code></pre>
<p>returns:</p>
<pre><code>\t foo
</code></pre>

<div class="post-text" itemprop="text">
<p>I've found a decoder workaround!</p>
<pre><code>from codecs import getencoder as ge, getdecoder as gd
v = "\t"
ge("unicode-escape")(v)[0]
v = gd("utf-8")(byteDelim)[0]
write(v)
</code></pre>
<p>using the encoder one can introduce the extra "\" to make a tuple <code>(b'\\t', 1)</code>. Then deconstruct the byte-like string in the tuple using another decoder that does <em>not</em> remove the "\", like <code>utf-8</code>.</p>
