<div class="post-text" itemprop="text">
<p>I'm writing a Python script where a file path is returned as a string. The file path is something like <code>C:\user\test\2016\1005\text.xml</code>. When I tried to give this string to <code>ET.parse()</code>, it incorrectly treats the <code>\20</code> and <code>\10</code> portion of the path as special characters, and so the parsing fails.</p>
<p>How can I handle such situations?</p>
</div>
<div class="post-text" itemprop="text">
<p>You either need to escape the backslashes in the strings:</p>
<pre><code>ET.parse("C:\\some\\path.xml")
</code></pre>
<p>or via</p>
<pre><code>ET.parse(r"C:\some\path.xml")
</code></pre>
<p>and I would recommend to use the <a href="https://docs.python.org/3/library/os.path.html" rel="nofollow">os.path</a> module:</p>
<pre><code>import os
ET.parse(os.path("C:\\some\\path.xml")
</code></pre>
<p>Other threads with similar problems: <a href="https://stackoverflow.com/questions/19351127/python-xml-etree-elementtree-directory-acess">e.g. here</a>.</p>
