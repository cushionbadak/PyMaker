<p>I am trying to implement an LCM finding algorithm. It needs to find LCM for very large numbers. </p>
<p>LCM is found using the formula,</p>
<pre><code>LCM(A, B) = (A * B) / GCD(A, B)
</code></pre>
<p>where A and B are two inputs.</p>
<p>Input: <code>226553150 1023473145</code><br/>
So, <code>LCM = (226553150 * 1023473145) / 5</code><br/>
It should be, <code>46374212988031350</code>.</p>
<p>But python is finding this as <code>46374212988031352</code>, which is obviously an error. 
How to solve this problem ?
<img alt="python screenshot" src="https://i.stack.imgur.com/myyag.png"/></p>
</div>
<div class="post-text" itemprop="text">
<p>You are using floating point math, because you used the <code>/</code> true division operator. Floating point can only approximate large numbers, and the difference you see is a result of that.</p>
<p>Use <code>//</code> floor division instead:</p>
<pre><code>&gt;&gt;&gt; (226553150 * 1023473145) // 5
46374212988031350
</code></pre>
<p>Floor division on integers never requires conversion to float, avoiding the precision issues.</p>
<p>Alternatively, use the <a href="https://docs.python.org/3/library/decimal.html" rel="nofollow noreferrer"><code>decimal</code> module</a> for higher-precision math with real numbers:</p>
<pre><code>&gt;&gt;&gt; from decimal import Decimal
&gt;&gt;&gt; Decimal('226553150') * Decimal('1023473145') / Decimal('5')
Decimal('46374212988031350')
</code></pre>
<p>This is slower than using <code>float</code>.</p>
</div>
