<p>I'd like to use map to get list of strings:</p>
<pre><code>value = '1, 2, 3'

my_list = list(map(strip, value.split(',')))
</code></pre>
<p>but got:</p>
<pre><code>NameError: name 'strip' is not defined
</code></pre>
<p>expected result: <code>my_list=['1','2','3']</code> </p>
</div>
<div class="post-text" itemprop="text">
<p><code>strip</code> is still just a variable, not a reference to the <code>str.strip()</code> method on each of those strings.</p>
<p>You can use the unbound <code>str.strip</code> method here:</p>
<pre><code>my_list = list(map(str.strip, value.split(',')))
</code></pre>
<p>which will work for any <code>str</code> instance:</p>
<pre><code>&gt;&gt;&gt; value = '1, 2, 3'
&gt;&gt;&gt; list(map(str.strip, value.split(',')))
['1', '2', '3']
</code></pre>
<p>In case you want to call a method named in a string, and you have a variety of types that all happen to support that method (so the unbound method reference wouldn't work), you can use a <a href="https://docs.python.org/3/library/operator.html#operator.methodcaller" rel="noreferrer"><code>operator.methodcaller()</code> object</a>:</p>
<pre><code>from operator import methodcaller

map(methodcaller('strip'), some_mixed_list)
</code></pre>
<p>However, instead of <code>map()</code>, I'd just use a list comprehension if you want to output a list object <em>anyway</em>:</p>
<pre><code>[v.strip() for v in value.split(',')]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can also use a <code>lambda</code> to achieve your purpose by using:</p>
<pre><code>my_list = map(lambda x:x.strip(), value.split(","))
</code></pre>
<p>where each element in <code>value.split(",")</code> is passed to <code>lambda x:x.strip()</code> as parameter <code>x</code> and then the <code>strip()</code> method is invoked on it. </p>
