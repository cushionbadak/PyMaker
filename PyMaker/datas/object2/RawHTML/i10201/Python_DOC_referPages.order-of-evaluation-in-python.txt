<pre><code>k = 10 / 3
l = k &lt;= 3 or True
</code></pre>
<p>The value for l in this code returns True. How is this possible? In the order of evaluation, comparisons are higher prioritised than the "or, not, and" operators.</p>
</div>
<div class="post-text" itemprop="text">
<p>They do have a higher priority, but, the <a href="https://docs.python.org/3/reference/expressions.html#boolean-operations" rel="nofollow"><code>or</code> operator works by</a>:</p>
<blockquote>
<p>The expression <code>x or y</code> first evaluates <code>x</code>; if <code>x</code> is true, its value is returned; otherwise, <code>y</code> is evaluated and the resulting value is returned.</p>
</blockquote>
<p>Since <code>k</code> is <code>3.3333333333333335</code>, the expression <code>k &lt;= 3</code> is <code>False</code>, <code>True</code> is going to be evaluated (to itself) and returned making <code>l == True</code>.</p>
</div>
