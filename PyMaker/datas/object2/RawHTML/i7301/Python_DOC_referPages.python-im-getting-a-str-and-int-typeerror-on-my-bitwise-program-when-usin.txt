When prompted "Enter Integer" you can provide a number (say, 12, for example) and then it will be converted from a string to an integer, then from base10 to binary. <p>As far as I understand the int() and bin() functions, the user provided number should be a bitwise interger when it's passed to the check_bit4() function. What am I missing?</p>

<div class="post-text" itemprop="text">
<p>There is <strong>no reason to use <code>bin(..)</code></strong>. <code>bin(..)</code> does <strong>not convert an integer to a binary equivalent</strong>, an int is already stored binary. It constructs a <strong>representation</strong> (something a <em>person</em> can read) of the integer with 0s and 1s as a string. Indeed:</p>
<pre><code>&gt;&gt;&gt; type(bin(1425))
&lt;class 'str'&gt;
</code></pre>
<p>So you can simply use:</p>
<pre><code>x = int(input("Enter Integer "))  # look ma, no bin(..)
check_bit4(x)
</code></pre>
<p>In your function, you write <code>0b00001000</code> but this is also an integer: if you would have written <code>8</code> directly, that would not have made a difference.</p>
</div>
<div class="post-text" itemprop="text">
<p>Well, It is because <strong>bin(x)</strong> returns a <strong>string</strong><br/> 
Reference: <a href="https://docs.python.org/3/library/functions.html#bin" rel="nofollow noreferrer">https://docs.python.org/3/library/functions.html#bin</a></p>
<p>Hence in the above line you are trying to <strong>&amp; (and)</strong> string and int which causes the error.<br/>
So we need to convert the string to int before using <strong>&amp; (and)</strong> operator. <br/><br/>Instead of these tedious tasks, I would suggest you to use <strong>&amp; (and)</strong> operator directly on the input number.</p>
<pre><code>def check_bit4(x):
    check = 8  #decimal representation of 0b00001000
    if x &amp; check:  #if condition will be true if the answer is non zero
        return("on")
    else:
        return("off")

x = int(input("Enter Integer "))
print(check_bit4(x))
</code></pre>
<p>I hope this solves your issue :)</p>
</div>

<span class="comment-copy">All numbers are stored in binary format internally (at the lowest level), thus bitwise operations, e.g. <code>&amp;</code> can be directly performed on <code>int</code>s without converting in any way.</span>
