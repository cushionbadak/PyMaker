<div class="post-text" itemprop="text">
<p>You need to slice your string <code>middle</code>.  <code>1s</code> is not a command Python understands but <code>[0]</code> is. See <a href="https://www.digitalocean.com/community/tutorials/how-to-index-and-slice-strings-in-python-3" rel="nofollow noreferrer">string slicing</a>.</p>
<p>Your working code:</p>
<pre><code>first_name = str(input("Please enter your first name: "))
middle_name = str(input("Please enter your middle name: "))
last_name = str(input("Please enter your last name: "))

first_name = first_name.capitalize()
middle_name = middle_name.upper()
last_name = last_name.capitalize()

name_format = "{first} {middle[0]} {last}"
print(name_format.format(first=first_name, middle=middle_name, last=last_name)) 
</code></pre>
<p>Modified:</p>
<pre><code>name_format = "{first} {middle[0]} {last}"
</code></pre>
<p>Slicing strings are versatile.  For example:</p>
<pre><code>&gt;&gt;&gt;a='Pythom'
&gt;&gt;&gt;a[0]
P
&gt;&gt;&gt;a[0:]
ythom
&gt;&gt;&gt;a[0:2]
pyt
&gt;&gt;&gt;a[:3]
Pyth
&gt;&gt;&gt;a[-1] 
m
&gt;&gt;&gt;a[-1] = 'n'
&gt;&gt;&gt;a
Python
</code></pre>
<p>You are doing the same here.  You input your name and slice at a point.  Note that string slicing starts at 0 for the first character and not 1.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>.1s</code> in a format string is the syntax from the <a href="https://docs.python.org/3/library/stdtypes.html#old-string-formatting" rel="nofollow noreferrer">older <code>printf</code>-style formatting</a>. There, it would have looked liked this:</p>
<pre><code>&gt;&gt;&gt; '%s %.1s %s' % ('Guido', 'van', 'Rossum')
'Guido v Rossum'
</code></pre>
<p>With the <a href="https://docs.python.org/3/library/string.html#formatstrings" rel="nofollow noreferrer">format string syntax</a>, which you are using, this looks a bit different. Format specifiers are to be specified after a colon, like this:</p>
<pre><code>&gt;&gt;&gt; '{} {:.1s} {}'.format('Guido', 'van', 'Rossum')
'Guido v Rossum'
</code></pre>
<p>Or using named groups:</p>
<pre><code>&gt;&gt;&gt; '{first} {middle:.1s} {last}'.format(first='Guido', middle='van', last='Rossum')
'Guido v Rossum'
</code></pre>
<p>This is using the <em>precision</em> to limit the string length, as <a href="https://docs.python.org/3/library/string.html#format-specification-mini-language" rel="nofollow noreferrer">documented</a> (emphasis mine):</p>
<blockquote>
<p>The <em>precision</em> is a decimal number indicating how many digits should be displayed after the decimal point for a floating point value formatted with <code>'f'</code> and <code>'F'</code>, or before and after the decimal point for a floating point value formatted with <code>'g'</code> or <code>'G'</code>. <strong>For non-number types the field indicates the maximum field size - in other words, how many characters will be used from the field content.</strong> The <em>precision</em> is not allowed for integer values. </p>
</blockquote>
<p>(Yes, I am aware that “van” is not a middle name…)</p>
