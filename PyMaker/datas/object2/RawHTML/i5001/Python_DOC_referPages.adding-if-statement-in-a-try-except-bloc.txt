<p>I' learning Python and I wonder if improving my code with a light <em>debug level</em> with if statement in my try/except bloc may be a good practice or if there is a better (or builtin) way to doing this.</p>
<pre><code>#!/usr/bin/env python3
""" Testing conditions in try/except """

DEBUG_MODE = True
number = input("Type a integer: ")

try:
    number = int(number)
except ValueError as except_detail:
    if DEBUG_MODE:
        print("ValueError: «{}»".format(except_detail))
    else:
        print("«{}» is not an integer".format(number))            
else:
    print("Your number {} is an integer".format(number))
</code></pre>
<p>I use a boolean, but a debug-level <code>list()</code> work either for more than 2 levels.</p>
<p>Error reporting may also be improved with <em>debug-level specific</em> logging instructions.</p>
</div>
<div class="post-text" itemprop="text">
<p>You could use Python's <code>logging</code> module for this, you can change the logging <code>level</code> argument to record the level of logging detail you want:</p>
<pre><code>import logging

logging.basicConfig(filename='log.txt', filemode='a', level=logging.DEBUG)

try:
    number = int(number)
except ValueError as except_detail:
    logging.warning("ValueError: «{}»".format(except_detail))
    logging.info("«{}» is not an integer".format(number))         
else:
    logging.info("Your number {} is an integer".format(number))
</code></pre>
</div>
