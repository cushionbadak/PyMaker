<div class="post-text" itemprop="text">
<p>A dictionary can have a key only once.
When you create <code>count = ({'John': 2, 'Sam': 1, 'Brian': 2, 'Brian': 2, 'Brian': 1, 'Sam': 2, 'John': 2, 'Henry': 2, 'Brian': 1})</code>, Python stores <code>{'John': 2, 'Brian': 1, 'Sam': 2, 'Henry': 2}</code> (values can change as there is no rule on what value to keep for a key which appears more than once). Cf <a href="https://docs.python.org/3.6/tutorial/datastructures.html#dictionaries" rel="nofollow noreferrer" title="the Python documentation for dictionaries">the Python documentation for dictionaries</a></p>
<p>So the count will always be 1.</p>
<p>If you want to have a key multiple times, don't use a dictionary but a list of pairs (tuples of size 2).</p>
</div>
<div class="post-text" itemprop="text">
<p>Since python dictionaries must have unique keys, counting how many times a key occurs wont work here. You can read the <a href="https://docs.python.org/3/tutorial/datastructures.html#dictionaries" rel="nofollow noreferrer">documentation</a> for more comprehensive details about this data structure. </p>
<p>Furthermore, You can store counts for each name in a dictionary:</p>
<pre><code>counts = {'Brian': 4, 'John': 2, 'Sam': 2, 'Henry': 1}
</code></pre>
<p>Then call each key to obtain the count value:</p>
<pre><code>&gt;&gt;&gt; counts['Brian']
4
&gt;&gt;&gt; counts['Sam']
2
</code></pre>
<p>You could also just keep the names as a list, and call <a href="https://docs.python.org/3.6/library/collections.html#collections.Counter" rel="nofollow noreferrer"><code>collections.Counter</code></a> to count how many times a name occurs:</p>
<pre><code>&gt;&gt;&gt; from collections import Counter
&gt;&gt;&gt; names = ['John', 'Sam', 'Brian', 'Brian', 'Brian', 'Sam', 'John', 'Henry', 'Brian']
&gt;&gt;&gt; Counter(names)
Counter({'Brian': 4, 'John': 2, 'Sam': 2, 'Henry': 1})
</code></pre>
<p>Which returns a <code>Counter()</code> object, a subclass of <code>dict</code>. </p>
