<div class="post-text" itemprop="text">
<p>The package you are using from <a href="https://github.com/BillMills/python-package-example" rel="nofollow noreferrer">https://github.com/BillMills/python-package-example</a> is using an import style not supported in Python 3. You can see in <a href="https://github.com/BillMills/python-package-example/blob/master/myPackage/__init__.py#L1" rel="nofollow noreferrer">python-package-example/__init__.py</a> that</p>
<pre><code>import somePython
</code></pre>
<p>is used to import a submodule, but python 3 will assume <code>somePython</code> exists as a top level module. See <a href="https://docs.python.org/3/whatsnew/2.5.html#pep-328-absolute-and-relative-imports" rel="nofollow noreferrer">PEP-328</a> for a better explanation.</p>
<p>A Python 3 compatible example package can be found at <a href="https://github.com/kennethreitz/samplemod" rel="nofollow noreferrer">https://github.com/kennethreitz/samplemod</a>. Notice that in <a href="https://github.com/kennethreitz/samplemod/blob/master/sample/__init__.py" rel="nofollow noreferrer">sample/__init__.py</a> submodules are imported using a relative import (indicated by the leading '.'):</p>
<pre><code>from .core import hmm 
</code></pre>
<p>If you wanted to modify python-package-example to be compatible you would need to change its __init__.py to use an absolute import:</p>
<pre><code>import myPackage.somePython as somePython
</code></pre>
<p>or a relative import:</p>
<pre><code>from . import somePython
</code></pre>
