<p><code>os.path.join()</code></p>
<p>This safely joins the paths together as a string.  It uses <code>os.sep</code> to ensure the correct path separator is used for the os platform.  It can correct extra separators.</p>
<pre><code>mypath = os.path.join("/some/path/", "/directory")
print(mypath) # "/some/path/directory"
</code></pre>
</div>
<span class="comment-copy">Keep in mind that this is <a href="https://docs.python.org/3/library/os.path.html#os.path.join" rel="nofollow noreferrer"><code>os.path.join</code></a>, not <code>str.join</code>.  <code>os.path.join</code> takes one or more paths and joins them together using <code>os.sep</code></span>

<span class="comment-copy"><code>data_directory</code> is some directory, say <code>/path/to/</code>.  <code>listdir(data_directory)</code> is an iterable of all of the things in that directory (files, other directories, etc.). So <code>path.join(data_directory, d)</code> is just the full path to something in <code>data_directory</code>.  Say there is some file <code>file</code>.  Then <code>d</code> is <code>'file'</code>, <code>data_directory</code> is <code>'/path/to/'</code>, and <code>path.join(data_directory, file)</code> is <code>/path/to/file</code></span>
