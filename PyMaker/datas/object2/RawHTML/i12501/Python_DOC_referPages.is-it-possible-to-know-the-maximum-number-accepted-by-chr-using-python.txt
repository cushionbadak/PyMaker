<p>From the Python's documentation of the <a href="https://docs.python.org/3.5/library/functions.html#chr" rel="nofollow"><code>chr</code></a> built-in function, the maximum value that <code>chr</code> accepts is 1114111 (in decimal) or 0x10FFFF (in base 16). And in fact</p>
<pre><code>&gt;&gt;&gt; chr(1114112)
Traceback (most recent call last):
  File "&lt;pyshell#20&gt;", line 1, in &lt;module&gt;
    chr(1114112)
ValueError: chr() arg not in range(0x110000)
</code></pre>
<p>My first question is the following, why exactly that number? The second question is: if this number changes, is it possible to know from a Python command the maximum value accepted by <code>chr</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/3/library/sys.html#sys.maxunicode" rel="nofollow"><code>sys.maxunicode</code></a>:</p>
<blockquote>
<p>An integer giving the value of the largest Unicode code point, i.e. <code>1114111</code> (<code>0x10FFFF</code> in hexadecimal).</p>
</blockquote>
<p>On my Python 2.7 UCS-2 build the maximum Unicode character supported by <code>unichr()</code> is 0xFFFF:</p>
<pre><code>&gt;&gt;&gt; import sys
&gt;&gt;&gt; sys.maxunicode
65535
</code></pre>
<p>but Python 3.3 and newer switched to a new internal storage format for Unicode strings, and the maximum is now <em>always</em> <code>0x10FFFF</code>. See <a href="https://www.python.org/dev/peps/pep-0393/" rel="nofollow">PEP 393</a>.</p>
<p><code>0x10FFFF</code> is the maximum Unicode codepoint as defined in the Unicode standard. Quoting the <a href="https://en.wikipedia.org/wiki/Unicode#Architecture_and_terminology" rel="nofollow">Wikipedia article on Unicode</a>:</p>
<blockquote>
<p>Unicode defines a codespace of 1,114,112 code points in the range 0 to 10FFFF.</p>
</blockquote>
