<p>I'm scraping a .txt file in python, and I'm using regex to capture ticket numbers and names:</p>
<pre><code>for line in Text:
    if re.match('Ticket:|Name:', line):
        print(line)
</code></pre>
<p>Based on my .txt file, this is capturing the correct fields as well as my blank lines (not yet filled out):</p>
<pre><code>Ticket:  123
Name:  Person123
Ticket:  124
Name:  Person124
Ticket:  
Name:  
Ticket:  
Name:
Ticket:  
Name:
</code></pre>
<p>If I append a <code>\n</code> to my regex it'll print my blank lines, but is there a way to keep my "filled out" lines and also exclude the blanks with their line break?</p>

<div class="post-text" itemprop="text">
<p>the following code will skip empty lines by adding <strong>\s+</strong> which Matches Unicode whitespace characters (which includes <code>[\t\n\r\f\v]</code>) one or more time then adding <strong>\w+</strong> which Matches Unicode word characters one or more; this includes most characters that can be part of a word in any language, as well as numbers and the underscore. If the ASCII flag is used, only <code>[a-zA-Z0-9_]</code> is matched </p>
<pre><code>for line in Text:
   if re.match('Ticket:\s+\w+|Name:\s+\w+', line):
        print(line)
</code></pre>
<p>output:</p>
<pre><code>Ticket:  123
Name:  Person123
Ticket:  124
Name:  Person124
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Perhaps you could use <a href="https://docs.python.org/3/library/stdtypes.html#str.rstrip" rel="nofollow noreferrer">rstrip</a> and specify a newline <code>\n</code></p>
<pre><code>for line in Text:
    if re.match('Ticket:|Name:', line):
        print(line.rstrip('\n'))
</code></pre>
