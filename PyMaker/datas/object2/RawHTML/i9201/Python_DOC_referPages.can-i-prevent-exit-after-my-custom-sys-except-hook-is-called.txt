<p>Problematic code:  </p>
<pre><code>import sys
def handler(exception_type, exception_value, traceback_object):
    print(exception_type, exception_value)    
sys.excepthook = handler

defined_thing = "Redacted faster than dissenting tweets"
print(defined_thing)
print(undefined_thing)
print("I wish I got here...")
</code></pre>
<p>Result:</p>
<pre><code>Redacted faster than dissenting tweets
&lt;class 'NameError'&gt; name 'undefined_thing' is not defined

Process finished with exit code 1
</code></pre>
<p>Problem: can I prevent the exit and proceed having printed the error?</p>
</div>
<div class="post-text" itemprop="text">
<h2>No.</h2>
<p>From <a href="https://docs.python.org/3/library/sys.html#sys.excepthook" rel="nofollow noreferrer">the docs</a> (emphasis mine):</p>
<blockquote>
<p>When an exception is raised and <strong>uncaught</strong>, the interpreter calls
  <code>sys.excepthook</code> with three arguments, the exception class, exception
  instance, and a traceback object. In an interactive session this
  happens just before control is returned to the prompt; <strong>in a Python
  program this happens just before the program exits</strong>.</p>
</blockquote>
<p>Your except hook will run, then the program will exit. If you want to catch exceptions, you need to use the usual <a href="https://docs.python.org/3/tutorial/errors.html#handling-exceptions" rel="nofollow noreferrer"><code>try-except</code></a> formula.</p>
</div>
<div class="post-text" itemprop="text">
<p>See the <a href="https://docs.python.org/2/library/sys.html" rel="nofollow noreferrer">documentation</a> for that function:</p>
<p><strong>sys.excepthook(type, value, traceback)</strong></p>
<p><em>This function prints out a given traceback and exception to sys.stderr.</em></p>
<p>It does nothing but provide additional debugging facilities.</p>
