<p>I'm attempting to read in a line of integer values separated by spaces and then perform a few simple statistical calculations on them. </p>
<p>An example line of input looks like so:</p>
<pre><code>64630 11735 14216
</code></pre>
<p>I decided to use the <code>statistics</code> module from the Python Standard Library documented here: <a href="https://docs.python.org/3/library/statistics.html" rel="nofollow noreferrer">https://docs.python.org/3/library/statistics.html</a></p>
<p>Here is it in action:</p>
<pre><code>import sys
import statistics

values = map(int, sys.stdin.readline().split())

print(statistics.mean(values))
print(statistics.median(values))
print(statistics.mode(values))
</code></pre>
<p>Upon executing, I get the following output from <code>stderr</code>:</p>
<pre><code>Traceback (most recent call last):
  File "file.py", line 7, in &lt;module&gt;
    print(statistics.median(values))
  File "/usr/lib/python3.4/statistics.py", line 313, in median
    raise StatisticsError("no median for empty data")
statistics.StatisticsError: no median for empty data
</code></pre>
<p>And so it seems the first call to <code>statistics.mean()</code> is consuming all of the elements in the <code>values</code> list before the call to <code>statistics.median()</code> can use them. (I do see the output from the first <code>print</code> statement from <code>stdout</code>, so the call to <code>statistics.mean()</code> worked fine.)</p>
<p>The input handling doesn't seem to have any issues. I have the following code that works as expected:</p>
<pre><code>import sys
import statistics

values = map(int, sys.stdin.readline().split())

for v in values:
    print(v)
</code></pre>
<p>Outputting the following from <code>stdout</code>:</p>
<pre><code>64630
11735
14216
</code></pre>
<p>Nothing in the standard library documentation mentions these functions consuming the parameter passed in. Each of these standard functions should only return a value and not alter the list itself.</p>
<p>Is there something specific to Python where a list can only be read or iterated on once? Or is there possibly a quirk with both of the compilers I've tried?</p>

<div class="post-text" itemprop="text">
<p><code>map()</code> produces an iterable, and iteration over that can only be done <em>once</em>. If you want to be able to re-use the result, produce a list first from the <code>map()</code> results first:</p>
<pre><code>values = list(map(int, sys.stdin.readline().split()))
</code></pre>
<p>You could also use a list comprehension:</p>
<pre><code>values = [int(i) for i in sys.stdin.readline().split()]
</code></pre>
<p>Demo, using <code>list(map(...))</code>:</p>
<pre><code>&gt;&gt;&gt; from io import StringIO
&gt;&gt;&gt; import statistics
&gt;&gt;&gt; demo = StringIO('64630 11735 14216\n')
&gt;&gt;&gt; values = list(map(int, demo.readline().split()))
&gt;&gt;&gt; print(statistics.mean(values))
30193.666666666668
&gt;&gt;&gt; print(statistics.median(values))
14216
&gt;&gt;&gt; print(statistics.mode(values))
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
  File "/Users/mjpieters/Development/Library/buildout.python/parts/opt/lib/python3.6/statistics.py", line 507, in mode
    'no unique mode; found %d equally common values' % len(table)
statistics.StatisticsError: no unique mode; found 3 equally common values
</code></pre>
<p>Your input of 3 <em>unique</em> values has no mode, hence the last exception.</p>
