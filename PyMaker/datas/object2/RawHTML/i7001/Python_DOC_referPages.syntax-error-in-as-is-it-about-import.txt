<div class="post-text" itemprop="text">
<p><code>as</code> is a <a href="https://docs.python.org/3/reference/lexical_analysis.html#keywords" rel="nofollow noreferrer"><em>reserved keyword</em></a> (as part of the <a href="https://docs.python.org/3/reference/compound_stmts.html#the-with-statement" rel="nofollow noreferrer"><code>with</code> statement</a>, an <code>except</code> handler in a <a href="https://docs.python.org/3/reference/compound_stmts.html#the-try-statement" rel="nofollow noreferrer"><code>try</code> statement</a>, or an <a href="https://docs.python.org/3/reference/simple_stmts.html#the-import-statement" rel="nofollow noreferrer"><code>import</code></a>); keywords can't be used as names.</p>
<p>Use a different name; a common approach is to add a <code>_</code> after the name:</p>
<pre><code>self.config.rng_address.random_numbers(as_ = rng, outsz =  6)
</code></pre>
<p>However, the <a href="https://github.com/PeterBorah/ethereum-powerball/blob/48f5d887620c64f3a857468eec26db3de3a8a690/contracts/lotto.se#L72" rel="nofollow noreferrer">code you copied</a> is <strong>not Python code</strong>:</p>
<blockquote>
<p>"Powerball"-style lottery <strong><em>written in Serpent</em></strong>, for prophetx's bounty.</p>
</blockquote>
<p>(Bold italic emphasis mine).</p>
<p>And, from the <a href="https://ethereumbuilders.gitbooks.io/guide/content/en/serpent_features.html" rel="nofollow noreferrer"><em>Serpent Builder's Guide</em></a>:</p>
<blockquote>
<p>Serpent is one of the high-level programming languages used to write Ethereum contracts. The language, as suggested by its name, <strong><em>is designed to be very similar to Python</em></strong>[.]</p>
</blockquote>
<p>It will fail on other issues too; all the files have <code>.se</code> extensions, not <code>.py</code> extensions, to indicate this is a different programming language. There are a lot of similarities with Python, but there are <em>more differences</em> that can't just be ignored. Use the <a href="https://github.com/ethereum/serpent" rel="nofollow noreferrer">serpent compiler</a> instead. </p>
