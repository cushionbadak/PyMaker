<div class="post-text" itemprop="text">
<p><code>sys.modules</code> only shows modules that have already been loaded into your environment.</p>
<blockquote>
<h1><a href="https://docs.python.org/3/library/sys.html#sys.modules" rel="nofollow noreferrer">sys.<strong>modules</strong></a></h1>
<p>This is a dictionary that maps module names to modules
  which have already been loaded. This can be manipulated to force
  reloading of modules and other tricks. However, replacing the
  dictionary will not necessarily work as expected and deleting
  essential items from the dictionary may cause Python to fail.</p>
</blockquote>
<p>To get the behaviour you want, you will need to import the module first:</p>
<pre><code>import sys
import packagename
'packagename' in sys.modules #True
</code></pre>
<h3>Console</h3>
<p>If you are in the console, and want to check if you have a module available, just try to import it. If it doesn't exist, you will get an error:</p>
<pre><code>&gt;&gt;&gt; import packagename
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
ModuleNotFoundError: No module named 'packagename'
</code></pre>
<h3>Code</h3>
<p>If you are writing code and want a clean way to check if the module exists before you use it, it is <a href="https://docs.python.org/3.4/glossary.html#term-eafp" rel="nofollow noreferrer">easier to ask for forgiveness than permission</a>.</p>
<pre><code>try:
    import packagename
except ImportError:
    #handle the error
</code></pre>
