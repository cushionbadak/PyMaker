<div class="post-text" itemprop="text">
<p>I'm having some issue to round up and down of a list of number to the nearest 1000.</p>
<p>Below is my code:</p>
<pre><code>rev_list = []

for i in range(12):
    rev = int(round(random.normalvariate(100000, 12000)))
    rev_list.append(rev)

print(rev_list)
</code></pre>
<p>The output is:</p>
<pre><code>[97277, 96494, 104541, 132060, 98179, 87862, 84718, 95391, 94674, 89773, 92790, 86122]
</code></pre>
<p>I would like to round the list to the nearest 1000. How can I do that?</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/functions.html#round" rel="noreferrer">The <code>round</code> function</a> can take <em>negative</em> digits to round to, which causes it to round off to the <em>left</em> of the decimal. For example:</p>
<pre><code>&gt;&gt;&gt; round(15768, -3)
16000
&gt;&gt;&gt; round(1218, -3)
1000
</code></pre>
<p>So the short answer is: Call <code>round</code> with the second argument of <code>-3</code> to round to the nearest 1000.</p>
</div>
<div class="post-text" itemprop="text">
<p>List comprehension is a one-line loop which allows you to apply a function to the list items. (for more read <a href="https://www.python.org/dev/peps/pep-0202/" rel="nofollow noreferrer">List Comprehensions</a>)</p>
<pre><code>[x for x in rev_list]
</code></pre>
<p>In this case, round(num, -3) is the function. </p>
<pre><code>&gt;&gt;&gt; round(1300,-3)
1000
&gt;&gt;&gt;
</code></pre>
<p><strong>The answer</strong></p>
<p>You can apply a function on a list by this code</p>
<pre><code>rev_list=[round(x,-3) for x in rev_list]
</code></pre>
<p>The example:</p>
<pre><code>&gt;&gt;&gt; rev_list=[97277, 96494, 104541, 132060, 98179, 87862, 84718, 95391, 94674, 89773, 92790, 86122]
&gt;&gt;&gt; rev_list=[round(x,-3) for x in rev_list]
&gt;&gt;&gt; rev_list
[97000, 96000, 105000, 132000, 98000, 88000, 85000, 95000, 95000, 90000, 93000, 86000]
&gt;&gt;&gt;
</code></pre>
</div>
<span class="comment-copy">Note: My example behavior is the Python 3 behavior for <code>round</code>; if you're on Python 2, the results would be <code>float</code> values, but wrapping in <code>int()</code> would convert back to <code>int</code>.</span>
