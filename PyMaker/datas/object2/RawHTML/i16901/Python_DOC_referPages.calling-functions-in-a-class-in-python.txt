<p>Suppose that I have the following in a single <code>.py</code> file:</p>
<pre><code>class Graph( object ):
    def ReadGraph( file_name ):

def ProcessGraph(file_name, verbose):
    g=ReadGraph(file_name)
</code></pre>
<p>where <code>ProcessGraph</code> is a driver class.  When I type</p>
<pre><code>ProcessGraph('testcase.txt', verbose=True)
</code></pre>
<p>I get this error</p>
<pre><code>NameError: global name 'ReadGraph' is not defined
</code></pre>
<p>Could someone explain how to fix this error?</p>

<p>Just decorate them with @staticmethod</p>
<pre><code>class Graph( object ):
    @staticmethod
    def ReadGraph( file_name ):
         print 'read graph'

    @staticmethod
    def ProcessGraph(file_name, verbose):
         g=ReadGraph(file_name)

if __name__=="__main__":
    Graph.ProcessGraph('f', 't')
</code></pre>
<p>Outputs 'hello'. </p>

<div class="post-text" itemprop="text">
<p><code>ReadGraph</code> is in the namespace of the <code>Graph</code> class, which is why you can't call it as a high-level function. Try this:</p>
<pre><code>class Graph(object):
     @classmethod
     def ReadGraph(cls, file_name):
         # Something

def ProcessGraph(file_name, verbose):
     g=Graph.ReadGraph(file_name)
</code></pre>
<p>The <a href="https://docs.python.org/3/library/functions.html#classmethod" rel="nofollow noreferrer"><code>@classmethod</code></a> decorator will let you call <code>ReadGraph</code> on a class without creating a class instance.</p>
</div>
