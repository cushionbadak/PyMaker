<p>To keep a couple modules I'm building in Python a bit consistent I want to do some automated code checking. For this I want to check the modules for their functions and the arguments the functions take. I can use hasattr() to check whether the module actually contains the expected functions. So far so good.</p>
<p>I now want to see which arguments the functions take. It would be enough to simply see the variable names. I have no idea how I would be able to do this though. Does anybody know how I can get the names of the arguments a function takes?</p>

<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/2/library/inspect.html#inspect.getargspec" rel="nofollow"><code>inspect.getargspec()</code></a> to see what arguments are accepted, and any default values for keyword arguments.</p>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; def foo(bar, baz, spam='eggs', **kw): pass
... 
&gt;&gt;&gt; import inspect
&gt;&gt;&gt; inspect.getargspec(foo)
ArgSpec(args=['bar', 'baz', 'spam'], varargs=None, keywords='kw', defaults=('eggs',))
&gt;&gt;&gt; inspect.getargspec(foo).args
['bar', 'baz', 'spam']
</code></pre>
<p>In Python 3, you want to use <a href="https://docs.python.org/3/library/inspect.html#inspect.getfullargspec" rel="nofollow"><code>inspect.getfullargspec()</code></a> as this method supports new Python 3 function argument features:</p>
<pre><code>&gt;&gt;&gt; def foo(bar: str, baz: list, spam: str = 'eggs', *, monty: str = 'python', **kw) -&gt; None: pass
... 
&gt;&gt;&gt; import inspect
&gt;&gt;&gt; inspect.getfullargspec(foo)
FullArgSpec(args=['bar', 'baz', 'spam'], varargs=None, varkw='kw', defaults=('eggs',), kwonlyargs=['monty'], kwonlydefaults={'monty': 'python'}, annotations={'baz': &lt;class 'list'&gt;, 'return': None, 'spam': &lt;class 'str'&gt;, 'monty': &lt;class 'str'&gt;, 'bar': &lt;class 'str'&gt;})
</code></pre>
<p><code>inspect.getargspec()</code> should be considered deprecated in Python 3. </p>
<p>Python 3.4 adds the <a href="https://docs.python.org/3/library/inspect.html#introspecting-callables-with-the-signature-object" rel="nofollow"><code>inspect.Signature()</code> object</a>:</p>
<pre><code>&gt;&gt;&gt; inspect.signature(foo)
&lt;inspect.Signature object at 0x100bda588&gt;
&gt;&gt;&gt; str(inspect.signature(foo))
"(bar:str, baz:list, spam:str='eggs', *, monty:str='python', **kw) -&gt; None"
&gt;&gt;&gt; inspect.signature(foo).parameters
mappingproxy(OrderedDict([('bar', &lt;Parameter at 0x100bd67c8 'bar'&gt;), ('baz', &lt;Parameter at 0x100bd6ea8 'baz'&gt;), ('spam', &lt;Parameter at 0x100bd69f8 'spam'&gt;), ('monty', &lt;Parameter at 0x100bd6c28 'monty'&gt;), ('kw', &lt;Parameter at 0x100bd6548 'kw'&gt;)]))
</code></pre>
<p>and many more interesting options to play with signatures.</p>
