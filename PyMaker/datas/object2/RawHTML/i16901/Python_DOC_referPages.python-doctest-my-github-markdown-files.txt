<p>As an alternative to <a href="https://docs.python.org/3/library/doctest.html" rel="nofollow">doctest</a> I wrote <a href="https://github.com/ryneeverett/mkcodes" rel="nofollow">mkcodes</a>, a script that pulls code blocks out of markdown files so they can be tested in a separate file.</p>
<p>Here's my actual test script using mkcodes:</p>
<pre><code>mkcodes --github --output tests/docs/test_{name}.py docs
py.test tests
pyflakes tests
pep8 tests
</code></pre>

<div class="post-text" itemprop="text">
<p>You can run <code>doctest</code> on your <code>README</code> on the command line using:</p>
<pre><code>python -m doctest -v README.md
</code></pre>
<p>The <a href="https://docs.python.org/3.6/using/cmdline.html#interface-options" rel="nofollow noreferrer"><code>-m</code> parameter</a> tells Python to run the following module as a script. <a href="https://docs.python.org/3.6/library/doctest.html#simple-usage-checking-examples-in-docstrings" rel="nofollow noreferrer">When run as a script</a>, the <code>doctest</code> module runs the <a href="https://docs.python.org/3.6/library/doctest.html#doctest.testmod" rel="nofollow noreferrer"><code>doctest.testmod</code></a> function on the following file. Lastly, the <code>-v</code> parameter makes <code>doctest</code> run in verbose mode; if it's turned off, <code>doctest</code> only produces output if at least one test fails (and will produce no output if everything is successful).</p>
</div>
