<p>No, the output is not the same; one is a Unicode value, the other an undecoded bytes value.</p>
<p>For ASCII, that <em>looks</em> the same, but when you load any web page that uses characters outside the ASCII characterset, the difference will be much clearer.</p>
<p>Take UTF-8 encoded data, for example:</p>
<pre><code>&gt;&gt;&gt; '–'
'–'
&gt;&gt;&gt; '–'.encode('utf8')
b'\xe2\x80\x93'
</code></pre>
<p>That's a simple <a href="http://codepoints.net/U+2013" rel="nofollow">U+2013 EN DASH</a> character. The bytes representation shows the 3 bytes UTF-8 uses to encode the codepoint.</p>
