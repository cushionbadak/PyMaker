<p>I have a connected socket. When I use:</p>
<pre><code>print (mySocket)
</code></pre>
<p>I get this:</p>
<p><code>&lt;socket.socket fd=376, family=AddressFamily.AF_INET, type=SocketKind.SOCK_STREAM, proto=0, laddr=('192.168.31.244', 4160), raddr=('192.168.31.244', 7061)&gt;</code></p>
<p>I can also successfully print:</p>
<pre><code>print (mySocket.family)
print (mySocket.proto)
</code></pre>
<p>But if I try to print the address:</p>
<pre><code>print(mySocket.laddr)
</code></pre>
<p>I get and error:</p>
<pre><code>AttributeError: 'socket' object has no attribute 'laddr'
</code></pre>
<p>How can I print the <em>laddr</em> and <em>raddr</em> attributes?</p>
</div>
<div class="post-text" itemprop="text">
<p>Try using the <a href="https://docs.python.org/3/library/socket.html#socket.socket.getsockname" rel="nofollow noreferrer"><code>.getsockname()</code></a> and <a href="https://docs.python.org/3/library/socket.html#socket.socket.getpeername" rel="nofollow noreferrer"><code>.getpeername()</code></a> methods instead. As noted <a href="https://docs.python.org/3/library/socket.html#socket-objects" rel="nofollow noreferrer">in the Socket object docs</a> only the <code>family</code>, <code>proto</code>, and <code>type</code> fields are available as attributes.</p>
<pre><code>&gt;&gt;&gt; s.bind(('localhost',12345))
&gt;&gt;&gt; s.getsockname()
('127.0.0.1', 12345)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>For <code>laddr</code> use <code>mySocket.getsockname()</code> and for <code>raddr</code> use <code>mySocket.getpeername()</code></p>

<span class="comment-copy">@1qazxsw2 when you print out an object, the things you see aren't necessarily attributes on the object - it can print whatever it wants via the <code>__str__</code> and <code>__repr__</code> methods. So chances are what's happening is that the object is looking at internal private variables to construct that string representation, but you can't access those variables directly.</span>
