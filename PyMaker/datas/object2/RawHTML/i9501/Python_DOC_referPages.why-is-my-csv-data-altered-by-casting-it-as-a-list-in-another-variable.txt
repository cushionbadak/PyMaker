<div class="post-text" itemprop="text">
<p>If I run</p>
<pre><code>input_file = csv.DictReader(open("input2.csv"))
for row in input_file:
      print (row)
</code></pre>
<p>It works as I would expect and prints out the csv file. However if I add the line</p>
<pre><code>input_file = csv.DictReader(open("input2.csv"))
data = list(input_file)
for row in input_file:
      print (row)
</code></pre>
<p>It now does not print anything out. Why does assigning the data variable to a list of the input_file, alter input_file? </p>
</div>
<div class="post-text" itemprop="text">
<p><code>input_file</code> is an <a href="https://docs.python.org/3/howto/functional.html#iterators" rel="nofollow noreferrer">iterator</a>. Your call <code>list(input_file)</code> exhausts it.
That is, it iterates through all lines and <em>yields</em> each one of them to <code>data</code>, after which it has no more lines to yield.</p>
<p>Try:</p>
<pre><code>data = list(input_file)
for row in data:
      print (row)
</code></pre>
<p>to see that the data is still there.</p>
