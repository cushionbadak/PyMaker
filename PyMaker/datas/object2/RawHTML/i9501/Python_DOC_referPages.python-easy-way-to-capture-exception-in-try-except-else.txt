<p>In Python, we can use <code>as</code> to capture the exception instance in the <code>except</code> statement. However, there does not seem to have an easy way to do the same in an <code>else</code> statement after the <code>try</code>. To be more clear, see the code below.</p>
<pre><code>try:
    raise Exception('Foo')
except ValueError as valueError:
    print(valueError)
    print('I can capture the exception instance with "as" keyword')
else:    # Cannot use "as" here
    print('Some exception other than ValueError')
    print('How can I capture the exception in order to, e.g. print its message?')
</code></pre>
<p>Any idea?</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>try:
    raise Exception('Foo')
except ValueError as valueError:
    print(valueError)
    print('I can capture the exception instance with "as" keyword')
except Exception as e:  
    print(e)
    print('Some exception other than ValueError')
else:
    print('no exception raised')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Use multiple <a href="https://docs.python.org/3/tutorial/errors.html#handling-exceptions" rel="nofollow noreferrer"><code>except</code></a> clauses:</p>
<p>Either like this:</p>
<pre><code>try:
    f = open('myfile.txt')
    s = f.readline()
    i = int(s.strip())
except OSError as err:
    print("OS error: {0}".format(err))
except ValueError:
    print("Could not convert data to an integer.")
except:
    print("Unexpected error:", sys.exc_info()[0])
    raise
</code></pre>
<p>or clubbing exceptions together:</p>
<pre><code>except (RuntimeError, TypeError, NameError):
    pass
</code></pre>
