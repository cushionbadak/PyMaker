<p>I have a string input, such as <code>100124</code>. I want to evaluate each digit of the string as an integer, so I do:</p>
<pre><code>for c in string:
    c = int(c)
    # do stuff with c
</code></pre>
<p>Is there a syntactically better way to do this? I tried doing <code>string = int(string)</code> before the loop, but a number isn't iterable.</p>
</div>
<div class="post-text" itemprop="text">
<p>You could <a href="https://docs.python.org/3/library/functions.html#map" rel="noreferrer"><code>map</code></a> the string to <code>int</code>:</p>
<pre><code>for c in map(int, string):
    # do stuff with c
</code></pre>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; for c in map(int, '100124'):
...     c
...
1
0
0
1
2
4
&gt;&gt;&gt;
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can use a list comprehension that return a generator :</p>
<pre><code>&gt;&gt;&gt; [int(i) for i in s]
[1, 0, 0, 1, 2, 4]
&gt;&gt;&gt; for digit in [int(i) for i in s] :
...    # do stuff with digit
</code></pre>
<p>or <a href="https://docs.python.org/2/library/functions.html#map" rel="nofollow"><code>map</code></a> function :</p>
<pre><code>&gt;&gt;&gt; map(int,s)
[1, 0, 0, 1, 2, 4]
</code></pre>
