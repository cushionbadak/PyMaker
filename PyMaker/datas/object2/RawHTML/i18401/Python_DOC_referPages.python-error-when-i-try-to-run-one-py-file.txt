<p><strong>Answer for Python 3.x</strong>
If you're using Python3.x change <code>from BaseHTTPServer</code> to <code>from http.server</code>.</p>
<p>If you wrote this code for Python 2.x and you are running it with Python3.x, The <a href="https://www.google.co.uk/url?sa=t&amp;rct=j&amp;q=&amp;esrc=s&amp;source=web&amp;cd=1&amp;cad=rja&amp;ved=0CDEQFjAA&amp;url=http%3A%2F%2Fdocs.python.org%2Flibrary%2F2to3.html&amp;ei=50grUqbZCcHb0QWgyYCIBw&amp;usg=AFQjCNGRPJXHYMfkDrULR3M3DdRFaW0vQg&amp;sig2=uqL39-GQaCooeI3s6ZBjzg&amp;bvm=bv.51773540,d.d2k" rel="nofollow">2to3</a> tool will automatically adapt imports when converting your sources to Python 3.</p>
<p><strong>Answer for Python 2.x</strong>
The error is telling you that <code>BaseHTTPServer</code> needs to be in your <code>PYTHONPATH</code>.</p>
<p>That is to say, Python cannot find the module <code>BaseHTTPServer</code> anywhere, you either need to install it, or if it is installed in a non-standard location, modify your <code>PYTHONPATH</code> environment variable to include it - however this would be a bit of a strange (though not impossible) situation since that module is normally included in Python2.x</p>

<div class="post-text" itemprop="text">
<p>If you're using Python 3.x, try following:</p>
<pre><code>import http.server
import ssl

httpd = http.server.HTTPServer(('localhost', 4443), http.server.SimpleHTTPRequestHandler)
httpd.socket = ssl.wrap_socket(httpd.socket, certfile='server.pem', server_side=True)
httpd.serve_forever()
</code></pre>
<p><a href="http://docs.python.org/2/library/basehttpserver.html" rel="nofollow"><code>BaseHTTPServer</code></a>, <a href="http://docs.python.org/2/library/simplehttpserver.html" rel="nofollow"><code>SimpleHTTPServer</code></a> modules in Python 2 have been merged into <a href="http://docs.python.org/3/library/http.server" rel="nofollow"><code>http.server</code></a> module in Python 3.</p>
<p><strong>UPDATE</strong></p>
<p>BTW, port number seems wrong. HTTPS port is 443, not 4443.</p>
