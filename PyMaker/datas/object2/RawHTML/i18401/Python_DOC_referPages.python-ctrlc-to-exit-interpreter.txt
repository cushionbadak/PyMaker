<p>Python 2.73</p>
<p>Why is it on my laptop when I hit <kbd>CTRL</kbd>+<kbd>C</kbd>, I can exit the interpreter and on my desktop hitting <kbd>CTRL</kbd>+<kbd>C</kbd> will make the interpreter shoot back at me a KeyboardInterrupt message. How can I get rid of this KeyboardInterrupt and go back to exiting with <kbd>CTRL</kbd>+<kbd>C</kbd>! </p>
<p>On my desktop it's required to input <kbd>CTRL</kbd>+<kbd>Z</kbd> and hitting enter to exit.</p>
<p>I am using PowerShell on both computer. Same 64bit, one is Win7 one is Win8</p>
</div>
<div class="post-text" itemprop="text">
<p>You could change the signal handler for <kbd>CTRL</kbd>-<kbd>C</kbd> to something that exits the interpreter:</p>
<pre><code>import signal
import sys
signal.signal(signal.SIGINT, lambda number, frame: sys.exit())
</code></pre>
<p>You could probably put that code in a file to be run automatically when an interactive session starts, then set the environment variable PYTHONSTARTUP to the name of that file:</p>
<p><a href="http://docs.python.org/3/using/cmdline.html?highlight=startup#envvar-PYTHONSTARTUP" rel="nofollow noreferrer">http://docs.python.org/3/using/cmdline.html?highlight=startup#envvar-PYTHONSTARTUP</a></p>
</div>
<div class="post-text" itemprop="text">
<p>A slightly shorter version of the previous answer:</p>
<pre><code>import signal
signal.signal(signal.SIGINT, signal.SIG_DFL)
</code></pre>
<p><code>SIG_DFL</code> means default signal handling, so Python does not catch it to
raise a <code>KeyboardInterrupt</code> exception.</p>
