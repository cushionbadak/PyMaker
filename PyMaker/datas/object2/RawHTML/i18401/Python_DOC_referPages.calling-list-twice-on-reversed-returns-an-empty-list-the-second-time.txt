<p>I don't understand the result of this code:  </p>
<pre><code>aa = 'hello, world'
bb = reversed(aa)
print(bb)
print(list(bb))
print(bb)
dd = list(bb)
print(dd)
print(''.join(dd))
</code></pre>
<p>The result:</p>
<pre><code>&lt;reversed object at 0x025C8C90&gt;
['d', 'l', 'r', 'o', 'w', ' ', ',', 'o', 'l', 'l', 'e', 'h']
&lt;reversed object at 0x025C8C90&gt;
[]
</code></pre>
<p>Why is <code>dd</code> <code>[]</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>That's because <a href="https://docs.python.org/3/library/functions.html#reversed" rel="nofollow"><code>reversed</code></a> creates an iterator, which is already spent when you're calling <code>list(bb)</code> for the second time.</p>
<pre><code>aa = 'hello, world'
bb = reversed(aa)    # Creates an iterator, not a list
print(bb)            # Prints "&lt;reversed object at 0x10c619b90&gt;" (that's the iterator)
print(list(bb))      # Pops items from the iterator to create a list, then prints the list
print(bb)            # At this point the iterator is spent, and doesn't contain elements anymore
dd = list(bb)        # Empty iterator -&gt; Empty list ([])
print(dd)
print(''.join(dd))
print('----------')
</code></pre>
<p>To fix that, just change <code>reversed(aa)</code> by <code>list(reversed(aa))</code>.</p>
