<pre><code>parser.add_argument('domain_name', metavar='domain name',
                    action='store', nargs=1, help='specify a domain name')
</code></pre>
<p>See the above line of code. It requires that the positional argument be specified, but it returns a list. To access <code>domain_name</code>, I need to access <code>args.domain_name[0]</code>. I'd like to access it as <code>args.domain_name</code> as it's confusing to see this list that can only ever be one item, and in fact must be one item as it's required.</p>
<p>If I specify <code>nargs='*'</code> it's no longer required but is returned as a string.</p>
</div>
<div class="post-text" itemprop="text">
<p>nargs isn't a required parameter and its removal will return a string and not a list as specified above.</p>
