<div class="post-text" itemprop="text">
<p>First, sizeof is no valid python function so your second try won't work.</p>
<p>Second, your attribute of the BOMRows Item object is named <code>__sizeof__</code>, so you have to use that and not just 'sizeof':</p>
<pre><code>oStructuredBOMView.BOMRows.Item(1).__sizeof__
</code></pre>
<p>Third, those are just attributes of the object, they may or may not be functions/methods, you can find out which are callable with:</p>
<pre><code>isMethod = callable(oStructuredBOMView.BOMRows.Item(1).__sizeof__)
</code></pre>
<p>See <a href="https://docs.python.org/3/library/functions.html#callable" rel="nofollow noreferrer">https://docs.python.org/3/library/functions.html#callable</a></p>
<p>Update: You can try to <code>print()</code> all the attributes (this is kind of obvious, so you maybe already tried that). As I don't know anything about AutoDesk Inventor models, I can't help you with a direct response, but you could try to dig deeper into the object. If the attribute is a 'primitive type' (read: printable object or object that has some string representation) you can use it directly. If your attribute returns another object, then you have to analyze it again with <code>dir()</code> or <code>vars()</code> again. You can ignore most of the double underscore attributes, as they are most likely from python.</p>
<p>Example of trying to print all attributes of an unknown object:</p>
<pre><code>for key in dir(my_obj):
    print('{}: {}'.format(key, getattr(my_obj, key))
</code></pre>
<p>But I am sure there should be some documentation available how to use the Autodesk Inventor objects.</p>

<p>The inventor object browser is your best friend to see the properties and methods of each class in inventor. To open it go to the VBA editor and press F2.</p>
<pre><code>&gt;&gt; print(oStructuredBOMView.BOMRows.Item(1))
&lt;win32com.gen_py.Autodesk Inventor Object Library.BOMRow instance at 0x88831536&gt;
</code></pre>
<p>This tells you that the class is BOMRow. Type the class name into the search field and go from there.</p>
