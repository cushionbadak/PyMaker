<p>Assume now I have a well-defined function</p>
<pre><code>def f1(a):
    ...
</code></pre>
<p>and i can not change its definition. Now I want to call it with a argument from a dictionary </p>
<pre><code>D={ 'a':1 , 'b':2}
</code></pre>
<p>If I call it like</p>
<pre><code>f1(**D)
</code></pre>
<p>there will be a syntax error (because b is not defined in f1). My question is if I can find a smart way to let the function to find the argument it needed only?</p>

<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/inspect.html#inspect.getargspec" rel="nofollow noreferrer"><code>inspect.getargspec</code></a>:</p>
<pre><code>d = {'a': 5, 'b': 6, 'c': 7}
def f(a,b):
    return a + b

f(*[d[arg] for arg in inspect.getargspec(f).args])
</code></pre>
<p>which gives <code>11</code>.</p>
<hr/>
<p>Thanks Eugene, <code>getargspec</code> is legacy, instead you should use <a href="https://docs.python.org/3/library/inspect.html#inspect.getfullargspec" rel="nofollow noreferrer"><code>getfullargspec</code></a> which has the same usage in this case.</p>
</div>
<div class="post-text" itemprop="text">
<p>You could get the argument names from the function's <a href="https://docs.python.org/3.7/reference/datamodel.html#index-55" rel="nofollow noreferrer">code object</a>, then lookup the values for them in the dictionary:</p>
<pre><code>&gt;&gt;&gt; def f(a):
...     b = 2
...     return a + b
...
&gt;&gt;&gt; D = {'a': 1, 'b': 2, 'c': 3}
&gt;&gt;&gt; nargs = f.__code__.co_argcount
&gt;&gt;&gt; varnames = f.__code__.co_varnames
&gt;&gt;&gt; f(*(D[arg] for arg in varnames[:nargs]))
3
</code></pre>
</div>
