<p>I would like to ask if there is any other way to write the following code and also if someone could explain what I'm doing wrong.</p>
<p>Suppose that <code>x,y</code> are integers.  My goal is to print the following calculations <code>x/y</code> and <code>x%y</code> with distance 10 between them.  In order to do that I have to take into account the length of each calculation, so I use <code>len(str())</code>, but I get <code>ValueError: Invalid format specifier</code></p>
<pre><code>z='{:(10+len(str(x/y)))f}{:10d}'.format(x/y,x%y)
print(z)
</code></pre>

<div class="post-text" itemprop="text">
<p>Why not simply put 10 spaces between?</p>
<pre><code>x = 5
y = 3
z = '{:f}{:&gt;10}{:10d}'.format(x/y," ",x%y)

print(z)
</code></pre>
<p>Output:</p>
<pre><code>      # 12345678901234567890
1.666667                   2
</code></pre>
<p>See <a href="https://docs.python.org/3/library/string.html#format-specification-mini-language" rel="nofollow noreferrer">format mini language</a></p>
<p>I am putting a single space right aligned to 10 places between the outputs. The decimal is also right-aligned by default and uses the specified 10 places.</p>
<hr/>
<p>You could also upp the righmost value by 10 and use <code>{:20d}</code> ...</p>
</div>
<span class="comment-copy">The problem is you wrapped all the <code>(10+len(str(x/y)))</code> in the string, and Python won't perform any calculation since it considers everything to be string characters.</span>
