<div class="post-text" itemprop="text">
<p>Using <a href="https://docs.python.org/3/library/functions.html#hex" rel="nofollow noreferrer">hex(...)</a> will only give you the least amount of leading 0:</p>
<pre><code>print( [hex(i) for i in [0,1]])  #  ['0x1', '0x2']
</code></pre>
<p>You can <a href="https://docs.python.org/3/library/functions.html#format" rel="nofollow noreferrer">format(...)</a> the string as hex and <a href="https://docs.python.org/3/library/stdtypes.html#str.zfill" rel="nofollow noreferrer">zfill()</a> the needed amount of zeros and prepending <code>'0x'</code> to it yourself:</p>
<pre><code>data = [0,1]

as_hex = [ "0x" + format(e,"x").zfill(2) for e in data]

print(as_hex)
</code></pre>
<p>Output:</p>
<pre><code>['0x00', '0x01']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As you fail to describe what your wanted outcome is, I can only guess:</p>
<ol>
<li><p>Do you want to pack some numbers to a string?</p>
<p>Then it doesn't matter which format you use, both should work as soon as you use the correct format string:</p>
<pre><code>a = [1, 2]
b = [0x01, 0x02]
struct.pack("&gt;hh", *a)
# &gt; '\x00\x01\x00\x02'
struct.pack("&gt;hh", *b)
# &gt; '\x00\x01\x00\x02'
</code></pre>
<p>This is because <code>a</code> and <code>b</code> are equal: it doesn't matter if you write <code>1</code> and <code>2</code> or if you prefer <code>0x01</code> resp. <code>0x02</code>. Be aware that the displayed strings are just representations of strings containing special characters with the values 0, 1 and 2, respectively.</p></li>
<li><p>Do you want to output your numbers in a hexadecimal representation?</p>
<p>In this case, see the other answers.</p></li>
</ol>

<p>Below is the another way of converting a list of integer to list of hex using a map function:</p>
<pre><code>a=[0,1]
print(map(hex,a))
# Output
# ['0x0', '0x1']
</code></pre>
