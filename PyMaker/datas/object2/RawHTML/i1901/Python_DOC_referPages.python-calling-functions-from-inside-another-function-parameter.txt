<p><strong>Question :</strong> Given a sentence, return a sentence with the words reversed</p>
<p><strong>My Solution:</strong></p>
<pre><code>def sentence_reverse(text):
    a = text.split()
    a.reverse()
    return ' '.join(a)
</code></pre>
<p><strong>Output:</strong></p>
<pre><code>sentence_reverse('I am home') --&gt; 'home am I'
sentence_reverse('We are ready') --&gt; 'ready are We'
</code></pre>
<p>the code works the way it is supposed to but my question is, why can't i make a function call directly from the returned result of the previous function. i,e</p>

<div class="post-text" itemprop="text">
<p>Because <code>reverse</code> returns None, and updates the list, so so None is not valid for <code>join</code>.</p>
<p>Yeah so <code>reverse</code> is <code>inplace</code> function so no way to fix that.</p>
<p>So Use <code>reversed</code>:</p>
<pre><code>return ' '.join(reversed(text.split()))
</code></pre>
<p>Or <code>[::-1]</code>:</p>
<pre><code>return ' '.join(text.split()[::-1])
</code></pre>
<p>From docs:</p>
<blockquote>
<p>You might have noticed that methods like insert, remove or sort that only modify the list have no return value printed â€“ they return the default None. [1] This is a design principle for all mutable data structures in Python.</p>
</blockquote>

<p>Here's the problem. In Python, calling <code>list.reverse()</code> doesn't <strong>return</strong> the reversed list. It just reverses it in place.</p>
<p>What you need is:</p>
<pre><code>return " ".join(text.split()[::-1])
</code></pre>
</div>

<pre><code>me = 'ready are We'
" ".join(me.split()[::-1])
&gt;&gt;&gt; 'We are ready'
</code></pre>
