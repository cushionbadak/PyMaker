<div class="post-text" itemprop="text">
<p>In the last line, you are assigning to <code>message.content</code>.</p>
<p>This is not possible with a built in <code>dict</code>.</p>
<pre><code>&gt;&gt;&gt; d = {}
&gt;&gt;&gt; d.content = 1
[...]
AttributeError: 'dict' object has no attribute 'content'
</code></pre>
<p>In fact, dictionaries don't even have a <code>__dict__</code> (obviously).</p>
<pre><code>&gt;&gt;&gt; vars(d)
[...]
TypeError: vars() argument must have __dict__ attribute
</code></pre>
<p>However, your custom type will share no such problems.</p>
<pre><code>&gt;&gt;&gt; class MyDict(dict):
...:    pass
...:
&gt;&gt;&gt; d = MyDict()
&gt;&gt;&gt; d.content = 0
&gt;&gt;&gt; vars(d)
&gt;&gt;&gt; {'content': 0}
</code></pre>
<p>If you plan to design a more sophisticated subclass of <code>dict</code>, inherit from <a href="https://docs.python.org/3/library/collections.html#collections.UserDict" rel="nofollow noreferrer"><code>collections.UserDict</code></a>. I have lined out some problems with inheriting from <code>dict</code> directly in the last part of <a href="https://stackoverflow.com/questions/36688966/let-a-class-behave-like-its-a-list-in-python/36690145#36690145">this</a> answer.</p>
