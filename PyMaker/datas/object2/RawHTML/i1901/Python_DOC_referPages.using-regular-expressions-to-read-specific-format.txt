<pre><code>str = "RegName1,Regname2,0x00000000,0x100000"

<pre><code>re.match('\s+,\s+,\d+,\d+',str)
</code></pre>
<p>Note: I want to ignore comments like <code>"//"</code>, <code>"#"</code>, etc..</p>

<div class="post-text" itemprop="text">
<p>Identify your fields this way:</p>
<pre><code>import re
text = "RegName1,Regname2,0x00000000,0x100000"
matches = re.match(r"(\w+),(\w+),(\w+),(\w+)", text)
</code></pre>
<p>Then, if you <code>print(matches)</code> and <code>print(matches[1])</code>, you'll get:</p>
<pre><code>Python 3.6.1 (default, Dec 2015, 13:05:11)
[GCC 4.8.2] on linux

&lt;_sre.SRE_Match object; span=(0, 37), match='RegName1,Regname2,0x00000000,0x100000'&gt;
RegName1
</code></pre>
<p>Not sure how to avoid comment lines. But, at least they won't be matched by regex. For example:</p>
<pre><code>text = "#Some comment"
matches = re.match(r"(\w+),(\w+),(\w+),(\w+)", text)
print(matches)
</code></pre>
<p>Throws:</p>
<pre><code>Python 3.6.1 (default, Dec 2015, 13:05:11)
[GCC 4.8.2] on linux

None
</code></pre>
<p>So if you do:</p>
<pre><code>if matches is not None:
    do something
</code></pre>
<p>..they will be avoided.</p>
