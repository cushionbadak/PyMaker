<div class="post-text" itemprop="text">
<p>You can use the <code>strftime()</code> function from the <code>time</code> module. This will store the current month in a variable called <code>current_month</code>.</p>
<pre><code>from time import strftime
current_month = strftime('%B')
</code></pre>
<p>If you want to get the current season from this, try this function:</p>
<pre><code>def get_season(month):
    seasons = {
    'Autumn': ['September', 'October', 'November', 'December', 'January'],
    'Spring': ['January', 'February', 'March', 'April'],
    'Summer': ['Apirl', 'May', 'June', 'July']
    }

    for season in seasons:
        if month in seasons[season]:
            return season
    return 'Invalid input month'
</code></pre>
<p>Currently, this will not solve the date conflicts that you specified, as any day in January will return <code>'Autumn'</code>, and any day in April will get you <code>'Spring'</code>.</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>time.strftime("%b") 
</code></pre>
<p>will tell you the current month (abbreviation)</p>
<pre><code>time.strftime("%m") 
</code></pre>
<p>will give you the numeric month (ie 1 = Jan / 12 = Dec)</p>
<p>(or even better)</p>
<pre><code>datetime.datetime.now().month
</code></pre>
<p>as that will give you an actual integer instead of a string (be for-warned though January is 0)</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/datetime.html#datetime.datetime.now" rel="nofollow"><code>datetime.datetime.now</code></a>:</p>
<pre><code>&gt;&gt;&gt; from datetime import datetime
&gt;&gt;&gt; datetime.now().month  # As a number
4
&gt;&gt;&gt; datetime.now().strftime("%B")  # As a name
'April'
&gt;&gt;&gt; datetime.now().strftime("%b")  # As an abbreviated name
'Apr'
&gt;&gt;&gt;
</code></pre>
