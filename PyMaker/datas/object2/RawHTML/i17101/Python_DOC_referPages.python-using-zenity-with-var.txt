<p>I'm trying to run Zenity in a python script, to display a variable.</p>
<pre><code>nmaj = 10
cmd = ["zenity" "--question" "--text='Are you " + str(nmaj) + "years old ?'"]
subprocess.call(cmd, shell=True)
</code></pre>
<p>Can I put a string in the command? How?
Thanks</p>

<div class="post-text" itemprop="text">
<p>You can try using <code>format</code> and putting <code>''</code> outer than <code>""</code>:</p>
<pre><code>nmaj = 10
cmd = 'zenity --question --text="Are you {} years old ?"'.format(nmaj)
subprocess.call(cmd, shell=True)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You miss comma(<code>,</code>)s between command line arguments:</p>
<pre><code>cmd = ["zenity", "--question", "--text='Are you " + str(nmaj) + "years old ?'"]
</code></pre>
<p>Otherwise the string literals are concatenated into a string (See <a href="https://docs.python.org/3/reference/lexical_analysis.html#string-literal-concatenation" rel="nofollow">String literal concatenation</a>):</p>
<pre><code>&gt;&gt;&gt; "zenity" "--question" "--text='Are you "
"zenity--question--text='Are you "
</code></pre>
</div>
<span class="comment-copy">Is that supposed to be one long string, or a list of strings. If the former you need to use <code>+</code> to append each string. If the latter, you need to use <code>,</code> to separate each element.</span>
<span class="comment-copy">As <a href="http://stackoverflow.com/a/22862944/4279">@falsetru demonstrates</a>: no need to use <code>shell=True</code>. Use <code>{:d}</code> to assert that <code>nmaj</code> is an integer.</span>
<span class="comment-copy"><code>--text</code> and its value could be passed as separated arguments: <code>[..., "--text", "Are you {n:d} years old?".format(n=nmaj)]</code></span>
