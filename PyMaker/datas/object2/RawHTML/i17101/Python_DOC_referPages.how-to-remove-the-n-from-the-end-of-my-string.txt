</code></pre>
<p>Using <code>rstrip</code> with <code>\n</code> avoids any unwanted side-effect except that it will remove multiple \n at the end, if present.</p>
<p>Otherwise, you need to use this less elegant function:</p>
<pre><code>def rstrip1(s, c):
    return s[:-1] if s[-1]==c else s
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/3/library/stdtypes.html#str.rstrip" rel="nofollow"><code>str.rstrip</code></a> to remove the newline character at the end of each line:</p>
<pre><code>databaselist = [line.rstrip("\n") for line in database.readlines()]
</code></pre>
<hr/>
<p>However, I recommend that you make three more changes to your code to improve efficiency:</p>
<ol>
<li><p>Remove the call to <code>readlines</code>. Iterating over a file object yields its lines one at a time.</p></li>
<li><p>Remove the <code>"r"</code> argument to <code>open</code> since the function defaults to read-mode.  This will not improve the speed of your code, but it will make it less redundant.</p></li>
<li><p>Most importantly, use a <a href="https://docs.python.org/3/reference/compound_stmts.html#the-with-statement" rel="nofollow">with-statement</a> to open the file.  This will ensure that it is closed automatically when you are done.</p></li>
</ol>
<p>In all, the new code will look like this:</p>
<pre><code>with open("database.txt") as database:
    databaselist = [line.rstrip("\n") for line in database]
</code></pre>
