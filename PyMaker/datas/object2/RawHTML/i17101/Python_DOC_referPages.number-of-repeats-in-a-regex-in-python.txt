<p>Say I have the following regex:</p>
<pre><code>"GGAGG.{5,13}?(ATG|GTG|TTG)(...)+?(TGA|TAA|TAG)"
</code></pre>
<p>Is there a way to see how many repeats are being done with for the part .{5,13}? 
Looking to know how far between GGAGG and a start codon. I could go and search for it manually later but wondering if there's a better way within the original regex.</p>

<div class="post-text" itemprop="text">
<p>You could do</p>
<pre><code>"GGAGG(.{5,13}?)(ATG|GTG|TTG)(...)+?(TGA|TAA|TAG)"
</code></pre>
<p>and then use code like</p>
<pre><code>rem = re.match(pat, s)
dist_between_ggagg_and_start_codon = len(rem.group(1))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can get the position of the entire match or groups using <a href="https://docs.python.org/3/library/re.html#re.match.start" rel="nofollow"><code>match.start</code></a> method. Using that information:</p>
<pre><code>&gt;&gt;&gt; import re
&gt;&gt;&gt; seq = 'xxxxGGAGGxxxxxxxATGxxxTGA'
&gt;&gt;&gt; pattern = "GGAGG.{5,13}?(ATG|GTG|TTG)(...)+?(TGA|TAA|TAG)"
&gt;&gt;&gt; match = re.search(pattern, seq)
&gt;&gt;&gt; match.start(1) - match.start() - 5 # 5 = len(GGAGG)
7
</code></pre>
</div>
