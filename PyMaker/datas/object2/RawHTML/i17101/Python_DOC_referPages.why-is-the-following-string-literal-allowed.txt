<pre><code>In [3]: str = 'abcd'

   File "&lt;ipython console&gt;", line 1

</code></pre>
<p>I'm aware of the condition where single quoted string can hold double quotes in between, and double quoted string can hold single quote in between.</p>
<p>Can someone please explain, why we are observing this behavior, where inside single quoted string two simultaneous single quotes are allowed, but single quote not.</p>

<div class="post-text" itemprop="text">
<p>Python implicitly concatenates nearby strings with nothing in between them. Observe:</p>
<pre><code>&gt;&gt;&gt; 'hel'  'lo'
'hello'
&gt;&gt;&gt; 'cat'        'egory'
'category'
&gt;&gt;&gt; 'ab''cd'
'abcd'
</code></pre>
<p>Strings with an odd number of one kind of quote are ambiguous and therefore not allowed.</p>
<pre><code>&gt;&gt;&gt; 'ab'cd'   # Here `cd` is a bareword and ' starts an unterminated string
  File "&lt;stdin&gt;", line 1
    'ab'cd'
         ^
SyntaxError: invalid syntax
&gt;&gt;&gt; 'ab\'cd'  # The middle single quote is escaped so this is OK
"ab'cd"
</code></pre>
<p>This can be a sort of "gotcha" when dealing with lists if you forget a comma:</p>
<pre><code>&gt;&gt;&gt; ['a', 'b', 'c' 'd', 'e']  # no comma between 'c' and 'd'
['a', 'b', 'cd', 'e']
</code></pre>
