<p>I read i text file line by line and want to find out if a line ends on a specifik word.</p>
<p>My code looks lika this, but result always get false</p>
<pre><code>#!/usr/bin/env python
# -*- coding: utf-8 -*-

import os
import wget

with open("data.txt","r") as f:
    text = f.readlines()

for line in text:
    print line
    result = line.endswith("jpg")
    print(result)
</code></pre>
<p>And the text file like this:</p>
<pre><code>file1.jpg
file2.jpg
file3.png
file4.html
</code></pre>

<div class="post-text" itemprop="text">
<p>From the <a href="https://docs.python.org/3/tutorial/inputoutput.html" rel="nofollow noreferrer">docs</a>, </p>
<blockquote>
<p>f.readline() reads a single line from the file; a newline character (\n) is left at the end of the string, and is only omitted on the last line of the file if the file doesnâ€™t end in a newline</p>
</blockquote>
<p>So really all your lines end with a hidden character after the text that's displayed. You can either include this in your check or find a way to strip it before running your check</p>
</div>
<div class="post-text" itemprop="text">
<p>The problem is that <code>f.readlines()</code> adds a new line <code>\n</code> in the end of the string.
If you try to <code>print (text)</code> you will get the following output that demonstrates this behaviour: </p>
<pre><code>['file1.jpg\n', 'file2.jpg\n', 'file3.png\n', 'file4.html']
</code></pre>
<p>You can use <code>rstrip('\n')</code> to remove newlines from the string:</p>
<pre><code>import os

with open("data.txt","r") as f:
    text = f.readlines()

for line in text:
    print (line.rstrip('\n'))
    result = line.rstrip('\n').endswith(".jpg")
    print(result)
</code></pre>
<p>which now gives the correct output:</p>
<pre><code>file1.jpg
True
file2.jpg
True
file3.png
False
file4.html
False
</code></pre>
</div>
