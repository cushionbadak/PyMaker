cmds.pointPosition(geo[0]+".cv[0]["+c+"2]", w=True)

<div class="post-text" itemprop="text">
<p>Avoid string concatenation when you can. This is a good use case for <a href="https://docs.python.org/3/library/stdtypes.html#str.format" rel="nofollow noreferrer"><code>str.format</code></a>.</p>
<pre><code>cmds.pointPosition("{}.cv[0][{}]".format(geo[0], c + 2), w=True)
</code></pre>
<p>Which works assuming <code>c</code> holds the value of an integer to begin with (otherwise, the <code>c + 2</code> step fails).</p>
</div>
<div class="post-text" itemprop="text">
<p>Python does not automatically convert an integer into its string representation like some languages. You need convert it explicitly yourself with <code>str()</code>, <code>format()</code>, or an f-string.</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>c = 5
cmds.pointPosition(geo[0]+".cv[0][" + str(c) + "2]", w=True)
</code></pre>
<p>In Python, you can't concatenate str and int.</p>
</div>
<span class="comment-copy">Try <code>str(c)</code> or <code>format()</code> or an f-string.</span>
