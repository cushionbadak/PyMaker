<div class="post-text" itemprop="text">
<p><a href="http://docs.python.org/3/library/functions.html#list" rel="nofollow noreferrer"><code>list()</code></a> creates a Python list object.</p>
<p>Given an iterable, all elements of the iterable are copied over to the resulting list:</p>
<blockquote>
<p>Return a list whose items are the same and in the same order as <em>iterable</em>â€˜s items.</p>
</blockquote>
<p>The expression <code>Tag.objects.filter(name__istartswith=q)[:10]</code> creates an iterable that results in 10 <code>Tag</code> objects, whose name starts with <code>q</code> (with a case insensitive match).</p>
<p>This simply creates a local copy of the query result, perhaps so the code can index directly into the list without re-issuing a new query each time.</p>
<p>This is specifically documented on the <a href="https://docs.djangoproject.com/en/stable/ref/models/querysets/#when-querysets-are-evaluated" rel="nofollow noreferrer">Django <code>QuerySet()</code> documentation</a>:</p>
<blockquote>
<p><strong><code>list()</code></strong>. Force evaluation of a <code>QuerySet</code> by calling <code>list()</code> on it. For example:</p>
<pre><code>entry_list = list(Entry.objects.all())
</code></pre>
<p>Be warned, though, that this could have a large memory overhead, because Django will load each element of the list into memory. In contrast, iterating over a <code>QuerySet</code> will take advantage of your database to load data and instantiate objects only as you need them.</p>
</blockquote>
