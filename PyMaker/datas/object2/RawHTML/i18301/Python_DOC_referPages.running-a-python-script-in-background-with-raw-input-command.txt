<p>I have a python script that has a raw input command, but I would like to run it in the background after the user inputs the raw_input part. The problem I have is if I try running the script in the background using &amp;, the raw input pops up as a linux command and the python script doesn't recognize it.</p>
<p>Any tips?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="http://docs.python.org/3/library/os.html#os.fork" rel="nofollow"><code>fork</code></a> to create a child process and then exit the parent.</p>
<pre><code>#!/usr/bin/env python

import os
import sys
import time

_ = raw_input('Enter the the secret code: ')
if os.fork(): # returns 0 in the child, pid of the child in the parent
    sys.exit()

time.sleep(2)
print('All good things must come to an end')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You probably want to run the script in the foreground, but then call <code>os.fork()</code> after the user has input the value.</p>
