<div class="post-text" itemprop="text">
<p>Apparently there's a long-standing <a href="https://github.com/mbr/flask-bootstrap/pull/159" rel="nofollow noreferrer">bug</a> in <a href="https://github.com/mbr/flask-bootstrap" rel="nofollow noreferrer">Flask-Bootstrap</a> that where <code>quick_form</code> omits the label for radio button fields. I found a <a href="https://stackoverflow.com/questions/27705968/flask-wtform-radiofield-label-does-not-render">fix</a> for it. First I applied the fix to my local pip installation of Flask-Bootstrap. Finding that worked, I forked the repository, applied to fix to my fork, executed pip to uninstall Flask-Bootstrap and install my fork.</p>
<p>But <code>pip freeze</code> lists <code>Flask-Bootstrap</code> and has nothing about my fork of it.</p>
<p>I need <code>requirements.txt</code> to use my fork instead of the original version. How can I do this? I know that I can <a href="https://stackoverflow.com/questions/16584552/how-to-state-in-requirements-txt-a-direct-github-source">add a line</a> in <code>requirements.txt</code> to indicate installation from a repository. But I'd rather avoid doing this manually every time I install a new Flask module.</p>
<p>Is there a way to automate the process of including the properly line in <code>requirements.txt</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>Performing these steps fixed the problem. First uninstall Flask-Bootstrap with <code>pip uninstall flask-bootstrap</code>. Then manually edit <code>requirements.txt</code> to include this line:</p>
<pre><code>-e git+https://github.com/chivalry/flask-bootstrap.git
</code></pre>
<p>Finally, run <code>pip install -r requirements.txt</code> and <code>pip freeze &gt; requirements.txt</code>. The last command will edit <code>requirements.txt</code> to include the commit ID and <code>#egg</code> name of the precise version used.</p>
<pre><code>-e git+https://github.com/chivalry/flask-bootstrap.git@c28095521664dea05f2adbf5e01fe1a36392ab6e#egg=Flask_Bootstrap
</code></pre>
</div>

<span class="comment-copy">It doesn't. If I manually edit <code>requirements.txt</code>, run <code>pip install -r requirements.txt</code> and then run <code>pip freeze</code>, it shows it at that point. Perhaps that will have to suffice.</span>
