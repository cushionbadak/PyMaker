<div class="post-text" itemprop="text">
<p>The <code>x for x in y</code> syntax is used in <a href="https://docs.python.org/3/tutorial/datastructures.html#list-comprehensions" rel="nofollow">list comprehensions</a>, <a href="https://docs.python.org/3/reference/expressions.html#generator-expressions" rel="nofollow">generator expressions</a>, set comprehensions, and  dictionary comprehensions in Python. It is not valid to use it without surrounding <code>()</code>, <code>[]</code>, or <code>{}</code>.</p>
<p>A more correct version would be </p>
<pre><code>&gt;&gt;&gt; m = [(1, 2), (3, 4), (5, 6)]
&gt;&gt;&gt; f = lambda x,y: x % y # Note no return
&gt;&gt;&gt; [f(*i) for i in m]
[1, 3, 5]
</code></pre>
<p>Or even better, the less verbose</p>
<pre><code>&gt;&gt;&gt; [x % y for x,y in ((1, 2), (3, 4), (5, 6))]
[1, 3, 5]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>actually, you can:</p>
<pre><code>&gt;&gt;&gt; [x%y for x,y in [(1,2),(3,4),(5,6)]]
[1, 3, 5]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Using <a href="https://docs.python.org/2/library/functions.html#map" rel="nofollow"><code>map</code></a>:</p>
<pre><code>&gt;&gt;&gt; m = [(1,2),(3,4),(5,6)]
&gt;&gt;&gt; f = lambda x: x[0] % x[1]  # No return statement in lambda.
&gt;&gt;&gt; map(f, m)  # list(map(f, m)) in Python 3.x
[1, 3, 5]
</code></pre>
<hr/>
<p>You can use tuple parameter unpacking in <code>lambda</code> if you use Python 2.x, but <a href="http://legacy.python.org/dev/peps/pep-3113/" rel="nofollow">it's gone in Python 3.x</a>:</p>
<pre><code>f = lambda (x, y): x % y
</code></pre>
