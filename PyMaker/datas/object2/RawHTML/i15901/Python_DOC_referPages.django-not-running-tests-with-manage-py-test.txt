<p>from the <a href="https://docs.python.org/3/library/unittest.html#unittest.TestCase" rel="nofollow">unittest documentation</a></p>
<blockquote>
<p>A testcase is created by subclassing unittest.TestCase. The three individual tests are defined with methods whose names start with the letters <code>test</code>. This naming convention informs the test runner about which methods represent tests.</p>
</blockquote>
<p>change your test function to <code>test_hard_no_more_than</code>, then the test runner whould be able to discover the testcase.</p>
</div>
<div class="post-text" itemprop="text">
<p>Test methods need to be named starting with "test" for the test runner to find them. So your method should be called "test_hard_no_more_than".</p>
