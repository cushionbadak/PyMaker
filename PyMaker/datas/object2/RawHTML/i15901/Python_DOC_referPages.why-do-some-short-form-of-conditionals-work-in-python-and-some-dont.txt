<p>Take the example:</p>
<pre><code> a = "foo" if True else "bar"
</code></pre>
<p>This executes fine and without problems.
now take:</p>
<pre><code>print("foo") if True else print("bar")
</code></pre>
<p>This throws an error.</p>
<p>I assume the first one works like a ternary operator. Is there a way to write the second statement without resorting to the full length:</p>
<pre><code>if True:
    print("foo")
else:
    print("bar")
</code></pre>
<p>Something akin to Perl's </p>

<div class="post-text" itemprop="text">
<ol>
<li><p>All paths of a conditional <em>expression</em>, must be evaluatable.</p></li>
<li><p>In Python 2.7, <a href="https://docs.python.org/2/reference/simple_stmts.html#print" rel="nofollow"><code>print</code> is a <em>statement</em></a>, not a function. So, it cannot be evaluated as an expression.</p></li>
</ol>
<p>Since <code>print</code> statement violates point 2, it cannot be used. But you can do</p>
<pre><code>print "foo" if True else "bar"
</code></pre>
<p>In Python 3.x, <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow"><code>print</code> is a <em>function</em></a>, so you can write as you mentioned in the question</p>
<pre><code>print("foo") if True else print("bar")
</code></pre>
<p>Since <code>print</code> is a function in Python 3.x, the result of the function call will be the result of the evaluation of the function call expression. You can check this like this</p>
<pre><code>print(print("foo") if True else print("bar"))
# foo
# None
</code></pre>
<p><code>print</code> function doesn't explicitly return anything. So, by default, it returns <code>None</code>. The result of evaluating <code>print("foo")</code> is <code>None</code>.</p>
</div>
