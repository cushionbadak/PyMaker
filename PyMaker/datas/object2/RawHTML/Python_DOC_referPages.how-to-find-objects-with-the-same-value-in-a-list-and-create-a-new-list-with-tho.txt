<p>Your defintion of car class is wrong. you don't create constructor <code>__init__</code> for it:</p>
<pre><code>Class car:
    def __init__(self, color, price, speed):
        self.color = color
        self.price = price
        self.speed = speed

def is_red(car):
    return car.color=="red"

# creating car1, car2, .... , car10 objects here
car_list = [car1,car2,car3,car4,car5,car6,car7,car8,car9,car10,]
</code></pre>
<p>you can use <code>filter</code> function to filter objects in a list your list:</p>
<pre><code>&gt;&gt;&gt; print(filter(is_red, car_list))
# [...list of car objects that their color is red]
</code></pre>

<div class="post-text" itemprop="text">
<p>You can use a list comprehension.</p>
<p>If <code>car_list</code> is the full list, and you want the red cars, you could do:</p>
<pre><code>red_cars = [c for c in car_list if c.color == "red"]
</code></pre>
<p>No need of custom methods to check the attribute values with this method.<br/>
However as @shotgunner said, you should fix your class definition. It's not like C++: in python  the constructor is called <code>__init__</code> and the first argument must be <code>self</code>. <a href="https://docs.python.org/3.6/tutorial/classes.html" rel="nofollow noreferrer">Here</a> more info on classes in python.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use this code:</p>
<pre><code>car_list = [car1, car2, car3, car4, car5, car6, car7, car8, car9, car10]
dd = {}

for i in range(len(car_list)):
    for j in range(len(car_list)):
        if i != j:
            if car_list[i].color == car_list[j].color:
                if car_list[i].color+'_cars' in dd:
                    dd[car_list[i].color+'_cars'] = dd[car_list[i].color+'_cars']
                else:
                    dd[car_list[i].color+'_cars'] = set()

                dd[car_list[i].color+'_cars'].add(car_list[i])
                dd[car_list[i].color+'_cars'].add(car_list[j])
print(dd)
</code></pre>
<p>Instead of creating <code>list</code>, I have created <code>sets</code> so that it will trim out the duplicate results (generated from my code) itself. And for the variable part, both the answers above have used static value (i.e. red) for conditioning as well as for the variable creation. So what I have done is, created a dictionary so that dynamic assignment of key for each color is possible (which is what I think the author wants).</p>
