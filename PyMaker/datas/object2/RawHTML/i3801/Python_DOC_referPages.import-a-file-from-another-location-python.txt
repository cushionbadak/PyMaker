<div class="post-text" itemprop="text">
<p>Python's <code>import</code> is going to look for the modules (<code>.py</code> files) in the python path. You can see what is in python path in the <a href="https://docs.python.org/3/library/sys.html#sys.path" rel="nofollow noreferrer"><code>sys.path</code></a> variable.</p>
<p>By default, <a href="https://docs.python.org/3/library/sys.html#sys.path" rel="nofollow noreferrer"><code>sys.path</code></a> will include directory where the script you ran is located. It will also include everything defined in the <a href="https://docs.python.org/3/using/cmdline.html#envvar-PYTHONPATH" rel="nofollow noreferrer"><code>PYTHONPATH</code></a> environment variable.</p>
<p>So, if you have two directories with <code>.py</code> files, you can either put both in python path, or you can make sure everything all of your source is under the same path entry.</p>
<h2>Option 1</h2>
<p>(this syntax depends on your shell, here it is for windows)</p>
<pre><code>set PYTHONPATH=%PYTHONPATH%;\path\to\second_folder

python \path\to\first_folder\a.py
</code></pre>
<p>Then, you can simply <code>import b</code>.</p>
<h2>Option 2</h2>
<p>Create an empty <code>__init__.py</code> in both directories and a <code>run.py</code> in the directory above them, so you have:</p>
<pre><code>root_dir
    run.py
    first_folder
        __init__.py
        a.py
    sedond_folder
        __init__.py
        b.py
</code></pre>
<p>Make <code>run.py</code> your entry point (run <code>python run.py</code>) and then you can import any of the modules from any other module using their full module names:</p>
<pre><code>import first_folder.a
import second_folder.b
</code></pre>

<div class="post-text" itemprop="text">
<p>Suppose you have files like this</p>
<pre><code>.
├── first_folder
│   └── a.py
└── second_folder
    └── b.py
</code></pre>
<p>You can use abs path to import <code>a.py</code> as a module in <code>b.py</code></p>
<pre><code>import importlib.util

spec = importlib.util.spec_from_file_location('a', 'path/to/first_folder/a.py')

foo = importlib.util.module_from_spec(spec)

spec.loader.exec_module(foo)

print(dir(foo))
</code></pre>
<p>There is another convenient way to load <code>b.py</code></p>
<p>$ cd path/to/second_folder</p>
<pre><code>$ ln -s path/to/first_folder ./first_folder
</code></pre>
<p>and import <code>a.py</code> as a normal python module</p>
<pre><code>import a from first_folder
</code></pre>
