<p>You can use the <a href="https://docs.python.org/3/library/subprocess.html" rel="nofollow noreferrer">subprocess module</a> to run commands from your script: <br/></p>
<pre><code>subprocess.run(cmd.split())
</code></pre>
<p>The <code>run()</code> function also takes parameters like <code>shell=True</code> if you want to keep the environment variables or <code>stdout=subprocess.PIPE</code> if you want to retreive the output of the command. <br/><br/>
Honestly, just take a look at the docs ;)<br/><br/>
EDIT: as Alan mentionned, the <code>shell=True</code> option is often considered not desirable, as you cannot be sure which shell is used by the system. More details <a href="https://stackoverflow.com/questions/3172470/actual-meaning-of-shell-true-in-subprocess">here</a>.</p>
