<div class="post-text" itemprop="text">
<p>In Python <code>3.x</code>, <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow noreferrer"><code>input</code></a> returns the entered value as is (<code>str</code>) instead of evaluating it, so you should do <code>int(input('Please input number:'))</code> to get the value as an <code>int</code>.</p>
<p>In Python <code>2.x</code>, however, <a href="https://docs.python.org/2/library/functions.html#raw_input" rel="nofollow noreferrer"><code>raw_input</code></a> would return the raw <code>str</code> value, while <a href="https://docs.python.org/2/library/functions.html#input" rel="nofollow noreferrer"><code>input</code></a> would evaluate the entered value, as you can read on the docs:</p>
<blockquote>
<p><strong>input([prompt])</strong></p>
<p>Equivalent to <code>eval(raw_input(prompt))</code>.</p>
<p>...</p>
<p>Consider using the <code>raw_input()</code> function for general input from users.</p>
</blockquote>
<p>You can mimic this behavior in Python <code>3.x</code> with <a href="https://docs.python.org/3/library/functions.html#eval" rel="nofollow noreferrer"><code>eval</code></a>: <code>eval(input('Please input number:'))</code> if you really need to, but first take a look at <a href="https://stackoverflow.com/questions/661084/security-of-pythons-eval-on-untrusted-strings">Security of Python's eval() on untrusted strings?</a></p>
</div>
