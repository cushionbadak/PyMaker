<div class="post-text" itemprop="text">
<p>I have been using <code>argparse</code> to parse command line options and worked great. Now I would like to use <code>+</code> instead of <code>-</code> for some options.</p>
<p>Ex: <code>script.py +opt -f &lt;filename&gt;</code></p>
<p>Is it possible to use such <code>+</code> options with <code>argparse</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can set the <a href="https://docs.python.org/3/library/argparse.html#prefix-chars" rel="nofollow noreferrer"><code>prefix_chars</code> option</a> to something other than <code>'-'</code>:</p>
<pre><code>parser = argparse.ArgumentParser(prefix_chars='-+')
</code></pre>
<p>at which point you can start using either <code>-</code> or <code>+</code> in the definition of arguments:</p>
<pre><code>&gt;&gt;&gt; import argparse
&gt;&gt;&gt; parser = argparse.ArgumentParser(prefix_chars='-+')
&gt;&gt;&gt; parser.add_argument('+opt', action='store_true')
_StoreTrueAction(option_strings=['+opt'], dest='opt', nargs=0, const=True, default=False, type=None, choices=None, help=None, metavar=None)
&gt;&gt;&gt; parser.add_argument('-f')
_StoreAction(option_strings=['-f'], dest='f', nargs=None, const=None, default=None, type=None, choices=None, help=None, metavar=None)
&gt;&gt;&gt; parser.parse_args(['+opt', '-f', '&lt;filename&gt;'])
Namespace(f='&lt;filename&gt;', opt=True)
</code></pre>
<p>The above defines <code>+opt</code>; you have to use the right prefix to invoke it, <code>-opt</code> won't work.</p>
</div>
<span class="comment-copy">Yes, simply set the <a href="https://docs.python.org/3/library/argparse.html#prefix-chars" rel="nofollow noreferrer"><code>prefix_chars</code></a> option.</span>
