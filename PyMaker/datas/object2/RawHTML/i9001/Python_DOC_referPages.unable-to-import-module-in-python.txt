<p>Why am I having an 'undefined bdir module' error, here is my directory,</p>
<p>a.py</p>
<p>bdir-&gt;bdir&gt;module.py</p>
<p>in a.py </p>

</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Put an <code>__init__.py</code> file (even empty) in your bdir folder.</p>
</div>
<div class="post-text" itemprop="text">
<p>You must create a <code>__init__.py</code> file, that's how Python knows which folders are packages that can be imported using the <code>import</code>. Here's the <a href="https://docs.python.org/3/tutorial/modules.html#packages" rel="nofollow noreferrer">documentation</a>:</p>
<blockquote>
<p>The <code>__init__.py</code> files are required to make Python treat the
  directories as containing packages; this is done to prevent
  directories with a common name, such as string, from unintentionally
  hiding valid modules that occur later (deeper) on the module search
  path. In the simplest case, <code>__init__.py</code> can just be an empty file, but
  it can also execute initialization code for the package or set the
  <code>__all__</code> variable.</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>Any folder without a <code>__init__.py</code> file inside of the folder is not considered a module. Furthermore, if you want to <code>import *</code> from a module, make sure that actually import the things you require into <code>__init__.py</code>, or declare a <code>__all__</code> list.</p>
<p>Also, if you want to make a <em>relative</em> import, meaning that you want to import a file from the package that a module is currently in, then you do a relative import. So, for example, if you have:</p>
<pre><code>bdir
  - bdir
     - __init__.py
     - module.py
     - a.py
</code></pre>
<p>In order to import anything from the <code>bdir.module</code>, you have to import it like so if you are in <code>a.py</code>:</p>
<pre><code>from .module import *
</code></pre>
<p>If outside the <code>bdir</code> module then:</p>
<pre><code>from bdir.module import *
</code></pre>
</div>
