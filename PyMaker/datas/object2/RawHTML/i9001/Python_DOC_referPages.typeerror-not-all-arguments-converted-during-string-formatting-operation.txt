<p>Use <a href="https://docs.python.org/3/library/string.html#format-string-syntax" rel="nofollow noreferrer">newer style string formatting</a>:</p>
<pre><code>&gt;&gt;&gt; var_range=24
&gt;&gt;&gt; print('{:&lt;{fill}}{:&lt;{fill}}{:&lt;{fill}}'.format(2, 12, 22, fill=var_range//3))
       2      12      22    
</code></pre>
<p>Since your pattern is the same for each, create the formatting for one field and repeat it 3 times.</p>
<pre><code>&gt;&gt;&gt; print(('{:&gt;{fill}}' * 3).format(2, 12, 22, fill=var_range//3))
       2      12      22
</code></pre>

<p>I not exactly sure what output you're attempting to produce, but this example might help:</p>
<pre><code>var_range = 99
print('%*s %*s %*s' % (8, var_range//3, 8, var_range//3, 8, var_range//3))
</code></pre>
<p>Output:</p>
<pre class="lang-none prettyprint-override"><code>  33       33       33
</code></pre>
<p>This format string requires 3 pairs of values for a total of 6 in the tuple that follows it. The first value of each pair is the field width (<code>8</code> in all three cases), and the second value is the numeric value to display (which is also the same in very case because you've repeated <code>var_range//3</code> three times as well).</p>
<p>I also changed the formatting for each value from <code>'%-*s</code> to <code>'%*s</code> so the value display is right-justify instead of left-justified with the width. See the <a href="https://docs.python.org/3/library/stdtypes.html#printf-style-string-formatting" rel="nofollow noreferrer">string formatting documentation</a> for details.</p>
</div>
<span class="comment-copy">The <code>%</code> <i>operator</i> expects the formatting string to contain as many <code>%</code> <i>formatting specifiers</i> as there are in the tuple following it. In your code snippet the formatting string being used is only the last <code>'%-*s'</code> yet the tuple following it contains 8 things (the <code>(8, "", (var_range//3), "", (var_range//3), "", (var_range//3), "")</code>).</span>
<span class="comment-copy">When I only use 4 things: (8,(var_range//3),(var_range//3),(var_range//3))), the same error exists</span>
<span class="comment-copy">The format string <code>'%-*s'</code> only expect one item to be in given because <i>that string</i> only contains one <code>'%'</code> character (not 8 or even 4). You might find it easier to use the <a href="https://docs.python.org/3/library/stdtypes.html#str.format" rel="nofollow noreferrer"><code>str.format()</code></a> method which doesn't using <code>'%'</code> characters at all. I think you're confusing the fact that they're used for two different things hereâ€”as both a string <i>operator</i> <b>and</b> what that operator expects in the formatting <i>string</i> it get passed as part of the entire expression.</span>
