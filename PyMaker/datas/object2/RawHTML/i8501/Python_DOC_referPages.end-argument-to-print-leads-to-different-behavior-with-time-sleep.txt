<p>I tried to write this script that just prints number after each other on the same line with a one second delay, but it does not work.</p>
<p>Ex this works:</p>
<pre><code>from time import sleep

for n in range(1, 11):
    print(n)
    sleep(1)
</code></pre>
<p>But this does not, the program just freezes:</p>
<pre><code>from time import sleep

for n in range(1, 11):
    print(n, end="")
    sleep(1)
</code></pre>
<p>Can anyone explain why this is?</p>
</div>
<div class="post-text" itemprop="text">
<p>The program does not freeze, you just need to flush the stream by passing <code>flush=True</code>:</p>
<pre><code>print(n, end="", flush=True)
</code></pre>
<p><code>sys.stdout</code>, the stream that <code>print</code> uses by default, is line-buffered (when in interactive mode), as stated in <a href="https://docs.python.org/3/library/sys.html#sys.stderr" rel="nofollow noreferrer">the documentation</a>:</p>
<blockquote>
<ul>
<li>When interactive, standard streams are line-buffered</li>
</ul>
</blockquote>
<p>This means that when the <em>default</em> string used for <code>end</code> (<code>\n</code>) is encountered, a call to <code>flush</code> will be made. Using other strings (i.e <code>''</code>) don't trigger <code>flush</code>ing so you'll have to force it by using <code>flush=True</code>.</p>
</div>
