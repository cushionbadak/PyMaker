<div class="post-text" itemprop="text">
<p>In python 3, comparison operations are stricter for the built in data types. </p>
<pre><code>&gt;&gt;&gt; min([123, 'xyz', 'zara', 'abc'])
TypeError: unorderable types: str() &lt; int()
</code></pre>
<p><code>min()</code> tries to compare <code>'xyz' &lt; 123</code> which was possible in python 2, but not in python 3.</p>
<p>If you want to get the smallest member of a list, use <code>min(list)</code>, not <code>list.min()</code>.</p>
<p>This should work, for instance:</p>
<pre><code>plt.plot([min(y), max(y)], [min(x), max(x)], color='blue', linewidth=3) 
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The issue is probably that you need to pass the lists into min and max like this:</p>
<pre><code>plt.plot([min(y), max(y)], [min(y), max(y)], color='blue', linewidth=3) 
</code></pre>
<p>Right now this will just plot a line between [1,1] and [5,5].</p>
<p>If you want to plot x and y against each other you could do this:</p>
<pre><code>plt.plot(x, y, color='blue', linewidth=3) 
</code></pre>
<p>In the case of list1 you have a list with ints and strings so the strings will just be ignored by min and max.</p>
</div>
<div class="post-text" itemprop="text">
<p>Numpy arrays have min and max methods, Python lists don't. Python does have <a href="https://docs.python.org/3/library/functions.html#min" rel="nofollow noreferrer">min</a> and <a href="https://docs.python.org/3/library/functions.html#max" rel="nofollow noreferrer">max</a> <a href="https://docs.python.org/3/glossary.html#term-function" rel="nofollow noreferrer">functions</a>, which are slower on numpy arrays in particular since they would iterate over all entries as Python objects. That tutorial erroneously refers to the functions as <a href="https://docs.python.org/3/glossary.html#term-method" rel="nofollow noreferrer">methods</a>, and they operate on iterables, not necessarily lists. </p>
