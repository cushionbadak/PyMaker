<div class="post-text" itemprop="text">
<p>The documentation (<a href="https://docs.python.org/3/library/functions.html#bytes" rel="nofollow noreferrer">here</a>) says a <code>bytes</code> is a sequence of integers. Further down the documentation of <code>zip</code> (<a href="https://docs.python.org/3/library/functions.html#zip" rel="nofollow noreferrer">here</a>) says, that zip iterates over sequences. That is why <code>b</code> becomes of type <code>int</code>.</p>
<p>You can see that by doing the following:</p>
<pre><code>&gt;&gt;&gt; b = b'b'
&gt;&gt;&gt; for i in b:
...    print(type(i), i)
&lt;class 'int'&gt; 98
</code></pre>
<p>Notice please, that by using <code>a</code> and <code>b</code> as loop variables you overwrite the original <code>a</code> and <code>b</code>.</p>
