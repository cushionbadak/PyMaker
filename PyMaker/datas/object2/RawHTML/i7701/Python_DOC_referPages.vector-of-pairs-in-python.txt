<div class="post-text" itemprop="text">
<p>I would suggest storing it in a dictionary (Hashmap)</p>
<pre><code>input = ['str1', 'str2', 'str3']
stored_as = {'str1': {'start': 1, 'end': 2}, 'str2': {'start': 0, 'end': 2}, 'str3': {'start': 1, 'end': 1}}
</code></pre>
<p>This gives you a better representation.
If you are tight on space, then you can store it as either of the following:</p>
<pre><code>stored_as = [(1,2), (0,2), (1,1)]
</code></pre>
<p>or</p>
<pre><code>stored_as = [[1,2], [0,2], [1,1]]
</code></pre>

<div class="post-text" itemprop="text">
<p>If you use <code>slice</code> objects, you can use them to select the substrings directly:</p>
<pre><code>In [924]: al = [slice(0,3), slice(2,5), slice(5,10)]
In [925]: astr = 'this is a long enough string'
In [926]: [astr[s] for s in al]
Out[926]: ['thi', 'is ', 'is a ']
</code></pre>
<p>Or with a list of tuples:</p>
<pre><code>In [927]: at = [(0,3), (2,5), (5,10)]
In [928]: [astr[s[0]:s[1]] for s in at]
Out[928]: ['thi', 'is ', 'is a ']
</code></pre>
<p>They could even be named tuples.  or a list of lists.</p>
<p>We can even hide that slice iteration with an <code>itemgetter</code>:</p>
<pre><code>In [933]: import operator
In [934]: f=operator.itemgetter(*al)
In [935]: f
Out[935]: operator.itemgetter(slice(0, 3, None), slice(2, 5, None), slice(5, 10, None))
In [936]: f(astr)
Out[936]: ('thi', 'is ', 'is a ')
</code></pre>
<p>This list of slices could also contain scalar indexes:</p>
<pre><code>In [945]: al = [0, slice(5,7), slice(10,14), -1]
In [946]: operator.itemgetter(*al)(astr)
Out[946]: ('t', 'is', 'long', 'g')
</code></pre>
