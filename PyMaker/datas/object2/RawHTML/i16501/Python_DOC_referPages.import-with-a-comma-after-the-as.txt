<p>I was looking at a repo and came across a somewhat weird line</p>
<pre><code>from flask.ext.testing import TestCase as Base, Twill
</code></pre>
<p>What does it mean to import like this? I have not seen it before and unfortunately it's rather hard to google.</p>
</div>
<div class="post-text" itemprop="text">
<p>That line tells Python to import <code>TestCase</code> and <code>Twill</code> from the package <code>flask.ext.testing</code>, but to import <code>TestCase</code> under the name of <code>Base</code>.</p>
<p>From the <a href="https://docs.python.org/3/reference/simple_stmts.html#grammar-token-import_stmt" rel="nofollow">docs</a>:</p>
<blockquote>
<p>If the module name is followed by <code>as</code>, then the name following <code>as</code> is
  bound directly to the imported module.</p>
</blockquote>
<p>Below is a demonstration with the <code>search</code> and <code>match</code> functions from the <a href="https://docs.python.org/3/library/re.html" rel="nofollow"><code>re</code> module</a>:</p>
<pre><code>&gt;&gt;&gt; from re import search as other, match
&gt;&gt;&gt; match  # The name match refers to re.match
&lt;function match at 0x02039780&gt;
&gt;&gt;&gt; other  # The name other refers to re.search
&lt;function search at 0x02048A98&gt;
&gt;&gt;&gt; search  # The name search is not defined because it was imported as other
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
NameError: name 'search' is not defined
&gt;&gt;&gt;
</code></pre>
</div>
