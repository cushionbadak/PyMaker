<div class="post-text" itemprop="text">
<p>You are using Python 2.7, but trying to follow a HOWTO written for Python <em>3</em>.</p>
<p>Use the <a href="https://docs.python.org/2/howto/urllib2.html" rel="nofollow">correct documentation instead</a>: <a href="https://docs.python.org/2/howto/urllib2.html" rel="nofollow">https://docs.python.org/2/howto/urllib2.html</a>, note how the URL contains a <code>2</code>, not a <code>3</code>, and the styling of the documentation differs materially.</p>
<p>Next, you are importing several names <em>from</em> the <code>urllib2</code> module:</p>
<pre><code>from urllib2 import Request, urlopen, URLError
</code></pre>
<p>This means now have bound the name <code>urlopen</code> (together with <code>Request</code> and <code>URLError</code>), so you don't (and <em>can't</em>) use the <code>urllib2</code> module name in your code:</p>
<pre><code>response = urlopen('http://beta.content.guardianapis.com/search?tag=football%2Fworld-cup-2014&amp;api-key=uexnxqm5bfwca4tn2m47wnhv')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Please use <code>requests</code> or if you really need <code>urllib</code> API, <code>urllib3</code> that is shipped with <code>requests</code>.</p>
<p>Everything else is has way too many gotchas, for example when it comes to ssl.</p>
