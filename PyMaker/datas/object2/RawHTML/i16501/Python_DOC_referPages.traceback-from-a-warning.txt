<div class="post-text" itemprop="text">
<p>You can turn warnings into exceptions:</p>
<pre><code>import warnings

warnings.simplefilter("error")
</code></pre>
<p>Now instead of printing a warning, an exception will be raised, giving you a traceback.</p>
<p>You can get the same effect with the <a href="https://docs.python.org/3/using/cmdline.html#cmdoption-W" rel="noreferrer"><code>-W</code> command line switch</a>:</p>
<pre><code>$ python -W error somescript.py
</code></pre>
<p>or by setting the <a href="https://docs.python.org/3/using/cmdline.html#envvar-PYTHONWARNINGS" rel="noreferrer"><code>PYTHONWARNINGS</code> environment variable</a>:</p>
<pre><code>$ export PYTHONWARNINGS=error
</code></pre>
<p>You can play with the other <a href="https://docs.python.org/2/library/warnings.html#warnings.simplefilter" rel="noreferrer"><code>warnings.simplefilter()</code> arguments</a> to be more specific about what warning should raise an exception. You could filter on <code>warnings.RuntimeWarning</code> and a line number, for example.</p>
