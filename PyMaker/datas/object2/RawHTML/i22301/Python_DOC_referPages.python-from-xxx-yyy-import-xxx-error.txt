<p>I'm working with the <code>PyFacebook</code> package in Python, and I've seen people mention numerous times that you can write an import statement as follows:</p>
<pre><code>from facebook.djangofb import facebook
</code></pre>
<p>However, it does not work. It states that <code>facebook.method_name</code> exists in the <code>facebook</code> module, rather than the <code>djangofb</code> module. I assume I'm importing the <code>facebook.method_name</code> as <code>facebook</code>, not that I'm receiving it from the <code>facebook</code> package itself.</p>
<p>I'm using Python 2.6.</p>
<p>How can I alias <code>facebook.djangofb</code> as <code>facebook</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>This is the proper way to alias a module via import:</p>
<pre><code>import facebook.djangofb as facebook
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>From the <a href="https://docs.python.org/3/reference/simple_stmts.html#import">python 3 documentation</a> (it works the same way in <a href="https://docs.python.org/2.6/reference/simple_stmts.html#import">python 2.6</a>, but I found the python 3 documentation explained it clearer):</p>
<blockquote>
<p>If the module name is followed by <code>as</code>, then the name following <code>as</code> is bound directly to the imported module.</p>
</blockquote>
<p>So your statement should look like:</p>
<pre><code>import facebook.djangofb as facebook
</code></pre>
<p>and then <code>facebook.method_name</code> will work.</p>
<p>Also see <a href="https://stackoverflow.com/questions/706595/can-you-define-aliases-for-imported-modules-in-python">Can you define aliases for imported modules in Python?</a> for additional aliasing options</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>from facebook import djangofb as facebook
</code></pre>
<p>If you're looking to import <code>djangofb</code> as <code>facebook</code>, that's how you need to do it.</p>
<p>This way, you can access <code>facebook.djangofb.method_name</code> like <code>facebook.method_name</code>.</p>
<p>That being said, it's more common to give it a non conflicting name, here it would be <code>fb</code> or <code>face</code>. Something that doesn't override the root <code>facebook</code> import.</p>
