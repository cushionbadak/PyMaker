<div class="post-text" itemprop="text">
<p>re.search returns a <a href="https://docs.python.org/3/library/re.html?highlight=match#match-objects" rel="nofollow">match object</a> as you can see the <code>groups</code> method always returns a tuple. Therefore either access the first element of the result <code>color[0]</code> or use the <code>group</code> function instead:</p>
<pre><code>color = match.group("color")
</code></pre>
<p>Also note that <code>match.groups("color")</code> does probably not do what you think it does, to quote the documentation:</p>
<blockquote>
<p>match.groups(default=None)</p>
<p>The default argument is used for groups that did not participate in
  the match; it defaults to None.</p>
</blockquote>
<p>Meaning you will set the color to "color" if no match is found for your color group. </p>
</div>
<div class="post-text" itemprop="text">
<p>for match object <code>m</code>, <code>m.groups</code> returns a <code>tuple</code>, an immutable sequence. You can index it or </p>
<pre><code>color = m.group(1) # m.group(n) for nth group
</code></pre>
<p>OR</p>
<pre><code>color = m.group('color') # for named group
</code></pre>
<p>You said <code>('red',)</code> to have contained <em>unwanted</em> chars. This probably means you need to brush up basic python as you can't recognize a <code>tuple</code>.</p>
</div>
