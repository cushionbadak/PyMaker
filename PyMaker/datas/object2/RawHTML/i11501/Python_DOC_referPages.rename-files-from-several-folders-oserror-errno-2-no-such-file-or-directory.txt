<div class="post-text" itemprop="text">
<p>You should use</p>
<pre><code>for i, j in zip(oldNames, newNames):
    os.rename(i, j)
</code></pre>
<p>instead of a nested for-loop. </p>
<p>The <a href="https://docs.python.org/3/library/functions.html#zip" rel="nofollow"><code>zip</code> function</a> iterates the two arrays in lock-step like</p>
<pre><code>os.rename(oldNames[0], newNames[0])
os.rename(oldNames[1], newNames[1])
os.rename(oldNames[2], newNames[2])
....
</code></pre>
<p>While the nested-loop will actually perform</p>
<pre><code>os.rename(oldNames[0], newNames[0])
os.rename(oldNames[0], newNames[1])
os.rename(oldNames[0], newNames[2])
os.rename(oldNames[0], newNames[3])
....
os.rename(oldNames[1], newNames[0])
os.rename(oldNames[1], newNames[1])
os.rename(oldNames[1], newNames[2])
os.rename(oldNames[1], newNames[3])
....
os.rename(oldNames[2], newNames[0])
os.rename(oldNames[2], newNames[1])
os.rename(oldNames[2], newNames[2])
os.rename(oldNames[2], newNames[3])
........
</code></pre>
</div>
<div class="post-text" itemprop="text">
<pre><code>for i in oldNames:
    for j in newNames:
</code></pre>
<p>This is the problem.  You're changing the filename of each oldName J times, even though after the first time the file doesn't exist anymore (because it's been renamed).</p>
</div>
<span class="comment-copy">Your nested loop is trying to rename each file multiple times.  Once the first file has been renamed it no longer exists for the second rename.  Loose the nested loop!</span>
