<div class="post-text" itemprop="text">
<p>I try this:</p>
<pre><code>def test(x):
    return x**2

a = map(test,[1,2,3])
</code></pre>
<p>If I get the value like this:</p>
<pre><code>for i in a:
    print(a)
</code></pre>
<p>I will get <code>1,4,9</code> and this works perfectly.</p>
<p>But if I do this: <code>a[0]</code>. The error will be raised.</p>
<p>I know it is because the result of map function is map class:</p>
<pre><code>type(map(test,[1,2,3])) == &lt;class 'map'&gt;
</code></pre>
<p>which is not subsciptable.</p>
<p>So how can I use the index to get the value of the result of map function?</p>
<p><strong>NOTE: This behavior is specific to python 3.</strong></p>
</div>
<div class="post-text" itemprop="text">
<p>Convert the <code>map</code> object into a <code>list</code> object:</p>
<pre><code>a = list(map(test,[1,2,3]))
</code></pre>
<p>Then you can use list indices to access the individual elements.</p>
</div>
<div class="post-text" itemprop="text">
<p>The built-in <a href="https://docs.python.org/3/library/functions.html#map" rel="nofollow">map()</a> function in python 3 returns an iterator. Once you consume an iterator or it is exhausted, it won't yield you result any more.</p>
<pre><code>a = map(lambda x: x**2, [1,2,3])
for i in a: print(a)
</code></pre>
<p>The result:</p>
<pre><code>1
4
9
</code></pre>
<p>Now try,</p>
<pre><code>a.__next__()
</code></pre>
<p>It will return you a <code>StopIteration</code> error, as the iterator is already exhausted.</p>
<p>To use index you can use <code>list</code> as suggested in the previous answer.</p>
</div>
