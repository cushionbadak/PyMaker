<p>How do I match my functions variable names to the default values using python and inspect?</p>
<pre><code>def foo(x:int,y:int=5) -&gt; int: return a+b

inspect.getfullargspec(foo) 
FullArgSpec(args=['x', 'y'], varargs=None, varkw=None, defaults=(4,), kwonlyargs=[], kwonlydefaults=None, annotations={'x': &lt;class 'int'&gt;, 'y': &lt;class 'int'&gt;, 'return': &lt;class 'int'&gt;})
</code></pre>
<p>I can get the args and get the defaults, but the aren't 1:1 when you try to line them up.</p>
<p>If you were to zip the two arrays into a dictionary, the x would get mapped to 4 and y isn't mapped to anything.  But the result should be y=4.</p>
<p>Thanks</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/inspect.html#inspect.signature" rel="nofollow noreferrer"><code>inspect.signature()</code></a> which returns a <a href="https://docs.python.org/3/library/inspect.html#inspect.Signature" rel="nofollow noreferrer"><code>Signature</code></a> object for your function. Then you can easily access to the arguments with their respective annotation and default values.</p>
<pre><code>In [96]: s = inspect.signature(foo)

In [97]: s.parameters
Out[97]: mappingproxy({'x': &lt;Parameter "x: int"&gt;, 'y': &lt;Parameter "y: int = 5"&gt;})

In [98]: s.parameters['x']
Out[98]: &lt;Parameter "x: int"&gt;

In [99]: s.parameters['y']
Out[99]: &lt;Parameter "y: int = 5"&gt;
</code></pre>
<p>Or separately access to annotation or defaults:</p>
<pre><code>In [106]: s.parameters['y'].annotation
Out[106]: int

In [107]: s.parameters['y'].default
Out[107]: 5
</code></pre>
