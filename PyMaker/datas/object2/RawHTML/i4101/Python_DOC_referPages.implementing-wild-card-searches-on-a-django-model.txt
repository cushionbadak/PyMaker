<div class="post-text" itemprop="text">
<p>I have a model called Host where I store all the information about all my servers. In the model I have a field <code>hostname</code>.</p>
<p>I would like for the user to be able to specify wildcard searches like: <code>*one</code> where they'd get all the hosts with hostnames ending in <code>one</code> or
<code>one*</code> where they'd get all the hosts with hostnames starting with <code>one</code>
and <code>one*two</code> and <code>*one*two</code> and <code>*one*two*</code> and <code>*one*two*three</code> and <code>one*two*three</code> and so forth.</p>
<p>I have tried iterating over the results of <code>hostname.split('*')</code> but I end up with confusing if/elif continue mess.  Does anyone know a more elegant way to handle wildcards?</p>
</div>
<div class="post-text" itemprop="text">
<p>Are you looking for <a href="https://docs.python.org/3/library/fnmatch.html" rel="nofollow noreferrer"><code>fnmatch</code></a> ?</p>
<pre><code>import fnmatch

print(fnmatch.fnmatch('foo.info', 'foo*'))
print(fnmatch.fnmatch('foo.info', '*.info'))
print(fnmatch.fnmatch('nope', '*.info'))
</code></pre>
<p>Gives</p>
<pre><code>True
True
False
</code></pre>
<p>You still have to iterate, but have to handle a single one <code>if</code>.</p>
</div>
