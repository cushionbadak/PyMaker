<span class="comment-copy">You can simply use <code>filter(None, str.split('/'))</code></span>
<span class="comment-copy"><code>print(",".join(filter(None, INPUT.split('/'))))</code></span>
<span class="comment-copy">Or <code>re.findall(r"[^/]+", s)</code></span>
<span class="comment-copy">are you perhaps looking for <a href="https://docs.python.org/3/library/re.html#re.finditer" rel="nofollow noreferrer"><code>re.finditer("(.+?)/", INPUT)</code></a>?</span>
<span class="comment-copy">This is generally a better option than regex. I would personally assign the output to a list however rather than using argument unpacking. If the list returned from <code>string.split()</code> is too short, you'll get an error.</span>
