<p>Why Python function can modify <code>list</code> or <code>dict</code> but not a <code>string</code> outside:</p>
<p>this makes sense, because function create scope, so the setit function create new variable:</p>
<pre><code>ttt = 'ttt'

def setit(it):
    ttt = it
    print(ttt)

def showit():
    print(ttt)

if __name__ == "__main__":
    setit("lsdfjlsjdf")
    showit()
</code></pre>
<p>But how to explain this, the setit function can modify the list outside:</p>
<pre><code>aaa = []

def setit(it):
    ttt = it
    aaa.append(it)

def showit():
    print(aaa)


if __name__ == '__main__':
    setit(123)
    showit()
    setit(234)
    showit()
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Because <strong>strings are immutable</strong>. You cannot edit strings, you can just create new strings.</p>
<p>Source: <a href="https://docs.python.org/3/faq/design.html#why-are-python-strings-immutable" rel="nofollow noreferrer">Python Docs</a></p>
