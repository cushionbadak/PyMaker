<p>I was under the impression that numbers in "" marks become strings.  Why do I get the following result?</p>
<pre><code>"2001".isdigit()
True
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>It says <code>.isdigit()</code> not <code>.isint()</code> which makes sense since <code>int</code> and <code>string</code> are Python types, but nobody says a string can't be digits. </p>
<p>In fact, strings can contain letters, alphanumeric, digits, and a bunch of other things.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>.isdigit()</code> is a method for the <code>str</code> object if this string only contains digits.</p>
<p>From <a href="https://docs.python.org/3/library/stdtypes.html#str.isdecimal" rel="nofollow noreferrer">the Python documentation</a>, <code>.isdigit()</code>:</p>
<blockquote>
<p>Return true if all characters in the string are digits and there is at least one character, false otherwise. Digits include decimal characters and digits that need special handling, such as the compatibility superscript digits. This covers digits which cannot be used to form numbers in base 10, like the Kharosthi numbers. Formally, a digit is a character that has the property value Numeric_Type=Digit or Numeric_Type=Decimal.</p>
</blockquote>
</div>
<span class="comment-copy"><code>help("2001".isdigit)</code> -&gt; <code>Return True if the string is a digit string</code>.</span>
