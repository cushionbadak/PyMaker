<p>Look what happens when you print the actual representation of your output string:</p>
<pre><code>&gt;&gt;&gt; repr(tags[0].text)
'\n\nMots-clefs :\n    A Gray State 4,\n    A Gray State 3,\n    A Gray State 2,                                     \n  \n'
</code></pre>
<p>You have a few issues here, both newline characters (which are <code>\n</code>, and not <code>/n</code>), as well as undesired continuous whitespace.  However, you can use <a href="https://docs.python.org/3/library/re.html#re.sub" rel="nofollow noreferrer"><code>re.sub</code></a> to replace both of these with single spaces, to achieve your desired result:</p>
<pre><code>from bs4 import BeautifulSoup
import re

html = """
&lt;div class="entry-footer"&gt;
  &lt;p&gt;
    &lt;strong&gt;Mots-clefs :&lt;/strong&gt;
    A Gray State 4,
    A Gray State 3,
    A Gray State 2,                                     
  &lt;/p&gt;
&lt;/div&gt;
"""

soup = BeautifulSoup(html)
tags = soup.find_all("div",{"class":"entry-footer"})[0].text
print(re.sub(r'(\s+|\n)', ' ', tags))
</code></pre>
<p>Output:</p>
<pre><code>Mots-clefs : A Gray State 4, A Gray State 3, A Gray State 2, 
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Or without regex:</p>
<pre><code>from bs4 import BeautifulSoup

content = """
 &lt;div class="entry-footer"&gt;
    &lt;p&gt;
        &lt;strong&gt;Mots-clefs :&lt;/strong&gt;
        A Gray State 4,
        A Gray State 3,
        A Gray State 2,                                     
    &lt;/p&gt;
&lt;/div&gt;
"""
soup = BeautifulSoup(content,"lxml")
for items in soup.select(".entry-footer p strong"):
    title = items.text
    data = ' '.join([' '.join(item.split()) for item in items.next_siblings])
    print(title,data)
</code></pre>
<p>Output:</p>
<pre><code>Mots-clefs : A Gray State 4, A Gray State 3, A Gray State 2,
</code></pre>

<h1>Newline</h1>
<p>Frequently called <strong>line ending</strong>, <strong>end of line</strong> (<strong>EOL</strong>), <strong>line feed</strong>, or <strong>line break</strong>, is a <em><a href="https://en.wikipedia.org/wiki/Control_character" rel="nofollow noreferrer" title="Control character">control character</a></em> in a <em><a href="https://en.wikipedia.org/wiki/Character_encoding" rel="nofollow noreferrer" title="Character encoding">character encoding</a></em> specification, like e.g. <a href="https://en.wikipedia.org/wiki/ASCII" rel="nofollow noreferrer" title="ASCII">ASCII</a>. It is used to signify the end of a <a href="https://en.wikipedia.org/wiki/Line_(text_file)" rel="nofollow noreferrer" title="Line (text file)">line of text</a> and the start of a new one. Text editors set this special character when pressing the <a href="https://en.wikipedia.org/wiki/Enter_key" rel="nofollow noreferrer" title="Enter key">Enter key</a>.</p>

<h1>Unicode</h1>
