<p>When i ping servers with os.system in python i get multiple response codes.
Command used -  <code>os.system("ping -q -c 30 -s SERVERANME")</code></p>
<ul>
<li>0  - Online</li>
<li>256 - Offline</li>
<li>512 - what does 512 mean ?</li>
</ul>
</div>
<div class="post-text" itemprop="text">
<p>Per <a href="https://docs.python.org/3/library/os.html#os.system" rel="nofollow noreferrer">the docs</a>:</p>
<blockquote>
<p>On Unix, the return value is the exit status of the process encoded in the format specified for <code>wait()</code>. Note that POSIX does not specify the meaning of the return value of the C <code>system()</code> function, so the return value of the Python function is system-dependent.</p>
</blockquote>
<p>And <a href="https://docs.python.org/3/library/os.html#os.wait" rel="nofollow noreferrer">the <code>wait</code> docs say</a>:</p>
<blockquote>
<p>Wait for completion of a child process, and return a tuple containing its pid and exit status indication: a 16-bit number, whose low byte is the signal number that killed the process, and whose high byte is the exit status (if the signal number is zero); the high bit of the low byte is set if a core file was produced.</p>
</blockquote>
<p>So 0, 256 and 512 correspond to <code>ping</code> exiting normally (not killed by signal) with exit statuses of <code>0 == 0 &lt;&lt; 8</code> (<code>0</code> traditionally means "success"), <code>256 == 1 &lt;&lt; 8</code> (<code>1</code> typically means "normal" failure) and <code>512 == 2 &lt;&lt; 8</code> (not consistent, but <code>2</code> is frequently used to indicate an argument parsing failure). In this case, you passed <code>-s</code> without providing the mandatory value (<code>packetsize</code>) that switch requires, so an exit status of <code>2</code> makes sense.</p>
</div>
<span class="comment-copy">The <code>-s</code> option of <code>ping</code> is used to set a packet size, but you have not specified one. You should either eliminate the <code>-s</code> or add a <code>packetsize</code>.</span>
