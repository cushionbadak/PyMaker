<p>Say I read a FITS file</p>
<pre><code>from astropy.io import fits
from astropy.utils.data import get_pkg_data_filename

image_file = get_pkg_data_filename('tutorials/FITS-images/HorseHead.fits')
cube = fits.open(image_file)[0]
</code></pre>
<p>And I want to make a copy of its header so I can modify the copy without changing the original header</p>
<pre><code>header_copy = cube.header
header_copy.remove('OBJCTY')
</code></pre>
<p>However, this also modifies <code>cube.header</code>. </p>
<p>How would I go about to make an actual copy of the header, rather than creating a new pointer to the header?</p>
<p><code>header_copy = ?</code></p>
</div>
<div class="post-text" itemprop="text">
<p>The <code>Header</code> class from <code>astropy.io.fits</code> has a <a href="https://astropy.readthedocs.io/en/stable/io/fits/api/headers.html#astropy.io.fits.Header.copy" rel="nofollow noreferrer">copy function</a>.</p>
<pre><code>header_copy = cube.header.copy()
header_copy.remove('OBJCTY')
</code></pre>
<p>Then <code>header_copy</code> is an actual copy of the object, rather than a pointer towards <code>cube.header</code></p>
<pre><code>if cube.header['OBJECTY'] != header_copy['OBJECTY']:
    print('All ok')

&gt;&gt;&gt; All ok
</code></pre>

<span class="comment-copy">Perhaps the <a href="https://docs.python.org/3/library/copy.html" rel="nofollow noreferrer">copy</a> module? <code>import copy</code> then <code>header_copy = copy.copy(cube.header)</code></span>
<span class="comment-copy">Indeed, <code>copy</code> seems to do what I expect. I was rather stumped by the fact that the behaviour of cube.header and cube.data are not the same when I tried to copy them, but the answer turned out to be quite simple.</span>
