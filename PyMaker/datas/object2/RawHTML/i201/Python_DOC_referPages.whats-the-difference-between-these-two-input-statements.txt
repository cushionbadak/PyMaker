<p>I've tried using these two input statements in python. Both the statements returns same output. What's the difference between using <code>split()</code> and <code>split(" ")</code> ?</p>
<pre><code>a=[int(i) for i in input().split(" ")]
print(a)
</code></pre>
<p>and</p>
<pre><code>a=[int(i) for i in input().split()]
print(a)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The default action of method <code>split</code> on a string is to split on <em>any</em> grouping of white space:</p>
<pre><code>&gt;&gt;&gt; 'foo   bar'.split()
['foo', 'bar']
&gt;&gt;&gt; 'foo \n \t   bar'.split()
['foo', 'bar']
</code></pre>
<p>If you pass a literal space as the argument, however, the split is done differently, with <em>only</em> a literal space as the splitter, and with empty strings resulting from adjacent literal spaces:</p>
<pre><code>&gt;&gt;&gt; 'foo \n \t   bar'.split(' ')
['foo', '\n', '\t', '', '', 'bar']
</code></pre>
<p>If the input has only single, ordinary spaces, there will be no observable difference.</p>
</div>
