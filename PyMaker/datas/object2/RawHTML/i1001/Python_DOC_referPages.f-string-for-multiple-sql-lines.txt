<p>The issue stems from essentially invalid multi lines and not really any fault of <a href="https://docs.python.org/3/reference/lexical_analysis.html#f-strings" rel="nofollow noreferrer">f-string.</a>. Note that you only need to use your variables inside the <code>{}</code> parenthesis and not pass them separately with <code>f-string</code> syntax. Use triple quotes or backslashes at end of strings to use multi line strings. <a href="https://docs.python.org/3/tutorial/introduction.html#strings" rel="nofollow noreferrer">Link to docs</a></p>
<pre><code>test = "this is a 
       bad multiline " #Raises SyntaxError: EOL while scanning string literal

test = ''' this is a 
         valid multiline'''

a = "answer"

print(f'''test
        is complete. check {a} ''')
</code></pre>
