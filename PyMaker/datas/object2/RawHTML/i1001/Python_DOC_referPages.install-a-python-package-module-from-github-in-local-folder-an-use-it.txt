<h2>Issue</h2>
<p>I would like to install with <code>pip3</code> a python module from github into a local folder named <code>local_lib/</code> and then use it in a script, <strong>without any virtualenv</strong>.</p>
<h2>Context</h2>
<p>Here is my folder structure :</p>
<pre><code>.
+-- local_lib/ // Folder where the package must be installed
+-- my_script.py
</code></pre>
<p>Here is the command line i use to install the <code>path.py</code> package from github into the <code>local_lib/</code> folder :</p>
<pre><code>pip3 install --upgrade --target local_lib git+https://github.com/jaraco/path.py.git 
</code></pre>
<p>Here is the content of the <code>local_lib/</code> folder after the command line :</p>
<pre><code>.
+-- local_lib/ // Folder where the package must be installed
|   +-- __pycache__
|   +-- importlib_metadata-0.8.dist-info
|   +-- path.py-11.5.1.dev20+g3684c4d.dist-info
|   +-- zipp-0.3.3.dist-info
|   +-- importlib_metadata
+-- my_script.py
</code></pre>
<p>Here is the content of <code>my_script.py</code> :</p>
<pre><code>#!/usr/bin/env python3
# -*- coding: utf-8 -*-

from local_lib.path import Path

if __name__ == '__main__'::
    print(Path('.') / 'generated_folder')
</code></pre>
<p>But when i execute the script with <code>python3 my_script.py</code>, i get the following error of import :</p>
<blockquote>
<p>Traceback (most recent call last): <br/>
  File "my_program.py", line 4, in module <br/>
  from local_lib.path import Path <br/>
  ModuleNotFoundError: No module named 'local_lib.path' <br/></p>
</blockquote>
<p>Should i change the way i import the package into my_scipt.py <strong>or</strong> should i change the command line to install the package ?</p>
</div>
<div class="post-text" itemprop="text">
<p>You have to tell Python that it has to look in <code>local_lib</code> for modules. E.g. by adding it to <a href="https://docs.python.org/3/library/sys.html?highlight=sys%20path#sys.path" rel="nofollow noreferrer"><code>sys.path</code></a> in your script (<em>before</em> importing from it) or by adding it to your <a href="https://docs.python.org/3/using/cmdline.html#envvar-PYTHONPATH" rel="nofollow noreferrer">PYTHONPATH</a> environment variable.</p>
</div>
<div class="post-text" itemprop="text">
<p>For this, you can just download the <a href="https://github.com/jaraco/path.py/blob/master/path.py" rel="nofollow noreferrer">path.py</a> file into your <code>local_lib</code> folder and your path.py use case should work. You don't need install it with <code>pip3</code>.</p>
<p>Should you want <code>path.py</code> to be available for scripts in any folder on the machine, using the same syntax, use the <a href="https://stackoverflow.com/a/54052435/6018688">solution from Roland Smith</a> or install it the generic way with <code>pip install path.py</code>.</p>
</div>
