<p>How do I have to configure so that I don't have to type <code>python script.py</code> but simply <code>script.py</code> in CMD on Windows?</p>
<p>I added my python directory to %PATH% that contains python.exe but still scripts are not run correctly.</p>
<p>I tried it with django-admin.py Running <code>django-admin.py startproject mysite</code> gives me <code>
Type 'django-admin.py help &lt;subcommand&gt;' for help on a specific subcommand.</code> Using python in front of it processes the command correctly. </p>
<p>What's the problem here?</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>C:\&gt; assoc .py=Python
C:\&gt; ftype Python="C:\python27\python.exe %1 %*"
</code></pre>
<p>Or whatever the relevant path is - you can also set command line args using ftype.</p>
<hr/>
<p>In order to make a command recognized without having to give the suffix (<code>.py</code>), similar to how it works for <code>.exe</code> files, add <code>.py</code> to the semi-colon separated list of the (global) <code>PATHEXT</code> variable.</p>
<p><strong>ETA 2017-07-27</strong></p>
<p>Seems like this is still getting eyeballs, wanted to elevate a useful comment for Win10 users (from @shadowrunner):</p>
<p><em>For me to get it work under Win10 the actual command was (note the placement of the quotes):</em> </p>
<pre><code>C:\&gt; ftype Python="c:\Anaconda2\python.exe" "%1" %*
</code></pre>
<p><strong>ETA 2019-02-01</strong></p>
<p>Talk about evergreen!</p>
<p>First of all, if you're newly installing Python, I highly recommend reviewing the answer by @NunoAndré .</p>
<p>Secondly, to clarify something from a recent comment, please note: you must do both parts (<code>assoc</code> and <code>ftype</code>), or use a pre-existing association label in the <code>ftype</code> command.</p>
<p>By default, at least for Python 3.7 under Windows 8.1, the association for <code>.py</code> is <code>Python.File</code>, so performing the <code>ftype</code> command I wrote above will not work correctly unless the association is first changed. Or you can just use <code>ftype</code> and give the default association instead. Up to you.</p>

<div class="post-text" itemprop="text">
<p>From Python 3.3 a <a href="https://docs.python.org/3.6/using/windows.html#python-launcher-for-windows" rel="nofollow noreferrer">launcher for Windows</a> is included: <code>py</code> (and <code>pyw</code> for GUI or non-UI applications)</p>
<blockquote>
<p>which aids in locating and executing of different Python versions. It allows scripts (or the command-line) to indicate a preference for a specific Python version, and will locate and execute that version.</p>
<p>Unlike the PATH variable, the launcher will correctly select the most appropriate version of Python. It will prefer per-user installations over system-wide ones, and orders by language version rather than using the most recently installed version.</p>
</blockquote>
<p>Python installer links Python's file extensions to <code>open</code> verb by default, so you can run a python file simply by typing its name (and args if needed).</p>
<ul>
<li><code>py</code>: <code>.py</code>, <code>.pyc</code> and <code>.pyo</code> (<a href="https://docs.python.org/3/distutils/apiref.html?highlight=pyc#distutils.util.byte_compile" rel="nofollow noreferrer">byte-compiled</a>), and <code>.pyz</code> (<a href="https://docs.python.org/3/library/zipapp.html" rel="nofollow noreferrer">zip-compressed</a>).</li>
<li><code>pyw</code>: <code>.pyw</code> and <code>.pyzw</code> (zip-compressed).</li>
</ul>
<p>Caveat: <a href="https://stackoverflow.com/questions/9705982/pythonw-exe-or-python-exe/30313091#30313091">be aware of the differences between python.exe and pythonw.exe</a></p>
<hr/>
<p>Among other advantages, Windows launcher reads 'nix shebangs, so you can specify Python version or <code>python.exe</code>'s <a href="https://docs.python.org/3/using/cmdline.html" rel="nofollow noreferrer">command line arguments</a></p>
<p>You can check this running this script (supposing py3 as default):</p>
<pre><code>#! /usr/bin/python2.7 -i
import sys
print(sys.version)
</code></pre>
<ul>
<li><code>myscript.py</code>: runs with <code>py</code>, launches <code>python2.7</code> and enters in interactive mode after finished (<code>-i</code>, great option for testing and debugging).</li>
<li><code>myscript.py -3</code>: runs with <code>py</code>, launches <code>python3</code> and keeps interactive mode.</li>
<li><code>python myscript.py</code>: runs with default python runtime, no interactive mode.</li>
</ul>
<hr/>
<p>You can change this default association with <code>ftype</code>, but I would strongly recommend:</p>
<ul>
<li>Nirsoft's <a href="http://www.nirsoft.net/utils/file_types_manager.html" rel="nofollow noreferrer">File Types Manager</a> utility.</li>
<li>Taking a look at msdn docs about file extensions and associations: <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/ff521657(v=vs.85).aspx" rel="nofollow noreferrer">Implementing a Custom File Format</a>.</li>
</ul>
<p>You can easily associate other verbs (like <code>edit</code>, <code>test</code>, <code>debug</code>...) to these files.</p>
<hr/>
<p>In addition, you can omit Python's extensions to run a file in a terminal by adding them to <code>PATHEXT</code> environment variable ordered by preference. (You must re-open the terminal for the change to take effect).</p>
<p><code>setx PATHEXT %PATHEXT%;.PYC;.PYZ;.PY</code></p>
