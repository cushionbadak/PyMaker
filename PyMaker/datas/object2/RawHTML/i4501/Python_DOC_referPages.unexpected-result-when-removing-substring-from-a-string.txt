<p>I have a string <code>s</code>, and I want to remove <code>'.mainlog'</code> from it. I tried:</p>
<pre><code>&gt;&gt;&gt; s = 'ntm_MonMar26_16_59_41_2018.mainlog'
&gt;&gt;&gt; s.strip('.mainlog')
'tm_MonMar26_16_59_41_2018'
</code></pre>
<p>Why did the <code>n</code> get removed from <code>'ntm...'</code>?</p>
<p>Similarly, I had another issue:</p>
<pre><code>&gt;&gt;&gt; s = 'MonMar26_16_59_41_2018_rerun.mainlog'
&gt;&gt;&gt; s.strip('.mainlog')
'MonMar26_16_59_41_2018_reru'
</code></pre>
<p>Why does python insist on removing <code>n</code>'s from my strings? How I can properly remove <code>.mainlog</code> from my strings?</p>

<div class="post-text" itemprop="text">
<p>If you read the docs for <a href="https://docs.python.org/3/library/stdtypes.html#str.strip" rel="nofollow noreferrer"><code>str.strip</code></a> you will see that:</p>
<blockquote>
<p>The chars argument is a string specifying the <strong>set of characters</strong> to be removed.</p>
</blockquote>
<p>So all the characters in <code>'.mainlog'</code> (<code>['.', 'm', 'a', 'i', 'n', 'l', 'o', 'g']</code>) are stripped just from the beginning and end.</p>
<hr/>
<p>What you want is <a href="https://docs.python.org/3/library/stdtypes.html#str.replace" rel="nofollow noreferrer"><code>str.replace</code></a> to replace all occurrences of <code>'.mainlog'</code> with nothing:</p>
<pre><code>s.replace('.mainlog', '')
#'ntm_MonMar26_16_59_41_2018'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>From Python documentation:</p>
<p><a href="https://docs.python.org/2/library/string.html#string.strip" rel="nofollow noreferrer">https://docs.python.org/2/library/string.html#string.strip</a></p>
<p>Currently, it tries to strip all the characters which you mentioned ('.', 'm', 'a', 'i'...)</p>
<p>You can use string.replace instead.</p>
<pre><code>s.replace('.mainlog', '')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You are using the wrong function. <code>strip</code> removes characters from the beginning and end of the string. By default spaces, but you can give a list of characters to remove.</p>
<p>You should use instead:</p>
<pre><code>s.replace('.mainlog', '')
</code></pre>
<p>Or:</p>
<pre><code>import os.path
os.path.splitext(s)[0]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The argument to the strip function, in this case, <code>.mainlog</code> is not a string, it's a set of individual characters.</p>
<p>That's removing all leading and trailing characters that are in that list.</p>
<p>We'd get the same result if we passed in the argument  <code>aiglmno.</code>.</p>
