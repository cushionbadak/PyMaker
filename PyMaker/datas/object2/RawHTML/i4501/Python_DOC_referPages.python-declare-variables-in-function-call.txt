<div class="post-text" itemprop="text">
<p>There is not a local variable called <code>axis</code> being initialized, you are just explicitly stating which parameter of the function you are passing.</p>
<p>You might use this if there are multiple optional parameters in a function:</p>
<p>Example:</p>
<pre><code>In [9]: def foo(x=2, y=2):
   ...:     print(x, y)
   ...:

In [10]: foo(y=3)
2 3
</code></pre>
<p>If I want to change <code>y</code>, but leave <code>x</code> as default, I must specify when I call the function.</p>
<p>Even when you aren't setting a particular optional parameter however, there is nothing wrong with passing an argument by its keyword.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>pixmap</code> is a <code>numpy</code> array with dimension &gt;= 2. You can test this by printing <code>pixmap.ndim</code>.</p>
<p>The line you refer to is calculating the <code>mean</code> of an array <em>along an axis</em>. In other words, <code>axis</code> is an <em>argument</em> of the <code>mean</code> method. See <code>numpy.mean</code> <a href="https://docs.scipy.org/doc/numpy/reference/generated/numpy.mean.html" rel="nofollow noreferrer">documentation</a> for more details.</p>
<p>You are right in that the <code>axis</code> parameter may be omitted, i.e. <code>arr.mean(2)</code> is equivalent to <code>arr.mean(axis=2)</code>. Most likely it was explicitly stated for clarity.</p>
</div>
<div class="post-text" itemprop="text">
<p>This looks a bit like the variable assignment syntax because of the <code>=</code>, but it is actually a very different feature:</p>
<p>When you call functions, you can specify which variables to set to what using the "keyword argument" syntax. This allows you to set specific parameters by name:</p>
<pre><code>some_func(arg1, arg2, arg5=None)
</code></pre>
<p>This is very useful with default arguments:</p>
<pre><code>def x(a, b, c=None, d=None):
    pass
</code></pre>
<p>If you wanted to set <code>d</code> with the normal syntax, you would have to specify an argument for <code>c</code> too.</p>
<pre><code>x(1, 2, None, "hi")
</code></pre>
<p>Using the keyword argument syntax, you can omit this:</p>
<pre><code>x(1, 2, d="hi")
</code></pre>
<p>Keyword arguments are often, like in this case, also used to make the purpose of a parameter clear to the reader. If you see:</p>
<pre><code>pixmap.mean(2)
</code></pre>
<p>It is not immediately clear what the <code>2</code> meas. However, if you see</p>
<pre><code>pixmap.mean(axis=2)
</code></pre>
<p>you immediately know what <code>2</code> refers to and why it's there</p>
<p>Also see <a href="https://docs.python.org/3/tutorial/controlflow.html#keyword-arguments" rel="nofollow noreferrer">this section of the python docs</a></p>
</div>
