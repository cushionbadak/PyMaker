<p>I am confused on how does the != operator works in python. I am new to python programming.</p>
<p>I know simple <code>!=</code> simply checks whether the LHS expression and RHS expression are not equal.</p>
<p>for eg: </p>
<pre><code>True != False
</code></pre>
<p>returns True.</p>
<p>My Question is how it works in series of <code>!=</code> operators.</p>
<p>eg: when I type </p>
<pre><code>-5 != False != True != True
</code></pre>
<p>in my python interactive session it returns <code>False</code>, but if I solve it step by step I get the answer <code>True</code>.</p>
<p>Solving it step by step:</p>
<p><code>-5 != False</code> returns True </p>
<p><code>True != True</code> returns False</p>
<p><code>False != True</code> returns True</p>
<p>So it should return <code>True</code> but it returns <code>False</code>. I don't know why.</p>
<p>A little help will be highly appreciated.
Thanks in advance.</p>
</div>
<div class="post-text" itemprop="text">
<p>In Python, this comparation is equivalent to:</p>
<pre><code>-5 != False and False != True and True != True
</code></pre>
<p>I.e., </p>
<pre><code>result = (-5 != False) and (False != True) and (True != True)
result =    (True)     and     (True)      and     (False)
result = False
</code></pre>
<p>See more in:<a href="https://docs.python.org/3/reference/expressions.html#comparisons" rel="nofollow noreferrer">https://docs.python.org/3/reference/expressions.html#comparisons</a>.</p>
</div>
<span class="comment-copy">There may be more exact duplicates, but the quote from the documentation in the accepted answer for this duplicate does explain what is happening.</span>
<span class="comment-copy">Your last operation that is carried out is 'True != True', which is false. -5 is being compared to False. False it being compared to True. But the second to last True is being compared to True. True != True is False.</span>
<span class="comment-copy">Read operation chaining here - <a href="https://docs.python.org/3/reference/expressions.html#comparisons" rel="nofollow noreferrer">docs.python.org/3/reference/expressions.html#comparisons</a></span>
<span class="comment-copy">Using parenthesis helps you control the order of evaluation and brings more clarity to the developer's intent.  ((-5 != False) != True) != True</span>
