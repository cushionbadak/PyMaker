<div class="post-text" itemprop="text">
<p>Python 3 input function returns string.</p>
<p>Try like this</p>
<pre><code>start = input("--&gt;: ")
if start == "3":
    print("successful")
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Some things you could do on your own to get to the root of the problem:</p>
<p>Ways to get to know the type of the object:</p>
<pre><code>print(type(start)) # prints &lt;class 'str'&gt;
print(repr(start)) # prints '3'
</code></pre>
<p>Unlike Python 2.x, the function <code>input()</code> returns a string object (and does not blindly <a href="https://docs.python.org/3/library/functions.html#eval" rel="nofollow noreferrer">evaluate</a> the expression provided by the user):</p>
<blockquote>
<p><strong>input([</strong><em>prompt</em><strong>]):</strong></p>
<p>If the prompt argument is present, it is written to standard output without a trailing newline. <strong>The function then reads a line from input, converts it to a string (stripping a trailing newline), and returns that.</strong> [...]</p>
</blockquote>
<p>This should give an idea how to fix it (compare numbers to numbers).</p>
