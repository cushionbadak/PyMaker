<p>Messages exchanged between a client and a server using the xmlrpclib using Python 2.6.x creates a type of 'instance' on server side instead of type 'datetime'. 
On the client side I create a new </p>
<pre><code>updateTime = datetime(year, month, day, hour, minute, second)
print type(updateTime)
print updateTime
</code></pre>
<p>results in</p>
<pre><code>&lt;type 'datetime.datetime'&gt;
2015-10-07 10:21:52
</code></pre>
<p>when being send, the dictionary looks like this on the client side:</p>
<pre><code>'updateTime': datetime.datetime(2015, 10, 7, 10, 21, 52)
</code></pre>
<p>but the incoming dictionary on the server side looks like this:</p>
<pre><code>'updateTime': &lt;DateTime '20151007T10:21:52' at 7f4dbf4ceb90&gt;
</code></pre>
<p>printing the type and its string representation looks like this:</p>
<pre><code>&lt;type 'instance'&gt;
20151007T10:21:52
</code></pre>
<p>We are using xmlrpclib.ServerProxy but changing <strong>use_datetime</strong> either to <em>True</em> or <em>False</em> does not make any difference at all.</p>
<pre><code>xmlrpclib.ServerProxy('https://'+rpc_server_addr, allow_none=True, use_datetime=True)
</code></pre>
<p>Why is this happening? I expected a tpye 'datetime.datetime' on the receiving side as well.</p>
</div>
<div class="post-text" itemprop="text">
<p>You have to convert the xmlrpc.datetime format to a python datetime.datetime object. </p>
<p>The simplest way I have found to transform the object is: </p>
<pre><code>import datetime

my_datetime = datetime.datetime.strptime(str(xmlrpc.datetime), '%Y%m%dT%H:%M:%S')
</code></pre>
</div>

<p>The <code>use_builtin_types=True</code> works for me. All datetime values have type <code>&lt;class 'datetime.datetime'&gt;</code>.</p>
<p>Without this parameter all datetime were <code>&lt;class 'xmlrpc.client.DateTime'&gt;</code></p>
<pre><code>rpc = xmlrpc.client.ServerProxy('https://..../', use_builtin_types=True)
</code></pre>
<p>The <a href="https://docs.python.org/3/library/xmlrpc.client.html#xmlrpc.client.ServerProxy" rel="nofollow noreferrer">Python3 XML-RPC client documentation</a> says: 
The obsolete <code>use_datetime</code> flag is similar to <code>use_builtin_types</code> but it applies only to date/time values.</p>
