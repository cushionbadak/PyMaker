<div class="post-text" itemprop="text">
<p>If you want to do something fast @AdamSmith's approach will be great.</p>
<p>On the other hand, if your program will grow a lot on options or you want to create more elegant parameters such as:</p>
<p><strong>your_script --src-dir=<em>src_dir</em> --dest-dir=<em>dest_dir</em></strong></p>
<p>you can use <a href="https://docs.python.org/3/library/argparse.html" rel="nofollow">argparse</a> which will do the job great.</p>
<p>Even more, if you want to print something like this:</p>
<pre><code>$ python copier.py -h
usage: copier.py [-h] [--src-dir] [--dest-dir] 

Copies a directory from source to destination

positional arguments:
--src-dir The source directory
--dest-dir The destination directory

optional arguments:
 -h, --help  show this help message and exit
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can access the whole argument values list with</p>
<pre><code>import sys

sys.argv
</code></pre>
<p>the first element is always the name of the file, so you're looking at <code>sys.argv[1:]</code></p>
<pre><code>src, dst = sys.argv[1:]
</code></pre>
<p>But is there some reason that you're using the <code>distutils</code> module for this? It's more often called from <code>shutil</code></p>
<pre><code>import shutil
import sys

src, dst = sys.argv[1:]
try:
    shutil.copytree(src, dst)
    # shutil.copytree(src, dst2)  # how are you generating dst2?
except Exception:
    # something went wrong. Unless there's a reasonable way to recover we should just
    raise
</code></pre>
