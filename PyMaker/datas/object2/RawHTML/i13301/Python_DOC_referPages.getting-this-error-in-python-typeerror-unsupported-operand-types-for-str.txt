<div class="post-text" itemprop="text">
<p>I believe your problem is here. </p>
<pre><code>sumatoriafinal = input("&gt;")
varianza = sumatoriafinal / datos
</code></pre>
<p>The <code>input()</code> function takes the user's input and <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow">converts it to a string</a>. This is why you're receiving your <code>TypeError: unsupported operand type(s) for /: 'str' and 'int'</code> error -- you're attempting to divide a string (<code>sumatoriafinal</code>) and an integer (<code>datos</code>).</p>
<p>Try doing this instead:</p>
<pre><code>sumatoriafinal = input("&gt;")
varianza = int(sumatoriafinal) / datos
</code></pre>
</div>
<span class="comment-copy">Just cast anthing you want as an int like this: int("4"). Specifically, anything input by the user should be cast as an int.</span>
<span class="comment-copy">On a side note, why are you importing inside of a function definition? That would lead to those imports being done every time the function is called, which is wasteful and unnecessary. Just import everything you need at the top of your script.</span>
<span class="comment-copy">Note: If this is Python 3, then yes, that's what <code>input</code> does (it needn't be AFAICT; the print is Py3 style, but a <code>__future__</code> import makes that available in Py2, and might be omitted). In Py2, it does stupid things (equivalent to wrapping Py3's <code>input</code> in an <code>eval</code>), and it's possible the OP saw an example that used Py2's <code>input</code> and borrowed it incorrectly for Py3.</span>
