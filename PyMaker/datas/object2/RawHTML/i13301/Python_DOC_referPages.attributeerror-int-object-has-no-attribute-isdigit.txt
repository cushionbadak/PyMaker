<div class="post-text" itemprop="text">
<p>As documented <a href="https://docs.python.org/3/library/stdtypes.html#str.isdigit" rel="nofollow">here</a> <code>isdigit()</code> is a string method. You can't call this method for integers.</p>
<p>This line, </p>
<pre><code>cpi = eval(input("Enter the CPI for July 2015: ")) 
</code></pre>
<p><code>evaluates</code> the user input to integer. </p>
<pre><code>&gt;&gt;&gt; x = eval(input("something: "))
something: 34
&gt;&gt;&gt; type(x)
&lt;class 'int'&gt;
&gt;&gt;&gt; x.isdigit()
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
AttributeError: 'int' object has no attribute 'isdigit'
</code></pre>
<p>But if you remove <code>eval</code> method (you should better do that), </p>
<pre><code>&gt;&gt;&gt; x = input("something: ")
something: 54
&gt;&gt;&gt; type(x)
&lt;class 'str'&gt;
&gt;&gt;&gt; x.isdigit()
True
</code></pre>
<p>everything will be fine. </p>
<p>by the way using eval without sanitizin user input may cause problems</p>
<p>consider this.</p>
<pre><code>&gt;&gt;&gt; x = eval(input("something: "))
something: __import__('os').listdir()
&gt;&gt;&gt; x
['az.php', 'so', 'form.php', '.htaccess', 'action.php' ...
</code></pre>
</div>

<p>If you want to catch the error if user didn't enter a number, just use <code>try...except</code> like this:</p>
<pre><code>numOfYears = 0

while numOfYears == 0:
    try:
        cpi = int(input("Enter the CPI for July 2015: "))
    except ValueError:
        print("Bad input")
    else:
        while cpi &lt; (cpi * 2):
            cpi *= 1.025
            numOfYears += 1

print("Consumer prices will double in", numOfYears, "years.")
</code></pre>
