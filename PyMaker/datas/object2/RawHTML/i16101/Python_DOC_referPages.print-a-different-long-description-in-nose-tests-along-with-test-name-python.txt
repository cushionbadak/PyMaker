<div class="post-text" itemprop="text">
<p>Unittest is <a href="https://docs.python.org/3/library/unittest.html#unittest.TestCase.shortDescription" rel="nofollow noreferrer">documented as only showing the first line</a> of the test method's docstring.  But you could override the default implementation of <code>shortDescription</code> method to customise that behaviour:</p>
<pre><code>import unittest

class TestClass(unittest.TestCase):

    def shortDescription(self):
        return self._testMethodDoc

    def test_1(self):
       """this is a long description //
              continues on second line """
       self.assertTrue(True)

    def test_2(self):
       """this is another or different long description //
              continues on second line which also gets printed :) """
       self.assertTrue(True)

if __name__ == '__main__':
    unittest.main(verbosity=2)
</code></pre>
<p>Demo:</p>
<pre><code>$ nosetests -v example.py 
this is a long description //
              continues on second line ... ok
this is another or different long description //
              continues on second line which also gets printed :) ... ok

----------------------------------------------------------------------
Ran 2 tests in 0.000s

OK
</code></pre>
<p>Someone wrote a nose plugin to fix this exact annoyance, maybe you'd be interested to use that.  Here it is:  <a href="https://bitbucket.org/natw/nose-description-fixer-plugin/" rel="nofollow noreferrer">https://bitbucket.org/natw/nose-description-fixer-plugin/</a></p>
