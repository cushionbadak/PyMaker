<p>Why is it possible to dynamically define a new attribute for every class instance in Python as long as it's not of type <code>object</code>?</p>
<pre><code>class NewStyle(object):
    pass

class OldStyle:
    pass

a = object()
# This line raises AttributeError
a.foo = 1

# All of these work fine
a = Exception()
a.foo = 1

a = OldStyle()
a.foo = 1

a = NewStyle()
a.foo = 1
</code></pre>
<p>This behaviour seems to be identical in Python 3</p>
</div>
<div class="post-text" itemprop="text">
<p>From <a href="https://docs.python.org/3/library/functions.html#object" rel="nofollow">the python doc</a>:</p>
<blockquote>
<p>Note: <code>object</code> does not have a <code>__dict__</code>, so you canâ€™t assign arbitrary attributes to an instance of the <code>object</code> class.</p>
</blockquote>
<p>Some class definition provide a <code>__dict__</code> attribute and then it is included in instances by default. </p>
<p>See also <a href="https://docs.python.org/3/reference/datamodel.html#slots" rel="nofollow"><code>__slots__</code></a></p>
