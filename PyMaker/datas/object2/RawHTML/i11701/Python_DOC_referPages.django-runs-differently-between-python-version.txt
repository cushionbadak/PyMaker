<div class="post-text" itemprop="text">
<p>Today I have come across wired error running the Django application with virtual environments. </p>
<p>The application runs without errors both locally and Heroku under python-3.5.1 with <code>Procfile</code> </p>
<p><code>web: gunicorn --pythonpath='src/' proj.wsgi:application --log-file -</code></p>
<p>And my project structure is:</p>
<p>```</p>
<pre><code>.
├── LICENSE.txt
├── Procfile
├── README.md
├── db.sqlite3
├── docs
├── env3
├── env
├── logs
├── manifest.yml
├── requirements
├── requirements.txt
├── run.sh
├── runtime.txt
├── site
└── src/proj
    ├── logger.py
    ├── settings
    │   ├── __init__.py
    │   ├── base.py
    │   ├── development.py
    │   └── production.py
    ├── urls.py
    ├── views.py
    ├── wsgi.py
</code></pre>
<p>```</p>
<p>But when I run it with python 2.7.11 I get the following error</p>
<p><code>ImportError: No module named proj.settings.development</code></p>
<p>I could easily guess the error is related pythonpath config. However I am curious why in first place I am getting an error?</p>
</div>
<div class="post-text" itemprop="text">
<p>Fixed now! The reason is that as the default django structure is rewritten every nested folder contains module need an <code>__init__.py</code>.</p>
<p><code>src/proj</code> missing the <code>__init__.py</code>. Adding it fixed the error for python2. From <code>python3.3</code> this is not an requirement </p>
<p>Reference <a href="https://docs.python.org/3/whatsnew/3.3.html#pep-420-implicit-namespace-packages" rel="nofollow">PEP420</a> <a href="https://www.python.org/dev/peps/pep-0420/" rel="nofollow">https://www.python.org/dev/peps/pep-0420/</a></p>
</div>
