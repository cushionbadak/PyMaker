</div>
<div class="post-text" itemprop="text">
<p>At a Python console:</p>
<pre><code>&gt;&gt;&gt; import string
&gt;&gt;&gt; type(string)
&lt;type 'module'&gt;

&gt;&gt;&gt; type("someValues")
&lt;type 'str'&gt;

&gt;&gt;&gt; help(dir)
</code></pre>
<blockquote>
<p>dir([object]) -&gt; list of strings</p>
<p>If called without an argument, return the names in the current scope.
  Else, return an alphabetized list of names comprising (some of) the &gt;attributes
  of the given object, and of attributes reachable from it.
  If the object supplies a method named <strong>dir</strong>, it will be used; otherwise
  the default dir() logic is used and returns:</p>
<pre><code>for a module object: the module's attributes.
for a class object:  its attributes, and recursively the attributes
  of its bases.
for any other object: its attributes, its class's attributes, and
  recursively the attributes of its class's base classes.
</code></pre>
</blockquote>
<p>Thus, <code>dir(string)</code> returns the attributes of the <a href="https://docs.python.org/2/library/string.html" rel="nofollow"><code>string</code></a> module while
<code>dir("someValues")</code> returns the attributes of the <a href="https://docs.python.org/3/library/stdtypes.html#text-sequence-type-str" rel="nofollow"><code>str</code></a> class as well as the attributes of instances thereof. For a proper distinction, take a look at <a href="http://learnpythonthehardway.org/book/ex40.html" rel="nofollow">Modules, Classes, and Objects</a>.</p>
</div>
<span class="comment-copy">Because <code>someString</code> is an instance of <code>str</code> and <code>string</code> is a module from standard library.</span>
<span class="comment-copy">Indeed, the <code>dir()</code> of two totally different things is likely to be different.</span>
