<p>Due to the way the Python C-level APIs developed, a lot of built-in

</blockquote>
<p>I found it really annoying cause I'm not be able to know it by looking at the doc. For instance, <a href="https://docs.python.org/3/library/functions.html#eval" rel="nofollow noreferrer">eval</a></p>

<p>Then I wrote this line of code</p>
<pre><code>print(eval('a+b', globals={'a':1, 'b':2}))
</code></pre>
<p>and got <code>TypeError: eval() takes no keyword arguments</code>. So is there a complete list of functions of this kind? How do I know if a function is allowed to have keyword arguments?</p>

<div class="post-text" itemprop="text">
<p>In Python 3.5 you can inspect the <code>__text_signature__</code> of the built-in function:</p>
<pre><code>&gt;&gt;&gt; eval.__text_signature__
'($module, source, globals=None, locals=None, /)'
</code></pre>
<p>or </p>
<pre><code>&gt;&gt;&gt; abs.__text_signature__
'($module, x, /)'
&gt;&gt;&gt; abs(x=5)
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: abs() takes no keyword arguments
</code></pre>
<p>(<code>x</code> cannot be used as a keyword argument)</p>
<p>The <code>/</code> means that the arguments following that can be used as keyword arguments. C.f. </p>
<pre><code>&gt;&gt;&gt; compile.__text_signature__
'($module, /, source, filename, mode, flags=0,\n        dont_inherit=False, optimize=-1)'
&gt;&gt;&gt; compile(source='foo', filename='bar', mode='exec')
&lt;code object &lt;module&gt; at 0x7f41c58f0030, file "bar", line 1&gt;
</code></pre>
<hr/>
<p>Of course there are bugs even in 3.5:</p>
<pre><code>&gt;&gt;&gt; sorted.__text_signature__
'($module, iterable, key=None, reverse=False)'
</code></pre>
<p>though according to <a href="https://bugs.python.org/issue26729" rel="nofollow">issue 26729 in the Python bug tracker</a>, there ought to be <code>/</code> after the <code>iterable</code> as the <code>iterable</code> cannot be used as a keyword argument.</p>
<hr/>
<p>Unfortunately this information is not yet available in the Python documentation itself.</p>
