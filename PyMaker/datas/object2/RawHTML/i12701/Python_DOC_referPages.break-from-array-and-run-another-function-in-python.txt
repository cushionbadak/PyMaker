<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/2/library/functions.html#enumerate" rel="nofollow">enumerate</a> and modulo:</p>
<pre><code>def exit():
    return "Exit"


arr = [1, 2, 3, 4, 5, 6, 7, 8, 9, 0]

def func():
    for ind, element in enumerate(arr, 1):
        print element
        if ind % 3 == 0:
            print(exit())
</code></pre>
<p>Output:</p>
<pre><code>1
2
3
Exit
4
5
6
Exit
7
8
9
Exit
0
</code></pre>
<p>If you actually want to use <code>n</code> elements at a time maybe passing them to a function:</p>
<pre><code>from itertools import islice

arr = [1, 2, 3, 4, 5, 6, 7, 8, 9, 0]

def func():
    it = iter(arr)
    for sli in iter(lambda: list(islice(it, 3)),[]):
        print(sli) # pass sli to function
        print(exit())

func()
</code></pre>
<p>Output:</p>
<pre><code>[1, 2, 3]
Exit
[4, 5, 6]
Exit
[7, 8, 9]
Exit
[0]
Exit
</code></pre>

<p>However, I question some parts of your code. Here are some notes:
<code>arr</code> is a python list, not an array, so the name is not the best one.
Your <code>exit</code> function simply returns the string "Exit", are you sure it does what it is supposed to do? I am printing the return value of <code>exit()</code>, because simply calling it would do nothing useful. In addition, it would probably be better to write <code>func</code> such that it takes a list as a parameter. Lastly, <code>exit</code> is already a built-in Python function, overriding it is a bad idea.</p>
