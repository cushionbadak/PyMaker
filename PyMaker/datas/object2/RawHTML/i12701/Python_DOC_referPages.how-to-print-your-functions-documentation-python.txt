<p>I have been searching for an answer for a lot of time now. Let's say I wrote a function in python and I made a brief documentation of what this function is doing. Is there any way to print the function's documentation from within main? Or from the function itself?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can either use <strong><a href="https://docs.python.org/3/library/functions.html#help" rel="nofollow"><code>help()</code></a></strong> or print the <code>__doc__</code>. <code>help()</code> prints a more verbose description of an object while <code>__doc__</code> holds <strong>only</strong> the documentation string you have defined with triple quotes <code>""" """</code> <em>in the beginning</em> of your function.</p>
<p>For example, using <code>__doc__</code> explicitly on the <code>sum</code> built-in function:</p>
<pre><code>print(sum.__doc__)
Return the sum of a 'start' value (default: 0) plus an iterable of numbers

When the iterable is empty, return the start value.
This function is intended specifically for use with numeric values and may
reject non-numeric types.
</code></pre>
<p>Additionally, since Python first compiles an object and during execution evaluates it you can call <code>__doc__</code> within the function with no problems:</p>
<pre><code>def foo():
    """sample doc"""
    print(foo.__doc__)

foo()  # prints sample doc
</code></pre>
<p>and remember, besides functions, modules and classes have a <code>__doc__</code> attribute holding their documentation.</p>
<p>Alternatively, using <code>help()</code> for <code>sum</code>:</p>
<pre><code>help(sum)
</code></pre>
<p>Will print: </p>
<pre><code>Help on built-in function sum in module builtins:

sum(iterable, start=0, /)
    Return the sum of a 'start' value (default: 0) plus an iterable of numbers

    When the iterable is empty, return the start value.
    This function is intended specifically for use with numeric values and may
    reject non-numeric types.
</code></pre>
<p>gives a bit more information, including the docstring.</p>
