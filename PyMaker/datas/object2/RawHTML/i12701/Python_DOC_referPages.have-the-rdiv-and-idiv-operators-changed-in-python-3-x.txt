<div class="post-text" itemprop="text">
<p>In Python 3.x the <code>__div__()</code> operator <a href="https://stackoverflow.com/questions/21859440/error-when-trying-to-overload-an-operator">was deprecated in favor of</a> <code>__floordiv__()</code> and <code>__truediv__()</code>.  </p>
<p>Are the <code>__rdiv__()</code> and <code>__idiv__()</code> operators still used to refer to the reversed and in-place versions of <code>__truediv__()</code>?  If so, what are the operator names for the reversed and in-place versions of <code>__floordiv__()</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>__rdiv__</code> and <code>__idiv__</code> no longer exist. The new names are the obvious choices of <code>__rtruediv__</code>, <code>__itruediv__</code>, <code>__rfloordiv__</code>, and <code>__ifloordiv__</code>, following the standard format. This is visible in the <a href="https://docs.python.org/3.3/reference/datamodel.html#object.__truediv__" rel="noreferrer">link</a> in the second answer on the question you linked, or by doing <code>dir(int)</code> in a Python 3 interpreter.</p>
</div>
