<p>I have a tuple (could also use list instead) like this:</p>
<pre><code> (1, 'jsmith', 'Ferrari', '10 million', 39, 16, 2, 0, '-')
</code></pre>
<p>What would be a sensible way to represent this as a JSON struct in python? The key is a string like so:</p>
<pre><code> { 
    "John Smith" : (1, 'jsmith', 'Ferrari', '10 million', 39, 16, 2, 0, '-') 
 }
</code></pre>
<p>Any suggestions? The JSON will be parsed in JS in the front end and will always be the same length. Missing entries are represented by '-'.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/3/library/json.html#json.dumps" rel="nofollow"><code>json.dumps</code></a>.</p>
<pre><code>&gt;&gt;&gt; import json
&gt;&gt;&gt; data = (1, 'jsmith', 'Ferrari', '10 million', 39, 16, 2, 0, '-')
&gt;&gt;&gt; json.dumps({'John Smith': data})
'{"John Smith": [1, "jsmith", "Ferrari", "10 million", 39, 16, 2, 0, "-"]}'
</code></pre>
<p>BTW, <a href="http://json.org/" rel="nofollow">JSON</a> has only array (<code>[...]</code>); no distinction between <code>list</code> and <code>tuple</code>.</p>
</div>
