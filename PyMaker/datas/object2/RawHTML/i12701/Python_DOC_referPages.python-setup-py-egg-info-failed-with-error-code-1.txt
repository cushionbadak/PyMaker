<div class="post-text" itemprop="text">
<p><code>execfile</code> is a <a href="https://docs.python.org/2/library/functions.html#execfile" rel="nofollow">standard library builtin</a> in Python 2. It was <a href="https://docs.python.org/3/whatsnew/3.0.html#builtins" rel="nofollow">removed</a> in Python 3, which means that <strong>pystashop</strong> does not support Python 3. You should try <a href="https://github.com/openlabs/pystashop/issues" rel="nofollow">contacting the developer(s)</a> and request Python 3 support, and in the meantime see if you can get by with Python 2.</p>
<p>You can attempt to remedy the situation yourself by cloning the <a href="https://github.com/openlabs/pystashop" rel="nofollow">GitHub repo</a>, making changes, and installing with <code>python3 setup.py install</code>, but there's no guarantee that it will work properly. The offending code seems to be <a href="https://github.com/openlabs/pystashop/blob/develop/setup.py#L12" rel="nofollow">here</a>:</p>
<pre><code>execfile(os.path.join('pystashop', 'version.py'))
</code></pre>
<p>You can replace this with the following:</p>
<pre><code>exec(open(os.path.join('pystashop','version.py')).read())
</code></pre>
<p>This will provide the expected functionality. From a cursory glance over the code, everything else <em>appears</em> to be compatible with Python 3, though I may have missed something.</p>
