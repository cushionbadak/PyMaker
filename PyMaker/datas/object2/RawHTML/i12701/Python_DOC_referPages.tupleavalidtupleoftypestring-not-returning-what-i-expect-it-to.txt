<p>In the standard Python IDE:</p>
<pre><code>&gt;&gt;&gt; tuple("(5,6,7)")
('(', '5', ',', '6', ',', '7', ')')
</code></pre>
<p>is (5,6,7) a valid tuple? If so, why is tuple("(5,6,7)") placing every character in the string into its own spot in the tuple? Thank you so much for your help!</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/stdtypes.html#tuple" rel="nofollow"><code>tuple</code></a> accepts an iterable and creates a tuple of the values in the iterable. Example:</p>
<pre><code>In [2]: tuple([1, 2, 42, 52, 1])
Out[2]: (1, 2, 42, 52, 1)
</code></pre>
<p><code>"(5,6,7)"</code> is an iterable, where each element is a single character. That is why you get a tuple of single characters. </p>
<p>In order to create a tuple from a string, you may use <a href="https://docs.python.org/3/library/ast.html#ast.literal_eval" rel="nofollow"><code>literal_eval</code></a> function from the <code>ast</code> module:</p>
<pre><code>In [3]: from ast import literal_eval

In [4]: literal_eval("(5,6,7)")
Out[4]: (5, 6, 7)
</code></pre>
</div>
<span class="comment-copy"><code>"(5,6,7)"</code> is a string with <code>(,5,6,7,)</code> as its elements. So when you iterate over it, each element gets to be an element of the resulting tuple.</span>
