<p>Using <a href="https://docs.python.org/3/library/functions.html#sorted" rel="nofollow"><code>sorted</code></a> with <a href="https://docs.python.org/3/library/functions.html#int" rel="nofollow"><code>int</code></a> as a key function:</p>
<pre><code>&gt;&gt;&gt; rows = [
...     ['Dan Danson', '1', '6', '9'],
...     ['John Johnson', '5', '7', '10'],
...     ['Mike Mikeson', '10', '7', '6'],
... ]
&gt;&gt;&gt;
&gt;&gt;&gt; rows = [row[:1] + sorted(row[1:], key=int, reverse=True) for row in rows]
&gt;&gt;&gt; sorted(rows, key=lambda row: sum(map(int, row[1:])), reverse=True)
[['Mike Mikeson', '10', '7', '6'],
 ['John Johnson', '10', '7', '5'],
 ['Dan Danson', '9', '6', '1']]
</code></pre>
<ul>
<li><code>sorted(row[1:], ..)</code>: separate number values and sort.</li>
<li><code>row[:1]</code>: name as a list, alternatively you can use <code>[row[0]]</code>. Should be a list to be concatenated to a list of number strings.</li>
</ul>

<p>You can approach the problem this way as well. I first convert all grades into integer type so that I can keep my lambda function clean. I could have done the conversion in lambda function but it does not fancy me much. </p>
<p>You can see splitting the each problem into different module, gives re-usability.     </p>
<pre><code>test = [['Dan Danson', '1', '6', '9'], ["Karthikeyan", 10, 10, 10], ['John Johnson', '5', '7', '10'], ['Mike Mikeson', '10', '7', '6']]

def covert_to_integer(test):
    """
    Coverting all grades into integer type
    """
    for outer_index, item in enumerate(test):
        for inner_index, element in enumerate(item[1:], 1):
            test[outer_index][inner_index] = int(element)
    return sorting_by_sum(test)

def sorting_by_sum(test):
    """
    Sorting the records by sum of the grades. 
    """
    return sorted(test, key=lambda record: record[1]\
                                   + record[2] \
                                   + record[3], reverse=True)

if __name__ == "__main__":
    print covert_to_integer(test)
