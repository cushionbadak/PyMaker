<div class="post-text" itemprop="text">
<p>The syntax to declare the types for the <code>name</code> and <code>id</code> fields you are using requires <a href="https://docs.python.org/3/whatsnew/3.6.html#pep-526-syntax-for-variable-annotations" rel="noreferrer">Python 3.6 or up</a>. Python 3.5 does not support the variable-level type hints required.</p>
<p>From the <a href="https://docs.python.org/3/library/typing.html#typing.NamedTuple" rel="noreferrer"><code>typing.NamedTuple</code> documentation</a>:</p>
<blockquote>
<p>Changed in version 3.6: Added support for <a href="https://www.python.org/dev/peps/pep-0526/" rel="noreferrer">PEP 526</a> variable annotation syntax.</p>
</blockquote>
<p>Use the <em>backwards compatible syntax</em> also included in the documentation:</p>
<pre><code>Employee = NamedTuple('Employee', [('name', str), ('id', int)])
</code></pre>
<p>so listing the field names as <code>(name, type)</code> tuples.</p>
<p>If you are using Python 3.5, you may want to switch to the <a href="https://docs.python.org/3.5/library/typing.html#typing.NamedTuple" rel="noreferrer">Python 3.5 version of the documentation</a> instead (there is a selector in the top-left corner, or you can just replace the <code>3</code> in the URL with <code>3.5</code>).</p>
</div>
