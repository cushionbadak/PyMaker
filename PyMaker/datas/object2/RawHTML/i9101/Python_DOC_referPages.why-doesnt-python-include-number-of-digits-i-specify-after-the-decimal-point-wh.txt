<div class="post-text" itemprop="text">
<p>You are confusing rounding with formatting. Rounding produces a new <code>float</code> object with the rounded value, which is still going to print the same way as any other float:</p>
<pre><code>&gt;&gt;&gt; print(31.00)
31.0
</code></pre>
<p>Use the <a href="https://docs.python.org/3/library/functions.html#format"><code>format()</code> function</a> if you need to produce a <em>string</em> with a specific number of decimals:</p>
<pre><code>&gt;&gt;&gt; print(format(31.0, '.2f'))
31.00
</code></pre>
<p>See the <a href="https://docs.python.org/3/library/string.html#formatspec"><em>Format Specification Mini-Language</em></a> section for what options you have available.</p>
<p>If the value is part of a larger string, you can use the <a href="https://docs.python.org/3/library/stdtypes.html#str.format"><code>str.format()</code> method</a> to embed values into a string template, using the same formatting specifications:</p>
<pre><code>&gt;&gt;&gt; a = 5 * 6.2
&gt;&gt;&gt; print('The value of "a" is {:.2f}'.format(a))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Python always prints at least one digit after the decimal point so you can tell the difference between integers and floats. </p>
<p>The <code>round()</code> function merely rounds the number to the specified number of decimal places. It does not control how it is printed. 7.36 and 7.360 are the same number, so the shorter is printed.</p>
<p>To control the printing, you can use formatting.  For example:</p>
<pre><code>print(".3f" % b)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Python does round to 3 decimal places. It is the printing that cuts additional zeros. Try something like <code>print("%.3f" % number)</code> </p>
