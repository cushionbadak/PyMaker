<pre><code>from datetime import datetime

<div class="post-text" itemprop="text">
<p>A 12-hour clock has no <code>0</code> hour; <code>%I</code> will only match 1 through to 12. Your timestamp has an impossible time in it:</p>
<pre><code>0:03 pm
</code></pre>
<p>From the <a href="https://docs.python.org/3/library/datetime.html#strftime-strptime-behavior" rel="nofollow noreferrer"><em><code>strftime()</code> and <code>strptime()</code> Behavior</em> documentation</a>:</p>
<blockquote>
<p><code>%I</code><br/>
  Hour (12-hour clock) as a zero-padded decimal number.<br/>
  01, 02, ..., 12</p>
</blockquote>
<p>Assuming <code>0</code> is really <code>12</code>, you could repair this by replacing the <code>' 0:'</code> with <code>'12:'</code> (note the leading space for the zero!):</p>
<pre><code>&gt;&gt;&gt; from datetime import datetime
&gt;&gt;&gt; latest_datetime = 'Feb 1, 2017  0:03 pm'
&gt;&gt;&gt; datetime.strptime(latest_datetime.replace(' 0:', '12:'), "%b %d, %Y %I:%M %p")
datetime.datetime(2017, 2, 1, 12, 3)
</code></pre>
<p>It doesn't really matter if you have one or two spaces between the year and the hour, the string will be parsed either way.</p>
