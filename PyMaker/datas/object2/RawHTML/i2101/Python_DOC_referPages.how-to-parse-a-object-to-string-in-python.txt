<p>You have the <em>exit code</em> of the process. See the <a href="https://docs.python.org/3/library/subprocess.html#subprocess.call" rel="nofollow noreferrer"><code>subprocess.call()</code> documentation</a>:</p>
<blockquote>
<p>Run the command described by args. Wait for command to complete, then return the <a href="https://docs.python.org/3/library/subprocess.html#subprocess.Popen.returncode" rel="nofollow noreferrer"><code>returncode</code></a> attribute.</p>
</blockquote>
<p>and for the linked <code>returncode</code> entry:</p>
<blockquote>
<p>The child return code, set by <code>poll()</code> and <code>wait()</code> (and indirectly by <code>communicate()</code>).</p>
</blockquote>
<p>If you expected the output the process produces on stdout, you should use <a href="https://docs.python.org/3/library/subprocess.html#subprocess.run" rel="nofollow noreferrer"><code>subprocess.run()</code></a>, and set <code>stdout</code> to <code>subprocess.PIPE</code> so you can read that output in Python:</p>
<pre><code>import subprocess

callme = 'whois %s ' % domain
proc = subprocess.run(callme, shell=True, stdout=subprocess.PIPE)
data = proc.stdout
</code></pre>
<p>This gives you a <code>bytes</code> object; you could set <code>text=True</code> to get a decoded string value.</p>
