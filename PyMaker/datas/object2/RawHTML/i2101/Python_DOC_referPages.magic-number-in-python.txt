<p>In the following code :</p>
<pre><code>magicnumber = 256
for n in range (500):
    if n is magicnumber:
        print ("the magic number is " , n)
        break
    else:
        print(n)
</code></pre>
<p>The loop breaks at 256, but if you set <code>magicnumber</code> to 257, it doesn't. Why ?</p>
</div>
<div class="post-text" itemprop="text">
<p>Because you are checking for <code>identity</code> with <code>is</code> (instead of checking for <code>equality</code> with <code>==</code>).</p>
<p>As an implementation detail, Python keeps an array of integer objects for all integers
between -5 and 256, when you create an int in that range you actually just get
back a reference to the existing object.</p>
<p>So integers above 256 will still be <code>equal</code>, but not <code>identical</code> (unless they refer to the same object, you can compare that with <code>id()</code>).</p>
<p>Source: <a href="https://docs.python.org/3/c-api/long.html#c.PyLong_FromLong" rel="noreferrer">https://docs.python.org/3/c-api/long.html#c.PyLong_FromLong</a></p>
