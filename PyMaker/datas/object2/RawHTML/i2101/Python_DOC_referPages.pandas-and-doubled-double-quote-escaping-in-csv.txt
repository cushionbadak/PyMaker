<div class="post-text" itemprop="text">
<p>Sometimes CVS data is formatted like this</p>
<pre><code>col1,col2,col3
a,b,"this field has an embedded quote character ("") in it"
</code></pre>
<p>Which is intended to be parsed as</p>
<pre><code>col1 | col2 | col3
a    | b    | this field has an embedded quote character (") in it
</code></pre>
<p>That is, the field-quoting character is escaped by doubling it.</p>
<p>The Python <code>csv.reader</code> module handles this just fine, as long as <a href="https://docs.python.org/3/library/csv.html#csv.Dialect.doublequote" rel="nofollow noreferrer"><code>csv.Dialect.doublequote</code></a> is <code>True</code>.</p>
<p>How can you do this in Pandas?</p>
</div>
<div class="post-text" itemprop="text">
<p><em>Note: I found the answer before I had even finished posting</em></p>
<p>Use <code>pd.read_csv(..., doublequote=True)</code></p>
<pre><code>import csv
import pandas as pd

data = pd.read_csv('data.csv', quotechar='"', doublequote=True, quoting=csv.QUOTE_NONNUMERIC)
</code></pre>
<p>swapping out <code>QUOTE_NONNUMERIC</code> for <code>QUOTE_MINIMAL</code>, or something else, as appropriate.</p>
</div>
<span class="comment-copy">Note that you can also escape the character or enclose the string in single quotes.</span>
