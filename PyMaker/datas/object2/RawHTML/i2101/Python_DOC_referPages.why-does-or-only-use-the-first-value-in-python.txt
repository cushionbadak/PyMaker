<div class="post-text" itemprop="text">
<p>Try evaluating the following expression in the python REPL:</p>
<p><code>'a' or 'b'</code></p>
<p>This gives <code>'a'</code>, because the <code>or</code> operator short-circuits; that is, it returns the first truthy argument it finds, in this case, <code>'a'</code>. Both <code>'a'</code> and <code>'b'</code> are truthy, so you're simply getting whichever one of those you put first.</p>
</div>
<div class="post-text" itemprop="text">
<p>If I understand your code correctly, you probably wanted to write:</p>
<pre><code>b = 'a'
print(b in ('a', 'b'))
</code></pre>
<p>This checks if <code>b</code> equals one of <code>'a'</code> and <code>'b'</code>.</p>

<span class="comment-copy">In strings a non-empty string evaluates as <code>True</code> and an empty string evaluates as <code>False</code>.  <code>(X or Y or Z)</code> will return the first "Truthy" value or if none are "Truthy" it will return the last "Falsey" value.  So when you compare <code>b == ('a' or 'b')</code> it first evaluates <code>('a' or 'b')</code> to <code>'a'</code> and then compares that to <code>b</code>.  What you want is <code>b in ('a', 'b')</code>.</span>
