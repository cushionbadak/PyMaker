<p>Operator <code>&gt;</code> in python 2.7 can be used with two different types like string and integer. Example:</p>
<pre><code>s = "xxx"
n = 123
s &gt; n // will output True
</code></pre>
<p>In python3 this operation is not allowed.</p>
<pre><code>TypeError: unorderable types: str() &gt; int()
</code></pre>
<p>Comparison operator <code>&gt;</code> is described in a detailed way in <a href="https://docs.python.org/3/reference/expressions.html#comparisons" rel="nofollow noreferrer">python docs</a></p>
<p>It seems that somewhere in your code you are trying to compare string vs integer.</p>
<p>EDIT (after OP posted the code)</p>
<p>The problem with <code>operator &gt;</code> unsupported arguments occurs, because your <code>RotatingFileHandler</code> setting <code>maxBytes</code> is interpreted as a string instead of integer. You can fix this problem by providing the list of <code>RotatingFileHandler</code> constructor arguments without using keyword notation. Like this:</p>
<pre><code>[handler_rotatingFileHandler]
class=logging.handlers.RotatingFileHandler
args=(r'c:\log\debug.log', 'a', 1000000, 3)
level=INFO
formatter=simpleFormatter
</code></pre>
<p>Second argument is <code>mode</code> and by default is equal to <code>'a'</code>. <a href="https://docs.python.org/2/library/logging.handlers.html#logging.handlers.RotatingFileHandler" rel="nofollow noreferrer">reference</a></p>
