<div class="post-text" itemprop="text">
<p><code>dir()</code> returns a list of all the defined names, but it is annoying to try to call function I see listed only to discover it is actually an attribute, or to try to access an attribute only to discover it is actually a callable.  How can I get <code>dir()</code> to be more informative?</p>
</div>
<div class="post-text" itemprop="text">
<p>To show a list of the defined names in a module, for example the math module, and their types you could do:</p>
<pre><code>[(name,type(getattr(math,name))) for name in dir(math)]
</code></pre>
<p>getattr(math,name) returns the object (function, or otherwise) from the math module, named by the value of the string in the variable "name". For example type(getattr(math,'pi')) is 'float'</p>
</div>
<div class="post-text" itemprop="text">
<p>There isn't a way to make <code>dir</code> 'more informative' as you put it, but you can use the <a href="https://docs.python.org/3/library/functions.html#callable" rel="nofollow"><code>callable</code></a> and <a href="https://docs.python.org/3/library/functions.html#getattr" rel="nofollow"><code>getattr</code></a> functions:</p>
<pre><code>[(a, 'func' if callable(getattr(obj, a)) else 'attr') for a in dir(obj)]
</code></pre>
<hr/>
<p>Obviously functions are still attributes themselves, but you get the idea.</p>
</div>
<div class="post-text" itemprop="text">
<p>Another way is to use <code>getmembers</code> function in <code>inspect</code>. You can get a similar result to James's one by </p>
<pre><code>from inspect import getmembers

getmembers(obj)  # =&gt; ...
</code></pre>
<p>For more information, please take a look at:</p>
<p><a href="https://docs.python.org/2/library/inspect.html#inspect.getmembers" rel="nofollow">https://docs.python.org/2/library/inspect.html#inspect.getmembers</a></p>
</div>
