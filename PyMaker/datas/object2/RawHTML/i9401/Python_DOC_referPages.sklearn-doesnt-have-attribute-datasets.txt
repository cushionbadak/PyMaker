<div class="post-text" itemprop="text">
<p><code>sklearn</code> is a <a href="https://docs.python.org/3/glossary.html#term-regular-package" rel="noreferrer">package</a>. <a href="https://stackoverflow.com/a/7948672/3642398">This answer</a> said it very succinctly:</p>
<blockquote>
<p>when you import a package, only variables/functions/classes in the <code>__init__.py</code> file of that package are directly visible, not sub-packages or modules.</p>
</blockquote>
<p><code>datasets</code> is a sub-package of <code>sklearn</code>. This is why this happens:</p>
<pre><code>In [1]: import sklearn

In [2]: sklearn.datasets
---------------------------------------------------------------------------
AttributeError                            Traceback (most recent call last)
&lt;ipython-input-2-325a2bfc35d0&gt; in &lt;module&gt;()
----&gt; 1 sklearn.datasets

AttributeError: module 'sklearn' has no attribute 'datasets'
</code></pre>
<p>However, the reason why this works: </p>
<pre><code>In [3]: from sklearn import datasets

In [4]: sklearn.datasets
Out[4]: &lt;module 'sklearn.datasets' from '/home/ethan/.virtualenvs/test3/lib/python3.5/site-packages/sklearn/datasets/__init__.py'&gt;
</code></pre>
<p>is that when you load the sub-package <code>datasets</code> by doing <code>from sklearn import datasets</code> it is automatically added to the namespace of the package <code>sklearn</code>. This is one of the lesser-known <a href="http://python-notes.curiousefficiency.org/en/latest/python_concepts/import_traps.html#the-submodules-are-added-to-the-package-namespace-trap" rel="noreferrer">"traps" of the Python import system</a>.</p>
<p>Also, note that if you look at the <a href="https://github.com/scikit-learn/scikit-learn/blob/master/sklearn/__init__.py#L60" rel="noreferrer"><code>__init__.py</code> for <code>sklearn</code></a> you <em>will</em> see <code>'datasets'</code> as a member of <a href="https://docs.python.org/3/tutorial/modules.html#importing-from-a-package" rel="noreferrer"><code>__all__</code></a>, but this only allows you to do:</p>
<pre><code>In [1]: from sklearn import *
In [2]: datasets
Out[2]: &lt;module 'sklearn.datasets' from '/home/ethan/.virtualenvs/test3/lib/python3.5/site-packages/sklearn/datasets/__init__.py'&gt;
</code></pre>
<p>One last point to note is that if you inspect either <code>sklearn</code> or <code>datasets</code> you will see that, although they are packages, their type is <code>module</code>. This is because all packages are considered modules - however, not all modules are packages.</p>
