<div class="post-text" itemprop="text">
<p><code>/Date(N)/</code> is usually the format for dates serialized by the built-in WCF JSON serializer in .NET. The value can then be reconstructed by calling <code>new Date(N)</code> in JavaScript.</p>
<p>Running that in JavaScript will show why this date is very likely just wrong:</p>
<pre><code>new Date(-62135596800000)
// Date 0001-01-01T00:00:00.000Z
</code></pre>
<p>As you can see, you get the <code>0001-01-01 00:00:00.00</code>. This also happens to be the <em>default</em> value in .NET for empty <code>DateTime</code> objects:</p>
<pre class="lang-csharp prettyprint-override"><code>default(DateTime).ToString("u") // 0001-01-01 00:00:00Z
new DateTime(0).ToString("u") // 0001-01-01 00:00:00Z
</code></pre>
<p>So, sorry to say it like that, but that server is just broken.</p>
<hr/>
<p>That all being said, once you get a valid date, you can parse the number in Python using <a href="https://docs.python.org/3/library/datetime.html#datetime.datetime.fromtimestamp" rel="nofollow noreferrer"><code>datetime.fromtimestamp</code></a> or <a href="https://docs.python.org/3/library/datetime.html#datetime.datetime.utcfromtimestamp" rel="nofollow noreferrer"><code>datetime.utcfromtimestamp</code></a>:</p>
<pre><code>&gt;&gt;&gt; from datetime import datetime
&gt;&gt;&gt; datetime.fromtimestamp(1483120713887 // 1000)
datetime.datetime(2016, 12, 30, 18, 58, 33)
</code></pre>

<p>Using python 3.x<br/>
Looks like epoch time (in nano seconds starting January 1, 1970 00:00:00.000 GMT). You could use something like this:</p>
<pre><code>from datetime import datetime
dt = datetime.fromtimestamp(62135596800000 // 1000000000)
print(dt)
</code></pre>
<p>As for the value you're getting, you can't have a negative epoch time stamp (-62135596800000 would be invalid). If the - indicates negative, it might be another time stamp format.</p>
