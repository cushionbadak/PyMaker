<div class="post-text" itemprop="text">
<p>Python 3.6:</p>
<pre><code>class Sim():
     def __init__(self, *args, **kwargs):
        self._sim_info: SimInfo = None
</code></pre>
<p>Other python:</p>
<pre><code>class Sim():
     def __init__(self, *args, **kwargs):
        self._sim_info = None  # type: SimInfo
</code></pre>
<p>It called "<a href="https://docs.python.org/3/library/typing.html" rel="nofollow noreferrer">type hints</a>"</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="https://www.jetbrains.com/help/pycharm/2016.3/type-hinting-in-pycharm.html#d417517e57" rel="nofollow noreferrer">type hinting</a> with pycharm using the @type docstring.</p>
<pre><code>def __init__(self, *args, **kwargs):
    # n.b., in your usage code you use `.sim_info`
    # but in your constructor you used `._sim_info`
    # since I didnâ€™t see a prop get function, I assumed
    # that you meant `.sim_info`
    self.sim_info = None 
    """@type: SimInfo"""
</code></pre>
<p>or</p>
<pre><code>def __init__(self, *args, **kwargs):
    self.sim_info = None
    """:type SimInfo"""
</code></pre>
<p>You can also specify the full path to the class including the module if just the class name did not work.  You can also use PEP484 in PyCharm to specify the type of member variables:</p>
<pre><code>def __init__(self):
    self.sim_info = None # type: SimInfo
</code></pre>
