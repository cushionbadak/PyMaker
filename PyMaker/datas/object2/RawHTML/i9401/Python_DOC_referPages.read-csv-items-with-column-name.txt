<pre><code>with open('info.csv') as f:

<div class="post-text" itemprop="text">
<p>You are looking for <a href="https://docs.python.org/2/library/csv.html#csv.DictReader" rel="noreferrer">DictReader</a></p>
<pre><code>with open('info.csv') as f:
    reader = csv.DictReader(f, delimiter=';')
    for row in reader:
        name = row['name']
        blah = row['blah']
</code></pre>
<p>to quote from the link:</p>
<blockquote>
<p>Create an object which operates like a regular reader but maps the
  information read into a dict whose keys are given by the optional
  fieldnames parameter.
  ...
  If the fieldnames parameter is omitted, the values in the first row of
  the csvfile will be used as the fieldnames.</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>Yes, there is. That's what <a href="https://docs.python.org/2/library/csv.html#csv.DictReader" rel="nofollow noreferrer"><code>csv.DictReader</code></a> function does - supplies the rows as an iterable of dicts.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use a <a href="https://docs.python.org/3/library/csv.html#csv.DictReader" rel="nofollow noreferrer">csv.DictReader</a> instance to get this behaviour.</p>
<p>Example from the docs:</p>
<pre><code>&gt;&gt;&gt; with open('names.csv') as csvfile:
...     reader = csv.DictReader(csvfile)
...     for row in reader:
...         print(row['first_name'], row['last_name'])
...
Eric Idle
John Cleese
</code></pre>
