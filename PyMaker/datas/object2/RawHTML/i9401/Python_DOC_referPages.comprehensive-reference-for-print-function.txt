<div class="post-text" itemprop="text">
<p>That's all covered under string formatting, which you can read up <a href="https://docs.python.org/3/library/string.html#format-string-syntax" rel="nofollow noreferrer">here</a>.</p>
<p>Basically, the <code>{...}</code> sequences are placeholders which will use the arguments given to <code>format()</code> to construct a single result string.</p>
<p>These placeholders identify things such as <em>which</em> argument to use and formatting information for the argument (justification, padding, decimal places and so on).</p>
<p>By way of example, the following expression:</p>
<pre><code>"{0:0&gt;16b}".format(27)
</code></pre>
<p>gives you the value <code>27</code> in binary:</p>
<pre><code>0000000000011011
</code></pre>
<p>That's because the format string works as follows (spaces added for readability, but they're not in the <em>actual</em> format specifier):</p>
<pre><code>{0:0 &gt; 16 b}
 | | | || |
 | | | |/ +-- binary
 | | | +---- width of 16
 | | +------ right justified
 | +------- zero padded
 +--------- argument zero (the first one)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Here is a summary of the syntax by using the str method <code>format</code>:</p>
<ul>
<li>"{}".format(42) ==&gt; "42"</li>
<li>"{0}".format(42) ==&gt; "42"</li>
<li>"{0:.2f}".format(42) ==&gt; "42.00"</li>
<li>"{0:.0f}".format(42.1234) ==&gt; "42"</li>
<li>"{answer}".format(no_answer=41, answer=42) ==&gt; "42"</li>
<li>"{answer:.2f}".format(no_answer=41, answer=42) ==&gt; "42.00"</li>
<li>"{answer} = {answer}".format(answer=42) ==&gt; "42 = 42"</li>
</ul>
<p>For your example, {0} means the first argument in the <code>format</code> method which is <code>key</code> in this case and {1} means the second argument which is <code>value</code> in this case.</p>
</div>
