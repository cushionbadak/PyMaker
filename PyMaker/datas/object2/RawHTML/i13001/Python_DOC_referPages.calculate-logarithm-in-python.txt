<p>I am wondering why the result of <code>log base 10 (1.5)</code> in python = 0.405465108108 while the real answer = 0.176091259.</p>
<p>This is the code that I wrote:</p>
<pre><code>import math
print math.log(1.5)
</code></pre>
<p>Can someone tell how to solve this issue?</p>

<div class="post-text" itemprop="text">
<p>From <a href="https://docs.python.org/2/library/math.html#math.log" rel="noreferrer">the documentation</a>:</p>
<blockquote>
<p>With one argument, return the natural logarithm of <em>x</em> (to base <em>e</em>).</p>
<p>With two arguments, return the logarithm of <em>x</em> to the given <em>base</em>, calculated as <code>log(x)/log(base)</code>.</p>
</blockquote>
<p>But the log<sub>10</sub> is made available as <code>math.log10()</code>, which does not resort to log division if possible.</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>math.log10(1.5)
</code></pre>
<p>Use the log10 function in the math module.</p>
</div>
<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/math.html#math.log" rel="noreferrer"><code>math.log</code></a> function is to the base <code>e</code>, i.e. natural logarithm. If you want to the base 10 use <a href="https://docs.python.org/3/library/math.html#math.log10" rel="noreferrer"><code>math.log10</code></a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>If you use log without base it uses <code>e</code>.</p>
<p>From the comment</p>
<blockquote>
<p>Return the logarithm of x to the given base.<br/>
  If the base not specified, returns the natural logarithm (base e) of x.</p>
</blockquote>
<p>Therefor you have to use:</p>
<pre><code>import math
print( math.log(1.5, 10))
</code></pre>
</div>
<span class="comment-copy">Because like in most languages, <code>log</code> is logarithm in base e (natural logarithm), not base 10. If you want log in base 10, use <code>log10</code>.</span>
<span class="comment-copy"><code>math.log(x, n)</code> is useful when you have custom base, but if you want logarithm base 2 or 10, use <code>math.log2</code> or <code>math.log10</code>.</span>
<span class="comment-copy">You are right, using <code>log10</code> directly has a better readability and is less prone to errors</span>
<span class="comment-copy">math.log2(x) doesn't exist until Python 3.3. For Python 2 you have to use math.log(x, 2).</span>
