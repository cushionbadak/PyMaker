<p>Your issue is here: <code>x=f.read(l)</code></p>
<p>When you pass a single argument to <code>read()</code> is expects and integer (how many bytes to read IIRC).</p>
<p>Use <code>x=f.read()</code> with no argument to read all data from the file.</p>
<p>Also <code>l=f.write(str(rate))</code> is unnecessary.   Just call <code>f.write(str(rate))</code>  no need to store return value, unless you're expecting something to go wrong.</p>
<p><strong>EDIT:</strong>  I have a few recommendations:  Use a string rather than an int and a key for your dictionaries.  This way <code>raw_input()</code> will provide a key you can use directly.</p>
<p>Also, you are reading a writing to the same file too often, it is messing up your reading.  See <a href="https://stackoverflow.com/questions/9282967/how-to-open-a-file-using-the-open-with-statement">here</a> for more pythonic handling of files.  I would recommend encapsulating your reads and write this way.</p>
<p>If you just want to check that you have written the data to a file, simply use <code>f.write(data)</code> and <strong>assume it work unless it throws an error.</strong>  See my answer <a href="https://stackoverflow.com/questions/31731841/how-to-check-if-writing-to-a-file-fails-in-python/31731966#31731966">here.</a></p>
