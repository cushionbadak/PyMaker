<div class="post-text" itemprop="text">
<p>There's a few bugs you need to fix here. First of all you overwrite the parameters in the division function:</p>
<pre><code>def division(x, y):  
        try:
            x = 0  #now x is 0 regardless of what you passed in
            y = 0  #same for y
</code></pre>
<p>Then in this line:</p>
<pre><code>if (x!= 0 or y!= 0): 
</code></pre>
<p>If x is not 0 this is true because the <code>or</code> operation will return true if any of the conditions are true. So for example if you have x = 1 and y = 0 you still perform the invalid division. You could fix this by checking just for <code>y != 0</code>.</p>
<p>A much better way to do this would just be to let python do the division and catch the <a href="https://docs.python.org/3/library/exceptions.html#ZeroDivisionError" rel="nofollow noreferrer">ZeroDivisionError</a> that is raised if you attempt to make an invalid division:</p>
<pre><code>def division(x, y):
    try:
        return x/y
    except ZeroDivisionError:
        print("This operation is not supported for given input parameters")
</code></pre>
<p>This is much more in line with the overall Python philosophy of <a href="https://stackoverflow.com/questions/11360858/what-is-the-eafp-principle-in-python">"Easier to ask for forgiveness than permission"</a>.</p>
