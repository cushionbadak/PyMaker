<p>I have made it a habbit that when i do <code>cur.execute()</code> functions, I do a <code>sql.commit()</code> function right after.</p>
<p>Is <code>sql.commit()</code> needed if I do not modify the database (i.e. only do a <code>SELECT</code> query)?</p>
<p>Here is an example:</p>
<pre><code>sql = sqlite3.connect('DB.db')
cur = sql.cursor()
cur.execute('SELECT * FROM table')
sql.commit()
</code></pre>

<div class="post-text" itemprop="text">
<p>Python does not begin a transaction for select statements.</p>
<p>From the <a href="https://docs.python.org/3/library/sqlite3.html#sqlite3-controlling-transactions" rel="noreferrer">Transaction control</a> section of the sqlite3 module docs:</p>
<blockquote>
<p>By default, the sqlite3 module opens transactions implicitly before a Data Modification Language (DML) statement (i.e. INSERT/UPDATE/DELETE/REPLACE), and commits transactions implicitly before a non-DML, non-query statement (i. e. anything other than SELECT or the aforementioned).</p>
</blockquote>
</div>
