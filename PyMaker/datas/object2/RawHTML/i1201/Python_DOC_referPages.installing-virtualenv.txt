<p>I installed Python 3.7.1 using home-brew, and on the command line typing <code>python</code> returns the default python 2.7.10 and <code>python3</code> gives the home-brew installed version. Then using </p>
<p><code>python3 -m pip install --user numpy scipy matplotlib</code></p>
<p>I installed the required packages. 
Now what I understand is that to use the home-brew version python I have to always use <code>python3</code>and the required packages are installed inside python 3's lib folder.</p>
<p>I want to install virtual env correctly. I want to know what command should I use if I want it to be usable with python3?</p>

<div class="post-text" itemprop="text">
<p>You don't need to install virtual env because <a href="https://docs.python.org/3/library/venv.html" rel="nofollow noreferrer"><code>venv</code></a> module is included in Python 3 standard library.  </p>
<p>To create a virtualenv under <code>.venv</code> subdirectory of the current working directory:</p>
<pre><code> python3 -m venv .venv
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>From my understanding you want to create python3 environments. First install virtualenv using pip.<br/>
<code>pip install virtualenv</code><br/>
Then you can create a python3 environment like this:<br/>
<code>virtualenv -p python3 env_name</code></p>
</div>
<span class="comment-copy">Do it the same way you did for numpy, scipy and matplotlib.</span>
