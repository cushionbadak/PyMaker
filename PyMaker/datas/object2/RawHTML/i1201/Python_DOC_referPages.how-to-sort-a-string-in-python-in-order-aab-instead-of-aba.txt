<p>I'm trying to sort a string (to make a genotype for a punnetsquare).
My current implementation is:</p>
<pre><code>unsorted_genotype = 'ABaB'
sorted_genotype = sorted(list(unsorted_genotype))

<div class="post-text" itemprop="text">
<p>You can specify a <a href="https://docs.python.org/3/library/functions.html#sorted" rel="nofollow noreferrer">sorting key</a> and use the sorting for tuples: </p>
<p><code>[(1,1),(1,2)]</code>: Tuples are sorted by 1st element, on equality by 2nd to nth:</p>
<pre><code>unsorted = 'ABaB'
s = sorted(unsorted, key= lambda x:(x.lower(),x))  
print(''.join(s))  # 'AaBB'
</code></pre>
<p>This ensures it is first sorted by the 'lower' character - grouping <code>a</code> and <code>A</code>  together and then by the actual character afterwards so they occure sorted as well: <code>ABaaAAaAaAAAB =&gt;  AAAAAAAaaaaBB</code></p>
<p>Readups:</p>
<ul>
<li><a href="https://docs.python.org/3/howto/sorting.html" rel="nofollow noreferrer">Overview</a></li>
<li><a href="https://docs.python.org/3/howto/sorting.html#key-functions" rel="nofollow noreferrer">by key (part of the first link)</a></li>
</ul>
<p>A simple <code>key = str.lower</code> as suggested in the allegated dupe does not group <code>A</code> to <code>A</code> and <code>a</code> to <code>a</code> which would be nice to have for <a href="https://en.wikipedia.org/wiki/Punnett_square" rel="nofollow noreferrer">Prunnet squares</a></p>
<hr/>
<p>If you had them rather stay in order of occurence then grouping lower- and uppercase together, you can use:</p>
<pre><code>unsorted = 'ABaaAAaAaAAAB'
s = sorted(unsorted, key=str.lower)  # not by tuple, just by lower
print(''.join(s))  
</code></pre>
<p>results in keeping the elements "in order":</p>
<pre><code>AaaAAaAaAAABB   # instead of  AAAAAAAaaaaBB
</code></pre>
<p>and is described in <a href="https://stackoverflow.com/questions/10269701/case-insensitive-list-sorting-without-lowercasing-the-result">case-insensitive list sorting, without lowercasing the result?</a></p>
</div>
