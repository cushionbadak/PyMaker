<p>I have the following to do two unit tests:</p>
<pre><code>import unittest
from unittest import TestCase


class TestUM(unittest.TestCase):

    def setUp(self):
        self.client = SeleniumClient()

    def test_login(self):
        self.client.login()
        self.assertIn("my-data", self.client.driver.current_url)
        print ('Log in successful.')

    def test_logout(self):
        self.client.logout()
        print ('Log out successful.')


if __name__ == '__main__':
    unittest.main()
</code></pre>
<p>However, it does <code>setUp</code> twice -- once for each of the unit tests. Is there a way I can do one setup across all the unittests for <code>TestUM</code> ? If so, how would I do that?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>setupClass</code> for that:</p>
<pre><code>class TestUM(unittest.TestCase):

    @classmethod
    def setUpClass(cls):
        cls.client = SeleniumClient()
</code></pre>
<p>From the <a href="https://docs.python.org/3/library/unittest.html#unittest.TestCase.setUpClass" rel="nofollow noreferrer">documentation</a>, this method is called only once before tests in class are run.</p>
