<p>I am familiar with using the 'and', 'not' and 'or' operators in Python and I have just learnt how the 'XOR' operator works. </p>
<p>But, it doesn't seem to work when I coded this:</p>
<pre><code>a = 2
b = 12

if a == 2 ^ b == 12:
    print("You must be broken") # because TRUE XOR TRUE IS FALSE

if a == 10 ^ b &gt; 12:
    print("You must also be broken") # because FALSE XOR FALSE IS FALSE

if a &lt; 10 ^ b &gt; 13:
    print("This should print because TRUE XOR FALSE IS TRUE")

if a &gt; 3 ^ b == 12:
    print("This should print because FALSE XOR TRUE IS TRUE")
</code></pre>
<p>The program does not print anything out?</p>

<div class="post-text" itemprop="text">
<p>You are misunderstanding <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer">Python's operator precedence</a>.</p>
<p>This condition:</p>
<pre><code>a &lt; 10 ^ b &gt; 13
</code></pre>
<p>means</p>
<pre><code>a &lt; (10^b) &gt; 13
</code></pre>
<p>which means</p>
<pre><code>(a &lt; 10^b) and (10^b &gt; 13)
</code></pre>
<p>So it is false.</p>
<p>Presumably you mean:</p>
<pre><code>(a &lt; 10) ^ (b &gt; 13)
</code></pre>
<p>Similarly for your other conditions.</p>
</div>

<span class="comment-copy">Look at <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer">Python's operator precedence</a>. Also keep in mind that in python <code>A == B == C</code> is the same as <code>A == B and B == C</code>, so <code>a == 2 ^ b == 12</code> is the same as <code>a == (2^b) == 12</code> which is the same as <code>a == (2^b) and (2^b) == 12</code>.</span>
<span class="comment-copy">While it happens to work, you should not apply bitwise operators to booleans. A more logical and readable version of an XOR would be <code>if (a == 2) != (b == 12)</code>.</span>
