<p>I have a EXE file, which could be called like this:</p>
<pre><code>Extractor64.exe -F C:\File1 -E  C:\file2
</code></pre>
<p>When trying to call this from subprocess.Popen, I got an could not find file error:</p>
<pre><code>t=subprocess.Popen("start C:\yardi_backup_v2\Extractor64.exe -F C:\yardi_backup_v2\miadmfggp_live_Full_TKDB1A112_201812180000.Lts.bak -E  C:\yardi_backup_v2\tmp\extracted.bak")
t.wait()
</code></pre>
<p>What is the right structure to call this?  </p>
</div>
<div class="post-text" itemprop="text">
<ol>
<li><code>start</code> is a shell built-in. It's not really an executable. So you cannot call it without <code>shell=True</code>. That's your primary error: there is no <code>start.exe</code> in the system. But there's more...</li>
<li>no need to start a process in background since you seem to wait for it. Either start it without <code>start</code> and don't wait, or just wait: don't use <code>start</code>, so now you can use <code>check_call</code> instead.</li>
<li>use lists not strings for your arguments.</li>
<li>using backslashes within a string needs escaping/raw string prefix, in your case <code>\t</code> becomes "TAB char"</li>
</ol>
<p>my proposal:</p>
<pre><code>t=subprocess.check_call([r"C:\yardi_backup_v2\Extractor64.exe",
                         "-F",r"C:\yardi_backup_v2\miadmfggp_live_Full_TKDB1A112_201812180000.Lts.bak",
                         "-E",r"C:\yardi_backup_v2\tmp\extracted.bak"])
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Have you tried run instead of Popen?</p>
<pre><code>subprocess.run(['Extractor64.exe', '-F', r'C:\File1','-E' , r'C:\file2'])
</code></pre>
<p>You can read all about here: <a href="https://docs.python.org/3/library/subprocess.html#subprocess.run" rel="nofollow noreferrer">https://docs.python.org/3/library/subprocess.html#subprocess.run</a></p>
<p>It allows shell=True, etc.</p>
</div>
