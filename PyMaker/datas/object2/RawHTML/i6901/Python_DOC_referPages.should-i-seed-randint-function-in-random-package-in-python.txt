<div class="post-text" itemprop="text">
<p>I want to check if the <code>randint</code> function is already seeded. </p>
<p>Should I manually seed it? If yes, how can I do that?</p>
</div>
<div class="post-text" itemprop="text">
<p>There is no need to explicitly seed, unless you have very specific requirements. The internal <code>Random()</code> instance is automatically seeded the first time you import the module.</p>
<p>From the <a href="https://docs.python.org/3/library/random.html" rel="nofollow noreferrer">module documentation</a>:</p>
<blockquote>
<p>The functions supplied by this module are actually bound methods of a hidden instance of the random.Random class.</p>
</blockquote>
<p>and the <a href="https://github.com/python/cpython/blob/v3.6.2/Lib/random.py#L87-L94" rel="nofollow noreferrer"><code>Random.__init__()</code></a>:</p>
<blockquote>
<pre><code>class Random(_random.Random):
    # ...
    def __init__(self, x=None):
        # ...
        self.seed(x)

# ...

_inst = Random()
</code></pre>
</blockquote>
<p>so the instance calls <code>self.seed(None)</code> when the module is created. <code>None</code> means 'best available source of a seed' (which may be <code>time.time()</code>), see <a href="https://docs.python.org/3/library/random.html#random.seed" rel="nofollow noreferrer"><code>random.seed()</code></a>:</p>
<blockquote>
<p>If <em>a</em> is omitted or <code>None</code>, the current system time is used (together with the PID of the process). If randomness sources are provided by the operating system, they are used instead of the system time (see the <a href="https://docs.python.org/3/library/os.html#os.urandom" rel="nofollow noreferrer"><code>os.urandom()</code></a> function for details on availability).</p>
</blockquote>

<span class="comment-copy">Try to call <code>randint</code> without manually seedig it. If it works then you know it's already seeded.</span>
<span class="comment-copy">You should seed it with a constant in case you need the pseudo-random numbers to be repeatable for each run. Otherwise, the module will use the current time as the seed value.</span>
