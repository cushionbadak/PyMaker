<div class="post-text" itemprop="text">
<p>How can I configure multiprocessing <strong>in Windows</strong> to use the 'forkserver' method?
whenever I start up IPython console and type:</p>
<pre><code>import multiprocessing
multiprocessing.set_start_method('forkserver')
</code></pre>
<p>the error:</p>
<pre><code>ValueError: cannot find context for 'forkserver'
</code></pre>
<p>occurs.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>forkserver</code> is only available in Python 3.4+ and only on some Unix platforms (not on Windows).</p>
<p>From the <a href="https://docs.python.org/3/library/multiprocessing.html#contexts-and-start-methods" rel="nofollow noreferrer">documentation</a>:</p>
<blockquote>
<p><code>forkserver</code></p>
<blockquote>
<p><strong>Available on Unix platforms which support passing file descriptors over Unix pipes.</strong></p>
</blockquote>
<p>Changed in version 3.4: [...] <code>forkserver</code> added for <strong>some unix
  platforms</strong>.</p>
</blockquote>
<p>The reason <code>forkserver</code> is not available on Windows is that it relies on <a href="https://en.wikipedia.org/wiki/Fork_(system_call)" rel="nofollow noreferrer"><code>fork()</code></a>, and there is no <code>fork()</code> on Windows. For more information, see <a href="https://stackoverflow.com/questions/9147941/how-can-i-start-a-sub-process-in-windows">How can I start a sub-process in Windows?</a></p>
</div>
