<p>I have this C SDK that I'm working on creating a Python extension for. The C SDK documentation says that it expects a <code>ffmpeg</code> executable in it's working directory. My current project structure is:</p>
<pre><code>my-extension/
    include/
        sdk-file.h
        sdk-file-2.h
    lib/
        sdk-lib.so
        sdk-lib2.so
    src/
        my-extension.c
    setup.py
</code></pre>
<p>My setup.py:</p>
<pre><code>from setuptools import setup
from setuptools.extension import Extension

setup(
    name='my_extension',
    version='develop',
    ext_modules=[Extension(
        'my_extension',
        ['src/my-extension.c'],
        include_dirs=['include'],
        library_dirs=['lib'],
        libraries=['sdk-lib', 'sdk-lib2',],
        runtime_library_dirs=['lib'],
    )]
)
</code></pre>
<p>My question is, how do I package a executable dependency (<code>ffmpeg</code> in this case) with my extension? The executable file was provided with the SDK.</p>
</div>
<div class="post-text" itemprop="text">
<p>I think <code>package_data</code> can be used for this:</p>
<pre><code>setup(
    …
    package_data={'': ['ffmpeg']},
    …
)
</code></pre>
<p><code>package_data</code> is a dict. Keys are submodule names; an empty string means the root of the package. Values are lists that list file patterns for named submodules. See the <a href="https://docs.python.org/3/distutils/setupscript.html#installing-package-data" rel="nofollow noreferrer">docs</a>.</p>
