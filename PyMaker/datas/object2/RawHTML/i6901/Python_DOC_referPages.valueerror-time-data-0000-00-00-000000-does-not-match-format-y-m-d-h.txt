<div class="post-text" itemprop="text">
<p><code>%Y</code> matches years in the range <code>0001</code> through to <code>9999</code>, see the <a href="https://docs.python.org/3/library/datetime.html#strftime-strptime-behavior" rel="nofollow noreferrer">table in the module documentation</a>:</p>
<blockquote>
<p><code>%Y</code><br/>
  Year with century as a decimal number.<br/>
  0001, 0002, …, 2013, 2014, …, 9998, 9999</p>
</blockquote>
<p>Your year <code>0000</code> falls outside of that range.</p>
<p>The same issue applies to <code>%m</code> and <code>%d</code> (the acceptable ranges are <code>01</code> through to <code>12</code>, and <code>01</code> through to <code>31</code>, respectively). </p>
<p>That's because <code>datetime.datetime()</code> objects have the same range for their attribute values, there is no <code>datetime.datetime(0, 0, 0, 0, 0)</code> object. <code>datetime.datetime.min</code> is set to <code>datetime.datetime(1, 1, 1, 0, 0)</code> (so <code>0001-01-01 00:00:00</code>).</p>
<p>Presumably <code>0000-00-00 00:00:00</code> means 'no date available'; you'll have to alter the input to replace those dates, or handle that value to replace it with something else (perhaps <code>None</code> would be acceptable?)</p>
