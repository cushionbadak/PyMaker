<div class="post-text" itemprop="text">
<p><code>in</code> is a separate test from <code>is</code>; you <a href="https://docs.python.org/3/reference/expressions.html#comparisons" rel="nofollow noreferrer">chained the comparisons</a>, and Python executes the test as:</p>
<pre><code>(1 in dummy_list) and (dummy_list is [0])
</code></pre>
<p>Both tests fail. There is no <code>1</code> integer in <code>dummy_list</code>, only a <code>"1"</code> string object. And <code>dummy_list</code> is not the exact same object as the list produced by the <code>[0]</code> expression; the latter will always produce a new list object so the <a href="https://docs.python.org/3/reference/expressions.html#is-not" rel="nofollow noreferrer"><code>is</code> identity test</a> always fails.</p>
<p>You can't make an assertion about the <code>in</code> test; you can't test what index the containment matched. Just test the index <em>explicitly</em>:</p>
<pre><code>if dummy_list[0] == "1":
</code></pre>
<p>or if <code>dummy_list</code> could be an empty list, add a test to make sure it is not empty:</p>
<pre><code>if dummy_list and dummy_list[0] == "1":
</code></pre>
</div>
