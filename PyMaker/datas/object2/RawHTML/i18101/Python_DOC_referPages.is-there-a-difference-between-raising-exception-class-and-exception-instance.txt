<div class="post-text" itemprop="text">
<p>In Python, I can raise an Exception in two ways</p>
<pre><code>raise ValueError
raise ValueError()
</code></pre>
<p>apart from the fact that you can supply exception message in latter case, is there any fundamental difference between these two styles? Should I choose one over the other?</p>
</div>
<div class="post-text" itemprop="text">
<p>from <a href="http://docs.python.org/2/tutorial/errors.html#raising-exceptions" rel="noreferrer">the doc</a> both are valid (no unexpected behaviour):</p>
<blockquote>
<p>The sole argument to raise indicates the exception to be raised. This must be either an exception instance or an exception class (a class that derives from Exception).</p>
</blockquote>
<p>In my opinion, an instance need to be used if you want it to hold data, whether it is a message (as you said) or custom data or whatever.</p>
<p>as @alko said, if you don't give an instance it will instantiate one with no argument.</p>
<p>this won't work if you need a mandatory argument:</p>
<pre><code>&gt;&gt;&gt; class MyError(Exception):
...    def __init__(self, message, data=None):
...       self.msg = message
...       self.data = data or {}
...
&gt;&gt;&gt; raise MyError
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: __init__() takes at least 2 arguments (1 given)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>To summarize comments: there is no difference. Either syntax will throw ValueError instance. Relevant <a href="http://docs.python.org/3/reference/simple_stmts.html#the-raise-statement">excerpt from the docs</a>:</p>
<blockquote>
<p>If it is a class, the exception instance will be obtained when needed
  by instantiating the class with no arguments.</p>
</blockquote>
