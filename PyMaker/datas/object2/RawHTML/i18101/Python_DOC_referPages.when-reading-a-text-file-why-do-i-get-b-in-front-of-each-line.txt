<div class="post-text" itemprop="text">
<p>That's because you have the <code>b</code> (binary) option enabled while reading. Use the following.</p>
<pre><code>with open('a.txt', 'r') as inp:
    lines = inp.readlines()

for line in lines:
    line_list=str(line).split(',')
    session=line_list[0]


&gt;&gt;&gt; print(session)
1999
</code></pre>

<span class="comment-copy">You probably want to use the <a href="http://docs.python.org/3/library/csv.html" rel="nofollow noreferrer"><code>csv</code> module</a>, which has fairly specific requirements for how you open the file (in Python 3, you want <code>open(filename, 'r', newline="")</code>).</span>
<span class="comment-copy">I haven't had luck with the csv module or pandas because of lines like the last one, which extends with ~33 more columns.</span>
<span class="comment-copy">I think the regular <code>csv.reader</code> can handle rows with arbitrary numbers of columns, though I've not actually tried it in that context. Parsing the list it gives you for each row might be a chore, but probably no more so than you're already facing using <code>line.split(',')</code>.</span>
