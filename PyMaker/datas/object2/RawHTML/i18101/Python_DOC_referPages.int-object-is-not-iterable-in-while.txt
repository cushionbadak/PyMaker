<p>This (sequence unpacking)[http://docs.python.org/3/tutorial/datastructures.html#tuples-and-sequences]:</p>
<pre><code>x, y = z
</code></pre>
<p>does something different to what you seem to think it does.</p>
<p>It actually means this:</p>
<pre><code>x = z[0]    # The first item in z
y = z[1]    # The second element of z
</code></pre>
<p>For instance, you could do this:</p>
<pre><code>x, y, z = (1, 2, 4)
</code></pre>
<p>Then:</p>
<pre><code>&gt;&gt;&gt; x
1
&gt;&gt;&gt; y
2
&gt;&gt;&gt; z
4
</code></pre>
<p>In your case, you this doesn't work, because <code>1</code> is an integer, it doesn't have elements, hence the error.</p>
<p>Useful features of sequence unpacking combined with tuples (and the splat operator - <code>*</code>):</p>
<p>This:</p>
<pre><code>a, b = b, a
</code></pre>
<p>swaps the values of <code>a</code> and <code>b</code>.</p>
<p>Unpacking <code>range</code>, useful for constants:</p>
<pre><code>&gt;&gt;&gt; RED, GREEN, BLUE = range(3)
&gt;&gt;&gt; RED
0
&gt;&gt;&gt; GREEN
1
&gt;&gt;&gt; BLUE
2
</code></pre>
<p>The splat operator:</p>
<pre><code>&gt;&gt;&gt; first, *middle, last = 1, 2, 3, 4, 5, 6, 7, 8, 9
&gt;&gt;&gt; first
1
&gt;&gt;&gt; middle
[2, 3, 4, 5, 6, 7, 8]

9

<div class="post-text" itemprop="text">
<p>When you do <code>m, n = 1</code> this is called <a href="http://docs.python.org/2.7/tutorial/datastructures.html#tuples-and-sequences" rel="noreferrer">tuple unpacking</a>, and it works like this:</p>
<pre><code>&gt;&gt;&gt; m, n = ('a','b')
&gt;&gt;&gt; m
'a'
&gt;&gt;&gt; n
'b'
</code></pre>
<p>Since <code>1</code> is an integer not a tuple, you get this weird error; because Python cannot "step through" (or <em>iterate</em>) an integer to unpack it. That's why the error is 'int' object is <strong><em>not iterable</em></strong></p>
