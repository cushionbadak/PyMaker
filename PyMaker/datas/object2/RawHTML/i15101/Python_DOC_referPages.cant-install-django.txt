<p>I can't install Django, I read that pip is included in python 3.4, but when I write following command:</p>
<blockquote>
<p>pip install django</p>
</blockquote>
<p>this error appears:</p>
<p><code>SyntaxError: invalid syntax.</code></p>
<p>What I should do? Am I missing something? I'm using Python 3.4.2</p>
</div>
<div class="post-text" itemprop="text">
<p>From the <code>SyntaxError: invalid syntax</code> it looks like you're typing it in at the python interpreter (the <code>&gt;&gt;&gt;</code>). <code>pip install django</code> needs to be run from a terminal.</p>
<p>pip is installed when you install python but it is a program in its own right not a python package/module. Docs are <a href="https://pip.pypa.io/en/latest/index.html" rel="nofollow">here</a></p>
</div>
<div class="post-text" itemprop="text">
<p>Since Python 3.4, pip installer is included by default with the Python binary installers.</p>
<pre><code>~$ python -m pip install django
</code></pre>
<p>Read the docs:</p>
<ul>
<li><a href="https://pip.pypa.io/en/latest/installing.html#pip-included-with-python" rel="nofollow">pip included with Python</a></li>
<li><a href="https://docs.python.org/3/installing/#installing-python-modules" rel="nofollow">Installing Python Modules</a></li>
</ul>
<p>or (on Debian/Ubuntu)</p>
<pre><code>~$ apt-get install python3-pip
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>I recommend using virtualenv for you django project, which includes pip. You can install virtualenv by:</p>
<pre><code>$ curl -O https://pypi.python.org/packages/source/v/virtualenv/virtualenv-X.X.tar.gz
$ tar xvfz virtualenv-X.X.tar.gz
$ cd virtualenv-X.X
$ [sudo] python setup.py install
</code></pre>
<p>Then cd to a folder you want to create a django project:</p>
<pre><code>$ virtualenv ENV
</code></pre>
<p>then simply run</p>
<pre><code>$ source bin/activate
</code></pre>
<p>You should be able to <code>pip install django</code> from there</p>
</div>
