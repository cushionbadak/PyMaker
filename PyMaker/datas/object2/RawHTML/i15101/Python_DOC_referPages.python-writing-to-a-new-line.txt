<div class="post-text" itemprop="text">
<p><code>"{},{},{}"+"\n".format(Name, Date, Score)</code></p>
<p>You are calling the <code>format</code> method on string <code>"\n"</code>.</p>
<p>Try <code>"{},{},{}\n".format(Name, Date, Score)</code></p>
</div>
<div class="post-text" itemprop="text">
<p>When you do:</p>
<pre><code>"{},{},{}"+"\n".format(Name, Date, Score)
</code></pre>
<p>Python interprets it as:</p>
<pre><code>"{},{},{}"+("\n".format(Name, Date, Score))
</code></pre>
<p>because the method invocation  <code>.format(...)</code> <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow">has a tighter binding</a> than the <code>+</code> operator.  Meaning, it applies to just the string <code>"\n"</code> and not the combined string <code>"{},{},{}"+"\n"</code>.</p>
<p>Moreover, calling the <code>str.format</code> method on the string <code>"\n"</code> is effectively a no-op:</p>
<pre><code>&gt;&gt;&gt; "\n".format(1, 2, 3)
'\n'
&gt;&gt;&gt;
</code></pre>
<p>So, the string that is written to the file is just <code>"{},{},{}"+"\n"</code> or <code>"{},{},{}\n"</code>.</p>
<hr/>
<p>To fix the problem, simply include the newline in the format string:</p>
<pre><code>myFile.write("{},{},{}\n".format(Name, Date, Score))
</code></pre>
</div>

<pre><code>myFile.write(("{},{},{}"+"\n").format(Name, Date, Score))
</code></pre>

<pre><code>myFile.write(("{},{},{}\n").format(Name, Date, Score))
</code></pre>
<p>Then the <code>.format</code> will be applied to the whole string.</p>
