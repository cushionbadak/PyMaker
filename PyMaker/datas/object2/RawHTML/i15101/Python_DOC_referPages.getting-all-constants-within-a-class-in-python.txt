<p>I have a class which is essentially used to define common constants for other classes.  It looks something like the following:</p>
<pre><code>class CommonNames(object):
    C1 = 'c1'
    C2 = 'c2'
    C3 = 'c3'
</code></pre>
<p>And I want to get all of the constant values "pythonically".  If I used <code>CommonNames.__dict__.values()</code> I get those values (<code>'c1'</code>, etc.) but I get other things such as:</p>
<pre><code>[&lt;attribute '__dict__' of 'CommonNames' objects&gt;,
 &lt;attribute '__weakref__' of 'CommonNames' objects&gt;,
 None]
</code></pre>
<p>Which I don't want.</p>
<p>I want to be able to grab all the values because this code will be changed later and I want other places to know about those changes.</p>
</div>
<div class="post-text" itemprop="text">
<p>You'll have to filter those out explicitly by filtering on names: </p>
<pre><code>[value for name, value in vars(CommonNames).iteritems() if not name.startswith('_')]
</code></pre>
<p>This produces a list of values for any name not starting with an underscore:</p>
<pre><code>&gt;&gt;&gt; class CommonNames(object):
...     C1 = 'c1'
...     C2 = 'c2'
...     C3 = 'c3'
... 
&gt;&gt;&gt; [value for name, value in vars(CommonNames).iteritems() if not name.startswith('_')]
['c3', 'c2', 'c1']
</code></pre>
<p>For enumerations like these, you'd be better of using the <a href="https://pypi.python.org/pypi/enum34/"><code>enum34</code> backport</a> of the new <a href="https://docs.python.org/3/library/enum.html"><code>enum</code> library</a> added to Python 3.4:</p>
<pre><code>from enum import Enum

class CommonNames(Enum):
    C1 = 'c1'
    C2 = 'c2'
    C3 = 'c3'

values = [e.value for e in CommonNames]
</code></pre>
</div>
