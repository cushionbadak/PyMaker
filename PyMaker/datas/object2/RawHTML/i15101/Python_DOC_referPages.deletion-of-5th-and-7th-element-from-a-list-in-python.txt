<div class="post-text" itemprop="text">
<p>Delete items from list with</p>
<pre><code>del items[x] # x - list index, starting with 0
</code></pre>
<p>While</p>
<pre><code>del items[x:y]
</code></pre>
<p>will delete the slice of list, its not what you are looking for. So, just call <code>del</code> twice, with index needed.
To create tuple from list, just use</p>
<pre><code>new_var = tuple(items)
</code></pre>

<div class="post-text" itemprop="text">
<p>You can use a list comprehension:</p>
<pre><code>print([items[i] for i in range(len(items)) if i not in (4,6)])
</code></pre>
<p>List comprehensions are well documented on <a href="https://docs.python.org/3/tutorial/datastructures.html#list-comprehensions" rel="nofollow">python.org</a>. "A list comprehension consists of brackets containing an expression followed by a for clause, then zero or more for or if clauses."</p>
<p>The result will be a new list, so if memory matters, then simply delete the items in the right order; i.e. in reverse order:</p>
<pre><code>del items[6]
del items[4]
</code></pre>
<p>And finally in your particular case you could have done:</p>
<pre><code>del items[4:7:2]
</code></pre>
<p>The slice function is well documented on <a href="https://docs.python.org/3/library/functions.html#slice" rel="nofollow">python.org</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>You were supposed to return a tuple of the sequence given with 5th and 7th removed:</p>
<pre><code>def fifth_and_seventh_removed(lst):
    lst = list(lst)
    del lst[6]
    del lst[4]  # order matters!
    return tuple(lst)
</code></pre>
<p>If you do <code>del lst[4]</code> followed by <code>del lst[6]</code>, you will remove the fifth and eighth element, clearly (the <em>why</em> part is left as an exercise for the reader).</p>
<hr/>
<p>Alternatively for a functional approach use <code>enumerate</code>, with generator expression:</p>
<pre><code>def tuple_with_elements_removed(l, to_remove=(5, 7)): 
    """Return a copy of a iterable with certain elements removed
       (whose ordinals are in the to_remove); by default 
       5th and 7th elements are removed:"""

    to_remove = set(to_remove)
    return tuple(e for i, e in enumerate(l, 1) if i not in to_remove)
</code></pre>
