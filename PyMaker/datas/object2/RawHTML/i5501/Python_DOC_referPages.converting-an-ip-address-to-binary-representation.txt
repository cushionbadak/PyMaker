<p>I want to get the bytes representation of an ip address (big endian, 4 bytes).</p>
<p>I'm using Python 2 and I am using this module: <a href="https://github.com/phihag/ipaddress" rel="nofollow noreferrer">https://github.com/phihag/ipaddress</a></p>
<p>If I want to get the bytes representation of <code>127.0.0.1</code>, I get the expected result.</p>
<pre><code>&gt;&gt;&gt; ip_address(u'127.0.0.1').packed
'\x7f\x00\x00\x01'
</code></pre>
<p>But for <code>52.4.20.183</code>, I am getting this result:</p>
<pre><code>&gt;&gt;&gt; ip_address(u'52.4.20.183').packed
'4\x04\x14\xb7'
</code></pre>
<p>This format looks odd to me, especially the leading <code>4</code>. I was expecting it to be <code>\x34\x04\x14\xB7</code>. But apparently, both are the same:</p>
<pre><code>&gt;&gt;&gt; ipaddress.IPv4Address(b'4\x04\x14\xb7')
IPv4Address(u'52.4.20.183')
&gt;&gt;&gt; ipaddress.IPv4Address(b'\x34\x04\x14\xB7')
IPv4Address(u'52.4.20.183')
</code></pre>
<p>Why is the format for <code>52.4.20.183</code> <code>4\x04\x14\xb7</code> and where can I find more information about it?</p>
</div>
<div class="post-text" itemprop="text">
<p>The characters <code>'4'</code> and <code>'\x34'</code> are equivalent. Both represent the value 52.</p>
</div>
<span class="comment-copy">Turns out that <code>'4'</code> is the ASCII representation for the byte value 52, see <a href="http://www.asciitable.com/" rel="nofollow noreferrer">asciitable.com</a>. In other words: <code>b'4' == b'\x34' == bytes([52])</code>.</span>
<span class="comment-copy">Per its description, <code>packed</code> returns a <code>bytes</code> object, which is more or less derived from a regular <code>string</code>. See <a href="https://docs.python.org/3/library/stdtypes.html#bytes-objects" rel="nofollow noreferrer"><code>bytes</code></a> for a longer explanation. You can find more information in that official documentation as well.</span>

<span class="comment-copy">@mitchkman every character has a value. The value of the character <code>4</code> is 52 in pretty much all modern encodings (ASCII / ANSI / UTF-8 / ISO-8859-*   / etc...).</span>
<span class="comment-copy">Thanks, looking at an ASCII table solved the mystery</span>
<span class="comment-copy">The leading \x escape sequence means  that the character is interpreted as a hex digit and in hex representation 34 translates to 52(decimal).</span>
