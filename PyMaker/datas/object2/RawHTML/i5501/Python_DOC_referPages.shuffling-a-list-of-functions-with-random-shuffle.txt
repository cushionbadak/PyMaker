<p>I have some functions:</p>
<pre><code>def feeling():
    ...

def homesick():
    ...

def miss():
     ...
</code></pre>
<p>I'd like to put them in a list, shuffle them, and call each of them in succession: </p>
<pre><code>import random
prompts = [feeling, homesick, miss]
</code></pre>
<p>My idea was to call each function like this:</p>
<pre><code>random.shuffle(prompts)()
</code></pre>
<p>But this throws a </p>
<pre><code>TypeError: 'NoneType' object is not callable
</code></pre>
<p>What am I doing wrong, and how can I get this to work?</p>
</div>
<div class="post-text" itemprop="text">
<p>You have a task to choose one of these functions at random. Here's a small demo does what you're doing, but correctly.</p>
<pre><code>&gt;&gt;&gt; f = [sum, max, min]
&gt;&gt;&gt; random.shuffle(f)
&gt;&gt;&gt; f.pop()([1, 2, 3])   # looks like we picked max. Alternatively, `f[0](...)`
3 
</code></pre>
<p>Or, if it's only <em>one</em> function you want, there's no need to use <code>random.shuffle</code> at all. Use <code>random.choice</code> instead. </p>
<pre><code>&gt;&gt;&gt; random.choice(f)([1, 2, 3])
&gt;&gt;&gt; 3
</code></pre>
<hr/>
<h2>Why Your Error Occurs</h2>
<p><a href="https://docs.python.org/3/library/random.html#random.shuffle" rel="nofollow noreferrer"><code>random.shuffle</code></a> performs an <em>inplace</em> shuffling, as the docs mention. </p>
<pre><code>&gt;&gt;&gt; y = list(range(10))
&gt;&gt;&gt; random.shuffle(y)
&gt;&gt;&gt; y
[6, 3, 4, 1, 5, 8, 9, 0, 7, 2]
</code></pre>
<p>So, when you call the function, expect nothing in return. In other words, expect <code>None</code> in return. </p>
<p>Further, calling <code>()</code> on an object invokes its <code>__call__</code> method. Since <code>NoneType</code> objects do not have such a method, this errors out with <code>TypeError</code>. For an object to be callable, you'd need - </p>
<pre><code>class Foo:
    def __init__(self, x):
        self.x = x
    def __call__(self, y):
        return self.x + y
</code></pre>
<p></p>
<pre><code>&gt;&gt;&gt; f = Foo(10)
&gt;&gt;&gt; f(20)
30
</code></pre>
<p>As an exercise, try removing <code>__call__</code> and rerunning the code. Calling <code>f(20)</code> should give you the same error.</p>
