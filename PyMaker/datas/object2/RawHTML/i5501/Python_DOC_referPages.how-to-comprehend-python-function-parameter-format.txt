<blockquote>
<p>class bytearray([source[, encoding[, errors]]])</p>
</blockquote>
<p>Normally I just figure this out through examples, but I want to learn why the format of optional parameters are written in this way. Specifically:</p>
<ol>
<li><p>What should I interpret when an optional parameter is within a [] of another parameter? If [<strong>a</strong>[, <strong>b</strong>]], then what does that mean for <strong>b</strong> in relation to <strong>a</strong>? If both are independent of each other, except by order, then why is <strong>b</strong> enclosed in the bracket for <strong>a</strong>? </p></li>
<li><p>For second and other parameters, why is there a comma preceding it within the brackets? What makes it different between [<strong>a</strong>[, <strong>b</strong>]] and [<strong>a</strong>[<strong>b</strong>]] given whatever the relation between the two from the first question above?</p></li>
<li><p>How is this format advantageous or suitable in situations than the one below besides us not having to specify/write the parameter names?</p></li>
</ol>
<blockquote>
<p>class bytearray(source = default_value1, encoding = default_value2,
  errors = default_value3)</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>These are optional parameters, and there is a dependency to their 'optionality'.</p>
<p>Given <code>class bytearray([source[, encoding[, errors]]])</code>, the valid uses will be</p>
<pre><code>class bytearray()
class bytearray(source)
class bytearray(source, encoding)
class bytearray(source, encoding, errors)
</code></pre>
<p>The nested brackets indicate that, for example, you may only specify <code>encoding</code> parameter when you have already specified the <code>source</code> parameter.</p>
<p>Why is there a comma inside the bracket? Because the comma is part of the optional parameter; if you don't choose to specify the parameter then you shouldn't provide the comma.</p>
<p>How is this advantageous over three optional parameters? As above - sometimes a parameter only makes sense when another parameter is provided. In this case the library author has decided they don't want to allow you to provide the <code>encoding</code> parameter unless you have provided the <code>source</code> parameter.</p>
</div>
