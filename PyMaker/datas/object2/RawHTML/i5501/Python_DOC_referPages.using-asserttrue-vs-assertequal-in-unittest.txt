<p>In the Python <code>unittest</code> module, are there any advantages or disadvantages of using <code>assertTrue()</code> vs. <code>assertEqual()</code> in the following case?</p>
<pre><code>self.assertTrue(a == b)
self.assertEqual(a, b)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Always use <code>assertEqual()</code>, as it <em>customizes failure output</em>.</p>
<p>The method delegates to various helper methods to show you how, for example, two strings or two lists differ when the assertion fails, provided the type of both arguments match and have a type-specific helper method registered.</p>
<p><code>assertTrue()</code> can only tell you about the assertion failing, not show you why.</p>
<p>From the <a href="https://docs.python.org/3/library/unittest.html#unittest.TestCase.assertEqual" rel="noreferrer"><code>assertEqual()</code> documentation</a>:</p>
<blockquote>
<p>In addition, if <em>first</em> and <em>second</em> are the exact same type and one of <code>list</code>, <code>tuple</code>, <code>dict</code>, <code>set</code>, <code>frozenset</code> or <code>str</code> or any type that a subclass registers with <code>addTypeEqualityFunc()</code> the type-specific equality function will be called in order to generate a more useful default error message (see also the <a href="https://docs.python.org/3/library/unittest.html#type-specific-methods" rel="noreferrer">list of type-specific methods</a>).</p>
</blockquote>
<p>Only use <code>assertTrue()</code> if there is no more specific assertion available. </p>
