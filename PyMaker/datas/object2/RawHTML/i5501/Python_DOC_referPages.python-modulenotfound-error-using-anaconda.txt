<p>That error might be due to having multiple python installations on your computer:<br/>
<a href="https://docs.python.org/3/installing/#work-with-multiple-versions-of-python-installed-in-parallel" rel="nofollow noreferrer">https://docs.python.org/3/installing/#work-with-multiple-versions-of-python-installed-in-parallel</a></p>
<p>Please add the following lines somewhere to your script (or to a new script) and run it once from Spyder, and once from the console and compare the output:</p>
<pre><code>import sys
print("python: {}".format(sys.version))
# also add the following if running from python 3
from shutil import which
print(which("python"))
</code></pre>
<p>Buildingspy has to be installed using pip, I would recommend to install it using the command:</p>
<pre><code>python -m pip install -U https://github.com/lbl-srg/BuildingsPy/archive/master.zip
</code></pre>
<p>Anaconda adds an <a href="https://stackoverflow.com/questions/37993175/difference-between-command-prompt-and-anaconda-prompt">Anaconda prompt</a> to the Start menu, use that to make sure the path to python.exe is correct.</p>
<p>Once BuildingsPy is installed correctly, you will run into the problem that on Windows multiprocessing will not work from Spyder (or, from IPython/Jupyter), please also read this issue:<br/>
<a href="https://github.com/lbl-srg/BuildingsPy/issues/179" rel="nofollow noreferrer">https://github.com/lbl-srg/BuildingsPy/issues/179</a><br/>
You will have to run your script from the command line.</p>
