<div class="post-text" itemprop="text">
<p>You can simple re-assign <code>pd.Series</code> to the correct value:</p>
<pre><code>pd.Series = pd.core.series.Series
</code></pre>
<p>This only works if there is a reference to the original function/class still around (in this case there is <code>pd.core.series.Series</code>). In case you overwrite something that isn't references somewhere else this won't work.</p>
<p>Just to give another example (based on a question in the comments), suppose you overwrote <code>list</code>, you can use the <code>builtins</code> module (or the <code>__builtin__</code> module in Python 2):</p>
<pre><code>list = [1,2,3]          # overwritten list
b = list(range(10))     # throws a "TypeError: 'list' object is not callable"

import builtins
list = builtins.list    # restored
b = list(range(10))     # works
</code></pre>
<p>However often it's much easier to restart the Python interpreter (which should also work) and in some cases it might only be the only option (in case nothing else references the overwridden value.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use the <a href="https://docs.python.org/2/library/functions.html#reload" rel="nofollow noreferrer">reload</a> buildin function to reload the module. </p>
<pre><code>reload(pd)
</code></pre>
<p>Showcase:</p>
<pre><code>&gt;&gt;&gt; import pandas as pd
&gt;&gt;&gt; pd.Series
&lt;class 'pandas.core.series.Series'&gt;
&gt;&gt;&gt; pd.Series=[1,2,3]
&gt;&gt;&gt; pd.Series
[1, 2, 3]
&gt;&gt;&gt; reload(pd)
&lt;module 'pandas' from '.local/lib/python2.7/site-packages/pandas/__init__.pyc'&gt;
&gt;&gt;&gt; pd.Series
&lt;class 'pandas.core.series.Series'&gt;
</code></pre>
</div>
