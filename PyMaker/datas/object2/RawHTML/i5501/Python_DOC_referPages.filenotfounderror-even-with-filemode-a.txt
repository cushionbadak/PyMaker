<p>I have code that looks like this:</p>
<pre><code>logPath = os.path.join(os.path.normpath(os.getcwd()) + "/datei.log") + os.sep
logFile = open(logPath, "a+")
logFile.write(foo)
logFile.close()
</code></pre>
<p>This <em>should</em> open the file <code>./datei.log</code> (or create it if it doesn't exist). However, I get a <code>FileNotFoundError</code>:</p>
<pre><code>FileNotFoundError: [Errno 2] No such file or directory: '/foo/bar/datei.log/'
</code></pre>
<p>All resources I have seen indicate that <code>"a+"</code> instead of <code>"a"</code> should fix this, but this clearly fails. How can I resolve it?</p>

<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow noreferrer"><code>open</code></a> function expects a path-like object that points to a file in an existing directory. When the directory does not exist an error is raised, in my case in Python 3.6 the error is:</p>
<blockquote>
<p>NotADirectoryError: [Errno 20] Not a directory: '/Users/grr/datei.log/'</p>
</blockquote>
<p>As @cᴏʟᴅsᴘᴇᴇᴅ pointed out the trailing <code>os.sep</code> adds a '/' at the end of your file descriptor making it appear to be a path instead of a file. This is what is causing the error. </p>
</div>
<span class="comment-copy">You may want to drop the trailing <code>+ os.sep</code> from the expression.</span>
