<p>Using the <a href="https://docs.python.org/3/using/windows.html#launcher" rel="nofollow noreferrer">Python Launcher</a> works fine: <code>py.exe -3.5 helloworld.py</code></p>
<p>This also works fine to run the corresponding <code>pip</code> (which is also not in <code>PATH</code>): <code>py.exe -3.5 -m pip</code>.</p>
<p>Is there a way to make this also work for installed scripts? </p>
<hr/>
<p>Background: This allows installing multiple Python versions in parallel, and using them easily even when they are not in <code>PATH</code>. But for example I can't run <a href="http://www.pyinstaller.org/#" rel="nofollow noreferrer">pyinstaller</a>.</p>
<ul>
<li><code>C:\Python35\Scripts\pyinstaller.exe</code> works.</li>
<li><code>py -3.5 C:\Python35\Scripts\pyinstaller-script.py</code> works, but still requires me to know the installation path.</li>
<li><code>py -3.5 -m pyinstaller</code> does not work. It just prints <code>C:\Python35\python.exe: No module named pyinstaller</code>.</li>
<li><code>py -3.5 -m pyinstaller-script</code> does not work. It just prints <code>C:\Python35\python.exe: No module named pyinstaller-script</code>.</li>
</ul>

<pre><code>py -3.5 -m PyInstaller
