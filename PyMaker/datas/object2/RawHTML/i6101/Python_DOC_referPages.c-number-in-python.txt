<div class="post-text" itemprop="text">
<ul>
<li>A <code>c_uint8</code> uses 1 byte (= 8 bits).</li>
<li>A <code>c_uint16</code> uses 2 bytes (= 16 bits).</li>
<li>A <code>c_uint32</code> uses 4 bytes (= 32 bits).</li>
</ul>
<p>2 <a href="https://en.wikipedia.org/wiki/Hexadecimal" rel="nofollow noreferrer">hexadecimal digits</a> define a byte. It means the number before <a href="https://docs.python.org/3/library/stdtypes.html#printf-style-string-formatting" rel="nofollow noreferrer"><code>x</code></a> should be a quarter of the number after <code>c_uint</code>. This way, all the hexadecimal numbers will have the same length.</p>
<pre><code>print("another  : %02x" % [a c_uint8 number])
print("another1 : %04x" % [a c_uint16 number])
print("some_num : %08x" % [a c_uint32 number])
</code></pre>
<p>Here's a small test:</p>
<pre><code>&gt;&gt;&gt; from numpy import iinfo, uint8, uint16, uint32
&gt;&gt;&gt; iinfo(uint8).max
255
&gt;&gt;&gt; '1 byte : %02x' % 0
'1 byte : 00'
&gt;&gt;&gt; '1 byte : %02x' % iinfo(uint8).max
'1 byte : ff'
&gt;&gt;&gt; '2 bytes : %04x' % 0
'2 bytes : 0000'
&gt;&gt;&gt; '2 bytes : %04x' % iinfo(uint16).max
'2 bytes : ffff'
&gt;&gt;&gt; '4 bytes : %08x' % 0
'4 bytes : 00000000'
&gt;&gt;&gt; '4 bytes : %08x' % iinfo(uint32).max
'4 bytes : ffffffff'
</code></pre>
</div>

<span class="comment-copy">I'm not sure what you're asking.  The <code>%x</code> formatting is for hexadecimal.  <code>%0Nx</code> means hexadecimal left-padded with zeros to at least <code>N</code> digits (e.g. <code>01</code> instead of just <code>1</code>).  This has nothing to do with them being ctype types though.</span>
