<p>I want to find the index of two substrings in a string of characters given like this:</p>
<pre><code>find_start = '1L'
find_end = 'L'
&gt;&gt;&gt; blah = 'A1LELST5W'
&gt;&gt;&gt; blah.index('1L')
1
&gt;&gt;&gt; blah.index('L')
2  # i want it to give me 4
</code></pre>
<p>If I use the index method, it gives me the "L" that's the third character in the string. But I want it to treat "1L" and "L" as separate strings and give me the fifth character instead. </p>
<p>Is there a simple way of doing this? Or would I have to store everything except <code>find_start</code> in a new string and then try to index through that? (But that would mess with the position of everything inside the string).</p>
</div>
<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/stdtypes.html#str.index" rel="nofollow noreferrer">str.index</a> method has <em>start</em> and <em>end</em> arguments that allow you to constrain the search. So you just need to start the second search where the first one ends:</p>
<pre><code>&gt;&gt;&gt; find_start = '1L'
&gt;&gt;&gt; find_end = 'L'
&gt;&gt;&gt; blah = 'A1LELST5W'
&gt;&gt;&gt; first = blah.index('1L')
&gt;&gt;&gt; first
1
&gt;&gt;&gt; blah.index('L', first + len(find_start))
4
</code></pre>
</div>
<span class="comment-copy">Why is <code>1L</code> not a real <code>L</code>, but <code>EL</code> is?</span>
<span class="comment-copy"><code>blah</code> is a string with value and <code>1L</code> is a string of size 2. Since you want to find the index of letter <code>L</code> in string <code>blah</code>, <code>index</code> will give you the index number of the first occurrence. Hence, <code>index</code> is give location 2 as it has encountered <code>L</code> at index 2</span>
