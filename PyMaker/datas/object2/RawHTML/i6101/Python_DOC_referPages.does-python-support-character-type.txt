<p>When I am checking with data types I wonder why a char value is returning as string type.</p>
<p>Please see my input and output.</p>
<p><strong>Input:</strong></p>
<pre><code>a = 'c'

type(a)
</code></pre>
<p><strong>Output:</strong></p>

</code></pre>

<div class="post-text" itemprop="text">
<p>No.</p>
<p>Python does not have a character or <code>char</code> type. All single characters are strings with length one.</p>
</div>
<div class="post-text" itemprop="text">
<p>Simply, no.</p>
<blockquote>
<p>Since there is no separate “character” type, indexing a string produces strings of length 1.</p>
</blockquote>
<p><a href="https://docs.python.org/3/library/stdtypes.html#text-sequence-type-str" rel="nofollow noreferrer">Source</a></p>
<p>In other words there's no such thing as a <code>char</code> type in C. A single character is simply a string of one character.</p>
</div>
<div class="post-text" itemprop="text">
<p>There is a <a href="https://docs.python.org/3/library/stdtypes.html#bytes-objects" rel="nofollow noreferrer">bytes type</a> which may be analogous depending on why you are asking.</p>
<pre><code>&gt;&gt;&gt; b'a'
=&gt; b'a'
</code></pre>
<hr/>
<p><a href="https://docs.python.org/3/whatsnew/3.0.html#text-vs-data-instead-of-unicode-vs-8-bit" rel="nofollow noreferrer">https://docs.python.org/3/whatsnew/3.0.html#text-vs-data-instead-of-unicode-vs-8-bit</a></p>

<div class="post-text" itemprop="text">
<p>There is no <a href="https://docs.python.org/3/library/stdtypes.html" rel="nofollow noreferrer">built-in</a> type for character in Python, there are <code>int</code>, <code>str</code> and <code>bytes</code>. If you intend to user character, you just can go with <code>str</code> of length 1.</p>
<p>Note that Python is weakly and dynamically typed, you do not need to declare type of your variables.</p>
<p>All string you create using quote <code>'</code>, double quote <code>"</code> and triple quote <code>"""</code> are string (<a href="https://docs.python.org/3/howto/unicode.html" rel="nofollow noreferrer">unicode</a>):</p>
<pre><code>type("x")
str
</code></pre>
<p>When invoking <a href="https://docs.python.org/3/library/functions.html" rel="nofollow noreferrer">built-in</a> function <code>type</code>, it returns a <code>type</code> object representing the type of your variable:</p>
<pre><code>type(type("x"))
type
</code></pre>
<p>Integer and character do have mapping function (encoding), natively the sober map is ASCII, see <code>chr</code> and <code>ord</code> builti-in functions. </p>
<pre><code>type(123)
int

type(chr(65))
str

type(ord("x"))
int
</code></pre>
<p>If you must handle special characters that are not available in default charset, you will have to consider encoding:</p>
<pre><code>x = "é".encode()
b'\xc3\xa9'
</code></pre>
<p>The function encode, convert your sting into bytes and can be decoded back:</p>
<pre><code>x.decode()
'é'
</code></pre>
<p>Method <code>encode</code> and <code>decode</code> belongs respectively to object <code>str</code> and  <code>bytes</code>.</p>
</div>
