<p>At the moment when a combo box gets selected only the value of the combo box gets passed on to the function on_combo_activated.</p>
<p>How would I do if I wanted to send another two variables?</p>
</div>
<div class="post-text" itemprop="text">
<p>One way to solve this is by using a <code>lambda</code> function:</p>
<pre><code>self.comboBox.activated[str].connect(
    lambda text: self.on_combo_activated(text, 'str1', 'str2'))
...

def on_combo_activated(self, text, arg1, arg2):
    print(text, arg1, arg2) 
</code></pre>
<p>The <a href="https://docs.python.org/3/library/functools.html#functools.partial" rel="nofollow noreferrer">functools.partial</a> function does something similar, but it can sometimes be awkward to use it with positional arguments. If you used it to connect the above slot like this:</p>
<pre><code>self.comboBox.activated[str].connect(
    partial(self.on_combo_activated, 'str1', 'str1')
</code></pre>
<p>the slot would actually print <code>str1 str2 text</code>, because extra positional arguments are always <em>appended</em>, rather than inserted. However, with keyword arguments, it would work as expected:</p>
<pre><code>self.comboBox.activated[str].connect(
    partial(self.on_combo_activated, arg1='str1', arg2='str1'))
</code></pre>
<p>This last approach might be preferred over the others as it is self-documenting, and therefore perhaps more readable.</p>
</div>
