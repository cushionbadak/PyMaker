<p>I would like to execute an .exe file from a Python script, on Windows, with a bunch of arguments. Part of those arguments are the paths to input files and outputs. </p>
<p>My question: can I save the paths in a variable, which I then use as an argument, or does the call take only the name of the argument instead of its value?</p>
<p>My idea:</p>
<pre><code>outputPath = C:\...\folder1\folder2
inputpath = D:\...\folder1\folder2
subprocess.call('runnable.exe', outputPath, inputPath)
</code></pre>
<p>instead of </p>
<pre><code>subprocess.call('runnable.exe', C:\...\folder1\folder2, D:\...\folder1\folder2)
</code></pre>
