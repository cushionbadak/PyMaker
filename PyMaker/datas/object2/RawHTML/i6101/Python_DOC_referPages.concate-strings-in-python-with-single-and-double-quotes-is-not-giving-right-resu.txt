<p>This has nothing to do with how the string actually looks like; try printing it:</p>
<pre><code>print(a+c)
</code></pre>
<p>It is simply a matter of how Python <em>represents</em> the string in an interactive session (or when <code>repr()</code>d).</p>
<p>See also: <a href="https://docs.python.org/3/library/functions.html#repr" rel="nofollow noreferrer">Built-in Functions: <code>repr()</code></a> in the Python docs.</p>
