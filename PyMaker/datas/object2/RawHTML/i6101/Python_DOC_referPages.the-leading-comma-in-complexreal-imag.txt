<p>Bear with me but I'm going to cite a bit here (emphasis mine):</p>
<blockquote>
<h3>dir(<em>[object]</em>)</h3>
<p><strong>Without arguments</strong>, return the list of names in the current local scope. <strong>With an argument</strong>, attempt to return a list of valid attributes for that object. (<a href="https://docs.python.org/3/library/functions.html#dir" rel="nofollow noreferrer">Python 3.6.3 Documentation source</a>)</p>
</blockquote>
<p>The above notation for the Python documentation implies that the parameter, <code>object</code>, is <em>optional</em> and supplying it affects the behavior of the method (as described in <code>dir()</code>'s description).</p>
<p>As for your example of <code>complex([real[, imag]])</code>, the syntax indicates that it is valid for you to supply <em>none, only <code>real</code>, or both</em> of the arguments. The behaviors of the method under those circumstances are also listed in the method's description (emphasis also mine):</p>
<blockquote>
<p>...<strong>If <code>imag</code> is omitted</strong>, it defaults to zero and the constructor serves as a numeric conversion like int and float. <strong>If both arguments are omitted</strong>, returns <code>0j</code>. (<a href="https://docs.python.org/3/library/functions.html#complex" rel="nofollow noreferrer">PyDocs source</a>)</p>
</blockquote>
<p>However, be careful as this is not the same as <code>complex([real, imag])</code>. <code>complex([real, imag])</code> only implies that you can supply <em>either none or both</em> of the arguments, not any single one of them.</p>
<p>Hope this helped!</p>
