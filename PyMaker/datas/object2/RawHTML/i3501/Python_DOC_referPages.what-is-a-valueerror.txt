<div class="post-text" itemprop="text">
<p>Take a look at <a href="https://docs.python.org/3/library/math.html#math.sqrt" rel="nofollow noreferrer"><code>math#sqrt</code></a> - it takes a number, but this number must be non-negative. If you try calling it with a negative number (which can't be done in real numbers' math), you'll get a <code>ValueError</code>:</p>
<pre><code>&gt;&gt;&gt; from math import sqrt
&gt;&gt;&gt; sqrt(-1)
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
ValueError: math domain error
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>An example that gives the specific <code>ValueError</code> is this:</p>
<pre><code>import 
math.log(0)
</code></pre>
<p>which gives back:</p>
<pre><code>Traceback (most recent call last):

  File "&lt;ipython-input-6-f7278b7c2ed1&gt;", line 1, in &lt;module&gt;
    math.log(0)

ValueError: math domain error
</code></pre>
<p>Many other math function will produce the same when given bad inputs (<code>math.sqrt(-1)</code>, <code>math.ceil(math.nan)</code> and so on...)</p>
</div>
