<p>I'm prepping some data to insert into MySQL. Unfortunately some of the data is in the form <code>100½</code>. I need to convert this to a float, but obviously <code>float(100½)</code> fails. </p>
<p>Is there a method in Python that can handle this conversion? I'm not opposed to hard coding something, but I may encounter <code>⅝</code> or other fractions in the future. </p>
</div>
<div class="post-text" itemprop="text">
<pre><code>&gt;&gt;&gt; float(sum(fractions.Fraction(x) for x in unidecode.unidecode('100½').split()))
100.5
</code></pre>
<p><a href="https://docs.python.org/3/library/fractions.html" rel="nofollow noreferrer"><code>fractions</code></a></p>
<p><a href="https://pypi.org/project/Unidecode/" rel="nofollow noreferrer"><code>unidecode</code></a></p>

<div class="post-text" itemprop="text">
<pre><code>from unidecode import unidecode
from fractions import Fraction
a = unidecode('100½').split()
print(float(int(a[0])+Fraction(a[1])))
</code></pre>
