<p>The documentation from JetBrains says, that there are two formats for creating reStructuredText <a href="https://www.jetbrains.com/help/pycharm/2017.1/using-docstrings-to-specify-types.html" rel="nofollow noreferrer">comments with type-hinting</a>:</p>
<pre><code>:param param_type param_name: parameter description (type description is on the same line as the parameter description).
:type param_name: param_type (type description is on a separate line)
</code></pre>

<div class="post-text" itemprop="text">
<p>It's in Settings -&gt; Tools -&gt; Python Integrated Tools -&gt; Docstrings -&gt; Docstring format. </p>
<p><a href="https://i.stack.imgur.com/zeHom.png" rel="nofollow noreferrer"><img alt="enter image description here" src="https://i.stack.imgur.com/zeHom.png"/></a></p>
<p>If I understand correctly the format you're after is the NumPy one.</p>
<p>Also, if you're using Python&gt;=3.5, you may already know about type hints in variables and function parameters, which looks super tidy and save you space in the docstring: <a href="https://docs.python.org/3/library/typing.html" rel="nofollow noreferrer">https://docs.python.org/3/library/typing.html</a></p>
<p>And sphinx supports them with one of:</p>
<ul>
<li><a href="https://github.com/agronholm/sphinx-autodoc-typehints" rel="nofollow noreferrer">https://github.com/agronholm/sphinx-autodoc-typehints</a></li>
<li><a href="https://github.com/nicolashainaux/sphinx-autodoc-annotation" rel="nofollow noreferrer">https://github.com/nicolashainaux/sphinx-autodoc-annotation</a></li>
</ul>
