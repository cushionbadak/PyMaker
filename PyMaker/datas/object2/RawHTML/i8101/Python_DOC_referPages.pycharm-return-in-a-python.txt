<p>When you use Pycharm to edit a <code>py</code> file, it inserts a template like this.</p>
<pre><code>def check_caller_log():
    """
    xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    :return: 
    """
</code></pre>
<p>What's the purpose of <code>:return:</code> here?</p>
</div>
<div class="post-text" itemprop="text">
<p>It describes the type of what your function returns (for example if it returns a string, you write <code>return: str</code>). Also you can add a comment about what this return value contains or what its purpose is. </p>
<pre><code>def my_function():
    """
    Some good explanation of what this function does

    :return: str
    """
</code></pre>
<p>Note: If your function doesn't return anything, it actually returns None and you can write it down: <code>return: None</code></p>
<pre><code>def my_function():
    """
    Some good explanation of what this function does

    :return: None
    """
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As stefan.stt's answer says, the <code>:return:</code> field lets you provide a description of the fuction's return value. However, stefan.stt's example shows the <code>:return:</code> being used to document the return <em>type</em> rather than to describe the value. To document the type, it's more usual to use the <code>:rtype:</code> field, thus:</p>
<pre class="lang-python prettyprint-override"><code>def user_name():
    """
    Description of the function goes here.

    :return: the name of the currently active user
    :rtype: str
    """
</code></pre>
<p>The field names are not fully standardizes, and <code>:returns:</code> is also widely used to document the return value; it's my impression that <code>:returns:</code> is the more commonly used of the two synonyms, but PyCharm's  "Python Integrated Tools" has standardized on <code>:result:</code>. The widely used Python documentation generator Sphinx <a href="http://www.sphinx-doc.org/en/master/usage/restructuredtext/domains.html#info-field-lists" rel="nofollow noreferrer">supports both variants</a>, but other documentation tools may not.</p>
<p>It's also worth noting that there are several different syntaxes in use to document parameters, return values, etc. in Python docstrings. The most common ones are documented in <a href="https://stackoverflow.com/a/24385103/6947739">this answer</a>.</p>
