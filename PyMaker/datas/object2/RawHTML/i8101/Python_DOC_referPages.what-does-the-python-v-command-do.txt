<p>This will show you the way your modules are loading for debugging reasons. (in this case it is just the base modules)</p>
<p>Documentation: <a href="https://docs.python.org/2/using/cmdline.html#cmdoption-v" rel="nofollow noreferrer">https://docs.python.org/2/using/cmdline.html#cmdoption-v</a></p>
</div>
<div class="post-text" itemprop="text">
<p>You didn't do anything wrong. From the man page for Python (accessed by <code>man python</code>):</p>
<pre><code>   -v     Print  a  message each time a module is initialized, showing the
          place (filename or built-in module) from  which  it  is  loaded.
          When  given twice, print a message for each file that is checked
          for when searching for a module.  Also provides  information  on
          module cleanup at exit.

   -V ,  --version
          Prints  the  Python  version number of the executable and exits.
          When given twice, print more information about the build.
</code></pre>
<p>Think of the lowercase <code>-v</code> as "verbose"—it's useful for debugging modules. The modules you listed are all from the standard library.</p>
