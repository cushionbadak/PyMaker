<div class="post-text" itemprop="text">
<p><code>cp949</code> is the default locale for your Windows system, and that's what <code>open()</code> defaults to. From the <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow noreferrer"><code>open()</code> documentation</a>:</p>
<blockquote>
<p><em>encoding</em> is the name of the encoding used to decode or encode the file. This should only be used in text mode. The default encoding is platform dependent (whatever <code>locale.getpreferredencoding()</code> returns), but any text encoding supported by Python can be used.</p>
</blockquote>
<p>Specify a different codec when opening the file:</p>
<pre><code>wfile = open(os.getcwd()+"/test1.txt", mode='w', encoding='utf8')   
</code></pre>
<p>Note that you don't need to pre-pend <code>os.getcwd()</code> when opening a file without a path, the default is to use the working directory for relative paths:</p>
<pre><code>wfile = open("test1.txt", mode='w', encoding='utf8')   
</code></pre>
<p>You'd be better off using <code>os.path.join()</code> to build paths for everything else.</p>
<p>Your code can otherwise be simplified further with <code>enumerate()</code> and a context manager. The <code>data</code> dictionary is not really useful here, just reference <code>tweet.text</code> everywhere:</p>
<pre><code>with open(os.getcwd()+"/test1.txt", mode='w') as wfile:
    for i, tweet in enumerate(c.items()):
        print("{}: {}".format(i, tweet.text))
        wfile.write(tweet.text + '\n')
        time.sleep(0.5)
</code></pre>
