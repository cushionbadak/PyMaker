<p>I have a question concerning <code>int()</code>. Part of my Python codes looks like this</p>
<pre><code>string = input('Enter your number:')
n = int(string)
print n
</code></pre>
<p>So if the input string of <code>int()</code> is not a number, Python will report ValueError and stop running the remaining codes.</p>
<p>I wonder if there is a way to make the program re-ask for a valid string? So that the program won't just stop there.</p>
<p>Thanks!</p>

<div class="post-text" itemprop="text">
<p>What you're looking for is<h1>Try / Except</h1></p>
<p><strong>How it works:</strong></p>
<pre><code>try:
    # Code to "try".
except:
    # If there's an error, trap the exception and continue.
    continue
</code></pre>
<p><strong>For your scenario:</strong></p>
<pre><code>def GetInput():
    try:
        string = input('Enter your number:') 
        n = int(string) 
        print n
    except:
        # Try to get input again.
        GetInput()
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>try except</code></p>
<pre><code>while True:
    try:
        string = input('Enter your number:')
        n = int(string)
        print n
        break
    except ValueError:
        pass
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Put the whole thing in an infinite loop. You should catch and ignore <code>ValueError</code>s but break out of the loop when you get a valid integer. </p>
</div>
<div class="post-text" itemprop="text">
<pre><code>n = None
while not isinstance(n, int):
    try:
        n = int(input('Enter your number:'))
    except:
        print('NAN')
</code></pre>
