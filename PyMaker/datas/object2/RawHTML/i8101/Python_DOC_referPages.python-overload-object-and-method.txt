<p>I'm writing a class and want to overload the __and__ function</p>
<pre><code>class Book(object):
    def __init__(self, name, pages):
        self.name = name
        self.pages = pages

    def __and__(self, other):
        return '{}, {}'.format(self.name, other.name)
</code></pre>
<p>When I run this</p>
<pre><code>Book('hamlet', 50) and Book('macbeth', 60)
</code></pre>
<p>I would expect to get 'hamlet, macbeth'</p>
<p>However, it appears the overload does nothing.
What am I doing wrong?</p>
</div>
<div class="post-text" itemprop="text">
<p>The <code>__and__</code> method is an override for the and operator <code>&amp;</code>:</p>
<pre><code>&gt;&gt;&gt; Book('hamlet', 50) &amp; Book('macbeth', 60)
'hamlet, macbeth'
</code></pre>
<p>Sadly, you <a href="https://stackoverflow.com/q/471546/182402">can not override the <code>and</code> operator</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>The <code>__and__</code> method is actually grouped with the <a href="https://docs.python.org/3/reference/datamodel.html#emulating-numeric-types" rel="nofollow noreferrer">numeric type methods</a>, therefore it does not represent <em>logical</em> and (which is the <code>and</code> keyword) but rather the <code>&amp;</code> operator</p>
<pre><code>&gt;&gt;&gt; Book('hamlet', 50) &amp; Book('macbeth', 60)
'hamlet, macbeth'
</code></pre>
</div>
