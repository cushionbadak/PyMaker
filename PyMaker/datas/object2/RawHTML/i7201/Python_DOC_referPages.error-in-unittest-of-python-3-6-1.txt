<div class="post-text" itemprop="text">
<p>In all likelihood you have a file that is named <code>operator.py</code> in your directory. Unfortunately, that name conflicts with Python's standard library, which has an <a href="https://docs.python.org/3/library/operator.html" rel="nofollow noreferrer"><code>operator</code></a> module.</p>
<p>In fact, look at the last two lines, one of which says:</p>
<pre><code>from operator import itemgetter as _itemgetter, eq as _eq
    ImportError: cannot import name 'itemgetter'
</code></pre>
<p>Which suggests it tries to get <code>itemgetter</code> from <code>operator</code>, but since you have an <code>operator.py</code> file in your working directory, it tries to import <code>itemgetter</code> from that file, not Python's standard library.</p>
<p>Try to rename your <code>operator.py</code> to something else and see what happens.</p>
