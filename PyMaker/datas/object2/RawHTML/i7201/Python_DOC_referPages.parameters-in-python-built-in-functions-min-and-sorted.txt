<div class="post-text" itemprop="text">
<p>The following applies to Python 3 (to which the OP linked).</p>
<hr/>
<p>The single star (<code>*</code>) in the function definition of <a href="https://docs.python.org/3/library/functions.html#sorted" rel="nofollow noreferrer"><code>sorted</code></a> denotes that all following named parameters must be supplied as keyword arguments (<em>"keyword-only arguments"</em>, see <a href="https://www.python.org/dev/peps/pep-3102/" rel="nofollow noreferrer">PEP 3102</a>). That means the following won't work:</p>
<pre><code>&gt;&gt;&gt; sorted([1, 2, 3], lambda x: x)
[...]
TypeError: must use keyword argument for key function
</code></pre>
<p>Instead you <em>must</em> supply those parameters via keyword arguments:</p>
<pre><code>&gt;&gt;&gt; sorted([1, 2, 3], key=lambda x: x)
[1, 2, 3]
</code></pre>
<hr/>
<p>I'm not completely sure what the <code>, *[, key, default]</code> syntax shall denote (it's not valid Python but a documentation purpose) however I think the developers want to express the same thing, namely that you have to provide those parameters as keyword arguments (because <code>min</code> and <code>max</code> can take an arbitrary number of positional arguments that is the only way to do it).</p>
</div>
<div class="post-text" itemprop="text">
<p>The <code>[, key, default]</code> part is not Python syntax, it's general programming languages grammar syntax denoting optional parts (here optional arguments).</p>
<p>EDIT: the <code>*</code> part is explained in a_guest's answer (nb : this is for Python 3 only)</p>
