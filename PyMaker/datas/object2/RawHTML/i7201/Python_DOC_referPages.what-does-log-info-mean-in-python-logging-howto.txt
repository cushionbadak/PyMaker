<p>I read the <a href="https://docs.python.org/3/howto/logging.html#logging-to-a-file" rel="nofollow noreferrer">Python Logging HOWTO</a>, and there is a description I don't understand:</p>
<blockquote>
<p>If you want to set the logging level from a command-line option such
  as:</p>
<pre><code>--log=INFO
</code></pre>
<p>and you have the value of the parameter passed for <code>--log</code> in some variable <em>loglevel</em>, you can use:</p>
<pre><code>getattr(logging, loglevel.upper())
</code></pre>
<p>to get the value which you’ll pass to <a href="https://docs.python.org/3/library/logging.html#logging.basicConfig" rel="nofollow noreferrer"><code>basicConfig()</code></a> via the <em>level</em> argument.</p>
</blockquote>
<p>What does it mean? There is no example, it would be nice if someone could give one.</p>
<p>Can I set the logging level like this?</p>
<pre><code>logging --log=INFO
</code></pre>

<div class="post-text" itemprop="text">
<p>The assumption is that you are using something like <code>argparse</code> to define command line arguments:</p>
<pre><code>import argparse
p = argparse.ArgumentParser()
p.add_argument("--log")

args = p.parse_args()
loglevel = args.log
</code></pre>
<p>The argument to the <code>--log</code> option should then be one of the level constants defined in the <code>logging</code> module; <code>getattr(logging, loglevel.upper())</code> would then be a way to get that value for a string.</p>
