<span class="comment-copy">This has nothing to do with SQLite. Your Python source code file contains a non-ASCII character, requiring you to declare an encoding. Follow the link in the error message, or remove that character.</span>
<span class="comment-copy">You appear to have a zero-width space and a zero-width joiner  between the word <code>created</code> and <code>_date</code>. This is usually inserted by Stack Overflow in long comments to ensure they break across lines.</span>
<span class="comment-copy">You probably copied and pasted a Stack Overflow comment into your code. select the <code>created_date</code> part of your code, delete it and manually re-type it to ensure you removed the invisible extra characters.</span>
<span class="comment-copy">Next, <b>do not use <code>str.format()</code></b> to insert values. You are opening yourself up to a SQL injection attack. Pass in the values as SQL parameters instead. Read the intro of the <a href="https://docs.python.org/3/library/sqlite3.html" rel="nofollow noreferrer"><code>sqlite3</code> module</a> documentation carefully. Use <code>?</code> placeholders, and pass your values to the <code>cursor.execute()</code> method <i>separately</i>.</span>
