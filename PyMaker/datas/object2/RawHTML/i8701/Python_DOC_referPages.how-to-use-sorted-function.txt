<div class="post-text" itemprop="text">
<p>As mentioned in <a href="https://docs.python.org/2/library/functions.html#sorted" rel="nofollow noreferrer">the <code>sorted</code> docs</a>, you need to pass your comparison function as a <code>cmp</code> argument not a <code>key</code> argument. So </p>
<pre><code>sorted(some_list, cmp=cmp_ignore_case)
</code></pre>
<p>However, using <code>cmp</code> is <em>much</em> less efficient than <code>key</code>, and so <code>cmp</code> has been deprecated for some time in Python 2, and it has been removed from Python 3.</p>
<p>A <code>key</code> function only gets called once for each unique item in the list (or other iterable) that you're sorting. In contrast, a <code>cmp</code> function <em>must</em> be called every time a comparison is made between two items.</p>
<p>It's possible to transform a <code>cmp</code> function into a <code>key</code> function with <a href="https://docs.python.org/3/library/functools.html#functools.cmp_to_key" rel="nofollow noreferrer"><code>functools.cmp_to_key</code></a>, but this is only provided for convenience: it doesn't prevent the inefficiency, it only hides it. So it should only be used as a temporary fix for old code that uses <code>cmp</code>, or in those rare cases where it's not possible to use a <code>key</code> function.</p>
<p>For further details, please see the <a href="https://docs.python.org/2/library/functions.html#sorted" rel="nofollow noreferrer">Sorting HOW TO</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>It's right in the documentation you show :</p>
<pre><code>&gt;&gt;&gt; sorted(['bob', 'about', 'Zoo', 'Credit'], key=str.upper)
['about', 'bob', 'Credit', 'Zoo']
&gt;&gt;&gt; sorted(['bob', 'about', 'Zoo', 'Credit'], key=str.lower)
['about', 'bob', 'Credit', 'Zoo']
</code></pre>
<p>The <code>key</code> function should only accept one argument : it is called for every single string automatically by <code>sorted</code>. Your function expects 2 arguments, and <code>sorted</code> doesn't know what <code>s2</code> should be.</p>
<p>So your code should be :</p>
<pre><code>def ignore_case(just_one_string):
    return just_one_string.upper()

print(sorted(['bob', 'about', 'Zoo', 'Credit'], key=ignore_case))
# ['about', 'bob', 'Credit', 'Zoo']
</code></pre>
<p>The confusion could come from the fast that in some other languages (e.g. Java), the method expects 2 arguments (see <a href="https://docs.oracle.com/javase/7/docs/api/java/util/Comparator.html" rel="nofollow noreferrer"><code>Comparator</code></a>)</p>
</div>
<div class="post-text" itemprop="text">
<p>In order to use a custom comparator function for Python 3's <code>sorted</code>, you must use <code>functools.cmp_to_key</code>, like so:</p>
<pre><code>from functools import cmp_to_key
sorted(['bob', 'about', 'Zoo', 'Credit'], key=cmp_to_key(cmp_ignore_case))
</code></pre>
<p>For further information, please refer to the documentation for <code>functools.cmp_to_key</code>: <a href="https://docs.python.org/3/library/functools.html#functools.cmp_to_key" rel="nofollow noreferrer">https://docs.python.org/3/library/functools.html#functools.cmp_to_key</a></p>
</div>
