<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/distutils/packageindex.html#the-upload-command" rel="nofollow noreferrer">Distutils <code>upload</code> command</a>, which uploads the distribution to PyPI, can specify to first sign the distribution and upload that signature:</p>
<blockquote>
<p>You can use the <code>--sign</code> option to tell <em>upload</em> to sign each uploaded file
  using GPG (GNU Privacy Guard) [â€¦] You can also specify which key to use for
  signing using the <code>--identity=name</code> option.</p>
</blockquote>
<p>How can I specify that, for a particular distribution, every upload should be signed with a particular key?</p>
</div>
<div class="post-text" itemprop="text">
<p>The Distutils commands can be <a href="https://docs.python.org/3/distutils/configfile.html" rel="nofollow noreferrer">configured with a <code>setup.cfg</code> file</a>, by putting option values in a section named for the command.</p>
<p>So, to specify <code>sign</code> and <code>identity</code> options for the <code>upload</code> command, write in the distribution's <code>setup.cfg</code> file:</p>
<pre><code>[upload]

# Sign distributions, and upload the signing public key?
sign = true

# Which GnuPG identity to use for signing?
identity = christina@example.org
</code></pre>
</div>
