<p><code>-3&lt;-2&lt;-1</code> returns <code>True</code>.</p>
<p>However I would expect it interpreted as </p>
<pre><code>(-3&lt;-2)&lt;-1
True&lt;-1
1&lt;-1
False
</code></pre>
<p>How is that possible ?</p>
</div>
<div class="post-text" itemprop="text">
<p>This is a <a href="https://docs.python.org/3/reference/expressions.html#comparisons" rel="noreferrer">chained comparison</a>. Instead of being left-associative like <code>(-3 &lt; -2) &lt; -1</code> or right-associative like <code>-3 &lt; (-2 &lt; -1)</code>, it's actually treated as</p>
<pre><code>(-3 &lt; -2) and (-2 &lt; -1)
</code></pre>
<p>except that <code>-2</code> is evaluated at most once.</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/2/reference/expressions.html#comparisons" rel="nofollow noreferrer">From the docs</a>:</p>
<blockquote>
<p>Unlike C, expressions like <code>a &lt; b &lt; c</code> have the interpretation that is conventional in mathematics</p>
<p>Comparisons can be chained arbitrarily, e.g., <code>x &lt; y &lt;= z</code> is equivalent
  to <code>x &lt; y and y &lt;= z</code>, except that <code>y</code> is evaluated only once (but in both
  cases <code>z</code> is not evaluated at all when <code>x &lt; y</code> is found to be <code>false</code>).</p>
</blockquote>
<p>Therefore </p>
<pre><code>-3 &lt; -2 &lt; -1  
</code></pre>
<p>is equivalent to </p>
<pre><code>-3 &lt; -2 and -2 &lt; -1  # where -2 is evaluated only once
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>It's stated in the <a href="https://docs.python.org/2/reference/expressions.html#comparisons" rel="nofollow noreferrer">documentation</a> that it is part of the language</p>
</div>
<span class="comment-copy">This is how Python works. One of the many great features of this language if you ask me.</span>
