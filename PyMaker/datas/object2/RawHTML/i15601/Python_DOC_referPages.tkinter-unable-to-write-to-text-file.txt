<p>Two problems here:</p>
<ol>
<li><p>You are not closing the file when you are done with it.  Some systems require you to do this in order to commit the changes.  Either call <code>dataFile.close()</code> at the end of your <code>write</code> function or use a <a href="https://docs.python.org/3/reference/compound_stmts.html#the-with-statement" rel="nofollow noreferrer">with-statement</a> to open the file (which will automatically close it when you are done):</p>
<pre><code>def write(x=fin_btn_text):
    click(x)
    with open("StoredRecipes.txt", "w") as dataFile:
        dataFile.write(str(input_item, ) + "\n")
    new_recipe.destroy()  # Remember to call this
</code></pre></li>
<li><p>As @Kevin noted in <a href="https://stackoverflow.com/questions/27281291/python-tkinter-unable-to-write-to-text-file/27281374#comment43031254_27281291">a comment</a>, you cannot call <code>.grid</code> on the same line as you create a widget.  The <code>.grid</code> method works in-place and always returns <code>None</code>.  Thus, it should be called on its own line after you create the widget:</p>
<pre><code>itemtext = Label(new_recipe, text="Item")
itemtext.grid(row=0, column=0)
</code></pre></li>
</ol>
</div>
<span class="comment-copy">I don't know if this is specifically the cause of your problem, but you must not create a Tkinter widget, grid it, and assign the result all on one line. All of your itemtext, input_item, quantitytext, input_quantity, unittext, input_unit variables are None. Create and assign on one line, then grid on a second line.</span>
<span class="comment-copy">Also, you are opening the file in write mode.  This will truncate it (erase what it contained originally).  Are you sure you don't want append mode (<code>open("StoredRecipes.txt", "a")</code>)?</span>
