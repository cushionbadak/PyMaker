<p>I am getting values from some source using which I need to create a dictionary of dictionary.
To solve such a task in PHP I would just write a <code>for</code> loop and:</p>
<pre><code>$arr[ifIndex][$key] = $val
</code></pre>
<p>I do not need to bother if  a <code>$key</code> value really exists in my associative array - it will be created if it isn't.
In Python though you get a key error so I have to check that and add a dictionary if it isn't there:</p>
<pre><code>if ifIndex in data:
     data[ifIndex][entry] = val.prettyPrint()
else:
     data[ifIndex] = { entry: val.prettyPrint() }
</code></pre>
<p>To me that looks very ugly and I think there is way of doing that as simple as in a PHP example.</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>a = collections.defaultdict(dict)
a['b']['c'] = 'd'
</code></pre>
<p>returns </p>
<pre><code>defaultdict(&lt;type 'dict'&gt;, {'b': {'c': 'd'}})
</code></pre>
</div>
<span class="comment-copy">Look into using <a href="https://docs.python.org/3/library/collections.html#collections.defaultdict" rel="nofollow noreferrer"><code>collections.defaultdict</code></a>, it was made just for this purpose (among a few other nifty features).</span>
