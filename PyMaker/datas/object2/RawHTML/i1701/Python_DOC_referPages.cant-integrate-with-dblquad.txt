<p>So I want to integrate a double integral with constants in it, like a, b, etc where the user can asign the value of this constants:</p>
<p>The limits of the integral are x[0,1] and y[-1,2]</p>
<pre><code>import numpy as np
import scipy.integrate as integrate

def g(y,x,a):
    return a*x*y

a = int(input('Insert a value --&gt; '))
result = integrate.dblquad(g, 0, 1, lambda x: -1, lambda x: 2, args=(a))[0]
print(result)
</code></pre>
<p>But I get this error and I don't understant why:</p>
<pre><code>TypeError: integrate() argument after * must be an iterable, not int
</code></pre>
<p>I don't understand it. Because when I do the same but with quad() Python does it correctly:</p>
<pre><code>import numpy as np

    return a*x

</code></pre>

<div class="post-text" itemprop="text">
<p>The problem here is that the value you provide in the optional argument args is a tuple. In the case of <a href="https://docs.scipy.org/doc/scipy/reference/generated/scipy.integrate.quad.html" rel="nofollow noreferrer">quad</a>, it is what the function expects, but for <a href="https://docs.scipy.org/doc/scipy/reference/generated/scipy.integrate.dblquad.html" rel="nofollow noreferrer">dblquad</a>, a sequence is required. Even though tuples are <a href="https://docs.python.org/3/library/stdtypes.html#typesseq" rel="nofollow noreferrer">sequences</a> (immutable ones), it seems that scipy makes a difference here and therefore it is why an error is raised. However it is misleading as a tuple is definitely an <a href="https://docs.python.org/3/glossary.html#term-iterable" rel="nofollow noreferrer">iterable</a>. Anyway, this should work:</p>
<pre><code>result = integrate.dblquad(g, 0, 1, lambda x: -1, lambda x: 2, args=[a])[0]
</code></pre>
</div>
<span class="comment-copy"><i>"The problem here is that the value you provide in the optional argument args is a tuple. "</i>  Actually, <code>(a)</code> is not a tuple; it is exactly the same as <code>a</code>.  To make a tuple of length 1 containing <code>a</code>, use <code>(a,)</code>.  (Your solution works too.)</span>
