<div class="post-text" itemprop="text">
<p><code>&lt;__main__.Dinosaurus object at 0x026B5210&gt;</code> is not an error – it's the <em>representation</em> of your object.</p>
<p>A little experimentation shows that <code>Dinosaur</code> has all the attributes you assigned it:</p>
<pre><code>&gt;&gt;&gt; Dinosaur = Dinosaurus ("Stegosaurus", "20", "sinine", "Liha")
&gt;&gt;&gt; Dinosaur
&lt;__main__.Dinosaurus object at 0x7f893d9b5320&gt;
&gt;&gt;&gt; Dinosaur.nimi
'Stegosaurus'
&gt;&gt;&gt; Dinosaur.suurus
'20'
&gt;&gt;&gt; Dinosaur.värv
'sinine'
&gt;&gt;&gt; Dinosaur.toit
'Liha'
</code></pre>
<p>If you want to change the representation that gets shown when you print instances of your class, you need to write a special <a href="https://docs.python.org/3/reference/datamodel.html#object.__repr__" rel="nofollow noreferrer"><code>__repr__</code></a> method to tell Python how they should be represented, for example:</p>
<pre><code>class Dinosaurus:

    def __init__(self, nimi, suurus, värv, toit):
        self.nimi = nimi
        self.suurus = suurus
        self.värv = värv
        self.toit = toit

    def __repr__(self):
        return 'Dinosaurus(%r, %r, %r, %r)' % (self.nimi, self.suurus, self.värv, self.toit)
</code></pre>
<p></p>
<pre><code>&gt;&gt;&gt; Dinosaur = Dinosaurus ("Stegosaurus", "20", "sinine", "Liha")
&gt;&gt;&gt; print(Dinosaur)
Dinosaurus('Stegosaurus', '20', 'sinine', 'Liha')
</code></pre>
<p>There's also a <a href="https://docs.python.org/3/reference/datamodel.html#object.__str__" rel="nofollow noreferrer"><code>__str__</code></a> method you can define, which Python will use in preference to <code>__repr__</code> when printing objects if both are defined. You should think of <code>__str__</code> as returning an informal, human-friendly representation of an object, and <code>__repr__</code> as the formal representation (which, if possible, should be a valid Python expression that could be used to make an exact copy of it, as in my example above).</p>
