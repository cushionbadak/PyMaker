<div class="post-text" itemprop="text">
<p>If you want to print lines which contain <code>'@Grab'</code> or <code>'@ Grab'</code>, you could use:</p>
<pre><code>test = page_parse[i].text
if ('@Grab' in test) or ('@ Grab' in test):
    print(test)
</code></pre>
<p>Or, if you have many strings you wish to test for, it might be easier to put them in a list like this:</p>
<pre><code>test = page_parse[i].text
if any(item in test for item in ['@Grab', '@ Grab']):
    print(test)
</code></pre>
<hr/>
<p>The reason why the condition <code>var or var2 in test</code> is always <code>True</code> is because Python parses this expression as <code>(var) or (var2 in test)</code> (due to <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer">operator precedence</a>, since <code>in</code> binds more "tightly" than <code>or</code>). Since <code>var</code> is a non-empty string it evaluates as <code>True</code> in a boolean context. So </p>
<pre><code>(var) or (var2 in test)
</code></pre>
<p>is equivalent to</p>
<pre><code>True or (var2 in test)
</code></pre>
<p>which always evaluates as <code>True</code>.  </p>

<span class="comment-copy">Notice that <code>'@Grab' not in '@ Grab'</code> is <code>True</code> -- because of the space between <code>@</code> and <code>Grab</code>. If you want to print lines that contain <code>'@Grab'</code>, then you should be using <code>if '@Grab' in test: ...</code>.</span>
