<p>Python 3.7</p>
<p>I am writing a Python (3.7) code for National Instruments hardware. There is a <a href="https://pypi.org/project/PyDAQmx/" rel="nofollow noreferrer">python library for Python 2.7</a>. There are <a href="https://pythonhosted.org/PyDAQmx/installation.html" rel="nofollow noreferrer">instructions</a> on how to convert the library to be with Python 3.</p>
<ol>
<li>I have downloaded PyDAQmx-1.4.1.tar.gz from the project <a href="https://pypi.org/project/PyDAQmx/#files" rel="nofollow noreferrer">website</a></li>
<li>extracted the content of the archive file to a directory</li>
<li>go to that directory in the terminal</li>
<li>to convert PyDAQmx library from 2 to 3, I run the 2to3 converter in the terminal following <a href="https://docs.python.org/3/library/2to3.html" rel="nofollow noreferrer">instruction</a>.</li>
</ol>
<p><code>C:\Downloads\&gt;2to3 --output-dir=PyDAQmx-1.4.1-3 -W -n PyDAQmx-1.4.1 &gt;2to3.log 2&gt;&amp;1</code></p>
<p>You can inspect the log file for error and warnings.</p>
<p>next,build</p>
<p><code>C:\Downloads\PyDAQmx-1.4.1-3&gt;py setup.py build &gt;build 2&gt;&amp;1
and install</code></p>
<p>and install</p>
<p><code>C:\Downloads\PyDAQmx-1.4.1-3&gt;py setup.py install &gt;install.log 2&gt;
&amp;1</code></p>

<p>Here is the solution:</p>
<p>follow the protocol posted on <a href="https://pythonhosted.org/PyDAQmx/installation.html" rel="nofollow noreferrer">the webpage</a></p>
<p>Briefly,</p>
<ol>
<li>download PyDAQmx-1.4.1.tar.gz from the project <a href="https://pypi.org/project/PyDAQmx/#files" rel="nofollow noreferrer">website</a></li>
<li>extract the content of the archive file to a directory</li>
<li>go to that directory in the terminal</li>
<li>to convert PyDAQmx library from 2 to 3, I run the 2to3 converter in the terminal following <a href="https://docs.python.org/3/library/2to3.html" rel="nofollow noreferrer">instruction</a>.</li>
</ol>
<p><code>C:\Downloads\&gt;2to3 --output-dir=PyDAQmx-1.4.1-3 -W -n PyDAQmx-1.4.1 &gt;2to3.log 2&gt;&amp;1</code></p>
<p>You can inspect the log file for error and warnings.</p>
<p>next, build</p>
<p><code>C:\Downloads\PyDAQmx-1.4.1-3&gt;py setup.py build &gt;build 2&gt;&amp;1
and install</code></p>
<p>if you install you indeed get an egg file but there is a corrupted file.</p>
<p><code>C:\Downloads\PyDAQmx-1.4.1-3&gt;py setup.py install &gt;install.log 2&gt;
&amp;1</code></p>
<p>the DAQmxCallBack file is corrupted
<a href="https://i.stack.imgur.com/0s2s0.jpg" rel="nofollow noreferrer"><img alt="enter image description here" src="https://i.stack.imgur.com/0s2s0.jpg"/></a></p>
<p>you need to go to \PyDAQmx-1.4.1-3 folder which you got after the conversion from 2 to 3 and copy the file DAQmxCallBack file from the \PyDAQmx\ subdirectory to site-packages subdirectory of Python 3 installation.</p>
<p>Another way would be to skip the install command and just directly copy PyDAQmx folder from \PyDAQmx-1.4.1-3 to C:\Python37-32\Lib\site-packages (or wherever you have you python3 installed)</p>
