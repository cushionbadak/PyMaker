<p>I have a dictionary which looks like this:</p>
<pre><code>my_dict = {"a":1, "e":2, "i":3, ...}
</code></pre>
<p>And I also have a list which consists of different graphemes:</p>
<pre><code>my_list = ["t", "a", "b", "l", "e", ...]
</code></pre>
<p>What I want to do is iterate over each of the graphemes in my list and, if the grapheme is part of the dictionary, then replace it by its value. </p>
<p>I tried using <code>my_dict.get()</code>, though it requires an argument (which corresponds to a given value). Since I don't know what the exact value will be, for it will be changing depending on the grapheme that the function is iterating over, what could I do to get my desired output?</p>
</div>
<div class="post-text" itemprop="text">
<p>Would something like this work?</p>
<pre><code>result = [my_dict.get(grapheme, grapheme) for grapheme in my_list]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Use a list-comprehension with <code>dict.get()</code> which takes two parameters: key and a default value (optional).</p>
<p>From <a href="https://docs.python.org/3/library/stdtypes.html#dict.get" rel="nofollow noreferrer">docs</a>:</p>
<blockquote>
<p><code>get(key[, default])</code>: 
  Returns the value for key if key is in the dictionary, else default.
  If default is not given, it defaults to <code>None</code>, so that this method
  never raises a <code>KeyError</code>.</p>
</blockquote>
<pre><code>my_dict = {"a": 1, "e": 2, "i": 3}
my_list = ["t", "a", "b", "l", "e"]

print([my_dict.get(x, x) for x in my_list])
# ['t', 1, 'b', 'l', 2]
</code></pre>
</div>
<span class="comment-copy">What do you mean by being part of dictionary, is it being its key or value?</span>
<span class="comment-copy"><code>my_dict.get(), though it requires an argument (which corresponds to a given value)</code>. No, that's not true. The argument is a <b>key</b>, not a value.</span>
<span class="comment-copy">@Looioe if the grapheme is the key. So, in this case, all "a" should be replaced by "1", and so on</span>
<span class="comment-copy"><code>my_dict.get()</code> takes a <b>key</b>, and a default value to return when the key is missing. In your case, both the key <i>and</i> that default value to return is the grapheme <i>itself</i>. So <code>my_dict.get('a', 'a')</code> returns <code>1</code>, but <code>my_dict.get('t', 't')</code> returns <code>'t'</code>. Use <code>[my_dict.get(grapheme, grapheme) for grapheme in my_list]</code>.</span>
