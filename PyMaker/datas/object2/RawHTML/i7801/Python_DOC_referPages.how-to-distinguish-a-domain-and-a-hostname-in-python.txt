<div class="post-text" itemprop="text">
<p>A domain is something like this: <strong>google.com, yahoo.com</strong>. It also have a whois record</p>
<p>A hostname is something like this: <strong>m.google.com, www.google.com, images.google.com</strong></p>
<p>A domain can have very interesting TLDs and ccTLDs: <strong>google.co.uk, google.academy, google.xxx</strong></p>
<p>A hostname can be also like this: <strong>mail.services.1.google.com, xxx.google.com</strong></p>
<p>Here is the question: I have a string variable and i want to decide that if the value is a hostname or a domain. Is there a clever way to distinguish them in python?</p>
</div>
<div class="post-text" itemprop="text">
<p>You already seem to know how to differentiate between them.</p>
<p>use <code>urllib.parse</code> to break down the string and then write your own logic to decide.</p>
<p>Docs: <a href="https://docs.python.org/3/library/urllib.parse.html" rel="nofollow noreferrer">https://docs.python.org/3/library/urllib.parse.html</a></p>
</div>
<div class="post-text" itemprop="text">
<p>I found the answer. We can do this with tldextract package.</p>
<pre><code>from tldextract import tldextract

test_str = 'mail.google.co.uk'
te_result = tldextract.extract(test_str)
domain = '{}.{}'.format(te_result.domain, te_result.suffix)
print('domain: {}'.format(domain))
print('is_hostname: {}'.format(test_str != domain))
print('is_domain: {}'.format(test_str == domain))
</code></pre>
<p>Answer:</p>
<pre><code>domain: google.co.uk
is_hostname: True
is_domain: False
</code></pre>
