<p>min() function should return the smallest of the numbers given. Then why</p>
<pre><code>min({0},{1})
</code></pre>
<p>returns {0}, while</p>
<pre><code>min({1},{0})
</code></pre>
<p>returns {1}?</p>
</div>
<div class="post-text" itemprop="text">
<p>In python sets are compared based on whether or not one <strong>is a subset</strong> of the other.</p>
<p>None is a subset of the other, so <code>&lt;</code> gives <code>False</code> in all cases, so the first one is returned.</p>
<p>So <code>{1}&lt;{0}</code> gives <code>False</code> just like <code>{2}&lt;{1}</code> gives <code>False</code> and <code>{2}&lt;{3,4,5}</code> gives False.
However <code>{1,2}&lt;{1,3,2}</code> gives <code>True</code>.
This means there is no <strong>total order</strong> defined on sets.</p>
</div>
<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/stdtypes.html#set.issubset" rel="noreferrer">comparison operators for sets</a> check for "subset" not for numerical comparisons:</p>
<blockquote>
<p><code>set &lt; other</code></p>
<p>Test whether the set is a proper subset of other, that is, set &lt;= other and set != other.</p>
</blockquote>
<p>And both your sets are disjoint so <code>{0} &lt; {1} == False</code> and <code>{1} &lt; {0} == False</code> so it will always return the "first" set you put into <code>min</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p>Because these are <strong>sets</strong>, not <strong>numbers</strong>. </p>
<p>Moreover, it should return {1}, not 1, as you can see here:</p>
<pre><code>&gt;&gt;&gt; min({0},{1})
set([0])
&gt;&gt;&gt; min({1},{0})
set([1])
</code></pre>
<p>That means that it will return the <em>set</em>, not the <em>number</em>.</p>
</div>
