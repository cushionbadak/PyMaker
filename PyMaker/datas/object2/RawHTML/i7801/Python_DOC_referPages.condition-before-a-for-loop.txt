<div class="post-text" itemprop="text">
<p>Since <code>b = True</code>, <code>zs.extend([b for i in range(k)])</code> is effectively <code>zs.extend([True for i in range(k)])</code> or <code>zs.extend([True, True, ..., True (k times)])</code>.</p>
<p>Update. I'm assuming that the intended (indented) code is the following:</p>
<pre><code>def uncompress(xs):
  zs = []
  b = True
  for k in xs:
    zs.extend([b for i in range(k)])
    b = not b
  return zs
</code></pre>
<p>Then if <code>xs</code> is, say, <code>[2, 1, 3]</code>, then:</p>
<ul>
<li><p>after the first for-loop execution (so called "iteration"), <code>zs = [] + [True, True] = [True, True]</code> and <code>b</code> becomes <code>False</code>,</p></li>
<li><p>then <code>zs = [True, True] + [False] = [True, True, False]</code> and <code>b</code> becomes <code>True</code>, </p></li>
<li><p>finally <code>zs = [True, True, False, True, True, True]</code> and <code>b</code> becomes <code>False</code>.</p></li>
</ul>
<p>So if <code>xs == [s, t, u, v, ...]</code> then <code>zs</code> is <code>True</code> <code>s</code> times, <code>False</code> <code>t</code> times, <code>True</code> <code>u</code> times, <code>False</code> <code>v</code> times and so on.</p>
</div>
