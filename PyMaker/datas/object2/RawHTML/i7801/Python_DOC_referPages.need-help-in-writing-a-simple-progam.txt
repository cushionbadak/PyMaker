<div class="post-text" itemprop="text">
<p>Because <code>"June 5, 2017"</code>  is a str object, and <code>webPageCopy</code> is a bytes-like object.</p>
<p>You need to  convert <code>webPageCopy</code> to str type.</p>
<pre><code>if "June 5, 2017" in str(webPageCopy): 
</code></pre>
<p>Or make "June 5, 2017" just be a byte-like object when defined as Wasi Ahmad mentioned.</p>
<pre><code>if b"June 5, 2017" in webPageCopy: 
</code></pre>

<div class="post-text" itemprop="text">
<p>Just do this.</p>
<pre><code>if b"June 5, 2017" in webPageCopy: 
   print ('success')
</code></pre>
<p>In your example, all the data that you got from <code>response.read()</code> is bytes objects, not string. You can see what is the type of an object in python as follows.</p>
<pre><code>print(type(webPageCopy)) # prints - &lt;class 'bytes'&gt;
</code></pre>
<p>So, adding a prefix (<code>b</code>) prefix before the target string solves your problem.</p>
<p>Other alternatives:</p>
<pre><code>if bytes("June 5, 2017", 'utf8') in webPageCopy: 
    print ('success')
</code></pre>
<p>or,</p>
<pre><code>if "June 5, 2017".encode('utf8') in webPageCopy: 
   print ('success')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>webPageCopy</code> is a <a href="https://docs.python.org/3/library/stdtypes.html#bytes" rel="nofollow noreferrer"><code>bytes</code></a> object, not a string, so in order to use the <code>in</code> operator with it, you'd have to supply another <code>bytes</code> (or bytes-like) object. Luckily, this is quite easy in Python - just use the <code>b</code> prefix when you denote the literal:</p>
<pre><code>if b"June 5, 2017" in webPageCopy: 
   print ('success')
</code></pre>
