<div class="post-text" itemprop="text">
<p>The error indicates that virtualenv is trying to make an environment in your python path. therefore, you have to specify your virtualenv destination</p>
<pre><code>virtualenv -p python3.6 /path/to/yourenv
</code></pre>

<span class="comment-copy">You run <code>python3.6</code> from your virtual environment. In Linux Mint 17 <code>python3</code> is installed per default and should be available under <code>/usr/bin/python3</code>. What happens if you do <code>which python3</code>?</span>
<span class="comment-copy">Your question is about the python standard library module <code>venv</code> (<a href="https://docs.python.org/3/library/venv.html" rel="nofollow noreferrer">docs.python.org/3/library/venv.html</a>). That is not the same as <code>virtualenv</code> (<a href="https://pypi.python.org/pypi/virtualenv" rel="nofollow noreferrer">pypi.python.org/pypi/virtualenv</a>). Maybe you should edit the tags to reflect this.</span>
