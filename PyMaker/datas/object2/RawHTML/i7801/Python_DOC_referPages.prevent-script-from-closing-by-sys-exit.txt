<div class="post-text" itemprop="text">
<p><code>sys.exit()</code> doesn't do more then raising <code>SystemExit</code> (a <code>BaseException</code> subclass), which can be caught like any exception e.g:</p>
<pre><code>import time
while True:
    try:
        execfile("test.py")
    except SystemExit:
        print("ignoring SystemExit")
    finally:
        time.sleep(300)
</code></pre>
</div>

<p>You should be able to use:</p>
<pre><code>try:
  # Your call here
except BaseException as ex:
  print("This should be a possible sys.exit()")
</code></pre>
<p>Check out <a href="https://docs.python.org/3/library/exceptions.html#SystemExit" rel="nofollow noreferrer">the documentation</a> for more information.</p>

<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/2/library/subprocess.html" rel="nofollow noreferrer">subprocess</a></p>
<pre class="lang-py prettyprint-override"><code>import subprocess  
import time

while(1==1):
    subprocess.call(['python', './test.py'])
    time.sleep(300)
</code></pre>
<p>You could even remove the <em>python</em> word if the <code>test.py</code> file has a shebang comment on the first line:</p>
<pre><code>#!/usr/bin/env python
</code></pre>
<p>This is not exactly the same, as it will start a new interpreter, but the results will be similar.</p>
</div>
