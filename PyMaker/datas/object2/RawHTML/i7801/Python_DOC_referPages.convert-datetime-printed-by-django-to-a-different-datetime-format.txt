<p>My web application works with datetime(s) as timestamps, which according to Django's error messages need to be in the following format: <code>YYYY-MM-DD HH:MM[:ss[.uuuuuu]][TZ]</code>.</p>

<p>How do I convert the format printed by Django on the web app (<code>12. Juni 2017 10:17</code>) to the format, required by Django for further processing (<code>YYYY-MM-DD HH:MM[:ss[.uuuuuu]][TZ]</code>)?</p>

<div class="post-text" itemprop="text">
<p>Your issue is that <code>"%y/%m/%d %B"</code> doesn't match the pattern <code>'12. June 2017 10:17'</code></p>
<p>You can try using the following to parse the date correctly:</p>
<pre><code>&gt;&gt;&gt; from dateutil.parser import parse

&gt;&gt;&gt; parse('12. June 2017 10:17')
datetime.datetime(2017, 6, 12, 10, 17)
</code></pre>
<p>Or with:</p>
<pre><code>&gt;&gt;&gt; from datetime import datetime

&gt;&gt;&gt; datetime.strptime('12. June 2017 10:17', '%d. %B %Y %I:%M')
datetime.datetime(2017, 6, 12, 10, 17)
</code></pre>
<p>You can work out what percentage values to use from this useful table in the documentation <a href="https://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow noreferrer">here</a></p>
</div>
<div class="post-text" itemprop="text">
<p>Yes it expects the format <code>YYYY-MM-DD HH:MM[:ss[.uuuuuu]][TZ]</code> so the Valid values are:</p>
<pre><code>2017-09-04 06:00
2017-09-04 06:00:00
2017-09-04 06:00:00.000000

# w/ optional TZ as timezone. 
2017-09-04 06:00Z # utc
2017-09-04 06:00:00+0800
2017-09-04 06:00:00.000000-08:00
</code></pre>
<p>This should do the trick:</p>
<pre><code>import datetime
d = datetime.datetime.strptime('12. June 2017 10:17', '%d. %B %Y %H:%M')
print(d) # or print(str(d)) if you want it as a string 
</code></pre>
<p>output:</p>
<pre><code>2017-06-12 10:17:00
</code></pre>
<p>which is in the valid accepted format <code>(YYYY-MM-DD HH:MM[:ss[.uuuuuu]][TZ])</code></p>
</div>
<div class="post-text" itemprop="text">
<p>(12. Juni 2017 10:17) is in Pseudocode:</p>
<p><code>day</code>. <code>month in locale full name</code> <code>year with century</code> <code>hour</code>:<code>minute</code></p>
<p>To convert this into a datetime object with <a href="https://docs.python.org/2/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow noreferrer">strptime</a> use:</p>
<pre><code>datetime.datetime.strptime(your_input, '%d. %B %Y %H:%M')
</code></pre>
<p>to convert back, use the equivalent <code>strftime</code>, with your format string.</p>
