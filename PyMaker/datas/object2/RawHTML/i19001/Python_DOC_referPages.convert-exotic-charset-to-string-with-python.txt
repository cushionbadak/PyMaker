<div class="post-text" itemprop="text">
<p>With a little help from <a href="https://stackoverflow.com/a/5863893/1126841">this answer</a>:</p>
<pre><code>&gt;&gt;&gt; table = dict([(x + 0xFF00 - 0x20, unichr(x)) for x in xrange(0x21, 0x7F)] + [(0x3000, unichr(0x20))])
&gt;&gt;&gt; data.translate(table)
u'A Deuce'
</code></pre>
<p>The <code>translate</code> method takes a dictionary that maps one Unicode code point to another. In this case, it maps the full-width Latin alphabet (which is essentially part of the ASCII character set shifted up to the range 0xFF01-0xFF5E) to the "normal" ASCII character set. For example, 0xFF21 (full-width A) maps to 0x41 (ASCII A), 0xFF22 (full-width B) maps to 0x42 (ASCII B), etc.</p>
</div>
<div class="post-text" itemprop="text">
<p>This functionality is built into the <a href="http://docs.python.org/3/library/unicodedata.html" rel="nofollow">unicodedata</a> module:</p>
<pre><code>&gt;&gt;&gt; unicodedata.normalize('NFKC', 'Ａ　Ｄｅｕｃｅ')
'A Deuce'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Consider using Python 3, which has better printing support for Unicode characters.  Here's a sample:</p>
<pre><code>&gt;&gt;&gt; s=u'\uff21\u3000\uff24\uff45\uff55\uff43\uff45'
&gt;&gt;&gt; print(s)
Ａ　Ｄｅｕｃｅ
&gt;&gt;&gt; s
'Ａ\u3000Ｄｅｕｃｅ'
&gt;&gt;&gt; import unicodedata as ud
&gt;&gt;&gt; ud.name('\u3000')
'IDEOGRAPHIC SPACE'
&gt;&gt;&gt; print(ascii(s))
'\uff21\u3000\uff24\uff45\uff55\uff43\uff45'
</code></pre>
