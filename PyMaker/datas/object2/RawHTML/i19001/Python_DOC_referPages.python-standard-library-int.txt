<p>I was reading through the documentation to the Python Standard Library. In section 4.4. <a href="http://docs.python.org/3/library/stdtypes.html#numeric-types-int-float-complex" rel="nofollow">Numeric Types</a> there is a note to <code>int</code>:</p>
<blockquote>
<p>Conversion from floating point to integer may round or truncate as in
  C</p>
</blockquote>
<p>What does this mean? I thought int always returned the floor? Is that not the case?</p>
<pre><code>&gt;&gt;&gt; print(int(0.4))
0
&gt;&gt;&gt; print(int(0.6))
0
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Conversion from a float to a integer truncates towards 0 as it does in C. This is basically equivalent to <code>math.floor(abs(x))*sgn(x)</code>, where <code>sgn(x)</code> gives the sign of a number.</p>
</div>
<span class="comment-copy">no, it doesn't alway return the floor: <code>math.floor(-.5)</code> gives <code>-1.0</code>, while <code>int(-.5)</code> gives <code>0</code></span>
<span class="comment-copy"><code>&gt;&gt;&gt; print(int(-0.4)) 0</code> &lt;- truncates, not the <code>floor</code>.</span>
