<p>I'd like to be able to use unicode in my python string. For instance I have an icon:</p>
<pre><code>icon = '&amp;#x25B2;'
print icon
</code></pre>
<p>which should create icon = '▲'</p>
<p>but instead it literally returns it in string form: <code>&amp;#x25B2;</code></p>
<p>How can I make this string recognize unicode?</p>
<p>Thank you for your help in advance.</p>

<div class="post-text" itemprop="text">
<p>You can use string escape sequences, as documented in <a href="http://docs.python.org/3/reference/lexical_analysis.html#index-18" rel="noreferrer">the “string and bytes literals” section</a> of the language reference. For Python 3 this would work simply like this:</p>
<pre><code>&gt;&gt;&gt; icon = '\u25b2'
&gt;&gt;&gt; print(icon)
▲
</code></pre>
<p>In <a href="http://docs.python.org/2/reference/lexical_analysis.html#index-15" rel="noreferrer">Python 2</a> this only works within unicode strings. Unicode strings have a <code>u</code> prefix before the quotation mark:</p>
<pre><code>&gt;&gt;&gt; icon = u'\u25b2'
&gt;&gt;&gt; print icon
▲
</code></pre>
<p>This is not necessary in Python 3 as all strings in Python 3 are unicode strings.</p>
</div>

<div class="post-text" itemprop="text">
<p>Use <code>\u</code> escaping in a unicode string literal:</p>
<pre><code>&gt;&gt;&gt; print u"\u25B2".encode("utf-8")
▲
</code></pre>
<p>Alternatively, if you want to use HTML entities, you can use this answer: <a href="https://stackoverflow.com/a/2087433/71522">https://stackoverflow.com/a/2087433/71522</a></p>
