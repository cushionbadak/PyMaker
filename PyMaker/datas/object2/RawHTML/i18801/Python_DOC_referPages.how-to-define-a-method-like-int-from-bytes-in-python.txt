<div class="post-text" itemprop="text">
<p>In python there is a method on the type <code>int</code>, namely <code>int.from_bytes</code>. It's not a method on a particular int, instead it's a method on the type. E.g.</p>
<pre><code>&gt;&gt;&gt; int.from_bytes(b'\xee\xff',"big")
61183

&gt;&gt;&gt; int.from_bytes
&lt;built-in method from_bytes of type object at 0x107fdb388&gt;
</code></pre>
<p>How do I define something like this? Let's say a class called "point" is defined, how do I define something like</p>
<pre><code>&gt;&gt;&gt; point.from_coordinates(3,5)
&lt;__main__.point object at 0x10c0c9310&gt;

&gt;&gt;&gt; point.from_keys(b'\x12\x3e')
&lt;__main__.point object at 0x10bed5890&gt;
</code></pre>
<p>? (Assuming points are initialized by some different method.)</p>
</div>
<div class="post-text" itemprop="text">
<p>You want <a href="http://docs.python.org/3/library/functions.html#classmethod" rel="noreferrer"><code>classmethod</code></a>, which is normally used as a decorator:</p>
<pre><code>class point(object):
    @classmethod
    def from_coordinates(cls, x, y):
        pt = cls()
        pt.x, pt.y = x, y
        return pt
</code></pre>
<hr/>
<p>This is sometimes called the "alternate constructor" idiom. If there are multiple different ways to construct your type, instead of putting them all into one <code>__init__</code> method with a mess of optional parameters or varags, put them all into separate <code>classmethod</code>s.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use a <code>classmethod</code>, such as the mocked up example whose <code>init</code> expects ints, but also provides a convenient <code>from_hex</code> that tries to take strings and convert them to integers...</p>
<pre><code>class MyClass(object):
    def __init__(self, a, b):
        self.a = a
        self.b = b
    @classmethod
    def from_hex(cls, a, b):
        return cls(int(a, 16), int(b, 16))
</code></pre>
<p>The <code>from_hex</code> knows which class it is associated with, so by calling <code>cls(...)</code> you're able to construct a type of <code>MyClass</code> as though you'd written <code>MyClass(a, b)</code> with valid integers yourself.</p>
</div>
