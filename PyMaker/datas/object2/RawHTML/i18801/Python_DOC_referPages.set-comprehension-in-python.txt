<p>In Python3 I wrote a simple one line code as follows :</p>
<pre><code>{ 2*x  for x in {1,2,3,4} } 
</code></pre>
<p>but I m getting answer like this (order changed).</p>
<pre><code>{8, 2, 4, 6}
</code></pre>
<p>Why I am getting answer <code>{8,2,4,6}</code> instead of <code>{2,4,6,8}</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>That's because sets don't have any order. They're unordered collection.</p>
<p>help on <code>set</code>:</p>
<blockquote>
<p>Build an <strong>unordered</strong> collection of unique elements.</p>
</blockquote>
<p>If you want the order to be preserved then you can use <code>list</code>, <code>tuple</code> or <code>collections.OrderedDict</code> here.</p>
</div>
<div class="post-text" itemprop="text">
<p>Because a set has no fixed order. Quoting <a href="http://docs.python.org/3/library/stdtypes.html#set-types-set-frozenset" rel="nofollow">the set documentation</a>:</p>
<blockquote>
<p>A <em>set</em> object is an unordered collection of distinct hashable objects.</p>
</blockquote>
<p>Use a list or tuple if you require ordering, or use an <code>OrderedDict()</code> object to create unique keys that preserve ordering:</p>
<pre><code>from collections import OrderedDict

ordered_unique_values = list(OrderedDict.fromkeys(2*x for x in (1,2,3,4)))
</code></pre>
</div>
