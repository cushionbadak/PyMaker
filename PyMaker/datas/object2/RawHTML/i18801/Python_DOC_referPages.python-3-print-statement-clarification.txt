<div class="post-text" itemprop="text">
<p>When you provide multiple arguments to <code>print</code> they usually get separated by a space:</p>
<pre><code>&gt;&gt;&gt; print(1, 2, 3)
1 2 3
</code></pre>
<p><code>sep</code> lets you change that to something else:</p>
<pre><code>&gt;&gt;&gt; print(1, 2, 3, sep=', ')
1, 2, 3
</code></pre>
<p>Normally, <code>print</code> will add a new line to the end. <code>end</code> lets you change that:</p>
<pre><code>&gt;&gt;&gt; print('Hello.', end='')
Hello.&gt;&gt;&gt;
</code></pre>
<p>Normally <code>print</code> will write to standard out. <code>file</code> lets you change that:</p>
<pre><code>&gt;&gt;&gt; with open('test.txt', 'w') as f:
...     print("Hello, world!", file=f)
...
</code></pre>
<p>Normally <code>print</code> does not explicitly flush the stream. If you want to avoid an extra <code>sys.stdout.flush()</code>, you can use <code>flush</code>. The effect of this is usually hard to see, but trying this without <code>flush=True</code> should make it visible:</p>
<pre><code>&gt;&gt;&gt; import time
&gt;&gt;&gt; while True:
...     print('.', end='', flush=True)
...     time.sleep(0.5)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Python 2 doesn't have a <code>sep</code> equivalent because <code>print</code> wasn't a function and couldn't be passed arguments. The closest you could do was with <code>join</code>:</p>
<pre><code> print ' '.join([value, ...])
</code></pre>
<p>As for <code>file</code>, you'd have to use this (awkward, in my opinion) syntax:</p>
<pre><code>print &gt;&gt; sys.stdout, ' '.join([value, ...])
</code></pre>
<p>I'm not going to copy/paste the documentation here, so read it if you want to know what those arguments are for.</p>
</div>
