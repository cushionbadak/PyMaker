<p>How do I delete a possibly non-empty dir in Python.</p>
<p>The directory may have nested subdirectories many levels deep.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="http://docs.python.org/library/shutil.html#shutil.rmtree" rel="noreferrer"><code>shutil.rmtree</code></a>:</p>
<pre><code>import shutil

shutil.rmtree(path)
</code></pre>
<p>See <a href="http://docs.python.org/library/shutil.html#shutil.rmtree" rel="noreferrer">the documentation</a> for details of how to handle and/or ignore errors.</p>
</div>
<div class="post-text" itemprop="text">
<p>The standard library includes <a href="http://docs.python.org/3/library/shutil.html#shutil.rmtree" rel="noreferrer">shutil.rmtree</a> for this. By default,</p>
<pre><code>shutil.rmtree(path)  # errors if dir not empty
</code></pre>
<p>will give <code>OSError: [Errno 66] Directory not empty: &lt;your/path&gt;</code>.</p>
<p>You can delete the directory and its contents anyway by ignoring the error:</p>
<pre><code>shutil.rmtree(role_fs_path, ignore_errors=True)
</code></pre>
<p>You can perform more sophisticated error handling by also passing <code>onerrror=&lt;some function(function, path, excinfo)&gt;</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p>You want <a href="http://docs.python.org/library/shutil.html#shutil.rmtree" rel="noreferrer">shutil.rmtree</a></p>
<blockquote>
<p>shutil.rmtree(path[, ignore_errors[,
  onerror]])</p>
<p>Delete an entire directory
  tree; path must point to a directory
  (but not a symbolic link to a
  directory). If ignore_errors is true,
  errors resulting from failed removals
  will be ignored; if false or omitted,
  such errors are handled by calling a
  handler specified by onerror or, if
  that is omitted, they raise an
  exception.</p>
</blockquote>
</div>
