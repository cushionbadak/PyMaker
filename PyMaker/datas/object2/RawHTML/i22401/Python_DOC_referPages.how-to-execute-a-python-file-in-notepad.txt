<h3>First option: (Easiest, recommended)</h3>
<p>Open Notepad++. On the menu go to: Run -&gt; Run.. (F5). Type in:</p>
<pre><code>C:\Python26\python.exe "$(FULL_CURRENT_PATH)"
</code></pre>
<p>Now, instead of pressing run, press save to create a shortcut for it.</p>
<p><strong><em>Notes</em></strong></p>
<ul>
<li>If you have Python 3.1: type in <code>Python31</code> instead of <code>Python26</code></li>
<li>Add <code>-i</code> if you want the command line window to stay open after the script has finished</li>
</ul>
<hr/>
<h3>Second option</h3>
<p>Use a batch script that runs the Python script and then create a shortcut to that from Notepad++.</p>
<p>As explained here: <a href="http://it-ride.blogspot.com/2009/08/notepad-and-python.html" rel="noreferrer">http://it-ride.blogspot.com/2009/08/notepad-and-python.html</a></p>
<hr/>
<h3>Third option: (Not safe)</h3>
<blockquote>
<p>The code opens “HKEY_CURRENT_USER\Software\Python\PythonCore”, if the key exists it will get the path from the first child key of this key.</p>
</blockquote>
<p>Check if this key exists, and if does not, you could try creating it.</p>

<div class="post-text" itemprop="text">
<p>All the answers for the Run-&gt;Run menu option go with the "/K" switch of cmd, so the terminal stays open, or "-i" for python.exe so python forces interactive mode - both to preserve the output for you to observe.</p>
<p>Yet in <code>cmd /k</code> you have to type <code>exit</code> to close it, in the <code>python -i</code> - <code>quit()</code>. If that is too much typing for your liking (for me it sure is :), the Run command to use is</p>
<pre><code>cmd /k C:\Python27\python.exe  "$(FULL_CURRENT_PATH)" &amp; pause &amp; exit
</code></pre>
<p><code>C:\Python27\python.exe</code> - obviously the full path to your python install (or just <code>python</code> if you want to go with the first executable in your user's path).</p>
<p><code>&amp;</code> is unconditional execution of the next command in Windows - unconditional as it runs regardless of the RC of the previous command (<code>&amp;&amp;</code> is "and" - run only if the previous completed successfully, <code>||</code> - is "or").</p>
<p><code>pause</code> - prints "Press any key to continue . . ." and waits for any key (that output can be suppressed if need).</p>
<p><code>exit</code> - well, types the exit for you :)</p>
<p>So at the end, <code>cmd</code> runs <code>python.exe</code> which executes the current file and keeps the window opened, <code>pause</code> waits for you to press any key, and <code>exit</code> finally close the window once you press that any key.</p>
</div>

<p>No answer here, or plugin i found provided what i wanted. A minimalist method to launch my python code i wrote on Notepad++ with the press of a shortcut, with preferably no plugins.</p>
<p>I have Python 3.6 (64-bit), for Windows 8.1 x86_64 and Notepad++ 32bit. After you write your Python script in Notepad++ and save it, Hit F5 for <code>Run</code>. Then write: </p>
<pre><code>"C:\Path\to\Python\python.exe" -i "$(FULL_CURRENT_PATH)"
</code></pre>
<p>and hit the Run button. The <code>i</code> flag forces the terminal to stay still after code execution has terminated, for you to inspect it. This command will launch the script in a cmd terminal and the terminal will still lie there, until you close it by typing <code>exit()</code>. </p>
<p>You can save this to a shortcut for convenience (mine is CTRL + SHIFT + P). </p>

Running Python &amp; wxPython</code></a> with this <a href="https://htmlpreview.github.io/?https://github.com/d0vgan/nppexec/blob/master/NppExec/doc/NppExec/NppExec_Manual/4.6.4.html" rel="nofollow noreferrer">code</a>:</p>
<pre><code>npp_console -  // disable any output to the Console
npp_save  // save current file (a .py file is expected)
cd "$(CURRENT_DIRECTORY)"  // use the current file's dir
set local @exit_cmd_silent = exit()  // allows to exit Python automatically
set local PATH_0 = $(SYS.PATH)  // current value of %PATH%
env_set PATH = $(SYS.PATH);C:\Python27  // use Python 2.7
npp_setfocus con  // set the focus to the Console
npp_console +  // enable output to the Console
python -i -u "$(FILE_NAME)"  // run Python's program interactively
npp_console -  // disable any output to the Console
env_set PATH = $(PATH_0)  // restore the value of %PATH%
npp_console +  // enable output to the Console
</code></pre>
<p>All you need to do is copy this code and change your python directory if you use some other python version (e.g.<a href="https://www.merriam-webster.com/dictionary/e.g." rel="nofollow noreferrer">*</a> I am using python 3.4 so my directory is <code>C:\Python34</code>). This code works perfectly, but there is one <a href="https://image.prntscr.com/image/WjlWk5yeQQ2ns6PCbAC6Fw.png" rel="nofollow noreferrer">line</a> I added to this code so I can run python program multiple times without loosing previous output:</p>
<pre><code>npe_console m- a+
</code></pre>
<p><a href="https://htmlpreview.github.io/?https://github.com/d0vgan/nppexec/blob/master/NppExec/doc/NppExec/NppExec_Manual/4.1.html" rel="nofollow noreferrer"><code>a+</code></a> is to enable the "append" mode which keeps the previous Console's text and does not clear it.</p>
<p><code>m-</code> turns off console's internal messages (those are in green color)</p>
<p>The final code that I use in NppExec's Execute window is:</p>
<pre><code>npp_console -  // disable any output to the Console
npp_save  // save current file (a .py file is expected)
cd "$(CURRENT_DIRECTORY)"  // use the current file's dir
set local @exit_cmd_silent = exit()  // allows to exit Python automatically
set local PATH_0 = $(SYS.PATH)  // current value of %PATH%
env_set PATH = $(SYS.PATH);C:\Python34  // use Python 3.4
npp_setfocus con  // set the focus to the Console
npe_console m- a+
npp_console +  // enable output to the Console
python -i -u "$(FILE_NAME)"  // run Python's program interactively
npp_console -  // disable any output to the Console
env_set PATH = $(PATH_0)  // restore the value of %PATH%
npp_console +  // enable output to the Console
</code></pre>

<p>There is one issue that I didn't see resolved in the above solutions. Python sets the current working directory to wherever you start the interpreter from. If you need the current working directory to be the same directory as where you saved the file on, then you could hit F5 and type this:</p>
<pre><code>cmd /K cd "$(CURRENT_DIRECTORY)"&amp;C:\Users\username\Python36-32\python.exe -i "$(FULL_CURRENT_PATH)"
</code></pre>
<p>Except you would replace C:\Users\username\Python36-32\python.exe with whatever the path to the python interpreter is on your machine.</p>
<p>Basically you're starting up command line, changing the directory to the directory containing the .py file you're trying to run, and then running it. You can string together as many command line commands as you like with the '&amp;' symbol.</p>
