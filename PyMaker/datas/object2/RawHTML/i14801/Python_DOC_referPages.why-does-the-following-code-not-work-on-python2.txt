<div class="post-text" itemprop="text">
<p><code>input()</code> needs to be <code>raw_input()</code> for Python 2.</p>
<p>This was documented in the <a href="https://docs.python.org/3/whatsnew/3.0.html" rel="nofollow">Python 3 change logs</a>:</p>
<p>"PEP 3111: raw_input() was renamed to input(). That is, the new input() function reads a line from sys.stdin and returns it with the trailing newline stripped. It raises EOFError if the input is terminated prematurely. To get the old behavior of input(), use eval(input())."</p>
<p>Also, as cdarke pointed out, print statements shouldn't have parentheses around the item to print.</p>
</div>
<div class="post-text" itemprop="text">
<p>In Python 2 <code>input</code> is used for a different purpose, you should use <code>raw_input</code> in Python 2.</p>
<p>In addition, your <code>print</code> should not use parentheses.  In Python 3 <code>print</code> is a function whereas in Python 2 it is a statement.  Alternatively:</p>
<pre><code>from __future__ import print_function
</code></pre>
<p>In this case you can achieve some portability with something like this:</p>
<pre><code>import sys
if sys.version_info[0] == 2:  # Not named on 2.6
    from __future__ import print_function
    userinput = raw_input
else:
    userinput = input
</code></pre>
<p>Then use <code>userinput</code> instead of <code>input</code> or <code>raw_input</code>.  Its not pretty though, and generally it is best to stick with just one Python version.</p>
