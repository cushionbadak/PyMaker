<div class="post-text" itemprop="text">
<p>Setting syntax errors aside, the spirit of your two statements are generally equivalent. The first is a <a href="https://docs.python.org/3/reference/expressions.html#conditional-expressions" rel="nofollow noreferrer">"conditional expression"</a> (also called a "ternary expression").</p>
<pre><code>self.__hours = 0 if self.__hours == 23 else self.__hours + 1
</code></pre>
<p>You are setting <code>self.__hours</code> to <em>something</em> -- either <code>0</code> or <code>self.__hours + 1</code> -- depending on the current value of <code>self.__hours</code>.</p>
<p>The equivalent <a href="https://docs.python.org/3/reference/compound_stmts.html#the-if-statement" rel="nofollow noreferrer"><code>if</code> statement</a> would be:</p>
<pre><code>if self.__hours == 23:
    self.__hours = 0
else:
    self.__hours = self.__hours + 1
</code></pre>
<p>(<code>self.__hours = self.__hours + 1</code> can also be replaced with <code>self.__hours += 1</code>.)</p>
</div>
<div class="post-text" itemprop="text">
<p>In general, they're going to provide equivalent answers and be interchangeable.</p>
<p>One obvious difference between the two forms is that the ternary form can be used in places where an expression can be used as the body of a lambda, while the second one can't:</p>
<pre><code>&gt;&gt;&gt; x = lambda y: 0 if y &gt; 0 else -1
&gt;&gt;&gt;
&gt;&gt;&gt; x(1)
0
&gt;&gt;&gt; x(-1)
-1
</code></pre>
<p>or </p>
<pre><code>&gt;&gt;&gt; def fn(val):
...    print val
...
&gt;&gt;&gt; y = 2
&gt;&gt;&gt; fn(0 if y &gt; 0 else -1)
0
</code></pre>
