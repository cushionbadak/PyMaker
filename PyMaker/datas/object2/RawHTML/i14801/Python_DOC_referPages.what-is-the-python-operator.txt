<div class="post-text" itemprop="text">
<p>What exactly is the <code>&lt;&gt;</code> operator in Python, and why is it undocumented (as far as I can tell)?</p>
<p>Is it the same as <code>!=</code> or <code>is not</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>In <a href="https://docs.python.org/2/reference/lexical_analysis.html#operators">Python 2.x</a>, <code>&lt;&gt;</code> is the same as <code>!=</code> (i.e. <em>"not equal to"</em>, rather than <code>is not</code> which is <em>"not identical to"</em>), but the latter is preferred:</p>
<blockquote>
<p>The comparison operators <code>&lt;&gt;</code> and <code>!=</code> are alternate spellings of the same operator. <code>!=</code> is the preferred spelling; <code>&lt;&gt;</code> is obsolescent.</p>
</blockquote>
<p>In 3.x, <code>&lt;&gt;</code> <a href="https://docs.python.org/3/whatsnew/3.0.html#removed-syntax">has been removed</a> and only <code>!=</code> exists.</p>
</div>
<div class="post-text" itemprop="text">
<p>It is documented, but you're not supposed to use it. Your guess about it being equivalent to <code>!=</code> is correct. Quoting the <a href="https://docs.python.org/2/library/stdtypes.html" rel="nofollow"><strong>Python 2</strong> documentation</a>:</p>
<blockquote>
<p><code>!=</code> can also be written <code>&lt;&gt;</code>, but this is an obsolete usage kept for backwards compatibility only. New code should always use <code>!=</code>.</p>
</blockquote>
<p>Then as part of the general cleanup of <strong>Python 3</strong>, the operator was <a href="https://docs.python.org/3/whatsnew/3.0.html#removed-syntax" rel="nofollow">removed entirely</a>:</p>
<blockquote>
<p>Removed <code>&lt;&gt;</code> (use <code>!=</code> instead).</p>
</blockquote>
<hr/>
<h3>Historical note</h3>
<p>It goes back quite a long way; at least as far as <strong>Python 1.4</strong>. I found an entry in the <a href="https://docs.python.org/release/1.4/lib/node7.html" rel="nofollow">old docs</a>:</p>
<blockquote>
<p><code>&lt;&gt;</code> and <code>!=</code> are alternate spellings for the same operator. (I couldn't choose between ABC and C! :-)</p>
</blockquote>
<p>The docs started recommending <code>!=</code> with Python 1.5.2p2.</p>
</div>
