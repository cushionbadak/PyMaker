<div class="post-text" itemprop="text">
<p><a href="http://docs.python.org/3.3/library/cgi.html" rel="nofollow">cgi.parse exists in python 3.3</a>, however you have a python file named cgi.py and it is finding that.</p>
<p>Here is where the badly named cgi.py file lives:</p>
<pre><code>&lt;module 'cgi' from 'C:\\Program Files (x86)\\Apache Software 
Foundation\\Apache2.2\\htdocs\\cgi.py'&gt;
</code></pre>
<p><a href="http://docs.python.org/3.1/whatsnew/3.0.html#builtins" rel="nofollow">python 3.3 does not have "has_key" though</a>...</p>
<p>So you'll get an error on this line:</p>
<pre><code>if pairs.has_key("word"):
</code></pre>
<p>Changing that to:</p>
<pre><code>if "word" in pairs:
</code></pre>
<p>Makes the code work properly for me.</p>

<div class="post-text" itemprop="text">
<p><a href="http://docs.python.org/3/library/cgi.html#cgi.parse" rel="nofollow"><code>cgi.parse()</code> exists in Python 3.3.</a> You've called some other file "cgi.py", and Python is finding this instead. Rename that file and any related .py[co] files.</p>
