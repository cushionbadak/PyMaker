<div class="post-text" itemprop="text">
<p>You are reraising the exception. It's working as designed.</p>
<p>The <code>test</code> is printed right at the top there before the traceback, but you used <code>raise</code> so the exception still is causing a traceback:</p>
<pre><code>&gt;&gt;&gt; def main():
...     try:
...         test = int("hello")
...     except ValueError:
...         print("test")
...         raise
... 
&gt;&gt;&gt; main()
test
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
  File "&lt;stdin&gt;", line 3, in main
ValueError: invalid literal for int() with base 10: 'hello'
</code></pre>
<p>Remove the <code>raise</code> and just the <code>test</code> print remains:</p>
<pre><code>&gt;&gt;&gt; def main():
...     try:
...         test = int("hello")
...     except ValueError:
...         print("test")
... 
&gt;&gt;&gt; main()
test
</code></pre>
</div>
<span class="comment-copy">This is expected as you are raising the exception using raise</span>
<span class="comment-copy">You should read the <a href="http://docs.python.org/3/reference/simple_stmts.html#the-raise-statement" rel="nofollow noreferrer">documentation</a> for <code>raise</code>.  The first sentence states: <code>If no expressions are present, raise re-raises the last exception that was active in the current scope.</code></span>
