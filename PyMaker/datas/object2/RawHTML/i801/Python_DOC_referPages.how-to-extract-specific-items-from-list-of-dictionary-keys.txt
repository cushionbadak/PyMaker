<p>In Python3, you can't slice a <a href="https://docs.python.org/3/library/stdtypes.html#dictionary-view-objects" rel="nofollow noreferrer">view</a> such as <a href="https://docs.python.org/3/library/stdtypes.html#dict.keys" rel="nofollow noreferrer"><code>dict.keys</code></a> in the same way as a <code>list</code>. But you can use <a href="https://docs.python.org/3/library/itertools.html#itertools.islice" rel="nofollow noreferrer"><code>itertools.islice</code></a> to slice <em>any</em> iterable and feed the result to <code>list</code>. Here's a demo:</p>
<pre><code>from itertools import islice

d = dict.fromkeys([f'section_00{i}' for i in range(1, 4)])
L = [d.keys() for _ in range(3)]

res = [list(islice(x, 0, 2)) for x in L]

# [['section_001', 'section_002'],
#  ['section_001', 'section_002'],
#  ['section_001', 'section_002']]
</code></pre>
</div>

<span class="comment-copy">Did you, by any chance wrote <code>some_dictionary.keys()</code>? Since <a href="https://stackoverflow.com/questions/tagged/python-3.x">python-3.x</a>, <code>some_dictionary.keys()</code> does <i>not</i> return a list, but some sort of <i>proxy object</i> that acts as a representation of the keys of the dictionary.</span>
<span class="comment-copy">You can call <code>list(some_dictionary.keys())</code> to convert it to a list.</span>
