<div class="post-text" itemprop="text">
<p>Your <code>%y</code> and <code>%Y</code> patterns are fine, the issue is that you used <code>%mmm</code> here. The <code>datetime.strptime()</code> method patterns are all <em>single letter</em> patterns, and <code>%mmm</code> is seen as the <code>%m</code> pattern followed by two literal <code>m</code> characters. <code>%m</code> matches a numeric month (1 or 2 digits, zero-padding optional). So <code>19-1mm-19</code> would match, but <code>19-Jan-19</code> does not because the month is not numeric and the two literal <code>m</code> characters are missing.</p>
<p>The correct pattern to use is <code>'%d-%b-%y'</code> here, where <code>%b</code> matches an abbreviated month name.</p>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; import datetime
&gt;&gt;&gt; df = ['19-Jan-19', '4-Jan-19']
&gt;&gt;&gt; [datetime.datetime.strptime(x,'%d-%b-%y').strftime('%Y-%m-%d') for x in df]
['2019-01-19', '2019-01-04']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Since you're specifying month in short hand notation, you should use %b instead of %mmm (That is not a valid format in datetime)</p>
</div>
