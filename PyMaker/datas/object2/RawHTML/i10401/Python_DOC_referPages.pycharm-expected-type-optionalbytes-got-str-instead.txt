<p>I am using <code>rsplit</code> to split up a path name, </p>
<pre><code>rootPath = os.path.abspath(__file__)
rootPath = (rootPath.rsplit('/', 1)[0]).rsplit('/', 1)[0]
</code></pre>
<p>But <strong>Pycharm</strong> warns,</p>
<blockquote>
<p>expected type <code>optional [bytes]</code>, got <code>str</code> instead</p>
</blockquote>
<p>In <code>python doc</code>, it stated <em>using <code>sep</code> as the delimiter string</em>. </p>
<p>So how to fix this?</p>
</div>
<div class="post-text" itemprop="text">
<p>It seems like <code>rootPath</code> is being treated as a bytes object (a small bug maybe?) or the warning is for another part. </p>
<p>In general, what PyCharm and the error is essentially warning you about is that the parameter has to either be <code>None</code> or <code>bytes</code>. That's  what <a href="https://docs.python.org/3/library/typing.html#typing.Optional" rel="nofollow noreferrer"><code>Optional</code></a> means, <code>Optional[type]</code> is either <code>None</code> or <code>type</code> which in your case is <code>bytes</code>.</p>
<p>In a simple Python REPL the message is slightly different but the gist is the same:</p>
<pre><code>b'hello/world'.rsplit('/') # error bytes-like object required
</code></pre>
<p>Instead you need to supply a <code>byte</code> separator:</p>
<pre><code>b'hello/world'.rsplit(b'/') 
</code></pre>
<p>or <code>None</code> in order to get it to work.</p>
<p>Either there's a small bug in PyCharm and it's reporting <code>rsplit</code> incorrectly here or the warning is for another part of your code.</p>
