<div class="post-text" itemprop="text">
<p>The answer by teivaz is correct, but maybe incomplete. The way you are using the ConfigParser object is almost correct in Python 3 (<a href="https://docs.python.org/3/library/configparser.html" rel="nofollow noreferrer">docs</a>), but not Python 2 (<a href="https://docs.python.org/2/library/configparser.html" rel="nofollow noreferrer">docs</a>).</p>
<p>Here is Python 2:</p>
<pre><code>import ConfigParser
config = ConfigParser.ConfigParser()
config.add_section('Testing')
config.set('Testing', 'name', 'Yohannes')
config.set('Testing', 'age', '10')  # note: string value for '10'!
</code></pre>
<p>And Python 3:</p>
<pre><code>import configparser  # note: lowercase module name
config = configparser.ConfigParser()
config['Testing'] = {'name': 'Yohannes', 'age': '10'}
</code></pre>
<p>Note: Python 2's <code>ConfigParser.set()</code> won't complain if you give it a non-string value (e.g. <code>config.set('Testing', 'age', 10)</code>), but it will throw a <code>TypeError</code> when you try to retrieve it. Python 3 will throw a <code>TypeError</code> when you use the <code>set()</code> method with a non-string value, but it will quietly convert values to strings with the <code>__setitem__</code> access. E.g.:</p>
<pre><code>config['Testing'] = {'name': 'Yohannes', 'age': 10}  # int value for 'age'
config['Testing']['age']  # returns '10' as a string, not an int
</code></pre>
</div>

<pre><code>config = ConfigParser.ConfigParser()
config.add_section('Testing')
config.set('Testing', 'name', 'Yohannes')

<blockquote>
<p><code>object.__setitem__(self, key, value)</code>
  Called to implement assignment to <code>self[key]</code>.</p>
</blockquote>
