<div class="post-text" itemprop="text">
<p><code>os.scandir()</code> returns an iterator containing the directory entries for a given path, see <code>help(os.scandir)</code>:</p>
<pre><code>scandir(...)
    scandir(path='.') -&gt; iterator of DirEntry objects for given path
</code></pre>
<p>To get a value from it call the built-in <code>next</code> on it:</p>
<pre><code>next(img_list2)
</code></pre>
<p>This will return values from the iterator until it is exhausted (and <code>StopIteration</code> is raised).</p>
<p>You can also wrap it in a <code>list</code> call which create a <code>list</code> out of all entries and make it indexable but if you only require the first element that is overkill. </p>
<p><code>iterator.next()</code> was removed as of <code>Python 3.0</code> with <a href="https://www.python.org/dev/peps/pep-3114/" rel="nofollow"><code>PEP 3114</code></a>; a dunder <code>iterator.__next__()</code> method replaced it along with the built-in <code>next()</code> that calls it.</p>
