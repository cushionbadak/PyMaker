<p>There are two variables. </p>
<p>A variable <strong>drive</strong> was assigned a drive path (string). 
A variable <strong>filepath</strong> was assigned a complete path to a file (string).</p>
<pre><code>drive = '/VOLUMES/TranSFER'
filepath = '/Volumes/transfer/Some Documents/The Doc.txt'
</code></pre>
<p>First I need to find if a string stored in a <strong>drive</strong> variable is in a string stored in <strong>filepath</strong> variable. 
If it is then I need to extract a string stored in <strong>drive</strong> variable from a string stored in a <strong>filepath</strong> variable without changing both variables characters case (no changing to lower or uppercase. The character case should stay the same).</p>
<p>So a final result should be:</p>
<p><strong><code>result = '/Some Documents/The Doc.txt'</code></strong></p>
<p>I could get it done with:</p>
<pre><code>if drive.lower() in filepath.lower(): result = filepath.lower().split( drive.lower()  )
</code></pre>
<p>But approach like this messes up the letter case (everything is now lowercase)
Please advise, thanks in advance!</p>
<h1>EDITED LATER:</h1>
<p>I coould be using my own approach. It appear IF portion of the statement</p>
<pre><code>if drive.lower() in filepath.lower():
</code></pre>
<p>is case-sensitive. And <strong>drive in filepath</strong> will return False if case doesn't match.
So it would make sense to lower()-case everything while comparing. But a .split() method splits beautiful regardless of the letter-cases:</p>
<pre><code>if drive.lower() in filepath.lower(): result = filepath.split( drive  )
</code></pre>
</div>

<pre><code>if filepath.lower().startswith(drive.lower() + '/'):
    result = filepath[len(drive)+1:]
</code></pre>
