<p><strong>Edit:</strong></p>
<p>On this line:</p>
<pre><code>bpy.ops.graph.sound_bake(filepath=r"C:\Users\Callum\Desktop\Teardrop.mp3", low= i*step, high=i*step+step)
</code></pre>
<p>the <code>\U</code> in the filepath string is being interpreted as an escape sequence:</p>
<pre><code>&gt;&gt;&gt; '\U'
  File "&lt;stdin&gt;", line 1
SyntaxError: (unicode error) 'unicodeescape' codec can't decode bytes in position 0-1: truncated \UXXXXXXXX escape
&gt;&gt;&gt;
</code></pre>
<hr/>
<p>To fix the problem, place an <code>r</code> before the string:</p>
<pre><code>bpy.ops.graph.sound_bake(filepath=r"C:\Users\Callum\Desktop\Teardrop.mp3", low= i*step, high=i*step+step)
</code></pre>
<p>This will convert it into a <a href="http://docs.python.org/3/reference/lexical_analysis.html#literals" rel="nofollow">raw-string</a>, which do not process escape sequences:</p>
<pre><code>&gt;&gt;&gt; r'\U'
'\\U'
&gt;&gt;&gt;
</code></pre>
<hr/>
<p>An alternate solution would be to turn the backslashes into forwardslashes:</p>
<pre><code>bpy.ops.graph.sound_bake(filepath="C:/Users/Callum/Desktop/Teardrop.mp3", low= i*step, high=i*step+step)
</code></pre>
<p>Even though you are running Windows, Python works just fine with forwardslashes in filepaths.</p>
