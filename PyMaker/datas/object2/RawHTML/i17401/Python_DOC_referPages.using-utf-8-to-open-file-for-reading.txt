<p>I am using the below code but need to open it for reading with utf-8 specified. How would I do that please?</p>
<pre><code>infile = file(logPath)
lines = infile.readlines()
</code></pre>

<div class="post-text" itemprop="text">
<p>Use <code>open</code> function of <a href="http://docs.python.org/2/library/codecs.html#codecs.open" rel="nofollow">codecs</a> module:</p>
<pre><code>import codecs

with codecs.open(logPath, encoding='utf8') as infile:
    lines = infile.readlines()
</code></pre>
<p>By default the <code>codecs.open</code> function, open the file in <code>rb</code> (read binary) mode:</p>
<blockquote>
<p>def open(filename, mode='rb', encoding=None, errors='strict',
  buffering=1):</p>
<pre><code>    ...
    Files are always opened in binary mode, even if no binary mode
    was specified. This is done to avoid data loss due to encodings
    using 8-bit values. The default file mode is 'rb' meaning to
    open the file in binary read mode.
</code></pre>
</blockquote>

<span class="comment-copy">No, in <code>Python 2</code>, you can't use <code>open(logPath, encoding='utf8')</code>, because it's a built-in <a href="http://docs.python.org/2/library/functions.html#open" rel="nofollow noreferrer">open</a> function, and is different from <code>codecs.open</code> function. But in <code>Python 3</code> you can use the built-in <a href="http://docs.python.org/3/library/functions.html#open" rel="nofollow noreferrer">open</a> function for that, however it is still different from <code>codecs.open</code> function.</span>
