<p>The magic methods <code>__and__</code>, <code>__or__</code> and <code>__invert__</code> are used to override the operators <code>a &amp; b</code>, <code>a | b</code> and <code>~a</code> respectively. That is, if we have a class</p>
<pre><code>class QueryImpl(object):
    def __and__(self, other):
        return ...
</code></pre>
<p>then </p>
<pre><code>a = QueryImpl(...)
b = QueryImpl(...)
c = a &amp; b
</code></pre>
<p>is equivalent to </p>
<pre><code>a = QueryImpl(...)
b = QueryImpl(...)
c = a.__and__(b)
</code></pre>
<p>These methods are overridden in <code>tinydb</code> to support this syntax:</p>
<pre><code>&gt;&gt;&gt; db.find(where('field1').exists() &amp; where('field2') == 5)
&gt;&gt;&gt; db.find(where('field1').exists() | where('field2') == 5)
#                                    ^
</code></pre>
