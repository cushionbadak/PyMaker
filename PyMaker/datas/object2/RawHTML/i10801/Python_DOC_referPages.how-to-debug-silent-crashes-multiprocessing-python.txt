<div class="post-text" itemprop="text">
<p>I have a program that spawns a bunch of worker daemons using the multiprocessing library. </p>
<p>Sometimes these daemons will sporadically fall over and die. This isn't caused by a python-level Exception. The crashes seem to be totally silent...</p>
<p>Can anyone provide tips on how I may go about debugging this issue?</p>
</div>
<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/faulthandler.html" rel="nofollow noreferrer">faulthandler</a> library might help debug such behavior. Run this code in every process <code>run</code> method to save tracebacks into separate files:</p>
<pre><code>f = open('fault_%s.log' % multiprocessing.current_process().name, 'w')
faulthandler.enable(file=f, all_threads=True)
</code></pre>
</div>
<span class="comment-copy"><code>multiprocessing.log_to_stderr()</code></span>
