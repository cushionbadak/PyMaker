<div class="post-text" itemprop="text">
<p>If you want to use <a href="https://docs.python.org/2/library/os.html#os.open" rel="nofollow">os.open</a> you have to use <a href="https://docs.python.org/2/library/os.html#open-constants" rel="nofollow">os flags</a> to identify mode in which you are going to open file:</p>
<pre><code>file = os.open("numbers.txt", os.O_WRONLY)
</code></pre>
<p>The way you are trying to open file is correct for <a href="https://docs.python.org/2/library/functions.html#open" rel="nofollow">built in open method</a></p>
<p><strong>Good Luck !</strong></p>

<p>From the <a href="https://docs.python.org/3/library/os.html" rel="nofollow">documentation</a> you linked:</p>
<blockquote>
<p>If you just want to read or write a file see <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow">open()</a>.</p>
</blockquote>
<p>As I said in the comment, change this:</p>
<pre><code>os.open(...)
</code></pre>
<p>To this:</p>
<pre><code>open(...)
</code></pre>

<p>Citing after Python docs available <a href="https://docs.python.org/2/library/os.html#os.open" rel="nofollow">here</a></p>
<blockquote>
<p>This function is intended for low-level I/O. For normal usage, use the built-in function open(), which returns a “file object” with read() and write() methods (and many more). To wrap a file descriptor in a “file object”, use fdopen().</p>
</blockquote>
<p>You should use the built-in <code>open</code> function as others has said already.</p>
