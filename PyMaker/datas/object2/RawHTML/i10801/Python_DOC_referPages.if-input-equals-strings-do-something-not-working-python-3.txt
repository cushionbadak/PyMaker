<pre><code>    b = input("Do you like video games? y/n")
    if b == "y":
        print("Good For You!")
    if b == "n":
        print("What!? (Just joking)")
</code></pre>
<p>I've researched a lot and this seems to be working for most other people. But when I use it I get this error:</p>
<pre><code>    Do you like video games? y/ny

      File "/home/ubuntu/workspace/Test.py", line 1, in &lt;module&gt;

<div class="post-text" itemprop="text">
<p>That error from using the <code>input</code> function is typical of Python 2 and not Python 3. I suspect you are actually using Python 2.</p>
<p>To determine which version of python runs when you use the <code>python</code> command:</p>
<pre><code>$ python --version
# example output: Python 2.7.3
</code></pre>
<p>To make sure you run your script using python 3, do:</p>
<pre><code>$ python3 scriptname.py
</code></pre>
<p><br/></p>
<h1>Python 2</h1>
<p>In Python 2, to get user input, you actually need to use the function <a href="https://docs.python.org/2/library/functions.html#raw_input" rel="nofollow"><code>raw_input</code></a>, as the <a href="https://docs.python.org/2/library/functions.html#input" rel="nofollow"><code>input</code></a> function tries to evaluate the user's input as if it were python code. </p>
<p>So when the user inputs <code>'y'</code>, the <code>input</code> function then tries to evaluate the <em>expression</em> <code>y</code>, which in this case would mean the <em>variable</em> named <code>y</code>, which doesn't exist.</p>
<p>So for python 2, your code should look like: </p>
<pre><code>b = raw_input("Do you like video games? y/n")
# etc
</code></pre>
<p><br/></p>
<h1>Python 3</h1>
<p>In python 3, the <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow"><code>input</code></a> function takes the place of <code>raw_input</code>, so your code would work as-is.</p>
</div>
<div class="post-text" itemprop="text">
<p>You are actually using Python 2, not 3 unfortunately. Since you are using <code>input()</code> in Python <em>2</em>, <code>input()</code> actually is evaluating the expression, instead of being Python 3's equivalent for Python 2's <code>raw_input()</code>. <code>input()</code> in this case, failed to find a variable with the name <code>y</code> (the input) and raised the error.</p>
<p>Simply change your Python version from 2.x to 3.x and you should be fine. If you are sticking with Python 2 though, use <code>raw_input()</code> instead:</p>
<pre><code>b = raw_input("Do you like video games? y/n")
</code></pre>
