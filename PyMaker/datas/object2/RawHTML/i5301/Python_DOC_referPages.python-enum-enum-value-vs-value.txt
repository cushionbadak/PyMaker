<div class="post-text" itemprop="text">
<pre><code>from enum import Enum
class Type(Enum):
  a = 1
  b = 2

print Type.a.value, Type.a._value_
</code></pre>
<p>This prints </p>
<pre><code>1 1
</code></pre>
<p>What is the difference between _ value_ and value?</p>
</div>
<div class="post-text" itemprop="text">
<p>The difference is that <code>.value</code> is the <code>property</code>-backed version and cannot be changed:</p>
<pre><code>&gt;&gt;&gt; from enum import Enum
&gt;&gt;&gt; class Color(Enum):
...   red = 1
...   green = 2
...   blue = 3
... 
&gt;&gt;&gt; Color.green
&lt;Color.green: 2&gt;
&gt;&gt;&gt; Color.green.value
2
&gt;&gt;&gt; Color(2)
&lt;Color.green: 2&gt;
&gt;&gt;&gt; Color.green.value = 3
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
  File "/usr/local/lib/python3.5/types.py", line 141, in __set__
    raise AttributeError("can't set attribute")
AttributeError: can't set attribute
</code></pre>
<p>But <code>._value_</code> is where the actual value is stored in the instance-dict, and can be changed:</p>
<pre><code>&gt;&gt;&gt; Color.green._value_ = 4
&gt;&gt;&gt; Color.green
&lt;Color.green: 4&gt;
</code></pre>
<p>As <a href="https://stackoverflow.com/a/48524795/208880">Tobias explained</a>, names that begin with an underscore should be avoided unless you have a really good reason, as you can break things using them:</p>
<pre><code>&gt;&gt;&gt; Color(4)
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
  File "/usr/local/lib/python3.5/enum.py", line 241, in __call__
    return cls.__new__(cls, value)
  File "/usr/local/lib/python3.5/enum.py", line 476, in __new__
    raise ValueError("%r is not a valid %s" % (value, cls.__name__))
ValueError: 4 is not a valid Color
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The docs are a bit unclear, but as I understand the docs, <code>_value_</code> is only to be used to set the value if you write your own <code>__new__</code> constructor. This will later result in <code>value</code> being set by the metaclass provided by the <code>enum</code> module.</p>
<p><a href="https://docs.python.org/3/library/enum.html#supported-sunder-names" rel="nofollow noreferrer">https://docs.python.org/3/library/enum.html#supported-sunder-names</a></p>
<p>Per convention, names starting with an underscore are "private" members of a class so definitely go with <code>.value</code>.</p>
</div>
