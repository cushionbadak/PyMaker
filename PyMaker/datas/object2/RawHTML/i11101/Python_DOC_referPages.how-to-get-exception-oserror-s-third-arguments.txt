<div class="post-text" itemprop="text">
<p>From <a href="https://docs.python.org/3/library/exceptions.html#OSError" rel="nofollow">Python document</a>:</p>
<blockquote>
<p>For backwards compatibility, if three arguments are passed, the args attribute contains only a 2-tuple of the first two constructor arguments.</p>
</blockquote>
<p>So How do we I the third arguments, like the 'invalid path'?</p>
<pre><code>try:
    open('invalid path')
except OSError as e:
    error = e
    print(error.args[0],error.args[1])
    print(e)
</code></pre>
<p>The results:</p>
<pre><code>2 No such file or directory
[Errno 2] No such file or directory: 'invalid path'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>From the same docs:</p>
<blockquote>
<p>For exceptions that involve a file system path (such as open() or
  os.unlink()), <strong>filename</strong> is the file name passed to the function. For
  functions that involve two file system paths (such as os.rename()),
  filename2 corresponds to the second file name passed to the function.</p>
</blockquote>
<p>So you simply do <code>error.filename</code>, because <code>filename</code> is an optional arg which only gets passed for certain types of errors.</p>
<p>To make a one for all check, you can use <code>hasattr</code>:</p>
<pre><code>if hasattr(error, 'filename'):
    print(error.filename)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>In your example, <code>error.args</code> contains <code>(2, 'No such file or directory')</code>.</p>
<p>In order to get the path, you should access <code>error.filename</code>:</p>
<pre><code>try:
    open('some/invalid path')
except OSError as e:
    print(e.filename)
    &gt;&gt; 'some/invalid path'
</code></pre>
</div>
