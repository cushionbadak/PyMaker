<p>That plist file isn't well formed xml at all, so I doubt it will work. I think you need (at a minimum) a root plist element and a dict wrapper:</p>
<pre><code>&lt;plist&gt;
  &lt;dict&gt;
    &lt;key&gt;Apple Car Version&lt;/key&gt;
    &lt;string&gt;1.0&lt;/string&gt;
  &lt;/dict&gt;
&lt;/plist&gt;
</code></pre>
<p>Then the keys of <code>pl</code> will be the keys you actually define in your file, so "Apple Car Version" in this case:</p>
<pre><code>print(pl['Apple Car Version'])
</code></pre>

<span class="comment-copy">I don't think an empty date is valid in a plist...I think the convention is to omit the key entirely if its value is null. In any case, plistlib doesn't support it, and as you saw will attempt to parse the entire file and fail if there is an empty date. If you need to pluck parts out of the file, and are willing to give up compatibility with the binary plist format, you could use any of the python xml parsers to get what you need directly out of the xml.
