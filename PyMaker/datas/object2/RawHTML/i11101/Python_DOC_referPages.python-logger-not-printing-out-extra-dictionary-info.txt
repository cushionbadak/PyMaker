<div class="post-text" itemprop="text">
<p>According to the <a href="https://docs.python.org/3/library/logging.html" rel="nofollow"><code>logging docs</code></a>, the <code>extra</code> keyword argument is used to add additional context to log records.</p>
<p>You can use info passed via <code>extra</code> parameter in the formatter by adding the dict key:</p>
<pre><code>format = '%(asctime)s - %(levelname)s - %(name)s - %(message)s - %(EXTRA_DICT_KEY)s'
</code></pre>
<p><em>Note that if you set up formatter like this and forget to pass the dict key, you'll get a string formatting exception.</em></p>
<hr/>
<p>As you can see in your example, you haven't specified any key in the formatter.</p>
<p>So, instead of:</p>
<pre><code>format = '%(asctime)s - %(levelname)s - %(name)s - %(message)s'
</code></pre>
<p>Try:</p>
<pre><code>format = '%(asctime)s - %(levelname)s - %(name)s - %(message)s - %(test)s'
</code></pre>
<hr/>
<ul>
<li>You may find some other useful info <a href="https://docs.python.org/3/library/logging.html#logging.Logger.debug" rel="nofollow">here</a>.</li>
</ul>
