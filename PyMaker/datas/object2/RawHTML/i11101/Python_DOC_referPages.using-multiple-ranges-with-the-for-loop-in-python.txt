<div class="post-text" itemprop="text">
<p>In <em>Python 2.7</em>, you can add <code>range</code>s because the result of calling <code>range(something)</code> <a href="https://docs.python.org/2.7/library/functions.html?highlight=range#range" rel="nofollow">is a list</a>, and lists <a href="https://docs.python.org/2.7/library/stdtypes.html#typesseq" rel="nofollow">can be added</a> to produce a new list:</p>
<pre><code>&gt;&gt;&gt; range(5)
[0, 1, 2, 3, 4]

&gt;&gt;&gt; range(8,12)
[8, 9, 10, 11]

&gt;&gt;&gt; range(5) + range(8,12)
[0, 1, 2, 3, 4, 8, 9, 10, 11]
</code></pre>
<p>But in <em>Python 3.5</em> you can't do this as the <code>range</code> type represents an <em>immutable sequence</em> of numbers. Check <a href="https://docs.python.org/3/library/stdtypes.html#typesseq-range" rel="nofollow">the docs</a> for more information.</p>
</div>
<div class="post-text" itemprop="text">
<p>Each range expression returns a list: <code>range(1,21)</code> returns <code>[1,2,...,20]</code>, and <code>range(30,100,10)</code> returns <code>[30,40,...,90]</code>. The <code>+</code> operator concatenates the two lists, so <code>range(1,21) + range(30,100,10)</code> evaluates to <code>[1,2,...,20,30,40,...,90]</code>.</p>
