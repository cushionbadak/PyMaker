<div class="post-text" itemprop="text">
<p><code>enumerate()</code> associates a sequence of integers with an iterable, i.e. it enumerates the items of the sequence. Its argument <code>start</code> is not meant to affect the starting position within the iterable, just the initial value from which to start counting.</p>
<p>One use is to start the enumeration from 1 instead of 0, e.g. if you wanted to number the lines in a file:</p>
<pre><code>with open('file.txt') as f:
    for line_number, line in enumerate(f, 1):
        print(line_number, line)
</code></pre>
<p>This outputs the line numbers starting from 1, which is where most users would expect line numbering to begin.</p>
<p>If you want to skip the first <em>n</em> items in a sequence you can just slice it:</p>
<pre><code>a = [2,3,4,5,6,7]
n = 2
for index, value in enumerate(a[n:]):
    print index, value
</code></pre>
<p>outputs</p>
<pre>
0 4
1 5
2 6
3 7
</pre>
<p>but you might like to start the enumeration from 3 as well:</p>
<pre><code>a = [2,3,4,5,6,7]
n = 2
for index, value in enumerate(a[n:], n+1):
    print index, value
</code></pre>
<p>which would output</p>
<pre>
3 4
4 5
5 6
6 7

</pre>
</div>
<div class="post-text" itemprop="text">
<p>Some people - most I suppose - aren't used to an enumerated list starting at zero. At the very least, this makes it easy to format the output so the enumeration starts at one, e.g.</p>
<pre><code>a = ['Fred', 'Ted', 'Bob', 'Alice']
for index, value in enumerate(a, start=1):
    print index, value
</code></pre>
<p>will print out:</p>
<pre><code>1 Fred
2 Ted
3 Bob
4 Alice
</code></pre>
<p>Without the <code>start=1</code> parameter, <code>enumerate(a)</code> would print</p>
<pre><code>0 Fred
1 Ted
2 Bob
3 Alice
</code></pre>

The <code>start</code> element just changes the index associated with the first element of the iterable.
