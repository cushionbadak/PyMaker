<div class="post-text" itemprop="text">
<p>As pointed out strings in python are immutable. Operations on them return new strings instead of changing the original. Your output requirement is also a list of integers, so you have to parse the strings:</p>
<pre><code>In [1]: x=[u'9-9', u'-5-5',u'-45-45',u'99-99']

In [2]: [int(i.rsplit('-', 1)[0]) for i in x]
Out[2]: [9, -5, -45, 99]
</code></pre>
<p>Using a <a href="https://docs.python.org/2/tutorial/datastructures.html#list-comprehensions" rel="nofollow">list comprehension</a> the string is <a href="https://docs.python.org/2/library/stdtypes.html#str.rsplit" rel="nofollow">split starting from right</a> 1 time only using <code>'-'</code> as delimiter and the first string is picked up and parsed as integer.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use a regular expression:</p>
<pre><code>import re

x = [u'9-9', u'-5-5',u'-45-45',u'99-99']
regex = "([-]?\d*)-.*"

x = [int(re.search(regex, m).group(1)) for m in x]
print(x)
# [9, -5, -45, 99]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>str.replace</code> is not in-place. It returns a new string with the replaced characters. Therefore:</p>
<pre><code>for i in x:
    string.replace(str(i),"-",' ')
</code></pre>
<p>Doesn't affect the strings in <code>x</code> whatsoever. </p>
<p>You should also call <code>replace</code> directly on the string object you have, instead of calling the general <code>replace</code> from the <code>string</code> module.</p>
<p>Instead, what you should do is:</p>
<pre><code>x = [u'9-9', u'-5-5',u'-45-45',u'99-99']
x_int = [int(string.replace('-', '')) for string in x]
print x_int
&gt;&gt; [99, 55, 4545, 9999]
</code></pre>
<p>And just a small bit of code to emphasis that <code>str.replace</code> returns a new string:</p>
<pre><code>x = [u'9-9', u'-5-5',u'-45-45',u'99-99']
for i in x:
    print i.replace(,"-",'')
    print i
&gt;&gt; 99
   9-9
   55
   -5-5
   4545
   -45-45
   9999
   99-99
</code></pre>
</div>

</code></pre>
