<p>Currently I'm writing a small python application where I get a date of the following format (<a href="https://tools.ietf.org/html/rfc3339" rel="nofollow">RFC3339</a>) from a webserver:</p>
<pre><code>2016-01-29T20:00:00+01:00
</code></pre>
<p>I have to parse this date to a python datetime object and for this I would like to use the <code>datetime.strptime</code> method where I have to specify the format of the date but I don't know how to parse the timezone (+01:00) I tried with:</p>
<pre><code>datetime.strptime(strDate, "%Y-%m-%dT%H:%M:%S%z")
</code></pre>
<p>but this does not work. Can you please tell me which format string I have to use?</p>

<div class="post-text" itemprop="text">
<p>You could use <code>dateutil</code>, however, if you are using Python 3 it only works for 3.2 or 3.3. It also supports Python 2 on 2.6 and 2.7.</p>
<p>The solution I would suggest:</p>
<pre><code>from dateutil.parser import parse

string = "2016-01-29T20:00:00+01:00"

date = parse(string)
</code></pre>
<p>This will give you a datetime object like so:</p>
<pre><code>Out[1]: 2016-01-29 20:00:00+01:00
Out[2]: datetime.datetime(2016, 1, 29, 20, 0, tzinfo=tzoffset(None, 3600))
</code></pre>
<p>If you'd like to know more, check dateutil <a href="https://labix.org/python-dateutil" rel="nofollow">documentation</a>.</p>
<p>Moreover, I believe the reason <code>strptime</code> doesn't work straight away here is due to that 'T' on your date string. It's not the format <code>strptime</code> expects. Luckily <code>dateutil</code> parse method works right out of the box to fix that string for you.</p>
</div>
