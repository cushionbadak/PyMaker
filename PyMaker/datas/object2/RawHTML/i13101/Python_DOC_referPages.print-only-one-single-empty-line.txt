<p>I am reading files in a folder in a python. I want print the each file content separate by a single empty line.</p>
<p>So, after the for loop I am adding <code>print("\n")</code> which adding two empty lines of each file content. How can I resolve this problem?</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>print()
</code></pre>
<p>will print a single new line in Python 3 (no parens needed in Python 2).</p>
<p>The docs for <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow noreferrer"><code>print()</code></a> describe this behavior (notice the <code>end</code> parameter), and <a href="https://stackoverflow.com/q/493386/113632">this question</a> discusses disabling it.</p>
</div>
<div class="post-text" itemprop="text">
<p>Because <code>print</code> automatically adds a new line, you don't have to do that manually, just call it with an empty string:</p>
<pre><code>print("")
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>From <code>help(print)</code> (I think you're using Python 3):</p>
<blockquote>
<p>print(<code>value</code>, ..., <code>sep=' '</code>, <code>end='\n'</code>, <code>file=sys.stdout</code>, <code>flush=False</code>)</p>
<p>Prints the values to a stream, or to sys.stdout by default.<br/>
     Optional keyword arguments:</p>
<p>file:  a file-like object (stream); defaults to the current sys.stdout.<br/>
     sep:   string inserted between values, default a space.<br/>
     end:   string appended after the last value, default a newline.<br/>
     flush: whether to forcibly flush the stream.</p>
</blockquote>
<p>So <code>print()</code>'s default <code>end</code> argument is <code>\n</code>. That means you don't need add a <code>\n</code> like <code>print('\n')</code>. This will print two newlines, just use <code>print()</code>.</p>
<p>By the way, if you're using Python 2,  use <code>print</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>print</code> has a <code>\n</code> embedded in it....so you don't need to add <code>\n</code> by yourself</p>
</div>
<div class="post-text" itemprop="text">
<p>Or, if you want to be really explicit, use</p>
<pre><code>sys.stdout.write('\n')
</code></pre>
<p>Write method doesn't append line break by default. It's probably a bit more intuitive than an empty print.</p>
</div>
