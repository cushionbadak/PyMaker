<div class="post-text" itemprop="text">
<p>When you use <code>import XXX</code>, you import all the contents of XXX under the namespace XXX, and you have access to them using XXX.abc, XXX.example etc...</p>
<p>When you use <code>from XXX import abc</code>, you only overwrite the variable abc of your <code>globals()</code> dictionnary. The special <code>from XXX import *</code> does the same, but for all variables whose name doesn't begins with an underscore.</p>
<p>Finally, the "as" keyword allows you to give to the imported module/function/variable the name you want.</p>
<p>When you have a module containing some folders, and you want to import from another file, . refers to the directory containing the current file, .. to the directory containing it, and so on.</p>
<p>For a less concise / more precise answer : <a href="https://stackoverflow.com/questions/9439480/from-import-vs-import">`from ... import` vs `import .`</a></p>
</div>

<pre><code>from . import foo, bar
</code></pre>
<p>It means import foo, bar from current directory.</p>
<pre><code>from foo import *
</code></pre>
<p>means from module foo import all items. But I think this is not a good practice.If you can and need only few items from a module do it like normal import rather than this one. For more info check <a href="https://www.python.org/dev/peps/pep-0328/" rel="nofollow">here.</a> </p>
