<div class="post-text" itemprop="text">
<p>That's because a default value is set to <code>y</code></p>
<p>Let's see with demo:</p>
<ul>
<li><p>with no default value</p>
<pre><code>&gt;&gt;b = lambda x, y: x + y
&gt;&gt;b()
TypeError: &lt;lambda&gt;() takes exactly 2 arguments (0 given)
&gt;&gt;b(1)
TypeError: &lt;lambda&gt;() takes exactly 2 arguments (1 given)
&gt;&gt;b(1,2)
3
</code></pre></li>
<li><p>With default value in <code>y</code></p>
<pre><code>&gt;&gt;b = lambda x, y=1: x + y
&gt;&gt;b()
TypeError: &lt;lambda&gt;() takes at least 1 argument (0 given)
&gt;&gt;b(1)
2
&gt;&gt;b(1,2)
3     
</code></pre></li>
<li><p>With two default value</p>
<pre><code>&gt;&gt;b = lambda x=2, y=1: x + y
&gt;&gt;b()
3
&gt;&gt;b(1)
2 
&gt;&gt;b(2,2)
4          
</code></pre></li>
</ul>
</div>
<span class="comment-copy"><code>y</code> is default parameter which is initialized to <code>2</code>. <i>P.s see the statement <code>y = 2</code></i></span>
<span class="comment-copy">You can read about <a href="https://docs.python.org/3/tutorial/controlflow.html#keyword-arguments" rel="nofollow noreferrer">keyword arguments</a> in the official tutorial. The examples there use <code>def</code> functions, but it works the same with <code>lambda</code>s.</span>
<span class="comment-copy">You passed it a value of <code>1</code> for <code>x</code>, it used the default value of <code>2</code> for <code>y</code>, and <code>1 + 2</code> is <code>3</code>.</span>
