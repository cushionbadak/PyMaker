<p>After deploying AskBot I observe an unusual error:</p>
<blockquote>
<p>AttributeError at /settings/QA_SITE_SETTINGS/<br/>
     'SortedDict' object has no attribute 'insert'</p>
</blockquote>
<p>This is the result of the following code:</p>
<pre><code>langs_dict = SortedDict(django_settings.LANGUAGES)
default_code = django_settings.LANGUAGE_CODE
default_name = langs_dict[default_code]
langs_dict.insert(0, default_code, default_name) 
</code></pre>
<p>According to <a href="https://code.djangoproject.com/wiki/SortedDict" rel="nofollow">Django wiki</a>, <code>SortedDict</code> seems to have <code>insert</code> method.</p>
<p>Any idea?</p>

<div class="post-text" itemprop="text">
<p>The <a href="https://docs.djangoproject.com/en/1.5/ref/utils/" rel="nofollow"><code>SortedDict.insert</code></a> method was deprecated in Django 1.5, and removed in Django 1.7. The <a href="https://code.djangoproject.com/wiki/SortedDict" rel="nofollow">wiki page</a> you linked to was out of date, so I removed the mention of the <code>insert</code> method from there.</p>
<p><code>SortedDict</code> itself is deprecated in Django 1.7, and will be removed in Django 1.9.</p>
<p>Now that Django only supports Python 2.7+, the correct fix is to use <code>collections.OrderedDict</code> instead of <code>SortedDict</code>.</p>
