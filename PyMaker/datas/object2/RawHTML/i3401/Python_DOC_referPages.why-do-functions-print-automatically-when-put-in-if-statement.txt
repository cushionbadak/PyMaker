<p>Why is it that when this code is executed, I would get 'hi'?</p>
<p>Thanks!      </p>
<pre><code>def b():
    print("hi")
def c():
    return True
if b() == 'hi':
    print("Done")
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You are confusing printing to the console with returning a value. Your function implicitly returns <code>None</code> if you do not return anything from it so it is never equal to <code>'hi'</code>. Your <code>b()</code> does print - and not return its <code>'hi'</code></p>
<pre><code>def b():
    print("hi")  # maybe uncomment it if you do not want to print it here
    return "hi"  # fix like this (which makes not much sense but well :o)

def c():
    return True
if b() == 'hi':
    print("Done")
</code></pre>
<p>You can test it like this:</p>
<pre><code>def test():
    pass

print(test())
</code></pre>
<p>Outputs:</p>
<pre><code>None
</code></pre>

<div class="post-text" itemprop="text">
<p>Essentially what you're doing is saying <code>if b()</code>, which runs the b() function and prints "hi" is equal to "hi", print "done", but since your function prints "hi", rather than returning "hi", it will never equal true.</p>
<p>Try this:</p>
<pre><code>def b():
    return "hi"
def c():
    return True
if b() == 'hi':
    print("Done")
</code></pre>
</div>
<span class="comment-copy"><code>if b()</code> is calling <code>b</code>, and a side effect of calling <code>b</code> is that 'hi' is printed - that's what calling <code>b</code> does.</span>
<span class="comment-copy">If there's no explicit <code>return</code> statement <code>None</code> is returned</span>
