<div class="post-text" itemprop="text">
<p>This line:</p>
<pre><code>if hiLo == 'l' and system_number == 1 or system_number == 2 or system_number == 3:
</code></pre>
<p>Is evaluated something like this:</p>
<pre><code>if   (hiLo == 'l' and system_number == 1): # Enter block
elif system_number == 2:                   # Enter block
elif system_number == 3:                   # Enter block
else:                                      # Don't enter block
</code></pre>
<p>So the <code>hiLo == 'l'</code> part is (currently) only affecting the first of your <code>system_number == 1</code> tests.  The code will enter the block irrespective of the value of <code>hiLo</code> if <code>system_number</code> is 2 or 3.</p>
<p>This is because the <code>or</code> operator has a <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow noreferrer"><strong>higher precedence</strong></a> than the <code>and</code> operator.</p>
<p>So you probably want something like:</p>
<pre><code>if hiLo == 'l' and (system_number == 1 or system_number == 2 or system_number == 3):
</code></pre>
<p>or</p>
<pre><code>if hiLo == 'l' and system_number in [1,2,3]:
</code></pre>
<p>or</p>
<pre><code>if hiLo == 'l' and (1 &lt;= and system_number &lt;= 3):
</code></pre>
</div>
