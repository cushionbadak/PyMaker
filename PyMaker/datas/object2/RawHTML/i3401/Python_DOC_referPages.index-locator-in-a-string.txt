<div class="post-text" itemprop="text">
<p>Use <code>str.find</code> and <code>str.rfind</code> to find the indices of the two characters. Using <code>str.rfind</code> its simpler to find the last character and avoids the possibility of out by one errors other approaches may be susceptible to. There is also a <code>str.rindex</code> which works a similar way.</p>
<p>Whether you prefer using <code>str.find</code>/<code>str.rfind</code> or <code>str.index</code>/<code>str.rindex</code> is a matter of preference on how you wish to handle the absence of the <code>first</code>/<code>last</code> strings. If you want an exception use <code>index</code> if you want a -1 in the returned tuple use <code>find</code> .</p>
<pre><code>def find_first_last(s, first='(', last=')'):
    return s.find(first), s.rfind(last)

print(find_first_last('a(bcdefghijkl(mno)p)q'))
# couple of error cases
print(find_first_last('a(bcdefghijkl(mnopq'))
print(find_first_last('abcdefghijklmno)p)q'))
print(find_first_last('abcdefghijklmnopq'))
print(find_first_last('abcdefgh)(ijklmnopq'))
</code></pre>
<p>output</p>
<pre><code>(1, 19) &lt;- all good
(1, -1)
(-1, 17)
(-1, -1)
(9, 8) &lt;- backwards
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>This function finds the indices of the first opening and final closing parentheses:</p>
<pre><code>def find_first_last(string, first='(', last=')'):
    return (
        string.index(first),
        len(string) - string[::-1].index(last)
    )

print(find_first_last('a(bcdefghijkl(mno)p)q'))
# (1, 20)
</code></pre>
<p>It does so by using the <a href="https://docs.python.org/3/library/stdtypes.html#str.index" rel="nofollow noreferrer"><code>index</code></a> string method, which returns the <em>lowest</em> index of a character in a string. For the closing parenthesis it reverses the string to find the last occurrence of that character.</p>
