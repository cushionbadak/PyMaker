<div class="post-text" itemprop="text">
<p>I have a directory structure</p>
<pre><code>foobar/
    __init__.py
    baz.py
</code></pre>
<p>where <code>__init__.py</code> is just</p>
<pre><code>from .baz import do_something
</code></pre>
<p>and <code>baz.py</code></p>
<pre><code>def do_something():
    return
</code></pre>
<p>Now, when I do <code>import foobar</code>, I get access to</p>
<pre><code>foobar
foobar.do_something()
</code></pre>
<p>Nice! However, one also has</p>
<pre><code>foobar.baz
</code></pre>
<p>although it's never explicitly imported. I suppose it has been imported along with <code>do_something</code>.</p>
<p>Is there a way to make <code>do_something</code> available as <code>foobar.do_something</code> without adding <code>foobar.baz</code> to the namespace? (And without putting everything in in <code>__init__.py</code>.)</p>
</div>
<div class="post-text" itemprop="text">
<p>This behaviour is documented in <a href="https://docs.python.org/3/reference/import.html#submodules" rel="nofollow noreferrer">chapter 5 of the Python Language Reference</a>. If you <em>really</em> want to disable it, you can do this in <code>__init__.py</code>:</p>
<pre><code>from .baz import do_something
del baz
</code></pre>
<p>which removes <code>baz</code> from <code>foobar</code>'s namespace when imported.</p>
<p>However, this isn't common practice as far as I know; is there a particular reason you're wanting to do this?</p>
</div>
