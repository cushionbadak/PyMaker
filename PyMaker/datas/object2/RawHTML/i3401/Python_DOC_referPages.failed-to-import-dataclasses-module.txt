<p>As suggested by @wim <code>python3.7 -m venv venv_dir</code></p>
<p>This command will:</p>
<ol>
<li>Use python3.7 to run the command</li>
<li>The <code>-m</code> flag tells the interpreter to run the next argument as a script</li>
<li><code>venv</code> is a module, and because of the <code>-m</code> flag it will be run as a script</li>
<li>Finally, the <code>venv_dir</code> is given to the <code>venv</code> module as an argument which
this module will use to create a virtual environment directory at</li>
</ol>
<p>Once this command is run now you'll have a nice sandbox for messing around/testing the dataclasses module.</p>
<ul>
<li>To activate this virtual environment be sure to run <code>source venv_dir/bin/activate</code> before you begin. This command will run the script at <code>venv_dir/bin/activate</code> to set up the necessary environment variables and other things for you</li>
<li>To deactivate, simply run <code>deactivate</code> after activating</li>
</ul>
