<p>This error message...</p>
<pre><code>File "/usr/lib/python2.7/httplib.py", line 402, in_read_status raise BadStatusLine(line) BadStatusLine
</code></pre>
<p>...implies that the <strong>BadStatusLine</strong> was raised as the server responded with a HTTP status code that we don’t understand.</p>
<hr/>
<h2>exception httplib.BadStatusLine <a href="https://docs.python.org/2/library/httplib.html" rel="nofollow noreferrer">Python2.x</a>/<a href="https://docs.python.org/3/library/http.client.html" rel="nofollow noreferrer">Python3.x</a></h2>
<p><a href="https://docs.python.org/2/library/httplib.html#httplib.BadStatusLine" rel="nofollow noreferrer"><strong>exception httplib.BadStatusLine</strong></a> is defined in <a href="https://docs.python.org/2/library/httplib.html" rel="nofollow noreferrer">Python2.x - httplib</a> / <a href="https://docs.python.org/3/library/http.client.html" rel="nofollow noreferrer">Python3.x - http.client</a> module and is subclass of <a href="https://docs.python.org/2/library/httplib.html#httplib.HTTPException" rel="nofollow noreferrer"><strong>HTTPException</strong></a> which is raised if a server responds with a HTTP status code that we don’t understand.</p>
<hr/>
<h2>Reasons and Solutions</h2>
<p>There can be many reason behind seeing the <strong>httplib.BadStatusLine</strong> exception. As per <a href="https://stackoverflow.com/questions/1767934/why-am-i-getting-this-error-in-python-httplib">Why am I getting this error in python ? (httplib)</a> some of the pottential reasons and solutions are:</p>
<ul>
<li>You may be trying to use <code>http://</code> instead of <code>https://</code></li>
<li>The <em>URL</em> string may be containing a trailing newline character. So make sure that your <em>URL</em> are stripped of any leading or trailing special characters.</li>
<li>The <em>Web Server</em> may be down and <em>Not Responding</em>.</li>
<li>The <em>Server</em> may be closing the <em>connection</em> before sending a valid response.</li>
<li>Presence of <code>Content-Length</code> within the <strong><code>http header</code></strong> can also create this exception.</li>
<li>If the time interval between two requests are more than he configured time of <code>Keep-Alive timeout=n</code>.</li>
<li>An easy solution would be to establish a new connection through <strong><code>conn.connect()</code></strong> before sending each request.</li>
</ul>
