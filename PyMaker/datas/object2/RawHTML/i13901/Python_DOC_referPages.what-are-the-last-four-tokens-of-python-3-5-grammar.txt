<p>What are last four tokens about in the token definition of Python 3.5 grammar?</p>
</div>
<div class="post-text" itemprop="text">
<h3>token.OP</h3>
<p>Is a generalization of the operator tokens. This is also mentioned in the <a href="https://docs.python.org/3/library/tokenize.html" rel="nofollow">tokenize</a> module:</p>
<blockquote>
<p>To simplify token stream handling, all Operators and Delimiters tokens are returned using the generic <code>token.OP</code> token type. The exact type can be determined by checking the <code>exact_type</code> property on the named tuple returned from <code>tokenize.tokenize()</code>.</p>
</blockquote>
<h3>token.ERRORTOKEN</h3>
<p>Is used to mark errors within the tokenize-process of the parser. This is mostly used to generate syntax errors that <a href="https://hg.python.org/cpython/file/b339a35e2e77/Parser/parsetok.c#l208" rel="nofollow">abort the parsing process</a>.</p>
<p>It’s also mentioned in the <a href="https://docs.python.org/3/library/tokenize.html#tokenize.TokenError" rel="nofollow">tokenize documentation</a>:</p>
<blockquote>
<p>Note that unclosed single-quoted strings do not cause an error to be raised. They are tokenized as <code>ERRORTOKEN</code>, followed by the tokenization of their contents.</p>
</blockquote>
<h3>token.N_TOKENS</h3>
<p>Is simply the number of tokens that are defined. It’s used in the parser for iterating over the list of tokens.</p>
<h3>token.NT_OFFSET</h3>
<p>Is used in <code>token.h</code> like this:</p>
<pre class="lang-c prettyprint-override"><code>/* Special definitions for cooperation with parser */
#define NT_OFFSET       256

#define ISTERMINAL(x)       ((x) &lt; NT_OFFSET)
#define ISNONTERMINAL(x)    ((x) &gt;= NT_OFFSET)
</code></pre>
<p>It basically separates <a href="https://en.wikipedia.org/wiki/Terminal_and_nonterminal_symbols" rel="nofollow">terminal and non-terminal tokens</a>.</p>
</div>
