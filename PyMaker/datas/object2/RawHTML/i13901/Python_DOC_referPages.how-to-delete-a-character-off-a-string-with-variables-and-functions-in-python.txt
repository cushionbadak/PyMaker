<div class="post-text" itemprop="text">
<p>You can pass the whole range to <code>print()</code> in one go:</p>
<pre><code>print(*range(50, 150), sep='; ')
</code></pre>
<p>Note the <code>*</code>; it expands the values produced by the <code>range()</code> into separate arguments to <code>print()</code>, which then uses the <code>sep</code> argument between those arguments.</p>
<p>The alternative would be to use the <a href="https://docs.python.org/3/library/stdtypes.html#str.join" rel="nofollow"><code>str.join()</code> method</a> to join the integers into a long string and <em>then</em> print those values, but you'd have to convert them all to strings first:</p>
<pre><code>print('; '.join(map(str, range(50, 150))))
</code></pre>
<p>Here the <a href="https://docs.python.org/3/library/functions.html#map" rel="nofollow"><code>map()</code> function</a> converts each integer yielded by <code>range()</code> into a <code>str</code> object.</p>
<p>If you <em>must</em> use a loop, then you should loop to 148 and use an extra, <em>separate</em> <code>print()</code> call to print the last value, and use the <code>end</code> argument to produce your separator:</p>
<pre><code>for x in range(50, 149):
    print(x, end='; ')
print(149)
</code></pre>
<p>Using <code>str.rstrip()</code> would only work if you had already built the whole string and needed to remove the trailing <code>'; '</code> from that string. It won't work when using <code>print()</code>, as you don't have the whole string to operate on at that time.</p>
</div>
