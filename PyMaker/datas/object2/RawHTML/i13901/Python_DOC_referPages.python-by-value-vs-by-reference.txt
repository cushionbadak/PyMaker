<p>Until recently, I thought Python passed parameters by value. For example, </p>
<pre><code>def method1(n):
   n = 5
   return n

n = 1
m = method1(n)
&gt;&gt;&gt; n == 5
&gt;&gt;&gt; False
</code></pre>
<p>But if I pass in a list to a method, like</p>
<pre><code>def method2(list):
   del list[0]
   return list

list1 = [0,1,2,3,4,5]
list2 = method2(list1)
&gt;&gt;&gt; list1
&gt;&gt;&gt; [1,2,3,4,5]
</code></pre>
<p>it mutates the list. I did another test:</p>
<pre><code>l1 = ['a','b','c','d']
x = method1(l1)
&gt;&gt;&gt; l1
&gt;&gt;&gt; ['a','b','c','d']
</code></pre>
<p>Here the list did not change. My question is</p>
<blockquote>
<p>Why do these different cases happen?</p>
</blockquote>

<div class="post-text" itemprop="text">
<p>Python always <a href="http://docs.python.org/3/faq/programming.html#how-do-i-write-a-function-with-output-parameters-call-by-reference" rel="nofollow">passes by assignment</a> , even in the first case, actually the object reference is passed. But when you do an assignment inside the function, you are causing the name to refer a new object. That would not mutate the name/variable that was used to pass the value to the function (Better explained in the below example).</p>
<p>Example -</p>
<pre><code>&gt;&gt;&gt; def func(n):
...     n = [1,2,3,4]
...     return n
...
&gt;&gt;&gt; l = [1,2,3]
&gt;&gt;&gt; func(l)
[1, 2, 3, 4]
&gt;&gt;&gt; l
[1, 2, 3]
</code></pre>
<hr/>
<p>In your case, when you do <code>del list[0]</code> you are actually mutating the <code>list</code> in place (deleting the first element from it) , and hence it reflects in the place where the function was called from as well.</p>
