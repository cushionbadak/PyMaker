<p>How to print something in Python without added line feed at the end?</p>
<p>Something like <code>printf("abc")</code> (no <code>\n</code> at the end) in C or <code>process.stdout.write("abc")</code> in Node.js.</p>
</div>
<div class="post-text" itemprop="text">
<h3>Python 3:</h3>
<p>Use the <code>end</code> parameter of the <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow"><code>print</code> function</a> to overwrite its default <code>'\n'</code> value, so that the function will not add a new line at the end:</p>
<pre><code>print(text, end='')
</code></pre>
<h3>Python 2:</h3>
<p>Add a trailing comma to the <a href="https://docs.python.org/2/reference/simple_stmts.html#the-print-statement" rel="nofollow"><code>print</code> statement</a>:</p>
<pre><code>print text,
</code></pre>
<hr/>
<p>If you need a solution that works for both Python 2 and Python 3, you should consider importing the <a href="https://docs.python.org/2/library/functions.html#print" rel="nofollow"><code>print</code> function in Python 2</a>, so you can use the above Python 3 syntax on both. You can do that with the following line at the top of your module:</p>
<pre><code>from __future__ import print_function
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>In python 2 (although this adds an extra space at the end of the text):</p>
<pre><code>print "abc",
</code></pre>
<p>In python 3 (or python2 with <code>from __future__ import print_function</code>):</p>
<pre><code>print("abc", end="")
</code></pre>
<p>This works in both 2 and 3:</p>
<pre><code>import sys
sys.stdout.write("abc")
</code></pre>
</div>
