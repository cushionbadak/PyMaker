<p>It's a known bug:</p>
<p><a href="http://code.google.com/p/selenium/issues/detail?id=5923" rel="noreferrer">http://code.google.com/p/selenium/issues/detail?id=5923</a></p>
<p>It's safe to ignore it though. If you're using Python 3, you can do:</p>
<pre><code>unittest.main(warnings='ignore')
</code></pre>
<p>See <a href="http://docs.python.org/3/library/unittest.html#unittest.main" rel="noreferrer">Python 3 unittest docs</a>. </p>
<p>In Python 2, you'd use something like this:</p>
<pre><code>with warnings.catch_warnings(record=True):
     unittest.main()
</code></pre>
<p>cf the <a href="http://docs.python.org/2/library/warnings.html#testing-warnings" rel="noreferrer">Python 2 warnings docs</a></p>
<p>If you'll forgive the shameless self-promotion, there's lots more info on selenium in a wee book I've written, <a href="http://www.obeythetestinggoat.com" rel="noreferrer">here</a>.</p>

<p>I run my tests with the <code>-W</code> flag thus:   </p>
<pre><code>python -W ignore -m unittest my_tests  
</code></pre>
<p>or  </p>
<pre><code>python -W ignore -m unittest my_tests.MyIndividualTest     
</code></pre>
<p>that suppresses the <code>ResourceWarning</code> but still allows assertion error reports.  </p>
<hr/>
<p>BTW I found that:  </p>
<pre><code>if __name__ == '__main__':
    unittest.main(warnings='ignore')  
</code></pre>
<p>works when invoking <code>python my_tests.py</code> when every test is to be run, but this invocation precludes running individual tests.  </p>
<p>I couldn't figure out how else to use <code>unittest.main(warnings='ignore')</code> without encountering errors I put down to recursive inclusion of <code>unittest</code> libraries. </p>
<p>(<code>selenium==2.44.0</code> under <code>Python 3.4.2</code>)</p>
