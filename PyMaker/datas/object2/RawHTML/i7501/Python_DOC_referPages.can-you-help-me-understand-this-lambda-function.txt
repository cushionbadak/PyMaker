<p>Consider the following:</p>
<pre><code>li = [2, 1, 3]
sorted_li = sorted(li, key=lambda x: x)
</code></pre>
<p>This example is very basic (and for the sake of the example, needlessly uses <code>key</code> and a lambda).</p>
<p><code>sorted</code> passes each element in <code>li</code> to the function it gets for the <code>key</code> argument and expects to get back a value that will tell it how to sort the list (basically, the <code>key</code> function tells <code>sorted</code> how the elements compare to each other).</p>
<p>In the above example the <code>key</code> function receives an element, refers to it as a local variable <code>x</code> and simply returns it as it is.</p>
<p>In your example, it sums the digits (in your example <code>x</code> is a group of digits).</p>
</div>
<div class="post-text" itemprop="text">
<p>The lambda is being passed to <code>sorted</code> as the argument <code>key</code>. Check out the <a href="https://docs.python.org/3/library/functions.html#sorted" rel="nofollow noreferrer">documentation</a> for the <code>sorted()</code> function. </p>
<blockquote>
<p>key specifies a function of one argument that is used to extract a comparison key from each list element: key=str.lower. The default value is None (compare the elements directly).</p>
</blockquote>
<p>In simple terms, <code>key</code>, is a function that is effectively used as the comparator for the sort. <code>sorted</code> sorts the list based on it's output, instead of figuring out the order the elements should be in on its own.</p>
<p>The lambda you pass to <code>sorted()</code> takes <code>x</code> as it's only argument. <code>sorted()</code> then passes each element of the list it is sorting to that lambda, so any reference to <code>x</code> in the lambda corresponds to the element.</p>
<pre><code>key=lambda x: sum(int(c) for c in x)))
#^         ^
#Key arg   x is the lambda's only argument
</code></pre>
<p>In your case, the element that is being passed to the lambda is a string, representing the number who's digits you want to get the sum of. The lambda then iterates through them, converts them to integers, and returns their sum.</p>
<p>Once that is done, <code>sorted()</code> uses those sums to sort the strings in ascending order.</p>
</div>
<div class="post-text" itemprop="text">
<p>The signature for <a href="https://docs.python.org/3.6/library/functions.html#sorted" rel="nofollow noreferrer"><code>sorted</code></a> is <code>sorted(iterable[, key][, reverse])</code>.</p>
<blockquote>
<p>key specifies a function of one argument that is used to extract a comparison key from each list element: key=str.lower. The default value is None (compare the elements directly).</p>
</blockquote>
<p>So, for the default value of <code>key=None</code>, <code>sorted</code> will just sort by comparing the values directly. (i.e. <code>[2,3,1]</code> is sorted by comparing <code>2 &lt; 3</code>, etc.) Now if you specify a function <code>foo</code> for <code>key</code>, that accepts one parameter and returns a value, <code>sorted</code> will instead use the value returned by <code>foo</code> in the comparison (i.e. <code>[2,3,1]</code> is sorted by comparing <code>foo(2) &lt; foo(3)</code> etc.)</p>
<p>In your case: <code>sorted(_str.split(' ')), key=lambda x: sum(int(c) for c in x))</code></p>
<p>creates a <code>list</code> by splitting the string, and sorts it with <code>key=lambda x: sum(int(c) for c in x)</code>, so it will pass each space separated substring <code>x</code> to the lambda in <code>key</code>, which will return the comparison value by summing its digits <code>sum(int(c) for c in x)</code>.</p>
<p>Note that this doesn't need a lambda function, it just needs a function with the right signature - that is, takes one parameter and returns one that can be compared. So we could define:</p>
<pre><code>def foo(x):
    return sum(int(c) for c in x)
</code></pre>
<p>and pass <code>foo</code> instead.</p>
<pre><code>&gt;&gt;&gt; print( ' '.join(sorted(sorted(_str.split(' ')), key=foo)) )
128 256 3217 555
</code></pre>
</div>
