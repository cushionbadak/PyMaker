<p>I'm writing a django webhook for a service that send data via <code>POST</code> in what I believe is the <code>application/x-www-form-urlencoded</code> format. Example <code>POST</code> show below:</p>
<pre><code>POST id=a5f3ca18-2935-11e7-ad46-08002720e7b4
 &amp;originator=1123456789
 &amp;recipient=1987654321
 &amp;subject=MMS+reply
 &amp;body=View+our+logo
 &amp;mediaUrls[0]=https://storage.googleapis.com/mms-assets/20170424/a0b40b77-30f8-4603-adf1-00be9321885b-messagebird.png
 &amp;mediaContentTypes[0]=image/png
 &amp;createdDatetime=2017-04-24T20:15:30+00:00
</code></pre>
<p>I understand how to parse <code>json</code> but I haven't encountered this format before. There doesn't appear to be any useful tutorials for how to handle this via <code>POST</code>. I'm stuck at this point so help would be greatly appreciated. </p>

<div class="post-text" itemprop="text">
<p>Python 2:</p>
<pre><code>&gt;&gt;&gt; from urlparse import parse_qs
&gt;&gt;&gt; parse_qs('foo=spam&amp;bar=answer&amp;bar=42')
{'foo': ['spam'], 'bar': ['answer', '42']}
</code></pre>
<p>Python 3:</p>
<pre><code>&gt;&gt;&gt; from urllib.parse import parse_qs
&gt;&gt;&gt; parse_qs('foo=spam&amp;bar=answer&amp;bar=42')
{'foo': ['spam'], 'bar': ['answer', '42']}
</code></pre>
<p>Both python 2/3:</p>
<pre><code>&gt;&gt;&gt; from six.moves.urllib.parse import parse_qs
</code></pre>
<p><strong>UPD</strong></p>
<p>There is also <code>parse_qsl</code> function that returns a list of two-items tuples, like</p>
<pre><code>&gt;&gt;&gt; parse_qsl('foo=spam&amp;bar=answer&amp;bar=42')
[('foo', 'spam'), ('bar', 'answer'), ('bar', '42')]
</code></pre>
<p>It is very suitable to passing such list to <code>dict()</code> constructor, meaning that you got a dict with only one value per name. Note that the last name/value pair takes precedence over early occurrences of same name (see <a href="https://docs.python.org/3/library/stdtypes.html#dict" rel="noreferrer">dict</a> in library reference).</p>
