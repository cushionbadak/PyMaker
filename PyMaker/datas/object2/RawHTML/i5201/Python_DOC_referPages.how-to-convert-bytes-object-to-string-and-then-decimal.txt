<div class="post-text" itemprop="text">
<p>In order to get a <a href="https://docs.python.org/3/library/functions.html#func-str" rel="nofollow noreferrer"><code>str</code></a> from <a href="https://docs.python.org/3/library/functions.html#func-bytes" rel="nofollow noreferrer"><code>bytes</code></a> object, you need to firstly decode your <a href="https://docs.python.org/3/library/functions.html#func-bytes" rel="nofollow noreferrer"><code>bytes</code></a> object using <a href="https://docs.python.org/3/library/stdtypes.html#bytes.decode" rel="nofollow noreferrer"><code>bytes.decode()</code></a> function as:</p>
<pre><code>&gt;&gt;&gt; my_bytes = b'6016.0000'

&gt;&gt;&gt; my_str = my_bytes.decode()
# where `my_str` holds the string value `'6016.0000'`
</code></pre>
<p>Or you may get the same result by passing <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow noreferrer"><code>encoding</code></a> parameter while type-casting your bytes object to string as:</p>
<pre><code>&gt;&gt;&gt; my_str = str(my_bytes, encoding='utf-8')
# Here also `my_str` holds the string value `'6016.0000'`
</code></pre>
<p>As the <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow noreferrer">Python's <code>str</code> document</a> says:</p>
<blockquote>
<p>if object is a <a href="https://docs.python.org/3/library/stdtypes.html#bytes" rel="nofollow noreferrer"><code>bytes</code></a> (or <a href="https://docs.python.org/3/library/stdtypes.html#bytearray" rel="nofollow noreferrer"><code>bytearray</code></a>) object, then <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow noreferrer"><code>str(bytes, encoding, errors)</code></a> is equivalent to <a href="https://docs.python.org/3/library/stdtypes.html#bytes.decode" rel="nofollow noreferrer"><code>bytes.decode(encoding, errors)</code></a></p>
</blockquote>
<p>Then in order to type-cast your number string <code>my_str</code> to <a href="https://docs.python.org/3.0/library/decimal.html#decimal.Decimal" rel="nofollow noreferrer"><code>decimal.Decimal</code></a>, you may simply do:</p>
<pre><code>&gt;&gt;&gt; from decimal import Decimal

&gt;&gt;&gt; Decimal(my_str)
Decimal('6016.0000')
</code></pre>
<hr/>
<p>In your code, when you type-casted a bytes object using <a href="https://docs.python.org/3/library/functions.html#func-str" rel="nofollow noreferrer"><code>str()</code></a> without the <code>encoding</code> parameter, it returned you a new string having <code>b'...'</code> at the start as:</p>
<pre><code>&gt;&gt;&gt; str(my_bytes)
"b'6016.0000'"
</code></pre>
<p>This behavior is also mentioned in <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow noreferrer">Python's <code>str</code> document</a>:</p>
<blockquote>
<p>Passing a <a href="https://docs.python.org/3/library/stdtypes.html#bytes" rel="nofollow noreferrer"><code>bytes</code></a> object to <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow noreferrer"><code>str()</code></a> without the encoding or errors arguments falls under the first case of returning the informal string representation (see also the <a href="https://docs.python.org/3/using/cmdline.html#cmdoption-b" rel="nofollow noreferrer"><code>-b</code> command-line option</a> to Python). </p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>Instead of strip(), try <code>Decimal(s.decode())</code>. This makes the bytes object <code>b'6016.0000'</code> a string and should work great.</p>
