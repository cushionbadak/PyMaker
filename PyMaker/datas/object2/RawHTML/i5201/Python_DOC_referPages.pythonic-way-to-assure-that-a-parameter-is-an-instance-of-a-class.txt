<div class="post-text" itemprop="text">
<p>Let's say that I have the following class:</p>
<pre><code>class MyClass(object):
    def __init__(self):
    ...
</code></pre>
<p>And also this function:</p>
<pre><code>def foo(my_class):
    ...
</code></pre>
<p>My question is: <strong>what is the pythonic way to assure that <code>my_class</code> parameter is an instance of <code>MyClass()</code>?</strong></p>
</div>
<div class="post-text" itemprop="text">
<p>Use the <code>isinstance</code> builtin:</p>
<pre><code>isinstance(my_class, MyClass)
</code></pre>
<p>Hope it helps.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use <code>assert</code> along with <code>isinstance</code>:</p>
<pre><code>assert isinstance(my_class, MyClass)
</code></pre>
<p>If <code>my_class</code> is not instance of <code>MyClass</code> an exception would be thrown.</p>
</div>
<div class="post-text" itemprop="text">
<p>May be simple one is, <a href="https://docs.python.org/3/library/functions.html#isinstance" rel="nofollow noreferrer">more help</a></p>
<pre><code>isinstance(obj, MyClass)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Another approach would be:</p>
<pre><code>type(obj) is MyClass
</code></pre>
<p>But the <strong>Pythonic way</strong>, of course:</p>
<pre><code>isinstance(my_class, MyClass)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The pythonic way of assuring is not checking at all. Let <a href="https://en.wikipedia.org/wiki/Duck_typing" rel="nofollow noreferrer">duck typing</a> do the job. Or you can use Python 3's type hints like said in the comments section. Otherwise, specifically check the type if you <strong>really really</strong> need to.</p>
</div>
