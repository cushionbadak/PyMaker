<p>From the official docs on the <a href="https://docs.python.org/3/library/logging.handlers.html#httphandler" rel="nofollow noreferrer">HTTPHandler class</a>:</p>
<blockquote>
<p><code>class logging.handlers.HTTPHandler(host, url, method='GET', secure=False, credentials=None, context=None)</code></p>
<p>Returns a new instance of the HTTPHandler class. The host can be of the form <code>host:port</code>, should you need to use a specific port number.</p>
</blockquote>
<p>There is no documentation of the <code>url</code> parameter, and I'm not quite sure what to put there.</p>
<p>For example, if I have a server listening on <code>http://localhost:8080</code>, then I think I should set:</p>
<ul>
<li><code>host='localhost:8080'</code></li>
<li><code>url='http://localhost:8080'</code></li>
</ul>
<p>But wouldn't that always be redundant information?</p>
<p>Or else, what should I set the <code>url</code> parameter to?</p>
<p>Thank you.</p>
</div>
<div class="post-text" itemprop="text">
<p>You only need to add the path, like:</p>
<pre><code>host='localhost:8080'
path='/'
handler = logging.handlers.HTTPHandler(host, path)
</code></pre>
</div>
<span class="comment-copy">Thank you. Why just a path, when the argument is <code>url</code>?</span>
<span class="comment-copy">Maybe it's not well documented. But <code>HTTPHandler</code> internally uses <code>http.client</code>, which will handle path. See comment in <code>putrequest</code> <a href="https://github.com/python/cpython/blob/3.6/Lib/http/client.py" rel="nofollow noreferrer">github.com/python/cpython/blob/3.6/Lib/http/client.py</a></span>
