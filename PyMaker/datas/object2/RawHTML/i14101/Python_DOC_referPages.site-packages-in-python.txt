<div class="post-text" itemprop="text">
<p>When I type in the following code :</p>
<pre><code>&gt;&gt;&gt; import site
&gt;&gt;&gt; site.getsitepackages()
</code></pre>
<p>the following result comes up - </p>
<pre><code>['/usr/local/lib/python2.7/dist-packages', '/usr/lib/python2.7/dist-packages']
</code></pre>
<p>What do the two different paths indicate ? On windows, there is just one folder for the site-packages. Why are they different in linux ?</p>
</div>
<div class="post-text" itemprop="text">
<p>The site packages directories are the directories to which Python installs third party extensions, and in which it looks for packages and modules whenever you import something.</p>
<p>On Linux, the <code>/usr</code> directory contains files that come with the system distribution, and the <code>/usr/local</code> directory contains files installed by the system administrator. So, if you install the distribution version of a third party extension, you're going to find it in the <code>/usr</code> sites-packages directory:</p>
<pre><code>$ sudo apt-get install python-twisted
...
$ find /usr -name twisted
/usr/lib/python2.7/dist-packages/twisted
</code></pre>
<p>And if you install the third party extension yourself, you're going to find it in the <code>/usr/local</code> site-packages directory:</p>
<pre><code>$ sudo pip install twisted
...
$ find /usr -name twisted
/usr/local/lib/python2.7/dist-packages/twisted
</code></pre>
<p>Windows doesn't make this distinction, so it only has one site packages directory.</p>
