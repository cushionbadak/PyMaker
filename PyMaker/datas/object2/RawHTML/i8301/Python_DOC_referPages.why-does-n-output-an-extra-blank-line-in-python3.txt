<p>In python3 <code>print('\n')</code> will generate an extra blank line. Could someone make a brief explanation about this? </p>
<p>Thanks in advance.</p>
</div>
<div class="post-text" itemprop="text">
<p>In <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow noreferrer">the documentation for <code>print</code></a> it is stated that:</p>
<blockquote>
<p>All non-keyword arguments are converted to strings like <code>str()</code> does and written to the stream, separated by <code>sep</code> and followed by <code>end</code>.</p>
</blockquote>
<p>The default value for <code>end</code> is <code>'\n'</code>, so Python <em>first</em> prints the supplied <code>'\n'</code> and <em>then</em> <code>end</code> which equals <code>'\n'</code> too; that's why you see two blank lines.</p>
<p>Change the default value if you don't want that:</p>
<pre><code>print('\n', end='')
</code></pre>
<p>Note that this also applies to Python 2.x's <code>print</code> statement, it also writes <code>'\n'</code> at the end. You can change the behavior there by appending a comma character.</p>
</div>
