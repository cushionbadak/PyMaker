<div class="post-text" itemprop="text">
<p>The problem at hand is that when you were able to import <code>jenkinsapi</code>, your working directory was <code>~/jenkinsapi-0.3.4</code>; after log out and log in, your working directory is <code>~</code>. In latter case, Python is unable to find module in the same directory and fails.</p>
<p>If you do <code>cd ~/jenkinsapi-0.3.4</code> and then open Python shell, <code>import jenkinsapi</code> should work.</p>
<p>As a side note, you might want to learn about Pythons virtual environments (see <a href="https://stackoverflow.com/q/5844869/3552063">this StackOverflow question</a> for pointers) and installing packages through <a href="https://docs.python.org/3/installing/" rel="nofollow noreferrer">pip</a>.</p>
