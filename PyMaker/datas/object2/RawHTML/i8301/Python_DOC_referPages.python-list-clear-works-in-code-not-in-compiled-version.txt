<p>I am working on a Linux machine with Python version 3.2.3. 
Whenever I try to do <code>list.clear()</code> I get an exception </p>
<pre><code>&gt;&gt;&gt; l = [1, 2, 3, 4, 5, 6, 7]
&gt;&gt;&gt; l.clear()
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
AttributeError: 'list' object has no attribute 'clear'
</code></pre>
<p>At the same time on my Mac with Python 3.4.3 the same code runs smoothly. 
Can it be due to the difference between Python versions or is there something I'm missing? </p>

<div class="post-text" itemprop="text">
<p><code>list.clear</code> was added in Python 3.3.</p>
<p>Citing the <a href="https://docs.python.org/3/library/stdtypes.html?highlight=clear#mutable-sequence-types" rel="noreferrer"><em>Mutable Sequence Types</em></a> section in the documentation:</p>
<blockquote>
<p><em>New in version 3.3</em>: <code>clear()</code> and <code>copy()</code> methods.</p>
<p><code>s.clear()</code> removes all items from <code>s</code> (same as <code>del s[:]</code>)</p>
</blockquote>
<p>See the <a href="http://bugs.python.org/issue10516" rel="noreferrer">issue #10516</a> for the relevant discussion and alternative ways of clearing lists. In summary, it is the same as <code>del l[:]</code> and <code>l[:] = []</code>.</p>
</div>
