<div class="post-text" itemprop="text">
<p>Don't use separate <code>choice()</code> calls. Use <a href="https://docs.python.org/3/library/random.html#random.sample" rel="noreferrer"><code>random.sample()</code></a>, picking from <a href="https://docs.python.org/3/library/string.html#string.ascii_lowercase" rel="noreferrer"><code>string.ascii_lowercase</code></a> rather than lowercasing:</p>
<pre><code># pick 10 random keys, all unique
keys = random.sample(string.ascii_lowercase, 10)
</code></pre>
<p>If all you are doing is shuffling <em>all 26 letters</em>, then use <code>random.shuffle()</code>:</p>
<pre><code># list of 26 letters in random order
random_letters = list(string.ascii_lowercase)
random.shuffle(random_letters)
# map the alphabet to the random letters
alphabet = dict(zip(string.ascii_lowercase, random_letters))
</code></pre>
</div>
<span class="comment-copy">Just a quick note on terminology. You aren't storing those random letters as <i>keys</i> you're storing them as <i>values</i>. In <code>{'a': 'xyz', 'b': 'uvw'}</code>, <code>'a'</code> is a key and <code>'xyz'</code> is the value associated with that key.</span>
<span class="comment-copy">BTW, it's rarely a good idea to create a whole bunch of separate but related named variables like your <code>a</code> to <code>z</code>. When you find yourself tempted to do that, see if you can put them into a collection (like a list or dict) instead.</span>
