<p>For example, is it guaranteed that float in Python 3 is the 64-bit IEEE-754 binary floating-point type?</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3.6/tutorial/floatingpoint.html#representation-error" rel="nofollow noreferrer">Python docs</a> describe this in a passing mention:</p>
<blockquote>
<p>almost all platforms map Python floats to IEEE-754 “double
  precision”.</p>
</blockquote>
<p>So, no, it's not a language guarantee but it does seem like CPython will use it everywhere it's available.</p>
<p><a href="https://www.python.org/dev/peps/pep-0754/" rel="nofollow noreferrer">Rejected PEP 754</a> describes in its rejection notice:</p>
<blockquote>
<p>Several ideas of this PEP were implemented for Python 2.6.
  float('inf') and repr(float('inf')) are now guaranteed to work on
  every supported platform with IEEE 754 semantics.</p>
</blockquote>
</div>
