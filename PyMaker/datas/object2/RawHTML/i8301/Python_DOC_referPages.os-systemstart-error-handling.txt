<div class="post-text" itemprop="text">
<p>Instead of using <code>system</code>, you might want to use the <code>subprocess</code> module.</p>
<p>You can call the <code>os.path.isfile</code> to check if the file exists, or you can raise an <code>exception</code> as:</p>
<pre><code>if os.path.isfile('your_file'):
    # If required, you can read your file's output through this way
    output = subprocess.Popen(['./your_file'], stdout = subprocess.PIPE)
</code></pre>
<p>Or,</p>
<pre><code>try:
    output = subprocess.Popen(['./your_file'], stdout = subprocess.PIPE)
except FileNotFoundError as e:
    print('Oops, file not found')
</code></pre>
<p>Take a look at the documentation for the <code>subprocess</code> module <a href="https://docs.python.org/3/library/subprocess.html" rel="nofollow noreferrer">here</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>Aside from suspending errors from your platform, I will seek a platform independent solution. You can use<code>os.path.exists</code> to check if a directory or a file exists, if it exists you pass the command to <code>os.system</code> to open the file: </p>
<pre><code>if os.path.exists(path):
    os.system(...)
else: 
    # file does not exist 
    ...
</code></pre>
<p>I wouldn't recommend using <code>os.system</code>, you're really making the security of your application brittle this way.</p>
