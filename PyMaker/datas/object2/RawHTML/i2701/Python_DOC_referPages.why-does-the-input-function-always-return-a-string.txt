<div class="post-text" itemprop="text">
<p>What you input to your program is <em>always</em> of type <code>str</code>. if you want Python to deduce the type of the input itself, use <code>data = eval(input())</code> or, for more safety:</p>
<pre><code>import ast 
data = ast.literal_eval(input())
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The built-in function <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow noreferrer"><code>input</code></a> always returns a string, regardless of whether that string might be comprised of solely numerical characters. If you are sure the result can represent a number, then convert it:</p>
<pre><code>varry = int(input())
</code></pre>
<p>Now the type of <code>varry</code> will be <code>int</code>, assuming this conversion doesn't raise a <code>ValueError</code></p>
</div>
