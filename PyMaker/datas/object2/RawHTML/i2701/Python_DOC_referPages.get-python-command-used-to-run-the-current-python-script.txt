<div class="post-text" itemprop="text">
<p>If I run a Python script, <code>foo.py</code> with</p>
<pre><code>$ python3.6 foo.py
</code></pre>
<p><code>sys.argv</code> will return <code>['foo.py']</code>. But I want something that will return the name of the command used to launch the script. In this case <code>'python3.6'</code>.</p>
<p>Is this possible?</p>
</div>
<div class="post-text" itemprop="text">
<p>It sounds like you’re looking for <a href="https://docs.python.org/3/library/sys.html#sys.executable" rel="nofollow noreferrer"><code>sys.executable</code></a>:</p>
<blockquote>
<p>A string giving the absolute path of the executable binary for the Python interpreter, on systems where this makes sense. If Python is unable to retrieve the real path to its executable, <code>sys.executable</code> will be an empty string or <code>None</code>.</p>
</blockquote>
<p>This is commonly used for things like running another Python script via <code>subprocess.run([sys.executable, 'spam.py', 'arg')</code>.</p>
<p>This gives you the absolute path to the Python interpreter. If you want just the name, use <a href="https://docs.python.org/3/library/os.path.html#os.path.basename" rel="nofollow noreferrer"><code>os.path.basename(sys.executable)</code></a>.</p>
<p>Keep in mind that it’s always possible that, say, <code>/usr/local/bin/python3.6</code> is actually just a symlink or even a shim launcher for something named, say, <code>/opt/local/python3.6/bin/python3</code>, in which case you might get the latter path, and basenaming it would then give you <code>python3</code>, but then running <code>python3</code> might  run a different <code>python3</code> that’s higher on the path. Using the absolute path, you won’t have that problem.</p>
</div>
