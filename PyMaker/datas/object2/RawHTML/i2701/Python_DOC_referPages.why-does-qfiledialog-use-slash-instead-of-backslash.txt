<p><strong>Why "\" and "/" are mixed?</strong></p>
<p><code>os.getcwd()</code> emits backslash string.</p>
<p>On the other hand, <code>QFileDialog</code> emits forward slash string.</p>
<p>Why?</p>
<p><strong>Example</strong></p>
<p>Please execute this sample code.</p>
<pre><code>from PySide import QtGui
from PySide import QtCore
import sys
import os

class DirectoryPrinter(QtGui.QWidget):
    def __init__(self,parent=None):
        super(DirectoryPrinter,self).__init__(parent=None)

        self.filedialog_pushbutton = QtGui.QPushButton("filedialog",self)
        self.connect(self.filedialog_pushbutton,QtCore.SIGNAL("clicked()"),self.filename_getter)

    def filename_getter(self):
        print("from os.getcwd()",os.getcwd())
        filename = QtGui.QFileDialog.getOpenFileName(self,"Select your file",os.path.expanduser("~"))[0]
        print("from QFileDialog",filename)


def main():
    try:
        QtGui.QApplication([])
    except Exception as e:
        print(22,e)
    directoryprinter = DirectoryPrinter()
    directoryprinter.show()

    sys.exit(QtGui.QApplication.exec_())
if __name__ == "__main__":
    main()
</code></pre>
<p><strong>Result (on my occasion)</strong></p>
<p>from <code>os.getcwd()</code>: <code>J:\</code></p>
<p>from <code>QFileDialog</code>: <code>C:/Users/******/setup.py</code></p>
</div>
<div class="post-text" itemprop="text">
<p>This is because <code>QFileDialog</code> <a href="https://www.qtcentre.org/threads/40381-QFiledialog-returns-forward-slashes-instead-of-backslashes" rel="nofollow noreferrer">uses forward slashes regardless of OS. This makes it easier to write path handling code</a>.</p>
<p>You can use <a href="https://docs.python.org/3/library/os.path.html#os.path.normpath" rel="nofollow noreferrer"><code>os.path.normpath</code></a> to converts forward slashes to backward slashes in a path on Windows.</p>
