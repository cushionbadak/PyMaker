<div class="post-text" itemprop="text">
<p>Are you using PostgreSQL by any chance? The <code>UUIDField</code> may be using the native <a href="https://www.postgresql.org/docs/10/static/datatype-uuid.html" rel="nofollow noreferrer">uuid type</a> for the column. It stores it efficiently using only 16 bytes (without dashes). If that is the case, it is <strong>not</strong> storing the dashes, only showing them when you <code>select</code>. </p>
<p>The good news, is that in python code, you are getting a <a href="https://docs.python.org/3/library/uuid.html#uuid.UUID" rel="nofollow noreferrer">UUID object</a>, so you can do <code>self.api_key.hex()</code> to get a string without dashes.</p>
</div>
<div class="post-text" itemprop="text">
<p>Use <strong><code>CharField</code></strong> field instead of <code>UUIDField</code>,<br/></p>
<pre><code>def generate_uuid():
    return uuid.uuid4().hex


class Profile(models.Model):
    api_key = models.CharField(<b>default=generate_uuid</b>, editable=False, unique=True, max_length=40)
</code></pre>
