<div class="post-text" itemprop="text">
<p>In python, when you have a variable which is a list, it has a specific length, and if you try to access a member with a number greater or equal to that length, it will spit out the index error. Now to avoid this you could instantiate the array in one lane using the [] notion like so: (In this code the inputsSp variable is the inputs splitted into the corresponding members: for example inputsSp[i] = ['4', '5', '||', '3', '6'])</p>
<pre><code>S = [[int(inp[i][0]), int(inp[3])] for inp in inputsSp]
E = [[int(inp[i][1]), int(inp[4])] for inp in inputsSp] 
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can also do this via:</p>
<pre><code>S, E = zip(*[map(int, input().split()) for _ in range(int(input()))])
</code></pre>
<p>or, the long version:</p>
<pre><code>num_of_pairs = int(input())
full_input = []
for _ in range(num_of_pairs):
    full_input.append(map(int, input().split()))  # add each pair to the full input
S, E = zip(*full_input)  # the * is important as this unpacks the list for zip
</code></pre>
<p>This works by taking in all inputs and then <a href="https://docs.python.org/3/library/functions.html#zip" rel="nofollow noreferrer"><code>zip</code></a>ping the result, which sorts them as you wish. See it on <a href="https://repl.it/repls/DirtyPalegreenSpecialist" rel="nofollow noreferrer">repl.it</a>:</p>
<pre><code>&gt;? 2
&gt;? 4 5
&gt;? 3 6
&gt;&gt;&gt; S
('4', '3')
&gt;&gt;&gt; E
('5', '6')
</code></pre>
