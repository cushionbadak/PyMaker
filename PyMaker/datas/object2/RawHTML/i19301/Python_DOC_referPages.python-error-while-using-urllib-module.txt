<p>I was making python http banner grabber using <code>urllib</code> but it gives me an error.</p>
<p>I was not able it import <code>urllib2</code> (module not found error) so tried with with <code>urllib</code> instead.</p>
<p>Here's the code , what's wrong with it?</p>
<pre><code>#!/usr/bin/env python
import urllib   
url=urllib.urlopen('www.bing.com')
print (url.info())
</code></pre>
<p>Error:</p>
<pre><code>AttributeError: 'module' object has no attribute 'urlopen'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You are using Python 3; use <a href="http://docs.python.org/3/library/urllib.request.html" rel="nofollow"><code>urllib.request</code></a> instead.</p>
<pre><code>import urllib.request

url = urllib.request.urlopen('http://www.bing.com')
print(url.info())
</code></pre>
<p>This is documented at the <a href="http://docs.python.org/2/library/urllib2.html" rel="nofollow">top of the <code>urllib2</code> page</a>:</p>
<blockquote>
<p><strong>Note</strong>: The <a href="http://docs.python.org/2/library/urllib2.html#module-urllib2" rel="nofollow"><code>urllib2</code></a> module has been split across several modules in Python 3 named <code>urllib.request</code> and <code>urllib.error</code>. The <a href="http://docs.python.org/2/glossary.html#term-to3" rel="nofollow"><code>2to3</code></a> tool will automatically adapt imports when converting your sources to Python 3.</p>
</blockquote>
