<div class="post-text" itemprop="text">
<p>Your output sample suggest you are looking for <em>url encoding</em>, not Latin-1.</p>
<p>The <a href="http://docs.python.org/2/library/urllib.html#urllib.quote" rel="nofollow"><code>urllib.quote()</code></a> and <a href="http://docs.python.org/2/library/urllib.html#urllib.quote_plus" rel="nofollow"><code>urllib.quote_plus()</code></a> functions can be used to do such quoting, but the <code>-</code> character does <strong>not</strong> need quoting and won't be quoted:</p>
<blockquote>
<p>Replace special characters in <em>string</em> using the <code>%xx</code> escape. Letters, digits, and the characters <code>'_.-'</code> are never quoted.</p>
</blockquote>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; from urllib import quote
&gt;&gt;&gt; quote('abc-def')
'abc-def'
&gt;&gt;&gt; quote('some data that needs quoting!')
'some%20data%20that%20needs%20quoting%21'
</code></pre>
<p>If you are using Python 3, the <code>quote</code> and <code>quote_plus</code> functions are found in the <a href="http://docs.python.org/3/library/urllib.parse.html#url-quoting" rel="nofollow"><code>urllib.parse</code> module</a>.</p>
