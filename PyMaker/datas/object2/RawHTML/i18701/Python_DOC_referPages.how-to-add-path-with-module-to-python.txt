<div class="post-text" itemprop="text">
<p>Obviously, the module gyp.py is not in the search path of modules (sys.path). sys.path is an array variable in sys module which contains all known paths of the modules. You can add the directory containing the module gyp.py manually by either of these methods:</p>
<ol>
<li><p>set via PYTHONPATH environment variable (see <a href="http://docs.python.org/3/using/cmdline.html?highlight=path#envvar-PYTHONPATH" rel="nofollow">http://docs.python.org/3/using/cmdline.html?highlight=path#envvar-PYTHONPATH</a>)</p></li>
<li><p>Add the path manually within your python script prior to importing gyp. For example, if the directory containing this module is /home/you/gyp:</p></li>
</ol>
<pre class="lang-py prettyprint-override"><code>import os, sys
sys.path.append('/home/you/gyp')

import gyp
#--------- That's it ------------
</code></pre>
<p>You can check if this path already exists using the debug lines</p>
<pre><code>import sys
print(sys.path) # version python 3.2
</code></pre>
<p>or </p>
<pre><code>print sys.path # version python 2.7
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Install the module will be fine.</p>
<pre><code>git clone https://chromium.googlesource.com/external/gyp
cd gyp
sudo ./setup.py install
</code></pre>
<p>enjoy it.</p>
