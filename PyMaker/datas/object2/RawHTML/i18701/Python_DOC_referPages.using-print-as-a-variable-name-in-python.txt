<p>While following a tutorial for python, I got to know that we can use <em>print</em> for a variable name, and it works fine. But after assigning the print variable, how do we get back the original print function?</p>
<pre><code>&gt;&gt;&gt; print("Hello World!!")
Hello World!!!
&gt;&gt;&gt; print = 5
&gt;&gt;&gt; print("Hi")
</code></pre>
<p>Now, the last call gives the error <strong>TypeError: 'int' object is not callable</strong>, since now print has the integer value 5.</p>
<p>But, how do we get back the original functionality of print now? Should we use the class name for the print function or something?
As in, <code>SomeClass.print("Hi")</code>?</p>
<p>Thanks in advance.</p>
</div>
<div class="post-text" itemprop="text">
<pre><code>&gt;&gt;&gt; print = 5
&gt;&gt;&gt; print = __builtins__.print
&gt;&gt;&gt; print("hello")
hello
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can actually delete the variable so the built-in function will work again:</p>
<pre><code>&gt;&gt;&gt; print = 5
&gt;&gt;&gt; print('cabbage')
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'int' object is not callable
&gt;&gt;&gt; del print
&gt;&gt;&gt; print('cabbage')
cabbage
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If you want to use as a temp way, do them but after that, apply <code>print</code> function to <code>print</code> variable:</p>
<pre><code>print = __builtins__.print
</code></pre>
