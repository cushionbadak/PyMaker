<p>You have linked to the Python 3.x documentation, but if <code>translate()</code> is being used with multiple arguments then this code is probably from Python 2.x where that is valid.  Here is the <a href="http://www.python.org/doc//current/library/stdtypes.html#str.translate" rel="nofollow">documentation</a>.</p>
<p>As you can see there, the second argument is optional and it specifies characters that should be deleted from the input string (on Python 3.x you would do this by mapping the characters to <code>None</code>).</p>
<p>So for <code>"Agh#$%#%2341- -!zdrkfd".translate(table, delete_table)</code>, first all characters present in <code>delete_table</code> are removed, and then a translation is performed using <code>table</code>.</p>
<p>Since <code>delete_table</code> is constructed using <code>string.maketrans()</code> translating all lowercase letters to space, it will be a string that contains every ASCII character except for lowercase letters:</p>
<pre><code>&gt;&gt;&gt; delete_table = string.maketrans(string.ascii_lowercase, ' '*len(string.ascii_lowercase))
&gt;&gt;&gt; delete_table

<p>So all other characters will be removed from the string, and then the translation with <code>table</code> will not modify anything since <code>string.maketrans('', '')</code> is used.</p>
