<pre><code>for i in range(n):
    a = list(map(int, input().rstrip().split()))
</code></pre>
<p>Each time through this loop, you're reassigning <code>a</code>, discarding its previous value.</p>

<div class="post-text" itemprop="text">
<p>In your first loop, you're replacing each input list result with a <em>new</em> input list.</p>
<p>The culprit:</p>
<pre><code>    a = list(map(int, input().rstrip().split()))
</code></pre>
<p>When your code exits the loop, <code>a</code> will only contain one value. To revise your approach, you could initialise a list before the loop, appending values into it <em>when inside the loop</em>.</p>
<pre><code>a = []
for i in range(n):
    a.append(int(input()))
</code></pre>
<p>Additionally, since there seems to be only one number per line, simply doing <code>int(input())</code> will get the number.</p>
<p>Your second loop might also need to be revised.</p>

<div class="post-text" itemprop="text">
<p>In your beginning loop you keep resetting the value of <code>a</code> instead of adding it to the existing value of <code>a</code>:</p>
<pre><code>for i in range(n):
    a = list(map(int, input().rstrip().split()))
</code></pre>
<p>The first time <code>a = [4]</code> but then you override it the next loop where <code>a = [6]</code></p>
<p>To append it to <code>a</code> instead, you should do:</p>
<pre><code>a = []
for i in range(n):
    a += list(map(int, input().rstrip().split()))
</code></pre>
<p>or what you are more likely going for:</p>
<pre><code>a = []
for i in range(n):
    a.append(int(input().rstrip()))
</code></pre>
<p>The way you are doing it, calling <code>.split()</code> on the input generates a list which wouldn't make sense in this case, if would be better to just get the number by <code>int(input().rstrip())</code> </p>
<p>Another way to do it is using list comprehension which is more pythonic:</p>
<pre><code>a = [int(input().rstrip()) for _ in range(n)]
</code></pre>
<p>Hope that helps!</p>
