<div class="post-text" itemprop="text">
<p>Try this</p>
<pre><code>subprocess.call('./process "%s"' % path, shell=True)
</code></pre>
<p>I guess problem is more with space in file name. File names with spaces in them should be enclosed in quotes like this <code>./process "foo bar.txt"</code> or escaped like this <code>./process foo\ bar.txt</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p>Don't use <code>shell=True</code>. It's trouble-prone (as in the OP) and enables <a href="http://en.wikipedia.org/wiki/Shell_injection#Shell_injection" rel="nofollow">shell injection attacks</a>.</p>
<p>Do it like this:</p>
<pre><code>subprocess.call(["./process", path])
</code></pre>
<p>If you insist on using <code>shell=True</code>, read the <a href="https://docs.python.org/3/library/subprocess.html?highlight=subprocess#security-considerations" rel="nofollow">Security Considerations</a> in the python documentation, and make sure you use <code>shlex.quote</code> to correctly escape all metacharacters.</p>
</div>
<span class="comment-copy">Do you need shell=True?  If not it will likely work fine with making the args a list and not letting the shell trip on special characters</span>
