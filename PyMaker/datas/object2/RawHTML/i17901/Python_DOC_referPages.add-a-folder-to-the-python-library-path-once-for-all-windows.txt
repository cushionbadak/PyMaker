<div class="post-text" itemprop="text">
<p>I use</p>
<pre><code>sys.path.append('D:/my_library_folder/')
import mymodule
</code></pre>
<p>in order to import some module.</p>
<p>How to add <em>permanently</em> this folder <code>D:/my_library_folder/</code> to the Python library path, so that I will be able to use <strong>only</strong></p>
<pre><code>import mymodule
</code></pre>
<p>in the future?</p>
<p>(Even after a reboot, etc.)</p>
</div>
<div class="post-text" itemprop="text">
<p>just put the folder in <code>site-packages</code> directory. ie:</p>
<pre><code>C:\PythonXY\Lib\site-packages
</code></pre>
<p>Note: you need to add an empty file <code>__init__.py</code> to the folder</p>
<hr/>
<p><strong>Files named <code>__init__.py</code> are used to mark directories on disk as a Python package directories.</strong></p>
<p>If you have the files:</p>
<pre><code>C:\PythonXY\Lib\site-packages\&lt;my_library_folder&gt;\__init__.py
C:\PythonXY\Lib\site-packages\&lt;my_library_folder&gt;\module.py
</code></pre>
<p>you can import the code in module.py as:</p>
<pre><code>from &lt;my_library_folder&gt; import module
</code></pre>
<p>If you remove the <code>__init__.py</code> file, Python will no longer look for submodules inside that directory, so attempts to import the module will fail.</p>
<p>If you have lots of folders, then create the empty <code>__init__.py</code> file in each folder. for eg:</p>
<pre><code>C:\PythonXY\Lib\site-packages\&lt;my_library_folder&gt;\
    __init__.py
    module.py        
    subpackage\
        __init__.py
        submodule1.py
        submodule2.py
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Set PYTHONPATH environment variable to D:/my_library_folder/</p>
</div>
<div class="post-text" itemprop="text">
<p>If <code>D:/my_library_folder</code> is a project you're working on and has a setup script, you could also do <code>python setup.py develop</code>.  Not entirely related to the question, but I also recommend using <a href="http://www.virtualenv.org/en/latest/" rel="nofollow">virtualenv</a>.</p>
</div>
