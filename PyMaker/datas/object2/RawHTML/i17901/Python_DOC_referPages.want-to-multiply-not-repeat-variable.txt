<p>I want to input a command-line variable and then multiply it, but when I print the variable it repeats by the number I want to multiply it by.</p>
<p>eg:</p>
<pre><code>#!/usr/bin/env python3
import sys
st_run_time_1 = sys.argv[1]*60

print ("Station 1 : %s" % st_run_time_1)
</code></pre>
<p>When I run the script I get the following:
    python3 test.py 2</p>
<pre><code>Station 1 : 222222222222222222222222222222222222222222222222222222222222
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>sys.argv[x]</code> is string. Multiplying string by number casue that string repeated.</p>
<pre><code>&gt;&gt;&gt; '2' * 5         # str * int
'22222'

&gt;&gt;&gt; int('2') * 5    # int * int
10
</code></pre>
<p>To get multiplied number, first convert <code>sys.argv[1]</code> to numeric object using <a href="http://docs.python.org/3/library/functions.html#int" rel="nofollow"><code>int</code></a> or <a href="http://docs.python.org/3/library/functions.html#float" rel="nofollow"><code>float</code></a>, ....</p>
<pre><code>import sys
st_run_time_1 = int(sys.argv[1]) * 60 # &lt;---

print ("Station 1 : %s" % st_run_time_1)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You are multiplying a string with an integer, and that always means repetition. Python won't ever auto-coerce a string to an integer, and <code>sys.argv</code> is always a list of strings.</p>
<p>If you wanted integer arithmetic, convert the <code>sys.argv[1]</code> string to an integer first:</p>
<pre><code>st_run_time_1 = int(sys.argv[1]) * 60
</code></pre>
</div>
