<p>I pass float number as string format in <code>int('.0')</code></p>
<p>0.0 is valid floating point number, so why it is giving error?</p>
</div>
<div class="post-text" itemprop="text">
<p>From the documentation of <a href="https://docs.python.org/3/library/functions.html#int" rel="nofollow noreferrer">int</a>:</p>
<blockquote>
<p>If x is not a number or if base is given, then x must be a string,
  bytes, or bytearray instance representing an integer literal in radix
  base. Optionally, the literal can be preceded by + or - (with no space
  in between) and surrounded by whitespace.</p>
</blockquote>
<p>So it gives you <code>ValueError</code> because the string <code>'.0'</code> does not represent an integer literal.</p>
</div>
<div class="post-text" itemprop="text">
<p>Because you can type cast one step at a time. For example you can convert float to int or string to int. Not a floating point string which here is 2steps.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can do it if you remove the quotes and make it a float like:</p>
<pre><code>int(.1)
</code></pre>
<p>But strings don't work if inside the string is a float, because they will think it's a number and will break saying <code>'.'</code> is not a numeric value, also the reason the above works is because:</p>
<pre><code>&gt;&gt;&gt; .1
0.1
&gt;&gt;&gt; 
</code></pre>
<p>And:</p>
<pre><code>float(0.1)
</code></pre>
<p>Works.</p>
<p>Note that even a real float in a string can't be converted into an integer:</p>
<pre><code>&gt;&gt;&gt; int('3.1')
Traceback (most recent call last):
  File "&lt;pyshell#1&gt;", line 1, in &lt;module&gt;
    int('3.1')
ValueError: invalid literal for int() with base 10: '3.1'
</code></pre>
</div>
