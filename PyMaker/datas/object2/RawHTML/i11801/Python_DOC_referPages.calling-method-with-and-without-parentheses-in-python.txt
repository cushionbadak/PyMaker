<div class="post-text" itemprop="text">
<p>I've realized that some methods should be called with <code>()</code>, while others can't.  How can I check, using IPython e.g., whether to use parentheses or not?  For example the following file <code>scratch.py</code></p>
<pre><code>import numpy as np

arr = np.random.randn(5)

print arr.sort, "\n"
print arr.sort(), "\n";
print arr.shape, "\n";
print arr.shape(), "\n";
</code></pre>
<p>produces this output:</p>
<pre><code>&lt;built-in method sort of numpy.ndarray object at 0x7fb4b5312300&gt; 

None 

(5,) 

Traceback (most recent call last):
  File "scratch.py", line 8, in &lt;module&gt;
    print arr.shape(), "\n";
TypeError: 'tuple' object is not callable
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Those are not methods, those are <a href="https://docs.python.org/3/library/functions.html#property" rel="nofollow"><strong>properties</strong></a>. The <a href="https://docs.python.org/3/howto/descriptor.html" rel="nofollow"><strong>descriptor</strong></a> is invoked behind the scenes by Python itself.</p>
</div>
<div class="post-text" itemprop="text">
<p>Methods in Python are always invoked with a ().
Best way to check if something is a method is to read the documentation of the library.</p>
</div>
<span class="comment-copy">Or they may just be attributes.</span>
