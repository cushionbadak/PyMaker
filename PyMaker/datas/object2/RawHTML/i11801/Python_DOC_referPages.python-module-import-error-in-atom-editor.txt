<p>User sjs wrote:</p>
<blockquote>
<p>If you're like me and you don't like the idea of deleting things that
  were part of the standard system installation (which others have
  suggested) then you might like the solution I ended up using:</p>
<p>Get Homebrew - it's a one-line shell script to install!</p>
<p>Edit your <code>.profile</code>, or whatever is appropriate, and put <code>/usr/local/bin</code> at the start of your <code>PATH</code> so that Homebrew binaries are
  found before system binaries</p>
<p><code>brew install python</code> - this installs a newer version of python in /usr/local</p>
<p><code>pip install pandas</code></p>
<p>This worked for me in OS X 10.8.2, and I can't see any reason it
  shouldn't work in 10.6.8.</p>
</blockquote>
<p>I had the problem that atom-runner within atom editor did not recognize pandas after installing it via pip. This is obviously a consequence of using Homebrew in combination with bash and python.</p>
<p>For my case, I edited <code>.bash_profile</code> and changed <code>PATH="/Library/Frameworks/Python.framework/Versions/3.5/bin:${PATH}"</code> to <code>PATH="/usr/local/bin/python3:${PATH}"</code>, and exported this path.</p>
<p>After, I reinstalled pandas using <code>pip install pandas</code>, and rerun the .py script that contained <code>import pandas as pd</code> without any problems in atom.</p>
