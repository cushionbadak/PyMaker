<pre><code>import random

o=['§','±','!','@','#','$','%','^','&amp;','*','(',')','','_','=','+','/','[']
q=['1','2','3','4','5','6','7','8','9','0']

    wordDic ={i: random.choice(o)+random.choice(q)}
print(wordDic)
</code></pre>
<p>(top_25 is an array of words, and the random.choice randomly selects characters from the respective arrays). As the for loop iterates, the wordDic keeps updating and overwriting itself-is there a way to add the new values of the wordDic into the dictionary, so it prints out all of them in one dictionary?</p>

<div class="post-text" itemprop="text">
<p>Because you are defining the dictionary inside the loop.</p>
<p>Which means that you are redifining/overwriting the dictionary during each iteration of the loop ( Comment from: @M.T ) </p>
<p>Define it outside the loop to get it work.</p>
<pre><code>word_dic = {}
for i in top_25:
    word_dic[i] = random.choice(o)+random.choice(q)
print(word_dic)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The compact way to do this is with a <a href="https://docs.python.org/3/reference/expressions.html#dictionary-displays" rel="nofollow">dictionary comprehension</a>. Also, there's no need to pass <code>random.choice</code> a list of single characters: you can just pass it a string.</p>
<pre><code>from random import choice

o = u"§±!@#$%^&amp;*()_=+/["
q = u"0123456789"

wordDic = {word: choice(o)+choice(q) for word in top_25}
print(wordDic)
</code></pre>
<p>I've used the <code>u</code> prefix on the strings to make the code compatible to Python 2 as well as Python 3.</p>
