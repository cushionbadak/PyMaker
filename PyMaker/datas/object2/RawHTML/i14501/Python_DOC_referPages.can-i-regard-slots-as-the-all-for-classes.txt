<p><a href="https://docs.python.org/3/tutorial/modules.html?highlight=__all__#importing-from-a-package" rel="nofollow"><code>__all__</code></a> only affects starred imports - only names from <code>__all__</code> will be imported after doing <code>from mymodule import *</code> (if <code>__all__</code> is present, of course). You can still access names not listed there:</p>
<pre><code>In [4]: import mymodule

In [5]: mymodule.__all__
Out[5]: ['a']

In [6]: mymodule.b
Out[6]: 'was not in __all__'
</code></pre>
<p><a href="https://docs.python.org/3/reference/datamodel.html#slots" rel="nofollow"><code>__slots__</code></a> prevents the <code>__dict__</code> attribute from being created (restricting the ability to set arbitrary attributes to instances of certain class):</p>
<pre><code>In [10]: class A():
   ...:     __slots__ = ('a',)
   ...:     

In [11]: a = A()

In [12]: a.b = 'was not in __slots__'

<p>These are 2 different things.</p>
