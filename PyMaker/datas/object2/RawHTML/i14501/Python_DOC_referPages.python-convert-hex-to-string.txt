<div class="post-text" itemprop="text">
<p>A value like <code>0x0F</code> or <code>0x05</code> has to be given to a function as a string without the 0x at the beginning so <code>0F</code> or <code>05</code>.</p>
<p>If I use <code>str(hex(0x0F))[2:4]</code> I only get <code>f</code>. It is crucial that the missing <code>0</code> is still present in my application.</p>
<p>How can i do that?</p>
</div>
<div class="post-text" itemprop="text">
<p>You'd use <a href="https://docs.python.org/3/library/stdtypes.html?#str.format" rel="nofollow"><code>str.format</code></a>:</p>
<pre><code>In [1]: '{:02X}'.format(0x0F)
Out[1]: '0F'
</code></pre>
<p>In this context <code>{:02X}</code> is equivalent to <code>{0:02X}</code>. Preceding the colon with <em>0</em> tells Python to apply this to the first argument of <code>str.format</code> (<em>0x0F</em> in this example). <em>02</em> sets the minimum field width to <em>2</em> and </p>
<blockquote>
<p>Preceding the width field by a zero ('0') character enables sign-aware
  zero-padding for numeric types</p>
</blockquote>
<p><em>X</em> is one of the available integer presentation types:</p>
<blockquote>
<p>Hex format. Outputs the number in base 16, using upper- case letters
  for the digits above 9.</p>
</blockquote>
<p>Both quotes are from <a href="https://docs.python.org/3/library/string.html#format-specification-mini-language" rel="nofollow">Format Specification Mini-Language</a>.</p>
</div>
