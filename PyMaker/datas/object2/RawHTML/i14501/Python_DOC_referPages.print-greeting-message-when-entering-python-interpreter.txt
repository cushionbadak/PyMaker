<div class="post-text" itemprop="text">
<p>How do I print a greeting message when I initialize a python interpreter?  For example, if I were to <a href="https://stackoverflow.com/questions/30065694/initialize-interpreter-with-variables">initialize a python interpreter with custom pre-defined variables</a>, how might I advertise those variables to the user?</p>
</div>
<div class="post-text" itemprop="text">
<p>There is an environment variable called <code>PYTHONSTARTUP</code> that describes a path to a Python file to be executed on the invocation of Python shell. The script can contain normal Python code that is executed on invocation, therefore variables, prints or whatever else you want. It can be set in your ~/.bashrc</p>
<pre><code>export PYTHONSTARTUP="$HOME/.pythonrc"
</code></pre>
<p>and then creating the file itself</p>
<pre><code>cat &gt; ~/.pythonrc &lt;&lt; EOF
print 'Hello World!'
EOF
</code></pre>
<p>The output when starting python then looks somewhat like this</p>
<pre><code>Python 2.7.8 (default, Oct 19 2014, 16:02:00)
[GCC 4.2.1 Compatible Apple LLVM 6.0 (clang-600.0.54)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
Hello World!
&gt;&gt;&gt;
</code></pre>
<p>Because it is a normal Python file, setting the variables and showing them/announcing availability can be done like this:</p>
<pre><code>foo = 'Hello'
bar = 12.4123

print 'The following variables are available for use\nfoo: {}\nbar: {}'.format(foo, bar)
</code></pre>
<p>Output when invoking a Python repl and printing variable <code>foo</code>:</p>
<pre><code>Python 2.7.8 (default, Oct 19 2014, 16:02:00)
[GCC 4.2.1 Compatible Apple LLVM 6.0 (clang-600.0.54)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
The following variables are available for use
foo: Hello
bar: 12.4123
&gt;&gt;&gt; print foo
Hello
</code></pre>
<p>iPython acts different in a sense that it does not execute your PYTHONSTARTUP file, but has it's own mechanism called profiles. Default profile can be modified in <code>~/.ipython/profile_default/startup/</code>, where each <code>*.py</code> and <code>*.ipy</code> file is executed (see <code>~/.ipython/profile_default/startup/README</code>).</p>
</div>
<div class="post-text" itemprop="text">
<p>You can embed a console from a script using the built-in console or IPython's console.</p>
<p>If you want to use Python's built-in <a href="https://docs.python.org/3/library/code.html#code.interact" rel="nofollow">console</a>, pass the <code>banner</code> argument.  Assuming you have a dictionary of variables to inject:</p>
<pre><code>from code import interact
vars = {'hello': 'world'}
message = 'Extra vars: {}'.format(', '.join(vars))
interact(banner=message, local={'hello': 'world'})
</code></pre>
<p>With IPython's <a href="http://ipython.org/ipython-doc/stable/api/generated/IPython.terminal.embed.html?highlight=embed#IPython.terminal.embed.embed" rel="nofollow">console</a>, pass <code>banner1</code>.</p>
<pre><code>from IPython import embed
embed(banner1=message, user_ns={'hello': 'world'})
</code></pre>
