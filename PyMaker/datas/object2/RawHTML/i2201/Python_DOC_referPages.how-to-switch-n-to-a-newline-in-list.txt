<p>There are many answers already, but no one explained the problem.</p>
<h2>Problem</h2>
<p><code>\n</code> <strong>is</strong> the newline character!</p>
<h2>Explanation</h2>
<p>In order to be able show the newline within a string literal, the escape sequence <code>\n</code> is used, e.g.:</p>
<pre><code>&gt;&gt;&gt; 'a\nb'
'a\nb'

&gt;&gt;&gt; print('a\nb')
a
b
</code></pre>
<p><code>print</code> function prints the string, if a string argument is passed to it, but if some other object is passed to it, it first has to be converted to a string, so <code>print(x)</code> is the same as <code>print(str(x))</code>.</p>
<p>When a <code>list</code> of strings is converted to string, that is done by calling <a href="https://docs.python.org/3/library/functions.html#repr" rel="nofollow noreferrer"><code>repr</code></a> on each of its items:</p>
<pre><code>&gt;&gt;&gt; ['a', 'a\nb']
['a', 'a\nb']

&gt;&gt;&gt; str(['a', 'a\nb'])
"['a', 'a\\nb']"

&gt;&gt;&gt; print("['a', 'a\\nb']")
['a', 'a\nb']
</code></pre>
<h2>Solution</h2>
<p>Now, if you want to print the last 10 lines, it means you should print each string in the list, not the list object itself, e.g.:</p>
<pre><code>for s in list_of_strings:
    print(s)
</code></pre>
<p>Now, since the <code>s</code> already contains a newline and <code>print</code> adds a newline itself, you should remove one of the newlines to make the solution complete:</p>
<pre><code>for s in list_of_strings:
    print(s.strip('\n'))
</code></pre>
<p>or:</p>
<pre><code>for s in list_of_strings:
    print(s, end='')
</code></pre>
<p>or create one string by concatenating items of the list and print that:</p>
<pre><code>print(''.join(list_of_strings))
</code></pre>
