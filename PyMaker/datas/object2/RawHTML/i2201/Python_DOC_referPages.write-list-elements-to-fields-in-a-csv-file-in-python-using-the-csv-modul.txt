<p>you're writing a list of strings using <code>csv.writerows</code>, which writes <em>several</em> rows.</p>
<p>This function accepts an iterable of iterables. So when passing a list of strings, you're satisfying the input but it's not what you're expecting: the strings are seen as iterables, and are split character by character.</p>
<p>To write all your data, 1 string per row, use:</p>
<pre><code>writer.writerows([x] for x in neue_liste)
</code></pre>
<p>(well, <code>csv</code> module is hardly useful in that case, better use <code>writelines</code> on a file handle)</p>
<p>To write all your data in 1 row use <code>writerow</code> (note the lack of <code>s</code> in the end):</p>
<pre><code>writer.writerow(neue_liste)
</code></pre>
<p>although comments suggest wisely that you want to split your strings again according to spaces, then use:</p>
<pre><code>writer.writerows(x.split() for x in neue_liste)
</code></pre>
<p>or</p>
<pre><code>writer.writerows(map(str.split,neue_liste))
</code></pre>

<span class="comment-copy">What do you want the row to look like?  Do you want it to be <code>A0 0194 7F9E 00 0001 B7AB</code> or <code>A0,0194,7F9E,00,0001,B7AB</code> or something else?</span>
<span class="comment-copy">On the off chance that he wants rows looking like <code>A0,0194,7F9E,00,0001,B7AB</code> he could do <code>neue_liste = [c[i:i+26].split() for i in range(0, len(c), 26)]</code> and leave everything else the same.</span>
<span class="comment-copy">To add. When you use <code>writerows</code>, then each element of the list is treated as iterable row. As it is a string, the iterator considers each letter of the string as the element of the row. This is the reason why you see each element separated. As @StevenRumbalski wrote, you may alternatively want to loop through each <code>element</code> of the <code>neue_liste</code>, get the <code>row = element.split()</code>, and use the <code>writer.writerow(row)</code>.</span>
