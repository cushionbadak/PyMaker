<p>I have a case where I get a string version of an INI file and not the file itself. I am getting it this way from an external source and hence cannot be changed. </p>
<p>Example of the string version:</p>
<pre><code>'[DEFAULT]\nScore = 0.1\n\n[dev]\nHost = abc.com\nPort = 0000\n\n[qa]\nHost = xyz.com\nEsPort = 1000\n\n[main]\nHost = pqr.com\nPort = 2000\n'
</code></pre>
<p>I tried parsing this string using the configParser library in python:</p>
<pre><code>&gt;&gt;&gt; config = configparser.ConfigParser()
&gt;&gt;&gt; config.read(my_ini_str)
[]
</code></pre>
<p>I get back nothing with the read function. Is there any other way to achieve this?</p>
<p>Thanks in advance!</p>
<p>UPDATE - Forgot to add that I am using Python 2.</p>
</div>
<div class="post-text" itemprop="text">
<p>This is what the <a href="https://docs.python.org/3/library/configparser.html#configparser.ConfigParser.read_string" rel="nofollow noreferrer"><code>read_string()</code></a> method is for.</p>
<blockquote>
<p>Parse configuration data from a string.</p>
</blockquote>
<p>Hence:</p>
<pre><code>&gt;&gt;&gt; config = configparser.ConfigParser()
&gt;&gt;&gt; config.read_string(my_ini_str)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>you have to use <code>read_string</code> in order to parse a string.</p>
<p>then you have to use getters on "config" object, such as :</p>
<pre><code>&gt;&gt;&gt; config.get('dev', 'Host')
'abc.com'
</code></pre>
<p>or</p>
<pre><code>&gt;&gt;&gt; config.getfloat(config.default_section, 'Score')
0.1
</code></pre>
</div>
