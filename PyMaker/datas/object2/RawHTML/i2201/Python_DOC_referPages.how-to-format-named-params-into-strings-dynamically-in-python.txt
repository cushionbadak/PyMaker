<div class="post-text" itemprop="text">
<p>I have an array with parameters - for each parameter I have a name and a value.
Is there a way to format it dynamically into a string with a placeholders?</p>
<p>array:</p>
<pre><code>[{'name': "a", 'value': "123"}, {'name': "b", 'value': "456"}]
</code></pre>
<p>string: <code>"blabla {a}"</code></p>
<p>required result: <code>"blabla 123"</code></p>
</div>
<div class="post-text" itemprop="text">
<p>Because your string input already uses valid <a href="https://docs.python.org/3/library/string.html#formatstrings" rel="nofollow noreferrer">string formatting placeholders</a>, all you need to do is convert your existing data structure to a dictonary mapping names to values:</p>
<pre><code>template_values = {d['name']: d['value'] for d in list_of_dictionaries}
</code></pre>
<p>then apply that dictionary to your template strings with the <code>**mapping</code> call syntax to the <a href="https://docs.python.org/3/library/stdtypes.html#str.format" rel="nofollow noreferrer"><code>str.format()</code> method</a> on the template string:</p>
<pre><code>result = template_string.format(**template_values)
</code></pre>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; list_of_dictionaries = [{'name': "a", 'value': "123"}, {'name': "b", 'value': "456"}]
&gt;&gt;&gt; template_string = "blabla {a}"
&gt;&gt;&gt; template_values = {d['name']: d['value'] for d in list_of_dictionaries}
&gt;&gt;&gt; template_values
{'a': '123', 'b': '456'}
&gt;&gt;&gt; template_string.format(**template_values)
'blabla 123'
</code></pre>
</div>
<span class="comment-copy">Is there a reason why your data is in that format? Instead of a list of dictionaries you should just store the name/value pairs in a dict.</span>
<span class="comment-copy">Note: Python uses the name <code>list</code>, not <code>array</code>, for the mutable sequence data structure. There is a <a href="https://docs.python.org/3/library/array.html" rel="nofollow noreferrer">separate <code>array</code> module</a> supporting a single numeric type for each value only.</span>
