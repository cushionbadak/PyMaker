<p>Install a plug-in called Anaconda for sublime using package control. It'll turn your text editor into IDE. With Anaconda you can manually set which python interpreter to use using project settings. You can even set it to custom Python Virtualenv.</p>

<p><a href="https://realpython.com/blog/python/setting-up-sublime-text-3-for-full-stack-python-development/" rel="nofollow noreferrer">Setting up Sublime Text 3 for Full Stack Python Development</a></p>

<div class="post-text" itemprop="text">
<p>On Windows, you dont need create a new build system or specify the absolute path. Edit already existing python build system.</p>
<p>In <code>SublimeText\Data\Packages\</code> create folder <code>Python</code> -&gt; like this <code>SublimeText\Data\Packages\Python\</code>
In this folder, create a file <code>Python.sublime-build</code> , it will overwrite the existing python build system.</p>
<p>In this file write <a href="https://docs.python.org/3/using/windows.html#python-launcher-for-windows" rel="nofollow noreferrer">(python launcher documentation)</a></p>
<pre><code>{
    "shell_cmd": "py -u \"$file\"",
    "file_regex": "^[ ]*File \"(...*?)\", line ([0-9]*)",
    "selector": "source.python",
    "env": {"PYTHONIOENCODING": "utf-8"},
}
</code></pre>
<p>For choosing version of python interpretator, in the first line of your scripts write<a href="https://docs.python.org/3/using/windows.html#shebang-lines" rel="nofollow noreferrer">(shebang lines)</a></p>
<p><code>#! python</code> or
<code>#! python3</code></p>
</div>

<span class="comment-copy">Normally linux/unix binaries (programs) do not have an extension (unlike windows that uses .exe). The default location for python should be in '/usr/bin/python', so in that case you would have: <code>{     "cmd": ["/usr/bin/python", "-u", "$file"],     "file_regex": "^[ ]*File \"(...*?)\", line ([0-9]*)",     "selector": "source.python" }</code> I'm not sure where other versions of python would be installed in linix so you will have to search for that using <code>locate python</code></span>
