<div class="post-text" itemprop="text">
<p>I wanted to have a look at the python deque class. When I checked <a href="http://hg.python.org/cpython/file/791034a0ae1e/Lib/collections/__init__.py" rel="nofollow">the source code</a> , I found the following at line 10</p>
<pre><code> from _collections import deque, defaultdict
</code></pre>
<p>where exactly can I find this _collections module? I searched on my copy of the python source, but couldn't spot it.</p>
<p>Where is this class located?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>_collections</code> is builtin extension module.</p>
<p>You can find source for _collection module <a href="http://hg.python.org/cpython/file/tip/Modules/_collectionsmodule.c">here</a>.</p>
<p><a href="http://hg.python.org/cpython/file/tip/Modules/Setup.dist">Setup.dist</a> contains mapping between builtin extension module name to source file.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>_collections</code> is a private implementation of a class according to this answer: <a href="https://stackoverflow.com/questions/551038/private-implementation-class-in-python">"Private" (implementation) class in Python</a>.</p>
<p>Being private, I don't think that you will be able to access its Python source but you can check out the C implementation <a href="http://hg.python.org/cpython/file/20557286cc35/Modules/_collectionsmodule.c" rel="nofollow noreferrer">here</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>These days the CPython source code (including built-in modules) is <a href="https://github.com/python/cpython" rel="nofollow noreferrer">hosted on Github</a>, so you can find the collections module source code <a href="https://github.com/python/cpython/blob/master/Modules/_collectionsmodule.c" rel="nofollow noreferrer">here</a>.</p>
<p>For the <a href="https://docs.python.org/3/library/collections.abc.html" rel="nofollow noreferrer"><code>collections.abc</code></a> module, see <a href="https://github.com/python/cpython/blob/master/Lib/_collections_abc.py" rel="nofollow noreferrer">here</a>.</p>
</div>
<span class="comment-copy">In this case, <code>_collections</code> is a module implemented in C to support the main <code>collections</code> python module; because it needs to be a distinct, separate module an 'internal' name is used. It is but an implementation detail, after all, not part of the public API.</span>
