<div class="post-text" itemprop="text">
<p>While using sqlite3 from C/C++ I learned that it has a open-in-read-only mode option, which is very handy to avoid accidental data-corruption. Is there such a thing in the Python binding? </p>
</div>
<div class="post-text" itemprop="text">
<p>As of Python 3.4.0 you can open the database in read only mode with the following:</p>
<pre><code>db = sqlite3.connect('file:/path/to/database?mode=ro', uri=True)
</code></pre>
<p>Also see <a href="https://docs.python.org/3/library/sqlite3.html#sqlite3.connect" rel="noreferrer">the documentation</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>Workaround for Python 2.x:</p>
<pre><code>fd = os.open(filename, os.O_RDONLY)
c = sqlite3.connect('/dev/fd/%d' % fd)
os.close(fd)
</code></pre>
<p>Not posix, but available on Linux, OS/X and most modern unixes.</p>
</div>
<div class="post-text" itemprop="text">
<p>Somewhat related, note that you can enable/disable modifications dynamically with a pragma:</p>
<pre><code>pragma query_only = ON;   -- disable changes
pragma query_only = OFF;  -- enable changes
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As by the link given by @Chris, no. But there is another wrapper for sqlite3, which is less PEP 249-compliant and that wraps sqlite3 more tightly, assimilating new features of the engine: <a href="http://code.google.com/p/apsw/" rel="nofollow">http://code.google.com/p/apsw/</a> . That wrapper does support opening the database in read-only mode, plus other niceties. </p>
</div>
