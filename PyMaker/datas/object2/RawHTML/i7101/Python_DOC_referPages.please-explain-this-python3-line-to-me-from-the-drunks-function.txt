<pre><code>def drunks(bottle, end):
    print(bottle or 'No more', 'bottle' + ('s' if bottle - 1 else ''), end)
</code></pre>
<p>Explanation:</p>
<ul>
<li>Your function <code>drunks()</code> takes 2 arguments (<code>bottle</code> and <code>end</code>).</li>
<li>The built-in <code>print()</code> function takes unlimited number of positional arguments (in your case, they are <strong>3</strong>).</li>
<li>So it will print the three values, separated by the default delimiter (which is <strong>a space</strong>).</li>
<li><p>First expression (<code>bottle or 'No more'</code>):</p>
<ul>
<li>Python supports <a href="https://docs.python.org/3/library/stdtypes.html#boolean-operations-and-or-not" rel="nofollow noreferrer"><strong>short-circuit evaluation</strong></a>. See the first row of the table in the link: <code>x or y</code> | <code>if x is false, then y, else x</code></li>
<li>So in your case <code>bottle or 'No more'</code>, if <code>bottle</code> is <a href="https://docs.python.org/3/library/stdtypes.html#truth-value-testing" rel="nofollow noreferrer"><strong>a falsy value</strong></a>, <code>No more</code> will be printed. But if <code>bottle</code> is a <strong>truthy</strong> one, it will be printed (I don't know what type <code>bottle</code> is, but I think it is an integer).</li>
</ul></li>
<li><p>Second expression (<code>bottle + ('s' if bottle - 1 else '')</code>):</p>
<ul>
<li>Here, at first <code>bottle</code> is always printed.</li>
<li>Then, we append the result of <code>'s' if bottle - 1 else ''</code>. We append the letter <code>s</code> if <code>bottle - 1</code> is a <strong>truthy</strong> value (<strong>not a falsy one</strong>), otherwise we append an empty string (<em>which is simply nothing</em>).</li>
</ul></li>
<li>In the end, we print the <code>end</code> parameter, which can also be any type of value.</li>
</ul>
