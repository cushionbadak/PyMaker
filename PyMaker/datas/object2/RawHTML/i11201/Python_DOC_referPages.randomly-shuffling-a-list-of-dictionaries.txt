<p>A similar question I saw on Stack Overflow dealt with a dict of lists (was a bad title), and when I tried using <code>random.shuffle</code> on my list of dicts per that answer, it made the whole object a None-type object.</p>
<p>I have a list of dictionaries kind of like this:</p>
<pre><code>[
{'a':'1211', 'b':'1111121','c':'23423'},
{'a':'101', 'b':'2319','c':'03431'},
{'a':'3472', 'b':'38297','c':'13048132'}
]
</code></pre>
<p>I want to randomly shuffle like this.</p>
<pre><code>[
{'a':'3472', 'b':'38297','c':'13048132'},
{'a':'1211', 'b':'1111121','c':'23423'},
{'a':'101', 'b':'2319','c':'03431'}   
]
</code></pre>
<p>How can I do this?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>random.shuffle</code> should work. The reason I think you <em>thought</em> it was giving you a None-type object is because you were doing something like</p>
<pre><code>x = random.shuffle(...)
</code></pre>
<p>but <code>random.shuffle</code> doesn't return anything, it modifies the list <em>in place</em>:</p>
<pre><code>x = [{'a':'3472', 'b':'38297','c':'13048132'},
     {'a':'1211', 'b':'1111121','c':'23423'},
     {'a':'101', 'b':'2319','c':'03431'}]
random.shuffle(x)  # No assignment
print(x)
</code></pre>
</div>
<span class="comment-copy"><code>random.shuffle</code> should work. Not sure why it's giving you a None-type object.</span>
<span class="comment-copy">If you want to randomly shuffle the elements of a list, independently of what they are, just use <a href="https://docs.python.org/3/library/random.html#random.shuffle" rel="nofollow noreferrer"><code>random.shuffle</code></a>. Otherwise define what you mean by randomly shuffle. Note that <code>random.shuffle</code> <b>modified</b> the original list. Use it as: <code>my_list = [...]; random.shuffle(my_list); print(my_list)</code></span>
<span class="comment-copy"><code>random.shuffle</code> shuffles the list in place and returns <code>None</code>.  You are probably doing <code>x = random.shuffle(x)</code> when you should be doing <code>random.shuffle(x)</code> without assignment.</span>
