<p>I'm wondering if there is a way to deal with dynamic config files using Python's ConfigParser. By dynamic, I mean that the config file isn't fixed in structure.</p>
<p>For example, I could have the config file below.</p>
<pre><code>[SectionA]
FieldA1 = stringA1
FieldA2 = stringA2
FieldA3 = stringA3

[SectionB]
FieldB1 = stringB1
FieldB2 = stringB2
FieldB3 = stringB3
</code></pre>
<p>I could also have the config file below (an extra field-value added to Section B).</p>
<pre><code>[SectionA]
FieldA1 = stringA1
FieldA2 = stringA2
FieldA3 = stringA3

[SectionB]
FieldB1 = stringB1
FieldB2 = stringB2
FieldB3 = stringB3
FieldB4 = stringB4
</code></pre>
<p>The same code should be used to parse both config files. So far, I've only seen examples where the fields are hard-coded, like the example given <a href="https://docs.python.org/3/library/configparser.html" rel="nofollow">here</a>, which would require prior knowledge of the fields in each section.</p>
<pre><code>import configparser
&gt;&gt;&gt; config = configparser.ConfigParser()
&gt;&gt;&gt; config['DEFAULT'] = {'ServerAliveInterval': '45',
...                      'Compression': 'yes',
...                      'CompressionLevel': '9'}
</code></pre>
<p>Is there a way to dynamically read the fields/values in each section?</p>
</div>
<div class="post-text" itemprop="text">
<p>from <a href="https://pymotw.com/2/ConfigParser/" rel="nofollow">pymotw.com</a></p>
<pre><code>from configparser import SafeConfigParser

parser = SafeConfigParser()
parser.read('config.ini')

for section_name in parser.sections():
    print ('Section:', section_name)
    print ('  Options:', parser.options(section_name))
    for name, value in parser.items(section_name):
        print(name, value)
    print()
</code></pre>
<p>I just made an adaptation for python3 since I see you use <code>configparser</code>. Enjoy!</p>
