<p>It is the shell that converts <code>~</code> to the home path.  Python does, however, have its own function for that, <a href="https://docs.python.org/3/library/os.path.html#os.path.expanduser" rel="nofollow"><code>os.path.expanduser()</code></a>:</p>
<pre><code>from os.path import expanduser

def save_csv(self, path, trend_name):
    fileName = expanduser("~/Videos/data.csv")
    with open(fileName, "wb") as f:
        f.write(self.decode_data.encode('utf8'))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As it was mentioned in the comments, you should use the path directly like this:</p>
<pre><code>fileName = "home/&lt;username&gt;/Videos/" + "data" + ".csv"
</code></pre>
<p>as only bash expands <code>~</code> to full <code>home/&lt;username&gt;</code> path.</p>
<p>Also, it uses memory, but then you don't know where does it write to i.e. <code>~/Videos</code>. So yeah, it's saved, but not in the place you want it to.</p>
<p>Similar may be:</p>
<pre><code>with open('file.txt', 'wb')
</code></pre>
<p>Do you know where it'll save? In the folder you open the interpreter from, not in the folder where your <code>main.py</code> is. And in this way you may think the file "isn't saved", but actually it is. Try to look to the interpreter folder if there is a folder <code>~</code> and file <code>Videos...</code> or whole file named <code>~/Videos...</code></p>
<p>Also <code>expanduser()</code> will solve this for you as another answer mentions + to make sure you have the correct path and you didn't leave something out, use <a href="https://docs.python.org/2/library/os.path.html#os.path.join" rel="nofollow"><code>os.path.join()</code></a>.</p>
<p>Example:</p>
<pre><code>from os.path import join
with open(join('/home/&lt;username&gt;', 'Videos', 'data.csv'), mode='wb') as f:
    f.write('test')
</code></pre>
