<div class="post-text" itemprop="text">
<p>PyCharm shows me that imp is deprecated so I wonder if there any analogue of imp.new_module for importlib.</p>
</div>
<div class="post-text" itemprop="text">
<p>Quoting from <a href="https://docs.python.org/3/library/imp.html#imp.new_module">documentation (Emphasis mine) -</a></p>
<blockquote>
<p><strong>imp.new_module(name)</strong></p>
<p>Return a new empty module object called name. This object is not inserted in sys.modules.</p>
<p><strong>Deprecated since version 3.4: Use <a href="https://docs.python.org/3/library/types.html#types.ModuleType">types.ModuleType</a> instead.</strong></p>
</blockquote>
<p>Example -</p>
<pre><code>&gt;&gt;&gt; import types
&gt;&gt;&gt; types.ModuleType('name')
&lt;module 'name'&gt;
</code></pre>
<p>To show how they are synonymous -</p>
<pre><code>&gt;&gt;&gt; import imp
&gt;&gt;&gt; imp.new_module('name')
&lt;module 'name'&gt;
</code></pre>
</div>
<span class="comment-copy">The only problem is, 3.6 documentation does recommend importlib's module_from_spec, but it seems impossible to use it in such a way. Modules constructed with types.ModuleType have None as <code>__spec__</code>, but <code>module_from_spec(None)</code> doesn't work.</span>
