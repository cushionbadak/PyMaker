<div class="post-text" itemprop="text">
<p>The output of below code is different if:</p>
<pre><code>def foo():
    try:
        print 1/0
    except Exception:
        return  'Caught in foo'

try:
    print 'FOOO'
    print foo()
except Exception:
    print 'Caught in main'
</code></pre>
<p>OUTPUT : FOOO \n Caught in foo</p>
<pre><code>try:
    print 'FOOO'
    print foo()
except Exception:
    print 'Caught in main'


def foo():
    try:
        print 1/0
    except Exception:
        return  'Caught in foo'
</code></pre>
<p>OUTPUT : FOOO \n Caught in main</p>
<p>Why is the output different in both cases?</p>
</div>
<div class="post-text" itemprop="text">
<p>Because they raise different exceptions.</p>
<p><a href="https://docs.python.org/3/library/exceptions.html#exception-hierarchy" rel="nofollow"><em>All built-in, non-system-exiting exceptions are derived from <code>Exception</code></em></a>. <code>except Exception</code> catches most of the exceptions and this behaviour is often confusing.</p>
<p>You should <em>always</em> catch specific exceptions e.g. <code>1/0</code> raises <a href="https://docs.python.org/3/library/exceptions.html#ZeroDivisionError" rel="nofollow"><code>ZeroDivisionError</code></a> and you should only catch that:</p>
<pre><code>try:
    print 'FOOO'
    print foo()
except ZeroDivisionError:
    print 'Caught in foo'
</code></pre>
<p>You'll see different error in the output</p>
<pre><code>FOOO
Traceback (most recent call last):
  File "&lt;file&gt;", line 3, in &lt;module&gt;
    print (foo())
NameError: name 'foo' is not defined
</code></pre>
<p>Because it was not caught by <code>except ZeroDivisionError</code> (<code>issubclass(NameError, ZeroDivisionError)</code> is <code>False</code>).</p>
</div>
<div class="post-text" itemprop="text">
<p>Because in the second case the exception is:</p>
<pre><code>NameError: name 'foo' is not defined
</code></pre>
<p>Try <em>not</em> catching the exception in main, or <code>print</code> it. <code>foo</code> isn't defined by the time you try to call it.</p>
</div>
<div class="post-text" itemprop="text">
<p>In the first case you get the following error:</p>
<pre><code>ZeroDivisionError: integer division or modulo by zero
</code></pre>
<p>In the second case however you get:</p>
<pre><code>NameError: name 'foo' is not defined
</code></pre>
<p>The second one occurs because the <code>foo()</code> function is not defined yet when you call it.</p>
</div>
<div class="post-text" itemprop="text">
<p>Because in the second case, foo is not defined yet and thus calling foo() throws an exception.</p>
</div>
<span class="comment-copy">The answers show you why it is always best to only catch specific exceptions, like <code>ZeroDivisionError</code> in your case, and not all exceptions,as it makes finding problems very hard.</span>
