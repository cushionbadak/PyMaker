<div class="post-text" itemprop="text">
<p>As an alternative to escaping the backslash, you can disable backslash-escaping entirely by using a raw string literal:</p>
<pre><code>&gt;&gt;&gt; print(r"abc\ndef")
abc\ndef
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You should escape the backslash so that it's not treated as escaping character itself:</p>
<pre><code>Sys.stdout.write("abc\\ndef")
</code></pre>
<p><strong>Background</strong></p>
<p>The backslash <code>\</code> tells the parser that the next character is something special and must be treated differently. That's why <code>\n</code> will not print as <code>\n</code> but as a newline. But how do we write a backslash then? We need to escape it, too, resulting in <code>\\</code> for a single backslash and <code>\\n</code> for the output <code>\n</code>.</p>
<p><a href="https://docs.python.org/3/reference/lexical_analysis.html#string-and-bytes-literals" rel="nofollow noreferrer">Docs here</a>, also <a href="https://stackoverflow.com/questions/2081640/what-exactly-do-u-and-r-string-flags-do-in-python-and-what-are-raw-string-l">see this SO question</a></p>
