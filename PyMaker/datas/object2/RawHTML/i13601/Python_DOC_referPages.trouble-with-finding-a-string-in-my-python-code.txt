<p>Yes, you did wrong in passing the email-id variable, the <code>.format()</code> should be outisde the string, not inside it. Example -</p>
<pre><code>content = urllib2.urlopen('https://somewebsite.com/uniqueEmail?email={email_id}'.format(email_id=email_id)).read()
</code></pre>
<p>Secondly, if your website returns a response like -</p>
<pre><code>{"params":{"method":"GET","controller":"GlobalUsers","action":"uniqueEmail","email":"theemailaddress_entered"},"uniqueEmail":true}
</code></pre>
<p>This seems like a json response, so you should parse it as json, using the <a href="https://docs.python.org/3/library/json.html" rel="nofollow"><code>json</code> module</a> and then that would return you a dictionary, and from the dictionary you can get the <code>uniqueEmail</code> information. Example -</p>
<pre><code>content = urllib2.urlopen('https://somewebsite.com/uniqueEmail?email={email_id}'.format(email_id=email_id)).read()

import json
contentdict = json.loads(content)
if contentdict.get('uniqueEmail':   
    print("email exists")
else:
    print("email doesnt exists")
</code></pre>
<p>I am using <code>dict.get()</code> as that returns <code>None</code> if the key is not present, in which case it would mean the email doesn't exist.</p>
