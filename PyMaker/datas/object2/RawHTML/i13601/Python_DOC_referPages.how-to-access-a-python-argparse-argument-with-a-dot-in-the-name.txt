<p>Python's argparse lets me define argument names containing a dot in the name. But how can I access these ?</p>
<pre><code>import argparse


parser = argparse.ArgumentParser()
parser.add_argument("inputfile.txt")
parser.add_argument("outputfile.txt")

args = parser.parse_args(['hello', 'world'])

# now args is:
# Namespace(inputfile.txt='hello', outputfile.txt='world')


# and this does not work
print(args.inputfile.txt)
&gt;&gt;&gt; AttributeError: 'Namespace' object has no attribute 'inputfile'
</code></pre>
<p>Obviously attribute names can be created with a dot in their name but how can these be accessed ?</p>
<p>Edit:
My goal was to let argparse display the inputfile.txt name (e.g. with --help) but call the attribute "inputfile".</p>
<p>After trying some of the suggestions the easiest way to accomplish this is using the metavar option:</p>
<pre><code>parser.add_argument("inputfile", metavar="inputfile.txt")
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Internally, <code>argparse</code> will add all attributes to the <code>Namespace()</code> instance with <a href="https://docs.python.org/2/library/functions.html#setattr" rel="nofollow"><code>setattr</code></a>. Accordingly, you can access the values with <a href="https://docs.python.org/2/library/functions.html#getattr" rel="nofollow"><code>getattr</code></a>:</p>
<pre><code>getattr(args, 'inputfile.txt')
</code></pre>
<p>However, this is not as intuitive and in general attribute names with dots in them should be avoided.  It is recommended to use the <a href="https://docs.python.org/3/library/argparse.html#dest" rel="nofollow"><code>dest</code></a> option of <a href="https://docs.python.org/3/library/argparse.html#argparse.ArgumentParser.add_argument" rel="nofollow"><code>argparse.add_argument</code></a> to define your own variable in which to store the value for that argument, as hd1 suggests in their answer.</p>
</div>
<div class="post-text" itemprop="text">
<p>Using the dest option, you can assign it to any thing:</p>
<pre><code>parser = argparse.ArgumentParser()
parser.add_argument("inputfile.txt", dest='infile')
parser.add_argument("outputfile.txt", dest='out')

args = parser.parse_args(['hello', 'world'])
# now args is:
# Namespace(infile='hello', out='world')
</code></pre>
<p>Hope that helps.</p>
</div>
<div class="post-text" itemprop="text">
<p>Why not use <a href="https://docs.python.org/2.7/library/argparse.html#filetype-objects" rel="nofollow">FileType</a> objects instead?</p>
<pre><code>&gt;&gt;&gt; import argparse
&gt;&gt;&gt;
&gt;&gt;&gt; parser = argparse.ArgumentParser()
&gt;&gt;&gt; parser.add_argument('--infile', type=argparse.FileType('r'))
_StoreAction(option_strings=['--infile'], dest='infile', nargs=None, const=None, default=None, type=FileType('r'), choices=None, help=None, metavar=None)
&gt;&gt;&gt; parser.add_argument('--outfile', type=argparse.FileType('w'))
_StoreAction(option_strings=['--outfile'], dest='outfile', nargs=None, const=None, default=None, type=FileType('w'), choices=None, help=None, metavar=None)
&gt;&gt;&gt; parser.parse_args(['--infile', 'input.txt', '--outfile', 'output.txt'])
Namespace(infile=&lt;open file 'input.txt', mode 'r' at 0x100f99e40&gt;, outfile=&lt;open file 'output.txt', mode 'w' at 0x100f99ed0&gt;)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Rename the argument to <code>inputfile</code> and use <code>metavar</code> to set the display value for the user.</p>
<pre><code>import argparse


parser = argparse.ArgumentParser()
parser.add_argument("inputfile", metavar = "inputfile.txt")
parser.add_argument("outputfile", metavar = "outputfile.txt")

args = parser.parse_args(['hello', 'world'])

# now args is:
# Namespace(inputfile='hello', outputfile='world')

print(args.inputfile)
</code></pre>
</div>
