<p>In Python 3, suppose I have </p>
<pre><code>&gt;&gt;&gt; thai_string = 'สีเ'
</code></pre>
<p>Using <code>encode</code> gives</p>
<pre><code>&gt;&gt;&gt; thai_string.encode('utf-8')
b'\xe0\xb8\xaa\xe0\xb8\xb5'
</code></pre>
<p>My question: how can I get <code>encode()</code> to return a <code>bytes</code> sequence using <code>\u</code> instead of <code>\x</code>?  And how can I <code>decode</code> them back to a Python 3 <code>str</code> type?</p>
<p>I tried using the <code>ascii</code> builtin, which gives</p>
<pre><code>&gt;&gt;&gt; ascii(thai_string)
"'\\u0e2a\\u0e35'"
</code></pre>
<p>But this doesn't seem quite right, as I can't decode it back to obtain <code>thai_string</code>.</p>
<p><a href="https://docs.python.org/3/reference/lexical_analysis.html" rel="nofollow noreferrer">Python documentation</a> tells me that </p>
<ul>
<li><code>\xhh</code> escapes the character with the hex value <code>hh</code> while </li>
<li><code>\uxxxx</code> escapes the character with the 16-bit hex value <code>xxxx</code></li>
</ul>
<p>The documentation says that <code>\u</code> is only used in string literals, but I'm not sure what that means.  Is this a hint that my question has a flawed premise?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>unicode_escape</code>:</p>
<pre><code>&gt;&gt;&gt; thai_string.encode('unicode_escape')
b'\\u0e2a\\u0e35\\u0e40'
</code></pre>
<p>Note that <code>encode()</code> will always return a byte string (bytes) and the <code>unicode_escape</code> encoding <a href="https://docs.python.org/3.3/library/codecs.html#python-specific-encodings" rel="noreferrer">is intended to</a>:</p>
<blockquote>
<p>Produce a string that is suitable as Unicode literal in Python source code</p>
</blockquote>
</div>
