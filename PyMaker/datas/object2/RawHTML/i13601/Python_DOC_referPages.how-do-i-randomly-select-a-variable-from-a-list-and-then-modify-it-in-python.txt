<div class="post-text" itemprop="text">
<h2>Problem:</h2>
<p><code>random.choice(cells)</code> returns a random value from your list, for example <code>"3"</code>, and you are trying to assign something to it, like:</p>
<pre><code>"3" = "X"
</code></pre>
<p>which is wrong. </p>
<p>Instead of this, you can modify the <code>list</code>, for example:</p>
<pre><code>cells[5] = "X"
</code></pre>
<h2>Solution:</h2>
<p>You can use <a href="https://docs.python.org/3/library/random.html#random.randrange" rel="nofollow"><code>random.randrange()</code></a>.</p>
<pre><code>import random
cells = [str(i) for i in range(1,10)] # your list
cpuletter = 'X'

print(cells)
random_index = random.randrange(len(cells)) # returns an integer between [0,9]
cells[random_index] = cpuletter
print(cells)
</code></pre>
<p><strong>Output:</strong></p>
<pre><code>['1', '2', '3', '4', '5', '6', '7', '8', '9']
['1', '2', '3', '4', '5', '6', '7', 'X', '9']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>Random.choice(cells)</code> returns a random <em>element</em> from <code>cells</code>, so if it returned element #0 and element #0 was the value <code>"1"</code>, your original statement would essentially be saying <code>"1" = "X"</code> - obviously you can't assign the string literal to be something else, it's not a variable.  </p>
<p>You'd instead want to get a <em>random element #</em>, and assign <code>cells[rand_elt_num]</code>.
You could get the random element number by something simply like:</p>
<pre><code>rand_elt_num = random.randint(0, len(cells)-1 )  #get index to a random element
cells[rand_elt_num] = "X"    # assign that random element
</code></pre>
<p>I think this is the same as what the other answer says, they just used <code>random.randrange()</code> instead.</p>
</div>
<span class="comment-copy">Both values in <a href="https://docs.python.org/2/library/random.html#random.randint" rel="nofollow noreferrer"><code>random.randint()</code></a> is inclusive. So, you need to do <code>random.randint(0, len(cells)-1)</code> instead of <code>random.randint(0, len(cells))</code> if you really insist on using <code>random.randint()</code>.</span>
