<p>If you mean that you want it to match a string like <code>/pid/0002</code> but not <code>/rapid.html</code>, then you need to exclude word characters on either side. Something like:</p>
<pre><code>&gt;&gt;&gt; re.search(r'\Wpid\W', '/pid/0002')
&lt;_sre.SRE_Match object; span=(0, 5), match='/pid/'&gt;
&gt;&gt;&gt; re.search(r'\Wpid\W', '/rapid/123')
None
</code></pre>
<p>If 'pid' might be at the start or end of the string, you'll need to add extra conditions: check for either the start/end of line or a non-word character:</p>
<pre><code>&gt;&gt;&gt; re.search(r'(^|\W)pid($|\W)', 'pid/123')
&lt;_sre.SRE_Match object; span=(0, 4), match='pid/'&gt;
</code></pre>
<p>See <a href="https://docs.python.org/3.4/library/re.html#regular-expression-syntax" rel="nofollow">the docs</a> for more information on the special characters.</p>
<p>You could use it like this:</p>
<pre><code>pattern = re.compile(r'(^|\W)pid($|\W)')
if pattern.search(a['href']) is not None:
    ...
</code></pre>
