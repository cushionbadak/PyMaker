<p>Why is the following python code throwing an exception?</p>
<pre><code>import urllib

htmlfile=urllib.urlopen("http://google.com")
htmltext=htmlfile.read()
print (htmltext)
</code></pre>
<p><em>Traceback</em></p>
<pre><code>Traceback (most recent call last):
  File "C:/Users/dhfur/Desktop/webcrawl1.py", line 3, in &lt;module&gt;
    htmlfile=urllib.urlopen("http://google.com")
AttributeError: 'module' object has no attribute 'urlopen'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Because the functionality was moved to the <a href="https://docs.python.org/3/library/urllib.request.html" rel="nofollow"><code>urllib.request</code> module</a>:</p>
<pre><code>import urllib.request

htmlfile=urllib.request.urlopen("http://google.com")
</code></pre>
<p>A whole lot of things were moved in Python 3 from their former Python 2 locations. The Python 2 documentation usually names the new location; the <a href="https://docs.python.org/2/library/urllib.html" rel="nofollow"><code>urllib</code> documentation page</a> is no exception:</p>
<blockquote>
<p><strong>Note</strong>: The <code>urllib</code> module has been split into parts and renamed in Python 3 to <code>urllib.request</code>, <code>urllib.parse</code>, and <code>urllib.error</code>. The <em>2to3</em> tool will automatically adapt imports when converting your sources to Python 3. Also note that the <code>urllib.request.urlopen()</code> function in Python 3 is equivalent to <code>urllib2.urlopen()</code> and that <code>urllib.urlopen()</code> has been removed.</p>
</blockquote>
<p>The <a href="https://docs.python.org/3/library/urllib.html" rel="nofollow">Python 3 documentation page for <code>urllib</code></a> names all new modules under the namespace.</p>
