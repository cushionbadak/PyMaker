<div class="post-text" itemprop="text">
<p>This error raised because the <code>data</code> is a unicode/str variable, change the second line of your code to resolve your error:</p>
<pre><code>data = json.loads(data)
</code></pre>
<p><code>json.load</code> get a file object in first parameter position and call the <code>read</code> method of this.</p>
<p>Also you can call the <code>json</code> method of the response to fetch data directly:</p>
<pre><code>response = requests.get('http://ws.audioscrobbler.com/2.0/?method=library.getartists&amp;api_key=4c22bd45cf5aa6e408e02b3fc1bff690&amp;user=joanofarctan&amp;format=json')
data = response.json()
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><a href="http://www.python-requests.org/en/latest/api/#requests.Response.text" rel="nofollow"><code>requests.get(…).text</code></a> returns the content as a single (unicode) string. The <a href="https://docs.python.org/3/library/json.html#json.load" rel="nofollow"><code>json.load()</code></a> function however requires a file-like argument.</p>
<p>The solution is rather simple: Just use <a href="https://docs.python.org/3/library/json.html#json.loads" rel="nofollow"><code>loads</code></a> instead of <code>load</code>:</p>
<pre><code>data = json.loads(data)
</code></pre>
<p>An even better solution though is to simply call <a href="http://www.python-requests.org/en/latest/api/#requests.Response.json" rel="nofollow"><code>json()</code></a> on the response object directly. So don’t use <code>.text</code> but <code>.json()</code>:</p>
<pre><code>data = requests.get(…).json()
</code></pre>
<p>While this uses <code>json.loads</code> itself internally, it hides that implementation detail, so you can just focus on getting the JSON response.</p>
