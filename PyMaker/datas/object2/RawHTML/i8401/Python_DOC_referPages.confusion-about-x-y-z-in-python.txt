<p>I'm newbie in python. I have a three variables <code>x</code>, <code>y</code>, <code>z</code> as a <code>int</code>. I have comparison three variables in <code>if</code> condition. I'm confused about following code result.</p>
<p>The expression <code>x &lt; y &lt;= z</code> evaluates to false.</p>
<p>Let's assume <code>x = 10</code>, <code>y = 5</code> and <code>z = 0</code>. if <code>x &lt; y</code> become <strong>False</strong>, then <code>False &lt;= 0</code> become <strong>True</strong>. <strong>but output is False. Why?</strong></p>
<p>My python script:</p>
<pre><code>#!/usr/bin/python

x = 10
y = 5
z = 0

if (x &lt; y &lt; z):
        print"True"
else:
        print"False"
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/stdtypes.html" rel="nofollow noreferrer">document</a> say:</p>
<blockquote>
<p>Comparisons can be chained arbitrarily; for example, x &lt; y &lt;= z is
  equivalent to x &lt; y and y &lt;= z, except that y is evaluated only once
  (but in both cases z is not evaluated at all when x &lt; y is found to be
  false).</p>
</blockquote>
<p><code>x &lt; y &lt;= z</code> neither means <code>(x &lt; y) &lt;= z</code> nor <code>x &lt; (y &lt;= z)</code>. <code>x &lt; y &lt;= z</code> is equivalent to <code>x &lt; y and y &lt;= z</code>, and is evaluates from left-to-right.</p>
<p><strong>Logical AND</strong> do not have associativity in Python unlike C and C++. There are separate rules for sequences of this kind of operator and cannot be expressed as associativity.</p>
<p><code>x &lt; y and y &lt;= z</code> only evaluates the second argument if the first one is <strong>true</strong> because <code>and</code> is a short-circuit operator.</p>
</div>
<div class="post-text" itemprop="text">
<p>That expression is evaluated as:</p>
<pre><code>if (x &lt; y and y &lt; z):
    pass
</code></pre>
<p>So now you see why it is false is because <code>x</code> is not less than <code>y</code></p>
