<p>I have problems to set up correctly my if statement.</p>
<p>This is my code:</p>
<pre><code>def task_13():
    Main_meal=['Meat','Cheese','Fish']
    addons=['Potatoes','Rice','Salad']
    my_meal=[(x+y) for x in Main_meal for y in addons if (x+y)!= 'FishRice' and 'CheeseRice']
    print(my_meal)
</code></pre>
<p>My question is why Python filter out the 'CheeseRice' when is it stated there but only filter out the 'FishRice' option.</p>
<p>This is my output:</p>
<pre><code>['MeatPotatoes', 'MeatRice', 'MeatSalad', 'CheesePotatoes', 'CheeseRice', 'CheeseSalad', 'FishPotatoes', 'FishSalad']
</code></pre>
<p>Thank you for your advice.</p>

<div class="post-text" itemprop="text">
<p>Here's the official <a href="https://docs.python.org/3/reference/expressions.html#operator-precedence" rel="nofollow">reference on Python operator precedence</a>, note that <code>and</code> is lower precedence than <code>!=</code>, so the <code>!=</code> is evaluated first. Also <code>and</code> is a simple operator that takes the booleans on either side and returns a boolean representing their logical AND, it doesn't do what you tried to make it do.</p>
<p>Instead of</p>
<pre><code>if (x+y)!= 'FishRice' and 'CheeseRice'
</code></pre>
<p>you need:</p>
<pre><code>if (x+y)!= 'FishRice' and (x+y) != 'CheeseRice'
</code></pre>
<p>or alternatively</p>
<pre><code>if (x+y) not in ('FishRice', 'CheeseRice')
</code></pre>
