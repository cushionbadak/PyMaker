<p>I have a spider with scrapy and I am using a regex to split text on every word. It works in English and Korean, but when I try to scrape Spanish it doesn't break things up the same.</p>
<pre><code>foo = 'the quick brown fox'
english = foo.xpath('//*[@id="mw-content-text"]//p').xpath('string(.)').re(ur'[a-zA-Z\'-]+')
english = ['the','quick','brown','fox',]

foo = '빨른 깨를 있어요'
korean = foo.xpath('//*[@id="mw-content-text"]//p').xpath('string(.)').re(ur'[\uac00-\ud7af]+')
korean = ['빨른','깨를','있어요']

foo = 'el zorro marron rapido'
spanish = foo.xpath('//*[@id="mw-content-text"]//p').xpath('string(.)').re(ur'[\u0000-\u00FF]+')
spanish = ['el zorro marron rapido']
</code></pre>
<p>I just can't see why this would be treated any differently than the other languages.</p>

<div class="post-text" itemprop="text">
<p>Unlike other regular expressions, <code>[\u0000-\u00FF]</code> include space character (<code>\u0020</code>).</p>
<pre><code>&gt;&gt;&gt; import re
&gt;&gt;&gt; re.search(ur'[a-zA-Z\'-]', ' ')  # does not match space
&gt;&gt;&gt; re.search(ur'[\uac00-\ud7af]', ' ')  # does not match space
&gt;&gt;&gt; re.search(ur'[\u0000-\u00FF]', ' ')  # match space
&lt;_sre.SRE_Match object at 0x7fe024fc4648&gt;
</code></pre>
<p>Adjust the regular expression to exclude the space character. For example: <code>[\u0000-\u001f\u0021-\u00FF]+</code></p>
<hr/>
<p>But, using regular expression here is not necessary. Simply using <a href="https://docs.python.org/3/library/stdtypes.html#str.split" rel="nofollow"><code>str.split</code></a> is enough to get words splitted by spaces:</p>
<pre><code>&gt;&gt;&gt; 'the quick brown fox'.split()
['the', 'quick', 'brown', 'fox']
&gt;&gt;&gt; '빨른 깨를 있어요'.split()
['빨른', '깨를', '있어요']
&gt;&gt;&gt; 'el zorro marron rapido'.split()
['el', 'zorro', 'marron', 'rapido']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>There are a good number of Unicode whitespace definitions both above and below your <code>\u00FF</code> boundary. See <a href="https://en.wikipedia.org/wiki/Whitespace_character#Unicode" rel="nofollow">here</a> for example. Can't you use <code>\w+</code> ? It would be the much preferred approach.</p>
