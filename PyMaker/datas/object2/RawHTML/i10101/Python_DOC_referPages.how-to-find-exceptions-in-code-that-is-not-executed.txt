<div class="post-text" itemprop="text">
<p>In this simple Python code if the value of <code>a</code> is changed from <code>0</code> then there will be a <code>NameError</code> showing that <code>b</code> is undefined. However, if the <code>else</code> clause is never executed, the error remains hidden. How can I detect these kind of errors?</p>
<pre><code>a=0
if a==0 :
  print "hello"
else :
  print b
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://www.pylint.org/">pylint</a> is able to find a lot of common (and not so common) errors in python code without executing it, example output from your example:</p>
<pre><code>C:  1, 0: Exactly one space required around assignment
a=0
 ^ (bad-whitespace)
C:  2, 0: Exactly one space required around comparison
if a==0 :
    ^^ (bad-whitespace)
C:  2, 0: No space allowed before :
if a==0 :
        ^ (bad-whitespace)
C:  4, 0: No space allowed before :
else :
     ^ (bad-whitespace)
C:  1, 0: Missing module docstring (missing-docstring)
C:  1, 0: Invalid constant name "a" (invalid-name)
E:  5,10: Undefined variable 'b' (undefined-variable)
</code></pre>
<p>The line you're interested in here is: <code>E:  5,10: Undefined variable 'b' (undefined-variable)</code></p>
</div>
<div class="post-text" itemprop="text">
<p>There are two ways to achieve this:</p>
<ol>
<li><p>Either include such statements within the <a href="https://docs.python.org/3/tutorial/errors.html#handling-exceptions" rel="nofollow"><code>try/except</code></a> block. For example:</p>
<pre><code>&gt;&gt;&gt; a = 1
&gt;&gt;&gt; if a == 0:
...     print a
... else:
...     try:
...         print b
...     except Exception as e:
...         print 'Caught Exception: ', e  # where e is the exception string
...
Caught Exception:  name 'b' is not defined
</code></pre>
<p>Use may also use <a href="https://docs.python.org/2/library/traceback.html" rel="nofollow"><code>traceback</code></a> to get all the information related to raised <code>Exception</code>.</p></li>
<li><p>If you want this behavior at the scope of function, create a <a href="http://simeonfranklin.com/blog/2012/jul/1/python-decorators-in-12-steps/" rel="nofollow"><code>decorator</code></a> and use that decorator with the functions you need this behavior.</p></li>
</ol>
