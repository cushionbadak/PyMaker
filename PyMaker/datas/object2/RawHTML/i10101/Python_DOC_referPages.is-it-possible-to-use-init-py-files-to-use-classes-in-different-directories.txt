I use those classes in different subdirectories. My files are organized that way : </p>

|-- MyClasses
|   |-- class1_file.py

<p>I was using <code>sys.path.append("absolute_path_to_/MyClasses")</code>to use my classes in the subdirectories program (for example in <code>MyPrograms1/program_using_class1.py</code>) , which was ok as long as I was working on my computer.

<div class="post-text" itemprop="text">
<p>You are going to need to be importing those classes as a single module by adding <code>__init__.py</code> inside the classes' directory. But instead of leaving it blank you need to import the classes:</p>
<p><code>from . import class1_file, class2_file</code></p>
<p>And then you'll need that directory with the classes in order to import them in another computer. </p>
</div>
<div class="post-text" itemprop="text">
<p>I solved my problem using <code>sys.path.append</code>,<code>os.chdir</code>,<code>os.path.dirname</code> and <code>os.path.abspath</code>. 
I added the path to my module to <code>PYTHONPATH</code> using <code>sys.path.append</code> so the program could find it. </p>
<p>I used this code : </p>
<pre><code>import os
import sys
os.chdir(os.path.dirname(os.path.dirname(__file__)))
sys.path.append(os.path.abspath('MyClasses')
from class1_file import Class1
</code></pre>
<p>I would like to thank Stam Kaly for making me discover the <code>chdir</code> function.
You have to adapt the <code>os.chdir</code> part to your file organization. </p>
<p>Thank you for all the help. </p>
