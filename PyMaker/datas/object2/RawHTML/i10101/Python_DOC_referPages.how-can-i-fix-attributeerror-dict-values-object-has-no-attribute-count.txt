<div class="post-text" itemprop="text">
<p>In Python3 <code>dict.values()</code> returns "views" instead of lists:</p>
<blockquote>
<ul>
<li>dict methods dict.keys(), dict.items() and dict.values() return "views" instead of lists.
  <a href="https://docs.python.org/3/whatsnew/3.0.html" rel="noreferrer">https://docs.python.org/3/whatsnew/3.0.html</a></li>
</ul>
</blockquote>
<p>To convert the "view" into a list, simply wrap <code>in_degrees.values()</code> in a <code>list()</code>:</p>
<pre><code>in_hist = [list(in_degrees.values()).count(x) for x in in_values]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The error is obvious <code>dict_values</code> object has no attribute <code>count</code>. If you want to count the number of unique values, the best way to go is using a <code>collections.Counter</code></p>
<pre><code>from collections import Counter
in_hist = Counter(in_degrees.values())
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If you want to count dictionary values you can do it like this:</p>
<pre><code>len(list(dict.values()))
</code></pre>
<p>same method works for keys</p>
<pre><code>len(list(dict.keys()))
</code></pre>
<p>Also keep in mind if you want to get all keys or values in list just use <code>list(dict.values())</code></p>
</div>
<span class="comment-copy">I assume you're using Python 3. If that's correct, please add the python-3.x tag to your question. Some of the <code>dict</code> methods behave differently in Python 3 vs Python 2.</span>
<span class="comment-copy"><code>len(mydict)</code> is sufficient, since in this context the len of a dict is the number of keys. No need to produce a list of the values or keys. And making a copy by the use of the <code>list</code> operator is unnecessary as well since both <code>mydict.keys()</code> and <code>mydict.values()</code> return lists.</span>
