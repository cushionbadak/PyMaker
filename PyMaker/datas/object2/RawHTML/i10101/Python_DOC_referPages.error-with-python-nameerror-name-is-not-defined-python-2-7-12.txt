<div class="post-text" itemprop="text">
<p>You need to use <code>raw_input()</code> as opposed to <code>input()</code> as you are using Python 2.x.</p>
<p>In Python 2.x <code>input()</code> will will try and evaluate your input whereas <code>raw_input()</code> will return a string which you can use. </p>

<div class="post-text" itemprop="text">
<p>Change <a href="https://docs.python.org/2/library/functions.html#input" rel="nofollow"><code>input</code></a> to <a href="https://docs.python.org/2/library/functions.html#raw_input" rel="nofollow"><code>raw_input</code></a> which returns a string.</p>
<p>On Python <code>2.x</code> <a href="https://docs.python.org/2/library/functions.html#input" rel="nofollow"><code>input</code></a> tries to evaluate its input, while <a href="https://docs.python.org/2/library/functions.html#raw_input" rel="nofollow"><code>raw_input</code></a> returns a string. On Python 3 <a href="https://docs.python.org/3/library/functions.html#input" rel="nofollow"><code>input</code></a> replaces <a href="https://docs.python.org/2/library/functions.html#raw_input" rel="nofollow"><code>raw_input</code></a></p>
</div>

<div class="post-text" itemprop="text">
<p>Difference between <code>raw_input()</code> and <code>input()</code>:  </p>
<h3>In Python2.x:</h3>
<p><strong><code>input()</code> is looking for an expression.</strong> </p>
<p>if you give <code>1+1</code> to <code>input()</code>, it will evaluate it and return <code>2</code>.<br/>
<code>jeff</code> being not an expression, it will throw an error.<br/>
if input is <code>"jeff"</code> or <code>'jeff'</code>, <code>input()</code> will return the same string.</p>
<p><strong><code>raw_input()</code> is looking for a string.</strong> </p>
<p>if you give <code>1+1</code> to <code>raw_input()</code>, it will return <code>"1+1"</code>(string).<br/>
if you give <code>jeff</code> to <code>raw_input()</code>, it will return <code>"jeff"</code>.</p>
<h3>In Python3.x:</h3>
<p><code>raw_input()</code> is renamed as <code>input()</code> and <code>raw_input()</code> is removed.</p>
