<div class="post-text" itemprop="text">
<p>From the <a href="https://docs.python.org/3/library/subprocess.html#subprocess.Popen.communicate" rel="nofollow">docs</a>:</p>
<pre><code>communicate(self, input=None)
   Interact with process: Send data to stdin.  Read data from
   stdout and stderr, until end-of-file is reached.  Wait for
   process to terminate.  The optional input argument should be a
   string to be sent to the child process, or None, if no data
   should be sent to the child.
</code></pre>
<p>Notice, <code>Wait for process to terminate.</code>  I think what you need is <a href="https://pypi.python.org/pypi/pexpect/" rel="nofollow">pexpect</a>.  It isn't in the standard library, but it'll do what you want.</p>
<p>Example:</p>
<pre><code>import pexpect

process = pexpect.spawn("python")
process.expect_exact("&gt;&gt;&gt; ")
process.sendline('print("It works")')
process.expect("\n.*\n")
response = process.after.strip()
print(response)
</code></pre>
<p>Output:</p>
<pre><code>It works
</code></pre>

<p>If you want to send input to your shell then you should pass it as an argument in <code>communicate</code>, like so:</p>
<pre><code>prcs = subprocess.Popen("MYSHELL",shell=True,
subprocess.PIPE, stdout=subprocess.PIPE, stderr=subprocess.PIPE)

outputPrcs, err =  prcs.communicate("cmnd1; cmnd2;")
print outputPrcs
</code></pre>
