<p>Can my Python script take its arguments from a file, rather than the command line? I don't mind passing the file containing the arguments on the command line.</p>
<p>I am using <a href="https://www.google.co.uk/url?sa=t&amp;rct=j&amp;q=&amp;esrc=s&amp;source=web&amp;cd=1&amp;cad=rja&amp;uact=8&amp;ved=0ahUKEwiaor2D3urKAhVDxxQKHWH7B3oQFggjMAA&amp;url=https%3A%2F%2Fdocs.python.org%2F3%2Flibrary%2Fargparse.html&amp;usg=AFQjCNGfxmw2zSSIQ5m3V5GRr0EyRzJsiw&amp;sig2=qXHNypz9R4ruJV3BQHNBRg" rel="nofollow">argparse</a>.</p>
<p>The reason is that I have a very complex argument list. I suppose that I could just wrap the call in a batch file, or another Python script, but I wondered if this is possible and thought that I would ask and maybe learn something.</p>
<p>So, instead of <code>myScript.py --arg_1=xxx --arg2_=xxx ... --arg_n=xxx</code>, can I</p>
<p><code>myScript.py --file args.txt</code> where args.txt contains</p>
<pre><code>--arg_1=xxx 
--arg_2=xxx 
... 
--arg_n=xxx
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can tell the parser that arguments beginning with certain characters are actually names of files containing more arguments. From the <a href="https://docs.python.org/3/library/argparse.html#fromfile-prefix-chars" rel="nofollow">documentation</a>:</p>
<pre><code>&gt;&gt;&gt; with open('args.txt', 'w') as fp:
...    fp.write('-f\nbar')
&gt;&gt;&gt; parser = argparse.ArgumentParser(fromfile_prefix_chars='@')
&gt;&gt;&gt; parser.add_argument('-f')
&gt;&gt;&gt; parser.parse_args(['-f', 'foo', '@args.txt'])
Namespace(f='bar')
</code></pre>
<p>The parser reads from <code>args.txt</code> and treats each line as a separate argument.</p>
</div>
<div class="post-text" itemprop="text">
<p>You can do this by taking command line argument as filename and then opening it. 
like</p>
<pre><code>file_name=sys.argv[1]
f=open(file_name)
arguments= f.read()
user_input=arguments.split()
user_argument=[]
for i in range():
    user_argument.append(user_input[i])
</code></pre>
<p>Here you get the list of user argument in the list user_argument. Perheps you will get what you want!</p>
