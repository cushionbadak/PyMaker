<p>Trying something out with partial, I observed the following behaviour:</p>
<p>First, I defined a function <code>foo</code> which takes 2 non-keyword arguments:</p>
<pre><code>&gt;&gt;&gt; def foo(salutation, name):
...     print(salutation, name)
</code></pre>
<p>Then, I use partial to create a higher order wrapper.</p>
<pre><code>&gt;&gt;&gt; from functools import partial 
&gt;&gt;&gt; f = partial(foo, name='Coldspeed')
</code></pre>
<p>Then, called it like this:</p>
<pre><code>&gt;&gt;&gt; f('Hey')
</code></pre>
<p>This gave me <code>Hey Coldspeed</code> which was expected. Next, I tried to apply <code>partial</code> to <a href="https://docs.python.org/3/library/functions.html#isinstance" rel="nofollow noreferrer"><code>isinstance</code></a>:</p>
<pre><code>&gt;&gt;&gt; f = partial(isinstance, classinfo=str)
&gt;&gt;&gt; f('hello')
</code></pre>
<p>But this gave me an error:</p>
<pre><code>Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: isinstance() takes no keyword arguments 
</code></pre>
<p>I tried <code>classinfo</code> as the argument name since that was what was specified in the docs (linked above). It seems that the actual definition does <em>not</em> have the second argument named <code>classinfo</code> as the docs might otherwise state.</p>
<p>How now, would I create a partial function with the second parameter without being forced to specify the argname? I definitely cannot do <code>f = partial(isinstance, str)</code> because <code>str</code> will be taken as the first argument, not the second. If possible, would like to find a way to do this without using <code>lambda</code> functions too.</p>
</div>
<div class="post-text" itemprop="text">
<p>Unfortunately, that's just not something <code>functools.partial</code> supports. You'd need to write your own version that does support it, or find one in some 3rd-party library somewhere.</p>
</div>
<span class="comment-copy">Just do <code>f = lambda item: isinstance(item, str)</code>.</span>
<span class="comment-copy">@jonrsharpe Trying to avoid <code>lambdas</code> completely... :p</span>
<span class="comment-copy">Then write it out longhand with <code>def</code>. Same principle: <code>def f(item): return isinstance(item, str)</code>.</span>
