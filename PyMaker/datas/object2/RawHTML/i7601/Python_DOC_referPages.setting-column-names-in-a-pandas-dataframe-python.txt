<p>When setting a column name for a pandas dataframe, why does the following work:</p>
<pre><code>df_coeff = pd.DataFrame(data = lm.coef_, index = X.columns, columns = ['Coefficient'])
</code></pre>
<p>While this does not work</p>
<pre><code>df_coeff = pd.DataFrame(data = lm.coef_, index = X.columns, columns = 'Coefficient')
</code></pre>
<p>Why are brackets required around the column name? What is Python doing here?</p>
<p>Thanks</p>

<div class="post-text" itemprop="text">
<p>The <a href="https://pandas.pydata.org/pandas-docs/stable/generated/pandas.DataFrame.html" rel="nofollow noreferrer"><code>DataFrame</code> constructor</a> expects an <em>"Index or array-like"</em> as the <code>column</code> argument.</p>
<ul>
<li><code>['Coefficient']</code> is a <a href="https://docs.python.org/3/tutorial/introduction.html#lists" rel="nofollow noreferrer">Python list</a> with a single entry, the string <code>'Coefficient'</code>.</li>
<li><code>'Coefficient'</code> is not a list, but just a <a href="https://docs.python.org/3/tutorial/introduction.html#strings" rel="nofollow noreferrer">string</a>.</li>
</ul>
<p><code>DataFrame</code> accepts a Python list, but not a string, as an <em>array-like</em>.</p>
