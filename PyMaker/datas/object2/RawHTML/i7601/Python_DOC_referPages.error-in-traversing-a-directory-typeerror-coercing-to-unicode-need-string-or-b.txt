<p><code>os.walk</code> returns a tuple with values. See the documentation on <a href="https://docs.python.org/2/library/os.html#os.walk" rel="nofollow noreferrer">https://docs.python.org/2/library/os.html#os.walk</a>.</p>
<p>This should fix it:</p>
<pre><code>for root, dirs, files in os.walk("Path of a directory"):
    for file in files:
        print(os.path.getmtime(os.path.abspath(file)))
</code></pre>

<div class="post-text" itemprop="text">
<p>From the <a href="https://docs.python.org/2/library/os.html#os.walk" rel="nofollow noreferrer"><code>os.walk</code></a> docs:</p>
<blockquote>
<p>Generate the file names in a directory tree by walking the tree either
  top-down or bottom-up. For each directory in the tree rooted at
  directory top (including top itself), it yields a 3-tuple <code>(dirpath, dirnames, filenames)</code>.</p>
<p><code>dirpath</code> is a string, the path to the directory. <code>dirnames</code> is a list of
  the names of the subdirectories in <code>dirpath</code> (excluding '.' and '..').
  <code>filenames</code> is a list of the names of the non-directory files in
  <code>dirpath</code>.</p>
</blockquote>
<p>The right way to use <code>os.walk</code> is:</p>
<pre><code>for root, dirs, files in os.walk('/some/path'):
    for file in files:
        ... # do something
</code></pre>
<p>Note that <code>os.walk</code> will recursively traverse down all subdirectories in a directory. If this is not what you want, use <a href="https://docs.python.org/3/library/os.html#os.listdir" rel="nofollow noreferrer"><code>os.listdir</code></a>.</p>
</div>

<p>You're using the wrong function. <code>os.walk</code> is for walking down the hierarchy of a filesystem, and returns a tuple of <code>(dirpath, dirnames, filenames)</code>. </p>
<p>To return the list of files in a directory, use <a href="https://docs.python.org/3/library/os.html#os.listdir" rel="nofollow noreferrer"><code>os.listdir(path)</code></a>.</p>
