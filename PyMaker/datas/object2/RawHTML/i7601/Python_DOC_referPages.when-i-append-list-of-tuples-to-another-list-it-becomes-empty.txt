<div class="post-text" itemprop="text">
<p><code>del tagged_words[:]</code> empties the list, yes.</p>
<p>You have <strong>one</strong> list object that you keep filling and emptying, and you add references to another list. You are <em>not</em> creating copies here:</p>
<pre><code>tagged_sentences_list.append(tagged_words)
</code></pre>
<p>Create new list objects:</p>
<pre><code>tagged_sentences_list = []
for i in range (len(sentences)):
    length_sentences = len(sentences[i].split(" "))

    words_in_the_sentence = []
    tagged_words = []

    for j in range (length_sentences):
        length_words_in_sentence = len(sentences[i].split(" ")[j].split("/")[1:])

        part_of_the_speech = sentences[i].split(" ")[j].split("/")[1:]
        word = sentences[i].split(" ")[j].split("/")[:1]
        words_in_the_sentence.append(word)

        zipped = zip(word,part_of_the_speech)
        tagged_words.append(zipped)

    tagged_sentences_list.append(tagged_words)
</code></pre>
<p>Python names are just references; you may want to read up on how Python's memory model works, I strongly recommend <a href="https://nedbatchelder.com/text/names.html" rel="nofollow noreferrer">Ned Batchelder's <em>Facts and myths about Python names and values</em></a>.</p>
<p>Your code is doing a lot of redundant splitting too. Use the fact that Python <code>for</code> loops are <a href="https://en.wikipedia.org/wiki/Foreach_loop" rel="nofollow noreferrer"><em>for each</em> constructs</a>; there is no need to generate an index when you can just loop over the list itself:</p>
<pre><code>tagged_sentences_list = []
for sentence in sentences:
    tagged_words = []

    for word in sentence.split(' '):
        parts = word.split('/')[:2]
        tagged_words.append(parts)

tagged_sentences_list.append(tagged_words)
</code></pre>
<p>Note that there is no need to use <code>zip()</code>; all you do is re-combine the first and second element of the <code>/</code> split result.</p>
<p>If you were to use <a href="https://docs.python.org/3/tutorial/datastructures.html#list-comprehensions" rel="nofollow noreferrer">list comprehensions</a>, that could be further reduced to:</p>
<pre><code>tagged_sentences_list = [
    [word.split('/')[:2] for word in sentence.split(' ')]
    for sentence in sentences]
</code></pre>

<div class="post-text" itemprop="text">
<p>Try this: </p>
<pre><code>tagged_sentences_list.append(tagged_words[:])
</code></pre>
<p>Or...</p>
<pre><code>import copy
tagged_sentences_list.append(copy.copy(tagged_words))
</code></pre>
<p>If you're on python3, you may also try</p>
<pre><code>tagged_sentences_list.append(tagged_words.copy())
</code></pre>
<p>What your current code is doing is, appending the list to the bigger list, and then clearing it using <code>del tagged_words[:]</code>. </p>
<p>Now, since the references are the same, you end up clearing the contents of what you stored inside the bigger list too.</p>
<p>Observe:</p>
<pre><code>&gt;&gt;&gt; x = []
&gt;&gt;&gt; y = [(1, 2), (3, 4)]
&gt;&gt;&gt; x.append(y)
&gt;&gt;&gt; id(x[0])
4433923464
&gt;&gt;&gt; id(y)
4433923464
&gt;&gt;&gt; del y[:]
&gt;&gt;&gt; x
[[]]
</code></pre>
<p>You've gotten an empty list because you appended and then cleared the original. Now, this is what happens when you make a copy of the list:</p>
<pre><code>&gt;&gt;&gt; x = []
&gt;&gt;&gt; y = [(1, 2), (3, 4)]
&gt;&gt;&gt; x.append(y[:])
&gt;&gt;&gt; del y[:]
&gt;&gt;&gt; x
[[(1, 2), (3, 4)]]
</code></pre>
