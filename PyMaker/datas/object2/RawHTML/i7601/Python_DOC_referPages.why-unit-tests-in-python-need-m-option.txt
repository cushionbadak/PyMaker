<p>I'm reading Python's documentation about unit tests and just noticed the <code>-m</code> flag when <a href="https://docs.python.org/3/library/unittest.html#command-line-interface" rel="nofollow noreferrer">running test cases</a>:</p>
<p>There are <a href="https://stackoverflow.com/questions/7610001/what-is-the-purpose-of-the-m-switch">two</a> SO <a href="https://stackoverflow.com/questions/22241420/execution-of-python-code-with-m-option-or-not">questions</a> about the purpose of the <code>-m</code> flag, so I know it runs modules as scripts and also there's a <a href="https://www.python.org/dev/peps/pep-0338/" rel="nofollow noreferrer">full explanation here</a> about why it is needed.</p>
<p>My question is specific to unit tests: why do they need to run as scripts?</p>

<div class="post-text" itemprop="text">
<p>You are <em>still</em> running a module. The command is: </p>
<pre><code>python -m unittest &lt;path_to_your_script&gt;
</code></pre>
<p><code>unittest</code> <em>is</em> the module and you are passing positional arguments to <em>unittest</em>.</p>
<p>If you actually run the help against unittest:</p>
<pre><code>python -m unittest --help
</code></pre>
<p>You will get a long output explaining how to use it. Namely, this particular piece that explains exactly how to use it: </p>
<pre><code>  python -m unittest test_module               - run tests from test_module
  python -m unittest module.TestClass          - run tests from module.TestClass
  python -m unittest module.Class.test_method  - run specified test method
</code></pre>
<p>Even explained in the help, is a statement about the <em>positional</em> arguments that are passed:</p>
<pre><code>positional arguments:
  tests           a list of any number of test modules, classes and test
                  methods.
</code></pre>
