<div class="post-text" itemprop="text">
<p>If you're simply looking to input via <code>CLI</code> while debugging; you could simply use <code>step into</code> as shown below. - Enable 'Run with console' under your run configuration first.</p>
<ol>
<li>Change tab to <code>console</code> in your debugger.</li>
<li>Click on <code>step into</code> until you see the question in the console.</li>
<li>Input your answer as needed.</li>
<li>Click on <code>continue</code> or any other action from your debugger as needs be. </li>
</ol>
<p>If you'd like to debug through running a script in <code>CLI</code> you're looking for something on the lines of <code>pdb</code> (Python Debugger). You can read more <a href="https://docs.python.org/3/library/pdb.html#module-pdb" rel="nofollow noreferrer">here</a>.</p>
<p><strong>Example:</strong></p>
<p><strong>my_example.py</strong></p>
<pre><code>try:
    pdb_test = 1 / 0
except ZeroDivisionError:
    print('Argh stop it!')
</code></pre>
<p><strong>Command Line:</strong></p>
<pre><code>(venv) $ python3 -m pdb my_example.py
&gt; /my_example.py(1)&lt;module&gt;()
-&gt; try:
(Pdb) s
&gt; /my_example.py(2)&lt;module&gt;()
-&gt; a = 1 / 0
(Pdb) s
ZeroDivisionError: division by zero
&gt; /my_example.py(2)&lt;module&gt;()
-&gt; a = 1 / 0
(Pdb) 
</code></pre>
<p>What the above is showing is merely me using <code>s</code> to command the <code>pdb</code> to <code>step</code> - in the documentation you can find all the commands you might want to use including <code>continue</code> et cetera.</p>
