<p>Why does </p>
<pre><code>class multiprocessing.Pool([processes[,initializer[,initargs[,maxtasksperchild]]]]) 
</code></pre>
<p>have all these <code>]]]]</code> included?</p>
<p>I don't understand how to read this structure?</p>
</div>
<div class="post-text" itemprop="text">
<p>Usually, in documentation, [<code>something</code>] is read like '<code>something</code> is optional'. In this particular case, it also implies dependency and should be read like this:</p>
<ul>
<li><code>processes</code> is optional, but if you use it you can also use:   </li>
<li><code>initializer</code>, which is optional, but if you use it you can also use:</li>
<li><code>initargs</code>, which is optional, but... and so on</li>
</ul>
</div>
<div class="post-text" itemprop="text">
<p>"a phrase enclosed in square brackets ([ ]) means zero or one occurrences (in other words, the enclosed phrase is optional)"</p>
<p>See <a href="http://docs.python.org/2/reference/introduction.html#notation" rel="nofollow">http://docs.python.org/2/reference/introduction.html#notation</a></p>
<p><code>[processes[,initializer[,initargs[,maxtasksperchild]]]]</code> means for instance that <code>initializer</code> is optional but if you use <code>initializer</code> you must also use <code>processes</code> and so on. This is what the embedded brackets mean.</p>
<p>If you do not name the parameters you can use in any of the following examples (but no other combination!):</p>
<pre><code>Pool() 
Pool(processes) 
Pool(processes, initializer) 
Pool(processes, initializer, initargs) 
Pool(processes, initializer, initargs, maxtasksperchild) 
</code></pre>
<p>Otherwise if you do name the parameteres you can use any of them optionally. The constructor has following default values:</p>
<pre><code>Pool(processes=None, initializer=None, initargs=(), maxtasksperchild=None)
</code></pre>
<p>See the source code of the constructor (<a href="https://bitbucket.org/pypy/pypy/src/9d88b4875d6e/lib-python/2.7/multiprocessing/pool.py" rel="nofollow">https://bitbucket.org/pypy/pypy/src/9d88b4875d6e/lib-python/2.7/multiprocessing/pool.py</a>)</p>
<p>For more on keyword arguments you can read the following:
<a href="http://docs.python.org/3/tutorial/controlflow.html#keyword-arguments" rel="nofollow">http://docs.python.org/3/tutorial/controlflow.html#keyword-arguments</a></p>
</div>
<div class="post-text" itemprop="text">
<p>The use of <code>[]</code> indicates that the enclosed parameter is optional and can be omitted.</p>
