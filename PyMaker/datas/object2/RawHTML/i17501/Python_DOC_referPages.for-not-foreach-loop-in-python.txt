<div class="post-text" itemprop="text">
<p>The construct from other languages</p>
<pre><code>for (int i = start; i &lt; stop; i += step) { … }
</code></pre>
<p>translates directly into a Python for loop over a <a href="http://docs.python.org/3/library/functions.html#func-range"><code>range</code></a>:</p>
<pre><code>for i in range(start, stop, step):
    …
</code></pre>
<p>By default, <code>step</code> is 1, so you don’t need to specify it for <code>i++</code>. <code>start</code> defaults to zero, so you can also leave that out, if you start with <code>i = 0</code>:</p>
<pre><code>for i in range(stop): # the same as `for (int i = 0; i &lt; stop; i++)`
    …
</code></pre>
<p>So, if you want to iterate <code>i</code> over the length of a string, you can just do this:</p>
<pre><code>for i in range(len(my_string)):
    …
</code></pre>
<p>Usually, you would want to access the character of that string though, so you can just iterate over the characters directly:</p>
<pre><code>for char in my_string:
    …
</code></pre>
<p>Or, if you really do need the index as well, you can use <a href="http://docs.python.org/3/library/functions.html#enumerate"><code>enumerate</code></a> to get both:</p>
<pre><code>for i, char in enumerate(my_string):
    …
</code></pre>
</div>
<div class="post-text" itemprop="text">
<pre><code>for i, item in enumerate(some_string):
    # i is the index, item is the actual element
</code></pre>
<p>I think you'd be surprised how rarely you actually need <code>i</code>, though. You're better off learning how to use Python idioms, rather than forcing yourself to write PHP/C++ code in Python.</p>
</div>
