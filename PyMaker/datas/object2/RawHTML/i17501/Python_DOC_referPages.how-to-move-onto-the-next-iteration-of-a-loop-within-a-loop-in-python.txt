<div class="post-text" itemprop="text">
<p>It keep iterate through <code>dict</code>.</p>
<p>If you want the loop  go back upto first <code>for</code> loop, use <a href="http://docs.python.org/3/reference/simple_stmts.html#the-break-statement" rel="nofollow"><code>break</code> statement</a>:</p>
<pre><code>for xy in set:
    for name in dict:
        if xy == name:
           print("Yay")
           break
</code></pre>
<p>As Jacob Krall commented, You don't need to iterate the dictionary object to check whether key(<code>name</code>) exist. Use <code>in</code> operator:</p>
<pre><code>&gt;&gt;&gt; d = {'sam': 'Samuel', 'bob': 'Robert'}
&gt;&gt;&gt; 'bob' in d
True
&gt;&gt;&gt; 'tom' in d
False
</code></pre>
<hr/>
<pre><code>for xy in set:
    if name in dict:
        print("Yay")
</code></pre>
<p>If you want print <code>Yay</code> only once, use <a href="http://docs.python.org/3/library/stdtypes.html#set.isdisjoint" rel="nofollow"><code>set.isdisjoint</code></a>:</p>
<pre><code>if not set.isdisjoint(dict):
    print("Yay")
</code></pre>
<p>BTW, do not use <code>set</code> or <code>dict</code> as variable name. It shadows builtin function <a href="http://docs.python.org/3/library/functions.html#func-set" rel="nofollow"><code>set</code></a>, <a href="http://docs.python.org/3/library/functions.html#func-dict" rel="nofollow"><code>dict</code></a>.</p>
</div>

<pre><code>for xy in set:
    for name in dict:
