<p>I have a very simple <code>java</code> class that I am trying to port to <code>Python</code>, but I'm stuck on the line:</p>

<div class="post-text" itemprop="text">
<p><code>Character.getNumericValue()</code> returns <code>10</code> for <code>A</code> because it is digit in number systems with bases beyond the decimal system (such as hexadecimal, all the way up to 35-base):</p>
<pre><code>&gt;&gt;&gt; int('A', 36)
10
&gt;&gt;&gt; int('Z', 36)
35
</code></pre>
<p>From the <a href="http://docs.oracle.com/javase/6/docs/api/java/lang/Character.html#getNumericValue%28char%29" rel="nofollow"><code>Character.getNumericValue()</code> documenation</a>:</p>
<blockquote>
<p>The letters A-Z in their uppercase (<code>'\u0041'</code> through <code>'\u005A'</code>), lowercase (<code>'\u0061'</code> through <code>'\u007A'</code>), and full width variant (<code>'\uFF21'</code> through <code>'\uFF3A'</code> and <code>'\uFF41'</code> through <code>'\uFF5A'</code>) forms have numeric values from 10 through 35. This is independent of the Unicode specification, which does not assign numeric values to these <code>char</code> values.</p>
</blockquote>
<p>There is no direct Python equivalent; <code>Character.getNumericValue()</code> also interprets roman numerals, for example, you'd have to code such mappings yourself.</p>
</div>
<div class="post-text" itemprop="text">
<p>There is no single ready-made Python function that's equivalent, but I think two functions combined cover the whole range. For ASCII characters 0..9 and A..Z, <code>int()</code> with a proper base argument (36) suffices. For getting the numeric value of a unicode code point, there's <a href="http://docs.python.org/3/library/unicodedata.html#unicodedata.numeric" rel="nofollow">unicodedata.numeric</a>, discounting the fact that it returns a float.</p>
</div>
