<div class="post-text" itemprop="text">
<p>Because in Python 3 <a href="http://docs.python.org/3/library/stdtypes.html#str.translate" rel="nofollow"><code>str.maketrans()</code></a> takes a <em>dictionary</em> instead; it appears you are looking at the documentation for Python 2.</p>
<p>The dictionary should map codepoints (integers) to their replacement value (integer or unicode value), or <code>None</code> to delete codepoints.</p>
<pre><code>intab = "aeiou"
outtab = "12345"
trantab = str.maketrans(intab, outtab)

mystr.translate(trantab)
</code></pre>
<p>There is no need to import <code>string.maketrans</code> for this job; the <a href="http://docs.python.org/3/library/stdtypes.html#str.maketrans" rel="nofollow"><code>str.maketrans()</code> static function</a> is available to do the job.</p>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; intab = "aeiou"
&gt;&gt;&gt; outtab = "12345"
&gt;&gt;&gt; trantab = str.maketrans(intab, outtab)
&gt;&gt;&gt; mystr = "this is string example...wow!!!"
&gt;&gt;&gt; mystr.translate(trantab)
'th3s 3s str3ng 2x1mpl2...w4w!!!'
</code></pre>
<p>From the <a href="http://docs.python.org/3/library/stdtypes.html#str.translate" rel="nofollow"><code>str.translate()</code> documentation</a>:</p>
<blockquote>
<p>Return a copy of the s where all characters have been mapped through the <em>map</em> which must be a dictionary of Unicode ordinals (integers) to Unicode ordinals, strings or <code>None</code>. Unmapped characters are left untouched. Characters mapped to <code>None</code> are deleted.</p>
</blockquote>
<p>Incidentally, Python 2 <code>unicode.translate()</code> also works this way, while in Python 3, <a href="http://docs.python.org/3/library/stdtypes.html#bytes.translate" rel="nofollow"><code>bytes.translate()</code></a> also uses the old API.</p>
</div>
