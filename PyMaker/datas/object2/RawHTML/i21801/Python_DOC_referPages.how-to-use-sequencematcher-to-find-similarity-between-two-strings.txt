<div class="post-text" itemprop="text">
<p>From the docs:</p>
<blockquote>
<p>The <a href="https://docs.python.org/2/library/difflib.html#sequencematcher-objects" rel="noreferrer">SequenceMatcher</a> class has this constructor:</p>
<p><code>class difflib.SequenceMatcher(isjunk=None, a='', b='', autojunk=True)</code></p>
</blockquote>
<p>The problem in your code is that by doing </p>
<pre><code>seq=difflib.SequenceMatcher(a,b)
</code></pre>
<p>you are passing <code>a</code> as value for <code>isjunk</code> and <code>b</code> as value for <code>a</code>, leaving the default <code>''</code> value for <code>b</code>. This results in a ratio of <code>0.0</code>.</p>
<p>One way to overcome this (already mentioned by Lennart) is to explicitly pass <code>None</code> as extra first parameter so all the keyword arguments get assigned the correct values.</p>
<p>However I just found, and wanted to mention another solution, that doesn't touch the <code>isjunk</code> argument but uses the <code>set_seqs()</code> method to specify the different sequences.</p>
<pre><code>&gt;&gt;&gt; import difflib
&gt;&gt;&gt; a = 'abcd'
&gt;&gt;&gt; b = 'ab123'
&gt;&gt;&gt; seq = difflib.SequenceMatcher()
&gt;&gt;&gt; seq.set_seqs(a.lower(), b.lower())
&gt;&gt;&gt; d = seq.ratio()*100
&gt;&gt;&gt; print d
44.44444444444444
</code></pre>
