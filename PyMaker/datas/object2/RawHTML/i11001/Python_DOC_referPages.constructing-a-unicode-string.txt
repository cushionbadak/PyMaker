<div class="post-text" itemprop="text">
<p><code>\uhhhh</code> is an escape sequence, a <em>notation</em> used in string literals. You can't construct that notation from parts, at least not directly like that.</p>
<p>Generally, you'd use the <a href="https://docs.python.org/3/library/functions.html#chr" rel="nofollow"><code>chr()</code> function</a> to produce individual characters from an integer instead:</p>
<pre><code>&gt;&gt;&gt; chr(int('0394', 16))
'Δ'
</code></pre>
<p>for example, where I first interpreted the hex string <code>0394</code> as an integer in base 16.</p>
<p>If you <strong>must</strong> generate the Python string literal escape notation, use <code>codecs.decode()</code> with the <code>unicode_escape</code> codec:</p>
<pre><code>&gt;&gt;&gt; import codecs
&gt;&gt;&gt; r'\u' + '0394'
'\\u0394'
&gt;&gt;&gt; codecs.decode(r'\u' + '0394', 'unicode_escape')
'Δ'
</code></pre>
