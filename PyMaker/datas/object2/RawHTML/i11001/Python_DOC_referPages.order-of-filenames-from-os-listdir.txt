<div class="post-text" itemprop="text">
<p>No, the j-th position will (or at least CAN) vary. From <a href="https://docs.python.org/3/library/os.html#os.listdir" rel="nofollow">the docs</a> (emphasis mine)</p>
<blockquote>
<p><code>os.listdir(path='.')</code><br/>
  Return a list containing the names of the entries in the directory given by path. <strong>The list is in arbitrary order</strong>, and does not include the special entries '.' and '..' even if they are present in the directory.</p>
</blockquote>
<p>That said, if you want it sorted, <code>sorted</code> produces a stable lexicographically sorted list. <code>sorted(os.listdir("your/path/here"))[n]</code> should always point to the n-th file (unless your directory changes contents!)</p>

<span class="comment-copy"><code>sorted</code> will be lexically stable, but keep in mind that it's case sensitive. So <code>A.jpg</code> and <code>a.jpg</code> will be nowhere near each other, and <code>C.jpg</code> will show up before <code>b.jpg</code>. There's also issues like <code>1.jpg</code> vs. <code>02.jpg</code>.</span>
