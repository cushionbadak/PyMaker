<div class="post-text" itemprop="text">
<p>Whatever the case, by calling <a href="https://docs.python.org/3/library/stdtypes.html#str.split" rel="nofollow"><strong><code>split()</code></strong></a> on <code>new</code> you create a <code>list</code> object and not an <code>str</code>; lists do not support <a href="https://docs.python.org/3/c-api/buffer.html" rel="nofollow">the Buffer API</a>. Maybe you were looking for <a href="https://docs.python.org/3/library/stdtypes.html#str.strip" rel="nofollow"><strong><code>strip()</code></strong></a> in order to remove any trailing/leading white space? </p>
<p>Either way, the resulting <code>str</code> from <code>new.strip()</code> (or <code>split()</code> if you select an element of the resulting list) should be <em>encoded</em> since unicode objects must be <a href="https://docs.python.org/3/library/hashlib.html#hash-algorithms" rel="nofollow">encoded before feeding it to a hashing algorithms' initializer</a>.</p>
<pre><code>new = new.strip() # or new.split()[index]
hs = hashlib.md5(new.encode()).hexdigest()
</code></pre>
