<pre><code>a = ['c','a','t']
b = ['t','a','c']
m = a.sort()
n = b.sort()
print(m,n)
</code></pre>
<p><strong>Actual output:</strong></p>
<pre><code>(None, None)
</code></pre>
<p><strong>Expected:</strong></p>
<pre><code>(['a','c','t'], ['a','c','t'])
</code></pre>

<div class="post-text" itemprop="text">
<p><code>.sort()</code> returns <code>None</code> as it sorts the list in place and doesn't return the list back. But wheras sorted will.</p>
<pre><code>a = ['c','a','t']
b = ['t','a','c']
m = sorted(a)
n = sorted(b)
print(m,n)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The <code>sort()</code> method changes <code>a, b</code> themselves, and doesn't return anything. If you want to create sorted copies, do instead</p>
<pre><code>m = sorted(a)
n = sorted(b)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>sort()</code> sorts the list inplace without creating new lists</p>
<p>you can either print a and b after <code>sort()</code></p>
<p>or use <code>sorted(a)</code> and <code>sorted(b)</code> which will create new lists keeping original lists as it is</p>
</div>
<div class="post-text" itemprop="text">
<p>From the docs: The method <code>sort()</code> does not return any value but it changes from the original list. You therefore either use</p>
<pre><code>a = ['c','a','t']
b = ['t','a','c']
a.sort()
b.sort()
print(a, b)
</code></pre>
<p>or use <code>sorted()</code>, which returns a sorted list from the given iterable.</p>
<pre><code>a = ['c','a','t']
b = ['t','a','c']
m = sorted(a)
n = sorted(b)
print(m, n)
</code></pre>
</div>
