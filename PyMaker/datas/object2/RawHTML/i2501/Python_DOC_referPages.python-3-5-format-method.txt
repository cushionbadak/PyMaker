<pre><code>a = 19/3 
print('just testing {a:1.4f}'.format (a))
</code></pre>
<p>In python 3.5 this results in an error <code>KeyError: 'a'</code>.<br/>
I cannot understand why. I can use a workaround to overcome the error but I would really appreciate if somebody could explain why am I getting the error. </p>

<div class="post-text" itemprop="text">
<p>the item before the colon is an index into the arguments of the format() function. See <a href="https://docs.python.org/3.5/library/stdtypes.html#str.format" rel="nofollow noreferrer">https://docs.python.org/3.5/library/stdtypes.html#str.format</a> for more details.</p>
<p>So your example should read:</p>
<pre><code>a = 19/3
print('just testing {0:.4f}'.format (a))
</code></pre>
<p>Alternatively, if you are running python 3.6 or above, you can use f-strings:</p>
<pre><code>a = 19/3
print(f'just testing {a:.4f}')
</code></pre>
<p>See <a href="https://docs.python.org/3/reference/lexical_analysis.html#f-strings" rel="nofollow noreferrer">https://docs.python.org/3/reference/lexical_analysis.html#f-strings</a> for details</p>
</div>
<div class="post-text" itemprop="text">
<p>Dict-like notation:</p>
<pre><code>&gt;&gt;&gt; a = 4
&gt;&gt;&gt; b = 10
&gt;&gt;&gt; 'just testing {num1}, {num2}'.format(num1 = a, num2 = b)
just testing 4, 10
</code></pre>
<p>Sequencial notation</p>
<pre><code>&gt;&gt;&gt; a = 4
&gt;&gt;&gt; b = 10
&gt;&gt;&gt; 'just testing {0}, {1}'.format(a, b)
just testing 4, 10
</code></pre>
<p>Either use the dict-like notation:
<code>print('just testing {num}'.format(num=a))</code>
or the sequential one:
<code>print('just testing {0}'.format(a))</code></p>
</div>
