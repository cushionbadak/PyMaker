<p>I am trying to convert a string into date format in Python.</p>
<p>I am using following statement</p>
<pre><code>datetime_obj = datetime.datetime.strptime("Sun Aug 19 16:24:31 PDT 2018", "%a %b %d %H:%M:%S %Z %Y")
</code></pre>
<p>However, I get an error - </p>
<blockquote>
<p><em>ValueError: time data 'Sun Aug 19 16:24:31 PDT 2018' does not match format '%a %b %d %H:%M:%S %Z %Y'</em></p>
</blockquote>
<p>If I remove the timezone from the date string and the format string, the code works perfect. Which leads me to believe that the issue is related to the timezone but I am not sure what actions should be taken. 
I am in eastern timezone and the time zone in the string is in Pacific timezone. </p>
<p>Appreciate any help on this.</p>

<div class="post-text" itemprop="text">
<p>As mentioned in <a href="https://stackoverflow.com/a/8525115/5299969">this answer</a> you can use <a href="http://labix.org/python-dateutil" rel="nofollow noreferrer">python-dateutil</a> for this:</p>
<pre><code>&gt;&gt;&gt; from dateutil import parser

&gt;&gt;&gt; datetime_obj = parser.parse("Sun Aug 19 16:24:31 PDT 2018")
datetime.datetime(2018, 8, 19, 16, 24, 31)
</code></pre>
<p>Standard <a href="https://docs.python.org/3/library/datetime.html#module-datetime" rel="nofollow noreferrer">datetime module</a> behaves very strangely with parsing timezones, as I see reading <a href="https://stackoverflow.com/a/3306887/5299969">this answer</a> in question related to similar problem.</p>
