<p>I know it is possible to change the keys in a dictionary. But is it possible to change all the keys using a defined pattern? In this case I have a dictionary that contain:</p>
<pre><code>{'&gt;Apple': 'Orange', '&gt;Grape': 'Hopz'}
</code></pre>
<p><strong>Question:</strong> All key's contain a '&gt;' + string. Can I remove these '&gt;' and update? Or update without '&gt;'?</p>
<p><strong>Little background:</strong>
In my script I try to compare one dictionaries values to this examples keys which results in an error. I assume this is due to the '&gt;' because v =! k if k = '&gt;foo' and v = 'foo' </p>
<p>If any of you are scientist this '&gt;' is a result of a fasta parser. Which I could probably just change the script to not write the '&gt;' at the start of a line (I haven't tried this..yet).</p>

<div class="post-text" itemprop="text">
<p>It's easy enough with a dict comprehension:</p>
<pre><code>updateddict = {k.lstrip('&gt;'): v for k, v in yourdict.iteritems()}
</code></pre>
<p>for Python 2.7. For Python 3, use <code>yourdict.items()</code> instead.</p>
<p>For Python 2.6 and earlier, where there is no dict comprehension syntax yet, use:</p>
<pre><code>updateddict = dict((k.lstrip('&gt;'), v) for k, v in yourdict.iteritems())
</code></pre>
<p><a href="http://docs.python.org/3/library/stdtypes.html#str.lstrip" rel="nofollow"><code>str.lstrip()</code></a> is a quick and concise method to remove any <code>&gt;</code> characters from the start of a key.</p>
