<div class="post-text" itemprop="text">
<p>I'm working on a python3 project where we use the <code>typing</code> module type hints throughout.</p>
<p>It seems that we use <code>typing.Dict</code> and <code>typing.Mapping</code> pretty much interchangeably.</p>
<p>Is there a reason to prefer one over the other?</p>
</div>
<div class="post-text" itemprop="text">
<p>Managed to answer this myself.</p>
<p><code>typing.Dict</code> should be used to indicate a literal <code>dict</code> type with support for element type hinting i.e. <code>Dict[bytes, str]</code></p>
<p><code>typing.Mapping</code> is an object which defines the <code>__getitem__,__len__,__iter__</code> magic methods</p>
<p><code>typing.MutableMapping</code> is an object which defines same as Mapping but with <code>__setitem__,__delitem__</code> magic methods as well.</p>
<p>typing.Mapping et al. are based on the abc types in <a href="https://docs.python.org/3/library/collections.abc.html#collections-abstract-base-classes" rel="nofollow noreferrer">this table</a></p>
</div>
<span class="comment-copy">dicts are a kind of mapping</span>
<span class="comment-copy"><code>dict</code> is a more specific kind of mapping, it's like a sequence vs a list</span>
<span class="comment-copy">To be a <code>Mapping</code> it must define <i>all</i> of those methods somehow, but the mixin methods will be provided if you implement the abstract methods and inherit from <code>collections.abc.Mapping</code></span>
