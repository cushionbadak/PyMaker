<div class="post-text" itemprop="text">
<p>I've been experimenting with QT5 for Python, using pyqt5.  I've notices that most tutorials recommend using <strong>pyuic5</strong> to convert the XML UI to Python code.  I've also seen a couple of tutorials where instead they use <strong>uic.loadUi("myui.ui")</strong> to dynamical load the XML UI.  This seems like a cleaner, more modular solution to me, but it appears to be an unpopular option.  Is there a reason converting your code with pyuic5 is a sounder solution?  </p>
</div>
<div class="post-text" itemprop="text">
<p>Both solutions are good, they have advantages and disadvantages that have to be weighed with what you want to do, and many times it will depend on the taste of the programmer.</p>
<h1><a href="http://pyqt.sourceforge.net/Docs/PyQt5/designer.html#pyuic5" rel="nofollow noreferrer">pyuic5</a>:</h1>
<ul>
<li><p>Allows inheritance <strong>[+]</strong></p></li>
<li><p>There is no additional load when running the application <strong>[+]</strong></p></li>
<li><p>Convert the .ui to .py every time the file is modified <strong>[-]</strong></p></li>
</ul>
<h1><a href="http://pyqt.sourceforge.net/Docs/PyQt5/designer.html#PyQt5.uic.loadUi" rel="nofollow noreferrer">uic.loadUi()</a>:</h1>
<ul>
<li><p>You do not have to modify anything when modifying the .ui <strong>[+]</strong></p></li>
<li><p>Compilation extra time <strong>[+]</strong></p></li>
<li><p>Does not allow inheritance (You could implement the inheritance using <a href="http://pyqt.sourceforge.net/Docs/PyQt5/designer.html#PyQt5.uic.loadUiType" rel="nofollow noreferrer"><code>uic.loadUiType()</code></a>) <strong>[-]</strong></p></li>
<li><p>Does not allow the use of <a href="https://docs.python.org/3/library/inspect.html" rel="nofollow noreferrer">inspect</a> <strong>[-]</strong>.</p></li>
</ul>
</div>
