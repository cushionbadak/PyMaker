<div class="post-text" itemprop="text">
<p>You only created a <em>string</em>, nothing more.</p>
<p>The <a href="https://docs.python.org/3/library/tempfile.html#tempfile.gettempdir" rel="nofollow noreferrer"><code>tempfile.gettempdir()</code> function</a> returns a string:</p>
<blockquote>
<p>Return the name of the directory used for temporary files</p>
</blockquote>
<p>It's just a name. Print it if you need to know the location on your system but don't want to step through the rules outlined in the documentation; there are 3 environment variables and another 3 standard locations to test to see where the current user has permission to create files.</p>
<p>Next, <code>os.path.join()</code> takes strings and outputs a string. Nothing is created on disk to build these strings. It'll depend on whether or not you actually do something with that string (like calling <a href="https://docs.python.org/3/library/os.html#os.makedirs" rel="nofollow noreferrer"><code>os.makedirs()</code></a> or <code>open()</code>) on what will be created.</p>
<p>If you use a plain <code>open()</code> call on that string then a regular file is created in the <code>tempfile.gettempdir()</code> location that is visible to other processes.</p>
<p>If you need to create a temporary file <em>securely</em>, so in a manner that an attacker can't access the file or influence where it is created, use the <a href="https://docs.python.org/3/library/tempfile.html#tempfile.TemporaryFile" rel="nofollow noreferrer"><code>tempfile.TemporaryFile()</code> constructor</a> or the <a href="https://docs.python.org/3/library/tempfile.html#tempfile.mkstemp" rel="nofollow noreferrer"><code>tempfile.mkstemp()</code> function</a>; on UNIX systems that'll result in a file that is not listed in the directory any more (the file exists but can't be opened by other processes). It will be created in the <code>tempfile.gettempdir()</code> location however, so it'll use up disk space on the relevant disk partition that that directory lives on.</p>
