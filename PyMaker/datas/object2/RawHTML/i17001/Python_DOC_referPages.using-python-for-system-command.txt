<div class="post-text" itemprop="text">
<p>You missed a comma:</p>
<pre><code>commands_to_run  = ['python', 'wkhtmltopdf', 'a.html', 'b.pdf']
                            ^
</code></pre>
<p>Without comma <code>'python' 'wkhtmltopdf'</code> is same as <code>'pythonwkhtmltopdf'</code> (<a href="https://docs.python.org/3/reference/lexical_analysis.html#string-literal-concatenation" rel="nofollow">String literal concatenation</a>):</p>
<pre><code>&gt;&gt;&gt; 'python' 'wkhtmltopdf'
'pythonwkhtmltopdf'
</code></pre>
<p><strong>UPDATE</strong></p>
<p>Remove the space after the second command line argument.</p>
</div>
<div class="post-text" itemprop="text">
<p>Just want to share 2 ways to make the calling easier.</p>
<p>1) Call with a string of command and "shell=True" so that you don't have to split the command. For example,</p>
<pre><code>subprocess.call("python wkhtmltopdf a.html b.pdf", shell=True)
</code></pre>
<p>2) If security is a concern with "shell=True" (<a href="https://docs.python.org/2/library/subprocess.html#frequently-used-arguments" rel="nofollow">https://docs.python.org/2/library/subprocess.html#frequently-used-arguments</a>) you can try the shlex module in standard library. For example,</p>
<pre><code>import shlex
command = "python wkhtmltopdf a.html b.pdf"
subprocess.call(shlex.split(command))
</code></pre>
<p>shlex is especially helpful when your command is complicated.</p>
