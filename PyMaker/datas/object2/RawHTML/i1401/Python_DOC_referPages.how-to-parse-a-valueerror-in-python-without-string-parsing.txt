<p>The <a href="https://docs.python.org/3/library/exceptions.html#ValueError" rel="nofollow noreferrer"><code>ValueError</code></a> exception class have an <a href="https://docs.python.org/3/library/exceptions.html#BaseException.args" rel="nofollow noreferrer"><code>args</code></a> attribute which is <code>tuple</code> of arguments given to the exception constructor.</p>
<pre><code>&gt;&gt;&gt; a = ValueError({'code': -123, 'message': 'This is the error'})
&gt;&gt;&gt; a
ValueError({'code': -123, 'message': 'This is the error'})
&gt;&gt;&gt; raise a
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
ValueError: {'code': -123, 'message': 'This is the error'}
&gt;&gt;&gt; dir(a) # removed all dunder methods for readability.
['args', 'with_traceback']
&gt;&gt;&gt; a.args
({'code': -123, 'message': 'This is the error'},)
&gt;&gt;&gt; a.args[0]['code']
-123
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>ValueError is a dict type. 
So you can use
    e.get("key")
to reach any field inside dict.</p>
</div>
<div class="post-text" itemprop="text">
<p>You should get your <code>values()</code> directly from your dictionary not <code>e</code>. Try this one:</p>
<pre><code>try:
     ValueError= {'code': -123, 'message': 'This is the error'}
     Value = ValueError.get('code')
     print Value
except Exception as e:
    pass
</code></pre>
