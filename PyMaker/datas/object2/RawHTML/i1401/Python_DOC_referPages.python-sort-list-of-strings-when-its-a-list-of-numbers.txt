<p>Newbie here with the python interpreter.</p>
<p>Why am I not able to sort this list of strings?</p>
<pre><code>&gt;&gt;&gt; list = ['296098', '12805', '10445635', '11679960']
&gt;&gt;&gt; print list.sort()
None
&gt;&gt;&gt; print list.sort(key=float)
None
</code></pre>
<p>Why isn't python able to sort this list of string for me?</p>

<div class="post-text" itemprop="text">
<p><code>list.sort()</code> mutates the list in place, instead of returning a new list.</p>
<pre><code>&gt;&gt;&gt; list = ['296098', '12805', '10445635', '11679960']
&gt;&gt;&gt; list.sort()
&gt;&gt;&gt; list
['10445635', '11679960', '12805', '296098']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Firstly, <code>list.sort()</code> manipulate the list in place, which means when you call it, the list is already sorted, and the default return value is <code>None</code></p>
<pre><code>list = ['296098', '12805', '10445635', '11679960']
list.sort()
list
Out[4]: ['10445635', '11679960', '12805', '296098']
</code></pre>
<p>Secondly, above is sort by string value, you want to sort by the numerical value, you can use <code>key=int</code>. Remember you can replace int with any custom methods:</p>
<pre><code>list.sort(key=int)
list
Out[6]: ['12805', '296098', '10445635', '11679960']
</code></pre>
<p>Last but not least, you should not use <code>list</code> as a variable name, as it's builtin function name already.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>list.sort</code> sorts the <code>list</code> in place. Some Python programmers would prefer <code>sorted</code> which will return a sorted <code>list</code> and leave the initial <code>list</code> untouched</p>
<pre><code>In [2]: l=['296098', '12805', '10445635', '11679960']

In [3]: sorted(l, key=float)
Out[3]: ['12805', '296098', '10445635', '11679960']

In [4]: l.sort(key=float)

In [5]: l
Out[5]: ['12805', '296098', '10445635', '11679960']
</code></pre>
</div>
