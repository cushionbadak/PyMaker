<div class="post-text" itemprop="text">
<p>I am to write a regex which captures a string: "N letters followed by 2 or 3 digits", where N is passed as an argument of a function.</p>
<p>What I did is this: <code>"[a-zA-Z]{" + "{number}".format(number=N) + "}[0,9]{2,3}"</code> but it's not right. Where's the mistake?</p>
<p>For <code>N=3</code>, it should match: <code>"abc12", "abc123"</code> but not <code>"a1", "abc1", "abc1234"</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p>Your second block is only capturing the numbers <code>0</code> and <code>9</code>, change it to a range:</p>
<pre><code>"[a-zA-Z]{%d}[0-9]{2,3}"%N 
</code></pre>
<p>EDIT:
To <em>not</em> capture something that doesn't match, you will want to add start and end conditions to your regex. This can be done in a few ways:</p>
<h1>Text Anchors</h1>
<p>Start and end line designations if your pattern is to be by itself on a line:</p>
<pre><code>"^[a-zA-Z]{%d}[0-9]{2,3}$"%N
</code></pre>
<p>Where <code>^</code> represents a start of line and <code>$</code> represents end of line</p>
<h1>Designating <em>Not</em> matches</h1>
<p>You can say to <em>not</em> match specific characters after the pattern:</p>
<pre><code>"[a-zA-Z]{%d}[0-9]{2,3}[^0-9]+"
</code></pre>
<p>The <code>^</code> inside the brackets will say "Do not match these chars." I'm not familiar enough with lookaheads/lookbacks to give you a good answer using those, but the docs can be found <a href="https://docs.python.org/3/library/re.html" rel="nofollow noreferrer">here</a></p>
</div>
<div class="post-text" itemprop="text">
<p>You need to add anchors to your expression:</p>
<pre><code>^[a-z]{3}\d{2,3}$
</code></pre>
<p>See <a href="https://regex101.com/r/8WfcYZ/1/" rel="nofollow noreferrer"><strong>a demo on regex101.com</strong></a>.
<hr/>
In general:</p>
<pre><code>^[a-z]{n}\d{2,3}$
#     ^^^
</code></pre>
