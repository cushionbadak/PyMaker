<p>After studying the docs I cannot find the answer to this.
I have a directory structure:</p>
<pre><code>views/
  __init__py
  view1.py
  view2.py
</code></pre>
<p>In view1 and view2 are my view-functions. What do I have to do to be able to import all view functions from view1.py and view2.py with one single import:</p>
<pre><code>import views
</code></pre>
</div>
<div class="post-text" itemprop="text">
<pre><code>from views import view1, view2
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Summarizing all comments and verifying below is the answer.</p>
<ul>
<li><p>If <code>__init__.py</code> is declared with below line</p>
<p><code>__all__ = ["view1", "view2"]</code> then</p>
<p><strong>Solution</strong> : <code>from views import *</code></p></li>
<li><p>Else If <code>__init__.py</code> is not defined with <code>__all__</code> line or missing view1 , view2 in the list then   </p>
<p><strong>Solution</strong> : <code>from views import view1, view2</code></p></li>
</ul>
<p>In the code file where you do the import do access the functions as below</p>
<pre><code>view1.view1()
view2.view2()
</code></pre>
<blockquote>
<p>Reference : <a href="https://docs.python.org/3/tutorial/modules.html#packages" rel="nofollow noreferrer">modules-packages documentation</a></p>
</blockquote>
</div>

<span class="comment-copy">@caliph: no, they are not. <code>import views</code> binds the name <code>views</code> to the module object. <code>from views import *</code> takes all attributes of the <code>views</code> module and creates separate names for them in the current namespace.</span>
<span class="comment-copy">@caliph: See the <a href="https://docs.python.org/3/tutorial/modules.html#packages" rel="nofollow noreferrer">Python tutorial on importing</a>.</span>
