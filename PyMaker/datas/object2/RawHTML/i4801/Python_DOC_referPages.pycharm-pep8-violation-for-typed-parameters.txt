<p>PEP8 suggests no spaces around equal operators in function parameters. </p>
<p>For example: </p>
<p>Correct:</p>
<pre><code>def func(a=0):
   print('PEP8 compliant spacing')
</code></pre>
<p>Incorrect:</p>
<pre><code>def func(a = 0):
   print('Invalid PEP8 spacing')
</code></pre>
<p>PyCharm's auto-formatter fails to pick up incorrect spacing when <a href="https://docs.python.org/3/library/typing.html" rel="nofollow noreferrer">typing</a> is included. </p>
<p>For example, PyCharm <strong>does not</strong> correctly format the following function:</p>
<pre><code>def func(a: int = 0):
    print('Invalid PEP8 spacing')
</code></pre>
<p>To:</p>
<pre><code>def func(a: int=0):
    print('PEP8 compliant spacing')
</code></pre>
<p>Has anyone found a way to have PyCharm's auto-formatter pick up spacing violations where typing is present?</p>

<p>You are mistaken in your quoting of <a href="https://www.python.org/dev/peps/pep-0008/#function-annotations" rel="nofollow noreferrer">PEP8</a>. The whitespace is supposed to be there in this case:</p>
<blockquote>
<p>When combining an argument annotation with a default value, use spaces around the = sign (but only for those arguments that have both an annotation and a default).</p>
<p><strong>Yes</strong>:</p>
<p><code>def munge(sep: AnyStr = None): ...</code> </p>
<p><strong>No</strong>:  </p>
<p><code>def munge(input: AnyStr=None): ...</code><br/>
<code>def munge(input: AnyStr, limit = 1000): ...</code></p>
</blockquote>
