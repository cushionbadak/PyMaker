<div class="post-text" itemprop="text">
<p>Trying to encode the text output received after parsing HTML data through BeautifulSoup library in Python 3. Getting following error:</p>
<blockquote>
<p>----&gt; gmtext.encode('ascii', errors='replace').replace("?", "")</p>
<p>TypeError: a bytes-like object is required, not 'str'</p>
</blockquote>
<p>Here is the code implementation:</p>
<pre><code>import urllib.request as urllib2
from bs4 import BeautifulSoup

articleURL = "http://digimon.wikia.com/wiki/Guilmon"

page = urllib2.urlopen(articleURL).read().decode('utf8', 'ignore')
soup = BeautifulSoup(page, 'lxml')
gmtext = soup.find('p').text

gmtext.encode('ascii', errors='replace').replace("?", "")
</code></pre>
<p>So far, all answers I found regarding this error have been about some sort of file open error.</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/stdtypes.html?highlight=encode#str.replace" rel="nofollow noreferrer"><code>.replace()</code></a> is a string function, but you're calling it after calling <a href="https://docs.python.org/3/library/stdtypes.html?highlight=encode#str.encode" rel="nofollow noreferrer"><code>.encode()</code></a>,<br/>
which returns <a href="https://docs.python.org/3/glossary.html#term-bytes-like-object" rel="nofollow noreferrer">"a bytes-like object"</a> that you can't call <code>.replace()</code> on.</p>
<p>If you want to, you can do replacement before encoding like so:</p>
<pre><code>gmtext.replace("?", "").encode('ascii', errors='replace')
</code></pre>
<p>Then it'll work.</p>
</div>
<div class="post-text" itemprop="text">
<p>you can do replace with bytes (using <code>b</code> before the string) like:</p>
<pre><code>gmtext.encode('ascii', errors='replace').replace(b"?", b"")
</code></pre>
</div>
