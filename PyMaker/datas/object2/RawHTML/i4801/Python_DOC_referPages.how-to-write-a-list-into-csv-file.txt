<p>I tried the below code to write a list to csv file, but it always shows the error:</p>
<pre><code>a bytes-like object is required, not 'str'
</code></pre>
<p>Below is the code:</p>
<pre><code>import csv
a=['a','b','c']
with open(r"result.csv",'wb') as resultFile:
    writer = csv.writer(resultFile, lineterminator='\n')
    for i in a:
        writer.writerows([[i]])
</code></pre>

<div class="post-text" itemprop="text">
<p>you <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow noreferrer"><code>open</code></a> your file in 'binary' mode instead of 'text' mode:</p>
<pre><code>with open(r"result.csv",'w') as resultFile:  # instead of 'wb'
</code></pre>
<p>should work.</p>
</div>
<div class="post-text" itemprop="text">
<p>Like @hiro protagonist said, you are opening the file in binary mode. <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow noreferrer">Here</a> is some helpful documentation.</p>
<blockquote>
<p>Python distinguishes between binary and text I/O. Files opened in binary mode (including 'b' in the mode argument) return contents as <a href="https://docs.python.org/3/library/stdtypes.html#bytes" rel="nofollow noreferrer">bytes</a> objects without any decoding. In text mode (the default, or when 't' is included in the mode argument), the contents of the file are returned as <a href="https://docs.python.org/3/library/stdtypes.html#str" rel="nofollow noreferrer">str</a>, the bytes having been first decoded using a platform-dependent encoding or using the specified encoding if given.</p>
</blockquote>
<p>You could also use pandas <a href="http://pandas-docs.github.io/pandas-docs-travis/generated/pandas.DataFrame.to_csv.html" rel="nofollow noreferrer">to_csv</a>:</p>
<pre><code>import pandas as pd

a = ['a', 'b', 'c']
df = pd.DataFrame(a)
df.to_csv('result.csv', header=None, index=False)
</code></pre>
