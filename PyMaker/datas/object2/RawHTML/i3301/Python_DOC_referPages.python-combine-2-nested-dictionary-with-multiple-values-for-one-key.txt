<div class="post-text" itemprop="text">
<p>Dictionary keys are unique. You can't have two values attached to one key.</p>
<p>However, you can construct a nested dictionary with sub-dictionaries mapping keys to lists of values. For this, you can use <a href="https://docs.python.org/3/library/collections.html#collections.defaultdict" rel="nofollow noreferrer"><code>collections.defaultdict</code></a>:</p>
<pre><code>d1 = {'Fun': {1: 'hi', 2: 'hello'}, 'fun2': {3: 'bye', 4: 'good bye'}}
d2 = {'Fun': {1: 'abc', 2: 'xyZ'}, 'fun2': {3: 'qpr', 4: 'jkl'}}

from collections import defaultdict

dd = defaultdict(lambda: defaultdict(list))

for top_dict in (d1, d2):
    for k1, v1 in top_dict.items():
        for k2, v2 in v1.items():
            dd[k1][k2].append(v2)

print(dd)

defaultdict({'Fun': defaultdict(list,  {1: ['hi', 'abc'], 2: ['hello', 'xyZ']}),
             'fun2': defaultdict(list, {3: ['bye', 'qpr'], 4: ['good bye', 'jkl']})})
</code></pre>

<span class="comment-copy">In python, dictionary keys are unique. It means you can only one 'Fun' in the same dictionary.</span>
<span class="comment-copy">Such a thing is not present in the standard library. You can use a defaultdict( <a href="http://docs.python.org/3.1/library/collections.html#collections.defaultdict" rel="nofollow noreferrer">docs.python.org/3.1/library/…</a>) though.  Example code:- &gt;&gt;&gt; from collections import defaultdict &gt;&gt;&gt; md = defaultdict(list) &gt;&gt;&gt; md[1].append('a') &gt;&gt;&gt; md[1].append('b') &gt;&gt;&gt; md[2].append('c') &gt;&gt;&gt; md[1] // output will be ['a', 'b'] &gt;&gt;&gt; md[2] // output will be ['c']</span>
