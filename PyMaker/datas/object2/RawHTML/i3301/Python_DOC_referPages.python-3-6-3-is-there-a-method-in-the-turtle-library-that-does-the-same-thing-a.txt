<span class="comment-copy">It's best to avoid <code>time.sleep</code> in GUI code, since it puts <i>everything</i> to sleep. Instead use <a href="https://docs.python.org/3/library/turtle.html#turtle.delay" rel="nofollow noreferrer"><code>turtle.delay</code></a>. And yes, it's good to avoid recursion; you should be able to use a simple <code>while</code> loop here instead. FWIW, the turtle module runs on top of Tkinter: turtle graphics are rendered on a Tkinter Canvas, and it's possible to use Tkinter methods in turtle via <a href="https://docs.python.org/3/library/turtle.html#turtle.getcanvas" rel="nofollow noreferrer"><code>turtle.getcanvas()</code></a> if you <i>really</i> need them.</span>
<span class="comment-copy">Sounds good to me: this is turtle's equivalent to the Tkinter <code>.after</code> method.</span>
