<p>There is the built-in method unicodedata.name which given a unicode character will return a human readable name, e.g.:</p>
<pre><code>unicodedata.name(chr(0x2704)) == "WHITE SCISSORS"
</code></pre>
<p>Is there anything out there that will provide the inverse? I'm looking for something like:</p>
<pre><code>unicodedata.name("WHITE SCISSORS") == chr(0x2704) or 0x2704
</code></pre>
<p>I could easily loop over all the possible values, build up a map, but that seems like it could be inefficient and was hoping something like this already exists. I am using Python 3 so am open to 3-only solutions.</p>

<div class="post-text" itemprop="text">
<p>You're looking for <a href="https://docs.python.org/3/library/unicodedata.html#unicodedata.lookup" rel="nofollow"><code>unicodedata.lookup</code></a>:</p>
<pre><code>In [5]: unicodedata.lookup("WHITE SCISSORS")
Out[5]: '‚úÑ'
</code></pre>
<p>This returns a character, use <code>ord</code> to get integer ordinal:</p>
<pre><code>In [7]: ord(unicodedata.lookup("WHITE SCISSORS"))
Out[7]: 9988 # hex(9988) is 0x2704
</code></pre>
