<div class="post-text" itemprop="text">
<blockquote>
<p>AttributeError: 'str' object has no attribute 'read'</p>
</blockquote>
<p>As you can see it tries to call <code>read()</code> on the passed object, but you've passed a string object not a file (or any other object that implements <code>read()</code>)</p>
<p><code>json.load()</code> wants a file object, not a string of its path, so use <code>open()</code> to open the file then pass it, or something like code below:</p>
<pre><code>with open('file.json') as json_file:    
    json_data = json.load(json_file)
</code></pre>
<p>If you're opening the file to be written, you should the <code>w</code> mode to <code>open()</code> like <code>open('file.json', 'w')</code></p>
<p>From the <a href="https://docs.python.org/3/library/json.html#json.load" rel="nofollow">docs</a></p>
<blockquote>
<p><strong>json.load(fp, ...)</strong></p>
<p>Deserialize fp (a .read()-supporting <strong>file-like object</strong> containing a
  JSON document) to a Python object using this conversion table.</p>
</blockquote>
<p>And you have same issue with <code>json.dump</code></p>
<blockquote>
<p>AttributeError: 'str' object has no attribute 'write'</p>
<p><strong>json.dump(obj, ...)</strong> Serialize obj as a JSON formatted stream to fp (a
  .write()-supporting <strong>file-like object</strong>) using this conversion table.</p>
</blockquote>
<h2>Example</h2>
<pre><code>import json
match_histories = {}
match_histories["age"] = 25

print 'Python dict is', match_histories

with open('match.json', 'w') as json_file:
    json.dump(match_histories, json_file)

with open('match.json', 'r') as json_file:
    match_histories = json.load(json_file)

print 'Python dict restored from JSON', match_histories
</code></pre>
<h2>Run it</h2>
<pre><code>python jsontest.py

Python dict is {'age': 25}
Python dict restored from JSON {u'age': 25}
</code></pre>
