<div class="post-text" itemprop="text">
<p>I created a dictionary for peoples' names as such:</p>
<pre><code>names = {"first": , "second": , "third":}
</code></pre>
<p>What I want to do is call each name and combine them as a string, being separated by a plus sign. Below is a tedious way of doing it:</p>
<pre><code>str(names["first"]) + "+" + str(names["second"]) + "+" + str(names["third"])
</code></pre>
<p>This line is from a larger module I am attempting to create. I tried to following for simplicity, but it resulted in a syntax error:</p>
<pre><code>str(names["first"] "%+d" + names["second"] "%+d" + names["third"])
</code></pre>
<p>Is there a way of simplifying the above so that it prints:</p>
<blockquote>
<p>firstname+secondname+thirdname</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>In python 2 you have to use the slightly hacky</p>
<pre><code>"{first}+{second}+{third}".format(**names)
</code></pre>
<p>However, as GingerPlusPlus points out, in python 3 you can get away from this <code>**kwargs</code> chicanery and instead use <a href="https://docs.python.org/3/library/stdtypes.html#str.format_map" rel="nofollow"><code>format_map</code></a></p>
<pre><code>"{first}+{second}+{third}".format_map(names)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If you can use <code>namedtuple</code> instead of <code>dict</code> to store names, I'd suggest:</p>
<pre><code>'+'.join(map(str, names))
</code></pre>
