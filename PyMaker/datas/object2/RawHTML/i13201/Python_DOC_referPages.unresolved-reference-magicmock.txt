<p>I want to Mock a method in a Python Test. 
In order to do this I need MagicMock which I try to import it like this:</p>
<pre><code>from unittest import TestCase
from unittest.mock import MagicMock
</code></pre>
<p>Even after I clicked several times on install in PyCharm and although I see magicmock 0.0.3 under Preferences/Project Interpreter, I get this error:</p>
<pre><code> from unittest.mock import MagicMock
 ImportError: No module named mock
</code></pre>
<p>The import statement I saw <a href="https://docs.python.org/3/library/unittest.mock.html" rel="nofollow">here</a>. <b>What do I wrong?</b></p>
</div>
<div class="post-text" itemprop="text">
<p>In <code>Py3.5</code> (basically versions <code>&gt;= 3.3</code>) <a href="http://www.voidspace.org.uk/python/mock/" rel="nofollow"><code>mock</code></a> has been incorporated in the Standard Library as you linked in the <a href="https://docs.python.org/3/library/unittest.mock.html" rel="nofollow">documentation</a>. So, the statement:</p>
<pre><code>from unittest.mock import MagicMock
</code></pre>
<p>should <strong>never</strong> raise <code>ImportErrors</code>. </p>
<hr/>
<p>I'm guessing you're either using a <code>Py3</code> version <code>&lt; 3.3</code> or are using <code>Py2.7</code>. </p>
<p>In these cases you should <code>pip install mock</code> first and then import:</p>
<pre><code>from mock import MagicMock
</code></pre>
</div>
