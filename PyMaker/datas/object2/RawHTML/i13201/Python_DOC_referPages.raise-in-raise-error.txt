<div class="post-text" itemprop="text">
<p>If you comment the <code>raise</code> statement there, it doesn't mean that you <strong>don't</strong> have an Error; it simply means that you <strong><em>handled</em></strong> the <code>Exception</code> -- which in your case is from what I can tell an <code>IndexError</code> -- by catching it with the <code>except Exception</code> and then calling <code>PrintException()</code>. </p>
<p>When you <a href="https://docs.python.org/3/tutorial/errors.html#raising-exceptions" rel="nofollow"><code>raise</code></a> an exception what you actually do is: </p>
<blockquote>
<p>The raise statement allows the programmer to force a specified exception to occur.</p>
</blockquote>
<p>So, by un-commenting, you allow the <code>IndexError</code> named <code>eb</code> to re-appear after catching it in the inner <code>try-except</code> block and get caught by the outer <code>try - except</code> clause in which you again re-raise it.</p>
<hr/>
<p>Generally, you don't want to catch exceptions in such a generic way because it might hide some unpredicted behaviour of the program that you would like to know about. </p>
<p>Limit the exceptions you catch in the except clause by simply specifying them, in your case, an except clause of the form:</p>
<pre><code>except IndexError as eb:
    PrintException() 
</code></pre>
<p>would probably suffice.  </p>
