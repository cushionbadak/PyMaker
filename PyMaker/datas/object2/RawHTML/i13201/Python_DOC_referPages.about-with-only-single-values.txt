<div class="post-text" itemprop="text">
<p><code>{var1, var2, var3, ... }</code> is the syntax for creating a Set. A Set is a collection that has no repeated elements and has no indices. </p>
<p>For future reference, you can use the <code>type()</code> method to find the type of any object, like this:</p>
<pre><code>&gt;&gt;&gt; type({'testing'})
&lt;type 'set'&gt;
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You make a set, there is the document:
<a href="https://docs.python.org/2/library/sets.html" rel="nofollow">https://docs.python.org/2/library/sets.html</a></p>
</div>
<div class="post-text" itemprop="text">
<p>When you use <code>{'word1', 'word2', 'word3'}</code> to define your value, actually it equals <code>set(['word1', 'word2', 'word3'])</code>.</p>
<p>You can try to check them:</p>
<pre><code>&gt;&gt;&gt; set(['word1', 'word2', 'word3'])
set(['word1', 'word3', 'word2'])

&gt;&gt;&gt; {'word1', 'word2', 'word3'}
set(['word1', 'word3', 'word2'])
</code></pre>
<p>So, they are same.</p>
<p>If you want to make a dictionary object, the format is necessary to be combined of <code>key</code> and value as a pair:</p>
<pre><code>&gt;&gt;&gt; dict_ = {'word1': 1, 'word2': 2, 'word3': 3}
&gt;&gt;&gt; print dict
{'word1': 1, 'word3': 3, 'word2': 2}
</code></pre>
<p>Although it's similar for brackets between <code>dictionary</code> and <code>set</code> , as Will Sherwood said, you can use <code>type</code> to check type of the object.</p>
