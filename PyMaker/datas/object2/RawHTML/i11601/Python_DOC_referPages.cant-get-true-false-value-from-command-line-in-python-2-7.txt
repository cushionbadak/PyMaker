<div class="post-text" itemprop="text">
<p>You are missing the parsing step</p>
<pre><code>args = parser.parse_args()
</code></pre>
<p><code>args = argparse.Namespace()</code> just creates an new empty <code>Namespace</code> object.</p>
<p><code>argparse</code> is the module.  <code>parser</code> is the <code>ArgumentParser</code> object.  <code>Namespace</code> is a class defined in that module.  <code>parse_args</code> creates a <code>Namespace</code>, fills it with values that that it parses from <code>sys.argv</code>, and returns it as <code>args</code>.</p>
<p>Defining the <code>parser</code> by itself does not do any parsing.</p>
<hr/>
<p>The very first example in the docs is:</p>
<pre><code>import argparse

parser = argparse.ArgumentParser(description='Process some integers.')
parser.add_argument('integers', metavar='N', type=int, nargs='+',
                   help='an integer for the accumulator')
parser.add_argument('--sum', dest='accumulate', action='store_const',
                   const=sum, default=max,
                   help='sum the integers (default: find the max)')

args = parser.parse_args()
print(args.accumulate(args.integers))
</code></pre>
<p>Some claim this is too advanced for beginners, but the key pieces are there.</p>
<pre><code>parser = ...
parser.add_argument...
args = parser.parse_args()
# look at args, print it, access attributes, etc.
</code></pre>
<p><code>argparse.Namespace</code> isn't mentioned until section <a href="https://docs.python.org/3/library/argparse.html#the-namespace-object" rel="nofollow">https://docs.python.org/3/library/argparse.html#the-namespace-object</a></p>
