<p>While trying to sum across timedeltas in pandas, it seems to work for a slice but not the whole column. </p>
<pre><code>&gt;&gt; d.ix[0:100, 'VOID-DAYS'].sum()
Timedelta('2113 days 00:00:00')

&gt;&gt; d['VOID-DAYS'].sum()


ValueError: overflow in timedelta operation
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If <code>VOID-DAYS</code> represents an integer number of days, convert the Timedeltas into integers:</p>
<pre><code>df['VOID-DAYS'] = df['VOID-DAYS'].dt.days
</code></pre>
<hr/>
<pre><code>import numpy as np
import pandas as pd
df = pd.DataFrame({'VOID-DAYS': pd.to_timedelta(np.ones((106752,)), unit='D')})
try:
    print(df['VOID-DAYS'].sum())
except ValueError as err:
    print(err)
    # overflow in timedelta operation


df['VOID-DAYS'] = df['VOID-DAYS'].dt.days
print(df['VOID-DAYS'].sum())
# 106752
</code></pre>
<hr/>
<p>If the Timedeltas include seconds or smaller units, then use </p>
<pre><code>df['VOID-DAYS'] = df['VOID-DAYS'].dt.total_seconds()
</code></pre>
<p>to convert the value to a float.</p>
<hr/>
<p>Pandas Timedeltas (Series and TimedeltaIndexes) store all timedeltas as ints compatible with NumPy's <code>timedelta64[ns]</code> dtype. This dtype uses 8-byte ints to store the timedelta in nanoseconds. </p>
<p>The largest number of days representable in this format is </p>
<pre><code>In [73]: int(float(np.iinfo(np.int64).max) / (10**9 * 3600 * 24))
Out[73]: 106751
</code></pre>
<p>Which is why </p>
<pre><code>In [74]: pd.Series(pd.to_timedelta(np.ones((106752,)), unit='D')).sum()
ValueError: overflow in timedelta operation
</code></pre>
<p>raises a <code>ValueError</code>, but</p>
<pre><code>In [75]: pd.Series(pd.to_timedelta(np.ones((106751,)), unit='D')).sum()
Out[75]: Timedelta('106751 days 00:00:00')
</code></pre>
<p>does not.</p>
