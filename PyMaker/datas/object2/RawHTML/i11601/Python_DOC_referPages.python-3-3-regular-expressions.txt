<p>I've been studying Python for GCSE but can't get my code to work like it should (It will accept 4 digit+ numbers even though it shouldn't) even though I've verified that the coding is fine through teachers etc.</p>
<pre><code>import re
val = input("Please enter a three-digit number: ")
valid = re.match("[0-9]{3}",val)
if valid:
       print("Accepted. ")
else:
       print("Rejected. Invalid input. ")
</code></pre>
<p>I really don't know why this isn't working right. Anyone got any ideas??</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/re.html#re.match" rel="nofollow"><code>re.match</code></a> matches as long as the pattern matches at the beginning of the given string. You need to use <code>$</code> to ensure the string also ends with the string:</p>
<pre><code>valid = re.match("[0-9]{3}$",val)
</code></pre>
<hr/>
<pre><code>&gt;&gt;&gt; re.match('[0-9]{3}', '123')
&lt;_sre.SRE_Match object at 0x7effd9905f38&gt;
&gt;&gt;&gt; re.match('[0-9]{3}', '1234')
&lt;_sre.SRE_Match object at 0x7effd9852030&gt;


&gt;&gt;&gt; re.match('[0-9]{3}$', '123')
&lt;_sre.SRE_Match object at 0x7effd9905f38&gt;
&gt;&gt;&gt; re.match('[0-9]{3}$', '1234')
&gt;&gt;&gt; 
</code></pre>
</div>
<span class="comment-copy"><code>re.match("[0-9]{3}$",val)</code> (or <code>re.match(r"[0-9]{3}\z",val)</code>)</span>
<span class="comment-copy">Thanks this helped very much, the program works seamlessly now!</span>
