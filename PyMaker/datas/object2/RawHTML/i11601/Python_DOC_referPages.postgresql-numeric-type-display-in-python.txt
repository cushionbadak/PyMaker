<p>I have a postgresql table with column defined as: <code>numeric(20,7)</code>
If I insert the value <code>0</code> it saves it as: <code>0.0000000</code></p>
<p>When I assign this value to Python (2.7) it shows me <code>0E-7</code> which is correct but a wrong format.</p>
<p>Why does it shows it like this? and how can I fix it?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can format a <a href="https://docs.python.org/3/library/decimal.html" rel="nofollow"><code>Decimal</code></a> simply by passing it to:</p>
<pre><code>"{:f}".format(Decimal("0E-7"))
</code></pre>
<p>The <code>Decimal</code> supports <a href="https://www.python.org/dev/peps/pep-3101/" rel="nofollow">advanced string formatting</a>:</p>
<pre><code># PEP 3101 support.  the _localeconv keyword argument should be
# considered private: it's provided for ease of testing only.
def __format__(self, specifier, context=None, _localeconv=None):
    """Format a Decimal instance according to the given specifier.

    The specifier should be a standard format specifier, with the
    form described in PEP 3101.  Formatting types 'e', 'E', 'f',
    'F', 'g', 'G', 'n' and '%' are supported.  If the formatting
    type is omitted it defaults to 'g' or 'G', depending on the
    value of context.capitals.
    """
</code></pre>
<p>By default it seems to use the precision of the <code>Decimal</code> itself:</p>
<pre><code>&gt;&gt;&gt; '{:f}'.format(Decimal('0E-7'))
'0.0000000'
&gt;&gt;&gt; '{:f}'.format(Decimal('0E-8'))
'0.00000000'
&gt;&gt;&gt; '{:f}'.format(Decimal('0E-9'))
'0.000000000'
</code></pre>
<p>If on the other hand you want to use a custom precision, pass it in the format string:</p>
<pre><code>&gt;&gt;&gt; print("{:.2f}".format(Decimal("0E-7")))
0.00
</code></pre>
<p>For an explanation on what the different letters in the format mean, consult the <a href="https://docs.python.org/3/library/string.html#format-specification-mini-language" rel="nofollow">"Format Specification Mini-Language"</a>.</p>
</div>
<div class="post-text" itemprop="text">
<p>If you want to control the way a float number is formatted, you could try:</p>
<pre><code>"{0:.2f}".format(n) # n is value read from db
</code></pre>
