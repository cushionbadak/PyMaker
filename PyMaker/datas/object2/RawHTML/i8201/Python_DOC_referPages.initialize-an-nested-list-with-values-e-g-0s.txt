<p>Mutating any row value mutates all row values for the shortcut methods of initialization. <br/>
The only way I can get non-mutating behavior is by writing out the whole list.
<br/>
There must be a better way(?), think of the case where you want to initialize the list based on the length of a different list.</p>

<div class="post-text" itemprop="text">
<p>Use a list comprehension. </p>
<pre><code>my_list = [ [0]*4 for _ in range(4)]
</code></pre>
<p>Note that lists are mutable but integers are not. Thus [0]*4 behaves like you would expect, but [[0]*4]*4 'leaks' changes. </p>
<p>Using a list comprehension creates a new outer list each time, avoiding the problematic behavior of your example.</p>
