<p>Suppose I have a set and I do while set: like the following, when does the loop stop?</p>
<pre><code>x={'3', '4', '2'}
while x:
    print(1)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>while</code> over <em>any</em> non-empty variable will never stop if you never modify that variable within the while body.</p>
</div>
<div class="post-text" itemprop="text">
<p>Never.</p>
<p>Because <code>while</code> evaluates the truthiness of x, which is True - and so you have an infinite loop.</p>
<p>You can try it yourself with <a href="https://docs.python.org/3/library/functions.html#bool" rel="nofollow noreferrer"><code>bool(x)</code></a> or have a look at the docs: <a href="https://docs.python.org/3/library/stdtypes.html#truth" rel="nofollow noreferrer">https://docs.python.org/3/library/stdtypes.html#truth</a></p>
</div>
<div class="post-text" itemprop="text">
<p>to understand what your code is doing you need to understand how a while statement works.</p>
<p>The <code>while</code> statement is a Boolean statement.</p>
<p>What this means is <code>(True,False)</code></p>
<p>for example:</p>
<pre><code>x = True
while x: # without saying true or false here "while" assumes you are checking for true.
    print ("X is True")
    x = False
</code></pre>
<p>The above code will run the commands inside of it until <code>x == False</code>.
So the code will run one time because I have said <code>x = False</code> at the end of the statement.</p>
<p>the above code is the same as saying.</p>
<pre><code>x = True
while x is True:
    print ("X is True")
    x = False
</code></pre>
<p>In your example:</p>
<pre><code>x={'3', '4', '2'}
while x:
    print(1)
</code></pre>
<p><code>x</code> will always be <code>True</code> if you do not change it to be False or something that is equal to False.
Therefore your loop will never end "MUAHAHAHAHAHAHHA"</p>
<p>For example:</p>
<pre><code>x = True
while x:
    print (1)
    x = 0
</code></pre>
<p>Or:</p>
<pre><code>x = 12
while x:
    print (1)
    x = 0
</code></pre>
<p>This will print out the number <code>1</code> only one time because x has been assigned <code>0</code> and is the equivalent of False for the while statement.</p>
