<p>Humor is usually not welcome on stackoverflow, but there are exceptions from this rule so let's this answer be such exception. </p>
<p>But let's go back to answering the actual question: </p>
<blockquote>
<p><code>Can't determine cause of SyntaxError: invalid character in identifier</code></p>
<p><code>I am not sure what is triggering this error.</code></p>
</blockquote>
<p>I have to admit that I write this fully correct answer with a chuckle :) :</p>
<blockquote>
<p><strong>The cause of the SyntaxError is an invalid character in identifier.</strong></p>
</blockquote>
<p>The invalid character in the identifier is the character: </p>
<p>[ <strong>’</strong> ]</p>
<p>U+2019  ’   e2 80 99    RIGHT SINGLE QUOTATION MARK</p>
<p>Here Python code with which you can find out for yourself details about the "character" which is causing the trouble: </p>
<pre><code>#!/usr/bin/ python3
# -*- coding: &lt;utf8&gt; -*-
# tags=[’coffeepi’]
coffeepi = "’"
print(ord(coffeepi), hex(ord(coffeepi)), bin(ord(coffeepi))) 
</code></pre>
<p>what prints: </p>
<blockquote>
<p>8217, <strong>0x2019</strong> , 0b10000000011001</p>
</blockquote>
<p>Check out this here </p>
<p><a href="http://www.utf8-chartable.de/unicode-utf8-table.pl" rel="nofollow noreferrer">http://www.utf8-chartable.de/unicode-utf8-table.pl</a>
where I have got the information about the character from, and this here 
<a href="https://docs.python.org/3/reference/lexical_analysis.html#identifiers" rel="nofollow noreferrer">https://docs.python.org/3/reference/lexical_analysis.html#identifiers</a>
for more detailed information about which characters are allowed in an indentifier, because not all are and apparently this one does not belong to the range of allowed characters. </p>
<p>By the way: this "character" is NOT a one byte character like it is for ASCII characters. How this character is encoded in UTF-8 is specified in the Unicode table (e2 80 99). </p>
<p>AND ... in the entire code you have provided in the question there is no trace of what you have specified to cause the error message. </p>
<p>How you can fix that?</p>
<blockquote>
<p><strong>Just replace these strange quotes with standard quotes " "</strong></p>
</blockquote>
<p>ADDENDUM: The strange quotes should be probably standard quotes around a string with a word put into the list. As the Python interpreter is running into this strange quotes it assumes they are part of a variable name (identifier) and not part of a string specification. That is the reason for the confusing error message as you don't see from the code at the first glance WHY the Python interpreter thinks it deals with an variable name (identifier) and not a quoted section of text which should be turned into a Python string. </p>
