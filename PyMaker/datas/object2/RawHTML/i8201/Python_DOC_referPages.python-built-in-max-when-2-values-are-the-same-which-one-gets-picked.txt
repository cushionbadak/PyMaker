<p>I have a code that processes data based on some dates. 
Lets say:</p>
<pre><code>case1:

values1 with date1 = '2002-02-01'
values2 with date2 = '2004-02-01'

case2:

values1 with date1 ='2001-01-01'
values2 with date2 ='2001-01-01'
</code></pre>
<p>I need to get the most recent record. Everything works fine when my values have different dates, but when records have the same dates <code>max(date1, date2)</code>.
<strong>Question. Which max value is returned when the values are equal, like in case 2?</strong> </p>
</div>
<div class="post-text" itemprop="text">
<p>For multiple values that are all the maximum, the first such value is returned:</p>
<pre><code>&gt;&gt;&gt; class Equal:
...     def __init__(self, id):
...         self.id = id
...     def __repr__(self):
...         return f"Equal({self.id!r})"
...     def __gt__(self, other):
...         return False
...
&gt;&gt;&gt; max([Equal(1), Equal(2), Equal(3)])
Equal(1)
</code></pre>
<p>This is explicitly <a href="https://docs.python.org/3/library/functions.html#max" rel="nofollow noreferrer">documented</a>:</p>
<blockquote>
<p>If multiple items are maximal, the function returns the first one encountered.</p>
</blockquote>
