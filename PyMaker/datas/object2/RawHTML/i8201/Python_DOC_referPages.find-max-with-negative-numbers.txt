<p>I see three general possibilities:</p>
<ol>
<li>Set <strong>m</strong> to a ridiculously small integer to start; I think that your current code will work from there.</li>
<li>Read the first input number <em>before</em> the loop; make that the starting value of <strong>m</strong>.</li>
<li>Don't use a value of m at all: append all of the input numbers to a list, and simply take the <strong>max</strong> of the list after the user enters <strong>0</strong> to stop.</li>
</ol>
<p>Also, <em>please</em> learn to use descriptive variable names.  <strong>finished</strong> is good, but <strong>m</strong> is semantically useless, and <strong>num</strong> is questionable (although it <em>is</em> rather non-descript).</p>
</div>
<div class="post-text" itemprop="text">
<p>Consider the following approach using built-in <a href="https://docs.python.org/3/library/functions.html#max" rel="nofollow noreferrer">max()</a> function:</p>
<pre><code>positives = []   # list for positive numbers
negatives = []   # list for negative numbers
finished = False

while not finished:
    print('Enter a number (0 to finish): ', end = '')
    num = int(input())
    if num &gt; 0:
        positives.append(num)
    elif num &lt; 0:
        negatives.append(num)
    else:
        finished = True

print('Max positive number: ', max(positives))
print('Max negative number: ', max(negatives))
</code></pre>
