<div class="post-text" itemprop="text">
<p>This is explained in <a href="http://docs.python.org/3/library/stdtypes.html#old-string-formatting" rel="noreferrer">the Python documentation</a>.  In short,</p>
<ul>
<li><code>%d</code> will format a number for display.</li>
<li><code>%s</code> will insert the presentation string representation of the object (i.e. <code>str(o)</code>)</li>
<li><code>%r</code> will insert the canonical string representation of the object (i.e. <code>repr(o)</code>)</li>
</ul>
<p>If you are formatting an integer, then these are equivalent.  For most objects this is not the case.</p>
</div>
<div class="post-text" itemprop="text">
<p>Here is an example to supplement James Henstridge's answer:</p>
<pre><code>class Cheese(float):
    def __str__(self):
        return 'Muenster'
    def __repr__(self):
        return 'Stilton'

chunk = Cheese(-123.4)

print(str(chunk))
# Muenster
print(repr(chunk))
# Stilton
print(int(chunk))
# -123
print('%s\t%r\t%d'%(chunk, chunk, chunk))
# Muenster  Stilton -123
</code></pre>
