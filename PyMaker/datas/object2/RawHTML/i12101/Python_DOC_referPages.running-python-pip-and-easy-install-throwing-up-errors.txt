<div class="post-text" itemprop="text">
<p>The <a href="https://github.com/PyMySQL/PyMySQL/blob/4017e31244d660957e633ea6e32721f3c08e9012/pymysql/converters.py#L59" rel="nofollow noreferrer">code of PyMySQL</a> uses Unicode literals.</p>
<p>The <code>py3.2</code> parts of the directory names in the traceback of your error message tells me you're using Python 3.2. While Python 2 had Unicode literals, Python 3.0 to 3.2 didn't: The syntax was <a href="https://docs.python.org/3/whatsnew/3.0.html#text-vs-data-instead-of-unicode-vs-8-bit" rel="nofollow noreferrer">removed with Python 3.0</a>:</p>
<blockquote>
<p>You can no longer use <code>u"..."</code> literals for Unicode text.</p>
</blockquote>
<p>It was <a href="https://docs.python.org/3/whatsnew/3.3.html#pep-414-explicit-unicode-literals" rel="nofollow noreferrer">re-introduced with Python 3.3</a> by <a href="https://www.python.org/dev/peps/pep-0414/" rel="nofollow noreferrer">PEP 414</a> for backwards compatibility reasons.</p>
<p>As <a href="https://stackoverflow.com/a/35828925/674064">Boaz stated</a> and <a href="https://stackoverflow.com/questions/35828760/running-python-pip-and-easy-install-throwing-up-errors#comment59324601_35828760">Padraic pointed out</a>, PyMySQL requires Python 2.x ≥ 2.6 <strong>or 3.x ≥ 3.3</strong>, thus why the authors of PyMySQL felt free to use the Unicode literal syntax in their code.</p>
</div>
