<div class="post-text" itemprop="text">
<p>The two options do <strong>not</strong> produce the same file output on Linux or Mac. Only on Windows will you see the same output.</p>
<p>The first instructs the <code>csv</code> module to write <code>\n</code> as the line terminator between lines. The file object it is writing to, being in text mode, will then translate that terminator to the <em>platform default</em>. The default is <code>\n</code> on Linux or Mac, but it is <code>\r\n</code> on Windows.</p>
<p>The second piece instructs the <em>file object</em> to not change the line endings being written to it (it would otherwise translate line separators). The <code>csv</code> module is left to the default, so it'll write two <code>\r\n</code> characters between lines. The result is that all lines are terminated with <code>\r\n</code> regardless of the platform.</p>
<p>See the <a href="https://docs.python.org/3/library/csv.html#csv.Dialect.lineterminator" rel="nofollow"><code>Dialect.lineterminator</code> documentation</a>:</p>
<blockquote>
<p>The string used to terminate lines produced by the <code>writer</code>. It defaults to <code>'\r\n'</code>.</p>
</blockquote>
<p>and the <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow"><code>open()</code> documentation</a>:</p>
<blockquote>
<p>When writing output to the stream, if <em>newline</em> is None, any <code>'\n'</code> characters written are translated to the system default line separator, <code>os.linesep</code>. If newline is <code>''</code> or <code>'\n'</code>, no translation takes place. If newline is any of the other legal values, any <code>'\n'</code> characters written are translated to the given string.</p>
</blockquote>
