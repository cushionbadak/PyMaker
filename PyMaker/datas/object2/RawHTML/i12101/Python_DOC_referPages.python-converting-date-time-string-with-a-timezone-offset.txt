<p>I also have some dates as Unix timestamp like <code>1344006622</code>, and here I am simply do:</p>
<pre><code>datetime.datetime.fromtimestamp(1344006622).strftime('%d-%m-%Y')
</code></pre>

<h2>Python 2 with <code>dateutil</code></h2>
<p>As far as I know, Python 2 only provides a <code>%Z</code> (capital Z) for timezone <em>names</em>,  but you have an <em>offset</em> (<code>+0000</code>), but there is a library called <a href="https://dateutil.readthedocs.org/en/latest/" rel="nofollow noreferrer">dateutil</a> that can help you:</p>
<pre><code>&gt;&gt;&gt; import dateutil
&gt;&gt;&gt; import dateutil.parser
&gt;&gt;&gt; dateutil.parser.parse('Fri Aug 03 15:16:37 +0000 2012')
datetime.datetime(2012, 8, 3, 15, 16, 37, tzinfo=tzutc())
</code></pre>
<p>In the above example I used Python 2.7.9 with dateutil 2.4.2
You can check your version with</p>
<pre><code>&gt;&gt;&gt; dateutil.__version__
'2.4.2'
</code></pre>
<p>You can install it with <code>pip install python-dateutil</code> or if you want to specify a version <code>pip install python-dateutil==2.4.2</code></p>
<h2>Python 3</h2>
<p>Since Python 3.2 there is a <code>%z</code> that parses the <code>+0000</code> timezone info:</p>
<pre><code>&gt;&gt;&gt; from datetime import datetime
&gt;&gt;&gt; datetime.strptime('Fri Aug 03 15:16:37 +0000 2012', '%a %b %d %H:%M:%S %z %Y')
datetime.datetime(2012, 8, 3, 15, 16, 37, tzinfo=datetime.timezone.utc)
</code></pre>
<p>For an explanation of the parameters used please refer to <a href="https://docs.python.org/3/library/time.html#time.strftime" rel="nofollow noreferrer">this table</a></p>
