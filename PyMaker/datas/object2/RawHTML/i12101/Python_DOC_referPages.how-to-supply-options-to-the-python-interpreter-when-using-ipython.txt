<p>For example, if I wanted to avoid generation of <code>.pyc</code> files when using the usual <code>python</code> command I would write:</p>
<pre><code>python -B script.py
</code></pre>
<p>But what if I want to use <code>ipython</code> instead? All I know is that </p>
<pre><code>ipython -B script.py
</code></pre>
<p>doesn't work</p>
</div>
<div class="post-text" itemprop="text">
<p>Most of the command line options for the Python interpreter have equivalent <a href="https://docs.python.org/3/using/cmdline.html#environment-variables" rel="nofollow">environment variables</a>. These will work when starting IPython.</p>
<p>For instance, instead of the <code>-B</code> flag, to avoid writing bytecode, you can use:</p>
<pre><code>PYTHONDONTWRITEBYTECODE=1 ipython
</code></pre>
</div>
