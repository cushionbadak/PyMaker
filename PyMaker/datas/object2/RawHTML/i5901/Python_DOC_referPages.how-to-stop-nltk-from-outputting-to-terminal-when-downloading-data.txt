<div class="post-text" itemprop="text">
<p>When I run my program, which is using:</p>
<pre><code>nltk.download('wordnet')
from nltk.corpus import wordnet
</code></pre>
<p>I get the following output to my terminal:</p>
<pre><code>[nltk_data] Downloading package wordnet to
[nltk_data]     /Users/.../nltk_data...
[nltk_data]   Package wordnet is already up-to-date!
</code></pre>
<p>My program relies on not having this information saved to the terminal and a resulting output file, so how can I prevent the above lines from occurring, or write it to <code>sys.stderr</code> so it doesn't get included instead of it being through <code>print</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>According to the <a href="https://github.com/nltk/nltk/blob/develop/nltk/downloader.py#L666" rel="nofollow noreferrer">source code</a>, <code>nltk</code> downloader uses straightforward <code>print()</code> calls to report progress. This means that there is no logger involved which you can control or pre-configure. </p>
<p>One of the options is to modify the <code>sys.stdout</code> temporarily on the fly -
 there is that <a href="https://docs.python.org/3/library/contextlib.html#contextlib.redirect_stdout" rel="nofollow noreferrer"><code>redirect_stdout()</code> context manager</a> in Python 3.4+:</p>
<pre><code>from contextlib import redirect_stdout
import os

import nltk
from nltk.corpus import wordnet


with redirect_stdout(open(os.devnull, "w")):
    nltk.download('wordnet')
</code></pre>
<p>Or some other options:</p>
<ul>
<li><a href="https://stackoverflow.com/q/8391411/771848">Suppress calls to print (python)</a></li>
<li><a href="https://stackoverflow.com/questions/2828953/silence-the-stdout-of-a-function-in-python-without-trashing-sys-stdout-and-resto">Silence the stdout of a function in Python without trashing sys.stdout and restoring each function call</a></li>
</ul>
</div>
<div class="post-text" itemprop="text">
<p>Use <code>quiet=True</code>:</p>
<pre><code>import nltk
nltk.download('wordnet', quiet=True)
</code></pre>
</div>
