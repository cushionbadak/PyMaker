<p>The <a href="https://docs.python.org/3/library/socket.html" rel="nofollow noreferrer">documentation</a> says:</p>
<blockquote>
<p>A pair (host, port) is used for the AF_INET address family, where host is a string representing either a hostname in Internet domain notation like 'daring.cwi.nl' or an IPv4 address like '100.50.200.5', and port is an integer.</p>
</blockquote>
<p><code>sendto</code> expects the message &amp; the address so 2 parameters. You're passing this:</p>
<pre><code>client_socket.sendto( (b'10.1.15.243'), (b'5000') )
</code></pre>
<p>so viewed from <code>sendto</code>, the message is <code>b'10.1.15.243'</code> and the address is <code>b'5000'</code>. <code>b"5000"</code> is a bytes object, which explains the error message. The parentheses here have no effect (and even if they <em>did</em> define tuples, that would still be incorrect). Moreover, you also forgot to pass your payload...</p>
<p>As <code>address</code> is a proper address tuple (<code>('10.1.15.243','5000')</code>), just do:</p>
<pre><code>data = b"Positie"  # needs to be bytes object
client_socket.sendto(data, address)
</code></pre>
<p>(address is properly defined above)</p>
