<br/>However, it seems that the modules <em>zope</em> and <em>sklearn</em> do not have the attributes that you are trying to use with them.</p>
<p>try:</p>
<blockquote>
<p><code>from zope.interface import Interface</code></p>

<blockquote>
<p><code>from sklearn import lda</code></p>
</blockquote>
<p>I hope this helps. If not, try reinstalling the packages.</p>

<p>The Attribute error that you are presenting is the expected behaviour of Python3. Did you upgrade from Python 2 to Python 3 Because there are many differences between both version families... read more about this here: 
<a href="https://docs.python.org/3/howto/pyporting.html" rel="nofollow noreferrer">https://docs.python.org/3/howto/pyporting.html</a></p>
<p>To load the sklearn.lda module you have to explicitly import the part of the module that you need.</p>
<blockquote>
<p>from sklearn import lda</p>
</blockquote>
<p>or </p>
<blockquote>
<p>import sklearn.lda</p>
</blockquote>
<p>I hope this answers your question!</p>

<div class="post-text" itemprop="text">
<p>A useful way to get around this problem in the future is to use virtual environments. It will allow you to have different versions of packages running on the same machine. This means that you can upgrade your packages for your new code, but you can still run your old code on the same machine. </p>
<p>If you are using Anaconda then <a href="https://conda.io/docs/using/envs.html" rel="nofollow noreferrer">https://conda.io/docs/using/envs.html</a> is the description. </p>
<p>If you are ever using a cpython distribution <a href="http://virtualenvwrapper.readthedocs.io/en/latest/install.html" rel="nofollow noreferrer">virtualenvwrapper.readthedocs.io/en/latest/install.html</a> is an amazing tool. </p>
<p>While it doesn't solve your problem directly, spending an hour getting comfortable with this will stop dependancy issues for you in the future saving you a lot of time in the future.</p>
