<span class="comment-copy">That looks fine? You could always use <code>from datetime import datetime</code> instead of <code>import datetime</code>. Then you only need <code>datetime</code> instead of <code>datetime.datetime</code>.</span>
<span class="comment-copy">I disagree with the comments above: formatting into a string just to parse it back out is indeed an ugly hack and a bad idea. The <code>datetime</code> constructor can take arbitrary fields, or you can use the <code>replace()</code> method to replace arbitrary fields, either of which are better than working with strings like this.</span>
<span class="comment-copy">Use <a href="https://docs.python.org/3/library/datetime.html#datetime.date.strftime" rel="nofollow noreferrer">strftime</a> instead, <code>datetime.today().strftime('%Y-%m-%d')</code></span>
<span class="comment-copy">"Today's date", as in the question title, is ambiguous. Do you want to end up with a datetime object or with a string?</span>
