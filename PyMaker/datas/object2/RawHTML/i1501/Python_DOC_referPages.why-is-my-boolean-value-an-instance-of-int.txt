<p><code>x</code> is of type <code>bool</code>. So why does my code have the output <code>True</code>? I am new to Python. What's going on here?</p>
<p>Code:</p>
<pre><code>x = True
print(isinstance(x, int))
</code></pre>
<p>Output: </p>
<pre><code>True
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>That's because <code>bool</code> is a subclass of <code>int</code>.</p>
<p>Quoting from the <a href="https://docs.python.org/3/reference/datamodel.html" rel="nofollow noreferrer">Python Data Model</a>:</p>
<blockquote>
<p>Booleans (<a href="https://docs.python.org/3/library/functions.html#bool" rel="nofollow noreferrer">bool</a>)<br/>
  These represent the truth values False and True. The two objects representing the values <code>False</code> and <code>True</code> are the only Boolean objects. <strong>The Boolean type is a subtype of the integer type</strong>, and Boolean values behave like the values 0 and 1, respectively, in almost all contexts, the exception being that when converted to a string, the strings <code>"False"</code> or <code>"True"</code> are returned, respectively.</p>
</blockquote>
<p>Answer to comment:</p>
<blockquote>
<p>Would it also return <code>True</code> if you did <code>isinstance(bool, int)</code>?</p>
</blockquote>
<p>No. <code>True</code> and <code>False</code> are <em>instances</em> of <code>bool</code> and therefore instances of the parent class <code>int</code>, but <code>bool</code> itself is not an <em>instance</em> of <code>int</code>. Being a class it's an instance of <code>type</code> and a <em>subclass</em> of <code>int</code>.</p>
<pre><code>&gt;&gt;&gt; isinstance(bool, int)
False
&gt;&gt;&gt; isinstance(bool, type)
True
&gt;&gt;&gt; issubclass(bool, int)
True
</code></pre>
</div>
