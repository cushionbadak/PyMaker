<p>If I was to use the following in my Jinja template:</p>
<pre><code>{% set data['enabled'] = True %}
</code></pre>
<p>I receive the error <code>TemplateSyntaxError: expected token '=', got '['</code>. Setting one word variables is fine, but as the error states, setting dictionary values through Jinja brings an error.</p>
<p>Is there a workaround for this issue? Thanks.</p>
</div>
<div class="post-text" itemprop="text">
<p>Jinja2 tries to limit assignments in its code to remove the logic from the view (check out an <a href="http://martinfowler.com/eaaDev/uiArchs.html" rel="nofollow noreferrer">MVC explanation</a>).</p>
<p>If you still want to do an assignment you will have to use <a href="https://docs.python.org/3/library/stdtypes.html#dict.update" rel="nofollow noreferrer">update</a> with a do block. For this you have to enable <a href="http://jinja.pocoo.org/docs/dev/extensions/#expression-statement" rel="nofollow noreferrer">Expression Statements</a>. After that you can try something like this:</p>
<pre><code>{% do data.update({'enabled':'True'}) %}
</code></pre>
