<div class="post-text" itemprop="text">
<p>Pandas uses numpy's datetime dtypes called <a href="https://docs.scipy.org/doc/numpy/reference/arrays.datetime.html" rel="nofollow"><code>datetime64</code></a> which is different from the datetime types in python's standard library module <a href="https://docs.python.org/3/library/datetime" rel="nofollow"><code>datetime</code></a>.  </p>
<p>When you are accessing <code>df_tradeDate['date'].dt.date</code>, pandas returns a <a href="https://docs.python.org/3/library/datetime.html#datetime.date" rel="nofollow"><code>datetime.date</code></a> object back which is a generic object in pandas world. Hence you see the dtype difference. </p>
<p>Note that while numpy has units such as <code>datetime64[D]</code> to represent dates, pandas converts all such units to nanoseconds. Hence the only posible datetime dtype in pandas is <code>datetime64[ns]</code>. This does not really lead to any issues in practice. </p>
</div>
