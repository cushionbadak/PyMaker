<pre><code>import os

src = "C:\&lt;Username&gt;\Desktop\test2.txt"

<div class="post-text" itemprop="text">
<p><code>\t</code> is a tab character. I'd suggest using raw strings for windows paths:</p>
<pre><code>src = r"C:\&lt;Username&gt;\Desktop\test2.txt"
dst = r"C:\Users\&lt;Username&gt;"
</code></pre>
<p>This will stop python from surprising you by interpreting some of your backslashes as escape sequences.</p>
</div>
<div class="post-text" itemprop="text">
<p>In addition to using raw string literals, use the <a href="https://docs.python.org/3/library/subprocess.html" rel="nofollow"><code>subprocess</code></a> module instead of <code>os.system</code> - it will take care of quoting your arguments properly if they contain spaces. Thus:</p>
<pre><code>import subprocess

src = r'C:\&lt;Username&gt;\Desktop\test2.txt'
dst = r'C:\Users\&lt;Username&gt;'

subprocess.call(['xcopy', src, dst])
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Try prefixing your strings with <code>r</code>. So <code>r"C:\&lt;Username&gt;\Desktop\test2.txt"</code>. The problem is that a backslash is treated as a special character within strings.</p>
</div>
