<p>I am trying to put a colorbar to my image using matplotlib. The issue comes when I try to force the ticklabels to be written in scientific notation. How can I force the scientific notation (ie, 1x10^0, 2x10^0, ..., 1x10^2, and so on) in the ticks of the color bar?</p>
<p>Example, let's create and plot and image with its color bar:</p>
<pre><code>import matplotlib as plot
import numpy as np

img = np.random.randn(300,300)
myplot = plt.imshow(img)
plt.colorbar(myplot)
plt.show()
</code></pre>

<div class="post-text" itemprop="text">
<p>You could use <a href="http://matplotlib.org/api/figure_api.html#matplotlib.figure.Figure.colorbar" rel="noreferrer"><code>colorbar</code>'s <code>format</code> parameter</a>:</p>
<pre><code>import matplotlib.pyplot as plt
import numpy as np
import matplotlib.ticker as ticker

img = np.random.randn(300,300)
myplot = plt.imshow(img)

def fmt(x, pos):
    a, b = '{:.2e}'.format(x).split('e')
    b = int(b)
    return r'${} \times 10^{{{}}}$'.format(a, b)

plt.colorbar(myplot, format=ticker.FuncFormatter(fmt))
plt.show()
</code></pre>

<span class="comment-copy">You could <a href="https://stackoverflow.com/a/22122723/190597">use <code>imshow(..., vmin=-4, vmax=4)</code></a> to set the min and max values. To get <code>(-4 ---&gt; deep blue and +4 deep red)</code> you could pass <code>cmap=plt.get_cmap('jet')</code> an a parameter to <code>imshow</code> as well (again see the link for an example). Note however that while the <code>jet</code> color map looks pretty, it is <a href="https://stats.stackexchange.com/q/223315/842">not recommended for scientific presentation</a> because it the color scheme can mask or enhance variations when it shouldn't. That's why viridis is the new default.</span>
