<div class="post-text" itemprop="text">
<p>You decoded the hex value to a <em><code>bytes</code> object</em>. If you expected (Unicode) text, decode the bytes with a valid encoding; you appear to have either Latin 1 or Windows Codepage 1252 data here:</p>
<pre><code>&gt;&gt;&gt; import codecs
&gt;&gt;&gt; codecs.decode("5469eb73746f2026204b53484d5220666561742e205661737379", "hex")
b'Ti\xebsto &amp; KSHMR feat. Vassy'
&gt;&gt;&gt; _.decode('latin1')
'Tiësto &amp; KSHMR feat. Vassy'
</code></pre>

<span class="comment-copy">@JoranBeasley: <code>str.encode()</code> and <code>bytes.decode()</code> are entirely consistent in Python 3. You need to use <code>codecs.encode()</code> / <code>codecs.decode()</code> to get to the <a href="https://docs.python.org/3/library/codecs.html#python-specific-encodings" rel="nofollow noreferrer">transforming codecs</a> such as <code>'hex'</code>.</span>
