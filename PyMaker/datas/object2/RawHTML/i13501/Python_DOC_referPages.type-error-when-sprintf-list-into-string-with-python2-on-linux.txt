<div class="post-text" itemprop="text">
<p>The <code>%</code> operator requires that the right-hand operand has to be a <code>tuple</code> or a subclass of <code>tuple</code>if the format has multiple positional placeholders.</p>
<p>That is why you need to convert the arguments into a <code>tuple</code>: </p>
<pre><code>&gt;&gt;&gt; f % tuple(x)
'aa        bbbb      ccccccc   '
</code></pre>
<hr/>
<p>On the other hand, if you use a format string with a single right-hand value, you should be cautious as well, if you don't know the type of the value:</p>
<pre><code>&gt;&gt;&gt; foo = [1, 2, 3]
&gt;&gt;&gt; 'The value is %s' % foo
'The value is [1, 2, 3]'
&gt;&gt;&gt; foo = (1, 2, 3)
&gt;&gt;&gt; 'The value is %s' % foo
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: not all arguments converted during string formatting
</code></pre>
<p>Thus if you do not know the type of the single argument, you should <em>always</em> wrap it into a 1-tuple:</p>
<pre><code>&gt;&gt;&gt; foo = (1, 2, 3)
&gt;&gt;&gt; 'The value is %s' % (foo,)
'The value is (1, 2, 3)'
</code></pre>
<hr/>
<p>That is why <strong>it is now better to use the new</strong> <a href="https://docs.python.org/3/library/stdtypes.html#str.format" rel="nofollow"><code>str.format</code></a> (<a href="https://www.python.org/dev/peps/pep-3101/" rel="nofollow">PEP 3101</a>) especially for complex strings, as it is more powerful, and less prone to errors:</p>
<pre><code>&gt;&gt;&gt; x = ['aa', 'bbbb', 'ccccccc']
&gt;&gt;&gt; '{:10}{:10}{:10}'.format(*x)
'aa        bbbb      ccccccc   '
</code></pre>
</div>
