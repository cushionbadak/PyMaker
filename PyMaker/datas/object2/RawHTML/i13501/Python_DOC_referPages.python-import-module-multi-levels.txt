<p>I've identified the cause. It's because the <strong>__init__.py</strong> under folder "reporting" is not empty, but doing some initiation operations for Flask app. When I <strong>made this "__init__.py" a empty file</strong>, the import was successful using</p>
<pre><code>from reporting.dashboard.helpers.grouper import Grouper
</code></pre>
<p>Also, python-2.6 may <strong>not recognize</strong> "from xxx import yyyClass", but only "import xxxModule"</p>

<a href="https://docs.python.org/3/reference/import.html" rel="nofollow">The import system</a></p>
