<p>I am learning about classes in Python, using Python 3.4.1 and am attempting to make a class and then call the methods from that class.  I have looked at other questions related to this and unfortunately, I can't seem to make it work.  Here is my class (copied straight from the book)</p>
<pre><code>class Counter(object):
    """Models a counter"""
    instances = 0
    def __init__(self):
        """Sets up counter"""
        Counter.instances += 1
        self.reset()
    def reset(self):
        """Sets counter to 0"""
        self._value=0
    def increment(self, amount = 1):
        """Adds amount to counter"""
        self._value += amount
    def decrement(self, amount = 1):
        """Subtracts amount from counter"""
        self._value -= amount
    def getValue(self):
        return self._value
    def __str__(self):
        return str(self._value)
    def __eq__(self, other):
        """Returns True if self == other and False if not"""
        if self is other:
            return True
        if type(self)!=type(other):
            return False
        return self._value==other._value
</code></pre>
<p>and this is how I'm calling it from another file (in the same folder):</p>
<pre><code>import Counter
h = Counter()
print(h.getValue())
</code></pre>
<p>and this is the error I'm getting:</p>
<pre><code>Traceback (most recent call last):
File "C:/Python34/learning/classtest.py", line 3, in &lt;module&gt;
h = Counter()
TypeError: 'module' object is not callable
</code></pre>
<p>I can type import Counter just fine into the shell, but when I get to h = Counter() I get the same error.  I know I'm doing something wrong, but what?</p>

<div class="post-text" itemprop="text">
<p>You named your module <code>Counter</code> too; the class is contained <em>in</em> the module:</p>
<pre><code>import Counter

h = Counter.Counter()
</code></pre>
<p>Alternatively, import the class from the module:</p>
<pre><code>from Counter import Counter

h = Counter()
</code></pre>
<p>This is why the <a href="http://www.python.org/dev/peps/pep-0008/" rel="nofollow">Python style guide</a> recommends you use all lower-case names for your modules. In Python, the module name does <strong>not</strong> have to match the class contained, and you are not limited to just one class in a module. A module can contain just functions or any other Python object too.</p>
<p>Had you named your module file <code>counter</code> (all lowercase) instead, it would perhaps have been more obvious that the module and contained class are two distinct concepts. :-)</p>
</div>
<div class="post-text" itemprop="text">
<p>In simple terms, the line:</p>
<pre><code>import Counter
</code></pre>
<p>only makes the module <code>Counter</code> available for use.  If you want to use one of the tools that it contains, such as the class <code>Counter</code>, you need to qualify it with the module name:</p>
<pre><code>import Counter
h = Counter.Counter()
</code></pre>
<p>Or, you can import the tools you want directly:</p>
<pre><code>from Counter import Counter
h = Counter()
</code></pre>
<p>Here is a reference on <a href="https://docs.python.org/3/reference/simple_stmts.html#grammar-token-import_stmt" rel="nofollow">importing in Python</a>.</p>
<hr/>
<p>Also, PEP 8, the official style guide for Python code, states that <a href="http://legacy.python.org/dev/peps/pep-0008/#package-and-module-names" rel="nofollow">module names should be lowercase</a>:</p>
<blockquote>
<p>Modules should have short, all-lowercase names. Underscores can be
  used in the module name if it improves readability. Python packages
  should also have short, all-lowercase names, although the use of
  underscores is discouraged.</p>
</blockquote>
<p>Therefore, it would be best if you renamed the <code>Counter</code> module to <code>counter</code>.</p>
</div>
<div class="post-text" itemprop="text">
<p>You want <code>Counter.Counter</code> instead, because your file is named <code>Counter.py</code> as well.</p>
<p>If you don't call <code>h = Counter.Counter()</code>, you are basically trying to invoke the module as a function:</p>
<pre><code>&gt;&gt;&gt; import math
&gt;&gt;&gt; math()
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'module' object is not callable
</code></pre>
<p>You have two options.</p>
<p>1. Call <code>import Counter</code> and then call <code>h = Counter.Counter()</code></p>
<pre><code>import Counter
h = Counter.Counter()
print(h.getValue())
</code></pre>
<p>2. Call <code>from Counter import Counter</code> and then call <code>h = Counter()</code></p>
<pre><code>from Counter import Counter
h = Counter()
print(h.getValue())
</code></pre>
</div>
