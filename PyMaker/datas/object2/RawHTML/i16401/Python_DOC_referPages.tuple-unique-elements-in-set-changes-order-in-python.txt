<p>I have a tuple of repeating numbers and would like to keep only the unique items but not change the order.  This works:</p>
<p><code>values = (30.0,30.0,30.0,15.0,30.0])
print set(values)</code></p>
<p>which returns:</p>
<pre><code>set([30.0, 15.0])
</code></pre>
<p>But when I try:</p>
<pre><code>values = (2, 1, 2, 1)
</code></pre>
<p>It returns:</p>
<pre><code>set([1, 2])
</code></pre>
<p>My question is why is it not keeping the order in the second example.</p>
</div>
<div class="post-text" itemprop="text">
<p>Sets have no concept of order, but you could use an <a href="https://docs.python.org/3/library/collections.html#collections.OrderedDict" rel="nofollow"><code>OrderedDict</code></a> to achieve what you want:</p>
<pre><code>&gt;&gt;&gt; from collections import OrderedDict
&gt;&gt;&gt;
&gt;&gt;&gt; values = (2, 1, 2, 1)
&gt;&gt;&gt; list(OrderedDict.fromkeys(values))
[2, 1]
&gt;&gt;&gt;
&gt;&gt;&gt; values = (30.0, 30.0, 30.0, 15.0, 30.0)
&gt;&gt;&gt; list(OrderedDict.fromkeys(values))
[30.0, 15.0]
</code></pre>
</div>
