<div class="post-text" itemprop="text">
<p>You simply need to make <code>cab</code> a string:</p>
<pre><code>cab = '6176'
</code></pre>
<p>As the error message states, you cannot do <code>&lt;int&gt; in &lt;string&gt;</code>:</p>
<pre><code>&gt;&gt;&gt; 1 in '123'
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'in &lt;string&gt;' requires string as left operand, not int
&gt;&gt;&gt; 
</code></pre>
<p>because <a href="https://docs.python.org/3/library/functions.html#int" rel="noreferrer">integers</a> and <a href="https://docs.python.org/3/library/functions.html#str" rel="noreferrer">strings</a> are two totally different things and Python does not embrace implicit type conversion (<a href="http://legacy.python.org/dev/peps/pep-0020/" rel="noreferrer">"Explicit is better than implicit."</a>). </p>
<p>In fact, Python <em>only</em> allows you to use the <code>in</code> operator with a right operand of type string if the left operand is also of type string:</p>
<pre><code>&gt;&gt;&gt; '1' in '123'  # Works!
True
&gt;&gt;&gt;
&gt;&gt;&gt; [] in '123'
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'in &lt;string&gt;' requires string as left operand, not list
&gt;&gt;&gt;
&gt;&gt;&gt; 1.0 in '123'
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'in &lt;string&gt;' requires string as left operand, not float
&gt;&gt;&gt;
&gt;&gt;&gt; {} in '123'
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'in &lt;string&gt;' requires string as left operand, not dict
&gt;&gt;&gt;
</code></pre>
