<div class="post-text" itemprop="text">
<p>What would be the more pythonic way to find the largest number multiple of <code>n</code> but lower than an upperbound <code>x</code>?</p>
<p>Practical example:</p>
<pre><code>n = 48 
x = 2636 
</code></pre>
<p><code>48 * 54 = 2592</code> is the nearest.</p>
<p>I would do a <code>for</code> loop till I don't go over <code>x</code>. What are nicer alternatives?</p>
</div>
<div class="post-text" itemprop="text">
<p>The simplest way is probably using <code>//</code>:</p>
<pre><code>(x // n) * n
</code></pre>
<p>If the number has to be strictly less than <code>x</code>, use <code>x - 1</code> instead:</p>
<pre><code>((x - 1) // n) * n
</code></pre>
<p>The expression <code>x // n</code> is the floor division of <code>x</code> by <code>n</code>, discarding any remainder.</p>
</div>
<div class="post-text" itemprop="text">
<p>You could use the modulo operation</p>
<pre><code>x-(x%n)
</code></pre>
<p>If the upper bound has to be strictly lower than x use</p>
<pre><code>x - 1 -((x-1) % n)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>This works too:</p>
<pre><code>print(x - (x%n))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The most pythonic way I could think of would be for either python 2 or 3:</p>
<pre><code>&gt;&gt;&gt; (x // n) * n
</code></pre>
</div>
