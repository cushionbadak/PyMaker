<div class="post-text" itemprop="text">
<p>For my application scope, I need to concatenate two one-dimension array into one multi-dimension array, both implemented using (eventually nested) <code>list</code>s in Python. The concatenations have to print all the possible combinations between the elements of the first array with the elements of the second array. </p>
<pre><code>vectA=[124,172,222,272,323,376,426,479,531]
vectB=[440,388,336,289,243,197,156,113,74]
</code></pre>
<p>The expected result is a multi-dimension array with the combinations of <code>vectA</code> with <strong>all</strong> the elements of <code>vectB</code> (cartesian product).</p>
<pre><code>output=[[124,440],[124,388],[124,336],[124,289]...[172,440],[172,388]...]
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>No need to import a package here.</p>
<p>You can do this with simple <a href="https://docs.python.org/3/tutorial/datastructures.html#list-comprehensions" rel="nofollow noreferrer">list comprehensions</a>, too: </p>
<pre><code>vectA = [124, 172, 222, 272, 323, 376, 426, 479, 531]
vectB = [440, 388, 336, 289, 243, 197, 156, 113, 74]

output = [[a, b] for a in vectA for b in vectB]
print(output)
</code></pre>
<hr/>
<p>Also, I would propose to output a list of <a href="https://docs.python.org/3/tutorial/datastructures.html#tuples-and-sequences" rel="nofollow noreferrer">tuples</a> instead of a list of lists:</p>
<pre><code>output = [(a, b) for a in vectA for b in vectB]
</code></pre>
<p>giving you: <code>[(124, 440), (124, 388), (124, 336), ... , (531, 74)]</code></p>
<p>Using tuples would, in my opinion, more clearly convey to someone else your intention of pairing all the values of vectA with all the values of vectB.</p>
<p>You can still do e.g. <code>output[0]</code> to get <code>(124, 440)</code> and <code>output[0][0]</code> to get <code>124</code> as you would with a list of lists.</p>
<p>Note though, that you can <strong>not</strong> overwrite the values of a tuple like you could with values of a list, since tuples are immutable.</p>
</div>
<div class="post-text" itemprop="text">
<p>use <a href="https://docs.python.org/2/library/itertools.html#itertools.product" rel="nofollow noreferrer">itertools.product</a>:</p>
<pre><code>from itertools import product

vectA=[124,172,222,272,323,376,426,479,531]
vectB=[440,388,336,289,243,197,156,113,74]

output = list(product(vectA,vectB))
output = [list(i) for i in output]
print(output)
</code></pre>
