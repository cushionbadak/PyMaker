<p>I would like to compose a form results URL whereas certain parts can be programatically changed. I've read similar questions were <code>requests</code> <code>urljoin()</code> is used but it can take only two parameters. Is there anything more appropriate for this scenario?</p>
<p>This is the base</p>
<pre><code>https://navlib.forth-crs.gr/italian_b2c/npgres.exe?func=TT&amp;ReservationType=npgres.exe%3FPM%3DBO&amp;Leg1i=
</code></pre>
<p>This part is the route which will need to change</p>
<pre><code>BEV&amp;Leg1ii=PRJ
</code></pre>
<p>This is the date  which will need to change</p>
<pre><code>&amp;Leg1Date=28%2F02%2F2019
</code></pre>
<p>This part will not need changing</p>
<pre><code>&amp;TotalPassengers=1&amp;TotalPassengersHuman=1&amp;TotalPassengersAcce=0&amp;TotalVehicles=0
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Use <a href="https://docs.python.org/3/library/urllib.parse.html#urllib.parse.urlencode" rel="nofollow noreferrer"><code>urllib.parse.urlencode</code></a>, it takes a mapping or two-element tuple sequence and produces a properly-encoded query string:</p>
<pre><code>import urllib.parse

params = urllib.parse.urlencode({
    "Leg1ii": "PRJ",
    "Leg1Date": "28/02/2019",
    "TotalPassengers": "1",
    "TotalPassengersHuman": "1",
    "TotalPassengersAcce": "0",
    "TotalVehicles": "0",
})

print(f"https://navlib.forth-crs.gr/italian_b2c/npgres.exe?{params}")
</code></pre>
<p>Result:</p>
<pre>
https://navlib.forth-crs.gr/italian_b2c/npgres.exe?Leg1ii=PRJ&amp;Leg1Date=28%2F02%2F2019&amp;TotalPassengers=1&amp;TotalPassengersHuman=1&amp;TotalPassengersAcce=0&amp;TotalVehicles=0
</pre>
