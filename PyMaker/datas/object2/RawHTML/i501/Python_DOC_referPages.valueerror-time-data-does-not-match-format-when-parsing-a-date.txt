<div class="post-text" itemprop="text">
<p>When I try to parse a string to a datetime, I do this (having imported datetime before):</p>
<pre><code>fecha_2 = datetime.strptime('22/01/2019 17:00', '%d/%m/%y %H:%M')
</code></pre>
<p>However, I receive this error</p>
<blockquote>
<p>ValueError: time data '22/01/2019 17:00' does not match format '%d/%m/%y %H:%M'</p>
</blockquote>
<p>anyone know why?</p>
</div>
<div class="post-text" itemprop="text">
<p>The <code>y</code> should be capitalized. This is referenced in the Python docs found <a href="https://docs.python.org/3/library/datetime.html#strftime-strptime-behavior" rel="nofollow noreferrer">here</a></p>
<p>This works fine</p>
<p><code>fecha_2 = datetime.strptime('22/01/2019 17:00', '%d/%m/%Y %H:%M')</code></p>
</div>
<div class="post-text" itemprop="text">
<p>You need to use a capital <code>Y</code> for a 4 digit year (lowercase <code>y</code> is for a 2 digit year like <code>19</code>). So:</p>
<pre><code>&gt;&gt;&gt; fecha_2 = datetime.strptime('22/01/2019 17:00', '%d/%m/%Y %H:%M')
&gt;&gt;&gt; fecha_2
datetime.datetime(2019, 1, 22, 17, 0)
</code></pre>
<p><a href="https://docs.python.org/2/library/datetime.html#strftime-strptime-behavior" rel="nofollow noreferrer">strptime docs</a></p>
</div>
<div class="post-text" itemprop="text">
<p>The above is correct. However, to avoid these kind of errors you can use dateutil.parser, which can automatically guess the correct format string. You can install it by doing pip install python-dateutil</p>
<pre><code>&gt;&gt;&gt; from dateutil import parser
&gt;&gt;&gt; parser.parse("2018-06-19 11:21:13.311")
datetime.datetime(2018, 6, 19, 11, 21, 13, 311000)
</code></pre>
</div>
<span class="comment-copy">lower case <code>'y'</code> means 2 digit year rather than 4 digit year</span>
