<p>I noticed something today while fiddling with my code:</p>
<pre><code>print("lol") if None else print("Not lol") # Displays 'Not lol'
print("lol") if 0 else print("Not lol") # Displays 'Not lol'
print("lol") if float("nan") else print("Not lol") # Displays 'lol'
</code></pre>
<p>Unlike <code>None</code> and <code>0</code>. Why is <code>float("nan")</code> not considered Falsy? </p>

<div class="post-text" itemprop="text">
<p>OP : <em>Unlike None and 0. Why is float("nan") not considered Falsy?</em></p>
<p>Ans: <a href="https://docs.python.org/2/library/stdtypes.html#truth-value-testing" rel="nofollow noreferrer">Because that's what the language designers decided would be most useful.</a></p>
<pre><code>print(bool(float('nan')))  # returns True
</code></pre>
<p>Hence:</p>
<pre><code>print("lol") if float("nan") else print("Not lol")
</code></pre>
<p>OUTPUT:</p>
<pre><code>lol
</code></pre>

<div class="post-text" itemprop="text">
<p>It's not falsy because it's a valid string argument for float. You can find more information in the documentation.
<a href="https://docs.python.org/3/library/functions.html?highlight=float#float" rel="nofollow noreferrer">https://docs.python.org/3/library/functions.html?highlight=float#float</a></p>
<blockquote>
<p>If the argument is a string, it should contain a decimal number, optionally preceded by a sign, and optionally embedded in whitespace. The optional sign may be '+' or '-'; a '+' sign has no effect on the value produced. <strong>The argument may also be a string representing a NaN (not-a-number)</strong>, or a positive or negative infinity.</p>
</blockquote>
