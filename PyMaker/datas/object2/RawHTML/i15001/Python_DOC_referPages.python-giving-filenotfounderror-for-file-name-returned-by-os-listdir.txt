<div class="post-text" itemprop="text">
<p>I was trying to iterate over the files in a directory like this:</p>
<pre><code>import os

path = r'E:/somedir'

for filename in os.listdir(path):
    f = open(filename, 'r')
    ... # process the file
</code></pre>
<p>But Python was throwing <code>FileNotFoundError</code> even though the file exists:</p>
<pre><code>Traceback (most recent call last):
  File "E:/ADMTM/TestT.py", line 6, in &lt;module&gt;
    f = open(filename, 'r')
FileNotFoundError: [Errno 2] No such file or directory: 'foo.txt'
</code></pre>
<p>So what is wrong here?</p>
</div>
<div class="post-text" itemprop="text">
<p>It is because <a href="https://docs.python.org/3/library/os.html#os.listdir" rel="nofollow noreferrer"><code>os.listdir</code></a> does not return the full path to the file, only the filename part; that is <code>'foo.txt'</code>, when open would want <code>'E:/somedir/foo.txt'</code> because the file does not exist in the current directory.</p>
<p>Use <a href="https://docs.python.org/3/library/os.path.html#os.path.join" rel="nofollow noreferrer"><code>os.path.join</code></a> to prepend the directory to your filename:</p>
<pre><code>path = r'E:/somedir'

for filename in os.listdir(path):
    with open(os.path.join(path, filename)) as f:
        ... # process the file
</code></pre>
<p>(Also, you are not closing the file; the <code>with</code> block will take care of it automatically).</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/2/library/os.html#os.listdir" rel="nofollow"><code>os.listdir(directory)</code></a> returns a list of file <em>names</em> in <code>directory</code>. So unless <code>directory</code> is your current working directory, you need to join those file names with the actual directory to get a proper absolute path:</p>
<pre><code>for filename in os.listdir(path):
    filepath = os.path.join(path, filename)
    f = open(filepath,'r')
    raw = f.read()
    # ...
</code></pre>
</div>
