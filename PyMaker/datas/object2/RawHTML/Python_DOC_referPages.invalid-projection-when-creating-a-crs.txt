<p>I am attempting to construct a CRS instance with PyProj to match a geographic shape to image bounds. With the following code</p>
<pre><code>pyproj.Proj(proj='aea', lat1=b[1], lat2=b[3])
</code></pre>
<p>I am getting the error:</p>
<pre><code>pyproj.exceptions.CRSError: Invalid projection: +proj=aea
 +lat1=34.82176739999999 +lat2=34.8219931 +type=crs
</code></pre>
<p>There doesn't seem to be a lot of documentation about what makes a projection valid or invalid, apart from an apparently <a href="http://trac.osgeo.org/proj/wiki/GenParms" rel="nofollow noreferrer">no longer extant documentation page</a> on the Proj4 wiki. Can anyone help me to understand what's going wrong here? The code (not written by me) appears to have worked in the past.</p>

<div class="post-text" itemprop="text">
<p>I think the issue here is you mispelled the projection parameters. </p>
<p><em>lat1</em> and <em>lat2</em> should be <em>lat_1</em> and <em>lat_2</em> (with the underscore).</p>
<p>The correct proj string should be:</p>
<p><code>+proj=aea +lat_1=34.82176739999999 +lat_2=34.8219931</code></p>
<p>You can find more details about the format of the proj string in the official documentation, linked here: <a href="https://proj4.org/operations/projections/aea.html" rel="nofollow noreferrer">https://proj4.org/operations/projections/aea.html</a></p>
</div>
