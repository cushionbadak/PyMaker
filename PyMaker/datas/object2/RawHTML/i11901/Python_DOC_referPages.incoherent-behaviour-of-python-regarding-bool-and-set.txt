<p>I noticed a strange behaviour of python3 regarding set and boolean values.</p>
<pre><code>&gt;&gt;{True}
&gt;&gt;type({True})

</code></pre>
<p>However,</p>
<pre><code>&gt;&gt;set(True)
Traceback (most recent call last):
File "&lt;stdin&gt;", line 1, in &lt;module&gt;
TypeError: 'bool' object is not iterable 
</code></pre>
<p>I'm quite puzzled here.</p>
</div>
<div class="post-text" itemprop="text">
<p>The set constructor needs an iterable, use <code>set((True,))</code> or <code>set([True])</code>, you're trying to cast a <code>bool</code> into a <code>set</code> which is throwing that error.
Read the <a href="https://docs.python.org/3/tutorial/datastructures.html#sets" rel="nofollow">docs</a> for more info.</p>
</div>
<span class="comment-copy">The first is a literal <code>set</code> with one element of type <code>bool</code>. The second is trying to cast a <code>bool</code> to a <code>set</code> - which isn't possible.</span>
<span class="comment-copy">There is a good clue in <i>'bool' object is not iterable</i></span>
