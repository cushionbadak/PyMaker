<p>Why does <code>print r"""</code> return the string <code>...</code> and doesn't exit?</p>
<p>When I press <kbd>Enter</kbd>, the console displays a new row preceded by the string <code>...</code></p>
<p><img alt="the display of console" src="https://i.stack.imgur.com/9Qrf2.png"/></p>

<div class="post-text" itemprop="text">
<p>Because the three quotes are seen as the beginning of a triple-quoted string. It is waiting for you to type another set of triple quotes.</p>
<pre><code>&gt;&gt;&gt; print r"""
... blah
... """

blah

&gt;&gt;&gt; 
</code></pre>
<p>For more information see:</p>
<ul>
<li><a href="https://stackoverflow.com/q/1520548/7432">How does Python's triple-quote string work?</a></li>
<li><a href="https://docs.python.org/2.7/tutorial/introduction.html#strings" rel="nofollow noreferrer">https://docs.python.org/2.7/tutorial/introduction.html#strings</a>; search for <code>"""</code></li>
</ul>
<p>(The second link is to 2.7 docs since your example used python 2.x syntax). </p>
<p>A direct quote from the python documentation:</p>
<blockquote>
<p>String literals can span multiple lines. One way is using triple-quotes: """...""" or '''...'''. End of lines are automatically included in the string, but it’s possible to prevent this by adding a \ at the end of the line. </p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p><code>print r"""</code> starts a multi-line string.</p>
<p>Because the string hasn't been terminated with another <code>"""</code> the interpreter keeps printing <code>...</code>, waiting for you to type the next line  or match the opening quotes with a closing one. So, if you type <code>"""</code> again the standard interpreter prompt <code>&gt;&gt;&gt;</code> will reappear or resume.</p>
<p><a href="https://docs.python.org/3/tutorial/introduction.html#strings" rel="nofollow">Python 3 implementation</a>:</p>
<pre><code>&gt;&gt;&gt; print("""
...   
... abc
... 
... ggg
... 
... """)


abc

ggg


&gt;&gt;&gt; 
</code></pre>
<p><a href="https://docs.python.org/2.7/tutorial/introduction.html#strings" rel="nofollow">Python 2.7 implementation</a>:</p>
<pre><code>&gt;&gt;&gt; print r"""
... blah
... black sheep
... """

black sheep
blah
</code></pre>
