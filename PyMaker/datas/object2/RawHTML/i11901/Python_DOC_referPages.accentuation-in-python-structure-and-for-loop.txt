<div class="post-text" itemprop="text">
<p>I guess you are using python 2 and it might be related to the default encoding behavior. The value stocked in your set is the "encoded" value and when you use <code>print</code> (which is based on the underlying <code>__repr__</code> and/or <code>__str__</code> methods of the object) you get the decoded/formated output (according to the default system encoding).</p>
<p>You can obtain information about the default encoding used with the function <code>sys.getdefaultencoding()</code></p>
<p>Note that in python 3, encoding is <code>utf-8</code> by default (ie. by default "any string created (...) is stored as Unicode", according to the <a href="https://docs.python.org/3/howto/unicode.html" rel="nofollow">documentation</a>) and you wont have the exact same behavior (you can see in the <em>python 2</em> snippet that the hashed values, as python <code>set</code>s are based on them, are the same if your input string is encoded or not) :</p>
<p><strong>Python 2 :</strong></p>
<pre><code>&gt;&gt;&gt; a = b'Path\xc3\xa9'
&gt;&gt;&gt; a
'Path\xc3\xa9'
&gt;&gt;&gt; print(a)
Pathé
&gt;&gt;&gt; sys.getdefaultencoding()
'ascii'
&gt;&gt;&gt; hash('Pathé')
8776754739882320435
&gt;&gt;&gt; hash(b'Path\xc3\xa9')
8776754739882320435
</code></pre>
<p><strong>Python 3:</strong></p>
<pre><code>&gt;&gt;&gt; a = b'Path\xc3\xa9'
&gt;&gt;&gt; a
b'Path\xc3\xa9'
&gt;&gt;&gt; print(a)
b'Path\xc3\xa9'
&gt;&gt;&gt; print(a.decode())
Pathé
&gt;&gt;&gt; sys.getdefaultencoding()
'utf-8'
&gt;&gt;&gt; hash("Pathé")
1530394699459763000
&gt;&gt;&gt; hash(b"Path\xc3\xa9")
1621747577200686773
</code></pre>
