<div class="post-text" itemprop="text">
<p><code>class Tk</code> is in:</p>
<ul>
<li>Py 2.7: <a href="https://github.com/python/cpython/blob/2.7/Lib/lib-tk/Tkinter.py#L1795" rel="nofollow noreferrer"><code>Tkinter.py</code></a></li>
<li>Py 3.6: <a href="https://github.com/python/cpython/blob/3.6/Lib/tkinter/__init__.py#L1996" rel="nofollow noreferrer"><code>__init__.py</code></a>.  </li>
</ul>
<p>In general, when you <code>import Foo</code>, if <code>Foo</code> is a module implemented using Python code, and there is a <code>Foo/__init__.py</code> in Python's <a href="https://docs.python.org/2/tutorial/modules.html#the-module-search-path" rel="nofollow noreferrer">search path</a>, that file runs.  <a href="http://mikegrouchy.com/blog/2012/05/be-pythonic-__init__py.html" rel="nofollow noreferrer">Some more information here,</a> plus official docs on modules for <a href="https://docs.python.org/2/tutorial/modules.html#packages" rel="nofollow noreferrer">Python 2</a> or <a href="https://docs.python.org/3/tutorial/modules.html#packages" rel="nofollow noreferrer">Python 3</a>.  That may not be the case for modules built in to the Python interpreter, such as 2.7's <code>Tkinter</code>.</p>
<h3><code>Tkinter</code> specifics</h3>
<ul>
<li>In Python 2, <code>Tkinter</code> is a Python module and <code>tkinter</code> (lowercase) is a C module that <code>Tkinter</code> uses (<a href="https://docs.python.org/2/library/tkinter.html#how-tk-and-tkinter-are-related" rel="nofollow noreferrer">source</a>).</li>
<li>In Python 3, <code>tkinter</code> (lowercase) is a Python module and <code>_tkinter</code> (with underscore) is the C module that <code>tkinter</code> uses (<a href="https://docs.python.org/3/library/tkinter.html#how-tk-and-tkinter-are-related" rel="nofollow noreferrer">source</a>).</li>
</ul>
