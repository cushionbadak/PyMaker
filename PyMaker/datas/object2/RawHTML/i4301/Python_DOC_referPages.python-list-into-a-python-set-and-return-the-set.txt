<div class="post-text" itemprop="text">
<p>When you declare <code>y = {}</code>, you create an empty <code>dict</code>, not an empty <code>set</code>.
To create an empty set, you need <code>y = set()</code>.</p>
<p>For your exercise, you could store your unique values in a set (if you don't consider using <code>set()</code> to create the empty set cheating):</p>
<pre><code>x =  [ 1, 1, 1, 3, 4, 5, 5]
y = set()

for z in x:
    if z in y:
        continue
    else:
        y.add(z)  # you add your new unique element to the set 

print(y)
</code></pre>
<p>Or you can use a list:</p>
<pre><code>x =  [ 1, 1, 1, 3, 4, 5, 5]
y = []

for z in x:
    if z in y:
        continue
    else:
        y.append(z)  # you append your new unique element to the list 

print(y)
</code></pre>
<p>But of course, the efficient, not-an-exercise way would be to just write <code>y = set(x)</code></p>
</div>
<div class="post-text" itemprop="text">
<p>I don't know how strict is the expected task, but you can just do this (you are not using <code>set()</code>):</p>
<pre><code>y = {element for element in x}
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Do this: </p>
<pre><code>A_list =  [ 1, 1, 1, 3, 4, 5, 5]
set_A_list = {*A_list}
</code></pre>
<p>About your error - you cannot create set with <code>y = {}</code> (empty braces). In that case the type of <code>y</code> will be <code>dict</code>. However, for example, <code>y = {1, 2}</code> will create a <code>set</code> (as it is <strong>not</strong> empty).</p>
