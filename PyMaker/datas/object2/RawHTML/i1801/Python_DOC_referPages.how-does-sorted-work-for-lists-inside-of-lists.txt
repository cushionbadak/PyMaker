<p>I am confused on how lists inside lists get sorted. </p>
<pre><code>L = [[1,1,1],[0,9,0],[2,1,1]]
</code></pre>
<p><code>sorted(L)</code> returns <code>[[0, 9, 0], [1, 1, 1], [2, 1, 1]]</code></p>
<p>This means that it is not based off sum as 0+9+0 is larger than both of the other ones.</p>
</div>
<div class="post-text" itemprop="text">
<p>No it is based on all the elements of the iterator starting from the first element of the iterator</p>
<pre><code>sorted(L,key=lambda x:(x[0],x[1],x[2]) #==sorted(L)
</code></pre>
<p>In case you need by sum</p>
<pre><code>sorted(L,key=sum) 
</code></pre>
<p>A more simplified version of above code to understand the key argument further</p>
<pre><code>print(sorted(L,key=lambda x: x[0]+x[1]+x[2]))
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Built-in <code>sorted</code> considers <strong>each element of an iterable in turn</strong>. So, for example, <code>[0, 9, 0]</code> appears first because <code>0 &lt; 1</code> and <code>0 &lt; 2</code>.</p>
<p>To help gain intuition, you can test a few examples:</p>
<pre><code>[0,9,0] &lt; [1,1,1]  # True
[0,9,0] &lt; [0,8,0]  # False
[1,1,1] &lt; [2,1,1]  # True
</code></pre>
<p>So <code>sorted</code> works consistently with how comparison operators are defined. Sequence objects in Python usually support <a href="https://docs.python.org/3/tutorial/datastructures.html#comparing-sequences-and-other-types" rel="nofollow noreferrer">lexicographic comparison</a>. To sort by the sum of each list, you need to feed a function to the <code>key</code> argument, in this case built-in <code>sum</code>:</p>
<pre><code>res = sorted(L, key=sum)
</code></pre>
</div>
