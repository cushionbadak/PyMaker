<p>I am trying to find a way to get two actions done in one line if statement, I can find a lot of answers to perform a single action in if statement one-liner but not for this. Is it even possible? I tried something like this but failed with ValueError - too many values to unpack (expected 2). Thanks.</p>
<pre><code>g, er = "id", "err" if no_of_errs &lt;= 1 else "ppd", "ers"
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Python is looking at your code like this:</p>
<pre><code> g, er = ( "id" )  ,  ( "err" if no_of_errs &lt;= 1 else "ppd" )   , ( "ers" )
</code></pre>
<p>3 things to unpack, only 2 to pack it into.</p>
<p>The reason behind the error is <em>operator priority</em>, you can either read lots of  lenghty text <a href="https://docs.python.org/3/reference/expressions.html" rel="nofollow noreferrer">here</a>  or google it and find a table like <a href="http://www.mathcs.emory.edu/~valerie/courses/fall10/155/resources/op_precedence.html" rel="nofollow noreferrer">here</a>.</p>
<hr/>
<p>Fix it by makeing the tuples explicit with parenthesis:</p>
<pre><code>no_of_errs = 0
g, er = ("id", "err") if no_of_errs &lt;= 1 else ("ppd", "ers")
print(g,er)

no_of_errs = 10
g, er = ("id", "err") if no_of_errs &lt;= 1 else ("ppd", "ers")
print(g,er)
</code></pre>
<p>Output:</p>
<pre><code>id err
ppd ers
</code></pre>
<p>With explicit tuples, no more operator confusion and it works.</p>
<hr/>
<p>Keywords for google: <code>operator</code> <code>priority</code> <code>precedence</code> or smth alike</p>
