<div class="post-text" itemprop="text">
<p>Since <code>imdb.Movie.Movie</code> is a subclass of <code>imdb.utils._Container</code> with a <code>get</code> method similar to <a href="https://docs.python.org/3.6/library/stdtypes.html?highlight=get#dict.get" rel="nofollow noreferrer">that of a <code>dict</code></a>, and which docstring reads:</p>
<pre><code>&gt;&gt;&gt; imdb.utils._Container.get.__doc__
"Return the given section, or default if it's not found."
</code></pre>
<p>That means you can do this to never throw an exception if there are no keywords:</p>
<pre><code>movie = ia.get_movie('5950092', info='keywords')

movie.get('keywords', [])
# Result: [], empty list
</code></pre>
<p>You can also work with an <code>Exception</code> if you want to:</p>
<pre><code>try:
    keywords = movie['keywords']
except KeyError:
    keywords = []
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>In IMDbPY, Movie instances behave like dictionaries, so you handle exceptions in the usual way (with a try/except clause).  See <a href="https://docs.python.org/3/tutorial/errors.html#handling-exceptions" rel="nofollow noreferrer">https://docs.python.org/3/tutorial/errors.html#handling-exceptions</a></p>
<p>Being dictionary-like objects, you can also test for the presence of a key with <code>'keywords' in black_panther</code> and get the value without raising n exception, but returning None if the key is missing, with <code>black_panther.get('keywords')</code>.</p>
