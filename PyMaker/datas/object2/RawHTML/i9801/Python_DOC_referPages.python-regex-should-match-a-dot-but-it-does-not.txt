<p>I have the following Python 3 code:</p>
<pre><code>import re
pattern=re.compile(r'\.')
print(pattern.match('abc.de'))
</code></pre>
<p>The output is:</p>
<pre><code>None
</code></pre>
<p>What am I doing wrong? Why the regex does not match the dot?</p>

<div class="post-text" itemprop="text">
<p>As per the documentation of <a href="https://docs.python.org/2/library/re.html#re.RegexObject.match" rel="nofollow noreferrer"><code>match</code></a> it checks from the beginning of a string.</p>
<blockquote>
<p>If zero or more characters at the beginning of string match this regular expression, return a corresponding MatchObject instance. Return None if the string does not match the pattern; note that this is different from a zero-length match.</p>
</blockquote>
<hr/>
<p>Use <a href="https://docs.python.org/2/library/re.html#re.RegexObject.search" rel="nofollow noreferrer"><code>search</code></a> instead  for search at any position.</p>
<pre><code>&gt;&gt;&gt; import re
&gt;&gt;&gt; pattern=re.compile(r'\.')
&gt;&gt;&gt; print(pattern.search('abc.de'))
&lt;_sre.SRE_Match object at 0x7fc7b5823648&gt;
&gt;&gt;&gt; print(pattern.search('abc.de').group())
.
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p><code>match</code> looks for matches at the beginning of the string, unless you tell it to do otherwise. The dot isn't at the beginning of the string, so it can't be found.</p>
<p>See the documentation here: <a href="https://docs.python.org/3/library/re.html#re.match" rel="nofollow noreferrer">https://docs.python.org/3/library/re.html#re.match</a></p>
