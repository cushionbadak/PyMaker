<div class="post-text" itemprop="text">
<p>I have a big file of text that is a list of dictionaries, like so</p>
<pre><code>[{'a':'1', 'b':'2'},{'a':'3','b':'4'}]
</code></pre>
<p>How do i read this into python as a list of dictionaries? </p>
</div>
<div class="post-text" itemprop="text">
<p>You can also use <code>ast.literal_eval</code>.</p>
<pre><code>import ast
ast.literal_eval("[{'a':'1', 'b':'2'},{'a':'3','b':'4'}]")
</code></pre>
<blockquote>
<p>Help on function literal_eval in module ast:</p>
<p>literal_eval(node_or_string)
      Safely evaluate an expression node or a string containing a Python
      expression.  The string or node provided may only consist of the  following
      Python literal structures: strings, bytes, numbers, tuples, lists, dicts,    sets, booleans, and None.</p>
</blockquote>
<p>I learnt this trick at <a href="https://stackoverflow.com/a/21154138/1415639">https://stackoverflow.com/a/21154138/1415639</a></p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>literal_eval</code> from <code>ast</code> which is a safer solution of eval. 
You can see why you shouldn't use <code>eval</code> <a href="https://stackoverflow.com/questions/1832940/is-using-eval-in-python-a-bad-practice">here</a> and <a href="https://stackoverflow.com/questions/1933451/why-should-exec-and-eval-be-avoided">here</a>.</p>
<p>So what you want would be:</p>
<pre><code>from ast import literal_eval


with open('file.txt') as f:
    dictionaries = literal_eval(f.read().strip()) 
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/functions.html#eval" rel="nofollow noreferrer">eval</a>:</p>
<pre><code>with open('filename') as f:
    d = eval(f.read())
</code></pre>
<p>Bear in mind that <code>eval</code> should be used with care, as mentioned in the comments, because you may end up executing harmful code if the input is not 100% in your control.</p>
</div>
