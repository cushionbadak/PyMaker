<p>I have a dictionary that looks something like this:</p>
<pre><code>d = {
    'last_price': '760.54',
    'change': 'N/A',
    'days_range': '760.00 - 775.00',
    'previous_close': '771.23',
    'change_percent': '-1.39%', 
}
</code></pre>
<p>All of the values in this dictionary are subject to change. The numbers can vary widely from single digits to numbers several digits long. I would like to format this information and consistently present it like this:</p>
<pre><code>Last:            $760.54         Change:         N/A (-1.39%)
Prev Close:      $771.23         Day's Range:    $760.00 - $775.00
</code></pre>
<p>So far, I've been trying to do this by doing this:</p>
<pre><code>print "\
Last: {last_price:&gt;17} \t Change: {change:&gt;11} ({change_percent}) \n\
Prev Close: {previous_close:&gt;11} \t Day's Range: {days_range:&gt;18}\
".format(**d)
</code></pre>
<p>This produces output that looks like this:</p>
<pre><code>Last:            760.54          Change:         N/A (-1.39%)
Prev Close:      771.23          Day's Range:    760.00 - 775.00
</code></pre>
<p>While this works well enough if the numbers don't change, if "days_range" is set to a value like "78.91 - 79.91" it will jump further to the right. For example:</p>
<pre><code>Last:             79.21          Change:         N/A (-1.19%)
Prev Close:       80.16          Day's Range:      78.91 - 79.91
</code></pre>
<p>I also can't say I'm very content with using "\t" to separate the columns, as I am concerned that if the values in the first column are too long this can cause the right column to get pushed further right than desired.</p>
<p>Is there an easy and elegant way to do this? I would prefer a solution that does not depend on nonstandard libraries and is relatively straightforward. </p>
<p>I'd also like to know an easy way to prepend values with dollar signs without having to modify the values in the dictionary.</p>

<div class="post-text" itemprop="text">
<p>You can change it to align to the left and it worked for me (I used <code>ljust()</code> but the way you're doing it with format should also work):</p>
<pre><code>def print_data(d,spaces):
    print('Last:'.ljust(spaces) + d['last_price'].ljust(spaces) + \
          'Change:'.ljust(spaces) + d['change'] + ' (' + d['change_percent'] + ')')
    print('Prev close:'.ljust(spaces) + d['previous_close'].ljust(spaces) +
          "Day's Range:".ljust(spaces) + d['days_range'])

data = {
    'last_price': '760.54',
    'change': 'N/A',
    'days_range': '760.00 - 775.00',
    'previous_close': '771.23',
    'change_percent': '-1.39%', 
}
print_data(data,15)

print()

data['days_range'] = '78.91 - 79.91'
print_data(data,15)
</code></pre>
<p>As to your second question about prepending the dollar signs, everything except the "Day's Range" should be easy. For "Day's Range" I can't think of an easier way than adding one to the front and splitting the string down <code>' - '</code> and adding one to the second half.</p>

<div class="post-text" itemprop="text">
<p>In Python 3 and newer version, you may do it via <a href="https://docs.python.org/3/library/stdtypes.html#str.format" rel="nofollow noreferrer"><code>str.format()</code></a> as:</p>
<pre><code>print('{:&lt;15} {:&lt;15} {:&lt;15} {:&lt;15}'.format('Last:', d['last_price'], 'Change:', '{} ({})'.format(d['change'], d['change_percent'])))
#         ^ Note this
</code></pre>
<p>which prints:</p>
<pre><code> Last:           760.54          Change:         N/A (-1.39%)
</code></pre>
<p>Here, <em><code>&lt;</code> means that you want the content to be left align from the cursor position you mentioned where as <code>&gt;</code> means right aligned.</em></p>
<p>Here is the result of data you are printing when printed simultaneously:</p>
<pre><code>&gt;&gt;&gt; '{:&lt;15} {:&lt;15} {:&lt;15} {:&lt;15}'.format('Prev Close:', 771.23, 'Day\'s Range:', '760.00 - 775.00')
"Prev Close:     771.23          Day's Range:    760.00 - 775.00"
&gt;&gt;&gt; '{:&lt;15} {:&lt;15} {:&lt;15} {:&lt;15}'.format('Prev Close:', 80.16, 'Day\'s Range:', '78.91 - 79.91')
"Prev Close:     80.16           Day's Range:    78.91 - 79.91  "
</code></pre>
<p>All the coulmns are left aligned.</p>
<hr/>
<p>Note: Above sysntax for <em>string formatting</em> is only available from Python 3 and newer. In older versions you have to do it with <code>%s</code> as:</p>
<pre><code>'%-15s %-15s %-15s %-15s' % ('Change: ', d['last_price'], 'Change:', '{} ({})'.format(d['change'], d['change_percent']))
#       ^  '-'ive value means align left
</code></pre>
<p>which prints:</p>
<pre><code>'Change:         760.54          Change:         N/A (-1.39%)   '
</code></pre>
