<div class="post-text" itemprop="text">
<p>Only <code>identifier</code>(s) can follow <a href="https://docs.python.org/3/reference/simple_stmts.html#the-global-statement" rel="nofollow noreferrer">the global statement</a>:</p>
<blockquote>
<p><code>global_stmt ::=  "global" identifier ("," identifier)*</code></p>
</blockquote>
<p>where identifiers are plain names such as:</p>
<pre><code>a = 20
b = "string" 
global a, b
</code></pre>
<p>when you break that requirement and python isn't able to find an identifier after <code>global</code>, a <code>SyntaxError</code> is going to get raised.</p>
<p>What you've entered after the <code>global</code> keyword isn't an identifier; it's <a href="https://docs.python.org/3/reference/expressions.html#calls" rel="nofollow noreferrer">a function call</a> which is syntactically recognized by the <code>()</code> after the identifier.</p>
