<p>As the comment on your question says, you are using ArraySize in the wrong place, and assigning <code>c</code> in a way that doesn't do anything useful, you aren't updating List2.</p>
<p>And there's no need to keep a counter for the index when you can use <a href="https://docs.python.org/3/library/functions.html#enumerate" rel="nofollow noreferrer"><code>enumerate()</code></a> (also: <a href="https://stackoverflow.com/questions/22171558/what-does-enumerate-mean">What does enumerate mean?</a> ) to do that for you, e.g.</p>
<pre><code>List1 = [0, 0, ...]
List2 = []

for index, value in enumerate(List1):
  if index &gt; 0 and List1[index-1] != value:
    List2.append(index)
</code></pre>
<p>Output is</p>
<pre><code>&gt; List2
=&gt; [93, 218, 219, 307, 526, 692, 1106, 1856, 2043]
</code></pre>
<p><a href="https://repl.it/E5p9/1" rel="nofollow noreferrer">Try online at http://repl.it</a></p>
</div>
<div class="post-text" itemprop="text">
<p>In order to achieve this, you may use <a href="https://docs.python.org/2/library/functions.html#zip" rel="nofollow noreferrer"><code>zip()</code></a> and <a href="https://docs.python.org/2/library/functions.html#enumerate" rel="nofollow noreferrer"><code>enumerate()</code></a> along with conditional <em>list comprehension</em> expression as:</p>
<pre><code>&gt;&gt;&gt; [n for n, (i, j) in enumerate(zip(my_list, my_list[1:]), 1) if i!=j]
[93, 218, 219, 307, 526, 692, 1106, 1856, 2043]
</code></pre>
<hr/>
<p><strong>Issue with you code:</strong> </p>
<p>You are assigning <code>ArraySize = len(List1)</code> which will hold <code>int</code> value. Then later you are doing <code>ArraySize[c]</code> i.e accessing it like a list. But since it hold integer value, you are getting the error:</p>
<pre><code>TypeError: 'int' object has no attribute '__getitem__'
#         Function called by list when you ^
#         try to access the items based on index
</code></pre>
