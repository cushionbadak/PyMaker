<p><code>maketrans</code> is deprecated in favor of new static methods</p>
<blockquote>
<p>The string.maketrans() function is deprecated and is replaced by new static methods, bytes.maketrans() and bytearray.maketrans(). This change solves the confusion around which types were supported by the string module. Now, str, bytes, and bytearray each have their own maketrans and translate methods with intermediate translation tables of the appropriate type.</p>
</blockquote>
<p>You can use <code>dir()</code> to verify that whenever you have this kind of issue:</p>
<pre><code>&gt;&gt;&gt; import string
&gt;&gt;&gt;
&gt;&gt;&gt; dir(string)
['Formatter', 'Template', '_ChainMap', '_TemplateMetaclass', '__all__', '__builtins__', '__cached__', '__doc__', '__file__', '__loader__', '__name__', '__package__', '__spec__', '_re', '_string', 'ascii_letters', 'ascii_lowercase', 'ascii_uppercase', 'capwords', 'digits', 'hexdigits', 'octdigits', 'printable', 'punctuation', 'whitespace']
&gt;&gt;&gt;
</code></pre>
<p>As you can see, there is no <code>maketrans</code> in the resulted list above.</p>

<span class="comment-copy">In Python 3 <code>maketrans</code> is a method of <code>str</code>.</span>
<span class="comment-copy">The error I get now is:                                                                                                          <code>&gt;&gt;&gt; for line in f: 	line=line.split() 	for word in line: 		word=word.lower() 		new_word = word.translate(str.maketrans("",""), string.punctuation) 		if new_word in genesis: 			genesis[new_word]+=1 		else: 			genesis[new_word]=1</code> <code>Traceback (most recent call last):   File "&lt;pyshell#67&gt;", line 5, in &lt;module&gt;     new_word = word.translate(str.maketrans("",""), string.punctuation) TypeError: translate() takes exactly one argument (2 given)</code></span>
<span class="comment-copy">You need to remove <code>string.punctuation</code></span>
