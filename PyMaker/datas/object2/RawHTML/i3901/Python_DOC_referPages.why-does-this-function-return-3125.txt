<p>Today I saw a function in a python textbook, the code is below:</p>
<pre><code>def power(x,y):
    return pow(x,y)
</code></pre>
<p>When I gave params to the <code>power</code> like below:</p>
<pre><code>&gt;&gt;&gt;params=(5,)*2
&gt;&gt;&gt;power(*params)  
3125
</code></pre>
<p>The function returned me a 3125. I know 3125 may be came from (5*5)*(5*5)*5,but I don't understand how working is the params in the code. What the meaning of the expression <code>(5,)*2</code>?</p>
</div>
<div class="post-text" itemprop="text">
<p>That's a concatenation of a tuple.</p>
<pre><code>&gt;&gt;&gt;(5,)*2
(5,5)
&gt;&gt;&gt;(5,)*5
(5,5,5,5,5)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If you inspect the value of <code>params</code>, you can see how it works:</p>
<pre><code>&gt;&gt;&gt; params=(5,)*2
&gt;&gt;&gt; params
(5, 5)
&gt;&gt;&gt; power(*params)
3125
</code></pre>
<p><code>*n</code> applied to a sequence returns that sequence appended to itself <code>n</code> times (<a href="https://docs.python.org/3/library/stdtypes.html#typesseq-common" rel="nofollow noreferrer">see documentation</a>).</p>
</div>
<div class="post-text" itemprop="text">
<p>Note that <code>(5,)*2</code> expands to <code>(5, 5)</code>, aka is a <em>tuple</em>.  See the docs on <a href="https://docs.python.org/3.3/tutorial/datastructures.html#tuples-and-sequences" rel="nofollow noreferrer">tuples</a> . </p>
<p>Re <code>pow()</code>: according to the <a href="https://docs.python.org/2/library/functions.html#pow" rel="nofollow noreferrer">docs</a>, <code>pow(x,y)</code> returns <em>x to the power y</em>: </p>
<p>In your script, you use the * in <code>power(*params)</code> to insert the two items in the <code>params</code> tuple as individual parameters into the function. You could have achieved the same by typing <code>power(5,5)</code>. </p>
<p>Thus, 5^5, which equals 3125. </p>
<p>A intuitive tutorial on tuples is <a href="https://www.tutorialspoint.com/python/python_tuples.htm" rel="nofollow noreferrer">here</a>.</p>
</div>
