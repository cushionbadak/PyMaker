<p>The logging level you specify in your configuration is the <strong>minimum</strong> level will be logged. Just to recap, here the predefined levels in order of importance (from lowest to maximum):</p>
<ul>
<li><code>DEBUG</code></li>
<li><code>INFO</code></li>
<li><code>WARNING</code></li>
<li><code>ERROR</code></li>
<li><code>CRITICAL</code></li>
</ul>
<p>So, if you put <code>INFO</code> in your configuration you will see all logs marked as <code>INFO</code>, <code>WARNING</code>, <code>ERROR</code>, and <code>CRITICAL</code>.</p>
<p>If you really want to see <strong>only</strong> <code>INFO</code> and <code>ERROR</code> you can try to write some custom code, but I discourage it as it smells as a bad design.</p>
<p><em>References</em>:</p>
<ul>
<li><a href="https://docs.djangoproject.com/en/dev/topics/logging/" rel="nofollow noreferrer">https://docs.djangoproject.com/en/dev/topics/logging/</a></li>
<li><a href="https://docs.python.org/3/library/logging.html?highlight=logging#module-logging" rel="nofollow noreferrer">https://docs.python.org/3/library/logging.html?highlight=logging#module-logging</a></li>
</ul>
<h3>Edit</h3>
<p>Here how your configuration would look like:</p>
<pre><code>LOGGING = {
    'version': 1,
    'disable_existing_loggers': True,
    'root': {
