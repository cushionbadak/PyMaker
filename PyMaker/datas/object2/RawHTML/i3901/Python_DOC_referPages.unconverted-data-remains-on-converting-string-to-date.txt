<p>A simple code as below always gives me an error complaining about <code>unconverted data remains: 16</code></p>
<pre><code>print (date)
print (type(date))
print (len(date))
dt = datetime.strptime(date, "%d/%m/%y")
</code></pre>
<p>The prints output 14/11/2016
, str and 10</p>
<p>What is wrong with this code and why does it complain about 16?</p>
</div>
<div class="post-text" itemprop="text">
<p>Quoting <a href="https://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow noreferrer">the documentation</a>, <code>%y</code> is "<em>Year without century as a zero-padded decimal number.    00, 01, …, 99</em>". This conversion requires exactly two characters. Thus your conversion eats <code>20</code> but leaves <code>16</code> unconverted.</p>
<p>You want to use <code>%Y</code>, "<em>Year with century as a decimal number.  0001, 0002, …, 2013, 2014, …, 9998, 9999</em>".</p>
<pre><code>dt = datetime.strptime(date, "%d/%m/%Y")
</code></pre>
