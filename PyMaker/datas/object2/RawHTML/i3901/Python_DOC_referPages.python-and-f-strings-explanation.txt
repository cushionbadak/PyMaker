<p>In the follow f-string, </p>
<pre><code>print("It's fleece was white as {}.".format('snow'))
</code></pre>
<p>Is <code>'snow'</code> a variable? I am confused with what exactly it would be judged as.</p>
</div>
<div class="post-text" itemprop="text">
<p>No, <code>'snow'</code> is a string literal, an expression that produces a string value. <code>snow</code> would be a variable name (note the lack of quotes).</p>
<p>Compare:</p>
<pre><code>&gt;&gt;&gt; 'snow'
'snow'
&gt;&gt;&gt; snow
Traceback (most recent call last):
  File "&lt;stdin&gt;", line 1, in &lt;module&gt;
NameError: name 'snow' is not defined
&gt;&gt;&gt; snow = 42
&gt;&gt;&gt; snow
42
&gt;&gt;&gt; snow = 'snow'
&gt;&gt;&gt; snow
'snow'
</code></pre>
<p>The variable snow at first wasn't yet assigned to, so trying to use it caused an exception. I then assigned an integer to the name, and then the string with value <code>'snow'</code>.</p>
<p>Formatting a string literal with another string literal is pretty meaningless. You'd <em>normally</em> use an actual variable, so you can vary the output produced:</p>
<pre><code>compared_to = 'snow'
print("It's fleece was white as {}.".format(compared_to))
</code></pre>
<p>Also, that's not an <code>f</code> string. An <code>f</code> string literal starts with a <code>f</code> character. What you have here is a regular, run-of-the-mill string literal, and a call to the <a href="https://docs.python.org/3/library/stdtypes.html#str.format" rel="nofollow noreferrer"><code>str.format()</code> method</a>. The following is the equivalent expression using an f-string:</p>
<pre><code>print(f"It's fleece was white as {'snow'}.")
</code></pre>
<p>See <a href="https://stackoverflow.com/questions/35745050/string-with-f-prefix-in-python-3-6">String with 'f' prefix in python-3.6</a> for more information on actual f-strings. </p>
</div>
