<div class="post-text" itemprop="text">
<p>There are many solutions to your question, however, I believe the easiest would be to adjust the <code>daterange(d, d1)</code> function, by simply adding 1 to the <code>range(int ((d1 - d).days))</code>.</p>
<pre><code>def daterange(d, d1):
    for n in range(int ((d1 - d).days) + 1):
        yield d + timedelta(n)
</code></pre>
<p>The reason for this is that, as per <a href="https://docs.python.org/3/library/stdtypes.html#range" rel="nofollow noreferrer">documentation</a>, <code>range(stop)</code> does not output the stop value, but only the values 'before' it.</p>
</div>
<span class="comment-copy">...change <code>range(int ((d1 - d).days))</code> to <code>range(int ((d1 - d).days) + 1)</code>?</span>
<span class="comment-copy"><code>range</code> is non-inclusive on the "max" value, meaning <code>range(10)</code> won't actually include <code>10</code>. If you want the value to be included, you have to add <code>1</code> to your range-max.</span>
