<div class="post-text" itemprop="text">
<p>Time zone in format <code>-05:00</code>, i.e. with a colon, is not supported by the <code>datetime</code> module.</p>
<p>But you can use <code>dateutil.parser</code> or <code>pandas</code>:</p>
<pre><code>from dateutil import parser
import pandas as pd

x = '2017-01-12T14:12:06.000-05:00'

res1 = parser.parse(x)
res2 = pd.to_datetime(x).to_pydatetime()

print(res1)  # 2017-01-12 14:12:06-05:00
print(res2)  # 2017-01-12 19:12:06
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The <a href="https://docs.python.org/3/library/datetime.html#strftime-and-strptime-behavior" rel="nofollow noreferrer">documentation for strptime</a>, for the <code>%z</code> format specifier:</p>
<pre><code>UTC offset in the form +HHMM or -HHMM 
</code></pre>
<p>If I run your second example in my Python shell (Python 3.6.5), I get a ValueError:</p>
<pre><code>ValueError: time data '2017-01-12T14:12:06.000-05:00' does not match format '%Y-%m-%dT%H:%M:%S.%f%z'
</code></pre>
<p>Based on that, offsets with <code>:</code> in are not valid. You can have to handle this yourself, or use another parsing library</p>
