<p>What are the differences between the <code>__str__()</code> and <code>str()</code> methods in python?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>__str__</code> (usually read dunder, for  double under) is an instance method that is called whenever you run <code>str(&lt;object&gt;)</code> and returns the string representation of the object.</p>
<p><code>str(foo)</code> acts as a function trying to convert <code>foo</code> into a string.</p>
<p><strong>Note:</strong><br/>
There is also a <code>__repr__()</code> method which is fairly similar to <code>__str__()</code>, the main difference being <code>__repr__</code> should return an <em>unambiguous</em> string and <code>__str__</code> is for a <em>readable</em> string. For a great response on the diffences between the two I'd suggest giving <a href="https://stackoverflow.com/a/2626364/4225467">this answer</a> a read.</p>
</div>
<div class="post-text" itemprop="text">
<p><code>__str__()</code> is a magic instance method that doee this: when you print a class instance variable with <code>print()</code>, it will give you a string that can be modified by changing the returned string in the <code>__str__()</code> method. There's probably a better explanation to it but I can show you with code:</p>
<pre><code>class Thing:
    def __init__(self):
        pass
    def __str__(self):
       return "What do you want?"  #always use return

a = Thing()
print(a)
</code></pre>
<p><strong>OUTPUT:</strong></p>
<pre><code>What do you want?
</code></pre>
<p><code>str()</code> just converts a variable into a string type variable.</p>
<pre><code>print(str(12.0))
</code></pre>
<p><strong>OUTPUT:</strong></p>
<pre><code>'12.0'
</code></pre>
<p>You can confirm it is a string using the <code>type()</code> function.</p>
<pre><code>print(type(str(12.)))
</code></pre>
<p>I don't know the exact output of that but it will peobably have <code>'str'</code> in it.</p>
