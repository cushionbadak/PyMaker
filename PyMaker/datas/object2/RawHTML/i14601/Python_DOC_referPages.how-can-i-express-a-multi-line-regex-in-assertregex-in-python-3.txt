<div class="post-text" itemprop="text">
<p>I'm using the <a href="https://docs.python.org/3/library/unittest.html#unittest.TestCase.assertRegex" rel="nofollow">assertRegex() function</a> from the <a href="https://docs.python.org/3/library/unittest.html" rel="nofollow"><code>unittest</code></a> library in Python 3.4. It only takes two relevant parameters: the text to match against and the regex. The seems to prevent me from expressing any regex-related constants such as <a href="https://docs.python.org/3.4/library/re.html#re.MULTILINE" rel="nofollow">re.MULTILINE</a>.</p>
<p>How can I construct or specify a multi-line regex and use it with <code>assertRegex()</code>? Alternatively, is there a related way of solving my problem?</p>
</div>
<div class="post-text" itemprop="text">
<p>See <a href="https://docs.python.org/3.4/library/re.html#regular-expression-syntax" rel="nofollow">here</a> in the Python docs:</p>
<pre><code>(?iLmsux)
</code></pre>
<blockquote>
<p>(One or more letters from the set 'i', 'L', 'm', 's', 'u', 'x'.) The group matches the empty string; the letters set the corresponding flags: re.I (ignore case), <code>re.L</code> (locale dependent), <code>re.M</code> (multi-line), <code>re.S</code> (dot matches all), <code>re.U</code> (Unicode dependent), and <code>re.X</code> (verbose), for the entire regular expression. (The flags are described in Module Contents.) This is useful if you wish to include the flags as part of the regular expression, instead of passing a flag argument to the <code>re.compile()</code> function.</p>
<p>Note that the <code>(?x)</code> flag changes how the expression is parsed. It should be used first in the expression string, or after one or more whitespace characters. If there are non-whitespace characters before the flag, the results are undefined.</p>
</blockquote>
