<div class="post-text" itemprop="text">
<p>If you look at the documentation for the <a href="https://docs.python.org/3/library/functions.html?highlight=print#print" rel="nofollow"><code>print</code></a>-function. It takes a variable numbers to print out separated by the given <code>sep</code> and ended with <code>end</code>. So you can simply use the <code>*syntax</code> to pass your sequence to <code>print</code>. Simply:</p>
<pre><code>&gt;&gt;&gt; print(*range(1,5))
1 2 3 4
&gt;&gt;&gt; print(*range(1,5), sep='+')
1+2+3+4
&gt;&gt;&gt; print(*range(1,5), sep='+', end='=' + str(sum(range(1,5))))
1+2+3+4=10
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You're using <code>numbers</code> in a for loop. Even though the variable name is plural, it only gets one value at a time.  i.e., if <code>s_num==1</code> and <code>e_num==2</code>, the for loop will be unfolded to the following:</p>
<pre><code>numbers=1
print(1, sep='')
numbers=2
print(2, sep='')
</code></pre>
<p>As you see, <code>numbers</code> takes a single value from the range each time through the for loop, not the whole range. <code>sep</code> is used to separate multiple values being printed out from a variable list of arguments passed to the print function. Since there is only a single value being printed out in each print statement, there is nothing to separate so setting <code>sep</code> does nothing.</p>
</div>
