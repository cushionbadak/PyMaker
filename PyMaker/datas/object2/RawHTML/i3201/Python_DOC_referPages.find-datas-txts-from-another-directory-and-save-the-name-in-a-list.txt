file_list = glob.glob("&lt;path_to_your_directory&gt;/*.txt")

<div class="post-text" itemprop="text">
<p>You could use <code>os</code>, <code>shlex</code> and <code>subprocess</code> combined to achieve this:</p>
<pre><code>import os
from subprocess import PIPE, Popen
import shlex

#Change directory to where you want to start looking
os.chdir("the_path_from where_you_want_to_start_looking")

#Shell command for searching for .txt files
command = 'find . -name "*.txt"'

#Run command in shell
args = shlex.split(command)
process = Popen(args, stdout=PIPE, stderr=PIPE)

#Get output of command
result = process.communicate()[0]

#result is bytes object so need to parse it to list
list_of_files = str(result).replace("b'", "").split("\\n")

#list_of_files will be desired list
</code></pre>
<p>Hope this helps.</p>

<div class="post-text" itemprop="text">
<p>You can use <a href="https://docs.python.org/3/library/subprocess.html#subprocess.check_output" rel="nofollow noreferrer"><code>subprocess.check_output()</code></a> to run a Linux <a href="http://man7.org/linux/man-pages/man1/find.1.html" rel="nofollow noreferrer"><code>find</code></a> command on a directory and collect the files ending with <em>.txt</em>:</p>
<pre><code>from subprocess import check_output

# shell command
# remove -printf "%f\n" if you want full paths to files
command = 'find &lt;PATH&gt; -type f -name *.txt -printf "%f\n"'

# get contents returned from command
command_str = check_output(command, shell=True).decode('utf-8')

# split string on newlines to get list of filenames
files = command_str.split()
</code></pre>
<p>Where <code>&lt;PATH&gt;</code> is the relative/absolute path to your directory containing the files. </p>
<p><strong>Note:</strong> Since <code>check_output()</code> encodes your content in <a href="https://docs.python.org/3/library/stdtypes.html#bytes" rel="nofollow noreferrer">bytes</a>, you need to use <a href="https://docs.python.org/3/library/stdtypes.html#bytes.decode" rel="nofollow noreferrer"><code>bytes.decode()</code></a> to decode it to a string instead. Then you can split this string to collect the filenames into a list. </p>
<p>If you want to find out more about <code>find</code>, type <code>man find</code> into the terminal. </p>
