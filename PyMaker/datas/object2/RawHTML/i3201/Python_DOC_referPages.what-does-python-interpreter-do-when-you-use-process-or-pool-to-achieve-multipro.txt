<div class="post-text" itemprop="text">
<p>What does python interpreter do when you use Process or Pool to achieve multiprocessing. Are all the codes and global variables created copied to the child process?</p>
</div>
<div class="post-text" itemprop="text">
<p>It depends on the Operating System and on the start method you choose.</p>
<p>You can find a description of start methods in the <a href="https://docs.python.org/3/library/multiprocessing.html#contexts-and-start-methods" rel="nofollow noreferrer">multiprocessing documentation</a>.</p>
<p>Unix Operating Systems generally support all three methods whereas NT Operating Systems (Windows) only support the <code>spawn</code> method.</p>
</div>
<span class="comment-copy">Yes, they are copied. Theoretically, depending on the OS, you could have copy-on-write behavior, but in Python, it is almost impossible not to write since CPython uses reference counting, so just assigning a value will cause the copy.</span>
