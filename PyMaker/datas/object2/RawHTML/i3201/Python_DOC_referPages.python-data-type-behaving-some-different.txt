<p>Basically <code>() small brackets</code> are block enclosing brackets but eventually this represents tuple too in <code>python</code>. whenever we declare a tuple into python we need to comma separate 2 values or assign <code>()</code> (small opening and closing brackets, this will create an empty tuple).</p>
<p>So in your <code>demo3</code> you made a <code>list</code> and enclosed with brackets that means your <code>demo3</code> have a <code>list vector</code>. If you provide <code>comma</code> after <code>[]</code> just like <code>([], )</code> it will make a tuple of <code>lists</code>.</p>

<div class="post-text" itemprop="text">
<p>To create an empty tuple, use <code>tuple()</code>. It will <em>print</em> as <code>()</code>, but that's not how you create an empty tuple. As minimum, a tuple must contain one item. So you need to create it as <code>([],)</code>.</p>
<p>Your code <code>([])</code> is the same as <code>[]</code>, the parenthesis in this case act as <em>continuation characters</em>` and not an empty tuple.</p>
<blockquote>
<p>Expressions in parentheses, square brackets or curly braces can be
  split over more than one physical line without using backslashes.</p>
</blockquote>
<p><em>See <a href="https://docs.python.org/3/reference/lexical_analysis.html#implicit-line-joining" rel="nofollow noreferrer">the documentation on lexical analysis</a> for more details.</em></p>
<p>You can also verify this at the interpreter:</p>
<pre><code>&gt;&gt;&gt; ([]) == []
True
</code></pre>
</div>
