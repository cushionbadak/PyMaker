<p>I want to classify a list of string in Python depending on whether they are upper case, lower case, or mixed case</p>
<p>How can I do this?</p>
</div>
<div class="post-text" itemprop="text">
<p>There are a number of "is methods" on strings. <a href="https://docs.python.org/library/stdtypes.html#str.islower"><code>islower()</code></a> and <a href="https://docs.python.org/library/stdtypes.html#str.isupper"><code>isupper()</code></a> should meet your needs:</p>
<pre><code>&gt;&gt;&gt; 'hello'.islower()
True

&gt;&gt;&gt; [m for m in dir(str) if m.startswith('is')]
['isalnum', 'isalpha', 'isdigit', 'islower', 'isspace', 'istitle', 'isupper']
</code></pre>
<p>Here's an example of how to use those methods to classify a list of strings:</p>
<pre><code>&gt;&gt;&gt; words = ['The', 'quick', 'BROWN', 'Fox', 'jumped', 'OVER', 'the', 'Lazy', 'DOG']
&gt;&gt;&gt; [word for word in words if word.islower()]
['quick', 'jumped', 'the']
&gt;&gt;&gt; [word for word in words if word.isupper()]
['BROWN', 'OVER', 'DOG']
&gt;&gt;&gt; [word for word in words if not word.islower() and not word.isupper()]
['The', 'Fox', 'Lazy']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>I want to give a shoutout for using <code>re</code> module for this. Specially in the case of case sensitivity. </p>
<p>We use the option <a href="https://docs.python.org/3/library/re.html#re.IGNORECASE" rel="nofollow noreferrer">re.IGNORECASE</a> while compiling the regex for use of in production environments with large amounts of data.</p>
<pre><code>&gt;&gt;&gt; import re
&gt;&gt;&gt; m = ['isalnum','isalpha', 'isdigit', 'islower', 'isspace', 'istitle', 'isupper', 'ISALNUM', 'ISALPHA', 'ISDIGIT', 'ISLOWER', 'ISSPACE', 'ISTITLE', 'ISUPPER']
&gt;&gt;&gt;
&gt;&gt;&gt;
&gt;&gt;&gt; pattern = re.compile('is')
&gt;&gt;&gt;
&gt;&gt;&gt; [word for word in m if pattern.match(word)]
['isalnum', 'isalpha', 'isdigit', 'islower', 'isspace', 'istitle', 'isupper']
</code></pre>
<p>However try to always use the <code>in</code> operator for string comparison as detailed in this post </p>
