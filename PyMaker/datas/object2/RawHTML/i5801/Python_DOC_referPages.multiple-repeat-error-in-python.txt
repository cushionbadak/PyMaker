<p>I am trying some regexes in python</p>
<pre><code>re.compile('in versions: (.+?) of '+name+' ')
</code></pre>
<p>and, if name is 'libcrypto++', cause <strong>multiple repeat error</strong></p>
<p>how can i handle it only in a string..?</p>
</div>
<div class="post-text" itemprop="text">
<p><code>+</code> is a quantifier in regex. So when you add <code>libcrypto++</code> to regex string, it brings two of them alongside that doesn't make sense.
See <a href="http://www.rexegg.com/regex-quickstart.html#quantifiers" rel="nofollow noreferrer">this.</a></p>
<p>To solve this you can use regex <a href="https://docs.python.org/3/library/re.html#re.escape" rel="nofollow noreferrer">escape</a> method, like:</p>
<pre><code>re.compile('in versions: (.+?) of '+ re.escape(name) +' ')
</code></pre>
</div>
<span class="comment-copy">use <code>re.escape(name)</code> since it contains <code>+</code> characters which have otherwise special meaning in the context of regular expressions</span>
