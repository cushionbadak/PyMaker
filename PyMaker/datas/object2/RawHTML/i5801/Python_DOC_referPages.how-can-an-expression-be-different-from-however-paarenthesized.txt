<p>Consider this statement:</p>
<pre class="lang-python prettyprint-override"><code>&gt; False == False in [False]
True
</code></pre>
<p>However:</p>
<pre class="lang-python prettyprint-override"><code>&gt; (False == False) in [False]
False
&gt; False == (False in [False])
False
</code></pre>
<p>It is incredible. What's the reason and the interpretation?</p>
</div>
<div class="post-text" itemprop="text">
<p><a href="https://docs.python.org/3/library/dis.html" rel="noreferrer">dis.dis</a> to the rescue and a bit of my interpretation.</p>
<p>Anyway, it looks like this. Consider three expression <code>X, Y, Z</code> and two operators <code>O1, O2</code>. Then</p>
<pre><code>X O1 Y O2 Z
</code></pre>
<p>is equivalent to</p>
<pre><code>(X O1 Y) and (Y O2 Z)
</code></pre>
<p>as can be seen by</p>
<pre><code>a &lt; b &lt; c
</code></pre>
<p>example. I find this behaviour very counterintuitive.</p>
<p>And <a href="https://docs.python.org/3/reference/expressions.html#comparisons" rel="noreferrer">the docs</a>:</p>
<blockquote>
<p>Comparisons can be chained arbitrarily, e.g., x &lt; y &lt;= z is equivalent to x &lt; y and y &lt;= z, except that y is evaluated only once (but in both cases z is not evaluated at all when x &lt; y is found to be false).</p>
</blockquote>
<p>Note that <code>in</code> is a comparison operator.</p>
