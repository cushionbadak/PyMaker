<div class="post-text" itemprop="text">
<p>Considering tkinter <a href="http://effbot.org/tkinterbook/variable.htm" rel="nofollow noreferrer">variable class'</a> <code>trace()</code> method is <a href="https://docs.python.org/3/whatsnew/3.6.html" rel="nofollow noreferrer">deprecated</a> since Python 3.6, new <code>trace_add()</code> method should be used:</p>
<ul>
<li><code>trace('w', method)</code> has become <code>trace_add('write', method)</code></li>
<li><code>trace('r', method)</code> has become <code>trace_add('read', method)</code></li>
</ul>
<p>I'm wondering how is the <code>'u'</code> argument now specified? 
For instance, consider the former:</p>
<p><code>trace('wu', method)</code></p>
<p>I can't find anything in the docs or anywhere else.</p>
</div>
<div class="post-text" itemprop="text">
<p>In general, in such cases you can always fire up your REPL and call <code>help</code> on <code>trace_add</code> to examine its docstring.</p>
<pre><code>&gt;&gt;&gt; from tkinter import StringVar
&gt;&gt;&gt; help(StringVar.trace_add)
</code></pre>
<p>For <code>trace_add</code> the following is stated:</p>
<blockquote>
<p><code>trace_add(self, mode, callback)</code></p>
<p>Define a trace callback for the variable.</p>
<p><em>Mode is one of "read", "write", "unset", or a list or tuple of
     such strings.</em>
     Callback must be a function which is called when the variable is
     read, written or unset.</p>
<p>Return the name of the callback.</p>
</blockquote>
<p>So, judging from that, <code>'wu'</code> should now be <code>['write', 'unset']</code>.</p>
</div>
