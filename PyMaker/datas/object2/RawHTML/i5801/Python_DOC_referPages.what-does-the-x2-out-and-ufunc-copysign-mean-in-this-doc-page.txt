<div class="post-text" itemprop="text">
<p>The brackets are standard Python documentation syntax for optional parameters to a function call. From the <a href="https://docs.python.org/3/reference/introduction.html#notation" rel="nofollow noreferrer">Python Language Reference Introduction</a>:</p>
<blockquote>
<p>a phrase enclosed in square brackets (<code>[ ]</code>) means zero or one occurrences (in other words, the enclosed phrase is optional)</p>
</blockquote>
<p>You'll notice it all over the place in Python &amp; its libraries' documentation.</p>
<p>The <code>= &lt;ufunc 'func_name'&gt;</code> bit is to let you know that the function is an instance of the <code>numpy.ufunc</code> class. From the <a href="https://docs.scipy.org/doc/numpy-1.13.0/reference/ufuncs.html" rel="nofollow noreferrer">NumPy docs on Universal Functions</a>:</p>
<blockquote>
<p>A universal function (or ufunc for short) is a function that operates
  on <code>ndarrays</code> in an element-by-element fashion, supporting array
  broadcasting, type casting, and several other standard features. That
  is, a ufunc is a “vectorized” wrapper for a function that takes a
  fixed number of scalar inputs and produces a fixed number of scalar
  outputs.</p>
<p>In NumPy, universal functions are instances of the <code>numpy.ufunc</code> class. Many of the built-in functions are implemented in compiled C code, but <code>ufunc</code> instances can also be produced using the <code>frompyfunc</code> factory function.</p>
</blockquote>
</div>
<span class="comment-copy">In regards to <code>[, out]</code> you will notice in the documentation: "out : ndarray, optional". Optional arguments are presented in square brackets. This is common to python in general. The representation gets a bit messy if they have to be supplied in order, for example <code>datetime.replace([year[, month[, day[, hour[, minute[, second[, microsecond[, tzinfo]]]]]]]])</code>.</span>
