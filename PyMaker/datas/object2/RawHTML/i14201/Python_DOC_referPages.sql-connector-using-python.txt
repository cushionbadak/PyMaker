<p>Consider this SQLite database.</p>
<pre><code>$ sqlite3 so.sqlite3 .dump 
PRAGMA foreign_keys=OFF;
BEGIN TRANSACTION;
CREATE TABLE t (f1 integer, f2 text);
INSERT INTO "t" VALUES(1,'foo');
INSERT INTO "t" VALUES(2,'bar');
INSERT INTO "t" VALUES(3,'baz');
COMMIT;
</code></pre>
<p>Python connects and queries this database like this.</p>
<pre><code>import sqlite3

con = sqlite3.connect("so.sqlite3")
cur = con.cursor()
cur.execute("select * from t where f1 = 2")
print(cur.fetchone())
</code></pre>
<p>Output:</p>
<pre><code>(2, 'bar')
</code></pre>
<p>You have to use one of <a href="https://docs.python.org/3/library/sqlite3.html#sqlite3.Cursor.fetchone" rel="nofollow"><code>cur.fetchone()</code></a>, <a href="https://docs.python.org/3/library/sqlite3.html#sqlite3.Cursor.fetchall" rel="nofollow"><code>cur.fetchall()</code></a>, or <a href="https://docs.python.org/3/library/sqlite3.html#sqlite3.Cursor.fetchmany" rel="nofollow"><code>cur.fetchmany()</code></a> to get rows from the cursor. Just doing <a href="https://docs.python.org/3/library/sqlite3.html#sqlite3.Cursor.execute" rel="nofollow"><code>cur.execute()</code></a> does not return the rows.</p>
