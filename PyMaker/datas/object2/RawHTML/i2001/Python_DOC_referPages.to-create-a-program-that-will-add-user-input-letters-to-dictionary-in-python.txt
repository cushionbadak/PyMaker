<p>I have to write a Python program that creates empty dictionary of alphabetic letters. The dictionary must be populated such that when a user enters any letter, it will be appended to the dictionary as a key (capital case) and value (small case). All letters of the dictionary must be unique, which means no duplicates are allowed in the dictionary.
For example:</p>

{'A': 'a','B': 'b', 'C': 'c', 'D': 'd'}
</code></pre>

<pre><code>letter_dictionary = {}
letters = input().split(' ')
for i in letters:
    if i.upper() not in letter_dictionary:
        letter_dictionary[i.upper] = i.lower

print(letter_dictionary)
</code></pre>
<p>But the problem is, that the program adds to dictionary not letters, but functions lower() and upper(), as I saw on pythontutor.com

<div class="post-text" itemprop="text">
<p>As mentioned, <code>str.upper</code> and <code>str.lower</code> are <a href="https://docs.python.org/3/tutorial/classes.html#method-objects" rel="nofollow noreferrer">method objects</a>, you need to follow them with parentheses to actually apply the methods:</p>
<pre><code>for i in letters:
    if i.upper() not in letter_dictionary:
        letter_dictionary[i.upper()] = i.lower()
</code></pre>
<p>But a regular <code>for</code> loop isn't required here. Even the <code>not in letter_dictionary</code> check isn't strictly necessary, since dictionary keys are <em>always</em> unique. If you want to apply a uniqueness check, just use <code>set</code> on your input first.</p>
<p>So you can rewrite using a dictionary comprehension:</p>
<pre><code>letter_diction = {k.upper(): k.lower() for k in set(letters)}    
</code></pre>
