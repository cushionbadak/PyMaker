<p>Well, I had a similar problem using <strong>venv</strong>, so I'll post what I did in my case.</p>
<p>I was using <code>python -m venv /app_env</code> or <code>app_env</code>/ and when I activated the enviroment, the name of the enviroment <code>(app_env)</code> wasn't showing.</p>
<p>But when I did </p>
<pre><code>python -m venv app_env
</code></pre>
<p>And activated, It worked:</p>
<pre><code>(budget_app_env) PS C:\Users\some_user\project\app_project&gt;
</code></pre>
<p>So maybe you had a problem with "/"s.</p>
<p>And by the way, <a href="https://docs.python.org/3/library/venv.html" rel="nofollow noreferrer">python doc</a> recommends using <strong>venv</strong> because <strong>pyenv</strong> was <em>deprecated</em>.</p>
