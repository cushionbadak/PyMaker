<p>100.avi</p>
<p>10000.avi</p>

<p>I need to do some process using the files in numeric order. If I use <code>sorted</code> it does not care about the values, it sorts on the first character, second character, and so on - string sorting. </p>

<div class="post-text" itemprop="text">
<p>You can use a custom <a href="https://docs.python.org/3/library/functions.html#sorted" rel="nofollow noreferrer">sorting function</a> to extract the number from each element of the list and use it as the comparison key. You'd need to convert it to <code>int</code> to ensure it compares like number:</p>
<pre><code>&gt;&gt;&gt; lst = ['100.avi', '10000.avi', '10002.avi', '1003.avi']
&gt;&gt;&gt; sorted(lst, key=lambda x: int(x.split('.', 1)[0]))
['100.avi', '1003.avi', '10000.avi', '10002.avi']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Since all the strings of your given <em>list</em> are ended in <code>.avi</code> extension, you could use a <em>slicing</em> feature.</p>
<pre><code>sorted(lst, key=lambda x: int(x[:-4]))
</code></pre>
