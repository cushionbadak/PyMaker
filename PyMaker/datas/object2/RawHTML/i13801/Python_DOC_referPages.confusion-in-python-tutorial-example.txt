<p>You can find this on <a href="https://docs.python.org/3/library/functions.html#print" rel="nofollow">the documentation</a>:</p>
<pre><code>print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)
</code></pre>
<p>If you don't specify the <code>end</code> parameter, it uses <code>'\n'</code> which is a new line. So every time you call <code>print(repr(x*x*x).rjust(4))</code>, it creates a new line. You first <code>print</code> call explicitly uses a space at then end of your string while your second call doesn't.</p>
<p>If you want to keep your output on one line, use the <code>end</code> parameter in your second call of <code>print</code></p>
<pre><code>for x in range(1, 11):
    print(repr(x).rjust(2), repr(x*x).rjust(3), end=' ')
    print(repr(x*x*x).rjust(4), end=' ')
</code></pre>
