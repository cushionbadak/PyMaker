<div class="post-text" itemprop="text">
<p>I would like to use <code>urllib.quote()</code>. But python (python3) is not finding the module.
Suppose, I have this line of code:</p>
<pre><code>print(urllib.quote("châteu", safe=''))
</code></pre>
<p>How do I import urllib.quote?</p>
<p><code>import urllib</code> or
<code>import urllib.quote</code> both give</p>
<pre><code>AttributeError: 'module' object has no attribute 'quote'
</code></pre>
<p>What confuses me is that <code>urllib.request</code> is accessible via <code>import urllib.request</code></p>
</div>
<div class="post-text" itemprop="text">
<p>In Python 3.x, you need to import <a href="https://docs.python.org/3/library/urllib.parse.html"><code>urllib.parse.quote</code></a>:</p>
<pre><code>&gt;&gt;&gt; import urllib.parse
&gt;&gt;&gt; urllib.parse.quote("châteu", safe='')
'ch%C3%A2teu'
</code></pre>
<p>According to <a href="https://docs.python.org/2/library/urllib.html">Python 2.x <code>urllib</code> module documentation</a>:</p>
<blockquote>
<p><strong>NOTE</strong></p>
<p>The <code>urllib</code> module has been split into parts and renamed in Python 3 to
  <code>urllib.request</code>, <code>urllib.parse</code>, and <code>urllib.error</code>.</p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>If you need to handle both Python 2.x and 3.x you can catch the exception and load the alternative.</p>
<pre><code>try:
    from urllib import quote  # Python 2.X
except ImportError:
    from urllib.parse import quote  # Python 3+
</code></pre>
<p>You could also use the python compatibility wrapper <a href="https://pythonhosted.org/six/" rel="noreferrer">six</a> to handle this.</p>
<pre><code>from six.moves.urllib.parse import quote
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>urllib went through some changes in Python3 and can now be imported from the parse submodule</p>
<pre><code>&gt;&gt;&gt; from urllib.parse import quote  
&gt;&gt;&gt; quote('"')                      
'%22'                               
</code></pre>
</div>
