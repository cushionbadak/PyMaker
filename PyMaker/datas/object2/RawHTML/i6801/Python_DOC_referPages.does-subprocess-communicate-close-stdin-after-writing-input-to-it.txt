<p>While it would seem logical, the <a href="https://docs.python.org/3/library/subprocess.html#subprocess.Popen.communicate" rel="nofollow noreferrer">documentation</a> does not explicitly state it:</p>
<blockquote>
<p>Interact with process: Send data to stdin. Read data from stdout and stderr, until end-of-file is reached. Wait for process to terminate. </p>
</blockquote>
</div>
<div class="post-text" itemprop="text">
<p>From looking at the source I think the answer is yes:</p>
<ul>
<li><p>when timeout is None, <code>communicate()</code> calls <code>self._stdin_write(input)</code><br/><a href="https://github.com/python/cpython/blob/master/Lib/subprocess.py#L826-L827" rel="nofollow noreferrer">https://github.com/python/cpython/blob/master/Lib/subprocess.py#L826-L827</a></p></li>
<li><p>otherwise <code>_communicate()</code> calls <code>self._stdin_write(input)</code><br/><a href="https://github.com/python/cpython/blob/master/Lib/subprocess.py#L1078-L1079" rel="nofollow noreferrer">https://github.com/python/cpython/blob/master/Lib/subprocess.py#L1078-L1079</a></p></li>
<li><p>in each case <code>_stdin_write()</code> calls <code>self.stdin.close()</code> after <code>self.stdin.write(input)</code><br/><a href="https://github.com/python/cpython/blob/master/Lib/subprocess.py#L793" rel="nofollow noreferrer">https://github.com/python/cpython/blob/master/Lib/subprocess.py#L793</a></p></li>
</ul>
</div>
