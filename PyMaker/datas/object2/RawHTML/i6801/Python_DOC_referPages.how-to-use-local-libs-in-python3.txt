<p>How do I import local libs in python3?</p>
<p><strong>setup.cfg</strong></p>
<pre><code>[install]
prefix=./
</code></pre>
<p><strong>requirements.txt</strong></p>
<pre><code># cat requirements.txt 
docopt==0.6.2
</code></pre>
<p><strong>test.py</strong></p>
<pre><code>"""
Usage:
  test.py facility-list

Options:
  -h --help     Show this screen.
"""
from docopt import docopt

if __name__ == '__main__':
  print('Hellow World')
</code></pre>
<p><strong>shell</strong></p>
<pre><code>pip3 install -r requirements.txt
python3 test.py 
Traceback (most recent call last):
  File "test.py", line 8, in &lt;module&gt;
    from docopt import docopt
ModuleNotFoundError: No module named 'docopt'
</code></pre>
<p>Why wont the import work? After the pip2 install I have a <code>./lib</code> folder in my project folder.</p>

<div class="post-text" itemprop="text">
<p>Python 3 allows you to easily use a virtual environment. In fact it's a built in feature nowadays. </p>
<p>Go to the folder with your project and type in </p>
<pre><code>python3 -m venv myvenv
</code></pre>
<p>And it'll create a folder called myvenv. The next step is to activate the virtualenv with the following command. </p>
<pre><code>source myvenv/bin/activate
</code></pre>
<p>You're now running inside the virtual environment and can do a pip install. </p>
</div>
<span class="comment-copy">Use <a href="https://virtualenv.pypa.io/en/stable/" rel="nofollow noreferrer">virtualenv</a> instead.</span>
