<p>Right now I'm trying to make a simple tic-tac-toe game, and while user chooses the sector of the board for their next move I need to check if the input is a <em>single-digit natural number</em>. I don't think just making a <code>['1','2','3'...'9']</code> list and calling an in statement for it is the most optimal thing. Could you suggest anything?</p>
</div>
<div class="post-text" itemprop="text">
<p>You can check if a string, <code>x</code>, is a single digit natural number by checking if the string contains digits and the integer equivalent of those digits is between 1 and 9, i.e.</p>
<p><code>x.isdigit() and 1 &lt;= int(x) &lt;= 9</code></p>
<p>Also, if <code>x.isdigit()</code> returns false, <code>int(x)</code> is never evaluated due to the expression using <code>and</code> (it is unnecessary as the result is already known) so you won't get an error if the string is not a digit.</p>
</div>
<div class="post-text" itemprop="text">
<p>Using <a href="https://docs.python.org/3/library/functools.html#len" rel="nofollow"><code>len</code></a> and <a href="https://docs.python.org/3/library/stdtypes.html#str.isdigit" rel="nofollow"><code>str.isdigit</code></a>:</p>
<pre><code>&gt;&gt;&gt; x = '1'
&gt;&gt;&gt; len(x) == 1 and x.isdigit() and x &gt; '0'
True
&gt;&gt;&gt; x = 'a'
&gt;&gt;&gt; len(x) == 1 and x.isdigit() and x &gt; '0'
False
&gt;&gt;&gt; x = '12'
&gt;&gt;&gt; len(x) == 1 and x.isdigit() and x &gt; '0'
False
</code></pre>
<p>Alternative: using <a href="https://docs.python.org/3/library/functools.html#len" rel="nofollow"><code>len</code></a> and chained comparisons:</p>
<pre><code>&gt;&gt;&gt; x = '1'
&gt;&gt;&gt; len(x) == 1 and '1' &lt;= x &lt;= '9'
True
&gt;&gt;&gt; x = 'a'
&gt;&gt;&gt; len(x) == 1 and '1' &lt;= x &lt;= '9'
False
&gt;&gt;&gt; x = '12'
&gt;&gt;&gt; len(x) == 1 and '1' &lt;= x &lt;= '9'
False
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>Why not just use</p>
<pre><code>&gt;&gt;&gt; natural = tuple('123456789')
&gt;&gt;&gt; '1' in natural
True
&gt;&gt;&gt; 'a' in natural
False
&gt;&gt;&gt; '12' in natural
False
</code></pre>
<p>Checking for membership in a small tuple you initialize once is <em>very</em> efficient, a <code>tuple</code> in particular over a <code>set</code> since it's optimized for small numbers of items. Using <code>len</code> and <code>isdigit</code> is overkill.</p>
</div>
