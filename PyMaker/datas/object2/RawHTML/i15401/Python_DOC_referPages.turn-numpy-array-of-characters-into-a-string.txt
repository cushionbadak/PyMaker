<p>I have a numpy array of characters and when I write it to file it writes as:</p>
<pre><code>['K' 'R' 'K' 'P' 'T' 'T' 'K' 'T' 'K' 'R' 'G' 'L']
</code></pre>
<p>I want it to write with just the letters and without the brackets or quotations i.e. as:</p>
<pre><code>KRKPTTKTKRGL 
</code></pre>
<p>I have looked at numpy documentation and from what I have gathered the solution is a chararray however it looks like this is not as functional as a normal array.</p>
<p>Any help would be great. Thanks!</p>
</div>
<div class="post-text" itemprop="text">
<p>You can use <code>tostring()</code> method of <code>numpy</code> as:</p>
<pre><code>&gt;&gt;&gt; st = np.array(['K' 'R' 'K' 'P' 'T' 'T' 'K' 'T' 'K' 'R' 'G' 'L'])
&gt;&gt;&gt; st.tostring()
'KRKPTTKTKRGL'
</code></pre>
<p>Since you have a <code>numpy</code> array, this method will be faster than <code>join()</code>.</p>
<p>For Python3x <code>tostring()</code> can be used as:</p>
<pre><code>&gt;&gt;&gt; st = np.array(['K','R','K','P','T','T','K','T','K','R','G','L'])
&gt;&gt;&gt; st.astype('|S1').tostring().decode('utf-8')
'KRKPTTKTKRGL' 
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>If you just have a numpy array then why not convert it to a string directly for writing to your file? You can do this using <a href="https://docs.python.org/3/library/stdtypes.html#str.join" rel="noreferrer"><code>str.join</code></a> which accepts an iterable (list, numpy array, etc).</p>
<pre><code>import numpy as np

arr = np.array(['K', 'R', 'K', 'P', 'T', 'T', 'K', 'T', 'K', 'R', 'G', 'L'])

s = ''.join(arr)
# KRKPTTKTKRGL
</code></pre>
</div>

<div class="post-text" itemprop="text">
<p>If you use the <a href="http://docs.scipy.org/doc/numpy/reference/generated/numpy.ndarray.tofile.html" rel="nofollow"><code>tofile()</code></a> method to save the array to file, the default separator is the empty string <code>""</code>.</p>
<p>So if your array is this,</p>
<pre><code>st = np.array(['K', 'R', 'K', 'P', 'T', 'T', 'K', 'T', 'K', 'R', 'G', 'L'])
</code></pre>
<p>then if you're using Python 2,</p>
<pre><code>&gt;&gt;&gt; st.tofile('myfile.txt')
</code></pre>
<p>creates a file with the following content:</p>
<pre><code>KRKPTTKTKRGL
</code></pre>
<p>If you're using Python 3, you may need to explicitly cast the array to the <code>S</code> string type first:</p>
<pre><code>&gt;&gt;&gt; st.astype('|S1').tofile('myfile.txt')
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>In a numpy way, you can do:</p>
<p>Using F-String <em>(Only available for Python 3.4+)</em></p>
<pre><code>s = arr.view(f'U{arr.size}')[0]
</code></pre>
<p>Using the default string:</p>
<pre><code>s = arr.view('U' + str(arr.size))[0]
</code></pre>
<p>In both, we convert the array into a usable <a href="https://www.numpy.org/devdocs/reference/generated/numpy.dtype.html" rel="nofollow noreferrer">unicode <em>(check the <code>kind</code> attribute at page bottom)</em></a> format of the size of the array. </p>
<p>Which is the <a href="https://www.numpy.org/devdocs/reference/generated/numpy.ndarray.dtype.html?highlight=numpy%20ndarray%20dtype#numpy.ndarray.dtype" rel="nofollow noreferrer"><code>dtype</code></a> of the string if you try to convert it to <a href="https://www.numpy.org/devdocs/reference/generated/numpy.array.html?highlight=numpy%20array#numpy.array" rel="nofollow noreferrer"><code>numpy.array</code></a></p>
<pre><code>In[15]: import numpy as np
In[16]: arr = np.array(['KRKPTTKTKRGL'])
In[17]: arr.dtype
Out[17]: dtype('&lt;U12')
</code></pre>
<p>Note: it works with non-English letters.</p>
</div>
