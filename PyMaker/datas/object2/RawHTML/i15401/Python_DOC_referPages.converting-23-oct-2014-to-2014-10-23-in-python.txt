<p>I have date strings in the form of <code>'23/Oct/2014'</code> and want to convert it to <code>'2014/10/23'</code> in Python 3. The final output I need is just as a string. I implementing my own function to do the conversion using dictionary (for month conversion) but I am wondering if there is more pythonic way of doing the same thing.</p>

<div class="post-text" itemprop="text">
<p>You can use the <a href="https://docs.python.org/3/library/datetime.html" rel="nofollow"><code>datetime</code> library</a> to parse the string, then format back to a string again:</p>
<pre><code>from datetime import datetime

converted = datetime.strptime(original, '%d/%b/%Y').strftime('%Y/%m/%d')
</code></pre>
<p>This parses the input string using <a href="https://docs.python.org/3/library/datetime.html#datetime.datetime.strptime" rel="nofollow"><code>datetime.datetime.strptime()</code></a>, then formats the resulting object to a string again with <a href="https://docs.python.org/3/library/datetime.html#datetime.datetime.strftime" rel="nofollow"><code>datetime.datetime.strftime()</code></a>.</p>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; from datetime import datetime
&gt;&gt;&gt; original = '23/Oct/2014'
&gt;&gt;&gt; datetime.strptime(original, '%d/%b/%Y').strftime('%Y/%m/%d')
'2014/10/23'
</code></pre>
</div>

<p>If you just want a string, your dict mapping and format would work fine, if you don't want datetime objects I don't see the point in using datetime:</p>
<pre><code>dt = '23/Oct/2014'
dates = dict(Jan="01", Feb="02", Mar="03", Apr="04", May="05", Jun="06", Jul="07", Aug="08", Sep="09", Oct="10",
             Nov="11", Dec="12")
day,mon, year = dt.split("/")
print("{}/{}/{}".format(year,dates[mon],day))
2014/10/23
</code></pre>
