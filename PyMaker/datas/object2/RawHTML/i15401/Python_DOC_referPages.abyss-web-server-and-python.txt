<p>I have got <a href="http://www.aprelium.com/abyssws" rel="nofollow">Abyss</a> web server on my <a class="post-tag" href="/questions/tagged/windows-7" rel="tag" title="show questions tagged 'windows-7'">windows-7</a> 64 bit computer. I followed the instructions regarded in <a href="http://www.aprelium.com/abyssws/python.html" rel="nofollow">this official link</a> to add <a class="post-tag" href="/questions/tagged/python" rel="tag" title="show questions tagged 'python'">python</a> support to the server, regarding that I did not downloaded the Active Python considered in the documentation, I just used a previously installed Python 3.3 on my machine.</p>
<p>I have wrote very simple python script:</p>
<pre><code>print("Hello")
</code></pre>
<p>Then I saved it as <code>test.py</code> in the server's htdocs, then I tried to access <code>http://localhost:9090/test.py</code> but the server returns <strong>Error 500</strong> Internal Server Error the log line that record this error is:</p>
<blockquote>
<p><strong>cgi.log:</strong></p>
<p>[21/Dec/2014:01:33:54 +0200]  CGI: [C:\Python33\python.exe
  test.py ] URI: /test.py   Bad CGI header line [Hello]</p>
<p><strong>access.log:</strong></p>
<p>127.0.0.1 - - [21/Dec/2014:01:33:54 +0200] "GET /test.py HTTP/1.1" 500 412 "" "Mozilla/5.0 (Windows NT 6.1; Win64; x64) AppleWebKit/537.36
  (KHTML, like Gecko) Chrome/39.0.2171.95 Safari/537.36"</p>
</blockquote>
<p>So how could I fix the CGI header line?</p>
</div>
<div class="post-text" itemprop="text">
<p>I found how to fix the CGI header, thanks to <a href="https://stackoverflow.com/users/1189040/himal">Himal</a>, the idea is printing out content type followed by a blank line, then the script works fine:</p>
<pre><code>print ("Content-Type: text/html")
print ()
print("&lt;h1&gt;Hello World!&lt;/h1&gt;")
</code></pre>
