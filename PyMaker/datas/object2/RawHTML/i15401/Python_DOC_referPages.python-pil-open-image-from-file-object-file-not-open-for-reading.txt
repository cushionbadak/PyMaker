<div class="post-text" itemprop="text">
<p>You opened the file for writing, not reading. You'd have to use a dual mode, and first rewind the file pointer:</p>
<pre><code>with open('myimage.png', 'w+b') as imgfile:
    imgfile.write(decodestring(base64_image))
    imgfile.seek(0)
    f = Image.open(imgfile)
</code></pre>
<p>Here <code>w+</code> means writing and reading, see the <a href="https://docs.python.org/3/library/functions.html#open" rel="nofollow"><code>open()</code> documentation</a>:</p>
<blockquote>
<p><code>'+'</code><br/>
  open a disk file for updating (reading and writing)</p>
<p>For binary read-write access, the mode <code>'w+b'</code> opens and truncates the file to 0 bytes. <code>'r+b'</code> opens the file without truncation.</p>
</blockquote>
<p>However, there isn't really any need to use a file on disk here; use an in-memory file instead:</p>
<pre><code>from io import BytesIO

imgfile = BytesIO(decodestring(base64_image))
f = Image.open(imgfile)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>The problem here is that you are opening the file for <em>writing</em> with <code>wb</code>, use <code>w+b</code> to open for reading as well.</p>
<p>Detailed explanation <a href="https://docs.python.org/2/library/functions.html#open" rel="nofollow">in the docs here</a>.</p>
</div>
