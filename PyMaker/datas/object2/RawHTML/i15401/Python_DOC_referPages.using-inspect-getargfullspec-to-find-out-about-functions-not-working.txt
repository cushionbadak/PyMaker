<div class="post-text" itemprop="text">
<p>The only functions that <code>inspect.getfullargspec</code> supports are those created in Python code with <code>def</code> or <code>lambda</code>.  From the <a href="https://docs.python.org/3/library/inspect.html#inspect.getfullargspec" rel="nofollow">docs</a>:</p>
<blockquote>
<p><code>inspect.getfullargspec(func)</code></p>
<p>Get the names and default values of a <strong>Python</strong> function's arguments.</p>
</blockquote>
<p>Emphasis mine.  <code>print</code> however is not technically a <em>Python</em> function because it is written in C:</p>
<pre><code>&gt;&gt;&gt; import types
&gt;&gt;&gt; # FunctionType represents Python functions
&gt;&gt;&gt; isinstance(print, types.FunctionType)
False
&gt;&gt;&gt; # BuiltinFunctionType represents C functions
&gt;&gt;&gt; isinstance(print, types.BuiltinFunctionType)
True
&gt;&gt;&gt;
</code></pre>
<p>That is why a <code>TypeError</code> is raised for passing it to <code>inspect.getfullargspec</code>.</p>
<hr/>
<p>Also, using <code>inspect.getfullargspec</code> to get information about a function is only really used in metaprogramming.  If you are simply looking for documentation or usage, you should use <a href="https://docs.python.org/3/library/functions.html#help" rel="nofollow"><code>help</code></a> in the interactive interpreter as @thomas said or read the <a href="https://docs.python.org/3/" rel="nofollow">documentation</a> online.</p>
</div>
<div class="post-text" itemprop="text">
<p>The best way to find about a function, is to read its docstring and see how it was implemented. This can be done by running the help() function</p>
<p>I.E - <code>help(int)</code> shows you what can be done with the <code>int()</code> function</p>
<pre><code>class int(object)
 |  int(x=0) -&gt; int or long
 |  int(x, base=10) -&gt; int or long
 |  
 |  Convert a number or string to an integer, or return 0 if no arguments
 |  are given.  If x is floating point, the conversion truncates towards zero.
 |  If x is outside the integer range, the function returns a long instead.
 |  
 |  If x is not a number or if base is given, then x must be a string or
 |  Unicode object representing an integer literal in the given base.  The
 |  literal can be preceded by '+' or '-' and be surrounded by whitespace.
 |  The base defaults to 10.  Valid bases are 0 and 2-36.  Base 0 means to
 |  interpret the base from the string as an integer literal.
 |  &gt;&gt;&gt; int('0b100', base=0)
 |  4
</code></pre>
<p>Or for example -</p>
<pre><code>import datetime
help(datetime.datetime.strptime)
</code></pre>
<p>will return</p>
<pre><code>strptime(...)
    string, format -&gt; new datetime parsed from a string (like time.strptime()).
</code></pre>
<p>You can read more about the <code>help()</code> function <a href="https://docs.python.org/2/library/functions.html#help" rel="nofollow">here</a></p>
