<p>I am attempting to read a JSON file containing data which I want to keep in order as it is a mathematical equation which can't change. I have tried using OrderedDict() however this doesn't work (or I can't get it to work).</p>
<p>Here is the code I am using:</p>
<pre><code>import json
from collections import OrderedDict

json_data = open('example3.json')
data = OrderedDict(json.load(json_data))

print (json.dumps(data))
</code></pre>
<p>Could anyone shine some light as to why this is not working?</p>

Craig</p>

<div class="post-text" itemprop="text">
<p><code>json.load</code> will load the json data into a dictionary. You then convert this dictionary to an <code>OrderedDict</code>. Unfortunately, because it was a dictionary <em>first</em> the order could have already changed before it is "set" in the <code>OrderedDict</code>.</p>
<p>To load the json directly into an <code>OrderedDict</code> you can use the keyword argument <code>object_pairs_hook</code> with <code>OrderedDict</code>. See <a href="https://docs.python.org/3/library/json.html#json.load" rel="nofollow">the documentation</a> for more details.</p>
<pre><code>import json
from collections import OrderedDict

json_data = open('example3.json')
data = json.load(json_data, object_pairs_hook=OrderedDict)
</code></pre>
</div>
<span class="comment-copy">Just note that the json spec explicitely mentions that json objects (which translate to <code>dict</code>s in Python) are "an <i>unordered</i> collection of key:value pairs" - IOW, you should <b>not</b> rely on keys order in json.</span>
