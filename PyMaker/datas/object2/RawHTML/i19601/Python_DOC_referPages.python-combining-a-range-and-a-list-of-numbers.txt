<p>The <code>range()</code> function returns a special <a href="http://docs.python.org/3/library/stdtypes.html#ranges" rel="nofollow">range object</a> to save on memory (no need to keep all the numbers in memory when only the start, end and step size will do). Cast it to a list to 'expand' it:</p>
<pre><code>list(yourrange) + otherlist
</code></pre>
<p>To quote the documentation:</p>
<blockquote>
<p>The advantage of the <a href="http://docs.python.org/3/library/stdtypes.html#range" rel="nofollow"><code>range</code></a> type over a regular <a href="http://docs.python.org/3/library/stdtypes.html#list" rel="nofollow"><code>list</code></a> or <a href="http://docs.python.org/3/library/stdtypes.html#tuple" rel="nofollow"><code>tuple</code></a> is that a <a href="http://docs.python.org/3/library/stdtypes.html#range" rel="nofollow"><code>range</code></a> object will always take the same (small) amount of memory, no matter the size of the range it represents (as it only stores the <code>start</code>, <code>stop</code> and <code>step</code> values, calculating individual items and subranges as needed).</p>
</blockquote>
