<div class="post-text" itemprop="text">
<p>You can get the type of any object in Python using the <a href="https://docs.python.org/3/library/functions.html#type" rel="nofollow"><code>type</code> built-in</a>:</p>
<pre><code>print type(a)
</code></pre>
<p>You may also be interested in the <a href="https://docs.python.org/3/library/stdtypes.html#class.__name__" rel="nofollow"><code>__name__</code></a> and <a href="https://docs.python.org/3/library/stdtypes.html#instance.__class__" rel="nofollow"><code>__class__</code></a> attributes, which will hold data corresponding to the widget's name and class:</p>
<pre><code>type(a).__name__ # Returns class name as a string
a.__class__      # Same as type(a)
</code></pre>
<p>Note however that if you need to perform a <em>typecheck</em>, you should use <a href="https://docs.python.org/3/library/functions.html#isinstance" rel="nofollow"><code>isinstance</code></a>:</p>
<pre><code>isinstance(a, type_to_test_for)
</code></pre>
<p>This will take into account inheritance.</p>
</div>

<span class="comment-copy">Did you actually read any of the documentation you linked to? The <code>__name__</code> attribute is specific to classes, not instances, so <code>a.__name__</code> will raise an <code>AttributeError</code>. Also, <code>a.__class__</code> returns the same thing as <code>type(a)</code>, not a string. So to get the class name, you want <code>a.__class__.__name__</code>, or <code>type(a).__name__</code>.</span>
