<div class="post-text" itemprop="text">
<p>As Adam mentioned, you'll want to change your parser's "action", as stated <a href="https://docs.python.org/3/library/argparse.html?highlight=argparse#action" rel="nofollow">in the docs</a>.</p>
<p>But it sounds like you want to indicate whether that special config is active, or not. In which case, using <code>action='store_true'</code> or <code>action='store_false'</code> would be correct. You just won't pass the myconfig argument.</p>
<pre><code>parser = argparse.ArgumentParser(description='So something')
parser.add_argument(
    '-c',
    '--c',
    dest='enable_config',
    action='store_true',
    required=False,
    help='Enable special config settings',
)
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>"-c" will store <code>True</code> or <code>False</code>, so why are you passing a value (myconfig) to it?  You would call it as:</p>
<pre><code>python mypython.py -c
</code></pre>
<p>If you want -c to take an argument, then it shouldn't have the action store_true.</p>
