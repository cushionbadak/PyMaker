<p>In python <code>dir</code> displays all the names defined in namespace. I checked and it worked fine. 
But when I used <code>dir(function_name)</code> it didn't display the variables I created inside the function. Why?</p>
<p>Here's a code sample:</p>
<pre><code>&gt;&gt;&gt; def printHello(greetings):
...     inside_func=2
... 
&gt;&gt;&gt; dir()
['__builtins__', '__doc__', '__name__', '__package__', 'printHello',]
&gt;&gt;&gt; dir(printHello)
['__call__', '__class__', '__closure__', '__code__', '__defaults__', '__delattr__', '__dict__', '__doc__', '__format__', '__get__', '__getattribute__', '__globals__', '__hash__', '__init__', '__module__', '__name__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__setattr__', '__sizeof__', '__str__', '__subclasshook__', 'func_closure', 'func_code', 'func_defaults', 'func_dict', 'func_doc', 'func_globals', 'func_name']
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>As to why <code>dir(youfunc)</code> doesn't list local vars, it's quite simply because as documented <code>dir(obj)</code> lists <code>obj</code>'s <em>attributes</em> , and local vars are not attributes of the function object. </p>
<p>Outside the function itself, you can get a listing of a function's local vars names with <code>yourfunc.func_code.co_varnames</code> (note that arguments are local variables):</p>
<pre><code>def foo(bar):
    baaz = 42
print(foo.func_code.co_varnames)
</code></pre>
<p>but you won't obviously be able to access the local vars values from outside the function since those values only exist when the function is being executed.</p>
<p>From within the function you can get the local vars using <code>locals()</code>:</p>
<pre><code>def foo(bar):
    baaz = 42
    print(locals())
</code></pre>
</div>
