<p>I am not looking for a constant recording the maximum value representable with an <code>int</code>; I am looking to find out the maximum amount of memory that an <code>int</code> can occupy. Consider the following code:</p>
<pre><code>import sys

def print_and_execute(*args):
    for s in args:
        print(s, end = " == ")
        r = eval(s)
        print(r)
    return

print_and_execute("type(5) ")     
print_and_execute("sys.getsizeof(5)")    
i = int()    
print_and_execute("i")    
print_and_execute("sys.getsizeof(i)")    
i = 2937    
print_and_execute("i")    
print_and_execute("sys.getsizeof(i)")
print_and_execute("sys.maxsize")
print_and_execute("sys.getsizeof(sys.maxsize)")
</code></pre>
<p>For my particular machine, the output is:</p>
<pre><code>type(5)  == &lt;class 'int'&gt;
sys.getsizeof(5) == 28
i == 0
sys.getsizeof(i) == 24
i == 2937
sys.getsizeof(i) == 28

<p>Apparently, some <code>int</code>s are 24 bytes long, others are 28, still others use 36. How many bytes are required to hold an <code>int</code> of any size?</p>

<div class="post-text" itemprop="text">
<blockquote>
<p>How many bytes are required to hold any size <code>int</code>?</p>
</blockquote>
<p>As many as are required. There's no limit on the size of <code>int</code>s, they are of arbitrary size as <a href="https://docs.python.org/3/library/stdtypes.html#typesnumeric" rel="nofollow noreferrer">their documentation</a> states.<sup>*</sup></p>
<p><a href="https://docs.python.org/3/library/sys.html#sys.maxsize" rel="nofollow noreferrer"><code>sys.maxsize</code></a> does not represent the maximum size of <code>int</code> objects. It is the maximum value of a <code>Py_ssize_t</code> which is used (in CPython) for holding values of indexes. </p>
<p>For example, just create a larger <code>int</code> than <code>maxsize</code> :</p>
<pre><code>&gt;&gt;&gt; sys.getsizeof(sys.maxsize * sys.maxsize)
44
</code></pre>
<p>So, you get the number of bytes for a given <code>int</code> by using <code>getsizeof</code> on it. </p>
<p><sup>*Bound, <em>eventually</em>, by available memory.</sup></p>
</div>
<span class="comment-copy">Python integers can expanded to be as large as necessary. In other words, the upper limit is determined by how much memory is available.</span>
