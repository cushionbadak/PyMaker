<div class="post-text" itemprop="text">
<p>According to the <a href="https://greentreesnakes.readthedocs.io/en/latest/nodes.html#Assign" rel="nofollow noreferrer">GreenTreeSnakes documentation on Assignment statements</a>:</p>
<blockquote>
<p>An assignment. <code>targets</code> is a list of nodes, and <code>value</code> is a single node.</p>
<p>Multiple nodes in <code>targets</code> represents assigning the same value to each.
  Unpacking is represented by putting a <code>Tuple</code> or <code>List</code> within <code>targets</code>.</p>
</blockquote>
<p>My question is, when does the unpacking put the targets in a List instead of a Tuple? The example given unpacks into a Tuple.</p>
</div>
<div class="post-text" itemprop="text">
<p>In assignments, targets can both be lists and tuples:</p>
<pre><code>a, b, c = value  # assign to a tuple of names
[a, b, c] = value  # assign to a list of names
</code></pre>
<p>The difference is cosmetic to Python; see the <a href="https://docs.python.org/3/reference/simple_stmts.html#assignment-statements" rel="nofollow noreferrer"><em>Assignment statement</em> reference documentation</a>.</p>
<p>Demo:</p>
<pre><code>&gt;&gt;&gt; parseprint('[a, b, c] = value')
Module(body=[
    Assign(targets=[
        List(elts=[
            Name(id='a', ctx=Store()),
            Name(id='b', ctx=Store()),
            Name(id='c', ctx=Store()),
          ], ctx=Store()),
      ], value=Name(id='value', ctx=Load())),
  ])
</code></pre>
</div>
