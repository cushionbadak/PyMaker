<p>Using Python3.6.2, I was trying to find the longest word in a sentence.
The code I came up with is, </p>
<pre><code> &gt;&gt;&gt; sen = "Argument goes here"

 &gt;&gt;&gt; max(sen.split())

 'here'
</code></pre>
<p>I would like to why is the answer '<strong>here'</strong> instead of <strong>'Argument'</strong>, rather than the answer to the problem.</p>

<div class="post-text" itemprop="text">
<p>As <strong>SungJin Steve Yoo</strong> pointed out, <a href="https://docs.python.org/3/library/functions.html#max" rel="nofollow noreferrer"><code>max</code></a> function by default compares <code>string</code> in <code>Lexological Order</code>, and here, the string length does not come into play.</p>
<p>So, when you do </p>
<pre><code>max( ['Argument', 'goes', 'here'] )
</code></pre>
<p>Here, the starting letter <code>h</code> has higher order compared to both <code>A</code> and <code>g</code>.</p>
<p>In case you wanted to do it by <code>length</code> and <code>lexological order</code>, use the <code>key</code> function :</p>
<pre><code>max(sen.split(), key= lambda x:(len(x), x) )
</code></pre>
<p><em>Below are a few cases for you to get the hang of it :</em></p>
<pre><code>&gt;&gt;&gt; sen = sen = 'hex men or -'

&gt;&gt;&gt; max(sen.split())
=&gt; 'or'
&gt;&gt;&gt; max(sen.split(), key=len)
=&gt; 'hex'
&gt;&gt;&gt; max(sen.split(), key= lambda x: (len(x), x))
=&gt; 'men'
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>When you do not mention the <code>key</code> function of <code>max</code>, it will just take default ordering of the itearbles; in this case, the lexical order. You need to mention that you need to order by <code>len</code>. You need to do this:</p>
<pre><code>&gt;&gt;&gt; sen = "Argument goes here"
&gt;&gt;&gt; max(sen.split(), key=len)
'Argument'
</code></pre>
</div>
