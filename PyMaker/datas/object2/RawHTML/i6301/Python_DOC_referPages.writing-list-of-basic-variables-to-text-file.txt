<p>I want to write a text file that has some lines in the following format:</p>
<pre><code>result: variable1 +/- error1
result: variable2 +/- error2
</code></pre>
<p>And so on... So far I have:</p>
<pre><code>f = open('file_{a}.txt'.format(a=some_name), 'w')
for i in range(len(variable)):
    f.write('result: ', variable[i], '+/-', error[i], '\n')
</code></pre>
<p>Variable and error are floats, and some_name is a string.</p>
<p>But I'm getting an error:</p>
<blockquote>
<p>TypeError: expected a string or other character buffer object</p>
</blockquote>
<p>I guess I need to format the <code>f.write</code> line differently but I can't figure out how. The file only needs to be read by humans, so that actual format can change.</p>
<p>Thanks!</p>

<div class="post-text" itemprop="text">
<h3>From the <a href="https://docs.python.org/3/tutorial/inputoutput.html#methods-of-file-objects" rel="nofollow noreferrer"><code>documentation</code></a>:</h3>
<blockquote>
<p><strong><code>f.write(string)</code></strong></p>
<p>writes the contents of string to the file, returning the number of characters written.</p>
</blockquote>
<p>So you can't pass in any number of <code>strings</code> separated by <code>commas</code> as <code>arguments</code>. This is different to the way <code>print()</code> works which excepts any number of arguments and <code>formats</code> them for you...</p>
<p>So that is why you are getting the <code>error</code>:</p>
<blockquote>
<p>TypeError: expected a string or other character buffer object</p>
</blockquote>
<p><strong>How to fix it:</strong></p>
<p>Fixing it is really easy, if you are sure <code>variable[i]</code> and <code>error[i]</code> are <code>strings</code>, you can either:</p>
<p><code>format</code> them with <code>.format</code>:</p>
<pre><code>f.write('result: {} +/- {}\n'.format(variable[i], error[i]))
</code></pre>
<p>or concatenate them with the <code>+</code> operand:</p>
<pre><code>f.write('result: ' + variable[i] + ' +/- ' + error[i] + '\n')
</code></pre>
<p>Hope this helps!</p>
</div>
