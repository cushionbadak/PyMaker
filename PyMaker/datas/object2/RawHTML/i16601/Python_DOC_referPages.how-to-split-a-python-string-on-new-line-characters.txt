<h3>‚ú® Splitting line in Python:</h3>
<p>Have you tried using <code>str.splitlines()</code> method?:</p>
<ul>
<li><a href="https://docs.python.org/2/library/stdtypes.html#str.splitlines" rel="noreferrer">Python <code>2.X</code> documentation here.</a></li>
<li><a href="https://docs.python.org/3/library/stdtypes.html#str.splitlines" rel="noreferrer">Python <code>3.X</code> documentation here.</a></li>
</ul>
<p>From the docs:</p>
<blockquote>
<p><a href="https://docs.python.org/3/library/stdtypes.html#str.splitlines" rel="noreferrer"><code>str.splitlines([keepends])</code></a></p>
<p>Return a list of the lines in the string, breaking at line boundaries.
  Line breaks are not included in the resulting list unless <code>keepends</code> is
  given and true.</p>
</blockquote>
<p>For example:</p>
<pre><code>&gt;&gt;&gt; 'Line 1\n\nLine 3\rLine 4\r\n'.splitlines()
['Line 1', '', 'Line 3', 'Line 4']

&gt;&gt;&gt; 'Line 1\n\nLine 3\rLine 4\r\n'.splitlines(True)
['Line 1\n', '\n', 'Line 3\r', 'Line 4\r\n']
</code></pre>
<h3>ü§î Which delimiters are considered?</h3>
<blockquote>
<p>This method uses the universal newlines approach to splitting lines.</p>
</blockquote>
<p>The main difference between Python <code>2.X</code> and Python <code>3.X</code> is that the former uses the universal newlines approach to splitting lines, so <code>"\r"</code>, <code>"\n"</code>, and <code>"\r\n"</code> are considered line boundaries for 8-bit strings, while the latter uses a superset of it that also includes:</p>
<ul>
<li><code>\v</code> or <code>\x0b</code>: Line Tabulation (added in Python <code>3.2</code>).</li>
<li><code>\f</code> or <code>\x0c</code>: Form Feed (added in Python <code>3.2</code>).</li>
<li><code>\x1c</code>: File Separator.</li>
<li><code>\x1d</code>: Group Separator.</li>
<li><code>\x1e</code>: Record Separator.</li>
<li><code>\x85</code>: Next Line (C1 Control Code).</li>
<li><code>\u2028</code>: Line Separator.</li>
<li><code>\u2029</code>: Paragraph Separator.</li>
</ul>
<h3>ü•ä splitlines VS split:</h3>
<blockquote>
<p>Unlike <a href="https://docs.python.org/3/library/stdtypes.html#str.split" rel="noreferrer"><code>str.split()</code></a> when a delimiter string <em>sep</em> is given, this method
  returns an empty list for the empty string, and a terminal line break
  does not result in an extra line:</p>
</blockquote>
<pre><code>&gt;&gt;&gt; ''.splitlines()
[]

&gt;&gt;&gt; 'Line 1\n'.splitlines()
['Line 1']
</code></pre>
<p>While <code>str.split('\n')</code> returns:</p>
<pre><code>&gt;&gt;&gt; ''.split('\n')
['']

&gt;&gt;&gt; 'Line 1\n'.split('\n')
['Line 1', '']
</code></pre>
<h3>‚úÇÔ∏è Removing additional whitespace:</h3>
<p>If you also need to remove additional leading or trailing whitespace, like spaces, that are ignored by <code>str.splitlines()</code>, you could use <code>str.splitlines()</code> together with <a href="https://docs.python.org/3/library/stdtypes.html#str.strip" rel="noreferrer"><code>str.strip()</code></a>:</p>
<pre><code>&gt;&gt;&gt; [str.strip() for str in 'Line 1  \n  \nLine 3 \rLine 4 \r\n'.splitlines()]
['Line 1', '', 'Line 3', 'Line 4']
</code></pre>
<h3>üóëÔ∏è Removing empty strings (''):</h3>
<p>Lastly, if you want to filter out the empty strings from the resulting list, you could use <a href="https://docs.python.org/3/library/functions.html#filter" rel="noreferrer"><code>filter()</code></a>:</p>
<pre><code>&gt;&gt;&gt; # Python 2.X:
&gt;&gt;&gt; filter(bool, 'Line 1\n\nLine 3\rLine 4\r\n'.splitlines())
['Line 1', 'Line 3', 'Line 4']

&gt;&gt;&gt; # Python 3.X:
&gt;&gt;&gt; list(filter(bool, 'Line 1\n\nLine 3\rLine 4\r\n'.splitlines()))
['Line 1', 'Line 3', 'Line 4']
</code></pre>
