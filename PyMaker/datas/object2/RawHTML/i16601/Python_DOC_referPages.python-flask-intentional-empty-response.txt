<p>Is there a way to return a response (from <code>make_response()</code> object or similar) with certain properties so that it doesn't render the page again and doesn't do anything else either. I am trying to run a code on the server without generating any output </p>
<p>A simple 'return None' produces:</p>
<pre><code>ValueError: View function did not return a response
</code></pre>
<p>This should be possible because the following only downloads a file and doesn't render the template: </p>
<pre><code>myString = "First line of a document"
response = make_response(myString)
response.headers["Content-Disposition"] = "attachment; filename=myFile.txt"
return response
</code></pre>
</div>
<div class="post-text" itemprop="text">
<p>You are responding to a request, your HTTP server <em>must</em> return <em>something</em>. The HTTP 'empty response' response is <code>204 No Content</code>:</p>
<pre><code>return ('', 204)
</code></pre>
<p>Note that returning a file to the browser is <em>not</em> an empty response, just different from a HTML response.</p>
